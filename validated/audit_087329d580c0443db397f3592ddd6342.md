After reading all the relevant factory and provider contracts, I can identify a direct analog to the external bug.

---

### Title
Stale Updater Grants Persist After `transferProviderOwnership`, Allowing Former Updaters to Corrupt `confidenceParam` and Reach Pool Swaps with Bad Bid/Ask Prices — (`PriceProviderFactory.sol`, `PriceProviderFactoryL2.sol`, `AnchoredProviderFactory.sol`)

### Summary

All three factory contracts implement `transferProviderOwnership()` but none of them clear the `isUpdater[provider][updater]` mapping for the previous owner's granted updaters. After ownership transfer, stale updaters retain the ability to call `setConfidence()` and manipulate `confidenceParam`, which directly controls the bid/ask spread fed to pool swaps.

### Finding Description

`transferProviderOwnership()` in `PriceProviderFactory`, `PriceProviderFactoryL2`, and `AnchoredProviderFactory` updates `providerOwner[provider]` and moves the provider between `_providersByCreator` sets, but leaves `isUpdater[provider][*]` untouched:

```solidity
// PriceProviderFactory.sol lines 92-102
function transferProviderOwnership(address provider, address newOwner) external override onlyProviderOwner(provider) {
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);
    // ← isUpdater[provider][*] is NEVER cleared
    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
``` [1](#0-0) 

The access gate for `setConfidence()` is `_requireUpdater()`:

```solidity
function _requireUpdater(address provider) internal view {
    if (msg.sender != providerOwner[provider] && !isUpdater[provider][msg.sender])
        revert NotProviderUpdater();
}
``` [2](#0-1) 

Because `isUpdater[provider][staleUpdater]` remains `true` after the transfer, any address the previous owner granted can still call `setConfidence()` and invoke `setConfidenceParam()` on the provider.

`confidenceParam` is the direct multiplier on the oracle spread inside `_getBidAndAskPrice()`:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [3](#0-2) 

Setting `confidenceParam = 0` collapses `adjustedSpread` to zero, making `bid = mid` and `ask = mid`. After the `_applyBidAdjustments` / `_applyAskAdjustments` step with `marginStep = 0`, `bidOut` and `askOut` are computed from the same value with Floor vs Ceil rounding. When `mid` is divisible by `ORACLE_DECIMALS` (1e8), `bidOut == askOut`, triggering the hard invariant check:

```solidity
if (bidOut >= askOut) return (0, type(uint128).max);
``` [4](#0-3) 

This causes `getBidAndAskPrice()` to revert with `FeedStalled()`, blocking all swaps on every pool that uses this provider. When `mid` is not divisible by 1e8, the spread collapses to 1 unit (effectively zero), allowing traders to execute at near-mid price and extract LP value.

The identical pattern exists in `PriceProviderFactoryL2.transferProviderOwnership()` and `AnchoredProviderFactory.transferProviderOwnership()`: [5](#0-4) [6](#0-5) 

For `AnchoredPriceProvider`, the impact is partially mitigated by the reference band clamp in `_computeBidAsk()` (the final quote cannot be tighter than `refBid`/`refAsk`), but the `confidenceParam = 0` path still reaches `_shapedQuote()` and can produce `sBid == sAsk`, returning the `(false, 0, 0)` sentinel and halting the pool. [7](#0-6) 

### Impact Explanation

All pools using the compromised provider are either DoS'd (FeedStalled on every swap) or forced to execute at near-zero spread, extracting value from LPs. The new owner cannot immediately restore the parameter because `setConfidenceParam` enforces a 1-minute `CONFIDENCE_COOLDOWN`. During that window, the stale updater can re-set `confidenceParam = 0` immediately after the new owner restores it, creating a sustained griefing loop until the new owner calls `revokeUpdater()`. [8](#0-7) 

### Likelihood Explanation

Granting updater access before transferring ownership is a normal operational pattern (e.g., a team member is an updater, then the provider is sold or transferred to a DAO). The previous owner need not be malicious at grant time — they only need to exploit the stale grant after the transfer. No privileged role beyond "former provider owner who previously granted an updater" is required.

### Recommendation

Clear all updater grants on ownership transfer. Since `isUpdater` is a flat mapping with no enumerable set of granted addresses, the factory should maintain an `EnumerableSet.AddressSet` of updaters per provider, and iterate to clear them in `transferProviderOwnership()`:

```diff
+mapping(address provider => EnumerableSet.AddressSet) private _updaters;

 function transferProviderOwnership(address provider, address newOwner) external override onlyProviderOwner(provider) {
     require(_providers.contains(provider), ProviderNotTracked());
     require(newOwner != address(0));
     address previousOwner = providerOwner[provider];

     providerOwner[provider] = newOwner;
     _providersByCreator[previousOwner].remove(provider);
     _providersByCreator[newOwner].add(provider);

+    // Clear all stale updater grants from the previous owner
+    address[] memory updaters = _updaters[provider].values();
+    for (uint256 i; i < updaters.length; ++i) {
+        isUpdater[provider][updaters[i]] = false;
+        _updaters[provider].remove(updaters[i]);
+    }

     emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
 }
```

Alternatively, add a per-provider `updaterGeneration` counter that is incremented on transfer, and include the generation in the `isUpdater` key so all prior grants are automatically invalidated.

### Proof of Concept

```
1. Alice calls PriceProviderFactory.createPriceProvider(...) → provider P
2. Alice calls grantUpdater(P, mallory)
   → isUpdater[P][mallory] = true
3. Alice calls transferProviderOwnership(P, carol)
   → providerOwner[P] = carol
   → isUpdater[P][mallory] still = true  ← stale grant
4. Mallory calls setConfidence([P], [0])
   → _requireUpdater passes (isUpdater[P][mallory] == true)
   → PriceProvider(P).setConfidenceParam(0) succeeds
5. Any pool using P calls getBidAndAskPrice():
   → adjustedSpread = spread * 0 = 0
   → bid = mid, ask = mid
   → bidOut >= askOut → returns (0, type(uint128).max)
   → getBidAndAskPrice() reverts FeedStalled()
   → all swaps on the pool revert
6. Carol calls revokeUpdater(P, mallory) to stop further changes
7. Carol must wait CONFIDENCE_COOLDOWN (1 min) before restoring confidenceParam
   → pool is unusable for at least 1 minute; Mallory can repeat step 4 if revoke races
```
