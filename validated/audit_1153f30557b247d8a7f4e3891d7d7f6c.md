### Title
Single `MAX_REF_STALENESS` Applied to Both Legs of Synthetic Ratio Feed Allows Stale Price to Reach Pool Swaps — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

### Summary

`AnchoredPriceProvider` supports a two-feed synthetic ratio mode (`quoteFeedId != 0`) where the final mid-price is `price(baseFeedId) / price(quoteFeedId)`. Both legs are validated for freshness using the **same** immutable `MAX_REF_STALENESS`. Because the two feeds can have fundamentally different update heartbeats, a single staleness threshold cannot be simultaneously correct for both: setting it to the slower feed's heartbeat allows the faster feed to be consumed up to that many seconds stale; setting it to the faster feed's heartbeat causes near-constant reverts on the slower feed. The factory validates `maxRefStaleness` only against the `baseFeedId` class envelope, with no per-leg staleness parameter for `quoteFeedId`. A stale denominator or numerator price in the ratio reaches the pool swap as the bid/ask quote.

---

### Finding Description

In `AnchoredPriceProvider._getBidAndAskPrice()`, when `quoteFeedId` is non-zero, both legs are read through `_readLeg()`: [1](#0-0) 

`_readLeg()` applies `_isStale(..., MAX_REF_STALENESS)` identically to whichever `feedId` it receives: [2](#0-1) 

`MAX_REF_STALENESS` is a single immutable set once at construction: [3](#0-2) 

In `AnchoredProviderFactory.createAnchoredProvider()`, the envelope lookup and staleness validation are keyed **only on `baseFeedId`**. The `quoteFeedId` has no separate envelope class, no separate staleness parameter, and no validation that its heartbeat is compatible with the chosen `maxRefStaleness`: [4](#0-3) 

The factory comment on `quoteFeedId` explicitly acknowledges this gap — "The envelope is keyed on `baseFeedId` (the provider's class); the ref feed only contributes its uncertainty and is validated for existence at provider construction" — but provides no staleness envelope for the quote leg: [5](#0-4) 

---

### Impact Explanation

Consider a synthetic BTC/ETH provider backed by `baseFeedId = BTC/USD` (1-hour heartbeat) and `quoteFeedId = ETH/USD` (1-hour heartbeat), or more critically `baseFeedId = ETH/USD` (1-hour) and `quoteFeedId = USDC/USD` (24-hour heartbeat). The creator must choose a single `maxRefStaleness`:

- **Set to 24 h** (to keep the USDC/USD leg alive): the ETH/USD leg passes staleness even when 23 h 59 m old. A 24-hour-stale ETH price in the denominator of ETH/USDC produces a ratio that is wildly wrong relative to the current market. This stale ratio is returned as the bid/ask to the pool swap.
- **Set to 1 h** (to keep the ETH/USD leg tight): the USDC/USD leg reverts almost every block → near-constant `FeedStalled` DoS.

The corrupted value is the synthetic `mid` computed at line 267: [6](#0-5) 

This stale `mid` flows directly into `_computeBidAsk`, producing a stale bid/ask that the pool accepts as the execution price. A trader can swap at the stale price, extracting value from LPs or the pool.

---

### Likelihood Explanation

- `createAnchoredProvider` is permissionless; any address can deploy a synthetic ratio provider.
- The factory enforces no per-leg staleness bound; the structural mismatch is invisible to the envelope system.
- Feeds with materially different heartbeats (e.g., volatile asset vs. stablecoin, or cross-chain feeds with different update cadences) are a normal deployment scenario explicitly described in the NatDoc.
- No on-chain guard prevents the stale ratio from reaching the pool once `MAX_REF_STALENESS` is set to the slower feed's heartbeat.

---

### Recommendation

Add a separate `maxQuoteStaleness` immutable to `AnchoredPriceProvider` and a corresponding `maxQuoteRefStaleness` parameter to `createAnchoredProvider`. Pass `maxQuoteStaleness` to `_readLeg` when reading the quote leg:

```solidity
// In _getBidAndAskPrice():
(uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLegWithStaleness(_quote, MAX_QUOTE_STALENESS);
```

The factory should validate `maxQuoteRefStaleness` against the envelope class of `quoteFeedId` (not `baseFeedId`), or at minimum against the `DEFAULT_CLASS` envelope, so the audit-once guarantee extends to both legs.

---

### Proof of Concept

1. Admin configures an envelope for `DEFAULT_CLASS` with `stalenessMax = 86400` (24 h).
2. Attacker (or well-intentioned creator) calls `AnchoredProviderFactory.createAnchoredProvider()`:
   - `baseFeedId` = ETH/USD (1-hour heartbeat, class → DEFAULT_CLASS)
   - `quoteFeedId` = USDC/USD (24-hour heartbeat)
   - `maxRefStaleness` = 86400 (passes envelope check on `baseFeedId` class)
3. The ETH/USD oracle is not updated for 23 hours (e.g., due to a network hiccup or intentional griefing of the push oracle).
4. A swap is initiated. `_readLeg(baseFeedId)` returns the 23-hour-old ETH/USD price; `_isStale(refTime, block.timestamp, 86400)` returns `false` — staleness check passes.
5. `_readLeg(quoteFeedId)` returns the fresh USDC/USD price.
6. `mid = Math.mulDiv(stale_ETH_price, ORACLE_DECIMALS, fresh_USDC_price)` — the ratio is computed with a 23-hour-stale ETH price.
7. `_computeBidAsk` produces bid/ask from this stale ratio; `getBidAndAskPrice()` returns it without reverting.
8. The pool executes the swap at the stale price. If ETH dropped 10% in those 23 hours, the pool sells ETH at a 10% premium to the attacker, draining LP value. [1](#0-0) [7](#0-6) [4](#0-3)

```text
    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }
```

```text
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
```
