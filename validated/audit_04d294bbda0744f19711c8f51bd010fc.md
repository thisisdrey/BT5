### Title
No Per-Block Price Caching Allows Same-Block Arbitrage via Oracle Price Updates — (`smart-contracts-poc/contracts/PriceProvider.sol`, `ProtectedPriceProvider.sol`, `AnchoredPriceProvider.sol`)

### Summary

Every price provider in the Metric OMM oracle stack fetches a live oracle price on every `getBidAndAskPrice()` call with no per-block caching. Because the underlying Pyth oracle allows permissionless price-feed updates between transactions in the same block, two swaps in the same block can execute against materially different bid/ask quotes, enabling risk-free arbitrage at LP expense.

### Finding Description

`MetricOmmPool.swap()` calls `_getBidAndAskPriceX64()` on every invocation: [1](#0-0) 

That helper delegates to whichever price provider is active. All four provider variants — `PriceProvider`, `PriceProviderL2`, `ProtectedPriceProvider`, and `AnchoredPriceProvider` — unconditionally call `IPricedOracle.price(feedId, pool)` on every invocation: [2](#0-1) [3](#0-2) 

The only guard against stale data is a time-delta check measured in seconds against `block.timestamp`: [4](#0-3) 

`block.timestamp` is identical for every transaction in the same block. The staleness window (`MAX_TIME_DELTA`, minimum 1 second for `PriceProvider`/`ProtectedPriceProvider`) therefore cannot distinguish between two oracle reads that occur in the same block but after an intra-block price update. No provider stores a per-block price snapshot or compares `block.number` against a cached value.

For `AnchoredPriceProvider`, `MAX_REF_STALENESS = 0` is allowed and documented as "same-block reference required": [5](#0-4) 

Even with `MAX_REF_STALENESS = 0`, two Pyth VAAs with the same `publishTime` (same second, same block) both satisfy `refTime == block.timestamp`, so two different prices can pass the staleness gate in the same block.

The band clamp in `AnchoredPriceProvider._computeBidAsk` bounds the quote relative to the anchor mid: [6](#0-5) 

But if the anchor oracle itself is updated intra-block (Pyth is permissionless), the entire band shifts, so the clamp does not prevent the price from changing between two swaps in the same block.

### Impact Explanation

LPs bear the loss. An attacker who controls transaction ordering (MEV) can:

1. Execute swap A at the current (lower) oracle price, acquiring token0 cheaply.
2. Push a new Pyth VAA with a higher price for the same feed.
3. Execute swap B at the updated (higher) oracle price, selling token0 for a profit.

The profit equals `Δprice × amount − total_fees`. Because the spread fee is derived from the bid/ask spread (which is typically a few basis points), any intra-block price move exceeding the spread makes the round-trip profitable. LP balances are directly reduced by the arbitrage delta; the pool cannot recover those funds.

### Likelihood Explanation

Pyth price feeds are updated permissionlessly via signed VAAs. On any chain where Metric OMM is deployed, a searcher can observe a pending Pyth price update (or construct one from the Pyth P2P network), sandwich it between two pool swaps in the same block, and extract value. This requires MEV infrastructure (flashbots, sequencer access on L2) but no privileged role. The attack is repeatable on every block where the oracle price moves by more than the spread.

### Recommendation

Cache the bid/ask price the first time it is read in a block and return the cached value for all subsequent reads within the same block. A minimal implementation stores `(uint64 cachedBlock, uint128 cachedBid, uint128 cachedAsk)` in each provider and skips the oracle call when `block.number == cachedBlock`. This is the mitigation suggested by the external report and eliminates the intra-block price inconsistency at the provider level without affecting cross-block price freshness.

### Proof of Concept

Assume a `PriceProvider`-backed pool for WBTC/USDC with `MAX_TIME_DELTA = 60 s`, `spreadFeeE6 = 5000` (0.5 %), and the Pyth oracle currently reporting WBTC = $50,000.

1. **Block B, Tx 1**: Attacker calls `pool.swap(zeroForOne=false, ...)` — buys 1 WBTC for 50,250 USDC (ask price including spread).
2. **Block B, Tx 2**: Attacker submits a Pyth VAA with `publishTime = block.timestamp` and WBTC price = $51,000. The oracle stores the new price; `_isStale` passes because `block.timestamp − publishTime = 0 ≤ MAX_TIME_DELTA`.
3. **Block B, Tx 3**: Attacker calls `pool.swap(zeroForOne=true, ...)` — sells 1 WBTC for 50,745 USDC (bid price at new oracle mid). Net profit ≈ $495 (after spread fees), funded entirely by LP reserves.

The pool has no mechanism to detect that Tx 1 and Tx 3 used different oracle prices within the same block. [7](#0-6) [8](#0-7)

```text
    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
```

```text
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {

            if (!ok2 || mid2 == 0) return (0, type(uint128).max);

            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }
```
