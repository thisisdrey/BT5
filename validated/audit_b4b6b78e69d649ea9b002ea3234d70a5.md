### Title
Missing Factory Contract for `ProtectedPriceProvider` Permanently Locks `confidenceParam` at Zero, Breaking Pool Swap Execution — (`smart-contracts-poc/contracts/ProtectedPriceProvider.sol`)

### Summary

`ProtectedPriceProvider.setConfidenceParam()` is gated exclusively to `factory`, but no factory contract exists in the codebase to deploy `ProtectedPriceProvider` and relay calls to `setConfidenceParam()`. This is the direct structural analog to H-03: a privileged setter exists on the provider, the calling path from the designated authority is absent, and the parameter is permanently frozen at its zero-initialized value.

### Finding Description

Every other provider type has a paired factory that both deploys it and exposes a managed `setConfidence()` relay:

| Provider | Factory |
|---|---|
| `PriceProvider` | `PriceProviderFactory` (has `setConfidence()`) |
| `PriceProviderL2` | `PriceProviderFactoryL2` (has `setConfidence()`) |
| `AnchoredPriceProvider` | `AnchoredProviderFactory` (has `setConfidence()`) |
| **`ProtectedPriceProvider`** | **No factory exists** |
| **`ProtectedPriceProviderL2`** | **No factory exists** |

`ProtectedPriceProvider` stores `factory` as an immutable set at construction and gates its only mutable parameter behind it:

```solidity
function setConfidenceParam(uint256 newValue) external override {
    require(msg.sender == factory, OnlyFactory());
    ...
    confidenceParam = newValue;
``` [1](#0-0) 

Because no factory contract exists to call this, `confidenceParam` is permanently `0` for any `ProtectedPriceProvider` deployed through the intended factory-managed pattern.

### Impact Explanation

`confidenceParam = 0` collapses the spread to zero before `marginStep` is applied:

```solidity
uint256 adjustedSpread = spread * confidenceParam;   // = 0
(uint256 bid, uint256 ask) = _getBidAskFrom(price, adjustedSpread); // bid == ask == mid
``` [2](#0-1) 

With `bid = ask = mid`, the downstream `_applyStepAdjustment` path produces:

- **`marginStep < 0`**: `stepBidFactor > BPS_BASE_U > stepAskFactor`, so `bidOut > askOut` always → `bidOut >= askOut` guard fires → returns `(0, type(uint128).max)` → `getBidAndAskPrice()` reverts `FeedStalled` on every swap. Pool is permanently unusable. [3](#0-2) 

- **`marginStep = 0`**: `bidOut = floor(mid·Q64/1e8)`, `askOut = ceil(mid·Q64/1e8)`. For any price where `mid·Q64` is divisible by `1e8` (e.g., `mid = 1e8` = $1.00, `mid = 2e8` = $2.00), `floor == ceil`, so `bidOut == askOut` → same revert. Common round prices permanently break the pool.

- **`marginStep > 0`**: Pool functions but `confidenceParam` can never be raised, so the oracle's native confidence interval is permanently ignored. The spread is fixed at the `marginStep` bias alone and cannot be widened during volatile markets, exposing LPs to adverse selection they cannot defend against.

### Likelihood Explanation

`ProtectedPriceProvider` is a production contract (not a mock or test artifact) with the same constructor signature as `PriceProvider`. Any operator following the established pattern — deploying via a factory contract and expecting factory-managed `setConfidence()` calls — will find `confidenceParam` permanently frozen. The `marginStep < 0` and `marginStep = 0` cases are valid constructor inputs (the constructor only rejects `|marginStep| >= BPS_BASE`), so broken pools are reachable without any malicious intent. [4](#0-3) 

### Recommendation

Add a `ProtectedPriceProviderFactory` (mirroring `PriceProviderFactory`) that:
1. Deploys `ProtectedPriceProvider` with `factory = address(this)`.
2. Tracks deployed providers in an `EnumerableSet` and exposes `isProvider()`.
3. Exposes `setConfidence(address[] providers, uint256[] values)` that calls `setConfidenceParam()` on each tracked provider, gated to `onlyProviderOwner` / `_requireUpdater` as in `PriceProviderFactory`.

### Proof of Concept

1. Deploy `ProtectedPriceProvider` with `_factory = address(someMultisig)`, `_marginStep = 0`, valid oracle/feed/tokens.
2. `confidenceParam` initializes to `0`.
3. `someMultisig` has no `setConfidenceParam()` call → `confidenceParam` stays `0`.
4. Pool sets this provider and a user calls `swap()`.
5. Pool calls `provider.getBidAndAskPrice()` → `_getBidAndAskPrice()` → `adjustedSpread = spread * 0 = 0` → `bid = ask = mid`.
6. For `mid = 100_000_000` (i.e., $1.00): `bidOut = floor(1e8 * 2^64 / 1e8) = 2^64`, `askOut = ceil(2^64) = 2^64` → `bidOut >= askOut` → returns `(0, type(uint128).max)`.
7. `getBidAndAskPrice()` reverts `FeedStalled`. Swap reverts. Pool is permanently broken for this price. [5](#0-4) [6](#0-5)
