### Title
Duplicate `baseFeedId`/`quoteFeedId` in `AnchoredProviderFactory.createAnchoredProvider` Produces a Permanently Wrong Synthetic Price — (`File: smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`AnchoredProviderFactory.createAnchoredProvider` performs no check that `quoteFeedId != baseFeedId`. When both are set to the same non-zero value, the synthetic-ratio path in `AnchoredPriceProvider._getBidAndAskPrice` divides the feed price by itself, producing a fixed mid of `ORACLE_DECIMALS = 1e8` (i.e., 1.0) regardless of the actual market price. The resulting provider passes `isProvider()` and can be attached to a public pool, causing every swap to execute at a catastrophically wrong price and draining LP funds.

---

### Finding Description

`AnchoredProviderFactory.createAnchoredProvider` validates the oracle allow-list, the envelope bounds for `baseFeedId`, and the constructor-level parameter ranges, but it never asserts `quoteFeedId == bytes32(0) || quoteFeedId != baseFeedId`. [1](#0-0) 

When a caller passes `quoteFeedId == baseFeedId` (both non-zero), the provider is deployed and immediately registered: [2](#0-1) 

`isProvider()` therefore returns `true` for this provider, satisfying the public-pool eligibility predicate.

At swap time, `AnchoredPriceProvider._getBidAndAskPrice` reads both legs independently: [3](#0-2) 

Because `baseFeedId == quoteFeedId`, both `_readLeg` calls return the same `(mid, spreadBps)`. The synthetic ratio is then:

```
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2)
    = Math.mulDiv(X, 1e8, X)
    = 1e8   // always, regardless of actual market price
spreadBps += spreadBps2   // doubled
``` [4](#0-3) 

`_computeBidAsk` then builds a bid/ask band around a mid of `1e8` (representing 1.0 in 8-decimal oracle terms). The pool's `_getBidAndAskPriceX64` accepts this output without further validation: [5](#0-4) 

Every subsequent swap executes against a price of ~1.0 instead of the real market rate.

The `AnchoredPriceProvider` constructor checks `_baseToken != _quoteToken` but has no analogous check for feed IDs: [6](#0-5) 

---

### Impact Explanation

For any real token pair (e.g., ETH/USDC at ~3 000 USDC/ETH), the provider returns a price of ~1.0. The pool sells the underpriced token at a ~3 000× discount. An attacker buys the underpriced token from the pool and sells it at market, extracting the full LP principal. Pool insolvency is immediate once the attacker exhausts the underpriced side. This is a direct loss of LP principal — High severity.

---

### Likelihood Explanation

`createAnchoredProvider` is permissionless for any caller who uses an allow-listed oracle. The only guard is the oracle allow-list and envelope bounds; no feed-ID uniqueness check exists. An honest integrator can make this mistake accidentally (copy-paste of the same feed ID), and a malicious actor can do it deliberately. The resulting provider passes `isProvider()` and can be attached to a pool immediately.

---

### Recommendation

Add a guard in `AnchoredProviderFactory.createAnchoredProvider` before deploying the provider:

```solidity
if (quoteFeedId != bytes32(0) && quoteFeedId == baseFeedId)
    revert DuplicateFeedId();
```

Optionally mirror the check in the `AnchoredPriceProvider` constructor as a defence-in-depth measure, analogous to the existing `_baseToken != _quoteToken` guard.

---

### Proof of Concept

1. Admin adds oracle `O` to the allow-list and configures a DEFAULT_CLASS envelope.
2. Attacker calls `createAnchoredProvider(O, FEED_X, FEED_X, ...)` — `baseFeedId == quoteFeedId == FEED_X`.
3. Factory deploys the provider; `isProvider(provider) == true`.
4. A pool is created (or updated via `setPriceProvider`) to use this provider.
5. At swap time, `_getBidAndAskPrice` computes `mid = price(FEED_X) * 1e8 / price(FEED_X) = 1e8`.
6. For an ETH/USDC pool (real price ~3 000), the pool quotes ~1 USDC per ETH.
7. Attacker calls `swap(zeroForOne=false, ...)`, buying ETH at ~1 USDC each.
8. Attacker sells ETH on any external market at ~3 000 USDC, pocketing the difference.
9. LP funds are fully drained.

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }
