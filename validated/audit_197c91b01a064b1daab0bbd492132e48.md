### Title
Pool Admin Transfer Cancellation Can Be Front-Run, Enabling Unauthorized Admin Takeover and Admin Fee Theft — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`MetricOmmPoolFactory` implements a two-step pool admin handover (`proposePoolAdminTransfer` → `acceptPoolAdmin`) with a cancellation path (`cancelPoolAdminTransfer`). Because `acceptPoolAdmin` has no timelock and no ordering constraint relative to the cancellation, the pending admin can front-run a cancellation transaction and seize the admin role before the cancellation is mined. Once in control, the attacker can immediately redirect all future admin fees to an arbitrary address via `setPoolAdminFeeDestination`.

### Finding Description

The three relevant functions in `MetricOmmPoolFactory`:

```
proposePoolAdminTransfer  → pendingPoolAdmin[pool] = newAdmin
acceptPoolAdmin           → poolAdmin[pool] = pending; delete pendingPoolAdmin[pool]
cancelPoolAdminTransfer   → delete pendingPoolAdmin[pool]
``` [1](#0-0) 

The attack sequence:

1. Legitimate admin calls `proposePoolAdminTransfer(pool, newAdmin)` — perhaps to rotate to a new multisig.
2. Admin later discovers `newAdmin` is compromised and broadcasts `cancelPoolAdminTransfer(pool)`.
3. Before that cancellation is mined, `newAdmin` observes it in the mempool and submits `acceptPoolAdmin(pool)` with a higher gas price.
4. `acceptPoolAdmin` executes first: `poolAdmin[pool] = newAdmin`, pending slot cleared.
5. The cancellation transaction then reverts with `NoPendingPoolAdminTransfer` — the slot is already empty.
6. `newAdmin` is now the permanent pool admin.

`acceptPoolAdmin` has no guard that would prevent this race: [2](#0-1) 

### Impact Explanation

Once the attacker holds `poolAdmin[pool]`, they can immediately call `setPoolAdminFeeDestination(pool, attacker)`: [3](#0-2) 

All subsequent calls to `collectPoolFees` will route the admin share of every swap's spread and notional fees to the attacker's address. This is a direct, ongoing loss of admin fees owed to the legitimate pool admin. Additionally, the attacker can:

- `pausePool` — freeze swaps and liquidity operations for all users.
- `proposePoolPriceProvider` — schedule a malicious price provider after the timelock, eventually routing bad bid/ask data into pool swaps.
- `setPoolAdminFees` — raise admin fees to the factory cap, extracting maximum value from traders.

The admin fee theft is a direct loss of protocol fees matching the "Critical/High/Medium direct loss of user principal, protocol fees, or owed LP assets" gate. The pause is "unusable withdraw/swap/liquidity flows."

### Likelihood Explanation

The scenario requires:
1. The current admin proposes a transfer (routine operational action).
2. The admin later decides to cancel — the exact scenario the `cancelPoolAdminTransfer` function is documented for: *"Use if a proposal was mistaken or the nominee will not accept."*
3. The pending admin monitors the mempool (trivial with standard tooling on any public chain).

The documentation explicitly describes this cancellation path as a safety mechanism, implying users will rely on it. No comment in the code marks this race as intentional (unlike the upstream `RecoverableWallet` which added an explicit acknowledgment). The trigger is a semi-trusted pending admin, not a fully unprivileged stranger. [4](#0-3) 

### Recommendation

Replace the instant `acceptPoolAdmin` with a time-bounded acceptance window, or add a block-number commit to the proposal so that `acceptPoolAdmin` reverts if the cancellation was submitted in the same or earlier block. The simplest fix is to record `proposedAtBlock` alongside `pendingPoolAdmin` and require `block.number > proposedAtBlock` in `acceptPoolAdmin` — this does not prevent normal acceptance but eliminates same-block and next-block front-running on chains with public mempools. Alternatively, adopt the `Ownable2Step` pattern used by the factory's own owner transfer, which has the same race but is widely understood to be intentional; if the pool admin race is also intentional, add an explicit NatSpec comment to that effect.

// Foundry test demonstrating the front-run
contract FrontRunCancelTest is Test {
    MetricOmmPoolFactory factory;
    address pool;
    address legitimateAdmin = address(0xA);
    address attacker       = address(0xB);

    function setUp() public {
        // ... deploy factory, create pool with legitimateAdmin ...

    function test_frontRunCancel() public {
        // 1. Legitimate admin proposes transfer to attacker
        vm.prank(legitimateAdmin);
        factory.proposePoolAdminTransfer(pool, attacker);
        assertEq(factory.pendingPoolAdmin(pool), attacker);

        // 2. Legitimate admin decides to cancel — broadcasts cancelPoolAdminTransfer
        //    Attacker sees it in the mempool and front-runs with acceptPoolAdmin

        // Simulate front-run: attacker's tx lands first
        vm.prank(attacker);
        factory.acceptPoolAdmin(pool);
        assertEq(factory.poolAdmin(pool), attacker); // attacker is now admin

        // 3. Legitimate admin's cancel now reverts — slot already cleared
        vm.prank(legitimateAdmin);
        vm.expectRevert(IMetricOmmPoolFactoryPoolAdmin.NoPendingPoolAdminTransfer.selector);
        factory.cancelPoolAdminTransfer(pool);

        // 4. Attacker redirects admin fees
        vm.prank(attacker);
        factory.setPoolAdminFeeDestination(pool, attacker);
        assertEq(factory.poolAdminFeeDestination(pool), attacker);
        // All future collectPoolFees() calls send admin share to attacker
    }
}

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function proposePoolAdminTransfer(address pool, address newAdmin) external override nonReentrant onlyPoolAdmin(pool) {
    if (newAdmin == address(0)) revert InvalidAdmin();
    if (newAdmin == poolAdmin[pool]) revert InvalidAdmin();
    pendingPoolAdmin[pool] = newAdmin;
    emit PoolAdminTransferProposed(pool, poolAdmin[pool], newAdmin);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function acceptPoolAdmin(address pool) external override nonReentrant {
    address pending = pendingPoolAdmin[pool];
    if (pending == address(0)) revert NoPendingPoolAdminTransfer();
    if (msg.sender != pending) revert NotPendingPoolAdmin(pool, msg.sender, pending);
    address previousAdmin = poolAdmin[pool];
    poolAdmin[pool] = pending;
    delete pendingPoolAdmin[pool];
    emit PoolAdminTransferred(pool, previousAdmin, pending);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function cancelPoolAdminTransfer(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    address pending = pendingPoolAdmin[pool];
    if (pending == address(0)) revert NoPendingPoolAdminTransfer();
    delete pendingPoolAdmin[pool];
    emit PoolAdminTransferCancelled(pool, pending);
  }
```
