### Title
Missing Chainlink Sequencer Uptime Check in `ProtectedPriceProviderL2` Allows Stale Pre-Downtime Price to Execute Swaps — (`smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

### Summary
`ProtectedPriceProviderL2.getBidAndAskPrice()` performs only a time-delta staleness check on the oracle's `refTime`. It does not consult a Chainlink sequencer uptime feed. When the L2 sequencer restarts after a downtime period shorter than `MAX_TIME_DELTA`, the last pre-downtime oracle price passes the staleness check and is served to the pool swap, violating the protocol's stated invariant.

### Finding Description
`_computeBidAsk` calls `_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)`. This returns `false` (i.e., "not stale") whenever `block.timestamp - refTime <= MAX_TIME_DELTA`. [6](#0-5) 

During sequencer downtime, the oracle is not updated. When the sequencer restarts, `refTime` is the timestamp of the last pre-downtime update. If the downtime was shorter than `MAX_TIME_DELTA` (which can be up to 7 days), the check passes and the stale price is used. [7](#0-6) 

The standard Chainlink pattern for L2 requires checking the sequencer uptime feed and enforcing a grace period after restart (e.g., 3600 seconds) before accepting any oracle data. This is entirely absent from `ProtectedPriceProviderL2`. The constructor accepts no `sequencerUptimeFeed` parameter and stores none. [8](#0-7) 

### Impact Explanation
An attacker observing a sequencer restart can immediately call `pool.swap()` using the stale pre-downtime price. If the market moved during downtime (e.g., a 2–5% move, common during volatile periods that often coincide with sequencer outages), the attacker extracts that delta from LP principal. This is direct LP fund loss via bad-price execution — exactly the "No trade on bad oracle... (L2) sequencer down" invariant the protocol declares must hold. [3](#0-2) 

### Likelihood Explanation
Base (a target deployment chain) has experienced sequencer outages. The attack requires no privileged role — any public `pool.swap()` caller can exploit the restart window. The window lasts until the oracle is updated post-restart, which can be minutes to hours. Severity is **Medium** (requires external condition: sequencer downtime) with potential for **High** if the price deviation during downtime exceeds 1% of LP principal.

### Recommendation
Add a Chainlink sequencer uptime feed check with a grace period to `ProtectedPriceProviderL2`, mirroring the `ChainlinkVerifierL2` pattern already present in the codebase: [4](#0-3) 

```solidity
// In _computeBidAsk, before the staleness check:
if (sequencerUptimeFeed != address(0)) {
    (, int256 answer, uint256 startedAt,,) =
        AggregatorV3Interface(sequencerUptimeFeed).latestRoundData();
    // answer == 0 means sequencer is up; 1 means down
    if (answer != 0 || block.timestamp - startedAt < GRACE_PERIOD) {
        return (0, type(uint128).max); // revert as FeedStalled
    }
}
```

Store `sequencerUptimeFeed` and `GRACE_PERIOD` as immutables set in the constructor.

    function _isStale(
        uint256 refTime,
        uint256 nowTs,
        uint256 maxDelta,
        uint256 futureTol
    ) internal pure returns (bool) {
        if (refTime == 0) return true;

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol
            return (refTime - nowTs) > futureTol;
        }

        // refTime in the past or equal: check age
        return (nowTs - refTime) > maxDelta;
    }
