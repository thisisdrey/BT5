### Title
Missing Arbitrum Sequencer Uptime Check in L2 Price Providers Allows Stale Bid/Ask to Reach Pool Swaps ‚Äî (`smart-contracts-poc/contracts/PriceProviderL2.sol`, `smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

---

### Summary

`PriceProviderL2` and `ProtectedPriceProviderL2` are the production L2 price providers that supply bid/ask quotes to `MetricOmmPool` swaps. Both contracts implement a `MAX_TIME_DELTA` staleness check on the oracle's `refTime`, but neither checks whether the Arbitrum (or other L2) sequencer is currently up or has recently restarted. When the sequencer is down for a duration shorter than `MAX_TIME_DELTA`, the last pre-downtime oracle price passes the staleness check and is used to price live swaps. Because `MAX_TIME_DELTA` is a constructor parameter bounded only by 7 days, providers deployed with large staleness windows are especially exposed.

---

### Finding Description

`PriceProviderL2._getBidAndAskPrice()` reads the oracle price and applies a single staleness guard:

```solidity
// PriceProviderL2.sol lines 208‚Äì216
function _getBidAndAskPrice() internal returns (uint128, uint128) {
    (uint256 mid, uint256 spread, , uint256 refTime) =
        IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

    // 2. Staleness check
    if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
        return (0, type(uint128).max);
    }
    ...
}
```

The `_isStale` helper only compares `refTime` against `block.timestamp`:

```solidity
// PriceProviderL2.sol lines 135‚Äì150
function _isStale(
    uint256 refTime, uint256 nowTs,
    uint256 maxDelta, uint256 futureTol
) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

There is no call to a Chainlink sequencer uptime feed (`AggregatorV3Interface`) and no grace-period enforcement after sequencer restart. The grep search across all `.sol` files confirms zero occurrences of `sequencerUptimeFeed`, `GRACE_PERIOD`, or any equivalent check in any production contract. The `ChainlinkVerifierL2` artifact visible in the registry JSON has no corresponding source file in the repository.

`ProtectedPriceProviderL2._computeBidAsk()` has the identical gap:

```solidity
// ProtectedPriceProviderL2.sol lines 203‚Äì208
function _computeBidAsk(uint256 price, uint256 spread, uint256 refTime)
    internal view returns (uint128, uint128)
{
    if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
        return (0, type(uint128).max);
    }
```

`PriceProviderFactoryL2.createPriceProvider()` deploys providers with caller-supplied `_maxTimeDelta` (validated only to be non-zero and ‚â§ 7 days), so any `MAX_TIME_DELTA` from 1 second to 604 800 seconds is accepted:

```solidity
// PriceProviderL2.sol lines 92‚Äì95
if (_maxTimeDelta == 0 || _maxTimeDelta > 7 days) revert MaxTimeDeltaOutOfBounds();
if (_futureTolerance > 1 hours) revert FutureToleranceOutOfBounds();
MAX_TIME_DELTA   = _maxTimeDelta;
FUTURE_TOLERANCE = _futureTolerance;
```

**Attack scenario:**

1. Arbitrum sequencer goes down at time `T`. The last oracle price update has `refTime = T`.
2. Sequencer is down for duration `D` where `D < MAX_TIME_DELTA`.
3. Sequencer restarts at `T + D`. `block.timestamp` is now `T + D`.
4. Staleness check: `(T + D) ‚àí T = D < MAX_TIME_DELTA` ‚Üí **passes**.
5. The price from time `T` ‚Äî which may be significantly stale relative to the true market ‚Äî is returned as the live bid/ask.
6. An attacker who observed the market move during the downtime submits a swap immediately on sequencer restart, trading against the stale price before the oracle is updated.

Additionally, even after the oracle is updated post-restart, there is no grace period enforced. Chainlink's recommended pattern enforces a `GRACE_PERIOD` (typically 3 600 seconds) after sequencer restart to allow prices to stabilize; this codebase has no equivalent.

---

### Impact Explanation

Stale bid/ask prices reach `MetricOmmPool` swaps. A trader can buy the underpriced token (or sell the overpriced token) at the pre-downtime price, extracting value from LP depositors. The magnitude scales with:
- The price movement during sequencer downtime.
- The configured `MAX_TIME_DELTA` (up to 7 days for permissionlessly deployed providers).
- The liquidity depth of the affected pool.

This is a direct loss of LP principal ‚Äî the pool pays out more tokens than the current oracle price warrants.

---

### Likelihood Explanation

Arbitrum sequencer downtime events occur multiple times per year (historically documented). The attack requires no special privilege: `PriceProviderFactoryL2.createPriceProvider()` is permissionless, and `MetricOmmPool.swap()` is callable by any EOA. The attacker only needs to monitor the sequencer status and submit a swap transaction on restart before the oracle is refreshed. With a large `MAX_TIME_DELTA`, the window is wide.

---

### Recommendation

Add a sequencer uptime check in `PriceProviderL2._getBidAndAskPrice()` and `ProtectedPriceProviderL2._computeBidAsk()` following Chainlink's documented pattern:

```solidity
// Store as immutable: AggregatorV3Interface public immutable sequencerUptimeFeed;
// uint256 public constant GRACE_PERIOD = 3600;

function _checkSequencer() internal view {
    if (address(sequencerUptimeFeed) == address(0)) return; // L1 / no feed configured
    (, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
    // answer == 0 means sequencer is up
    if (answer != 0) revert SequencerDown();
    if (block.timestamp - startedAt < GRACE_PERIOD) revert GracePeriodNotOver();
}
```

Call `_checkSequencer()` at the top of `_getBidAndAskPrice()` / `_computeBidAsk()` before reading the oracle price. Pass `_sequencerUptimeFeed` through `PriceProviderFactoryL2.createPriceProvider()` (the registry ABI already includes this parameter, confirming the intent). Allow `address(0)` to skip the check for non-L2 deployments.

### Proof of Concept

```

  - Deploy PriceProviderL2 with MAX_TIME_DELTA = 300 (5 minutes), on Arbitrum.
  - Pool uses this provider.

Attack:
  T=0:   Oracle price = $1000. refTime = 0.
  T=0:   Arbitrum sequencer goes down. No oracle updates possible.

         Pool sells base token at $1000 ask (true market = $1100).
