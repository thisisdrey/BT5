### Title
Synthetic Ratio Legs in `AnchoredPriceProvider._getBidAndAskPrice` Are Never Cross-Validated for Timestamp Proximity, Allowing a Stale-vs-Fresh Price Ratio to Reach Pool Swaps - (File: `smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports a **synthetic ratio mode** (`quoteFeedId != 0`) where the pool's bid/ask band is derived from `price(baseFeedId) / price(quoteFeedId)`. Each leg is individually validated for staleness against `MAX_REF_STALENESS`, but the two legs' `refTime` values are **never compared against each other**. One leg can be fresh (timestamp = T) while the other is at the staleness boundary (timestamp = T − MAX_REF_STALENESS), and the ratio is computed and delivered to the pool swap without any inter-leg timestamp proximity check.

---

### Finding Description

In `_getBidAndAskPrice`, both legs are read via `_readLeg`, which returns `(mid, spreadBps, refTime, ok)`:

```solidity
// AnchoredPriceProvider.sol L258-272
function _getBidAndAskPrice() internal returns (uint128, uint128) {
    (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);   // refTime discarded
    if (!ok) return (0, type(uint128).max);

    bytes32 _quote = quoteFeedId;
    if (_quote != bytes32(0)) {
        (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote); // refTime discarded
        if (!ok2 || mid2 == 0) return (0, type(uint128).max);
        mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // ratio from potentially mismatched timestamps
        spreadBps += spreadBps2;
    }
    return _computeBidAsk(mid, spreadBps);
}
```

Inside `_readLeg`, each leg is checked independently:

```solidity
// AnchoredPriceProvider.sol L283
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
```

The `refTime` is returned as the third tuple element but is silently discarded (`,`) at both call sites in `_getBidAndAskPrice`. No code ever computes `|refTime1 - refTime2|` or enforces a maximum inter-leg skew.

The constructor allows `MAX_REF_STALENESS` up to 7 days:

```solidity
// AnchoredPriceProvider.sol L150
if (_maxRefStaleness > 7 days) revert MaxRefStalenessOutOfBounds();
```

The factory's `createAnchoredProvider` is permissionless and passes `maxRefStaleness` through an envelope bound that can be as wide as the admin configures:

```solidity
// AnchoredProviderFactory.sol L176-180
if (
    minMargin < env.minMarginMin || minMargin > env.minMarginMax
    || maxRefStaleness < env.stalenessMin || maxRefStaleness > env.stalenessMax
    ...
) revert ParamsOutOfEnvelope();
```

No existing guard — not `MAX_SPREAD_BPS`, not `priceGuard`, not the individual staleness check — prevents the two legs from having timestamps that are `MAX_REF_STALENESS` apart.

---

### Impact Explanation

The synthetic mid price `mid1 / mid2` is computed from prices at potentially very different points in time. During volatile periods:

- If `baseFeedId` (e.g., BTC/USD) is current and `quoteFeedId` (e.g., ETH/USD) is stale at a higher value, the synthetic BTC/ETH ratio is **understated** → the pool quotes BTC too cheaply → traders drain LP value by buying BTC at a discount.
- If `quoteFeedId` is stale at a lower value, the synthetic ratio is **overstated** → the pool quotes BTC too expensively → traders sell BTC to the pool at an inflated price, again at LP expense.

In reference mode (`source == 0`), the wrong synthetic mid **directly becomes the bid/ask** delivered to the pool swap — there is no further clamping layer. In source mode, the wrong band still bounds the source, so the source can quote prices that are wrong by up to the full band width centered on the wrong mid.

This is a **bad-price execution** impact: the pool swap settles at a price derived from a temporally mismatched ratio, causing direct loss of LP principal.

---

### Likelihood Explanation

- Synthetic ratio mode is an explicitly supported, documented, and factory-deployable configuration (`quoteFeedId` is a named constructor parameter with its own `ProviderDeployed` event field).
- The two feeds in a synthetic pair (e.g., BTC/USD on Pyth vs. ETH/USD on Chainlink) can have materially different update cadences. Pyth updates sub-second; Chainlink updates on deviation or heartbeat (minutes to hours).
- No privileged action is required to trigger the discrepancy — it is a normal market condition during oracle delays, network congestion, or any period where one feed's publisher is temporarily slow.
- The `AnchoredProviderFactory.createAnchoredProvider` is permissionless, so any curator can deploy a synthetic provider and attach it to a pool.

---

### Recommendation

After both legs pass their individual staleness checks, enforce a maximum inter-leg timestamp skew before computing the ratio:

```solidity
// After both _readLeg calls succeed, add:
if (refTime > refTime2 ? refTime - refTime2 : refTime2 - refTime > MAX_LEG_SKEW) {
    return (0, type(uint128).max);
}
```

`MAX_LEG_SKEW` should be an immutable set at construction (e.g., bounded to a small fraction of `MAX_REF_STALENESS`, such as 5 minutes for majors). Alternatively, use `min(refTime1, refTime2)` as the effective timestamp for the combined read and apply the staleness check to that minimum, which naturally rejects any pair where one leg is significantly older than the other.

---

### Proof of Concept

The existing test suite (`AnchoredPriceProviderSyntheticTest`) confirms the synthetic path is live and that each leg is checked independently. The following scenario demonstrates the gap:

```
MAX_REF_STALENESS = 3600 (1 hour)
T = 10_000

baseFeedId  (BTC/USD): mid = 65_000e8, refTime = 10_000   (fresh)
quoteFeedId (ETH/USD): mid = 2_000e8,  refTime = 6_401    (3599s old — just inside staleness)

True BTC/ETH at T=10_000: 65_000 / 3_000 ≈ 21.67   (ETH has risen to 3_000)
Computed synthetic mid:    65_000 / 2_000 = 32.50   (50% overstatement)

Both _readLeg calls return ok=true.
_computeBidAsk receives mid = 32.50e8, spreadBps = u1+u2.
Pool swap executes at a 50%-inflated BTC/ETH price.
LPs who hold ETH receive BTC worth far less than the ETH they give up.

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
