### Title
`confidenceParam` Zero-Initialized in `PriceProvider` Silently Discards Oracle Spread, Producing Unclamped Tight Bid/Ask Quotes at Swap Time — (`smart-contracts-poc/contracts/PriceProvider.sol`)

---

### Summary

`PriceProvider.confidenceParam` is a storage variable that defaults to `0` (Solidity zero-initialization). Every provider deployed via `PriceProviderFactory.createPriceProvider` starts in this state. Because the oracle's reported spread is multiplied by `confidenceParam` before being applied, a zero value silently discards the oracle's entire uncertainty signal. The resulting bid/ask spread is determined solely by the immutable `marginStep`, with no floor, no band clamp, and no staleness-of-spread guard — the exact analog of `callRestricted = false` by default.

---

### Finding Description

In `PriceProvider._getBidAndAskPrice()`, the oracle spread is scaled by `confidenceParam`:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [1](#0-0) 

`_getBidAskFrom` computes:

```solidity
uint256 delta = midPrice * confidence / CONFIDENCE_BASE;
bid = delta >= midPrice ? 0 : midPrice - delta;
ask = midPrice + delta;
``` [2](#0-1) 

When `confidenceParam == 0`, `adjustedSpread = 0`, `delta = 0`, and `bid = ask = mid`. The oracle's spread — which encodes real-time market uncertainty — is completely discarded. The only spread that survives is the fixed `marginStep` bias baked in at construction:

```solidity
stepBidFactor = uint256(BPS_BASE - _marginStep);
stepAskFactor = uint256(BPS_BASE + _marginStep);
``` [3](#0-2) 

`confidenceParam` is declared as a plain storage slot with no constructor initialization:

```solidity
uint256 public confidenceParam;
uint256 public lastConfidenceUpdate;
``` [4](#0-3) 

`PriceProviderFactory.createPriceProvider` is **permissionless** and passes no initial `confidenceParam`:

```solidity
PriceProvider p = new PriceProvider(
    address(this), _oracle, _feedId, _marginStep, _maxTimeDelta, _baseToken, _quoteToken
);
``` [5](#0-4) 

There is no oracle allow-list, no envelope validation, and no requirement that `confidenceParam` be set before the provider is attached to a pool. The factory owner must call `setConfidence` explicitly after deployment; until then every swap through the provider executes against a spread that ignores oracle uncertainty entirely.

**Contrast with `AnchoredPriceProvider`**: that contract's `_computeBidAsk` enforces a hard band clamp (`min(refBid, cBid)` / `max(refAsk, cAsk)`) so that even with `confidenceParam = 0` the final quote cannot be tighter than `mid ± (spreadBps + minMargin)`. `PriceProvider` has no such clamp. [6](#0-5) 

---

### Impact Explanation

When `confidenceParam = 0` and `marginStep` is small (e.g., 1 bps), the pool quotes a 2 bps spread regardless of oracle-reported uncertainty. During high-volatility periods the oracle may report a 3–10% spread, but the pool ignores it and continues to offer a 2 bps market. Arbitrageurs can repeatedly trade against this mispriced spread, extracting value from LPs at a rate proportional to the gap between the oracle's true uncertainty and the `marginStep`-only spread. Because the pool's bin-based liquidity is concentrated around the oracle mid, each such trade moves the active bin and leaves LPs holding the adverse side of the position. This is a direct loss of LP principal, satisfying the Critical/High/Medium impact gate.

---

### Likelihood Explanation

- `PriceProviderFactory.createPriceProvider` is permissionless; any address can deploy a provider.
- The provider is immediately usable (no activation step, no `confidenceParam` requirement).
- The factory owner must notice the omission and call `setConfidence` before the provider is attached to a pool; there is no on-chain enforcement of this ordering.
- A `CONFIDENCE_COOLDOWN` of 1 minute applies to *updates*, not to the initial set, but the initial set still requires an explicit transaction that can be front-run or simply forgotten.
- The window between provider deployment and `confidenceParam` initialization is an unprivileged-trigger window: any user who initiates a swap through the pool during that window executes against the zero-spread provider. [7](#0-6) 

---

### Recommendation

**Short term:** Require `confidenceParam` to be supplied as a constructor argument and validated (`> 0`, `<= CONFIDENCE_MAX`) so no provider can be deployed in the zero-spread state:

```solidity
constructor(
    ...
    uint256 _confidenceParam,   // NEW: must be > 0
    ...
) {
    require(_confidenceParam > 0 && _confidenceParam <= CONFIDENCE_MAX);
    confidenceParam = _confidenceParam;
    lastConfidenceUpdate = block.timestamp;
}
```

**Long term:** Add a band clamp analogous to `AnchoredPriceProvider._computeBidAsk` so that even a misconfigured `confidenceParam` cannot produce a spread tighter than the oracle's own reported uncertainty. Alternatively, gate pool attachment on `confidenceParam != 0` inside the pool factory.

---

### Proof of Concept

1. Attacker (or any user) calls `PriceProviderFactory.createPriceProvider` with `_marginStep = 1e14` (1 bps), `_maxTimeDelta = 1 hours`, valid `_oracle`/`_feedId`/tokens.
2. `confidenceParam` is `0` by default; no `setConfidence` call is made.
3. Oracle reports `mid = 100_000_000` (100 USD, 8 dec), `spread = 500` (5% uncertainty).
4. Inside `_getBidAndAskPrice`: `adjustedSpread = 500 * 0 = 0`; `bid = ask = 100_000_000`.
5. After `marginStep` (1 bps): `bidOut ≈ mid × (1e18 − 1e14) / 1e18`, `askOut ≈ mid × (1e18 + 1e14) / 1e18` → effective spread ≈ 2 bps.
6. Pool executes swaps at 2 bps spread while true oracle uncertainty is 10% (5% × 2 sides).
7. Arbitrageur buys at `bidOut` (2 bps below mid) and sells on an external venue at mid − 5%, netting ~4.98% per round-trip at LP expense, repeating until the bin is drained or `confidenceParam` is corrected. [8](#0-7) [9](#0-8)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
```
