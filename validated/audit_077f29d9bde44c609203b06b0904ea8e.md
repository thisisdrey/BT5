### Title
Stale Updater Permissions Persist After `transferProviderOwnership`, Allowing Unauthorized `confidenceParam` Manipulation — (`PriceProviderFactory.sol`, `PriceProviderFactoryL2.sol`, `AnchoredProviderFactory.sol`)

### Summary

`transferProviderOwnership` in all three factory contracts updates `providerOwner` but never clears the `isUpdater[provider][*]` mappings. Old updaters granted by the previous owner retain the ability to call `setConfidence` on the provider after ownership has transferred, bypassing the new owner's intended access control.

### Finding Description

In `PriceProviderFactory.transferProviderOwnership` (and the identical logic in `PriceProviderFactoryL2` and `AnchoredProviderFactory`), ownership is transferred by updating `providerOwner[provider]` and adjusting the `_providersByCreator` sets. However, the `isUpdater[provider][address]` mapping for every updater previously granted by the old owner is never cleared:

```solidity
// PriceProviderFactory.sol L92-102
function transferProviderOwnership(address provider, address newOwner) external override onlyProviderOwner(provider) {
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);
    // ← isUpdater[provider][oldUpdater] is NEVER cleared
    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
```

The `_requireUpdater` check used by `setConfidence` passes for any address where `isUpdater[provider][msg.sender] == true`, regardless of whether the granting owner still owns the provider:

```solidity
// PriceProviderFactory.sol L34-37
function _requireUpdater(address provider) internal view {
    if (msg.sender != providerOwner[provider] && !isUpdater[provider][msg.sender])
        revert NotProviderUpdater();
}
```

So after `transferProviderOwnership(provider, newOwner)`, any address that the **old** owner had granted via `grantUpdater` can still call `setConfidence([provider], [value])` successfully, because:
1. `_providers.contains(provider)` → still `true`
2. `_requireUpdater(provider)` → `isUpdater[provider][oldUpdater]` is still `true` → passes

This is the direct analog of the `__abdicate()` bug: the "pending" state (`isUpdater`) is not cleared when the authority (`providerOwner`) is transferred away, allowing the stale grantee to continue exercising the delegated power.

### Impact Explanation

For `PriceProvider` and `ProtectedPriceProvider` (used via `PriceProviderFactory` / `PriceProviderFactoryL2`), `confidenceParam` directly controls the bid/ask spread with **no band clamp**:

```solidity
// PriceProvider.sol L216-217
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
```

A stale updater can set `confidenceParam = 0`, collapsing the spread to zero before the `marginStep` adjustment. If `marginStep > 0`, the pool then quotes at `mid × (1 ± marginStep/BPS_BASE)` — the tightest possible spread — instead of the new owner's intended wider spread. Every swap executed at this tighter spread represents LP fee revenue that is permanently lost: the pool sells base tokens at a lower ask and buys at a higher bid than the new owner configured, with the difference going to traders rather than LPs.

For `AnchoredPriceProvider` (via `AnchoredProviderFactory`), the band clamp in `_computeBidAsk` limits the damage: the final quote cannot be tighter than `mid ± (spreadBps + minMargin)`, so `confidenceParam` manipulation is neutralized. The impact there is limited to DoS (FeedStalled) if the shaped quote inverts before clamping.

### Likelihood Explanation

Provider ownership transfer is a normal operational event (e.g., protocol team handoff, DAO takeover, sale of a provider). The old owner may have granted updater rights to automated bots, team members, or third-party keepers. The new owner has no on-chain enumeration of who holds `isUpdater == true` (the mapping is not indexed), making it difficult to revoke all stale updaters. A malicious or negligent old updater can exploit the window between transfer and revocation — or indefinitely if the new owner is unaware of the stale grants.

### Recommendation

Clear all updater permissions for the provider at transfer time, or implement a two-step transfer with an explicit updater-reset. The minimal fix is to require the caller to supply a list of updaters to revoke atomically, or to track updaters in an enumerable set per provider so they can be iterated and cleared:

```solidity
function transferProviderOwnership(address provider, address newOwner) external override onlyProviderOwner(provider) {
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);

+   // Clear all updater grants so the new owner starts with a clean slate.
+   // Requires updaters to be tracked in an EnumerableSet per provider.
+   address[] memory oldUpdaters = _updatersByProvider[provider].values();
+   for (uint256 i; i < oldUpdaters.length; ++i) {
+       isUpdater[provider][oldUpdaters[i]] = false;
+       _updatersByProvider[provider].remove(oldUpdaters[i]);
+   }

    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
```

Alternatively, document clearly that the new owner must call `revokeUpdater` for every previously granted address before the transfer is considered complete, and provide a view function to enumerate all active updaters for a provider.

### Proof of Concept

```
1. Alice deploys PriceProvider P via PriceProviderFactory.createPriceProvider(...)
   → providerOwner[P] = Alice

2. Alice grants Bob as updater:
   factory.grantUpdater(P, Bob)
   → isUpdater[P][Bob] = true

3. Alice transfers ownership to Carol:
   factory.transferProviderOwnership(P, Carol)
   → providerOwner[P] = Carol
   → isUpdater[P][Bob] is STILL true  ← bug

4. Carol sets confidenceParam to 50_000 (5% spread):
   factory.setConfidence([P], [50_000])   // succeeds, Carol is owner

5. Bob (old updater, no longer trusted) calls:
   factory.setConfidence([P], [0])
   → _providers.contains(P) = true ✓
   → _requireUpdater(P): isUpdater[P][Bob] = true ✓ → passes
   → PriceProvider(P).setConfidenceParam(0) succeeds

6. Pool calls P.getBidAndAskPrice():
   adjustedSpread = spread * 0 = 0
   bid = ask = mid
   After marginStep: bid = mid*(1-marginStep), ask = mid*(1+marginStep)
   → Pool quotes at tightest possible spread; LP fee revenue is lost on every swap
      until Carol notices and revokes Bob (or resets confidenceParam).

        address[] calldata providers,
        uint256[] calldata values
    ) external override {
        uint256 l = providers.length;
        if (l != values.length) revert LengthMismatch();

        for (uint256 i; i < l; ++i) {
            require(_providers.contains(providers[i]), ProviderNotTracked());
            _requireUpdater(providers[i]);
            PriceProvider(providers[i]).setConfidenceParam(values[i]);
        }
    }
