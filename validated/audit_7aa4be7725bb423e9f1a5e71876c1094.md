### Title
Identical `baseFeedId` and `quoteFeedId` in synthetic mode collapses mid-price to a constant `1e8`, delivering a permanently wrong bid/ask to pool swaps — (`AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports a synthetic ratio mode (`quoteFeedId != bytes32(0)`) that computes `mid = price(baseFeedId) / price(quoteFeedId)`. Neither the constructor nor the permissionless factory function `createAnchoredProvider` checks that `baseFeedId != quoteFeedId`. When both are identical, the division collapses to the constant `ORACLE_DECIMALS` (`1e8`), a fixed price of 1.0 in 8-decimal format, regardless of the actual market price. Every subsequent `getBidAndAskPrice()` call returns a bid/ask band anchored to this wrong constant, and every pool swap executes against it.

---

### Finding Description

In `_getBidAndAskPrice()`, when `quoteFeedId != bytes32(0)`, the synthetic ratio path reads both legs and divides:

```solidity
(uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
...
(uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote); // _quote == baseFeedId
...
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // mid / mid2 == ORACLE_DECIMALS == 1e8
spreadBps += spreadBps2;                          // doubled spread
``` [1](#0-0) 

When `baseFeedId == quoteFeedId`, `_readLeg` is called twice on the same feed, returning the same `mid` value both times. The division `Math.mulDiv(mid, 1e8, mid)` evaluates to exactly `1e8` — a constant price of 1.0 — for any non-zero `mid`. The actual market price is completely discarded.

The constructor validates tokens are distinct:

```solidity
require(_baseToken != address(0) && _quoteToken != address(0) && _baseToken != _quoteToken);
``` [2](#0-1) 

But there is no analogous check for feed IDs:

```solidity
baseFeedId = _baseFeedId;
quoteFeedId = _quoteFeedId;   // no require(_baseFeedId != _quoteFeedId)
``` [3](#0-2) 

The permissionless factory `createAnchoredProvider` passes both IDs straight through without any equality check:

```solidity
AnchoredPriceProvider p = new AnchoredPriceProvider(
    address(this), oracle,
    baseFeedId, quoteFeedId,   // no baseFeedId != quoteFeedId guard
    ...
);
``` [4](#0-3) 

The downstream `_computeBidAsk` then builds a bid/ask band around `mid = 1e8`. In Q64.64 terms this represents a price of exactly 1.0 token0 per token1, regardless of the true market ratio. For any pair where the tokens have different values (e.g., BTC/ETH ≈ 20:1), the pool executes every swap at a massively wrong price.

---

### Impact Explanation

Every call to `getBidAndAskPrice()` on the misconfigured provider returns a bid/ask band centered on `1e8` (price = 1.0). The pool's swap math uses this price to determine how much input is required for a given output. A trader can:

1. Swap the undervalued token for the overvalued token at the 1:1 oracle price, receiving far more value than they pay.
2. Drain the pool's reserves of the overvalued token until liquidity is exhausted.

LP principal is directly at risk. The pool is insolvent relative to LP claims the moment the first exploiting swap executes.

---

### Likelihood Explanation

`createAnchoredProvider` is **permissionless** — any address can call it. An attacker can:

1. Call `createAnchoredProvider(oracle, feedId, feedId, ...)` with `baseFeedId == quoteFeedId`.
2. Deploy a pool using this provider.
3. Seed the pool with liquidity (or attract honest LPs via normal UI).
4. Execute swaps against the constant-1.0 price to extract value.

No privileged role is required. The only prerequisite is that the oracle is on the factory's allow-list, which is a normal operational state. An honest curator making a copy-paste error achieves the same outcome without malicious intent.

---

### Recommendation

Add the following check in the `AnchoredPriceProvider` constructor, mirroring the existing token-equality guard:

```solidity
// In constructor, after setting baseFeedId / quoteFeedId:
if (_quoteFeedId != bytes32(0)) {
    require(_baseFeedId != _quoteFeedId, "synthetic: base and quote feed must differ");
}
```

Optionally add the same guard in `AnchoredProviderFactory.createAnchoredProvider` for early revert with a clear factory-level error before the constructor is reached.

---

### Proof of Concept

```solidity
// Assume: oracle has feedId registered, factory has oracle allow-listed and envelope set.

bytes32 feedId = keccak256("BTC/USD");

// Attacker (or mistaken curator) passes the same feedId for both base and quote:
address provider = factory.createAnchoredProvider(
    oracle,
    feedId,   // baseFeedId
    feedId,   // quoteFeedId  <-- same as baseFeedId
    minMargin, maxRefStaleness, maxSpreadBps,
    false, 0,
    address(BTC), address(ETH)
);

// Oracle returns: mid = 3_000_000_000_000 (BTC = $30,000 in 8-decimal)
// _readLeg(baseFeedId) → mid  = 3_000_000_000_000
// _readLeg(quoteFeedId) → mid2 = 3_000_000_000_000  (same feed, same value)
// mid = mulDiv(3_000_000_000_000, 1e8, 3_000_000_000_000) = 1e8  ← constant 1.0

// Pool swap executes at price 1.0 (1 BTC = 1 ETH in oracle terms).
// Attacker pays 1 ETH, receives 1 BTC worth ~20 ETH at market.
// Repeat until pool is drained.
```

The corrupted value is `mid = ORACLE_DECIMALS = 1e8` (line 267 of `AnchoredPriceProvider.sol`), produced whenever `baseFeedId == quoteFeedId` in the synthetic path, and it flows unchanged through `_computeBidAsk` into the bid/ask returned to the pool. [5](#0-4) [6](#0-5)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }
```
