### Title
Single `MAX_REF_STALENESS` Applied to Both Legs of Synthetic Ratio Feeds Allows Stale Quote-Leg Price to Reach Pool Swaps — (`File: smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports a two-feed synthetic ratio mode (`quoteFeedId != 0`) where the final mid price is `price(baseFeedId) / price(quoteFeedId)`. Both legs are validated by `_readLeg`, which applies the **same** single immutable `MAX_REF_STALENESS` to both feeds. Because real-world oracle feeds have different heartbeats (e.g., BTC/USD: 3600 s on Ethereum; ETH/USD: 27 s), a single staleness bound cannot correctly guard both legs simultaneously. The deployer is forced to choose between a DoS on the slower feed or accepting a stale faster feed, and the stale price propagates directly into the bid/ask quotes delivered to pool swaps.

---

### Finding Description

`_getBidAndAskPrice` reads both legs through `_readLeg`:

```solidity
// AnchoredPriceProvider.sol L258-271
function _getBidAndAskPrice() internal returns (uint128, uint128) {
    (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
    if (!ok) return (0, type(uint128).max);

    bytes32 _quote = quoteFeedId;
    if (_quote != bytes32(0)) {
        (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
        if (!ok2 || mid2 == 0) return (0, type(uint128).max);
        mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // ratio
        spreadBps += spreadBps2;
    }
    return _computeBidAsk(mid, spreadBps);
}
```

`_readLeg` applies `MAX_REF_STALENESS` identically to whichever feed it is called with:

```solidity
// AnchoredPriceProvider.sol L277-295
function _readLeg(bytes32 feedId)
    internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
{
    (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);
    if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
    ...
}
```

`MAX_REF_STALENESS` is a single immutable set once at construction:

```solidity
// AnchoredPriceProvider.sol L77, L150-151
uint256 public immutable MAX_REF_STALENESS;
...
if (_maxRefStaleness > 7 days) revert MaxRefStalenessOutOfBounds();
MAX_REF_STALENESS = _maxRefStaleness;
```

`createAnchoredProvider` in `AnchoredProviderFactory` accepts a single `maxRefStaleness` parameter and passes it to the constructor — there is no per-leg staleness parameter:

```solidity
// AnchoredProviderFactory.sol L156-194
function createAnchoredProvider(
    ...
    uint256 maxRefStaleness,
    ...
) external override returns (address provider) {
    ...
    AnchoredPriceProvider p = new AnchoredPriceProvider(
        address(this), oracle, baseFeedId, quoteFeedId,
        minMargin, maxRefStaleness, ...
    );
```

The envelope only bounds `stalenessMin`/`stalenessMax` as a single range applied to the single parameter — it cannot express different bounds for the two legs.

**Concrete scenario (Ethereum mainnet):**

| Feed | Chainlink heartbeat |
|---|---|
| BTC/USD (`baseFeedId`) | 3 600 s |
| ETH/USD (`quoteFeedId`) | 27 s |

- If `MAX_REF_STALENESS = 27 s`: `_readLeg(baseFeedId)` reverts almost every block → swap DoS.
- If `MAX_REF_STALENESS = 3 600 s` (set to accommodate the slower feed): `_readLeg(quoteFeedId)` accepts an ETH/USD price up to 3 600 s old. A 10 % ETH move in that window produces a synthetic BTC/ETH ratio that is 10 % wrong. The corrupted ratio flows through `_computeBidAsk` and is returned as the live bid/ask to the pool swap.

---

### Impact Explanation

The stale `mid2` (quote-leg price) corrupts the synthetic ratio `mid = mulDiv(mid, ORACLE_DECIMALS, mid2)`. This wrong mid is then used to compute the reference band (`refBid`, `refAsk`) and the final bid/ask returned to the pool. The band clamp does **not** protect here because the reference itself is derived from the stale price — the clamp only clips the custom source or shaped quote to the (already-stale) band edges.

A trader who observes the true current ETH price can execute swaps against the stale BTC/ETH quote, extracting value directly from LP positions. This is a bad-price execution impact: the pool receives less input token than the true market rate warrants, or delivers more output token, causing direct loss of LP principal.

---

### Likelihood Explanation

- The synthetic ratio mode (`quoteFeedId != bytes32(0)`) is explicitly documented and supported as a first-class feature.
- `createAnchoredProvider` is permissionless; any creator deploying a BTC/ETH-style synthetic pair faces this structural constraint.
- The envelope admin cannot fix it: the envelope has only one staleness range, not two per-leg ranges.
- The deployer has no correct choice: any single value for `MAX_REF_STALENESS` is either too tight (DoS) or too loose (stale price) when the two feeds have materially different heartbeats.
- Chainlink feed heartbeats routinely differ by 100× or more across common pairs.

---

### Recommendation

Add a second per-leg staleness parameter for the quote feed:

```solidity
uint256 public immutable MAX_REF_STALENESS;       // for baseFeedId
uint256 public immutable MAX_QUOTE_STALENESS;     // for quoteFeedId (0 = use MAX_REF_STALENESS)
```

Pass `MAX_QUOTE_STALENESS` to `_readLeg` when reading the quote leg:

```solidity
if (_quote != bytes32(0)) {
    uint256 quoteStaleness = MAX_QUOTE_STALENESS == 0 ? MAX_REF_STALENESS : MAX_QUOTE_STALENESS;
    (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLegWithStaleness(_quote, quoteStaleness);
    ...
}
```

Update `AnchoredProviderFactory.createAnchoredProvider` and the envelope struct to accept and validate a separate `maxQuoteStaleness` parameter, allowing the admin to set per-leg heartbeat-appropriate bounds.

### Proof of Concept

1. Admin deploys `AnchoredProviderFactory`, sets envelope with `stalenessMin=1, stalenessMax=7 days`.
2. Creator calls `createAnchoredProvider(oracle, BTC_USD_FEED, ETH_USD_FEED, ..., maxRefStaleness=3600, ...)`.
3. ETH/USD feed is last updated 3 599 s ago (within `MAX_REF_STALENESS`). True ETH price has moved 8 % since then.
4. Pool calls `getBidAndAskPrice()` → `_readLeg(ETH_USD_FEED)` passes staleness check → stale `mid2` used.
5. Synthetic BTC/ETH ratio is 8 % off from true market.
6. Attacker swaps BTC for ETH at the stale (favorable) BTC/ETH ask price, receiving 8 % more ETH than the true market rate warrants.

```text
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
```

```text
        address oracle,
        bytes32 baseFeedId,
        bytes32 quoteFeedId,

        uint256 maxRefStaleness,

        bool    mutableParams,

    ) external override returns (address provider) {
        if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);

        // Feeds without an explicit class fall back to the admin-configured DEFAULT_CLASS envelope.
        bytes32 classId = feedClass[baseFeedId];
        if (classId == bytes32(0)) classId = DEFAULT_CLASS;

        Envelope storage env = envelopes[classId];
        if (!env.exists) revert EnvelopeNotFound(classId);
        if (

        AnchoredPriceProvider p = new AnchoredPriceProvider(
            address(this),
            oracle,
            baseFeedId,
            quoteFeedId,
            minMargin,
            maxRefStaleness,
            maxSpreadBps,
            mutableParams,
            marginStep,
            baseToken,
```
