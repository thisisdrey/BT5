### Title
`AnchoredPriceProvider` Has No L2 Variant — L1 Staleness Check Silently Halts All Swaps on L2 - (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

Every other price-provider type in the codebase ships an L1 contract **and** a matching L2 contract that tolerates oracle timestamps slightly ahead of `block.timestamp` (sequencer clock skew). `AnchoredPriceProvider` — described as "the one standard provider for public pools" — has **no L2 variant**. Its `_isStale` function is explicitly labelled `(L1)` and unconditionally rejects any `refTime > block.timestamp`. On L2 chains this causes every swap that uses an anchored provider to revert with `FeedStalled`, making the pool permanently unusable whenever the oracle's publication timestamp is even one second ahead of the sequencer's `block.timestamp`.

---

### Finding Description

The codebase maintains a deliberate L1/L2 split for all price-provider types:

| Contract | L1 | L2 |
|---|---|---|
| `PriceProvider` | ✓ | `PriceProviderL2` ✓ |
| `ProtectedPriceProvider` | ✓ | `ProtectedPriceProviderL2` ✓ |
| `AnchoredPriceProvider` | ✓ | **missing** ✗ |
| `AnchoredProviderFactory` | ✓ | **missing** ✗ |

`AnchoredPriceProvider._isStale` (3-parameter, L1 form):

```solidity
/// @dev Pure staleness check (L1). Any future refTime is stale.
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← unconditional rejection
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

This is called inside `_readLeg`, which is the only path that feeds `_getBidAndAskPrice`:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS))
    return (mid, spreadBps, refTime, false);   // ok = false → caller halts
``` [2](#0-1) 

When `ok = false`, `_getBidAndAskPrice` returns the stale sentinel `(0, type(uint128).max)`, and the public entry point reverts:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [3](#0-2) 

The L2-aware variant (`PriceProviderL2._isStale`, 4-parameter form) correctly handles sequencer clock skew by tolerating `refTime` up to `FUTURE_TOLERANCE` seconds ahead of `block.timestamp`:

```solidity
if (refTime > nowTs) {
    return (refTime - nowTs) > futureTol;   // tolerate within FUTURE_TOLERANCE
}
``` [4](#0-3) 

`FUTURE_TOLERANCE` is bounded to at most 1 hour at construction:

```solidity
if (_futureTolerance > 1 hours) revert FutureToleranceOutOfBounds();
FUTURE_TOLERANCE = _futureTolerance;
``` [5](#0-4) 

`AnchoredPriceProvider` has no equivalent parameter and no L2 factory. `AnchoredProviderFactory.createAnchoredProvider` always deploys the L1 contract:

```solidity
AnchoredPriceProvider p = new AnchoredPriceProvider(
    address(this), oracle, baseFeedId, quoteFeedId, ...
);
``` [6](#0-5) 

---

### Impact Explanation

On L2 chains (Arbitrum, Optimism), the sequencer's `block.timestamp` can lag behind the wall-clock time at which an oracle (e.g. Pyth) publishes a price update. The oracle's `refTime` therefore legitimately exceeds `block.timestamp` by seconds to minutes. The L1 `_isStale` check treats every such timestamp as stale, returning `ok = false` from `_readLeg`. The sentinel `(0, type(uint128).max)` propagates to `getBidAndAskPrice`, which reverts with `FeedStalled`. Every swap routed through a pool backed by an `AnchoredPriceProvider` is blocked for the duration of the clock skew — which on L2 can be persistent and recurring. This is broken core pool functionality causing unusable swap flows, matching the allowed impact gate.

---

### Likelihood Explanation

The `AnchoredPriceProvider` is explicitly described as "the one standard provider for public pools." The existence of `PriceProviderL2` and `ProtectedPriceProviderL2` confirms the protocol is intended for L2 deployment. On Arbitrum and Optimism, Pyth oracle `refTime` values ahead of `block.timestamp` are a documented, normal occurrence — exactly the condition `FUTURE_TOLERANCE` was introduced to handle in the other two provider families. Because `AnchoredProviderFactory` has no L2 counterpart, there is no compliant deployment path for anchored providers on L2; any deployment uses the L1 contract and inherits the broken staleness check.

---

### Recommendation

1. Implement `AnchoredPriceProviderL2` that adds an immutable `FUTURE_TOLERANCE` parameter and replaces the 3-parameter `_isStale` with the 4-parameter L2-aware version (identical to `PriceProviderL2._isStale`).
2. Implement `AnchoredProviderFactoryL2` that deploys `AnchoredPriceProviderL2` instances, accepting and forwarding `_futureTolerance` (bounded to `≤ 1 hours` at construction, matching the other L2 providers).
3. Validate `_futureTolerance` inside the envelope check so the parameter is subject to the same admin-curated bounds as `minMargin`, `maxRefStaleness`, and `maxSpreadBps`.

---

### Proof of Concept

1. Deploy `AnchoredProviderFactory` on Arbitrum (or any L2 where `block.timestamp` lags oracle publication time).
2. Call `createAnchoredProvider(...)` — this deploys an `AnchoredPriceProvider` (L1 contract).
3. Attach the provider to a `MetricOmmPool`.
4. At any moment when the Pyth oracle's `refTime` for the feed is 1 second ahead of `block.timestamp` (routine on Arbitrum), call `pool.swap(...)`.
5. The pool calls `provider.getBidAndAskPrice()`.
6. `_readLeg` calls `_isStale(refTime, block.timestamp, MAX_REF_STALENESS)`.
7. `refTime > block.timestamp` → `_isStale` returns `true` → `ok = false`.
8. `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
9. `getBidAndAskPrice` reverts with `FeedStalled`.
10. The swap reverts. All swaps on this pool are blocked for the duration of the clock skew.
