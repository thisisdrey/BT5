### Title
Missing Sequencer Restart Grace Period in `ProtectedPriceProviderL2` Allows Stale Pre-Downtime Price Execution — (`smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

---

### Summary

`ProtectedPriceProviderL2` contains no L2 sequencer uptime feed check and no post-restart grace period. Its only L2-specific addition over the L1 variant is `FUTURE_TOLERANCE` (tolerates oracle `refTime` slightly ahead of `block.timestamp` for clock skew). When the sequencer restarts after a downtime of up to `MAX_TIME_DELTA` seconds, oracle data published just before the outage passes the staleness check and is used to price a live swap — delivering a pre-downtime, economically stale bid/ask pair to the pool.

---

### Finding Description

`_computeBidAsk` in `ProtectedPriceProviderL2` delegates staleness entirely to `_isStale`:

```solidity
// ProtectedPriceProviderL2.sol L207
if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
    return (0, type(uint128).max);
}
```

`_isStale` is a pure age check:

```solidity
// ProtectedPriceProviderL2.sol L152
return (nowTs - refTime) > maxDelta;
``` [1](#0-0) 

There is no reference to a sequencer uptime feed, no `GRACE_PERIOD` constant, and no check that the sequencer has been live for a minimum period after restart. A grep across all production `.sol` files under `smart-contracts-poc/contracts/` returns zero matches for `sequencerUptimeFeed`, `GRACE_PERIOD`, or `ChainlinkVerifierL2`. The `ChainlinkVerifierL2` ABI that does carry those fields appears only in the registry artifact JSON, not as a source file integrated into `ProtectedPriceProviderL2`. [2](#0-1) 

**Attack path:**

1. The L2 sequencer goes offline. No transactions execute; oracle price updates stop being pushed to L2.
2. The last oracle update on-chain has `refTime = T_down` (published just before downtime).
3. The sequencer is down for `D` seconds where `D < MAX_TIME_DELTA`.
4. At restart (`block.timestamp = T_down + D`), the attacker immediately submits a swap.
5. The pool calls `getBidAndAskPrice()` → `_getBidAndAskPrice()` → `IPricedOracle.price(feedId, pool)` → `_computeBidAsk(mid, spread, refTime=T_down)`.
6. Staleness check: `(T_down + D) - T_down = D`. Since `D < MAX_TIME_DELTA`, the check passes.
7. The pre-downtime price is used to compute bid/ask and the swap executes. [3](#0-2) 

The attacker knows the current real market price (from L1, CEX, or cross-chain data) and can exploit the gap between the stale on-chain price and the true price at restart.

---

### Impact Explanation

This is **bad-price execution**: a stale oracle bid/ask pair — one that was valid before the sequencer went down but is economically stale at restart — reaches `pool.swap`. The pool's swap math executes against this price. LPs bear the loss: the attacker buys the underpriced asset (or sells the overpriced one) at the pre-downtime rate, extracting value that should have been repriced by the oracle after restart. The magnitude of loss scales with price movement during downtime and swap size.

---

### Likelihood Explanation

L2 sequencer outages are documented historical events on Arbitrum, Optimism, and Base. The attacker requires no privileged access — only the ability to submit the first swap transaction after restart, which is achievable by monitoring the sequencer status and submitting with a competitive gas price. The condition `D < MAX_TIME_DELTA` is easily satisfied for any downtime shorter than the configured staleness window (which can be up to 7 days per the constructor bound). [4](#0-3) 

---

### Recommendation

Add a sequencer uptime feed check (Chainlink `AggregatorV3Interface` sequencer uptime feed) and enforce a `GRACE_PERIOD` (e.g., 3600 seconds) after restart before allowing swaps. In `_computeBidAsk` (or a dedicated `_checkSequencer` helper), revert or return the stall sentinel `(0, type(uint128).max)` if the sequencer has been live for fewer than `GRACE_PERIOD` seconds. This is the standard pattern used by Chainlink's own L2 documentation and is already present in the `ChainlinkVerifierL2` artifact but absent from `ProtectedPriceProviderL2`.

---

### Proof of Concept

Foundry fork test outline:

```solidity
// 1. Deploy ProtectedPriceProviderL2 with MAX_TIME_DELTA = 1 hours.
// 2. Push oracle data at block.timestamp = T0 (refTime = T0).
// 3. vm.warp(T0 + 3599); // sequencer was "down" for 3599s < MAX_TIME_DELTA
// 4. Call pool.swap() → getBidAndAskPrice().
// 5. Assert: bid != 0 && ask != type(uint128).max  (swap proceeds with pre-downtime price).
// 6. Assert: the returned bid/ask matches the T0 oracle price, not any updated market price.
```

The staleness check `(3599 < 3600)` passes, confirming the pre-downtime price is accepted and the swap executes against economically stale data. [5](#0-4)

        uint256 refTime,
        uint256 nowTs,
        uint256 maxDelta,
        uint256 futureTol
    ) internal pure returns (bool) {
        if (refTime == 0) return true;

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol
            return (refTime - nowTs) > futureTol;
        }

        // refTime in the past or equal: check age
        return (nowTs - refTime) > maxDelta;
    }
