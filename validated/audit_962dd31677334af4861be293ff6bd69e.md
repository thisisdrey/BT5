### Title
`transferProviderOwnership()` Does Not Clear Stale Updater Grants, Allowing Revoked Addresses to Manipulate `confidenceParam` and Corrupt Bid/Ask Prices Reaching Pool Swaps - (`smart-contracts-poc/contracts/PriceProviderFactory.sol`, `PriceProviderFactoryL2.sol`, `AnchoredProviderFactory.sol`)

---

### Summary

`transferProviderOwnership()` in all three factory contracts updates `providerOwner` and the `_providersByCreator` sets but never clears the `isUpdater[provider][updater]` mapping. Any address previously granted updater rights by the old owner retains the ability to call `setConfidence()` after the transfer, directly setting `confidenceParam` on the live provider. For `PriceProvider` / `PriceProviderL2`, `confidenceParam` is the sole multiplier on the oracle spread before the bid/ask is computed; there is no downstream clamp. A stale updater can therefore push an arbitrarily tight (or zero-spread) quote into every pool swap that reads from the provider, causing LPs to give away value at near-mid prices.

---

### Finding Description

**Stale state in `isUpdater` after ownership transfer**

All three factories share the same pattern. In `PriceProviderFactory.sol`:

```solidity
// line 20
mapping(address provider => mapping(address updater => bool)) public isUpdater;

// lines 92-102
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);

    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
``` [1](#0-0) 

The function touches only `providerOwner` and `_providersByCreator`. The `isUpdater[provider][*]` mapping is never touched.

The identical omission exists in `PriceProviderFactoryL2.sol`: [2](#0-1) 

And in `AnchoredProviderFactory.sol`: [3](#0-2) 

**The updater gate that is bypassed**

`setConfidence()` checks `_requireUpdater`, which passes for any address where `isUpdater[provider][msg.sender] == true`:

```solidity
function _requireUpdater(address provider) internal view {
    if (msg.sender != providerOwner[provider] && !isUpdater[provider][msg.sender])
        revert NotProviderUpdater();
}
``` [4](#0-3) 

Because `isUpdater` is never cleared, a stale updater `X` (granted by the old owner) passes this check even after the provider has been transferred to a new owner who has no knowledge of `X`.

**How `confidenceParam` reaches the pool swap**

In `PriceProvider` / `PriceProviderL2`, `confidenceParam` is the direct multiplier on the oracle's raw spread:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [5](#0-4) 

`_getBidAskFrom` computes `delta = midPrice * adjustedSpread / CONFIDENCE_BASE`, so `bid = mid − delta`, `ask = mid + delta`. With `confidenceParam` set to a near-zero value by the stale updater, `delta → 0`, and the provider returns `bid ≈ ask ≈ mid` (only the `marginStep` spread remains). This quote is then consumed directly by the pool's swap math with no further clamp. [6](#0-5) 

---

### Impact Explanation

When `confidenceParam` is driven to a very low value (e.g., `1` instead of the intended `100_000`), the oracle confidence spread is reduced by a factor of 100,000. The pool executes every swap at near-mid prices with essentially zero spread. LPs earn no spread income and, in a volatile market, absorb adverse selection losses that the spread was designed to compensate. This is a direct, continuous loss of LP principal on every swap routed through the affected provider. The new owner has no mechanism to detect the stale updater without enumerating all addresses ever granted, and the stale updater can re-apply the manipulation after any cooldown window.

The `AnchoredPriceProvider` is partially protected: its clamp guarantees the final quote is never tighter than `mid ± (spreadBps + minMargin)`, so the stale-updater impact there is bounded to widening (not tightening) quotes. The unmitigated path is `PriceProvider` / `PriceProviderL2`. [7](#0-6) 

---

### Likelihood Explanation

The trigger requires three conditions, all of which are normal operational events:

1. A provider owner grants at least one updater (routine operational practice for multi-sig / bot setups).
2. The provider is transferred to a new owner (documented, permissionless operation).
3. The stale updater acts adversarially or negligently after the transfer.

The new owner has no on-chain way to enumerate all previously granted updaters (the `isUpdater` mapping is not iterable), so they cannot proactively revoke them. The old owner may not remember or may be adversarial. This matches the medium-severity profile of the reference bug: multiple plausible conditions, but not requiring a single-step exploit.

---

### Recommendation

Clear all updater grants for the provider being transferred inside `transferProviderOwnership()`. Because the `isUpdater` mapping is not enumerable as written, the fix requires either:

1. **Enumerable updater set**: Replace `mapping(address updater => bool)` with an `EnumerableSet.AddressSet` per provider, and iterate to delete all entries on transfer.
2. **Generation counter**: Add `mapping(address provider => uint256 generation)` incremented on every ownership transfer; store the generation at grant time and check it in `_requireUpdater`. Stale grants from a prior generation are automatically invalidated without enumeration.

Option 2 is gas-efficient and avoids unbounded loops.

### Proof of Concept

```
// State before transfer
Owner A creates provider P via PriceProviderFactory.createPriceProvider(...)
  → providerOwner[P] = A

A calls factory.grantUpdater(P, X)
  → isUpdater[P][X] = true

// Ownership transfer (does NOT clear isUpdater)
A calls factory.transferProviderOwnership(P, B)
  → providerOwner[P] = B
  → isUpdater[P][X] still == true   ← BUG

// Stale updater acts
X calls factory.setConfidence([P], [1])
  → _requireUpdater(P): isUpdater[P][X] == true → passes
  → PriceProvider(P).setConfidenceParam(1)

// Pool swap reads provider
Pool calls P.getBidAndAskPrice()
  → adjustedSpread = oracleSpread * 1 = oracleSpread (e.g. 300 bps → 300)
  → delta = mid * 300 / CONFIDENCE_BASE (CONFIDENCE_BASE = 1e10)
  → delta ≈ 0 for any realistic mid
  → bid ≈ mid, ask ≈ mid (only marginStep spread remains)
  → Pool executes swap at near-mid price
  → LPs absorb full adverse selection with zero spread compensation
``` [8](#0-7) [9](#0-8)
