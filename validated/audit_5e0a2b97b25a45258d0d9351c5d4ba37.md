### Title
L2 Price Providers Lack Sequencer Uptime Check, Allowing Stale Prices to Reach Pool Swaps During Grace Period - (`smart-contracts-poc/contracts/PriceProviderL2.sol`, `smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

---

### Summary

`PriceProviderL2` and `ProtectedPriceProviderL2` are explicitly designed for L2 deployment and include a `FUTURE_TOLERANCE` parameter to handle sequencer clock skew, but neither contract checks whether the L2 sequencer is currently down or in its post-restart grace period. A `ChainlinkVerifierL2` contract with `sequencerUptimeFeed` and `GRACE_PERIOD` fields exists in the deployed registry ABI but is not imported or called anywhere in the price provider flow.

---

### Finding Description

Both L2 price providers call `_getBidAndAskPrice()` → `_computeBidAsk()` during every pool swap. The only time-based guard is a staleness check against `MAX_TIME_DELTA`:

```solidity
// PriceProviderL2.sol _isStale()
if (refTime > nowTs) {
    return (refTime - nowTs) > futureTol;
}
return (nowTs - refTime) > maxDelta;
``` [1](#0-0) 

This check only catches staleness if the sequencer downtime exceeds `MAX_TIME_DELTA`. For short outages (e.g., 20 minutes with a `MAX_TIME_DELTA` of 1 hour), the pre-downtime price passes the staleness check.

More critically, when the Arbitrum sequencer restarts, there is a grace period during which old transactions queued before the outage are processed. During this window:

1. `block.timestamp` has advanced past the downtime.
2. Oracle keepers have not yet pushed updated reports.
3. The stored `refTime` is from before the outage.
4. `nowTs - refTime` < `MAX_TIME_DELTA` → staleness check passes.
5. The pre-downtime price is served to the pool swap.

The `ChainlinkVerifierL2` contract (present in the deployed registry with `sequencerUptimeFeed` and `GRACE_PERIOD`) is the intended guard for this scenario, but it is never called by either L2 provider:

```
grep: sequencerUptimeFeed|ChainlinkVerifierL2|GRACE_PERIOD
→ matches only in PriceProviderL2.sol and ProtectedPriceProviderL2.sol
→ those matches are only comments about "clock skew", not uptime feed calls
``` [2](#0-1) [3](#0-2) 

The `PriceProviderFactoryL2.createPriceProvider()` deploys `PriceProviderL2` with no sequencer uptime feed parameter, confirming the check is absent end-to-end: [4](#0-3) 

---

### Impact Explanation

During the grace period after sequencer restart, pool swaps execute at the last pre-downtime price. If the market moved significantly during the outage (e.g., a 5–20% move is common during Arbitrum outages), LPs suffer direct principal loss: traders buy underpriced tokens or sell overpriced tokens against LP reserves at the stale quote. The pool's bin accounting settles at the wrong price, and LPs cannot recover the difference.

---

### Likelihood Explanation

Arbitrum has experienced multiple sequencer outages. Any user who submitted a swap transaction before the outage (or immediately after restart before oracle keepers update) can exploit this without any special privilege. The `PriceProviderFactoryL2.createPriceProvider()` is permissionless, so any pool using an L2 provider is affected. Likelihood is medium: outages are infrequent but historically real, and the exploit requires only normal swap behavior timed around the restart.

---

### Recommendation

Integrate the existing `ChainlinkVerifierL2` sequencer uptime check into `_computeBidAsk()` in both L2 providers. Before the staleness check, call the sequencer uptime feed and revert (or return the stalled sentinel `(0, type(uint128).max)`) if the sequencer is down or within `GRACE_PERIOD` of restart:

```solidity
// pseudocode addition to _computeBidAsk()
(, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
if (answer == 1) return (0, type(uint128).max);                    // sequencer down
if (block.timestamp - startedAt < GRACE_PERIOD) return (0, type(uint128).max); // grace period
```

Pass `sequencerUptimeFeed` as an immutable to both L2 provider constructors and expose it as a parameter in `PriceProviderFactoryL2.createPriceProvider()`.

---

### Proof of Concept

1. Deploy `PriceProviderL2` via `PriceProviderFactoryL2` with `MAX_TIME_DELTA = 3600` (1 hour) on Arbitrum.
2. Oracle keeper pushes a report at `T=0` with price `P0`. `refTime = T=0`.
3. Arbitrum sequencer goes down at `T=600` (10 minutes). Market price moves from `P0` to `P0 * 1.10` during outage.
4. Sequencer restarts at `T=1800` (30 minutes of downtime). Grace period begins.
5. Attacker's pre-queued swap transaction executes: `block.timestamp = 1800`, `refTime = 0`, `nowTs - refTime = 1800 < MAX_TIME_DELTA = 3600` → staleness check passes.
6. `_computeBidAsk` returns bid/ask based on stale `P0`. Attacker buys base token at 10% below current market price.
7. LPs receive quote tokens valued at `P0` but the market has moved to `1.1 * P0` — direct LP loss of ~9% on the swapped notional. [5](#0-4) [6](#0-5)

    function _isStale(
        uint256 refTime,
        uint256 nowTs,
        uint256 maxDelta,
        uint256 futureTol
    ) internal pure returns (bool) {
        if (refTime == 0) return true;

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol
            return (refTime - nowTs) > futureTol;
        }

        // refTime in the past or equal: check age
        return (nowTs - refTime) > maxDelta;
    }

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
