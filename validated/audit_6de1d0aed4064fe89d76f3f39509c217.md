### Title
Pool admin can front-run swaps by instantly raising `adminSpreadFeeE6` / `adminNotionalFeeE8` to the maximum cap with no timelock — (`MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.setPoolAdminFees` lets the per-pool admin change both the spread fee and the notional fee to any value up to the factory-enforced cap in a single transaction, with no cooldown or timelock. Because the same fees are read atomically during every `MetricOmmPool.swap`, a pool admin can sandwich any pending swap: raise fees to the cap, let the swap execute at the inflated rate, then restore them — all without the trader's knowledge.

---

### Finding Description

`MetricOmmPoolFactory.setPoolAdminFees` is the only path for a pool admin to update `adminSpreadFeeE6` and `adminNotionalFeeE8`:

```solidity
// MetricOmmPoolFactory.sol  lines 408-435
function setPoolAdminFees(address pool, uint24 newAdminSpreadFeeE6, uint24 newAdminNotionalFeeE8)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
    ...
    IMetricOmmPoolFactoryActions(pool)
      .setPoolFees(c.protocolSpreadFeeE6 + c.adminSpreadFeeE6,
                   c.protocolNotionalFeeE8 + c.adminNotionalFeeE8);
``` [1](#0-0) 

There is **no timelock, no cooldown, and no minimum delay** between consecutive calls. The factory hard caps are:

```solidity
uint24 internal constant HARD_MAX_SPREAD_FEE_E6   = 200_000;  // 20 %
uint24 internal constant HARD_MAX_NOTIONAL_FEE_E8 = 1_000_000; // 1 %
``` [2](#0-1) 

Both fees are written directly into the pool's packed `Slot0` via `setPoolFees` and are read at the very start of every `swap` call:

```solidity
// MetricOmmPool.sol  lines 804-813
function _getBidAndAskPriceX64() internal returns (uint128 bidPriceX64, uint128 askPriceX64) { ... }
// then immediately:
(uint256 midPriceX64, uint256 baseFeeX64) =
    SwapMath.midAndSpreadFeeX64FromBidAsk(uint256(bidPriceX64), uint256(askPriceX64));
``` [3](#0-2) 

`spreadFeeE6` (which includes `adminSpreadFeeE6`) is forwarded into every bin-level swap helper:

```solidity
// MetricOmmPool.sol  lines 906-915
SwapMath.buyToken0InBinSpecifiedOut(
    binState, curPosInBinCache, state,
    params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6),
    lowerPriceX64, upperPriceX64, params.priceLimitX64,
    spreadFeeE6          // ← total spread fee, includes admin portion
);
``` [4](#0-3) 

`notionalFeeE8` is applied on the output (exact-in) or input (exact-out) token amount inside `_executeSwap`:

```solidity
// MetricOmmPool.sol  lines 756-761
uint256 notionalFeeScaled = uint256(-amount1DeltaScaled) * notionalFeeE8 / 1e8;
if (notionalFeeScaled > 0) {
    amount1DeltaScaled = amount1DeltaScaled + int256(notionalFeeScaled);
    notionalFeeToken1Scaled = ...;
}
``` [5](#0-4) 

The user-supplied `priceLimitX64` is compared against the **marginal bin price**, not the effective post-fee price, so it does not protect against a fee spike.

By contrast, the protocol already recognises the need for rate-limiting on sensitive parameters: `PriceProvider.setConfidenceParam` and `ProtectedPriceProvider.setConfidenceParam` both enforce a `CONFIDENCE_COOLDOWN = 1 minutes` between updates:

```solidity
// PriceProvider.sol  lines 97-99
if (block.timestamp < lastConfidenceUpdate + CONFIDENCE_COOLDOWN) {
    revert CooldownNotElapsed();
}
``` [6](#0-5) 

No equivalent guard exists for `setPoolAdminFees`.

---

### Impact Explanation

A malicious or compromised pool admin can:

1. Advertise low fees (`adminSpreadFeeE6 = 0`, `adminNotionalFeeE8 = 0`) to attract volume.
2. Observe a pending swap in the mempool.
3. Front-run it with `setPoolAdminFees(pool, 200_000, 1_000_000)` — raising the spread fee to 20 % and the notional fee to 1 %.
4. The victim's swap executes at the inflated rate; the admin collects the excess via `collectPoolFees`.
5. Immediately call `setPoolAdminFees(pool, 0, 0)` to restore the advertised rate.

The victim receives materially fewer output tokens than the oracle mid-price and their `priceLimitX64` would suggest, with no on-chain signal that fees changed. The stolen value flows to `poolAdminFeeDestination`, which the admin controls.

---

### Likelihood Explanation

- The pool admin role is per-pool and can be any EOA or contract — it is not the factory owner (which is out of scope as fully trusted).
- The attack requires only two cheap factory calls sandwiching the victim's swap; no special permissions beyond `poolAdmin` are needed.
- The attack is repeatable on every block with no cooldown.
- Pools that advertise low fees to attract retail flow are the highest-value targets.

---

### Recommendation

Introduce a minimum timelock between consecutive `setPoolAdminFees` calls, mirroring the `CONFIDENCE_COOLDOWN` already applied to `setConfidenceParam`. A timelock of at least several minutes (ideally matching the `priceProviderTimelock` pattern already used for provider changes) would prevent same-block front-running. Alternatively, emit a `FeeChangeProposed` event and enforce a mandatory delay before the new fee takes effect, consistent with the two-step `proposePoolPriceProvider` / `executePoolPriceProviderUpdate` pattern already present in the factory. [7](#0-6)

### Proof of Concept

```solidity
// Pseudocode — can be added to the existing Foundry test suite

function test_poolAdmin_frontRuns_swap_with_higher_fees() public {
    // Setup: pool deployed with adminSpreadFeeE6 = 0
    address pool = factory.createPool(params); // adminSpreadFeeE6 = 0

    // Victim approves and queues a swap (pending in mempool)

    // Pool admin front-runs: raise fees to maximum
    vm.prank(poolAdmin);
    factory.setPoolAdminFees(pool, 200_000, 1_000_000); // 20% spread, 1% notional

    // Victim's swap executes at inflated fees
    vm.prank(victim);
    (int128 d0, int128 d1) = pool.swap(
        victim, true, int128(amountIn), 0, callbackData, ""
    );
    // d1 is materially worse than at 0% admin fee

    // Pool admin resets fees
    vm.prank(poolAdmin);
    factory.setPoolAdminFees(pool, 0, 0);

    // Admin collects the excess
    factory.collectPoolFees(pool);
}
```

    nonReentrant

    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();

  function executePoolPriceProviderUpdate(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    address pending = pendingPriceProvider[pool];
    if (pending == address(0)) revert NoPriceProviderChangeProposed();
    uint256 execAfter = pendingPriceProviderExecuteAfter[pool];
    // forge-lint: disable-next-line(block-timestamp) -- timelock enforcement legitimately relies on `block.timestamp`.

    delete pendingPriceProviderExecuteAfter[pool];
    emit PoolPriceProviderUpdated(pool, pending);
