### Title
Stale updater access persists after provider ownership transfer, allowing unauthorized `confidenceParam` manipulation — (`smart-contracts-poc/contracts/PriceProviderFactory.sol` / `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`transferProviderOwnership` in both `PriceProviderFactory` and `AnchoredProviderFactory` transfers the `providerOwner` slot but never clears the `isUpdater[provider][updater]` mappings that the previous owner granted. Every updater the old owner registered retains the ability to call `setConfidence` on the provider indefinitely, bypassing the intended post-transfer access boundary.

---

### Finding Description

Both factories share the same ownership-transfer implementation:

```solidity
// PriceProviderFactory.sol  (AnchoredProviderFactory.sol is identical)
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;                          // owner updated
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);
    // ← isUpdater[provider][*] is NEVER cleared
    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
``` [1](#0-0) 

The gate that guards `setConfidence` is `_requireUpdater`:

```solidity
function _requireUpdater(address provider) internal view {
    if (msg.sender != providerOwner[provider] && !isUpdater[provider][msg.sender])
        revert NotProviderUpdater();
}
``` [2](#0-1) 

Because `isUpdater[provider][exUpdater]` is never set to `false` during transfer, the second branch of the `&&` remains `true` for every updater the previous owner granted. The new owner has no way to discover which addresses are stale updaters without off-chain event scanning, and the new owner must manually call `revokeUpdater` for each one — a step that is easy to miss and not enforced by the protocol. [3](#0-2) [4](#0-3) 

**Attack path (concrete):**

1. Alice deploys provider `P` via `createPriceProvider` / `createAnchoredProvider`.
2. Alice calls `grantUpdater(P, Bob)` → `isUpdater[P][Bob] = true`.
3. Alice calls `transferProviderOwnership(P, Carol)` → `providerOwner[P] = Carol`, but `isUpdater[P][Bob]` is **not** cleared.
4. Carol is unaware of Bob's stale entry.
5. Bob calls `factory.setConfidence([P], [value])`. `_requireUpdater` passes because `isUpdater[P][Bob]` is still `true`. The call reaches `PriceProvider.setConfidenceParam(value)` (gated only by `OnlyFactory`, which is satisfied). [5](#0-4) [6](#0-5) 

---

### Impact Explanation

**`PriceProvider` / `ProtectedPriceProvider` (no reference-band clamp):**

`confidenceParam` directly shapes the bid/ask delivered to the pool:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [7](#0-6) 

- **Confidence → 0, `marginStep` > 0**: `adjustedSpread = 0`, so `bid = ask = mid`. After the step factors, `bidOut = mid·stepBidFactor/BPS_BASE` and `askOut = mid·stepAskFactor/BPS_BASE`. The spread collapses to the `marginStep` bias alone — far tighter than the oracle's actual uncertainty. Every swap executes at near-mid price; the pool earns near-zero spread fee. Protocol and LP fee revenue is drained on every swap for as long as the stale updater holds this setting.
- **Confidence → `CONFIDENCE_MAX` (1 000 000)**: `delta ≥ mid` for any non-trivial oracle spread, so `bid = 0` → `_applyBidAdjustments` returns `(0, false)` → provider returns the halt sentinel → pool reverts `PriceProviderFailed` on every swap (DoS).

**`AnchoredPriceProvider` (reference-band clamp present):**

The shaped quote is clamped to `[refBid, refAsk]` before reaching the pool, so the final bid/ask cannot be tighter than the audited band regardless of `confidenceParam`. Impact here is bounded and does not produce a bad price. [8](#0-7) 

The high-severity path is therefore through `PriceProvider` / `ProtectedPriceProvider` pools where the stale updater can silently collapse the spread to near-zero, causing the pool to execute all swaps at mid price and collect no spread fee — a continuous, silent loss of owed LP and protocol fee revenue.

---

### Likelihood Explanation

- Ownership transfer is a documented, expected lifecycle event.
- The previous owner may have granted multiple updaters over time; none are cleared automatically.
- The new owner has no on-chain enumeration of stale updaters and must rely on off-chain event logs to discover them.
- The 1-minute `CONFIDENCE_COOLDOWN` limits update frequency but does not prevent the attack; a single update to `confidenceParam = 0` (with `marginStep > 0`) persists until the new owner notices and revokes the stale updater.
- No special precondition beyond a normal ownership transfer is required.

---

### Recommendation

Clear all updater entries for the provider at transfer time, or require the new owner to re-grant updaters explicitly. The simplest safe fix is to track updaters per provider in an enumerable set and iterate over them during `transferProviderOwnership`:

```solidity
// In transferProviderOwnership, after updating providerOwner:
EnumerableSet.AddressSet storage updaters = _updatersByProvider[provider];
uint256 len = updaters.length();
for (uint256 i = len; i > 0; ) {
    --i;
    address u = updaters.at(i);
    isUpdater[provider][u] = false;
    updaters.remove(u);
    emit UpdaterRevoked(provider, u);
}
```

Alternatively, add a `providerOwnerVersion[provider]` nonce incremented on every transfer and gate `isUpdater` checks on `isUpdater[provider][version][updater]`, so all prior grants are invalidated atomically without iteration.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Assume PriceProviderFactory is deployed at `factory`,
// a PriceProvider with marginStep > 0 at `provider`.

// Step 1 – Alice grants Bob as updater
factory.grantUpdater(provider, bob);          // isUpdater[provider][bob] = true

// Step 2 – Alice transfers ownership to Carol
factory.transferProviderOwnership(provider, carol);

address[] memory ps = new address[](1); ps[0] = provider;
uint256[] memory vs = new uint256[](1); vs[0] = 0;  // confidence = 0
factory.setConfidence(ps, vs);   // _requireUpdater passes; confidenceParam set to 0

// Step 4 – Every subsequent pool swap executes at near-mid price,
//           earning near-zero spread fee for LPs and protocol.
