### Title
Stale updater permissions persist after `transferProviderOwnership`, allowing unauthorized `confidenceParam` manipulation that corrupts bid/ask prices — (`smart-contracts-poc/contracts/PriceProviderFactory.sol`, `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

### Summary
When provider ownership is transferred via `transferProviderOwnership`, the `isUpdater[provider][updater]` mapping entries granted by the old owner are never cleared. Stale updaters retain the ability to call `setConfidence`, manipulating `confidenceParam` to corrupt the bid/ask spread computation used in every pool swap.

### Finding Description

`PriceProviderFactory.transferProviderOwnership` updates `providerOwner[provider]` to the new owner and moves the provider between `_providersByCreator` sets, but does **not** clear any `isUpdater[provider][*]` entries that the old owner previously granted: [1](#0-0) 

The updater authorization check used by `setConfidence` is: [2](#0-1) 

Because `isUpdater[provider][oldUpdater]` is never deleted, any address the old owner granted via `grantUpdater` passes this check indefinitely after ownership transfer. The `setConfidence` batch path confirms the provider is still tracked and then calls through: [3](#0-2) 

`confidenceParam` is the direct multiplier on the oracle spread inside `PriceProvider._getBidAndAskPrice`: [4](#0-3) 

The identical structural defect exists in `AnchoredProviderFactory.transferProviderOwnership`: [5](#0-4) 

This is the direct analog of the external bug: just as `getOperatorIdForAddress[oldAddress]` was never deleted, `isUpdater[provider][oldUpdater]` is never deleted, leaving the old binding active after the controlling party changes.

### Impact Explanation

**Path A — DoS (unusable swap flows):** A stale updater calls `setConfidence([P], [0])`. With `confidenceParam = 0`, `adjustedSpread = 0`, so `bid = ask = mid`. After the `_applyBidAdjustments` / `_applyAskAdjustments` step, when `marginStep ≤ 0` the floor-rounded bid equals or exceeds the ceil-rounded ask, triggering the hard invariant guard: [6](#0-5) 

The provider returns `(0, type(uint128).max)`, `getBidAndAskPrice` reverts with `FeedStalled`, and every swap on every pool using this provider fails. The 1-minute cooldown limits frequency but does not prevent the attack — the stale updater can re-trigger it every minute indefinitely.

**Path B — Bad-price execution (unbounded spread):** A stale updater calls `setConfidence([P], [CONFIDENCE_MAX])` (1 000 000). With a typical oracle spread of 100 bps, `adjustedSpread = 1e8`, producing `delta = mid / 100`. The resulting bid/ask reaches the pool swap: [7](#0-6) 

`SwapMath.midAndSpreadFeeX64FromBidAsk` derives a `baseFeeX64` from this inflated spread, charging traders up to 100× the intended spread fee on every swap — a direct loss of trader principal that accrues as excess LP/protocol fees.

### Likelihood Explanation

The trigger is semi-trusted: the old owner must have granted updater rights to an address that later acts adversarially. This is realistic in any ownership-transfer scenario (e.g., a protocol sells or reassigns a provider, or an internal key is compromised). The new owner has no on-chain way to enumerate all stale `isUpdater` entries before accepting ownership, and no automatic revocation occurs. The attack requires no privileged role after the transfer — the stale updater is an ordinary EOA or contract.

### Recommendation

In `transferProviderOwnership` (both factories), enumerate and delete all `isUpdater[provider][*]` entries before updating `providerOwner`. Because Solidity mappings are not iterable, the factories should maintain a per-provider updater enumerable set (analogous to `_providersByCreator`) so that all grants can be revoked atomically on transfer. Alternatively, key updater authorization on `providerOwner` at the time of the grant (e.g., store a `grantedByOwner` epoch) and invalidate grants from prior owners automatically.

### Proof of Concept

```
1. Owner A deploys provider P via createPriceProvider (PriceProviderFactory).
   → providerOwner[P] = A, isUpdater[P][*] = {}

2. Owner A grants updater rights to adversarial address U:
   grantUpdater(P, U)
   → isUpdater[P][U] = true

3. Owner A transfers ownership to new owner B:
   transferProviderOwnership(P, B)
   → providerOwner[P] = B
   → isUpdater[P][U] = true  ← NOT cleared (root cause)

4. U calls setConfidence([P], [0]):
   _requireUpdater(P):
     msg.sender (U) != providerOwner[P] (B)  → check isUpdater[P][U] → true → passes
   PriceProvider(P).setConfidenceParam(0) succeeds.

5. Next swap on any pool using P:
   _getBidAndAskPrice():
     adjustedSpread = spread * 0 = 0
     bid = ask = mid
     bidOut >= askOut  (when marginStep <= 0)
     → return (0, type(uint128).max)
   getBidAndAskPrice() reverts FeedStalled
   Pool.swap() reverts PriceProviderFailed
   → all swaps on the pool are bricked until B discovers and revokes U.
``` [1](#0-0) [8](#0-7) [5](#0-4)

        address[] calldata providers,
        uint256[] calldata values
    ) external override {
        uint256 l = providers.length;
        if (l != values.length) revert LengthMismatch();

        for (uint256 i; i < l; ++i) {
            require(_providers.contains(providers[i]), ProviderNotTracked());
            _requireUpdater(providers[i]);
            PriceProvider(providers[i]).setConfidenceParam(values[i]);

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
```
