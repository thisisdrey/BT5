### Title
Missing L2 Sequencer Uptime Feed Check Allows Stale Pre-Downtime Prices to Execute Swaps — (`smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

---

### Summary

`ProtectedPriceProviderL2` is explicitly designed for L2 deployments but contains no check against a Chainlink L2 sequencer uptime feed. Its only temporal guard is `_isStale()`, which compares the oracle's `refTime` against `block.timestamp`. When the sequencer restarts after a downtime period shorter than `MAX_TIME_DELTA`, the pre-downtime oracle price passes the staleness check and reaches the pool swap unchanged.

---

### Finding Description

The full price read path is:

```
pool.swap()
  → ProtectedPriceProviderL2.getBidAndAskPrice()
      → _getBidAndAskPrice()
          → IPricedOracle(offchainOracle).price(offchainFeedId, msg.sender)
              returns (mid, spread, spread1, refTime)
      → _computeBidAsk(mid, spread, refTime)
          → _isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)
```

`_computeBidAsk` applies five guards in sequence: [1](#0-0) 

The only temporal guard is: [2](#0-1) 

`_isStale` is a pure arithmetic check — it has no awareness of whether the sequencer was down: [3](#0-2) 

There is no sequencer uptime feed address, no `startedAt` check, and no grace period anywhere in the contract. The two occurrences of "sequencer" in the file are comments only (lines 40–41), not logic: [4](#0-3) 

The constructor accepts `MAX_TIME_DELTA` in the range `[1, 7 days]` with no further constraint: [5](#0-4) 

**Attack scenario (concrete):**

| Time | Event |
|------|-------|
| T=0 | Sequencer goes down. Oracle `refTime = T=0`, price = P_old (pre-downtime). |
| T=0 to T=D | Market moves 10–20% (e.g., ETH drops). Oracle cannot be updated on-chain. |
| T=D | Sequencer restarts. `block.timestamp = D`. |
| T=D | Attacker calls `pool.swap()`. `_isStale` evaluates `(D - 0) > MAX_TIME_DELTA`. If `D < MAX_TIME_DELTA`, returns `false`. |
| T=D | `_computeBidAsk` proceeds with P_old. Bid/ask Q64 values reflect pre-downtime price. |
| T=D | Attacker buys the underpriced token at P_old, immediately sells at true market price P_new. LPs absorb the loss. |

The `_isStale` check is structurally insufficient because it cannot distinguish "oracle not updated because no price change" from "oracle not updated because the sequencer was down." The standard fix — checking the Chainlink L2 sequencer uptime feed and enforcing a grace period after restart — is entirely absent.

This is directly called out as a required protection in the audit's own Smart Audit Pivots: *"Protected/provider path: price update gates, maxTimeDelta/maxRefStaleness, **L2 sequencer-down checks**, and factory-created provider assumptions must block stale or bad-price execution."*

---

### Impact Explanation

Stale pre-downtime bid/ask Q64 values reach `pool.swap()`. An arbitrageur buys the underpriced token at the stale price and sells at the true post-downtime market price. LP token balances are drained by the spread between P_old and P_new. This is a direct loss of LP principal — a High/Critical impact under the contest's "bad-price execution" and "pool insolvency" gates.

---

### Likelihood Explanation

L2 sequencer downtime is a documented real-world event (Arbitrum experienced ~45 minutes of downtime in December 2022; Optimism has had similar incidents). The attack requires only that:
1. The sequencer is down for a period shorter than `MAX_TIME_DELTA` (which can be up to 7 days).
2. The market price moves during downtime.
3. The attacker is the first to call `pool.swap()` after restart.

All three conditions are realistic and require no privileged access. The attacker is an unprivileged trader calling the public `swap()` entrypoint.

---

### Recommendation

Add a Chainlink L2 sequencer uptime feed check with a grace period to `_computeBidAsk` (or a dedicated internal function called before it):

```solidity
// Immutables to add to constructor:
AggregatorV2V3Interface public immutable sequencerUptimeFeed;
uint256 public immutable SEQUENCER_GRACE_PERIOD; // e.g., 3600 (1 hour)

function _checkSequencer() internal view {
    (, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
    // answer == 0 means sequencer is up; 1 means down
    if (answer != 0) revert SequencerDown();
    // Enforce grace period after restart
    if (block.timestamp - startedAt < SEQUENCER_GRACE_PERIOD) revert GracePeriodNotElapsed();
}
```

Call `_checkSequencer()` at the top of `_computeBidAsk` before the staleness check. This closes the window entirely: any swap attempted while the sequencer is down or within the grace period after restart will revert.

function test_staleSwapAfterSequencerRestart() public {

        // refTime in the past or equal: check age

    function _computeBidAsk(uint256 price, uint256 spread, uint256 refTime)
        internal view returns (uint128, uint128)
    {
        // 1. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
            return (0, type(uint128).max);
        }

        // 2. Basic validity — price must be positive, spread must not be stalled marker
        if (price == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 3. Price guard check
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (price < guardMin || price > guardMax) {
            return (0, type(uint128).max);
        }

        // 4. Compute bid/ask from mid + confidence-adjusted spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(price, adjustedSpread);

        // 5. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 6. Hard invariant: bid must be strictly less than ask.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
