### Title
`AnchoredPriceProvider` Uses L1-Only Staleness Check With No L2 Variant, Causing Permanent `FeedStalled` Revert on L2 — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` — described as "the one standard provider for public pools" — implements an L1-only staleness check that unconditionally rejects any oracle `refTime` ahead of `block.timestamp`. On L2 networks (Arbitrum, Optimism), oracle timestamps (especially Pyth) routinely arrive slightly ahead of the sequencer's `block.timestamp` due to clock skew. The protocol already recognized this and created `PriceProviderL2` and `ProtectedPriceProviderL2` with a `FUTURE_TOLERANCE` window, but **no L2 variant of `AnchoredPriceProvider` exists**. If the `AnchoredProviderFactory` is deployed on L2, every swap through any public pool using this provider will permanently revert with `FeedStalled()`.

---

### Finding Description

`AnchoredPriceProvider._isStale()` is explicitly annotated as an L1 check:

```solidity
/// @dev Pure staleness check (L1). Any future refTime is stale.
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← hard rejection of any future timestamp
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

This is called from `_readLeg()`, which returns `ok=false` whenever `_isStale()` is true: [2](#0-1) 

`_getBidAndAskPrice()` propagates `ok=false` as the `(0, type(uint128).max)` sentinel: [3](#0-2) 

And `getBidAndAskPrice()` converts that sentinel into a hard revert: [4](#0-3) 

The L2-aware counterparts (`PriceProviderL2`, `ProtectedPriceProviderL2`) solve this with an immutable `FUTURE_TOLERANCE` parameter that allows `refTime` to be up to N seconds ahead of `block.timestamp`: [5](#0-4) 

`AnchoredPriceProvider` has no such parameter and no L2 variant. The `AnchoredProviderFactory.createAnchoredProvider()` is permissionless and contains no chain-ID guard: [6](#0-5) 

---

### Impact Explanation

Every swap through a public pool whose provider is an `AnchoredPriceProvider` deployed on L2 will revert with `FeedStalled()` whenever the oracle's `refTime` exceeds `block.timestamp` by even 1 second. On Pyth-based L2 deployments this is a routine condition, not an edge case. The result is a complete, persistent DoS of all swaps and liquidity operations for every pool using this provider — broken core pool functionality with no recovery path short of migrating to a new provider.

---

### Likelihood Explanation

The protocol explicitly targets L2 deployment (evidenced by `PriceProviderL2`, `ProtectedPriceProviderL2`, and `PriceProviderFactoryL2`). `AnchoredPriceProvider` is the designated standard provider for public pools. `AnchoredProviderFactory.createAnchoredProvider()` is permissionless with no chain-ID restriction. On Arbitrum and Optimism, Pyth oracle `publishTime` values routinely exceed `block.timestamp` by seconds. The combination makes this a near-certain failure mode whenever the factory is deployed on L2.

---

### Recommendation

Introduce `AnchoredPriceProviderL2` (mirroring the pattern already applied to `PriceProviderL2` and `ProtectedPriceProviderL2`) with an immutable `FUTURE_TOLERANCE` parameter (bounded to ≤ 1 hour, matching the existing L2 variants) and an L2-aware `_isStale()`:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Create a corresponding `AnchoredProviderFactoryL2` that deploys `AnchoredPriceProviderL2` instances and enforces the `FUTURE_TOLERANCE` bound at construction, consistent with `PriceProviderFactoryL2`. [7](#0-6) 

---

### Proof of Concept

1. Deploy `AnchoredProviderFactory` on Arbitrum.
2. Call `createAnchoredProvider(pythOracle, btcFeedId, 0, minMargin, staleness, maxSpread, false, 0, WBTC, USDC)`.
3. Assign the resulting `AnchoredPriceProvider` to a public pool.
4. At any moment when Pyth's `publishTime` for the BTC feed is 1 second ahead of `block.timestamp` (routine on Arbitrum), call `pool.swap(...)`.
5. The pool calls `provider.getBidAndAskPrice()` → `_getBidAndAskPrice()` → `_readLeg(baseFeedId)` → `_isStale(refTime=T+1, nowTs=T, ...)` returns `true` → `ok=false` → returns `(0, type(uint128).max)` → `getBidAndAskPrice()` reverts `FeedStalled()`.
6. Every swap reverts. The pool is permanently unusable until the provider is replaced.

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }

        if (refTime == 0) return true;

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol

        }

        // refTime in the past or equal: check age
