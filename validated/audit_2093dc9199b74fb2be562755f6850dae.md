### Title
Identical `baseFeedId` and `quoteFeedId` in Synthetic-Ratio Mode Produces a Constant Wrong Price of 1.0 — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`, `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`AnchoredPriceProvider._getBidAndAskPrice()` computes a synthetic ratio price as `mid1 / mid2` and adds both legs' spreads. Neither the provider constructor nor `AnchoredProviderFactory.createAnchoredProvider` validates that `baseFeedId != quoteFeedId`. When both are identical, the ratio collapses to the constant `ORACLE_DECIMALS` (= `1e8` = 1.0 in 8-decimal format) and the spread is doubled — a permanently wrong price that reaches every pool swap.

---

### Finding Description

In `AnchoredPriceProvider._getBidAndAskPrice()`, the synthetic-ratio branch reads both legs independently and combines them:

```solidity
(uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
...
if (_quote != bytes32(0)) {
    (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
    if (!ok2 || mid2 == 0) return (0, type(uint128).max);
    mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // ratio
    spreadBps += spreadBps2;                          // uncertainties add
}
``` [1](#0-0) 

When `baseFeedId == quoteFeedId`, both `_readLeg` calls hit the same oracle feed and return identical values (`mid2 == mid`, `spreadBps2 == spreadBps`). The ratio becomes:

```
mid = mulDiv(mid, ORACLE_DECIMALS, mid) = ORACLE_DECIMALS = 1e8
```

This is the constant `1.0` in 8-decimal format — completely independent of the actual market price. The spread is simultaneously doubled (`spreadBps += spreadBps`), widening the band around the wrong anchor.

The `AnchoredPriceProvider` constructor stores both feed IDs without any equality check:

```solidity
baseFeedId = _baseFeedId;
quoteFeedId = _quoteFeedId;
``` [2](#0-1) 

The only token-level guard present is `_baseToken != _quoteToken`; there is no analogous `_baseFeedId != _quoteFeedId` guard:

```solidity
require(_baseToken != address(0) && _quoteToken != address(0) && _baseToken != _quoteToken);
``` [3](#0-2) 

`AnchoredProviderFactory.createAnchoredProvider` is permissionless and performs envelope validation only on `minMargin`, `maxRefStaleness`, and `maxSpreadBps`. It does not check `baseFeedId != quoteFeedId`:

```solidity
if (
    minMargin < env.minMarginMin || minMargin > env.minMarginMax
    || maxRefStaleness < env.stalenessMin || maxRefStaleness > env.stalenessMax
    || maxSpreadBps < env.maxSpreadMin || maxSpreadBps > env.maxSpreadMax
) revert ParamsOutOfEnvelope();
``` [4](#0-3) 

After deployment the provider is unconditionally registered:

```solidity
_providers.add(provider);
providerOwner[provider] = creator;
``` [5](#0-4) 

making `isProvider(provider) == true` — the public-pool eligibility predicate — for a provider that will always quote price 1.0.

The downstream `_computeBidAsk` receives `mid = 1e8` and computes a valid (non-stalling) bid/ask band around 1.0, so `getBidAndAskPrice()` returns without reverting:

```solidity
uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
    return (0, type(uint128).max);
}
``` [6](#0-5) 

The wrong price propagates silently to every pool swap.

---

### Impact Explanation

For any pair whose true price deviates from 1.0 (e.g., ETH/USD at $3 000, BTC/ETH at 15), the pool quotes a price that is off by the full market ratio. Arbitrageurs can buy the underpriced asset at price 1.0 and sell it at market, draining the pool's reserves in a single block. LP principal is lost in full. This is a direct, contest-relevant loss of user funds matching the "bad-price execution" and "pool insolvency" impact categories.

---

### Likelihood Explanation

`createAnchoredProvider` is permissionless. Any caller can supply `baseFeedId == quoteFeedId` — either by accident (copy-paste of the same feed ID into both arguments) or deliberately. The resulting provider passes `isProvider()` and can be attached to a public pool without further factory-level rejection. No privileged action is required after deployment.

---

### Recommendation

Add a feed-ID equality guard in both the factory and the constructor:

In `AnchoredProviderFactory.createAnchoredProvider`, before deploying:
```solidity
if (quoteFeedId != bytes32(0) && quoteFeedId == baseFeedId)
    revert SameFeedId();
```

In `AnchoredPriceProvider` constructor, after storing the feed IDs:
```solidity
require(
    _quoteFeedId == bytes32(0) || _quoteFeedId != _baseFeedId,
    "baseFeedId == quoteFeedId"
);
```

This mirrors the existing `_baseToken != _quoteToken` guard and closes the gap in the factory's audit-once guarantee.

---

### Proof of Concept

1. Admin has configured an oracle and a DEFAULT_CLASS envelope in `AnchoredProviderFactory`.
2. Attacker (or mistaken user) calls:
   ```solidity
   factory.createAnchoredProvider(
       oracle,
       ETH_USD_FEED_ID,   // baseFeedId
       ETH_USD_FEED_ID,   // quoteFeedId — same feed
       minMargin, maxRefStaleness, maxSpreadBps,
       false, 0,
       WETH, USDC
   );
   ```
3. Factory validates envelope bounds (pass), deploys `AnchoredPriceProvider`, and records `isProvider(p) == true`.
4. A pool is created with provider `p`. At swap time the pool calls `p.getBidAndAskPrice()`.
5. Inside `_getBidAndAskPrice`:
   - `_readLeg(ETH_USD_FEED_ID)` → `mid = 300000000000` (ETH at $3 000, 8-dec), `spreadBps = 5`
   - `_readLeg(ETH_USD_FEED_ID)` again → `mid2 = 300000000000`, `spreadBps2 = 5`
   - `mid = mulDiv(300000000000, 1e8, 300000000000)` = **`1e8`** (= 1.0)
   - `spreadBps = 10`
6. `_computeBidAsk(1e8, 10)` returns a valid bid/ask around price 1.0 in Q64.
7. The pool executes swaps at price 1.0 instead of $3 000. An arbitrageur buys 1 ETH for 1 USDC and sells it on Uniswap for $3 000, repeating until the pool is drained.
