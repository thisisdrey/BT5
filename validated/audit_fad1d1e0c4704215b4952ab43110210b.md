### Title
Missing L2 Sequencer Uptime Circuit Breaker in `ProtectedPriceProviderL2` Allows Stale-Price Swap Execution After Sequencer Restart — (`smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

---

### Summary

`ProtectedPriceProviderL2` is deployed on L2 chains (Arbitrum, Base, Avalanche, etc.) but contains **no sequencer uptime feed check**. Its only L2-specific addition over the L1 variant is `FUTURE_TOLERANCE` — a tolerance for oracle `refTime` being slightly ahead of `block.timestamp` due to clock skew. There is no grace-period gate after a sequencer restart. As a result, immediately after the sequencer comes back online, a swap can execute against a price that was published before the downtime, bypassing any staleness rejection as long as the downtime duration `S < MAX_TIME_DELTA`.

---

### Finding Description

The staleness check in `_computeBidAsk` is:

```solidity
// ProtectedPriceProviderL2.sol line 207
if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
    return (0, type(uint128).max);
}
```

And `_isStale` is:

```solidity
// lines 138-153
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

**Scenario:**

| Variable | Value |
|---|---|
| `T_push` | Price published to oracle at time T |
| Sequencer down | Duration `S` seconds, `S < MAX_TIME_DELTA` |
| `T_resume` | `T_push + S` — sequencer restarts |
| `block.timestamp` at first post-restart swap | `T_push + S` |
| `_isStale(T_push, T_push + S, MAX_TIME_DELTA, ...)` | `S > MAX_TIME_DELTA` → **false** |

The price from `T_push` passes the staleness check. No other guard in `_computeBidAsk` (price guard, spread check, bid/ask inversion) detects that the sequencer was recently down. The price flows through to `getBidAndAskPrice()` and into the pool swap.

The contract has **no** `sequencerUptimeFeed` variable, no `GRACE_PERIOD` constant, and no uptime check anywhere in the call path. The registry ABI confirms a separate `ChainlinkVerifierL2` contract exists with `sequencerUptimeFeed` and `GRACE_PERIOD`, but `ProtectedPriceProviderL2` does not integrate it. ---

### Impact Explanation

After a sequencer restart, the first swap(s) execute against the pre-downtime price. During downtime, the market can move arbitrarily (no on-chain activity, but off-chain prices move). An arbitrageur who monitors the sequencer restart can immediately submit a swap at the stale price, extracting value from LPs. This is **bad-price execution** leading to direct LP principal loss. The magnitude scales with price movement during downtime and pool TVL.

`MAX_TIME_DELTA` is bounded only by `> 0` and `<= 7 days` at construction — a deployer setting even a moderate value (e.g., 30 minutes) creates a 29-minute window of exploitable stale prices after a 1-minute downtime. [4](#0-3) 

---

### Likelihood Explanation

L2 sequencer outages are documented historical events on Arbitrum, Base, and Optimism. The attack requires no privileged access: any public swap caller can submit a transaction immediately after sequencer restart. The attacker only needs to monitor the sequencer status feed off-chain (publicly available) and front-run the first oracle price update.

---

### Recommendation

Add a sequencer uptime feed check (Chainlink's `AggregatorV3Interface` sequencer uptime feed) with a `GRACE_PERIOD` (e.g., 1 hour) to `ProtectedPriceProviderL2`. In `_computeBidAsk` (or `getBidAndAskPrice`), revert or return the stalled sentinel `(0, type(uint128).max)` if the sequencer is down or if `block.timestamp - sequencerRestartedAt < GRACE_PERIOD`. The `ChainlinkVerifierL2` contract already in the registry implements this pattern and should be integrated.

// Foundry fork test sketch
function test_stalePrice_afterSequencerRestart() public {
    uint256 T_push = block.timestamp;
    oracle.setData(FEED_ID, 100_000_000, 300, 0, T_push); // price at T_push

    // Simulate sequencer downtime: warp S < MAX_TIME_DELTA

    vm.warp(T_push + S);

    // No new oracle price pushed (sequencer was down)
    // Swap executes immediately after restart
    (uint128 bid, uint128 ask) = poolContract.getBidAndAskPrice();

    // Succeeds with the pre-downtime price — no sequencer circuit breaker
    assertGt(bid, 0);
    assertLt(bid, ask);
    // Market may have moved 5-20% during downtime; LP is exposed
}
