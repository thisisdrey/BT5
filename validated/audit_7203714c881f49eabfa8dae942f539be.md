### Title
Missing L2 Sequencer Uptime Feed Check in `PriceProviderL2` and `ProtectedPriceProviderL2` Allows Stale-Price Swap Execution After Sequencer Recovery — (`smart-contracts-poc/contracts/PriceProviderL2.sol`, `smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

---

### Summary

`PriceProviderL2` and `ProtectedPriceProviderL2` are explicitly designed for L2 deployment (they carry a `FUTURE_TOLERANCE` immutable to handle sequencer clock skew) but contain **no Chainlink L2 sequencer uptime feed check**. After an L2 sequencer outage, the first blocks produced on recovery carry a `block.timestamp` that has jumped forward to real time, while the last oracle price pushed before the outage remains in the Metric oracle store. If the outage lasted less than `MAX_TIME_DELTA`, that pre-downtime price passes the staleness check and pool swaps execute at it — at a price that may be arbitrarily far from the current market.

---

### Finding Description

Both L2 provider variants implement `_isStale()` as a pure time-delta check: [1](#0-0) 

```
if (refTime == 0) return true;
if (refTime > nowTs) { return (refTime - nowTs) > futureTol; }
return (nowTs - refTime) > maxDelta;
```

`MAX_TIME_DELTA` is configurable up to 7 days: [2](#0-1) 

There is no call to a Chainlink sequencer uptime feed, no grace-period enforcement after sequencer recovery, and no other mechanism that detects sequencer downtime. The `FUTURE_TOLERANCE` immutable only handles clock skew — it does not address sequencer liveness: [3](#0-2) 

The same gap exists identically in `ProtectedPriceProviderL2`: [4](#0-3) 

The full swap-time price read path is:

```
pool.swap()
  → priceProvider.getBidAndAskPrice()
    → _getBidAndAskPrice()
      → IPricedOracle.price(feedId, pool)   // returns last pushed price + refTime
      → _isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)
      // NO sequencer uptime check here
``` [5](#0-4) 

---

### Impact Explanation

When an L2 sequencer (Arbitrum, Optimism) goes down:

1. No new L2 blocks are produced; no oracle updates can be pushed.
2. The last oracle price before the outage remains in the Metric oracle store with its original `refTime`.
3. When the sequencer recovers, `block.timestamp` jumps to current real time. If the outage lasted `T < MAX_TIME_DELTA`, the condition `(nowTs - refTime) > maxDelta` is **false** — the stale price passes.
4. An attacker immediately calls `pool.swap()` before the oracle updater pushes a fresh price.
5. The swap executes at the pre-downtime price. If the real-world price moved significantly during the outage (common during volatile events that cause sequencer stress), the attacker extracts the difference from LP reserves.

This is **bad-price execution** causing **direct LP fund loss** — a qualifying impact under the contest rules.

---

### Likelihood Explanation

- Arbitrum and Optimism have documented sequencer outages (multiple incidents on record).
- `MAX_TIME_DELTA` can be set up to 7 days, meaning even a multi-hour outage leaves the pre-downtime price valid.
- The attack requires no privileged access: any EOA can call `pool.swap()` immediately after sequencer recovery.
- The oracle updater's push transaction competes with the attacker's swap in the same recovery block; the attacker can front-run it.

---

### Recommendation

Integrate the Chainlink L2 sequencer uptime feed into both L2 provider variants. The standard pattern:

```solidity
// In constructor: store sequencerUptimeFeed address as immutable
// In _getBidAndAskPrice():
(, int256 answer, uint256 startedAt, ,) = sequencerUptimeFeed.latestRoundData();
if (answer != 0) return (0, type(uint128).max);                    // sequencer down
if (block.timestamp - startedAt < GRACE_PERIOD) return (0, type(uint128).max); // grace period
```

A grace period of at least 1 hour after recovery is standard practice (Chainlink recommends this). This ensures the oracle has time to publish a fresh price before swaps are permitted.

### Proof of Concept

Foundry fork test outline on Arbitrum:

```solidity
// 1. Fork Arbitrum mainnet at block B (sequencer up, oracle price = P0)
// 2. vm.warp(block.timestamp + OUTAGE_DURATION) where OUTAGE_DURATION < MAX_TIME_DELTA
//    (simulates sequencer recovery: block.timestamp jumps, oracle refTime stays at B)
// 3. Real price has moved to P1 (P1 >> P0, e.g. +20%)
// 4. Call pool.swap() — PriceProviderL2.getBidAndAskPrice() returns bid/ask based on P0
// 5. Assert swap succeeds and attacker receives tokens at P0 price
// 6. Assert LP reserves are drained by (P1 - P0) * swapAmount
```

The staleness check at step 4 passes because `(block.timestamp - refTime) = OUTAGE_DURATION < MAX_TIME_DELTA`. [6](#0-5)
