### Title
`AnchoredProviderFactory` Permits `baseFeedId == quoteFeedId`, Producing a Synthetic Ratio Price of 1.0 That Reaches Pool Swaps — (`smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`AnchoredProviderFactory.createAnchoredProvider()` is permissionless and performs no check that `baseFeedId != quoteFeedId`. When both IDs are identical and non-zero, `AnchoredPriceProvider._getBidAndAskPrice()` computes the synthetic ratio as `price(X) / price(X) = ORACLE_DECIMALS` (i.e., 1.0 in 8-decimal format), regardless of the actual market price. This corrupted mid passes every downstream guard — staleness, spread, priceGuard — and reaches `_computeBidAsk`, which produces a valid-looking bid/ask band anchored at 1.0. Any pool using such a provider executes swaps at a price of 1.0 instead of the real market price.

---

### Finding Description

`AnchoredProviderFactory` advertises its `isProvider()` predicate as the "public-pool eligibility predicate: deployed by this factory ⇒ clamp-bounded quotes with parameters that were inside the envelope at deploy time." [1](#0-0) 

However, `createAnchoredProvider` is fully permissionless and validates only the oracle allowlist and the envelope bounds for `baseFeedId`. It never checks that `baseFeedId != quoteFeedId`: [2](#0-1) 

The `AnchoredPriceProvider` constructor likewise stores both IDs without any equality check: [3](#0-2) 

At quote time, `_getBidAndAskPrice` reads both legs independently and computes the ratio:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // price(X) / price(X) = 1e8
spreadBps += spreadBps2;                          // doubled spread
``` [4](#0-3) 

Each leg passes its own `_readLeg` guards (staleness, mid > 0, spreadBps < ORACLE_BPS, priceGuard) because the individual feed data is valid. The corrupted ratio `mid = 1e8` is never checked against any post-ratio price guard. `_computeBidAsk` then builds a valid-looking band around 1.0 and returns it to the pool. [5](#0-4) 

---

### Impact Explanation

A pool whose `IPriceProvider` is an `AnchoredPriceProvider` with `baseFeedId == quoteFeedId` will execute every swap at a price of approximately 1.0 (in Q64 units scaled from 1e8). For any real asset pair where the true price deviates from 1.0 (e.g., BTC/ETH ≈ 20, ETH/USDC ≈ 2500), the pool will:

- Sell the overpriced asset far below market value, or
- Buy the underpriced asset far above market value.

LPs suffer direct loss of principal equal to the price deviation times the swap volume. The pool's bid/ask band is internally consistent (bid < ask, non-zero, within uint128) so no revert occurs — the loss is silent.

---

### Likelihood Explanation

`createAnchoredProvider` is permissionless. Any caller can supply `baseFeedId == quoteFeedId` with a valid oracle and envelope-compliant parameters; the factory registers the result as `isProvider() == true`. The most realistic trigger is an integrator or pool deployer who intends to set up a synthetic ratio pair (e.g., BTC/ETH via BTC/USD ÷ ETH/USD) but accidentally passes the same feed ID for both parameters. The factory emits no warning and the provider behaves normally until a swap is executed. The missing check is a single-line omission with no compensating guard anywhere in the quote path.

---

### Recommendation

Add an equality check in `AnchoredProviderFactory.createAnchoredProvider()` before deploying the provider:

```solidity
if (quoteFeedId != bytes32(0) && quoteFeedId == baseFeedId)
    revert IdenticalFeedIds();
```

Optionally mirror the check in the `AnchoredPriceProvider` constructor for defense-in-depth, analogous to the existing `_baseToken != _quoteToken` guard: [6](#0-5) 

---

### Proof of Concept

1. Admin adds oracle `O` to the allowlist and sets a DEFAULT_CLASS envelope with valid bounds.
2. Attacker calls:
   ```solidity
   factory.createAnchoredProvider(
       O,
       BTC_USD_FEED_ID,   // baseFeedId
       BTC_USD_FEED_ID,   // quoteFeedId  ← same as base
       minMargin,
       maxRefStaleness,
       maxSpreadBps,
       false,
       0,
       WBTC,
       WETH
   );
   ```
3. Factory registers the provider; `isProvider()` returns `true`.
4. A pool is created with this provider for the WBTC/WETH pair.
5. LPs deposit WBTC and WETH at the true market ratio (~20 WETH per WBTC).
6. At swap time, `_getBidAndAskPrice()` computes:
   - `mid1 = price(BTC_USD_FEED_ID)` = e.g. `10_000_000_000_000` (1e5 USD × 1e8)
   - `mid2 = price(BTC_USD_FEED_ID)` = same value
   - `mid = mulDiv(mid1, 1e8, mid2)` = `1e8` → represents price 1.0
7. Pool quotes bid/ask around 1.0 WETH per WBTC.
8. Attacker swaps 1 WBTC → receives ~20 WETH (pool's actual reserve ratio) while paying only 1 WETH worth of input at the quoted price, draining LP funds.

    function _computeBidAsk(uint256 mid, uint256 spreadBps)
        internal view returns (uint128, uint128)
    {
        // Circuit breaker: extreme (combined) uncertainty means the feed is clearly broken.
        if (spreadBps > MAX_SPREAD_BPS) {
            return (0, type(uint128).max);
        }

        // Reference band: mid ± (spreadBps + minMargin), bid rounded down, ask rounded up.
        uint256 half = spreadBps * ONE_BPS_E18 + minMargin; // < BPS_BASE_U by construction (spreadBps <= MAX_SPREAD_BPS here)
        uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
        uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
        if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
            return (0, type(uint128).max);
        }

        // Custom quote: source (both variants) or shaped reference quote (customizable variant).
        //    Immutable reference mode quotes the band directly — zero knob SLOADs.
        address _source = source;
        uint256 cBid;
        uint256 cAsk;
        if (_source != address(0)) {
            // 7a. Source mode: any failure (revert, OOG, garbage, zero, inverted) halts — fail
            //     closed. Knobs do NOT post-process the source output (the source shapes itself).
            bool ok;
            (ok, cBid, cAsk) = _readSource(_source);
            if (!ok) {
                return (0, type(uint128).max);
            }
        } else if (MUTABLE_PARAMS) {
            // 7b. Shaped reference quote: mid ± mid·spreadBps·confidence, then the marginStep step
            //     factors — PriceProvider semantics, clamped into the band below.
            bool ok;
            (ok, cBid, cAsk) = _shapedQuote(mid, spreadBps);
            if (!ok) {
                return (0, type(uint128).max);
            }
        } else {
            return (uint128(refBid), uint128(refAsk));
        }

        // 8. Clamp: out-of-band custom quotes are clipped silently to the band edge.
        //    bid ≤ refBid < refAsk ≤ ask, so bid < ask holds by construction.
        uint256 bidOut = Math.min(refBid, cBid);
        uint256 askOut = Math.max(refAsk, cAsk);
        if (bidOut == 0 || bidOut >= askOut) {
            return (0, type(uint128).max);
        }

        return (uint128(bidOut), uint128(askOut));
    }
