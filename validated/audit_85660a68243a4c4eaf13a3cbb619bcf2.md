### Title
`PriceProvider.getBidAndAskPrice()` Reverts with `FeedStalled` When `confidenceParam = 0` and `marginStep = 0`, Breaking Pool Swaps — (`smart-contracts-poc/contracts/PriceProvider.sol`)

---

### Summary

`PriceProvider` zero-initialises `confidenceParam` and provides no lower-bound guard on it. When `confidenceParam = 0` and `marginStep = 0` (both the default post-deployment state), `_getBidAndAskPrice()` computes `adjustedSpread = 0`, collapses bid and ask to the same oracle mid value, and then hits the `bidOut >= askOut` invariant guard — returning the `(0, type(uint128).max)` sentinel and causing `getBidAndAskPrice()` to revert with `FeedStalled()`. Every pool swap that calls this provider fails. The same defect exists verbatim in `ProtectedPriceProvider`.

---

### Finding Description

In `PriceProvider._getBidAndAskPrice()`, step 5 computes the spread-adjusted bid/ask:

```solidity
uint256 adjustedSpread = spread * confidenceParam;   // line 216
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [1](#0-0) 

`_getBidAskFrom` computes:

```solidity
uint256 delta = midPrice * confidence / CONFIDENCE_BASE;
bid = delta >= midPrice ? 0 : midPrice - delta;
ask = midPrice + delta;
``` [2](#0-1) 

When `confidenceParam = 0`: `delta = 0`, so `bid = ask = mid`.

After the step-factor adjustment with `marginStep = 0` (`stepBidFactor = stepAskFactor = BPS_BASE_U`):

```
bidOut = floor(mid × Q64 / ORACLE_DECIMALS)
askOut =  ceil(mid × Q64 / ORACLE_DECIMALS)
``` [3](#0-2) 

`bidOut = askOut` whenever `mid` is divisible by `5^8 = 390 625` (the odd factor of `ORACLE_DECIMALS = 1e8` after cancelling the `2^64` in Q64). Concrete examples: ETH/USD at exactly \$3 000.00 (`mid = 300 000 000 000`), BTC/USD at exactly \$60 000.00 (`mid = 6 000 000 000 000`), any "round" price. The guard then fires:

```solidity
if (bidOut >= askOut) return (0, type(uint128).max);
``` [4](#0-3) 

And the public entry point reverts:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [5](#0-4) 

`confidenceParam` is a plain storage slot, zero-initialised at deployment. `PriceProviderFactory.createPriceProvider()` never sets it:

```solidity
PriceProvider p = new PriceProvider(
    address(this), _oracle, _feedId, _marginStep,
    _maxTimeDelta, _baseToken, _quoteToken
);
``` [6](#0-5) 

`setConfidenceParam` accepts `0` as a valid value (no lower-bound check), so the broken state can also be re-entered deliberately or accidentally after deployment:

```solidity
if (newValue > CONFIDENCE_MAX) {
    revert ConfidenceParamOutOfBounds();
}
``` [7](#0-6) 

The identical defect exists in `ProtectedPriceProvider._computeBidAsk()`: [8](#0-7) 

For oracle prices that are *not* divisible by 390 625, `askOut = bidOut + 1` (one Q64 unit ≈ 5 × 10⁻²⁰ in price space), so swaps succeed but with a spread of effectively zero — LPs earn no spread fee and the pool is fully exposed to arbitrage at mid price.

---

### Impact Explanation

- **Intermittent swap DoS**: every swap through a pool backed by this provider reverts with `FeedStalled()` whenever the oracle mid price is divisible by 390 625 (common for round-dollar prices). Core swap functionality is broken.
- **Near-zero spread on other prices**: when the price is not divisible by 390 625, `askOut = bidOut + 1` (one Q64 unit). The pool executes swaps at essentially the oracle mid with no LP spread fee, constituting a swap conservation failure — traders receive better execution than the curve permits and LPs receive no compensation.

Both outcomes match the allowed impact gate: broken core pool functionality and swap conservation failure.

---

### Likelihood Explanation

Every `PriceProvider` (and `ProtectedPriceProvider`) deployed with `marginStep = 0` starts in this broken state. The factory does not set `confidenceParam` at creation. A pool can be created and funded before the owner calls `setConfidenceParam`, and the owner can inadvertently reset it to 0 at any time (no lower-bound guard). Round oracle prices (ETH at \$3 000, BTC at \$60 000, etc.) are common, making the DoS path reliably triggerable by any swap caller.

---

### Recommendation

1. **Enforce a non-zero lower bound** in `setConfidenceParam` (e.g., `require(newValue > 0)`), or
2. **Treat `confidenceParam = 0` as "use oracle spread directly"** — replace `spread * confidenceParam` with `confidenceParam == 0 ? spread * CONFIDENCE_BASE : spread * confidenceParam` so that zero means "1× oracle spread" rather than "zero spread", or
3. **Require `confidenceParam` to be set at construction** and passed as a constructor argument, mirroring how `marginStep` is handled.

The `AnchoredPriceProvider` avoids this class of failure entirely because its band clamp (`Math.min(refBid, cBid)` / `Math.max(refAsk, cAsk)`) restores valid ordering even when the shaped quote degenerates — `PriceProvider` has no equivalent safety net.

---

### Proof of Concept

1. Deploy `PriceProvider` via `PriceProviderFactory.createPriceProvider(oracle, feedId, 0 /*marginStep*/, maxTimeDelta, base, quote)`. `confidenceParam` is 0.
2. Create a `MetricOmmPool` pointing to this provider.
3. Oracle reports ETH/USD mid = `300_000_000_000` (exactly \$3 000.00, 8 decimals).
4. Call `pool.swap(...)`. Pool calls `provider.getBidAndAskPrice()`.
5. `adjustedSpread = 300_000_000_000 * 0 = 0` → `bid = ask = 300_000_000_000`.
6. `bidOut = floor(300_000_000_000 × 2^64 / 1e8) = floor(55_340_232_221_128_654_848)` = `55_340_232_221_128_654_848`.
7. `askOut = ceil(...)` = same value (exact division: `300_000_000_000 × 2^64 / 1e8 = 300_000_000_000 × 184_467_440_737 / 1_000_000 ...` — `300_000_000_000` is divisible by `5^8 = 390_625`, so the division is exact).
8. `bidOut >= askOut` → returns `(0, type(uint128).max)` → `getBidAndAskPrice()` reverts `FeedStalled()` → `MetricOmmPool` reverts `PriceProviderFailed`. [9](#0-8) [10](#0-9)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
```
