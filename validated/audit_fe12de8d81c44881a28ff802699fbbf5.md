### Title
Missing `msg.sender` Binding in Pool Deployment Salt Enables Front-Running with Malicious Price Provider — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.createPool()` accepts a caller-supplied `params.salt` and forwards it verbatim to `MetricOmmPoolDeployer.deploy()` without binding `msg.sender`. Because the only price-provider validation is a token-address match, an attacker can front-run a legitimate pool creation with the same salt but a malicious price provider, causing any user who pre-computed the deterministic pool address and batched subsequent interactions to execute against an attacker-controlled pool with bad-price quotes.

---

### Finding Description

`createPool()` passes `params.salt` directly to the deployer: [1](#0-0) 

`msg.sender` is never folded into the salt. The protocol uses CREATE3 for deterministic deployment (confirmed by the wiki and the explicit `salt` field in `PoolParameters`), so the deployed address is fully predictable from `(deployer, salt)` alone.

The only price-provider guard is `_validatePriceProvider()`: [2](#0-1) 

This check verifies only that `priceProvider.token0()` and `priceProvider.token1()` match the pool's tokens. It does **not** call `AnchoredProviderFactory.isProvider()` or any equivalent registry check. A malicious price provider trivially satisfies the token check while returning inverted, unbounded, or stale bid/ask quotes from `getBidAndAskPrice()`.

The pool admin is set to `params.admin` supplied by the front-runner: [3](#0-2) 

---

### Impact Explanation

A user who pre-computes the deterministic pool address and batches `createPool()` with `addLiquidity()` (or any subsequent pool interaction) will have those follow-on transactions execute against the attacker's pool. The attacker's price provider controls every bid/ask quote returned to the pool during swaps. By returning extreme or inverted quotes, the attacker can cause the pool's swap math to settle trades at prices that drain the victim's deposited principal. This is a direct loss of user funds through bad-price execution reaching pool swaps.

---

### Likelihood Explanation

Medium. Batching pool creation with immediate liquidity provision at the pre-computed address is a standard DeFi pattern (used in Uniswap v3, Balancer, etc.). The attack is fully permissionless — any address can call `createPool()` with an arbitrary salt — and requires only a standard mempool front-run. No privileged role or special setup is needed beyond deploying a two-function mock price provider.

---

### Recommendation

1. **Bind `msg.sender` to the salt** so each caller's salt-space is disjoint:
   ```solidity
   bytes32 boundSalt = keccak256(abi.encode(msg.sender, params.salt));
   // pass boundSalt to MetricOmmPoolDeployer.deploy(...)
   ```

2. **Enforce factory-registry membership** in `_validatePriceProvider()`:
   ```solidity
   require(anchoredProviderFactory.isProvider(priceProvider), NotRecognizedProvider());
   ```
   The `AnchoredProviderFactory.isProvider()` predicate already exists for exactly this purpose: [4](#0-3) 

---

### Proof of Concept

1. Bob computes the deterministic address `P` for salt `S` using the CREATE3 formula against `MetricOmmPoolDeployer`.
2. Bob signs and broadcasts a batch: `createPool(salt=S, priceProvider=legitimate, admin=Bob, ...)` followed by `addLiquidity(pool=P, amount=X)`.
3. Alice observes the pending batch in the mempool. She deploys `MaliciousProvider` — a contract that returns `token0 = params.token0`, `token1 = params.token1` (passing `_validatePriceProvider`) but returns inverted or extreme values from `getBidAndAskPrice()`.
4. Alice front-runs with `createPool(salt=S, priceProvider=MaliciousProvider, admin=Alice, ...)`. The pool is deployed at address `P` with Alice as admin and `MaliciousProvider` as the price provider.
5. Bob's `createPool` reverts (address already deployed). His `addLiquidity(pool=P, ...)` executes against Alice's pool.
6. Alice's `MaliciousProvider.getBidAndAskPrice()` returns quotes that cause the pool's swap math to settle any swap at a price that transfers Bob's deposited tokens to Alice, constituting direct loss of principal. [5](#0-4) [6](#0-5)

  function _validatePriceProvider(address token0, address token1, address priceProvider) internal view {
    if (priceProvider == address(0)) revert InvalidPriceProvider();
    if (IPriceProvider(priceProvider).token0() != token0 || IPriceProvider(priceProvider).token1() != token1) {
      revert PriceProviderTokenMismatch();
    }
  }
