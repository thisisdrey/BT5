### Title
Inactive provider owner permanently locks pool swaps when `marginStep < 0` and `confidenceParam` is never initialized — (File: `smart-contracts-poc/contracts/PriceProvider.sol`, `ProtectedPriceProvider.sol`)

---

### Summary

`confidenceParam` is zero-initialized in `PriceProvider` and `ProtectedPriceProvider`. When `marginStep < 0` (a valid, explicitly permitted constructor parameter), a `confidenceParam` of `0` causes `bidOut > askOut` in Q64 space, triggering the `bidOut >= askOut` guard and returning the `FeedStalled` sentinel on every swap. `confidenceParam` can only be updated by the provider owner (or a granted updater) through the factory. If the provider owner is inactive and no updaters are set, `confidenceParam` can never be changed. For pools configured with an immutable price provider (`priceProviderTimelock == type(uint256).max`), the pool admin has no escape hatch — `proposePoolPriceProvider` reverts with `PriceProviderImmutable`.

---

### Finding Description

**Root cause — inverted Q64 bid/ask when `marginStep < 0` and `confidenceParam = 0`:**

In `PriceProvider._getBidAndAskPrice()`, when `confidenceParam = 0`:

```
adjustedSpread = spread * 0 = 0
→ _getBidAskFrom(mid, 0): delta = 0, bid = mid, ask = mid  (8-decimal oracle units)
``` [1](#0-0) 

The step adjustment is then applied to the identical `bid = ask = mid`:

```
stepBidFactor = BPS_BASE_U − marginStep   (when marginStep < 0 → stepBidFactor > BPS_BASE_U)
stepAskFactor = BPS_BASE_U + marginStep   (when marginStep < 0 → stepAskFactor < BPS_BASE_U)
``` [2](#0-1) 

Because `bid = ask = mid` and `stepBidFactor > stepAskFactor` when `marginStep < 0`:

```
bidOut  = mulDiv(mid, Q64 * stepBidFactor, STEP_DENOM, Floor)   > mid_Q64
askOut  = mulDiv(mid, Q64 * stepAskFactor, STEP_DENOM, Ceil)    < mid_Q64
→ bidOut > askOut  →  returns (0, type(uint128).max)
``` [3](#0-2) 

The public `getBidAndAskPrice()` then reverts with `FeedStalled` on every call: [4](#0-3) 

**Single point of failure — only the provider owner can fix it:**

`setConfidenceParam` is gated to `msg.sender == factory`: [5](#0-4) 

The factory's `setConfidence` requires the caller to be the provider owner or a granted updater: [6](#0-5) 

If the provider owner is inactive and no updaters were granted, `confidenceParam` is permanently stuck at `0`.

**No escape hatch for immutable price provider pools:**

`proposePoolPriceProvider` explicitly reverts when `immutablePriceProvider != address(0)`: [7](#0-6) 

`_resolvedPriceProvider()` always returns `IMMUTABLE_PRICE_PROVIDER` when set, so `setPriceProvider` has no effect: [8](#0-7) 

**Design inconsistency with `AnchoredPriceProvider`:**

`AnchoredPriceProvider` explicitly documents that negative `marginStep` is safe *because* of its load-bearing band clamp: [9](#0-8) 

`PriceProvider` and `ProtectedPriceProvider` have no such clamp, yet accept the same `marginStep` range. The same applies to `ProtectedPriceProviderL2`: [10](#0-9) 

---

### Impact Explanation

All swaps on any pool using this provider permanently revert with `FeedStalled`. LPs can still call `removeLiquidity` (which does not invoke the price provider), so principal is not directly lost, but the pool's core swap functionality is permanently disabled. Accumulated fees cannot be earned, and the pool is entirely unusable for its primary purpose. This matches the allowed impact: **broken core pool functionality causing unusable swap flows**.

---

### Likelihood Explanation

- `marginStep < 0` is a valid, explicitly permitted constructor parameter (only `|marginStep| >= BPS_BASE` is rejected).
- `confidenceParam = 0` is the Solidity default for uninitialized storage — no constructor or factory validation enforces a non-zero value before pool use.
- `priceProviderTimelock == type(uint256).max` (immutable provider) is a documented pool creation option.
- The factory's `createPool` only validates `token0()/token1()` matching, not that `getBidAndAskPrice()` returns a valid quote.
- Provider ownership is a single address with no on-chain succession mechanism; key loss or abandonment is a realistic operational risk.

Trigger path is permissionless: any caller can deploy a `PriceProvider` with `marginStep < 0` via `PriceProviderFactory.createPriceProvider`, then create an immutable-provider pool against it via `MetricOmmPoolFactory.createPool`.

---

### Recommendation

1. **Reject `marginStep < 0` in `PriceProvider` and `ProtectedPriceProvider`** unless a minimum `confidenceParam` is enforced at construction, since there is no band clamp to neutralize the inversion.
2. **Require `confidenceParam > 0` at construction** (or enforce a minimum via the factory) when `marginStep <= 0`, mirroring the `AnchoredPriceProvider`'s band-clamp safety net.
3. **Add a time-based fallback** analogous to the Fantom recommendation: allow any caller to set `confidenceParam` to a safe default after a timeout if the provider owner has not acted, e.g.:
   ```solidity
   require(msg.sender == factory || block.timestamp > deployedAt + OWNER_TIMEOUT);
   ```
4. **Validate provider liveness** in `MetricOmmPoolFactory._validatePriceProvider` by calling `getBidAndAskPrice()` in a try/catch and reverting if it returns the stall sentinel.

---

### Proof of Concept

1. Deploy `PriceProvider` via `PriceProviderFactory.createPriceProvider` with `_marginStep = -1e14` (−0.01%, valid), any live oracle feed, and `_maxTimeDelta = 1 hours`. `confidenceParam` is `0` by default.

2. Call `MetricOmmPoolFactory.createPool` with this provider and `priceProviderTimelock = type(uint256).max` (immutable). Pool is created successfully — `_validatePriceProvider` only checks token addresses.

3. LPs call `addLiquidity` — succeeds (no price provider call).

4. Any trader calls `swap(...)`. The pool calls `_getBidAndAskPriceX64()` → `IPriceProvider.getBidAndAskPrice()` → `_getBidAndAskPrice()`:
   - `adjustedSpread = spread * 0 = 0`
   - `bid = ask = mid` (8-decimal)
   - `bidOut = mulDiv(mid, Q64 * (1e18 + 1e14), 1e26, Floor)` > `mid_Q64`
   - `askOut = mulDiv(mid, Q64 * (1e18 − 1e14), 1e26, Ceil)` < `mid_Q64`
   - `bidOut >= askOut` → returns `(0, type(uint128).max)`
   - `getBidAndAskPrice()` reverts with `FeedStalled`
   - Pool reverts with `PriceProviderFailed`

5. Provider owner loses their key. `confidenceParam` can never be set. `proposePoolPriceProvider` reverts with `PriceProviderImmutable`. Pool swaps are permanently broken.
