### Title
Prior Caller's Stranded Native ETH Consumed as Swap Input by Subsequent WETH Payer — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

### Summary

`PeripheryPayments.pay()` uses `address(this).balance` — the **total** contract ETH balance — as the native-ETH portion of a WETH payment. Any ETH left on the router by a prior caller (via a `payable` multicall or direct `exactInputSingle` with excess `msg.value`) is silently consumed as part of a subsequent caller's swap input, causing the prior caller to lose their stranded ETH.

### Finding Description

The `pay()` function in `PeripheryPayments.sol` handles WETH payments with a three-branch native-balance check: [1](#0-0) 

```solidity
} else if (token == WETH) {
    uint256 nativeBalance = address(this).balance;   // ← global, not per-caller
    if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
    } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
    } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
    }
```

`address(this).balance` is the **entire** contract balance, not `msg.value` of the current transaction. ETH can be stranded on the router because `multicall` and `exactInputSingle` are both `payable`: [2](#0-1) [3](#0-2) 

The `receive()` guard only blocks **direct** ETH transfers (not via calldata), so it does not prevent ETH from accumulating via `payable` function calls: [4](#0-3) 

**Attack path:**

1. **Caller A** calls `multicall{value: 15 ether}(...)` with a WETH swap for `amountIn = 10 ether`. `pay()` enters the `nativeBalance >= value` branch, wraps 10 ETH, and sends WETH to the pool. The remaining **5 ETH stays on the contract**. Caller A omits `refundETH()` (or the multicall reverts after the swap, leaving residue).

2. **Caller B** calls `exactInputSingle{value: 0}(tokenIn=WETH, amountIn=10 ether)`. Inside `_justPayCallback` → `pay()`: [5](#0-4) 

   `nativeBalance = 5 ether` (Caller A's residue), `value = 10 ether`. The `nativeBalance > 0` branch fires: the router wraps Caller A's 5 ETH, transfers 5 WETH to the pool, then pulls only 5 WETH from Caller B via `safeTransferFrom`. Caller B pays 5 WETH instead of 10 WETH.

3. **Caller A loses 5 ETH**. Caller B receives a 5 ETH subsidy. The pool is made whole (receives 10 WETH total), so no pool insolvency — but Caller A suffers a direct, unrecoverable principal loss.

### Impact Explanation

Direct loss of user principal. Caller A's ETH is irreversibly consumed as part of Caller B's swap input. The loss equals the stranded ETH amount. `refundETH()` cannot help Caller A after the fact because the ETH is already gone. [6](#0-5) 

### Likelihood Explanation

Moderate. The pattern of sending ETH with a multicall and omitting `refundETH()` is common (e.g., slippage causes less ETH to be used than sent, or a downstream call in the multicall reverts after the swap). Any on-chain observer can monitor `address(router).balance > 0` and immediately call `exactInputSingle(tokenIn=WETH, msg.value=0)` to harvest the residue.

### Recommendation

Track per-transaction ETH contribution rather than using the global `address(this).balance`. The standard fix is to record `msg.value` at the top-level entry point (e.g., in `exactInputSingle` / `multicall`) and pass it down to `pay()`, consuming only that amount as native ETH. Alternatively, enforce that `address(this).balance` equals `msg.value` at the start of every top-level `payable` entry point and revert otherwise.

### Proof of Concept

```solidity
// Foundry integration test sketch
function test_strandedEthConsumedBySubsequentCaller() public {
    // Caller A: sends 15 ETH, swaps 10 WETH, forgets refundETH
    vm.deal(callerA, 15 ether);
    vm.prank(callerA);
    router.exactInputSingle{value: 15 ether}(ExactInputSingleParams({
        tokenIn: address(weth), amountIn: 10 ether, ...
    }));
    // 5 ETH stranded on router
    assertEq(address(router).balance, 5 ether);

    // Caller B: sends 0 ETH, swaps 10 WETH — only needs 5 WETH from wallet
    uint256 wethBefore = weth.balanceOf(callerB);
    vm.prank(callerB);
    router.exactInputSingle{value: 0}(ExactInputSingleParams({
        tokenIn: address(weth), amountIn: 10 ether, ...
    }));
    // Caller B spent only 5 WETH; router balance is now 0
    assertEq(weth.balanceOf(callerB), wethBefore - 5 ether);
    assertEq(address(router).balance, 0);
    // Caller A's 5 ETH is gone — unrecoverable
}
```
