Looking at the `AnchoredProviderFactory.createAnchoredProvider` function and the `AnchoredPriceProvider` constructor, I need to trace whether the factory's eligibility gate (`isProvider()`) can be satisfied by a provider whose token pair is misbound to the wrong oracle feed — the direct analog to HoneyLocker's codehash-passes-but-params-differ bug.

### Title
`AnchoredProviderFactory.createAnchoredProvider` Validates Oracle and Envelope but Not Token/Feed Binding, Allowing a Misbound Provider to Pass `isProvider()` and Deliver Wrong-Price Quotes to Pool Swaps — (File: `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`AnchoredProviderFactory.createAnchoredProvider` is permissionless. It validates that the oracle is on the admin allow-list and that band parameters fall within the per-feed-class envelope, but it **never verifies that `baseToken`/`quoteToken` correspond to what `baseFeedId`/`quoteFeedId` actually prices**. A caller can deploy a provider whose token pair is entirely mismatched to its feed (e.g., ETH/USD feed bound to BTC/USDC tokens). That provider is registered in `_providers`, so `isProvider()` returns `true` — the machine-checkable public-pool eligibility predicate — and the pool factory accepts it. Every subsequent swap in that pool is priced off the wrong oracle feed, causing direct fund loss to LPs and traders.

---

### Finding Description

The factory's stated invariant is:

> "public-pool eligibility is then the machine-checkable predicate `recognizedFactory.isProvider(p)`" [1](#0-0) 

`createAnchoredProvider` enforces two structural checks before registering a provider:

1. Oracle must be in the admin allow-list.
2. `minMargin`, `maxRefStaleness`, `maxSpreadBps` must be within the envelope for `feedClass[baseFeedId]`. [2](#0-1) 

Neither check touches the relationship between `baseToken`/`quoteToken` and `baseFeedId`/`quoteFeedId`. The `AnchoredPriceProvider` constructor itself documents this gap explicitly:

> "Tokens live ONLY here (the oracles are token-free): the pair is an explicit, mandatory input" [3](#0-2) 

The oracle read path in `_readLeg` calls `offchainOracle.price(feedId, pool)` — it returns the price for whatever `feedId` was baked in at construction, with no awareness of the token pair: [4](#0-3) 

After the factory registers the provider, `isProvider()` returns `true`: [5](#0-4) 

The pool factory can only check `provider.token0()`/`provider.token1()` against the pool's tokens — it cannot detect that the feed behind those tokens is wrong. A provider with `baseToken = BTC`, `quoteToken = USDC`, `baseFeedId = ETH/USD_FEED` returns `token0() == BTC` and `token1() == USDC`, so any token-equality check at pool creation passes, yet every `getBidAndAskPrice()` call during a swap returns ETH/USD quotes for a BTC/USDC pool.

---

### Impact Explanation

**High.** Every swap in a pool backed by a misbound provider executes at the wrong oracle price. If the misbound feed prices the asset at a fraction of its true value (e.g., ETH/USD ≈ \$3 000 used for a BTC/USDC pool where BTC ≈ \$100 000), LPs suffer immediate, unbounded loss as arbitrageurs drain the pool at the mispriced rate. The bid/ask returned by `getBidAndAskPrice` is structurally valid (non-zero, non-inverted, within the band) so no circuit-breaker fires: [6](#0-5) 

---

### Likelihood Explanation

**Medium.** `createAnchoredProvider` is fully permissionless — no role is required. Any caller can supply a legitimate allow-listed oracle and envelope-valid parameters while freely choosing any `baseToken`/`quoteToken` pair. The only prerequisite is that the admin has configured at least one oracle and one envelope (normal operational state). Creating a pool with the resulting provider is also permissionless (pool factory is open). The exploit requires no privileged action after initial protocol setup.

---

### Recommendation

Add a feed-to-token registry in `AnchoredProviderFactory` (admin-curated, analogous to `feedClass`) that maps each `feedId` to its canonical `(baseToken, quoteToken)` pair. In `createAnchoredProvider`, require:

```solidity
(address expectedBase, address expectedQuote) = feedTokens[baseFeedId];
if (expectedBase != address(0)) {
    require(baseToken == expectedBase && quoteToken == expectedQuote, TokenFeedMismatch());
}
```

Alternatively, if the oracle itself exposes a token-pair getter per feed, query it and compare against the caller-supplied tokens before deploying the provider.

        address oracle,
        bytes32 baseFeedId,
        bytes32 quoteFeedId,
        uint256 minMargin,
        uint256 maxRefStaleness,
        uint16  maxSpreadBps,
        bool    mutableParams,
        int256  marginStep,
        address baseToken,
        address quoteToken
    ) external override returns (address provider) {
        if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);

        // Feeds without an explicit class fall back to the admin-configured DEFAULT_CLASS envelope.
        bytes32 classId = feedClass[baseFeedId];
        if (classId == bytes32(0)) classId = DEFAULT_CLASS;

        Envelope storage env = envelopes[classId];
        if (!env.exists) revert EnvelopeNotFound(classId);
        if (
            minMargin < env.minMarginMin || minMargin > env.minMarginMax
            || maxRefStaleness < env.stalenessMin || maxRefStaleness > env.stalenessMax
            || maxSpreadBps < env.maxSpreadMin || maxSpreadBps > env.maxSpreadMax
        ) revert ParamsOutOfEnvelope();

        AnchoredPriceProvider p = new AnchoredPriceProvider(
            address(this),
            oracle,
            baseFeedId,
            quoteFeedId,
            minMargin,
            maxRefStaleness,
            maxSpreadBps,
            mutableParams,
            marginStep,
            baseToken,
            quoteToken
        );

        provider = address(p);
        address creator = msg.sender;

        _providers.add(provider);
        _providersByCreator[creator].add(provider);
        providerOwner[provider] = creator;

        emit ProviderDeployed(
            provider,
            creator,
            baseFeedId,
            quoteFeedId,
            classId,
            p.baseToken(),
            p.quoteToken(),
            minMargin,
            maxRefStaleness,
            maxSpreadBps,
            mutableParams,
            marginStep,
            oracle
        );
    }

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
