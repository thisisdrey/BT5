### Title
`AnchoredPriceProvider` with `baseFeedId == quoteFeedId` Produces a Hardcoded Price of 1.0, Enabling Pool Drain via Bad-Price Execution — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`, `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`AnchoredProviderFactory.createAnchoredProvider` is permissionless and accepts any `baseFeedId` and `quoteFeedId` without validating they are distinct. When both are set to the same non-zero feed ID, the synthetic-ratio path in `AnchoredPriceProvider._getBidAndAskPrice` computes `mid = mulDiv(P, 1e8, P) = 1e8` — a hardcoded constant of 1.0 in 8-decimal oracle format — regardless of the actual market price. The factory marks the resulting provider `isProvider() == true`, its broken safety predicate. A pool admin (semi-trusted) who sets this provider causes every swap to execute at a fixed price of ~1.0 token0-per-token1, enabling full pool drain by arbitrage.

---

### Finding Description

`AnchoredPriceProvider._getBidAndAskPrice` implements synthetic ratio quoting when `quoteFeedId != bytes32(0)`:

```solidity
// AnchoredPriceProvider.sol lines 258-271
function _getBidAndAskPrice() internal returns (uint128, uint128) {
    (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
    if (!ok) return (0, type(uint128).max);

    bytes32 _quote = quoteFeedId;
    if (_quote != bytes32(0)) {
        (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
        if (!ok2 || mid2 == 0) return (0, type(uint128).max);
        mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // ← ratio
        spreadBps += spreadBps2;
    }
    return _computeBidAsk(mid, spreadBps);
}
``` [1](#0-0) 

When `baseFeedId == quoteFeedId`, both `_readLeg` calls query the **same oracle slot**, returning the same value `P`. The ratio collapses:

```
mid = mulDiv(P, 1e8, P) = 1e8   (always, for any market price P)
spreadBps = 2 × oracle_spread
```

`_computeBidAsk(1e8, 2×spread)` then calls `_bandEdge`:

```solidity
// AnchoredPriceProvider.sol lines 244-250
function _bandEdge(uint256 mid, uint256 edgeFactor, Math.Rounding rounding)
    internal pure returns (uint256) {
    return Math.mulDiv(mid, Q64 * edgeFactor, STEP_DENOM, rounding);
    // STEP_DENOM = 1e8 * 1e18 = 1e26
}
``` [2](#0-1) 

With `mid = 1e8`:

```
refBid = mulDiv(1e8, Q64 × (1e18 − half), 1e26) = Q64 × (1 − half/1e18)
refAsk = mulDiv(1e8, Q64 × (1e18 + half), 1e26) = Q64 × (1 + half/1e18)
```

This is a Q64.64 price of approximately **1.0 token0 per token1**, independent of the real market price.

Neither the `AnchoredPriceProvider` constructor nor `AnchoredProviderFactory.createAnchoredProvider` validates `baseFeedId != quoteFeedId`:

```solidity
// AnchoredPriceProvider.sol lines 136-141
offchainOracle = IOffchainOracle(_oracle);
baseFeedId = _baseFeedId;
quoteFeedId = _quoteFeedId;
// ← no require(_baseFeedId != _quoteFeedId)
``` [3](#0-2) 

```solidity
// AnchoredProviderFactory.sol lines 156-194
function createAnchoredProvider(
    address oracle,
    bytes32 baseFeedId,
    bytes32 quoteFeedId,   // ← accepted equal to baseFeedId
    ...
) external override returns (address provider) {
    if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);
    // envelope validation only checks minMargin/staleness/spread
    // ← no check: baseFeedId != quoteFeedId
    AnchoredPriceProvider p = new AnchoredPriceProvider(...);
    _providers.add(provider);   // ← isProvider() == true for broken provider
``` [4](#0-3) 

The factory's documented safety predicate — `isProvider(p) == true` implies clamp-bounded, market-reflective quotes — is broken. The pool factory's `_validatePriceProvider` only checks token address matching, not feed-ID sanity:

```solidity
// MetricOmmPoolFactory.sol lines 541-546
function _validatePriceProvider(address token0, address token1, address priceProvider) internal view {
    if (priceProvider == address(0)) revert InvalidPriceProvider();
    if (IPriceProvider(priceProvider).token0() != token0 || IPriceProvider(priceProvider).token1() != token1) {
        revert PriceProviderTokenMismatch();
    }
}
``` [5](#0-4) 

The pool's `_getBidAndAskPriceX64` then consumes the corrupted quote directly in every swap:

```solidity
// MetricOmmPool.sol lines 804-813
function _getBidAndAskPriceX64() internal returns (uint128 bidPriceX64, uint128 askPriceX64) {
    address activePriceProvider = _resolvedPriceProvider();
    try IPriceProvider(activePriceProvider).getBidAndAskPrice() returns (uint128 bid, uint128 ask) {
        if (bid >= ask) revert BidGreaterThanAsk();
        if (bid == 0) revert BidIsZero();
        return (bid, ask);
    } catch (bytes memory reason) {
        revert PriceProviderFailed(reason);
    }
}
``` [6](#0-5) 

The corrupted bid/ask passes both guards (`bid < ask` and `bid > 0`) because the band arithmetic produces a valid-looking but wrong spread around 1.0.

---

### Impact Explanation

For any real asset pair where the true price deviates from 1.0 (e.g., ETH/USDC at ~3000, BTC/ETH at ~20), every swap executes at the hardcoded price of ~1.0. Arbitrageurs can immediately drain the pool by buying the underpriced token at 1.0 and selling it at market. All LP principal is at risk. This is a direct, complete loss of user funds.

---

### Likelihood Explanation

`createAnchoredProvider` is permissionless — any address can deploy a provider with `baseFeedId == quoteFeedId`. The resulting provider passes `isProvider() == true`. A pool admin (semi-trusted) who relies on the factory's safety predicate and sets this provider triggers the loss. The timelock on provider updates does not prevent the attack once the provider is set and the timelock elapses. The factory's own documentation describes `isProvider()` as the machine-checkable eligibility predicate, making it a reasonable trust anchor for pool admins.

---

### Recommendation

Add a validation in both `AnchoredPriceProvider` constructor and `AnchoredProviderFactory.createAnchoredProvider`:

```solidity
// In AnchoredPriceProvider constructor, after setting baseFeedId/quoteFeedId:
require(
    _quoteFeedId == bytes32(0) || _baseFeedId != _quoteFeedId,
    "base and quote feed IDs must differ in synthetic mode"
);

// In AnchoredProviderFactory.createAnchoredProvider, before deploying:
if (quoteFeedId != bytes32(0) && baseFeedId == quoteFeedId) revert SameFeedId();
```

### Proof of Concept

```solidity
// Setup: oracle has feed F with price P = 100_000_000 (1.0 in 8-decimal)
// Attacker calls (permissionless):
factory.createAnchoredProvider(
    oracle,
    FEED_ID,   // baseFeedId = F
    FEED_ID,   // quoteFeedId = F  ← same as baseFeedId
    minMargin, staleness, maxSpread,
    false, 0,
    baseToken, quoteToken
);
// provider is now isProvider() == true

// Pool admin (semi-trusted) sets this provider on the pool.
// On every swap:
//   _readLeg(F) → mid = P, spread = S
//   _readLeg(F) → mid2 = P, spread2 = S
//   mid = mulDiv(P, 1e8, P) = 1e8  ← hardcoded 1.0
//   spreadBps = 2S
//   refBid ≈ Q64 × (1 − 2S/1e4 − minMargin/1e18)
//   refAsk ≈ Q64 × (1 + 2S/1e4 + minMargin/1e18)
// Pool executes all swaps at price ≈ 1.0 token0/token1.
// Arbitrageur buys token0 at 1 token1 each; real market price is 3000 token1 per token0.
// Pool is drained in one transaction.
```
