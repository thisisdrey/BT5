## Title
Missing Sequencer Uptime Feed Check in `ProtectedPriceProviderL2` Allows Stale Pre-Downtime Prices to Reach Pool Swaps — (`smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

## Summary

`ProtectedPriceProviderL2` is the L2-specific price provider for Metric OMM pools. It is named and documented as the L2-aware variant, yet its constructor accepts no `_sequencerUptimeFeed` parameter and its `_computeBidAsk` path contains no sequencer uptime or grace-period check. The only L2-specific logic it adds over the L1 variant is `FUTURE_TOLERANCE` — a clock-skew tolerance that allows `refTime` slightly ahead of `block.timestamp`. This is unrelated to sequencer downtime. The sole staleness guard is `_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)`, which only checks the age of the oracle's stored `refTime`. After a sequencer restart, if the stored pre-downtime `refTime` is still within `MAX_TIME_DELTA` of `block.timestamp`, the check passes and the stale price is consumed by the pool.

## Finding Description

**Constructor — no sequencer feed:** [1](#0-0) 

The constructor takes `_factory`, `_oracle`, `_offchainFeedId`, `_marginStep`, `_maxTimeDelta`, `_futureTolerance`, `_baseToken`, `_quoteToken`. There is no `_sequencerUptimeFeed` parameter and no `GRACE_PERIOD` constant anywhere in the file.

**`_computeBidAsk` — the only guards:** [2](#0-1) 

The five checks are: (1) `_isStale` on `refTime`, (2) `price == 0 || spread >= ORACLE_BPS`, (3) price guard bounds, (4) confidence-adjusted bid/ask math, (5) `bid < ask`. No sequencer uptime feed is consulted at any point.

**`_isStale` — only checks age of stored `refTime`:** [3](#0-2) 

This is a pure function that compares `refTime` to `block.timestamp`. It has no knowledge of whether the sequencer was down.

**Oracle layer — no sequencer check either:**

`OracleBase.price()` simply reads stored data and returns it: [4](#0-3) 

`ChainlinkOracle._store()` checks `revertIfAfterBlockTimeWithDrift` only at **ingestion** time (when a pusher calls `updateReport`), not at read time: [5](#0-4) 

Neither `PythOracle` nor `OracleBase` performs any sequencer uptime check at read time. [6](#0-5) 

**Registry confirms a sequencer-aware variant exists but is not wired here:**

The registry ABI contains a separate contract with `GRACE_PERIOD()`, `sequencerUptimeFeed()`, and a constructor parameter `_sequencerUptimeFeed` — but that contract also exposes `cexStep`, `maxClDeviation`, `setClOracle`, `removeClOracle`, which are absent from `ProtectedPriceProviderL2.sol`. This is a distinct legacy provider. The `ProtectedPriceProviderL2` source has none of these. [7](#0-6) 

## Impact Explanation

**Attack scenario:**

1. L2 sequencer goes down. Oracle pushers cannot submit new reports. The last stored `oracleData[feedId]` has `refTime = T_down` (the last pre-downtime update).
2. Market prices move significantly during downtime (e.g., a 5–10% move is common during volatile periods that cause sequencer outages).
3. Sequencer restarts at `T_restart`. `block.timestamp = T_restart`.
4. If `T_restart - T_down < MAX_TIME_DELTA`, `_isStale` returns `false` — the pre-downtime price passes.
5. Before any pusher submits a fresh oracle update, an attacker calls `pool.swap()`. The pool calls `ProtectedPriceProviderL2.getBidAndAskPrice()`, which reads the stale pre-downtime price and returns bid/ask Q64 values computed from it.
6. The attacker swaps at the stale price, arbitraging the difference between the stale bid/ask and the true post-downtime market price. LP token balances are drained by the arbitrage profit.

`MAX_TIME_DELTA` is configurable up to 7 days: [8](#0-7) 

Even with a short `MAX_TIME_DELTA` (e.g., 1 hour), a sequencer outage of 59 minutes leaves a window where the pre-downtime price passes the staleness check. The attacker only needs to be the first transaction after sequencer restart, before any oracle pusher updates the feed.

**Impact gate satisfied:** Bad-price execution — stale bid/ask quote reaches a pool swap. LP principal is at risk.

## Likelihood Explanation

- L2 sequencer outages are documented historical events on Arbitrum, Optimism, Base, and other chains where `PriceProviderFactoryL2` is deployed.
- The attacker needs only a public `pool.swap()` call — no privileged access.
- The window is bounded by how quickly oracle pushers react after restart, but MEV bots routinely front-run such windows.
- `MAX_TIME_DELTA` up to 7 days means even long outages can leave the price "fresh" by the contract's own check.

## Recommendation

Add a sequencer uptime feed check (Chainlink `AggregatorV3Interface` sequencer uptime feed) with a `GRACE_PERIOD` (e.g., 3600 seconds) to `ProtectedPriceProviderL2._computeBidAsk`. Reject any price read if the sequencer is down or if `block.timestamp - sequencerStartedAt < GRACE_PERIOD`. The registry already contains a `ChainlinkVerifierL2` component with `GRACE_PERIOD` and `sequencerUptimeFeed` — wire this check into `ProtectedPriceProviderL2` at the `_computeBidAsk` entry point, returning the stalled sentinel `(0, type(uint128).max)` when the sequencer is in its grace period.

```text
    function _isStale(
        uint256 refTime,
        uint256 nowTs,
        uint256 maxDelta,
        uint256 futureTol
    ) internal pure returns (bool) {
        if (refTime == 0) return true;

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol
            return (refTime - nowTs) > futureTol;
        }

        // refTime in the past or equal: check age
        return (nowTs - refTime) > maxDelta;
    }
```

```text
    function _computeBidAsk(uint256 price, uint256 spread, uint256 refTime)
        internal view returns (uint128, uint128)
    {
        // 1. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
            return (0, type(uint128).max);
        }

        // 2. Basic validity — price must be positive, spread must not be stalled marker
        if (price == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 3. Price guard check
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (price < guardMin || price > guardMax) {
            return (0, type(uint128).max);
        }

        // 4. Compute bid/ask from mid + confidence-adjusted spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(price, adjustedSpread);

        // 5. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 6. Hard invariant: bid must be strictly less than ask.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
```
