Looking at the `transferProviderOwnership` flow in both factory contracts and the `isUpdater` mapping, I need to check whether old updaters are cleared on ownership transfer, and what `setConfidence` can do to bid/ask prices.

### Title
Previous provider owner's granted updaters retain `setConfidence` access after `transferProviderOwnership`, enabling manipulation of bid/ask prices reaching pool swaps — (`PriceProviderFactory.sol` / `AnchoredProviderFactory.sol`)

---

### Summary

`transferProviderOwnership()` in both `PriceProviderFactory` and `AnchoredProviderFactory` updates `providerOwner` but **never revokes previously granted updaters**. An updater granted by the old owner retains the ability to call `setConfidence()` after the transfer, directly manipulating `confidenceParam` inside `PriceProvider` / `AnchoredPriceProvider`. Because `confidenceParam` is a live multiplier on the oracle spread used to compute bid/ask prices, the stale updater can push the spread to zero or to an extreme, causing bad-price execution or feed stall on every pool swap that consumes that provider.

---

### Finding Description

**Step 1 — Ownership transfer does not clear updaters.**

`PriceProviderFactory.transferProviderOwnership()`:

```solidity
providerOwner[provider] = newOwner;
_providersByCreator[previousOwner].remove(provider);
_providersByCreator[newOwner].add(provider);
``` [1](#0-0) 

The `isUpdater[provider][*]` mapping is never touched. The identical omission exists in `AnchoredProviderFactory`: [2](#0-1) 

**Step 2 — `setConfidence` only checks `_requireUpdater`, which still passes for old updaters.**

```solidity
function _requireUpdater(address provider) internal view {
    if (msg.sender != providerOwner[provider] && !isUpdater[provider][msg.sender])
        revert NotProviderUpdater();
}
``` [3](#0-2) 

Because `isUpdater[provider][oldUpdater]` is still `true`, the old updater passes this check and can call `setConfidence()` on the transferred provider at any time (subject only to the 1-minute cooldown). [4](#0-3) 

**Step 3 — `confidenceParam` is a live multiplier on the oracle spread used to compute bid/ask.**

Inside `PriceProvider._getBidAndAskPrice()`:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [5](#0-4) 

Setting `confidenceParam = 0` collapses the spread to zero (`delta = 0`, `bid = ask = mid`). After the step-factor adjustment, if `marginStep ≤ 0` this produces `bidOut ≥ askOut`, which triggers the hard invariant guard and returns the stale sentinel `(0, type(uint128).max)`. `getBidAndAskPrice()` then reverts with `FeedStalled()`, blocking every pool swap that uses this provider. [6](#0-5) 

Setting `confidenceParam = CONFIDENCE_MAX (1_000_000)` with a non-trivial oracle spread can push `delta ≥ mid`, collapsing `bid` to zero — again returning the stale sentinel and blocking swaps. [7](#0-6) 

Even without triggering the stale sentinel, the old updater can oscillate `confidenceParam` between 0 and `CONFIDENCE_MAX` every minute, producing artificially tight or wide bid/ask quotes that reach pool swaps — a direct bad-price execution path.

**Step 4 — The new owner cannot discover stale updaters without off-chain event scanning.**

`isUpdater` is a public mapping but is not enumerable on-chain. The new owner has no `getUpdaters(provider)` view; they must replay `UpdaterGranted` / `UpdaterRevoked` events to reconstruct the updater set. Until they do and call `revokeUpdater()` for each one, the old updater retains full `setConfidence` access. [8](#0-7) 

---

### Impact Explanation

The corrupted value is `confidenceParam` inside `PriceProvider` / `AnchoredPriceProvider`. It is consumed on every call to `getBidAndAskPrice()`, which is the sole price source for pool swaps. An old updater can:

1. **Cause feed stall** — set `confidenceParam = 0` (when `marginStep ≤ 0`) or to a value that collapses `bid` to zero, making every swap on the affected pool revert with `FeedStalled()`.
2. **Inject bad-price execution** — oscillate `confidenceParam` to produce artificially tight spreads (traders receive prices closer to mid than the oracle uncertainty warrants) or artificially wide spreads (traders pay more than the oracle uncertainty warrants), constituting unclamped bid/ask execution.

Both outcomes match the allowed impact gate: "bad-price execution: stale, inverted, unbounded, or unclamped bid/ask quote reaches a pool swap" and "broken core pool functionality causing loss of funds or unusable swap flows."

---

### Likelihood Explanation

The trigger is **unprivileged and requires no malicious initial setup beyond normal use**:

- Provider creation is permissionless; any actor can create a provider and grant themselves as an updater before transferring ownership.
- The attack requires only two normal factory calls (`grantUpdater` before transfer, then `setConfidence` after), both of which are documented, expected operations.
- The 1-minute `CONFIDENCE_COOLDOWN` limits frequency but does not prevent the attack.
- The new owner has no on-chain mechanism to enumerate and revoke stale updaters atomically.

---

### Recommendation

In `transferProviderOwnership()` in both `PriceProviderFactory` and `AnchoredProviderFactory`, either:

1. **Enumerate and revoke all updaters atomically** — maintain a per-provider `EnumerableSet` of updaters and iterate over it during transfer, clearing each entry.
2. **Add a generation counter** — store `uint256 updaterGeneration[provider]` and include it in the `isUpdater` key; incrementing it on transfer instantly invalidates all prior grants without enumeration.
3. **Require the caller to supply a revocation list** — add a `transferProviderOwnership(provider, newOwner, address[] calldata updatersToRevoke)` overload that atomically revokes before transferring.

---

### Proof of Concept

```solidity
// 1. Old owner creates provider and grants themselves as updater
address provider = factory.createPriceProvider(oracle, feedId, step, delta, base, quote);
factory.grantUpdater(provider, oldOwner); // isUpdater[provider][oldOwner] = true

// 2. Old owner transfers ownership to victim
factory.transferProviderOwnership(provider, victim);
// providerOwner[provider] == victim, but isUpdater[provider][oldOwner] still == true

// 3. Old owner (still a valid updater) manipulates confidenceParam
// (after 1-minute cooldown from any prior update)
address[] memory ps = new address[](1); ps[0] = provider;
uint256[] memory vs = new uint256[](1); vs[0] = 0; // collapse spread
factory.setConfidence(ps, vs);
// PriceProvider(provider).confidenceParam() == 0

// 4. Any pool swap using this provider now calls getBidAndAskPrice()
//    adjustedSpread = spread * 0 = 0 → bid == ask → bidOut >= askOut
//    → returns (0, type(uint128).max) → getBidAndAskPrice() reverts FeedStalled()
//    → all swaps on the pool revert

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
```
