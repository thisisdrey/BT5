### Title
`AnchoredPriceProvider` Uses L1 Staleness Check Without Future-Tolerance, Causing Swap DoS on L2 Deployments — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

### Summary

`AnchoredPriceProvider` implements the 3-parameter `_isStale(refTime, nowTs, maxDelta)` that unconditionally rejects any oracle `refTime > block.timestamp`. The sibling providers `PriceProviderL2` and `ProtectedPriceProviderL2` both carry a 4-parameter variant `_isStale(refTime, nowTs, maxDelta, futureTol)` that tolerates sequencer clock skew. `AnchoredPriceProvider` has no L2 variant, yet the protocol is deployed on L2 chains (Base, HyperEVM). When the oracle publishes a price whose `refTime` is even 1 second ahead of `block.timestamp` — normal on L2 — `_readLeg` returns `ok = false`, `getBidAndAskPrice()` reverts with `FeedStalled`, and every pool swap using this provider reverts with `PriceProviderFailed`.

### Finding Description

`PriceProvider` and `ProtectedPriceProvider` each have an L2 variant that adds `FUTURE_TOLERANCE` to handle the well-known sequencer clock-skew problem on optimistic rollups:

**L1 variant (3-param) — used in `AnchoredPriceProvider`:**
```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← any future timestamp = stale
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

**L2 variant (4-param) — present in `PriceProviderL2` / `ProtectedPriceProviderL2`:**
```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) {
        return (refTime - nowTs) > futureTol;  // ← tolerates small skew
    }
    return (nowTs - refTime) > maxDelta;
}
``` [2](#0-1) 

`AnchoredPriceProvider._readLeg` calls the 3-param version:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [3](#0-2) 

When `ok = false`, `_getBidAndAskPrice` returns the sentinel `(0, type(uint128).max)`: [4](#0-3) 

`getBidAndAskPrice()` then reverts with `FeedStalled`: [5](#0-4)

### Impact Explanation

Every pool that uses an `AnchoredPriceProvider` on an L2 chain becomes unable to execute swaps whenever the Pyth/Chainlink oracle's `refTime` is ahead of `block.timestamp`. This is not an edge case on optimistic rollups — sequencer timestamps routinely lead EVM block timestamps by 1–2 seconds. The result is a recurring, unpredictable DoS of the core swap function, matching the allowed impact gate of "broken core pool functionality causing unusable swap flows."

### Likelihood Explanation

The protocol explicitly targets Base and HyperEVM (L2 chains). The README states CREATE3 is used for deterministic multi-chain deployment. `AnchoredProviderFactory` has no L2 variant, so any `AnchoredPriceProvider` deployed on L2 carries the L1 staleness check. Oracle `refTime` leading `block.timestamp` is a documented, recurring property of L2 sequencers. No attacker action is required — normal oracle operation triggers the revert.

### Recommendation

Introduce `AnchoredPriceProviderL2` (and a matching `AnchoredProviderFactoryL2`) that adds an immutable `FUTURE_TOLERANCE` parameter and uses the 4-parameter `_isStale` identical to `PriceProviderL2`:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Apply this in `_readLeg` for both the `baseFeedId` and `quoteFeedId` legs. Bound `FUTURE_TOLERANCE` at construction (e.g., `≤ 1 hours`, matching `PriceProviderL2`). [7](#0-6) 

### Proof of Concept

1. Deploy `AnchoredProviderFactory` + `AnchoredPriceProvider` on Base (L2) with `MAX_REF_STALENESS = 60`.
2. Pyth publishes a price update; the oracle stores `refTime = block.timestamp + 2` (sequencer clock skew).
3. Pool calls `swap(...)` → `_getBidAndAskPriceX64()` → `getBidAndAskPrice()` → `_getBidAndAskPrice()` → `_readLeg(baseFeedId)`.
4. `_isStale(refTime=T+2, nowTs=T, maxDelta=60)` → `refTime > nowTs` → returns `true`.
5. `_readLeg` returns `ok = false` → `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
6. `getBidAndAskPrice()` reverts `FeedStalled` → pool reverts `PriceProviderFailed`.
7. All swaps on this pool are blocked until `block.timestamp` catches up to the oracle's `refTime`.

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
