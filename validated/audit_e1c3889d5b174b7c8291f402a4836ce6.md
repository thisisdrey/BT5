### Title
Stale Updater Permissions Persist After Provider Ownership Transfer, Enabling Unauthorized `confidenceParam` Manipulation That Reaches Pool Swaps — (`smart-contracts-poc/contracts/PriceProviderFactory.sol`, `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

Both `PriceProviderFactory.transferProviderOwnership` and `AnchoredProviderFactory.transferProviderOwnership` transfer the `providerOwner` mapping to a new address but never clear the `isUpdater[provider][*]` entries that were granted by the previous owner. Updaters authorized by the old owner retain the ability to call `setConfidence` on the factory indefinitely, modifying the `confidenceParam` of the provider even after they should have lost all access. For `PriceProvider`-backed pools (no anchor band clamp), this allows a stale updater to push a zero-spread or max-spread quote into live pool swaps.

---

### Finding Description

`transferProviderOwnership` in `PriceProviderFactory`:

```solidity
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);

    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
``` [1](#0-0) 

The `isUpdater[provider][updater]` mapping is never touched. The identical omission exists in `AnchoredProviderFactory.transferProviderOwnership`. [2](#0-1) 

The access gate for `setConfidence` is `_requireUpdater`:

```solidity
function _requireUpdater(address provider) internal view {
    if (msg.sender != providerOwner[provider] && !isUpdater[provider][msg.sender])
        revert NotProviderUpdater();
}
``` [3](#0-2) 

Because `isUpdater[provider][staleUpdater]` remains `true` after ownership transfer, the stale updater passes this check and can call `setConfidence` to invoke `PriceProvider.setConfidenceParam`: [4](#0-3) 

`confidenceParam` directly controls the bid/ask spread in `PriceProvider._getBidAndAskPrice`:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [5](#0-4) 

There is no anchor band clamp in `PriceProvider`; whatever `confidenceParam` produces flows directly into the pool's `_getBidAndAskPriceX64` call during `swap`: [6](#0-5) 

The new owner has no way to enumerate stale updaters because `isUpdater` is a plain `mapping(address provider => mapping(address updater => bool))` with no enumeration support — they cannot discover and revoke all stale entries. [7](#0-6) 

---

### Impact Explanation

**Path A — Zero-spread bad-price execution (confidenceParam → 0):**
Setting `confidenceParam = 0` makes `adjustedSpread = 0`, so `bid = ask = mid`. After `_applyBidAdjustments` / `_applyAskAdjustments` with `marginStep = 0`, `bidOut == askOut`, triggering the `bidOut >= askOut` guard and returning `(0, type(uint128).max)` → `FeedStalled` revert → swap DoS. With `marginStep > 0`, `bidOut < askOut` but the spread is purely the step bias with no oracle uncertainty component — a tighter-than-oracle quote reaches the pool, allowing traders to extract value from LPs at a spread the oracle does not support.

**Path B — Max-spread DoS (confidenceParam → CONFIDENCE_MAX = 1,000,000):**
With a non-trivial oracle `spread`, `delta = mid * spread * 1_000_000 / 1e10` can exceed `mid`, making `bid = 0` → `FeedStalled` → swap DoS. All swaps on the affected pool are blocked until the new owner resets `confidenceParam`, but the stale updater can re-trigger the condition every `CONFIDENCE_COOLDOWN` (1 minute).

Both paths affect live pool swaps. LP withdrawals are unaffected (no price read required), so principal is not directly at risk, but fee income is lost during DoS and LP positions are exposed to adverse selection during the tight-spread window.

---

### Likelihood Explanation

The trigger requires:
1. A provider owner grants updater rights to a third party (normal operational pattern for delegating confidence updates).
2. Provider ownership is transferred to a new party (e.g., team rotation, sale, protocol upgrade).
3. The old updater — now unauthorized — exploits the stale `isUpdater` entry.

Step 3 requires no special privilege; the old updater is an ordinary EOA or contract. The new owner cannot easily discover or revoke stale updaters. The `CONFIDENCE_COOLDOWN` (1 minute) limits frequency but does not prevent the attack. Likelihood is **Medium**: requires a prior grant + ownership transfer, but the window is permanent once opened.

---

### Recommendation

Clear all updater grants for the previous owner in `transferProviderOwnership`, or — since the mapping is not enumerable — require the outgoing owner to explicitly revoke all updaters before transferring, enforced on-chain:

```solidity
// Option A: require explicit revocation before transfer (revert if any updater remains)
// Option B: emit a TransferPending event and require a two-step accept by the new owner,
//           giving the old owner time to revoke updaters first.
// Option C: replace the flat isUpdater mapping with a per-owner nonce:
//   mapping(address provider => uint256 ownerNonce) public ownerNonce;
//   mapping(address provider => mapping(uint256 nonce => mapping(address updater => bool))) public isUpdater;
//   // increment ownerNonce on transferProviderOwnership to invalidate all prior grants
```

The simplest safe fix is to add a `ownerEpoch[provider]` counter incremented on every ownership transfer, and key `isUpdater` on `(provider, epoch, updater)`. All prior grants become unreachable in O(1) without enumeration.

### Proof of Concept

```
1. Alice deploys PriceProvider P via PriceProviderFactory F.
   providerOwner[P] = Alice.

2. Alice calls F.grantUpdater(P, Mallory).
   isUpdater[P][Mallory] = true.

3. Alice calls F.setConfidence([P], [500_000]) — sets confidenceParam to 500_000.
   Pool using P now quotes mid ± 50% of oracle spread.

4. Alice calls F.transferProviderOwnership(P, Bob).
   providerOwner[P] = Bob.
   isUpdater[P][Mallory] is still true — NOT cleared.

5. Bob is unaware of Mallory. Bob does not call revokeUpdater(P, Mallory)
   (he doesn't know Mallory exists; isUpdater has no enumeration).

6. Mallory calls F.setConfidence([P], [0]).
   _requireUpdater passes (isUpdater[P][Mallory] == true).
   PriceProvider(P).setConfidenceParam(0) succeeds.
   confidenceParam = 0.

7. Next swap on the pool backed by P:
   adjustedSpread = spread * 0 = 0
   bid = ask = mid
   If marginStep == 0: bidOut >= askOut → FeedStalled → swap reverts (DoS).
   If marginStep > 0: tight quote (no oracle spread) reaches pool → bad-price execution.

8. Mallory repeats step 6 every CONFIDENCE_COOLDOWN (1 minute) to maintain the attack.
``` [8](#0-7) [9](#0-8) [10](#0-9)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
