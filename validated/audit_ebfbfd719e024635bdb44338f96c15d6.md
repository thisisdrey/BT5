### Title
`PriceProvider` with `marginStep < 0` and default `confidenceParam = 0` permanently stalls all pool swaps — (`File: smart-contracts-poc/contracts/PriceProvider.sol`)

### Summary

`PriceProvider` (and `ProtectedPriceProviderL2`) zero-initialize `confidenceParam` in storage. `PriceProviderFactory.createPriceProvider` accepts any `marginStep` in `(-BPS_BASE, BPS_BASE)` — explicitly including negative values — but never accepts or sets `confidenceParam`. When `marginStep < 0` and `confidenceParam == 0`, the bid/ask computation always produces `bidOut ≥ askOut`, returning the stall sentinel `(0, type(uint128).max)` on every call. Every pool swap then reverts with `FeedStalled` / `PriceProviderFailed` until a separate `setConfidenceParam` call is made.

### Finding Description

`PriceProvider._getBidAndAskPrice()` computes the quote in two stages:

**Stage 1 — spread from confidence:**
```solidity
uint256 adjustedSpread = spread * confidenceParam;   // = 0 when confidenceParam == 0
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
// → delta = 0, bid = mid, ask = mid
``` [1](#0-0) 

**Stage 2 — marginStep scaling:**
```solidity
stepBidFactor = BPS_BASE_U - marginStep   // > BPS_BASE_U when marginStep < 0
stepAskFactor = BPS_BASE_U + marginStep   // < BPS_BASE_U when marginStep < 0

bidOut = floor(mid * Q64 * stepBidFactor / STEP_DENOM)   // LARGER than mid-scaled
askOut = ceil (mid * Q64 * stepAskFactor / STEP_DENOM)   // SMALLER than mid-scaled
``` [2](#0-1) 

Because `stepBidFactor > stepAskFactor` and both are applied to the same `mid` (no spread), `bidOut > askOut` always. The hard-invariant guard then returns the stall sentinel:

```solidity
// 7. Hard invariant: bid must be strictly less than ask.
//    Can be violated when marginStep < 0 and confidence is too small.
if (bidOut >= askOut) return (0, type(uint128).max);
``` [3](#0-2) 

`getBidAndAskPrice()` then reverts with `FeedStalled`: [4](#0-3) 

The pool's `_getBidAndAskPriceX64` wraps this in a `try/catch` and re-reverts with `PriceProviderFailed`, blocking every swap: [5](#0-4) 

**The factory gap:** `createPriceProvider` accepts `_marginStep` (negative values pass the bounds check) but has no `_confidenceParam` parameter and never calls `setConfidenceParam` internally. The provider is deployed in a permanently broken state: [6](#0-5) 

The code itself acknowledges the invariant can be violated (line 227 comment), but the factory provides no guard against deploying into this state.

Note: `AnchoredPriceProvider` is **not** affected — its `_computeBidAsk` applies a band clamp (`Math.min(refBid, cBid)` / `Math.max(refAsk, cAsk)`) that restores ordering regardless of `marginStep` sign. [7](#0-6) 

### Impact Explanation

All swaps on any pool whose active price provider is a `PriceProvider` (or `ProtectedPriceProviderL2`) deployed with `marginStep < 0` and `confidenceParam == 0` revert unconditionally. No swap can settle, no LP can rebalance, and no trader can exit a position through the pool. This satisfies the "broken core pool functionality / unusable swap flows" impact gate.

### Likelihood Explanation

`createPriceProvider` is **permissionless** — any address can deploy a provider with any valid `marginStep`, including negative values. The factory documentation and constructor both treat negative `marginStep` as a legitimate configuration (it is intended to tighten the pre-clamp quote in the `AnchoredPriceProvider` context). A pool operator who deploys a `PriceProvider` with `marginStep < 0` and attaches it to a pool before calling `setConfidenceParam` — a two-step flow with no atomic enforcement — triggers the DoS. The `Multicall` mixin exists on the factory but is not required, and there is no on-chain check that prevents the provider from being set on a pool in the broken state.

### Recommendation

1. Add a `confidenceParam` argument to `createPriceProvider` (and `createPriceProviderL2`) and call `setConfidenceParam` atomically during deployment when `marginStep <= 0`.
2. Alternatively, add a constructor-time validation: if `_marginStep <= 0`, require that a minimum `confidenceParam` is supplied and stored immediately.
3. At minimum, add a view function `isUsable() → bool` that pools or integrators can call to verify the provider will not stall before attaching it.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Scenario: factory deploys provider with marginStep = -1e14 (negative, valid range),
// confidenceParam stays 0 (never set). Pool swap reverts.

PriceProviderFactory factory = new PriceProviderFactory(admin);

// Permissionless: anyone can call this
address provider = factory.createPriceProvider(
    address(oracle),
    FEED_ID,
    -1e14,          // negative marginStep — passes bounds check
    1 days,
    BASE_TOKEN,
    QUOTE_TOKEN
);
// confidenceParam == 0 at this point; setConfidenceParam NOT called

// Pool admin attaches provider (semi-trusted action, but factory gave no warning)
pool.setPriceProvider(provider);

// Any swap now:
// _getBidAndAskPriceX64() → getBidAndAskPrice() →
//   adjustedSpread = spread * 0 = 0
//   bid = ask = mid
//   bidOut = floor(mid * Q64 * (BPS_BASE_U + 1e14) / STEP_DENOM)  > askOut
//   askOut = ceil (mid * Q64 * (BPS_BASE_U - 1e14) / STEP_DENOM)
//   bidOut >= askOut → return (0, type(uint128).max)
//   → revert FeedStalled()
//   → revert PriceProviderFailed(...)
pool.swap(...); // ← always reverts
```
