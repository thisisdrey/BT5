### Title
Single `MAX_REF_STALENESS` Applied to Both Legs of Synthetic Pair Allows Stale Quote-Feed Price to Reach Pool Swaps — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports synthetic ratio pricing (e.g., BTC/ETH = BTC/USD ÷ ETH/USD). A single immutable `MAX_REF_STALENESS` is applied identically to both the base and quote feed legs inside `_readLeg`. The `AnchoredProviderFactory` validates `maxRefStaleness` only against the **base feed's** class envelope; the quote feed receives no per-leg staleness envelope check. When the quote feed has a shorter heartbeat than `MAX_REF_STALENESS` (e.g., ETH/USD at 1 h vs. BTC/USD at 24 h), a stale quote-leg price passes the staleness guard and a corrupted synthetic mid reaches `getBidAndAskPrice`, which the pool uses directly for swap execution.

---

### Finding Description

`AnchoredPriceProvider._readLeg` applies the same `MAX_REF_STALENESS` to every feed it reads:

```solidity
// AnchoredPriceProvider.sol line 283
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
```

`_getBidAndAskPrice` calls `_readLeg` for both legs:

```solidity
// lines 259-268
(uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
...
(uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
...
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // synthetic ratio
```

`MAX_REF_STALENESS` is an immutable set at construction time. The factory validates it only against the **base feed's** class envelope:

```solidity
// AnchoredProviderFactory.sol lines 171-180
bytes32 classId = feedClass[baseFeedId];
if (classId == bytes32(0)) classId = DEFAULT_CLASS;
Envelope storage env = envelopes[classId];
if (
    ...
    || maxRefStaleness < env.stalenessMin || maxRefStaleness > env.stalenessMax
    ...
) revert ParamsOutOfEnvelope();
```

The factory's own NatSpec confirms this design gap:

> *"The envelope is keyed on `baseFeedId` (the provider's class); the ref feed only contributes its uncertainty and is validated for existence at provider construction."*

No envelope, no per-leg staleness bound, and no separate `MAX_REF_STALENESS` exist for `quoteFeedId`. A permissionless creator can therefore deploy a synthetic provider where `MAX_REF_STALENESS` is calibrated to the base feed's 24 h heartbeat while the quote feed (e.g., ETH/USD) has a 1 h heartbeat. After the quote feed goes 2 h without an update, its `refTime` is still within `MAX_REF_STALENESS`, the staleness guard passes, and the stale ETH/USD price is used to compute the synthetic mid.

---

### Impact Explanation

A stale quote-leg price corrupts the synthetic mid that `_computeBidAsk` uses to derive the reference band and the final bid/ask returned to the pool. The pool calls `getBidAndAskPrice()` at swap time and executes at those prices. If ETH/USD has drifted 3–5 % during the stale window, the synthetic BTC/ETH price is proportionally wrong. Traders can exploit the mispriced band to extract value from LPs: buying the underpriced asset or selling the overpriced one at the stale rate. This is a direct loss of LP principal, matching the "bad-price execution" and "swap conservation failure" impact gates.

---

### Likelihood Explanation

- `AnchoredProviderFactory.createAnchoredProvider` is permissionless; any address can deploy a synthetic provider.
- The admin sets class envelopes keyed on the base feed only; no mechanism forces the creator to account for the quote feed's heartbeat.
- Feeds with mismatched heartbeats (e.g., BTC/USD 24 h, ETH/USD 1 h) are common in production Chainlink deployments.
- The stale window opens naturally during low-volatility periods when the quote feed's deviation threshold is not triggered; no attacker action is needed to create the stale condition.

Likelihood: **Low-Medium** (requires a synthetic pair with mismatched heartbeats and a price move during the stale window, but no privileged or malicious action is required).

---

### Recommendation

1. **Per-leg staleness**: Add a second immutable `MAX_REF_STALENESS_QUOTE` (or pass it as a constructor argument) and apply it exclusively to the quote-leg read in `_readLeg`.
2. **Factory envelope for quote feed**: In `AnchoredProviderFactory.createAnchoredProvider`, resolve the quote feed's class and validate `maxRefStalenessQuote` against its envelope, symmetrically with the base feed.
3. **Minimum-of-two fallback**: As a simpler alternative, enforce `MAX_REF_STALENESS = min(baseFeedHeartbeat, quoteFeedHeartbeat)` by requiring the creator to supply both heartbeats and taking the minimum.

---

### Proof of Concept

**Setup:**
- Admin sets BTC/USD class envelope with `stalenessMax = 86400` (24 h).
- Creator calls `createAnchoredProvider(oracle, BTC_USD_FEED, ETH_USD_FEED, ..., maxRefStaleness=86400, ...)`.
- Factory validates `86400 <= stalenessMax` for BTC/USD class — passes. No check on ETH/USD.
- Provider deployed with `MAX_REF_STALENESS = 86400`.

**Attack:**
1. ETH/USD oracle last updated 2 h ago (`refTime = block.timestamp - 7200`). ETH price has dropped 4 % since then.
2. BTC/USD oracle is fresh.
3. Attacker calls pool swap (buy BTC with ETH).
4. Pool calls `provider.getBidAndAskPrice()`.
5. `_readLeg(ETH_USD_FEED)`: `nowTs - refTime = 7200 < 86400` → staleness check **passes**.
6. Stale ETH/USD mid (4 % too high) is used: synthetic BTC/ETH mid = BTC_USD / ETH_USD_stale → BTC/ETH is 4 % **underpriced**.
7. Attacker buys BTC at the stale (underpriced) rate, extracting ~4 % of the swap value from LPs. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

```text
    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }
```

```text
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
```

```text
    function createAnchoredProvider(
        address oracle,
        bytes32 baseFeedId,
        bytes32 quoteFeedId,
        uint256 minMargin,
        uint256 maxRefStaleness,
        uint16  maxSpreadBps,
        bool    mutableParams,
        int256  marginStep,
        address baseToken,
        address quoteToken
    ) external override returns (address provider) {
        if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);

        // Feeds without an explicit class fall back to the admin-configured DEFAULT_CLASS envelope.
        bytes32 classId = feedClass[baseFeedId];
        if (classId == bytes32(0)) classId = DEFAULT_CLASS;

        Envelope storage env = envelopes[classId];
        if (!env.exists) revert EnvelopeNotFound(classId);
        if (
            minMargin < env.minMarginMin || minMargin > env.minMarginMax
            || maxRefStaleness < env.stalenessMin || maxRefStaleness > env.stalenessMax
            || maxSpreadBps < env.maxSpreadMin || maxSpreadBps > env.maxSpreadMax
        ) revert ParamsOutOfEnvelope();
```
