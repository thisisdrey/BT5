### Title
Stale Updater Permissions Persist After Provider Ownership Transfer, Enabling Unauthorized `confidenceParam` Manipulation That Reaches Pool Swaps - (File: `smart-contracts-poc/contracts/PriceProviderFactory.sol`, `smart-contracts-poc/contracts/PriceProviderFactoryL2.sol`, `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`transferProviderOwnership` in all three factory contracts updates `providerOwner` but never clears the `isUpdater[provider][*]` mapping. Old updaters granted by the previous owner retain the ability to call `setConfidence`, which writes `confidenceParam` directly into the live `PriceProvider` / `PriceProviderL2`. Because `confidenceParam` is the sole multiplier on the oracle spread that produces the bid/ask prices consumed by pool swaps, a stale updater can silently corrupt or halt the price feed for any pool using that provider.

---

### Finding Description

The structural analog to the external bug is exact:

| External (LimitOrderManager) | Metric OMM (PriceProviderFactory) |
|---|---|
| `_limitPositions[tokenId]` (operator) not cleared on `_transfer` | `isUpdater[provider][updater]` not cleared on `transferProviderOwnership` |
| Old operator can burn/collect on behalf of new NFT owner | Old updater can call `setConfidence` on behalf of new provider owner |

In `PriceProviderFactory.transferProviderOwnership`:

```solidity
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;                          // owner updated
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);
    // ← isUpdater[provider][*] is NEVER cleared
    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
``` [1](#0-0) 

The same omission exists verbatim in `PriceProviderFactoryL2.transferProviderOwnership` and `AnchoredProviderFactory.transferProviderOwnership`. [2](#0-1) [3](#0-2) 

The `_requireUpdater` check that gates `setConfidence` passes for any address where `isUpdater[provider][msg.sender] == true`, regardless of whether the granting owner still owns the provider:

```solidity
function _requireUpdater(address provider) internal view {
    if (msg.sender != providerOwner[provider] && !isUpdater[provider][msg.sender])
        revert NotProviderUpdater();
}
``` [4](#0-3) 

`setConfidence` then calls `PriceProvider.setConfidenceParam`, which writes `confidenceParam` — the sole multiplier on the oracle spread:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [5](#0-4) 

This `bid`/`ask` pair is what the pool consumes directly during every swap via `getBidAndAskPrice()`. [6](#0-5) 

---

### Impact Explanation

**Case 1 — Pool halt (broken core swap functionality):**
When `marginStep = 0` (the common default), setting `confidenceParam = 0` makes `adjustedSpread = 0`, so `bid = mid` and `ask = mid`. After the step adjustment both sides are equal, triggering the hard invariant check:

```solidity
if (bidOut >= askOut) return (0, type(uint128).max);
``` [7](#0-6) 

`getBidAndAskPrice()` then reverts with `FeedStalled`, halting all swaps through every pool that uses this provider. LPs cannot exit via swap; the pool is effectively bricked until the new owner discovers the stale updater and revokes it.

**Case 2 — Bad-price execution (wide spread):**
Setting `confidenceParam` to a large value (up to `CONFIDENCE_MAX = 1_000_000`, a 100× multiplier) produces an extremely wide spread. Traders execute at prices far from the oracle mid, suffering direct principal loss on every swap. The pool does not revert — it silently executes at the manipulated price. [8](#0-7) 

For `AnchoredPriceProvider` (mutable variant), the band clamp limits tightening but not widening; an excessively high confidence causes `bid8 = 0` → `_shapedQuote` returns `(false, 0, 0)` → pool halts via the same `FeedStalled` path. [9](#0-8) 

---

### Likelihood Explanation

Provider ownership transfer is an explicitly supported operation (`transferProviderOwnership` exists in all three factories). The previous owner may have granted updater rights to automated bots, co-curators, or third-party services. The new owner has no on-chain mechanism to enumerate all granted updaters (the `isUpdater` mapping is not iterable), so they cannot know which addresses to revoke. The stale updater can act immediately after the transfer with no additional preconditions, subject only to the 1-minute `CONFIDENCE_COOLDOWN`. [10](#0-9) 

---

### Recommendation

Clear all updater grants atomically inside `transferProviderOwnership`. Because the `isUpdater` mapping is not iterable, the cleanest fix is to introduce a per-provider `updaterEpoch` counter and include it in the `_requireUpdater` check:

```solidity
mapping(address provider => uint256) public updaterEpoch;
mapping(address provider => mapping(uint256 epoch => mapping(address updater => bool))) public isUpdater;

function transferProviderOwnership(address provider, address newOwner) external onlyProviderOwner(provider) {
    ...
    updaterEpoch[provider]++;   // invalidates all existing updater grants atomically
    ...
}
```

Alternatively, emit a `ProviderOwnershipTransferred` event that off-chain tooling can use to warn the new owner, and document that all updaters must be manually revoked before or immediately after transfer.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Setup:
// 1. Alice deploys a PriceProvider via PriceProviderFactory (marginStep = 0).
// 2. Alice grants updater rights to Bob: factory.grantUpdater(provider, bob).
// 3. Alice sets confidenceParam to a working value: factory.setConfidence([provider], [100_000]).
// 4. Alice transfers provider ownership to Carol: factory.transferProviderOwnership(provider, carol).
//    → isUpdater[provider][bob] is still true.
// 5. Carol is unaware of Bob's updater grant.
// 6. Bob waits 1 minute (CONFIDENCE_COOLDOWN), then calls:

//    → PriceProvider.confidenceParam = 0
// 7. Any pool using this provider now reverts FeedStalled on every swap.
//    → All swaps halted; LPs cannot exit via swap path.

// Alternatively in step 6, Bob sets confidenceParam = 1_000_000 (CONFIDENCE_MAX):
//    → adjustedSpread = oracleSpread * 1_000_000 (100× wider)
//    → Traders execute at prices 100× further from mid than intended.
