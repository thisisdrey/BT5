### Title
`AnchoredPriceProvider` L1-Only Staleness Check Causes Permanent `FeedStalled` on L2 Deployments — (File: `smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider`, documented as "the one standard provider for public pools," hard-codes an L1 staleness check that unconditionally rejects any oracle `refTime` ahead of `block.timestamp`. On L2 chains (Arbitrum, Base), the sequencer's clock routinely lags behind oracle publication time, so valid oracle data arrives with `refTime > block.timestamp`. The provider treats this as stale, reverts every swap with `FeedStalled`, and renders the pool permanently unusable — the direct structural analog to the Convex Booster interface mismatch on Arbitrum.

---

### Finding Description

`AnchoredPriceProvider._isStale` uses L1 semantics:

```solidity
// AnchoredPriceProvider.sol L222-230
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← hard L1 rule: any future refTime = stale
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

This is called from `_readLeg`, which returns `ok=false` on a stale result:

```solidity
// AnchoredPriceProvider.sol L280-295
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [2](#0-1) 

`_getBidAndAskPrice` propagates `ok=false` as the sentinel `(0, type(uint128).max)`:

```solidity
// AnchoredPriceProvider.sol L258-260
(uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
if (!ok) return (0, type(uint128).max);
``` [3](#0-2) 

And `getBidAndAskPrice` reverts:

```solidity
// AnchoredPriceProvider.sol L214-217
function getBidAndAskPrice() external override returns (uint128 bid, uint128 ask) {
    (bid, ask) = _getBidAndAskPrice();
    if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
}
``` [4](#0-3) 

The pool catches this revert and re-reverts with `PriceProviderFailed`, blocking every swap:

```solidity
// MetricOmmPool.sol L804-813
function _getBidAndAskPriceX64() internal returns (uint128 bidPriceX64, uint128 askPriceX64) {
    address activePriceProvider = _resolvedPriceProvider();
    try IPriceProvider(activePriceProvider).getBidAndAskPrice() returns (uint128 bid, uint128 ask) {
        ...
    } catch (bytes memory reason) {
        revert PriceProviderFailed(reason);
    }
}
``` [5](#0-4) 

**The protocol already knows L2 requires a different staleness check.** Both `PriceProviderL2` and `ProtectedPriceProviderL2` accept a `FUTURE_TOLERANCE` constructor parameter and use a four-argument `_isStale` that tolerates oracle timestamps slightly ahead of `block.timestamp`:

```solidity
// PriceProviderL2.sol L135-150
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) {
        return (refTime - nowTs) > futureTol;   // ← L2-aware: tolerate clock skew
    }
    return (nowTs - refTime) > maxDelta;
}
``` [6](#0-5) 

`AnchoredPriceProvider` — the "standard provider for public pools" — has **no L2 variant** and no `FUTURE_TOLERANCE` parameter. `AnchoredProviderFactory.createAnchoredProvider` is permissionless; any curator can deploy one on L2 without any warning. [7](#0-6) 

---

### Impact Explanation

Every swap on a pool whose price provider is an `AnchoredPriceProvider` deployed on L2 reverts with `PriceProviderFailed(FeedStalled)` whenever the oracle's `refTime` exceeds `block.timestamp` — a routine condition on Arbitrum and Base. Swaps are permanently blocked; LP positions are stranded (liquidity cannot be rebalanced or arbitraged). This satisfies the allowed impact gate: **broken core pool functionality causing unusable swap/liquidity flows**.

---

### Likelihood Explanation

Medium. The protocol ships L2 variants for every other provider type, signalling planned L2 deployment. `AnchoredProviderFactory.createAnchoredProvider` is permissionless, so any pool curator can deploy an `AnchoredPriceProvider` on L2 without realising it carries L1-only staleness semantics. The oracle clock-skew condition (`refTime > block.timestamp`) is a normal, non-adversarial occurrence on L2 sequencer chains.

---

### Recommendation

Introduce `AnchoredPriceProviderL2` mirroring the pattern already used by `PriceProviderL2` and `ProtectedPriceProviderL2`:

```solidity
uint256 public immutable FUTURE_TOLERANCE;

function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta) internal view returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) {
        return (refTime - nowTs) > FUTURE_TOLERANCE;
    }
    return (nowTs - refTime) > maxDelta;
}
```

Add a corresponding `AnchoredProviderFactoryL2` that validates `FUTURE_TOLERANCE <= 1 hours` (matching the bound in `PriceProviderL2`) and deploys the L2 variant.

---

### Proof of Concept

1. Curator calls `AnchoredProviderFactory.createAnchoredProvider(oracle, feedId, ...)` on Arbitrum — succeeds, returns a valid `AnchoredPriceProvider`.
2. Pool is wired to this provider.
3. Pyth/Chainlink oracle publishes a price update; the oracle contract stores `refTime = T`.
4. Due to Arbitrum sequencer lag, `block.timestamp = T - 2` when the swap is executed.
5. Pool calls `swap(...)` → `_getBidAndAskPriceX64()` → `AnchoredPriceProvider.getBidAndAskPrice()`.
6. `_readLeg` calls `_isStale(T, T-2, MAX_REF_STALENESS)` → `T > T-2` → returns `true`.
7. `_readLeg` returns `ok=false`; `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
8. `getBidAndAskPrice` reverts `FeedStalled`.
9. Pool catches and reverts `PriceProviderFailed`.
10. Every swap reverts for as long as the sequencer lag persists — which is the normal operating condition on L2.

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
