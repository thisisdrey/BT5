### Title
`AnchoredPriceProvider` Uses L1-Only Staleness Check With No L2 Variant, Causing Valid Oracle Prices to Be Rejected as Stale on Base — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` is documented as "the one standard provider for public pools." Its `_isStale()` function carries an explicit `L1` annotation and unconditionally rejects any oracle `refTime` that is ahead of `block.timestamp`. On Base (an L2 the protocol explicitly targets), Pyth's `publishTime` can be a few seconds ahead of the L2 sequencer's `block.timestamp` due to clock skew. Every other provider family in the codebase ships a paired L2 variant (`PriceProviderL2`, `ProtectedPriceProviderL2`) that adds a `FUTURE_TOLERANCE` window to absorb this skew. `AnchoredPriceProvider` has no such variant and no such window, so on Base it will intermittently reject fresh, valid oracle prices as stale, reverting every swap with `FeedStalled` → `PriceProviderFailed`.

---

### Finding Description

`AnchoredPriceProvider._isStale()` is the L1 variant:

```solidity
/// @dev Pure staleness check (L1). Any future refTime is stale.
function _isStale(
    uint256 refTime,
    uint256 nowTs,
    uint256 maxDelta
) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← hard L1 assumption
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

This is called from `_readLeg`, which is the sole path for reading the anchor oracle:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [2](#0-1) 

When `ok == false`, `_getBidAndAskPrice` returns the sentinel `(0, type(uint128).max)`, and `getBidAndAskPrice` reverts with `FeedStalled`: [3](#0-2) 

The pool propagates this as `PriceProviderFailed`, blocking every swap: [4](#0-3) 

The codebase already knows this is an L2 problem. `PriceProviderL2` and `ProtectedPriceProviderL2` both carry a `FUTURE_TOLERANCE` immutable (capped at 1 hour) and an L2-aware `_isStale`:

```solidity
/// @dev L2 sequencer timestamp can lag behind oracle publication time.
///      Allows refTime up to FUTURE_TOLERANCE seconds ahead of block.timestamp.
uint256 public immutable FUTURE_TOLERANCE;
``` [5](#0-4) 

```solidity
if (refTime > nowTs) {
    return (refTime - nowTs) > futureTol;
}
``` [6](#0-5) 

`AnchoredPriceProvider` has no L2 counterpart and no `FUTURE_TOLERANCE`. `AnchoredProviderFactory` likewise has no L2 variant:

---

### Impact Explanation

On Base, when Pyth's `publishTime` is even 1 second ahead of the L2 sequencer's `block.timestamp` (a normal occurrence due to clock skew), `_isStale` returns `true`, `_readLeg` returns `ok = false`, and every call to `getBidAndAskPrice()` reverts with `FeedStalled`. Because `AnchoredPriceProvider` is "the one standard provider for public pools," this breaks swap execution for all pools using it on Base. Swaps revert, LPs cannot rebalance, and traders cannot exit positions — matching the "broken core pool functionality / unusable swap flows" impact gate.

---

### Likelihood Explanation

Base is an explicitly targeted deployment chain. Pyth oracle `publishTime` values being slightly ahead of L2 `block.timestamp` is a documented, recurring phenomenon on Optimism-stack chains — it is precisely why `PriceProviderL2` and `ProtectedPriceProviderL2` exist. The omission of an equivalent `AnchoredPriceProviderL2` means every public pool on Base using the anchored provider is exposed to intermittent swap failures during normal oracle operation. No attacker action is required; normal Pyth price updates trigger the condition.

---

### Recommendation

Introduce `AnchoredPriceProviderL2` (and a matching `AnchoredProviderFactoryL2`) that mirrors the L2-aware `_isStale` already present in `PriceProviderL2` and `ProtectedPriceProviderL2`:

```solidity
uint256 public immutable FUTURE_TOLERANCE; // capped at 1 hours in constructor

function _isStale(
    uint256 refTime,
    uint256 nowTs,
    uint256 maxDelta,
    uint256 futureTol
) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Pass `FUTURE_TOLERANCE` through `_readLeg` and validate it against a per-class envelope bound in `AnchoredProviderFactoryL2.setEnvelope`.

---

### Proof of Concept

1. Deploy `AnchoredPriceProvider` on Base with `MAX_REF_STALENESS = 60` (60 s).
2. Pyth publishes a price update with `publishTime = block.timestamp + 2` (2 s ahead — normal on Base).
3. Pool calls `swap(...)` → `_getBidAndAskPriceX64()` → `getBidAndAskPrice()` → `_getBidAndAskPrice()` → `_readLeg(baseFeedId)`.
4. `_isStale(publishTime, block.timestamp, 60)` evaluates `publishTime > block.timestamp` → returns `true`.
5. `_readLeg` returns `ok = false` → `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
6. `getBidAndAskPrice` reverts `FeedStalled`.
7. Pool catches and reverts `PriceProviderFailed` — swap is blocked.
8. This repeats for every Pyth update whose `publishTime` is ahead of the sequencer clock, making the pool intermittently unusable on Base.
