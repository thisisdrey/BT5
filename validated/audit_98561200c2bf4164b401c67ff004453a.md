### Title
`AnchoredProviderFactory::createAnchoredProvider` Permits `baseFeedId == quoteFeedId`, Collapsing Synthetic Ratio to a Constant 1.0 and Enabling Bad-Price Execution — (`smart-contracts-poc/contracts/AnchoredProviderFactory.sol`, `smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredProviderFactory.createAnchoredProvider` is permissionless (no role gate beyond the oracle allow-list) and accepts `baseFeedId` and `quoteFeedId` as independent `bytes32` parameters. Neither the factory nor the `AnchoredPriceProvider` constructor validates that `baseFeedId != quoteFeedId`. When both are equal, the synthetic-ratio path in `_getBidAndAskPrice` divides the feed price by itself, producing a hard-coded mid of exactly `1e8` (1.0 in 8-decimal format) regardless of the actual market price. The resulting provider passes `isProvider(p) == true`, is indistinguishable from a legitimate factory-deployed provider, and if attached to a pool will execute every swap at a price of ~1.0, allowing arbitrageurs to drain LP funds.

---

### Finding Description

**Root cause — missing `baseFeedId != quoteFeedId` guard in both the factory and the constructor.**

`AnchoredProviderFactory.createAnchoredProvider` validates the oracle allow-list and envelope bounds, but never checks whether the two feed IDs are distinct: [1](#0-0) 

The `AnchoredPriceProvider` constructor stores both IDs without any equality check: [2](#0-1) 

The only token-level guard (`_baseToken != _quoteToken`) does not protect the feed-ID binding: [3](#0-2) 

**Corrupted value — synthetic ratio degenerates to 1.0.**

When `quoteFeedId != bytes32(0)` the provider reads both legs and computes:

```
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // mid * 1e8 / mid = 1e8
spreadBps += spreadBps2;                          // spread doubled
``` [4](#0-3) 

If `baseFeedId == quoteFeedId`, both `_readLeg` calls return the same oracle value. The division `mid * 1e8 / mid` yields exactly `1e8` — a constant mid of 1.0 in 8-decimal format — for every block, every price level, every market condition. The doubled spread is then applied around this constant, producing a valid-looking `bid < ask` pair that passes all downstream guards: [5](#0-4) 

**Pool consumption — bad price reaches swap execution.**

The pool calls `getBidAndAskPrice()` unconditionally at swap time and uses the returned pair to derive `midPriceX64` and `baseFeeX64` for all bin math: [6](#0-5) [7](#0-6) 

A mid of 1.0 (Q64 ≈ `1 << 64`) is accepted as a valid price; the pool's only post-call checks (`bid >= ask`, `bid == 0`) both pass. Every subsequent swap executes against a price of ~1.0 token1 per token0, regardless of the true market rate.

---

### Impact Explanation

For any non-1:1 pool (e.g., ETH/USDC where the true price is ~3 000), the pool will quote and settle swaps at ~1 USDC per ETH. An arbitrageur can buy ETH from the pool at 1 USDC and sell it on the open market at 3 000 USDC, repeating until all token0 liquidity is drained. LP principal is lost in full; the pool is insolvent. This is a direct, quantifiable loss of user funds matching the "bad-price execution" and "pool insolvency" impact gates.

---

### Likelihood Explanation

`createAnchoredProvider` carries no role restriction — any address can call it with an allow-listed oracle: [8](#0-7) 

The resulting provider is registered as `isProvider(p) == true` and is indistinguishable from a correctly configured provider. The factory's documented "audit-once guarantee" — that every provider it deploys is clamp-bounded and safe — is broken for this case. A pool deployer or admin who relies on `isProvider` as the safety predicate will accept the provider without further inspection. The misconfiguration can also occur accidentally (copy-paste of the same feed ID for both parameters).

---

### Recommendation

Add a guard in both `AnchoredProviderFactory.createAnchoredProvider` and the `AnchoredPriceProvider` constructor:

```solidity
// In AnchoredProviderFactory.createAnchoredProvider, before deploying:
if (quoteFeedId != bytes32(0) && quoteFeedId == baseFeedId)
    revert SameFeedId();

// In AnchoredPriceProvider constructor, after storing the IDs:
require(
    _quoteFeedId == bytes32(0) || _quoteFeedId != _baseFeedId,
    "baseFeedId == quoteFeedId"
);
```

The factory-level check is the primary defence (it is the "audit-once" boundary); the constructor check is a defence-in-depth backstop for direct deployments.
