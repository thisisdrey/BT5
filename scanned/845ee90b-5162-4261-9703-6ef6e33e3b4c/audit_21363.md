# The keygen binary saves keyﬁles with overly broad permissions

## Summary
A bug has been found in the mobilecoin/consensus/scp/src/slot.rs target where the keygen binary built by the util/keyfile project creates private key files with 0666 permissions. This could potentially allow an attacker to read the private key file if they gain access to the machine on which the keys are generated. To prevent this, the write_keyfile function should be changed to create private key files with 0700 permissions. In the long term, it is recommended to investigate whether private keys could be offloaded to a service such as Vault to avoid handling sensitive information.

## Description  
The `keygen` binary built by the `util/keyfile` project creates private key files with `0666` permissions. Depending on the configuration of Linux `umask`, this may allow an attacker to read the private key file if he gets filesystem access to the machine on which the keys are generated.  

Figure 8.1 shows a way to see the permissions that the keyfiles are created with.  
```bash
$ strace -e file ./keygen --name keyfile
// (...)
openat(AT_FDCWD, "/home/disconnect3d/mobilecoin/target/debug/keyfile.json",
O_WRONLY|O_CREAT|O_TRUNC|O_CLOEXEC,  0666  ) = 3
openat(AT_FDCWD, "/home/disconnect3d/mobilecoin/target/debug/keyfile.pub",
O_WRONLY|O_CREAT|O_TRUNC|O_CLOEXEC,  0666  ) = 3
```
**Figure 8.1:** Using `strace` to see the permissions that the keyfiles are created with  

## Exploit Scenario  
Alice generates her keys through the `keygen` tool. Eve, another user on the same system, reads the generated private key file due to its overly broad permissions and hijacks Alice's node or account.  

## Recommendations  
**Short term:** Change the `write_keyfile` function to create private key files with `0700` permissions. This will prevent other users on the same system from accessing private keys generated with the `keygen` utility.  

**Long term:** Investigate whether the handling of users’ private keys could be offloaded to a service such as `Vault`. Avoid handling sensitive information to prevent its disclosure.
