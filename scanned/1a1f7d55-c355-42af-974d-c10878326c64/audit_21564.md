# Certain functions lack input validation routines ✓ Fixed

## Summary
This bug report is about the lack of argument checks in the functions of the Orchestrator and Loihi contracts. In particular, the functions `includeAsset` and `includeAssimilator` do not do any checks before changing the contract state. Additionally, the user facing function `swapByOrigin` does not check any of the arguments when it calls the `MainnetDaiToDaiAssimilator`. The `checks-effects-interactions` pattern is recommended to be implemented throughout the code. This pattern should include checks for `uint` values larger than `0`, `uint` values within constraints, positive `int` values in some cases, array lengths matching if more arrays are sent as arguments, and addresses not being `0x0`. Tests should also be added to check if all of the arguments have been validated.

## Details
#### Resolution



*Comment from the development team*:


1. Now all functions in the Orchestrator revert on incorrect arguments.
2. All functions in Loihi in general revert on incorrect arguments.




#### Description


The functions should first check if the passed arguments are valid first. The [checks-effects-interactions](https://solidity.readthedocs.io/en/develop/security-considerations.html#use-the-checks-effects-interactions-pattern) pattern should be implemented throughout the code.


These checks should include, but not be limited to:


* `uint` should be larger than `0` when `0` is considered invalid
* `uint` should be within constraints
* `int` should be positive in some cases
* length of arrays should match if more arrays are sent as arguments
* addresses should not be `0x0`

#### Recommendation


Implement the `checks-effects-interactions` as a pattern to write code. Add tests that check if all of the arguments have been validated.


Consider checking arguments as an important part of writing code and developing the system.
