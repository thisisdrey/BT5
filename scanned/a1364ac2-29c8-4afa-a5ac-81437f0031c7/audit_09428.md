# Protocol functionality lockout upon adding a new token

## Summary
This bug report is about an issue where adding a supported token to the project requires three separate function calls. These functions are addNewSupportedAsset() in LRTConfig, updateAssetStrategy() in LRTConfig, and updatePriceOracleFor() in LRTOracle. The problem arises when these function calls are made in different transactions, causing the protocol to be in an intermediate state for some time. This results in the updateRSETHPrice() function being blocked. 

To fix this issue, the recommendation is to consolidate these functions into one and make the token addition process atomic. This means that the entire process will be completed as a single unit, avoiding any intermediate states and ensuring that all functions are executed successfully. This will prevent the updateRSETHPrice() function from being blocked and improve the overall functionality of the project.

## Details
##### Description
To add a supported token to the project, the administrator and manager have to call 3 different functions:

- addNewSupportedAsset() in LRTConfig
- updateAssetStrategy() in LRTConfig
- updatePriceOracleFor() in LRTOracle

If the sequence of calls to these functions is contained in different transactions, the protocol is in an intermediate state for some time, during which the updateRSETHPrice() function is blocked.
##### Recommendation
We recommend consolidating the mentioned functions into one and making the token addition procedure atomic.
