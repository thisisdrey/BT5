# StakedToken holders can avoid losses

## Summary
This bug report discusses a vulnerability found in the StakedToken contract that could potentially lead to losses for its holders. The issue is related to the `accrue` function, which can be exploited by users to avoid these losses by redeeming their tokens right before the function is executed. The report recommends implementing a minimum unstaking period for StakedToken holders to prevent this from happening. The bug has been fixed by InfiniFi and verified by Spearbit.

## Details
## Security Report

## Severity
**Medium Risk**

## Context
StakedToken.sol#L64

## Description
Holders of StakedToken are exposed to losses when the `accrue` function triggers `applyLosses`. However, they can circumvent these losses by frontrunning the `accrue` function, redeeming their tokens moments before execution. This vulnerability enables users to sidestep their intended risk exposure entirely.

## Recommendation
Consider implementing a minimum unstaking period for StakedToken holders to prevent avoidance from loss.

## InfiniFi
Fixed in 24513bd by reverting if there are pending unaccrued losses when a user does `siUSD ! iUSD`, `liUSD ! iUSD`, or `iUSD ! USDC`.

## Spearbit
Fix verified.
