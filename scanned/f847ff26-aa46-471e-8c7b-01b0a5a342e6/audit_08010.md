# Removing owners via livenessModule does not update the guard lastlive mapping 

## Summary
The report states that there is a bug in the code related to the LivenessGuard and LivenessModule contracts. The LivenessGuard has a mapping that keeps track of when a signer was last active. When an owner is removed through a normal safe transaction, their address is removed from this mapping. However, when the LivenessModule removes owners, the transaction bypasses the guard and the mapping is not updated, resulting in a removed owner still having their lastLive value set. This breaks the rule that removed owners should not be present in the mapping. The likelihood of this bug happening is high and it impacts the functionality of the code. The recommendation is to introduce a new function in the LivenessGuard contract called removeLiveness() that can be called by the module after removing owners to update the mapping.

## Details
## Security Analysis: LivenessGuard

## Context
- **Files**: 
  - `LivenessGuard.sol#L146`
  - `LivenessModule.sol#L133-L169`

## Description
`LivenessGuard` has a mapping where the last time a signer was active is saved:

```solidity
mapping(address => uint256) public lastLive;
```

When an owner is removed through a normal safe transaction, this mapping is updated, and the old owner's address is removed from it:

```solidity
delete lastLive[ownerBefore];
```

However, when `LivenessModule.removeOwners()` is called, the transaction bypasses the guard, and the delete logic won't be executed. This will result in a removed owner still having their `lastLive` value set.

## Impact
**Severity**: Medium  
This breaks the invariant that removed owners must not be present in the `lastLive` mapping.

## Likelihood
**Likelihood**: High  
This issue occurs every time the module removes owners.

## Recommendation
To address this issue, similar to the existing `showLiveness()` function, a `removeLiveness()` function may be introduced to the Guard:

```solidity
function removeLiveness(address _account) external {
    require(!SAFE.isOwner(_account), "LivenessGuard: Account is owner");
    delete lastLive[_account];
}
```

This function can then be called by the module after removing an owner.
