# [M-01] An error in the `minPayoutSize` check in the `sell()` function

## Summary
This bug report describes a problem with a function called "sell" in a program written in a programming language called "JavaScript." The function is used to sell a type of digital token and it has two parameters: "tokensToSell" and "minPayoutSize." The severity of the bug is considered to be medium, meaning it could cause some problems but not major ones. The likelihood of the bug occurring is also considered to be medium. 

The bug is related to the "minPayoutSize" parameter, which is used to protect the user from losing money when selling the token. However, the current code does not properly check this parameter, which means that the user could still lose money even if they set a minimum payout size. The report recommends removing the check from one part of the code and moving it to another, in order to properly protect the user's funds.

## Details
## Severity

**Impact:** Medium

**Likelihood:** Medium

## Description

```javascript
 function sell(uint256 tokensToSell, uint256 minPayoutSize) external nonReentrant returns (uint256) {
        // Ensure the sender has enough liquidity to sell

```

`minPayoutSize` is a slippage protection parameter set by the user, meaning it guarantees that the user will receive at least minPayoutSize ETH in the end. Checking truePayoutSize does not achieve this protection, because payoutAfterFee could still be smaller than minPayoutSize.

## Recommendations

Remove the minPayoutSize check from the \_handleBondingCurveSell() function and move it to the sell() function, as follows:

```diff
// Calculate the payout after the fee
             uint256 payoutAfterFee = truePayoutSize - fee;
+             if (payoutAfterFee < minPayoutSize) revert SlippageTooHigh();
```
