# Path Traversal

## Summary
A bug has been reported in the client/cli/flags.go:557 file. The function `ReadProposalFlag` does not properly check the path of the file it reads, allowing any file in the filesystem to be accessed. This could potentially cause security issues. The recommended strategy to fix this bug is to implement input validation. The client has commented that the function should be able to read any file that the operating system allows, and it is not the responsibility of the code to determine this. The severity of this bug is medium and it has been acknowledged by the team.

## Details
**Severity**: Medium

**Status**: Acknowledged

**File**: client/cli/flags.go:557

**CWE**: 22

**Details**: 

The `ReadProposalFlag` function reads the file contents out of the filesystem, given by the variable propFN, into a `propFileContents` variable. However, this function doesn't check the path, and it's possible to point to any file in the filesystem.

**Recommended strategy**: 

Input Validation

**Client comment**: 

That function is supposed to be able to read any file that the file system allows the process to read. It's the OS's job to determine that and our stuff should not have any opinion on it
