# Malfunctioning Address Book

## Summary
Summary: The bug report states that when initializing the snap, the addresses attribute is not properly inserted into the snap state. This causes issues with remote procedure calls and decentralized applications cannot manage addresses using the snap. The suggested fix is to initialize the addresses attribute as an empty object in the snap state. The bug has been fixed in a recent commit by adding an initial state with the necessary attributes.

## Details
## Snap Initialization Issues with Addresses Attribute

When initializing the snap, the `addresses` attribute is not inserted into the snap state. However, the `addAddress`, `deleteAddress`, and `getAddresses` functions require it to be a valid JSON stringified object.

## Code Snippet

```typescript
// state.ts
if (data?.addresses == undefined || data?.addresses == null) {
    throw new Error(
        "Address book was not found. Add an address to address book to initialize."
    );
}
if (typeof data?.addresses !== "string") {
    throw new Error(
        "Invalid address book data. Addresses should be a JSON string."
    );
}
```

In particular, both remote procedure calls do not function correctly, which means that decentralized applications are unable to manage addresses using the snap.

## Remediation

Initialize the `addresses` attribute in the snap state as an empty object.

## Patch

Fixed in commit `ac642da` by changing the following part of the code:

```diff
// index.ts
- // add all the default chains into Metamask state
- res = await ChainState.addChains(chains);
+ // Initialize with initial state
+ await snap.request({
+     method: "snap_manageState",
+     params: {
+         operation: "update",
+         newState: { chains: chains.string(), addresses: JSON.stringify([]) },
+     },
+ });
```
