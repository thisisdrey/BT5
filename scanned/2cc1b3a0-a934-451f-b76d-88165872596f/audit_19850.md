# [M-03] Setting `Factory.bondPercentDiv` to zero cause Denial of Service in `Auction.bondForRebalance()`

## Summary
This bug report was submitted by pants and relates to the function `Factory.setBondPercentDiv()`. This function allows the owner to set the state variable `Factory.bondPercentDiv` to zero. If `Factory.bondPercentDiv` equals zero, then the function `Auction.bondForRebalance()` will always revert due to a division by zero. The tool used to identify this bug was manual code review.

To mitigate this bug, it is recommended to add an appropriate require statement to the function `Factory.setBondPercentDiv()` to disallow setting `Factory.bondPercentDiv` to zero. This bug was acknowledged by frank-beard (Kuiper) and Alex the Entreprenerd (judge) commented that they agree with the finding and raised it to medium.

## Details
_Submitted by pants_.

The function `Factory.setBondPercentDiv()` allows the owner to set the state variable `Factory.bondPercentDiv` to zero.

#### Impact

If `Factory.bondPercentDiv` equals zero then the function `Auction.bondForRebalance()` will always revert due to a division by zero:

    bondAmount = basketToken.totalSupply() / factory.bondPercentDiv();

#### Tool Used

Manual code review.

#### Recommended Mitigation Steps

Add an appropriate require statement to the function `Factory.setBondPercentDiv()` to disallow setting `Factory.bondPercentDiv` to zero.
