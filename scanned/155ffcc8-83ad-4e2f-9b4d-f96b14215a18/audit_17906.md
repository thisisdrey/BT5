# [M-04] RubiconRouter: Excess ether did not return to the user

## Summary
This bug report is about a vulnerability in the RubiconRouter contract, which is part of the code-423n4/2022-05-rubicon project. The vulnerability exists in the swapWithETH/buyAllAmountWithETH/offerWithETH/depositWithETH functions of the contract, where if the user sends more ether than the max_fill_withFee/pay_amt/amount/amtWithFee value, the excess ether will not be returned to the user. This vulnerability is demonstrated by the proof of concept code provided in the report. The recommended mitigation steps to fix this vulnerability are to return the excess ether to the msg.sender, or to require that msg.value == max_fill_withFee/pay_amt/amount/amtWithFee.

## Details
