# Claimable soft clay amount can be claimed unlimited times.

## Summary
There is a bug in the SoftClayContract code where users are able to claim soft clay multiple times. This is because the value for mapping_claimableSoftClay is not being reset to zero after the clay is claimed. The recommendation is to fix this by setting the value to zero after the clay is claimed. The issue has been resolved by the client.

## Details
**Description**

SoftClayContract.sol: function userClaimSoftClay(). After a user has claimed soft clay, the value from mapping_claimableSoftClay is not set to zero, which means that user can claim the same soft clay multiple times.

**Recommendation**

Set the value for mapping_claimableSoftClay[passportld] to zero after clay is claimed.

**Re-audit comment**

Resolved.

From client:

Fixed, now claimable amount is adjusted to zero once claimed.
