# Improper Conversion Of Units In Swap

## Summary
The MathHelper Library has a bug where the swap function is converting the quote amount into an integer twice, which can be exploited by attackers to steal funds without paying the quote balance. This is because the value is divided by 1e18 again, resulting in a zero quote balance. To fix this, one of the two toInt() functions over quoteSwappedX18 should be removed. This has been fixed in version #116.

## Details
## MathHelper Library Vulnerability

In the MathHelper Library, `swap` returns to base and quote amounts in which the quote was converted into an integer twice. For example, `toInt` was applied on `quoteSwappedX18` twice in the function, dividing the actual integer’s value with `1e18` again.

## Code Snippet
```
contracts/libraries/MathHelper.sol SOLIDITY

int256 quoteSwappedX18 = (k / (base + baseSwapped) - quote).fromInt();

if (amountSwap > 0) {
    quoteSwappedX18 = quoteSwappedX18.mul(keepRateX18).toInt();
} else {
    quoteSwappedX18 = quoteSwappedX18.div(keepRateX18).toInt();
}

return (baseSwapped.fromInt(), quoteSwappedX18.toInt().toInt());
```

## Vulnerability Analysis

An attacker may abuse this to manipulate the quoted value by which they steal the funds using `swapAMM` without paying the quote balance.

### Proof of Concept

The attacker swaps the quote to base, for which the quote balance a user would have to pay becomes zero if the value is less than `1e18`. This is because the amount was divided by `1e18` again. Therefore, no quote balance is deducted from the attacker’s account.

An attacker may receive the base amounts for free and withdraw them, throwing the entire protocol into an unresolved state.

## Remediation

Remove one of the two `toInt()` functions over `quoteSwappedX18`.

### Patch

`toInt` was converted to `fromInt` at the return. Fixed in #116.
