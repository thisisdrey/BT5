# Unutilized Accumulated Funds

## Summary
The increase_lambo_fund function in the vault account is not properly utilizing the accumulated amounts in treasury_sy and treasury_emission, resulting in unnecessary wastage of funds. This bug has been fixed in PR#521 and the vault logic has been updated to correctly utilize these amounts.

## Details
## Increase Lambo Fund

The `increase_lambo_fund` function in the vault updates the `treasury_sy` and `treasury_emission` fields within the vault account. However, in the current implementation, `treasury_sy` and `treasury_emission` are never retrieved, accessed, or utilized in any subsequent logic after they are updated in `increase_lambo_fund`. Consequently, the surplus emission in `earned_emission_surpluses` and the surplus amount of SY are never actually utilized in the treasury, resulting in unnecessary wastage of funds.

## Code Snippet

```rust
// exponent_core/src/state/vault.rs
pub fn increase_lambo_fund(&mut self, earn_all_result: &EarnAllResult) {
    self.treasury_sy = self
        .treasury_sy
        .checked_add(earn_all_result.earned_sy_surplus)
        .unwrap();
    for (index, earned_emission) in earn_all_result.earned_emission_surpluses.iter().enumerate() {
        self.emissions[index].treasury_emission = self.emissions[index]
            .treasury_emission
            .checked_add(*earned_emission)
            .unwrap();
    }
}
```

## Remediation

Ensure that the vault logic is updated so that the accumulated amounts in `treasury_sy` and `treasury_emission` are correctly utilized.

## Patch

Resolved in PR#521.
