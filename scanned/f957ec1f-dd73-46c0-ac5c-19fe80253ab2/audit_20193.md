# [M-04] Old yield source still has infinite approval

## Summary
This bug report highlights an issue with PoolTogether's Swappable Yield Source. After swapping a yield source, the old yield source still holds infinite approval, which can be used in large attacks if the yield source is not perfectly secure. The recommendation is to decrease the approval after swapping the yield source. This was confirmed by PierrickGT (PoolTogether) and a pull request was made to address the issue.

## Details
_Submitted by tensors, also found by hickuphh3, cmichel and GalloDaSballo_

After swapping a yield source, the old yield source still has infinite approval. Infinite approval has been used in large attacks if the yield source isn't perfectly safe ([see furucombo](https://github.com/pooltogether/swappable-yield-source/blob/89cf66a3e3f8df24a082e1cd0a0e80d08953049c/contracts/SwappableYieldSource.sol#L268)).

Recommend decreasing approval after swapping the yield source.
