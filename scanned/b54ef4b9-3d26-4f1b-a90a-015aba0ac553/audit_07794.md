# Failure To Validate AMM Configuration

## Summary
The bug report states that there is an issue with the contract::instantiate function in the AMM configuration for a launchpad. This can cause problems with the market_id, quote_decimals, and base_decimals, which can result in the IDO (initial decentralized offering) not being finalized and users not being able to claim their tokens. This is due to incorrect values being used for calculations and can lead to imbalances and reduced liquidity. The report recommends comprehensive validation checks for these values to be included in the instantiate function. The issue has been noted for future improvement.

## Details
## Contract Validation Issues in AMM Configuration

`contract::instantiate` fails to validate `quote_decimals`, `base_decimals`, and `market_id` in the AMM configuration for a launchpad. 

If the `market_id` does not correctly correspond to the specified `project_token_denom` and `quote_denom`, the contract may end up with a `market_id` that is not valid for the intended trading pair. If the `market_id` is incorrect, the IDO will not be finalized because the underlying market does not exist or is inactive. Consequently, users will be restricted from claiming their tokens because the IDO contract will not correctly interact with the market.

> **Location:** `mito-launchpad/src/contract.rs`  
> **Language:** Rust

Additionally, the `quote_decimals` and `base_decimals` are utilized to scale the amounts correctly when performing arithmetic operations involving token amounts. If these values are incorrect, the calculations will be wrong. 

- If the decimals are smaller than expected, the initial amounts for `initial_quote_subscription_amount` and `initial_proj_subscription_amount` will be smaller than intended, resulting in less capital being initially committed to the AMM, which may result in imbalances and reduced liquidity provisioning.
- If the decimals are larger than expected, the calculated subscription amounts may be larger than what is available. Thus, the IDO may fail during finalization due to insufficient funds.

## Remediation

Ensure `instantiate` includes comprehensive validation checks for `quote_decimals`, `base_decimals`, and `market_id`.

## Patch

The issue has been noted as a future improvement.
