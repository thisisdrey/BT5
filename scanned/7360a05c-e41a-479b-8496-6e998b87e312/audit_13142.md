# getKeysFromAddress - Possible Unchecked Null Dereference When Looking Up Private Key ✓ Fixed

## Summary
This bug report is about the `getKeysFromAddress()` function in the `starknet-snap` package of Consensys. This function may return `null` if an invalid address is provided, but most callers of the function do not check for the `null` condition and blindly dereference or unpack the return value, resulting in an exception. The report provides several examples of code that do not check for the `null` condition. 

The bug was fixed with [Consensys/starknet-snap@7231bb7fa4671283b2e7b4cbf5a519d56a57697a](https://github.com/Consensys/starknet-snap/tree/7231bb7fa4671283b2e7b4cbf5a519d56a57697a) by throwing an exception on error. The client also provided the statement that instead of returning null, an error should be raised and the caller should catch the exception. 

The report also provides a recommendation to explicitly check for the `null` or `{}` case and consider returning `{}` to not allow unpacking followed by an explicit null check.
