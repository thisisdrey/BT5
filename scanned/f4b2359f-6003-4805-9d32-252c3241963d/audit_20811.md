# Incorrect use of a library function

## Summary
This bug report is about an issue with the `safeApprove()` function from the `SafeERC20` library. This function is used in two lines of code (https://github.com/yearn/yearn-vaults/tree/e390c2a6b2ba6e2ecc8f3a72a1ea4adfabd17544/contracts/BaseWrapper.sol#L117 and https://github.com/yearn/yearn-vaults/tree/e390c2a6b2ba6e2ecc8f3a72a1ea4adfabd17544/contracts/BaseWrapper.sol#L198). The `safeApprove()` function is used to give permission for a certain amount of tokens, but it must be zeroed before doing so. The problem is that when the function is called twice, the second call will be blocked.

The recommendation to solve this issue is to make the correct call to the `safeApprove()` function. This should be done in two steps: first, zero the amount with `token.safeApprove(addressValue, 0)` and then approve the desired amount with `token.safeApprove(addressValue, amount)`. Following this recommendation should solve the issue.

## Details
##### Description
At the lines 
- https://github.com/yearn/yearn-vaults/tree/e390c2a6b2ba6e2ecc8f3a72a1ea4adfabd17544/contracts/BaseWrapper.sol#L117
- https://github.com/yearn/yearn-vaults/tree/e390c2a6b2ba6e2ecc8f3a72a1ea4adfabd17544/contracts/BaseWrapper.sol#L198
use the `safeApprove()` function from the `SafeERC20` library.
But before giving permission for a certain amount of tokens, you first need to zero this value.
Now the `safeApprove()` function is called only once with a specific value.
After the first call, everything will work, but after the second call, the function will be blocked. 

##### Recommendation
It is necessary to make a correct call to the `safeApprove()` function:
```
  token.safeApprove(addressValue, 0);
  token.safeApprove(addressValue, amount);
```
