# Investors will not be able to withdraw tokens after the end of the vesting duration for STAKING, MARKETING and RESERVES in UnmarshalTokenVesting

## Summary
The bug report is about an error that occurs when using a function called _calculateAvailablePercentage. This function is used to calculate the amount of a cryptocurrency that can be accessed after a certain period of time. The error happens when the function is used after 38 months have passed since the initial time. This is because the length of a specific array called marketingReserveVesting is set to 38, so the function is unable to access any values beyond that. The recommendation to fix this bug is to add a check for the number of months and if it is greater than 38, the function should return the last value in the marketingReserveVesting array. This will prevent the error from occurring and allow the function to work properly.

## Details
**Description**

Array marketingReserveVesting length is equal to 38. If more than 38 months have passed
since the initial time function _calculateAvailablePercentage will lead to error.

**Snippet**:
```solidity
if (currentTimeStamp > _initialTimestamp) {
if (distributionType == DistributionType.ECOSYSTEM) {
return ecosystemVesting[noOfMonths];
} else if (
distributionType == DistributionType.MARKETING ||
distributionType == DistributionType.RESERVES ||
distributionType == DistributionType.STAKING
) {
return marketingReserveVesting[noOfMonths];
```
**Recommendation**:

Add check for noOfMonths > 38 if true return last value of marketingReserveVesting.
