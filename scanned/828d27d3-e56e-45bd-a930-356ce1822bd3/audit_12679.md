# M-11: Calculation B0 meets devision 0 error when a token has small decimal and high price with a small kBid

## Summary
This bug report is related to a calculation error when a token has a small decimal and a high price with a small kBid. The Protocol Team found that in the _calB0WithPriceLimit function, small bid price and kBid will cause temp2 =0, so that temp3 couldn't be calculated correctly. This was demonstrated with a proof of concept code, which resulted in a revert when the maker sets the right parameters but traders can't swap. This would make the swap model invalid. The bug was found using manual review and fixed with a pull request. It is important to note that this issue is not part of the contest submissions and is not eligible for contest rewards. To fix the issue, it is recommended to either fix the formula for this corner case by making temp2 = 1, or to improve calculation accuracy by consistently using precision 18 for calculations and converting to real decimal when processing amounts.
