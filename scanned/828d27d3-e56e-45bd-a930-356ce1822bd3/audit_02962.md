# H-7: Uninitialized feeRecipient in Leverager.sol

## Summary
This bug report is about an issue found in the Leverager contract, which is a part of the Yieldoor protocol. The problem is that the feeRecipient variable is not initialized and there is no way to set it after the contract has been deployed. This can lead to a loss of protocol fees as the liquidatePosition function will send funds to address 0 instead of the intended recipient. To fix this, the feeRecipient should be initialized in the constructor and a function should be created to change it in the future.
