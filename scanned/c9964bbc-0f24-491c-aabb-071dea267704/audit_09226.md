# Denial Of Settlement

## Summary
This report describes a bug in the FCN Vault Logic code that can cause the vault to become permanently frozen. This happens when the function fcnCheckTradeExpiry attempts to add a large value to the number of price observations, but reverts if the prior value is larger than zero. This can be exploited by a malicious user, resulting in the vault being unable to be settled and preventing any further state transitions. The bug has been fixed in the latest version of the code.

## Details
## Vaults Without a Knock-In Event

Vaults without a knock-in event may be permanently frozen by calling `fcnCheckBarriers` before settlement. This happens because `fcnCheckTradeExpiry` attempts to add `type(uint16).max` to the number of price observations, reverting if the prior value is larger than zero.

> _contracts/cega-strategies/fcn/lib/FCNVaultLogic.sol_

```solidity
function checkBarriers(
    /* ... */
) internal returns (bool isKnockedIn, uint16 observationsCount) {
    /* ... */
    if (isKnockedIn) {
        fcnVault.observationsDone = type(uint16).max;
        fcnVault.isKnockedIn = true;
    } else {
        fcnVault.observationsDone += maxObservations;
    }
}
function checkTradeExpiry(
    /* ... */
) internal {
    /* ... */
    (bool isKnockedIn, ) = checkBarriers(
        cgs,
        addressManager,
        vaultAddress,
        type(uint16).max
    );
    /* ... */
}
```

## Proof of Concept

The vault may be permanently frozen via the following sequence:
1. A malicious user calls `fcnCheckBarriers` with a max observation count of one. This results in `observationsDone` to become one.
2. All Knock-In (KI) barriers are not reached, resulting in a non-KI vault.
3. The market maker calls `checkTradeExpiry` for settlement, but encounters a revert due to an integer overflow. Because the vault cannot be settled, any further state transitions may not occur.

## Cega Audit 04 — Vulnerabilities 

## Remediation

Modify `fcnCheckBarriers` to increase `fcnVault.observations` by `observationsCount` and not `maxObservations`.

## Patch

Fixed in `5ab692e`.
