# Unexpected `PERMIT_CODE` in `RouterV2`

## Summary
The bug report states that there is an issue with the `PERMIT_CODE` operation in the `RouterV2.sol` contract. This operation is assumed to always be the first in the list, but if it is not, the `_checkTo()` method does not anticipate it and returns zero. The recommendation is to either make sure the `PERMIT` operation is always first or remove it altogether.

## Details
##### Description

The `PERMIT_CODE` is assumed to always be the initial operation in the list. As a result, the `_checkTo()` method does not anticipate this code to follow and consequently returns zero.

- https://github.com/eywa-protocol/eywa-clp/blob/d68ba027ff19e927d64de123b2b02f15a43f8214/contracts/RouterV2.sol#L274

##### Recommendation

We recommend ensuring that the `PERMIT` operation is either the first in the sequence or not included at all.
