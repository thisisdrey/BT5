# Missing check for successful transfer in ApeMexMargin autoOpen function

## Summary
The report highlights a potential issue in a contract called Apex MexMargin. The function autoOpen at line 462 does not have a check to ensure that a transfer has been successfully completed. This can cause problems as some popular rc20 tokens do not use require statements in their transfer operations and only return a boolean value. The contract already uses a library called SafeERC20 in other functions, which is considered a safer method. The recommendation is to use the safeTransfer method instead of the transfer method to maintain consistency and avoid potential issues. This issue has been resolved in a re-audit.

## Details
**Description**

In contract Apex MexMargin, function autoOpen, at line 462, there is no check to see if the transfer have been done successfully, there are some implementations of popular rc20 tokens that are using no require statements in the transfer operation and at the final are only using the return of an boolean value, the contract is already making use of SafeERC20 library and in other functions the method safe Transfer is used instead of transfer so we would recommend to implement the same pattern everywhere.

**Recommendation**

Use safeTransfer method instead of transfer method.

**Re-audit comment**

Resolved
