# GG-2 | Reenter Dividends

## Summary
This bug report is about a malicious contract that can continually drain funds from a user's account. The malicious contract is able to do this because the dividends paid to a user are only updated after an external call sends them funds. To resolve this issue, the Bridges Team has added the "lock" modifier to the "deposit", "depositLocked", "relock", and "withdraw" functions. This will prevent the malicious contract from continuing to drain funds.

To prevent such a situation from happening again, it is recommended to add a "nonReentrant" modifier from OpenZeppelin's ReentrancyGuard or to use the check-effects-interactions pattern. This will ensure that the malicious contract cannot re-enter and keep draining funds from the user's account.

## Details
**Description**

Because the dividends paid to a user is only updated after the external call sending them funds, it is possible for a malicious contract to re-enter and keep draining `div` amount of BNB on each call.

**Recommendation**

Add a `nonReentrant` modifier from OpenZeppelin’s ReentrancyGuard or utilize the check-effects-interactions pattern.

**Resolution**

Bridges Team:

- Added the `lock` modifier to `deposit`, `depositLocked`, `relock`, and `withdraw`.
