# Block of the emergency withdrawal of arbitrary tokens.

## Summary
This bug report discusses a medium severity issue that has been resolved. The problem was related to the calculation of a local variable, availableBalance, in a function called "emergency WithdrawErc20()" in the TestStake contract. The calculation did not take into account the address of the withdrawn ERC20 token, which could lead to an underflow error if the total amount of tokens locked and fees collected exceeded the balance. The recommendation is to only calculate this variable when the withdrawn token is the same as the stake token. After an audit, it was confirmed that the available balance is now being calculated correctly.

## Details
**Severity**: Medium

**Status**: Resolved

**description**

TestStake::emergency WithdrawErc20()::availableBalance. The local variable availableBalance is calculated independently of the address of the withdrawn ERC20 token. Consequently, during the withdrawal of an arbitrary token when token != stakeToken, underflow may occur because the sum totalTokenLocked + totalFeeCollected exceeds the balance.

**Recommendation:**

Calculate this variable only when token == stakeToken'.
Post audit. Available balance is calculated correctly now.
