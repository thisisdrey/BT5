# Not equal softClay amount

## Summary
The bug report is about a method called updateRank in a file called AccessTokenContract.sol. The report states that the method does not work properly when the value of softClay is the same as any of the levels. The recommendation is to add a way to handle this situation. The re-audit comment says that the issue has been resolved.

## Details
**Description**

AccessTokenContract.sol

The updateRank method does not take into account cases where softClay is equal to any of the levels.

**Recommendation**

Add variant handling when softClay is equal to levels.

**Re-audit comment**

Resolved
