# Checks In issue() And redeemZero()

## Summary
This bug report concerns the functions issue() and redeemZero() in Divider.sol. These functions are only allowed to be executed when called from an adapter, but the contract Periphery is also calling these functions. This is potentially too strict, so it is recommended to double check if calling from Periphery should also be allowed. As a solution, when redeemZeroRestricted() in Divider.sol is set, then when _removeLiquidity() in Periphery.sol is called at or after maturity, zeros will not be redeemed and will be transferred back to the user. This change can be seen on PR#160.

## Details
## Severity: Medium Risk

## Context
- `Divider.sol` Lines: 183-231, 279-313

## Situation
The functions `issue()` in `Divider.sol` (Lines 183-231) and `redeemZero()` in `Divider.sol` have extra checks when the flags `level.issueRestricted()` or `level.redeemZeroRestricted()` are set. In that case, the function can only be executed when called from an adapter. The contract `Periphery`, specifically `Periphery.sol` Lines: 410, 514, 557, also calls these functions. However, these calls would not be allowed. The extra checks are potentially too strict.

```sol
function issue(...) external nonReentrant whenNotPaused returns (uint256 uBal) {
    ...
    if (level.issueRestricted()) {
        require(msg.sender == adapter, Errors.IssuanceRestricted);
    }
    ...
}

function redeemZero(...) external nonReentrant whenNotPaused returns (uint256 tBal) {
    ...
    if (level.redeemZeroRestricted()) {
        require(msg.sender == adapter, Errors.RedeemZeroRestricted);
    }
}
```

## Recommendation
Double check if calling from `Periphery` should also be allowed.

## Sense
After seeing this issue, we’ve decided that if `redeemZeroRestricted()` in `Divider.sol` (Lines 289-291) is set, then when `_removeLiquidity()` in `Periphery.sol` (Lines 551-578) is called at or after maturity, we will not be redeeming zeros and will transfer the zeros back to the user. You can see this change on PR#160.
