# Incorrect dataPricer model update in ProgramKeepalive, causing lower cost and demand

## Summary
This bug report discusses an issue with the ProgramKeepalive function in the Arbitrum Stylus program. The function is not passing the correct number of program bytes to the dataPricer.UpdateModel function, resulting in lower than intended demand and cost values. This can potentially lead to cheaper execution of the keepalive function. The report recommends fixing the ProgramKeepalive function and adding tests for this functionality in the future.

Note that when a program is activated, the `DataPricer.UpdateModel` is called correctly with the number of program bytes instead of kilobytes (`stylus/arbos/programs/programs.go#L246â€“L263`). This is because it is called with the `info.asmEstimate` variable (from the `activationInfo.asmEstimate` field), which is in bytes, instead of the `estimateKb` variable, which is in kilobytes and which is saved into the `Program.asmEstimate` field.

## Exploit Scenario

A chain owner sets a keepalive for a program, resulting in an incorrect data price model update and a cheaper execution of the keepalive function.

## Recommendations

Short term, take the following actions:

- Fix the `ProgramKeepalive` function so that it passes in the number of program bytes in bytes instead of kilobytes to the `dataPrice.UpdateModel` function. Note that this may require code changes in the `ActivateProgram` function as well so that both price model update calls receive the same value for the program bytes amount.
- Change the name of the `asmEstimate` field in the `Program` type to `asmEstimateKb` to prevent similar issues in the future (unless the field is refactored to hold the number of bytes).

Long term, add tests for this functionality.
