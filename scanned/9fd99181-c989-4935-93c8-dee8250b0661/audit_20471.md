# Exchange owner might steal users’ funds using reentrancy

## Summary
A bug was reported in the Exchange contract on BSC (Binance Smart Chain) that could allow an external call to drain user funds. The bug was addressed in commit 34c6b355795027d27ae6add7360e61eb6b01b91b. The bug occurs when the Exchange contract owner uses external calls to pull funds from a user (by using `safeTransferFrom`) and then later pushing (some) of the funds back to the user. Examples of this practice were seen in the code for `Exchange.convertFundsFromInput`, `Exchange.convertFundsFromOutput`, `Exchange.joinPoolFromInput`, `Exchange.recoverLostFunds`. 

The recommendation to fix this bug is to add reentrancy guard protection to at least the functions mentioned above, and in general to all public/external functions since gas price considerations are less relevant for contracts deployed on BSC.

## Details
#### Resolution



The client communicated this issue was addressed in commit 34c6b355795027d27ae6add7360e61eb6b01b91b.


#### Description


The practice of pulling funds from a user (by using `safeTransferFrom`) and then later pushing (some) of the funds back to the user occurs in various places in the `Exchange` contract. In case one of the used token contracts (or one of its dependent calls) externally calls the `Exchange` owner, the owner may utilize that to call back `Exchange.recoverLostFunds` and drain (some) user funds.

#### Recommendation


Reentrancy guard protection should be added to `Exchange.convertFundsFromInput`, `Exchange.convertFundsFromOutput`, `Exchange.joinPoolFromInput`, `Exchange.recoverLostFunds` at least, and in general to all public/external functions since gas price considerations are less relevant for contracts deployed on BSC.
