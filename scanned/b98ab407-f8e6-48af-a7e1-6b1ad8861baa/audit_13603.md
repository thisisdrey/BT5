# An incorrect version calculation will lead to a token loss for a user

## Summary
This bug report concerns a token version calculation error. The version is incorrect and falls within the range of 10_000 to 990_000. This means that a user can lose a token from their current collection and receive a token from another collection that may not even exist yet. The issue can be found in the `TokenVersionUtil` library in the `getTokenInfo()` function. The recommendation is to change the calculation of the token version logic in this function.

## Details
##### Description
A token version is calculated incorrectly (the version is in range [10_000:990_000]) which means that a user will lose a token from the current collection and receive a token from another collection (which probably hasn't existed yet).

##### Recommendation
We recommend changing the calculation of the token version logic in the `getTokenInfo()` function in the `TokenVersionUtil` library.
