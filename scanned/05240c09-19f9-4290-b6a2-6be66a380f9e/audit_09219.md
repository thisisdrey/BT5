# Incorrect Handling Of Partial Fills

## Summary
The report discusses a vulnerability in the AcrossSwapHandler function, specifically in the handleAcrossMessage function. The code assumes that the amount parameter represents the entire amount to be transferred in a fill, but in a cross-pool swap scenario, the transfer may occur in multiple steps and each step may involve a partial fill. The code checks for fillCompleted before calling the _handleSwap function, but if it is false, the function returns early assuming the partial fill is incomplete. This can lead to incorrect transfer amounts and the functionality is currently deprecated. The suggested solution is to remove this functionality as it is no longer needed. The issue has been resolved in version 65e6d0f.

## Details
## Vulnerability in AcrossSwapHandler

The vulnerability in `AcrossSwapHandler` pertains to the treatment of the `amount` parameter within `handleAcrossMessage`. In its current state, `handleAcrossMessage` presumes that the `amount` parameter signifies the entire sum to be transferred in the ongoing fill. However, in an across-pools swap scenario, the transfer may occur in multiple steps, and each step might involve a partial fill.

> _pkg/standalone-utils/contracts/AcrossSwapHandler.sol - Solidity  
> // Implementation of the AcrossMessageHandler interface allowing the  
> // SpokePool to call into the handler to conduct the swap when relaying  
> // a message (v2)

```solidity
function handleAcrossMessage(address tokenSent, uint256 amount,
bool fillCompleted, address /* relayer */, bytes memory message) override external {
    // If this was a partial fill, we wait for
    // it complete before conducting the swap
    if (!fillCompleted) return;
    _handleSwap(IERC20(tokenSent), amount, message);
}
```

The code checks if `fillCompleted` is true before calling `_handleSwap`. If it is false, the function returns early, assuming that the partial fill is incomplete. The problem is that the `amount` parameter represents the amount transferred during the current (potentially partial) fill, and it could be less than the full amount intended for the swap. Furthermore, this functionality is currently deprecated.

## Remediation

Remove this functionality since it is deprecated.

## Patch

Resolved in 65e6d0f.
