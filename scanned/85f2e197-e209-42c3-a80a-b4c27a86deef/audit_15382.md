# Insufficient constraint checks

## Summary
This bug report is about insufficient constraint checks which increase the human error probability in the Fantium-Smart-Contracts platform. The `addCollection()` function (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L266) does not have a zero address check for `athleteAddress` or constraint checks for `athletePrimarySalesPercentage`, `athleteSecondarySalesPercentage`. There is also no check for the `collectionId` existence in the functions `updateCollectionName`, `updateCollectionAthleteAddress`, `toggleCollectionIsPaused`, `updateCollectionPrice`, `updateCollectionMaxInvocations`, `updateCollectionTier`, `updateCollectionBaseURI`, `updateCollectionAthleteName`, `updateCollectionAthletePrimaryMarketRoyaltyPercentage`, and `updateCollectionAthleteSecondaryMarketRoyaltyPercentage`. Additionally, `updateCollectionMaxInvocations` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L342) does not have a `maxInvocations < ONE_MILLION` check, and `updateCollectionAthleteAddress`, `updateFantiumPrimarySaleAddress`, `updateFantiumSecondarySaleAddress`, `updateFantiumMinterAddress`, and `updateFantiumNFTAddress` do not have a zero address check. Furthermore, `updateTiers` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L493) does not have a check for `_priceInWei > 0` or `_maxInvocations < ONE_MILLION`. ## Details
##### Description

Insufficient constraint checks increase the human error probability: a platform manager may enter incorrect data and break the contract logic. 

`addCollection()` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L266):
* no zero address check for `athleteAddress`
* no constraint checks for `athletePrimarySalesPercentage`, `athleteSecondarySalesPercentage`

No check for the `collectionId` existence:
* `updateCollectionName` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L297)

It is recommended to add all necessary checks.
