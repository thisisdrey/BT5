# Signatures can be replayed across chains and contracts

## Summary
This bug report discusses a potential issue with a file called "Airdrop.sol". The problem is that the signature used in the file does not check for the specific chain or contract address, which could lead to the signature being used on different chains or contracts with the same code. The severity of this issue is considered medium and it has been marked as resolved. The recommendation is to include the chain id and contract address in the signature and verify them during the claim process to prevent this issue from occurring.

## Details
**Severity**: Medium

**Status**: Resolved

Location: Airdrop.sol

**Description**

It is possible that the signature is replayed on the different chains as the signature does not check for the chain id. Also it does not check for the address and can be replayed across 2 different contracts on the same chain but with the same code and different addresses.

**Recommendation**

It is advised to include the chain id as well as the address of the contract itself(i.e. address(this)) in the signature and do its verification during the claim process.
