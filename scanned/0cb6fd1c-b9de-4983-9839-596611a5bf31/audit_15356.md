# No Return Of Funds If DepositCollateral Fails

## Summary
The Endpoint contract has a function called submitSlowModeTransaction that takes ERC20 tokens and stores the transaction. However, if the processSlowModeTransaction function fails, the funds taken from users are not returned. This can cause issues for users who submit DepositCollateral transactions, as the funds are not returned if the transaction fails. To fix this, the _executeSlowModeTransaction function should be updated to return the funds to the user in case of a failed transaction. This issue has been fixed in the latest update.

## Details
## Submit Slow Mode Transaction Issue

The `submitSlowModeTransaction` function in the Endpoint contract takes a specified amount of ERC20 tokens and stores the transaction to `slowModeTxs`. However, if `processSlowModeTransaction` fails, the `_executeSlowModeTransaction` function does not return the funds taken from users. Instead, it silently ignores the issue.

## Proof of Concept

Consider this scenario:
1. User submits `DepositCollateral` transactions, for which the specified amount of tokens is transferred from the user account to the endpoint.
2. If `processSlowModeTransaction` call fails, `_executeSlowModeTransaction` is ignoring it silently.

## Remediation

In `_executeSlowModeTransaction`’s catch case at the end, if the `DepositCollateral` transaction failed, then funds should be returned to the user.

## Patch

Implemented `tryReturnFunds` to return the funds. Fixed in #116.
