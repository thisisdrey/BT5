# ChainlinkAtlasWrapper allows retransmitting old reports

## Summary
A bug has been reported in the ChainlinkAtlasWrapper code that could potentially be exploited by a single bad actor. When the transmit() function is called, there is no check to ensure that the report and corresponding signatures have not been used before. This means that even though the transmitters are trusted, the system can still be compromised. The recommendation is to implement a system to ensure that a report can only be transmitted once, possibly by using increasing ids similar to the Chainlink BASE_FEED. This issue has been solved in PR 184 and verified by Spearbit. The severity of this bug is medium risk.

## Security Assessment Report

## Severity
**Medium Risk**

## Context
`ChainlinkAtlasWrapper.sol#L108`

## Description
When `transmit()` is called on the `ChainlinkAtlasWrapper`, there is nothing checking that the report and corresponding signatures haven't been used before. While the transmitters are whitelisted and trusted to an extent, this behavior means a single bad actor can exploit the system (which is otherwise secured by multiple independent parties).

## Recommendation
Ensure that a report can only be transmitted once. This might be achieved by using a system of increasing IDs, similar to the Chainlink `BASE_FEED`.
