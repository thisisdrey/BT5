# Verifier Doesn’t Support Zero or Multiple BSB22 Commitments  Acknowledged

## Summary
This bug report is about the verifier not supporting multiple BSB22 commitments, as Gnark only supports a single `Commit` call. An example of this bug can be found in the `tmpl/template_verifier.go` file on lines 57-58. The recommendation is to add support for zero or multiple BSB22 commitments to the verifier so that proof verification will not fail.

## Details
#### Description


The verifier currently supports single BSB22 commitment as Gnark only supports single `Commit`(..) call. If there is no or multiple BSB22 commitments/`Commit` calls, the verifier would fail in proof verification.


#### Examples


**tmpl/template\_verifier.go:L57-L58**



```
uint256 constant vk\_selector\_commitments\_commit\_api\_0\_x = {{ (fpptr .Qcp.X).String }};
uint256 constant vk\_selector\_commitments\_commit\_api\_0\_y = {{ (fpptr .Qcp.Y).String }};

```
#### Recommendation


Add support for zero or multiple BSB22 commitments
