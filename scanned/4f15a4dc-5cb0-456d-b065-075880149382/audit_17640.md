# UnaccruedSeconds do not increase even if nobody is actively staking

## Summary
This bug report is about a high-risk issue found in the Lock.sol contract. The unstreamed variable tracks whether someone is staking in the contract. However, due to precision loss at Locke.sol#L164-L166 and Locke.sol#L187, unstreamed may be greater than 0 even when all deposited tokens have been withdrawn from the contract.

To illustrate the issue, consider a scenario with two users, Alice and Bob. Alice stakes 1052 wei of deposit tokens at t = startTime, and Bob stakes 6733 wei of deposit tokens at t = startTime + 99. Then, at t = startTime + 36, both Alice and Bob exit from the contract. At this point, Alice’s and Bob’s tokens are both 0 but unstreamed = 1 wei.

The recommendation is to consider using totalVirtualBalance == 0 instead of unstreamed == 0. This should help to resolve the issue.

## High Risk Severity Report

## Context
Lock.sol.sol#L180

## Description
The `unstreamed` variable tracks whether someone is staking in the contract or not. However, due to division precision loss at `Locke.sol#L164-L166` and `Locke.sol#L187`, `unstreamed > 0` may occur even when everyone has withdrawn all deposited tokens from the contract, i.e., `ts.token = 0` for everyone.

Consider the following proof of concept with only two users, Alice and Bob:
- `streamDuration = 8888`
- At `t = startTime`, Alice stakes `1052` wei of deposit tokens.
- At `t = startTime + 99`, Bob stakes `6733` wei of deposit tokens.
- At `t = startTime + 36`, both Alice and Bob exit from the contract.

At this point, Alice’s and Bob’s `ts.tokens` are both `0`, but `unstreamed = 1 wei`. The aforementioned numbers are the result of a fuzzing campaign and were not carefully crafted; therefore, this issue can also occur under normal circumstances.

### Code Snippet
```solidity
function updateStreamInternal() internal {
    ...
    uint256 tdelta = timestamp - lastUpdate;
    if (tdelta > 0) {
        if (unstreamed == 0) {
            unaccruedSeconds += uint32(tdelta);
        } else {
            unstreamed -= uint112(tdelta * unstreamed / (endStream - lastUpdate));
        }
    }
    ...
}
```

## Recommendation
Consider using `totalVirtualBalance == 0` instead of `unstreamed == 0`.
