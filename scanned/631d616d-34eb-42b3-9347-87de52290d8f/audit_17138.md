# Transfer to zero address might revert in MyStaking.

## Summary
The bug report is about an issue with the function_withdraw() in the MyStaking.sol file. It is mentioned that transferring fees to the zero address may cause the transfer to fail, depending on how the ERC20 is implemented. This can prevent users from being able to withdraw their funds. However, the issue is not considered critical as a custom implementation of ERC20 can be used instead, which allows transfers to the zero address. The recommendation is to ensure that only a custom implementation of ERC20 is used or to transfer the fees to a different address, such as address(1). The report also mentions that the issue has been resolved by transferring the fees to address(1).

## Details
**Description**

MyStaking.sol: function_withdraw(), line 314. Transferring fees to zero address might revert, depending on the implementation of ERC20. For example, standard implementation by OpenZeppelin has such a restriction. As a result, with some of the ERC20 tokens, withdrawal won't be possible. Thessue is not marked as critical since a custom implementation of ERC20 can be used, which allows transfers to zero address.

**Recommendation**

Verify that the team will only use a custom implementation of ERC20 that allows transferring to zero address. Otherwise, transfer fees to the other address. For example, to address(1).

**Re-audit comment**

Resolved.

Post-audit:

The issue was resolved by transferring fees to address(1).
