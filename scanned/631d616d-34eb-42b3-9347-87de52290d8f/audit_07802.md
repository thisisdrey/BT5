# Airdrop pool can be drawn by a maintainer.

## Summary
This bug report discusses a medium severity issue that has been resolved. The bug involves two contracts, ChainportMainBridge.sol and ChainportSideBridge.sol, where the caller is able to withdraw more tokens than expected. This is because the caller can set any airdropAmount, even if it exceeds the maximum available. This is a problem because airdrop funds may not even be present in the contract. The recommendation is to either include the airdropAmount in the signature or provide a maximum value for airdrop withdrawals. The post-audit update states that the airdropAmount parameter has been included in the signature.

## Details
**Severity**: Medium

**Status**: Resolved 

**Description**

ChainportMainBridge.sol, releaseTokens()
ChainportSideBridge.sol, mintTokens()

The caller can set any airdropAmount up to the max available, drawing all available liquidity.
Despite that the airdrop funds are considered extra and may not be present at the contract
at all, the user still has the ability to withdraw more than expected.

**Recommendation:**

Include airdropAmount into the signature payload or provide the max airdrop value which
can be withdrawn.

**Post audit**

airdropAmount parameter is now included into the signature
