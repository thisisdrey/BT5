# PaymentForwarder can be unhalted

## Summary
This bug report is about the Haltable contract, which is designed to be used in the case of an emergency to halt a working contract. The PaymentForwarder, however, requires a different functionality - to completely finalize the processing of payments at the end of the crowdsale. The bug report suggests removing the ‘unhalt’ feature so that the contract can’t resume receiving payments after finalization, as well as using more appropriate variable and modifier names. It was later revealed that this was the intended behavior.

## Details
The [Haltable contract](https://github.com/Storj/storj-contracts/blob/2bdeb27c0216d2f0889b6e7363d8a84b54cd7f39/contracts/Haltable.sol) is designed for the use case of halting a working contract in case of an emergency. This can be used, for example, to fix something manually before resuming normal operation.


The functionality required by PaymentForwarder ([as documented](https://github.com/Storj/storj-contracts/blob/2bdeb27c0216d2f0889b6e7363d8a84b54cd7f39/contracts/PaymentForwarder.sol#L11)), on the other hand, is to completely finalize the processing of payments at the end of the crowdsale. We recommend to remove the `unhalt` feature so that the contract can’t resume receiving payments after finalization, and to use more appropriate variable and modifier names.


**EDIT**: This was intended behavior.
