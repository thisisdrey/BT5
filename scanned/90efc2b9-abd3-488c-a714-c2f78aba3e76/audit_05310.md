# [M-03] No price limit in `zapWETH`

## Summary
The report discusses a bug in the `zapWETH` function, which deposits WETH to `UniV3TokenizedLp`. The bug occurs when there is too much WETH in `UniV3TokenizedLp`, causing some of it to be swapped without a price limit. This can result in a smaller value for LPs instead of a smaller number. The `LockZap` contract, which calls `zapWETH`, does not check for slippage, which can lead to further issues. The recommendation is to add a slippage check to prevent this bug from occurring.

## Details
## Severity

**Impact:** Medium

**Likelihood:** Medium

## Description

The `zapWETH` function deposits some WETH to `UniV3TokenizedLp`. If there is too much WETH in `UniV3TokenizedLp`, some WETH will be swapped without a price limit. Furthermore, its caller, `LockZap` contract, does not check slippage. `LockZap` only checks the number of LPs received from the deposit. However, this problem results in a smaller value for LPs rather than a smaller number.

## Recommendations

Add slippage check.
