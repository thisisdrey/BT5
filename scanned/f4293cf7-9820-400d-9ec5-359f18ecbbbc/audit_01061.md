# Unsafe Set Mutation During Iteration in executeAddIncentiveTokens and executeRemoveIncentive-

## Summary
This bug report discusses an issue with a function called `executeAddIncentiveTokens` in a file called `SuperGovernor.sol`. The issue is that when removing a token from a set while iterating over it, the loop will end early and not all tokens in the set will be processed. This same issue also exists in another function called `executeRemoveIncentiveTokens`. The recommendation is to modify the function to use a `while` loop instead of a `for` loop and to add test cases to ensure that the issue is fixed.

## Details
## Tokens

**Severity:** Medium Risk  
**Context:** SuperGovernor.sol#L772-L791  

**Description:**  
In `executeAddIncentiveTokens`, removing a token from the `_proposedWhitelistedIncentiveTokens` set while iterating over it causes the loop to terminate early; not every token in `_proposedWhitelistedIncentiveTokens` will be iterated over.

```solidity
for (uint256 i; i < _proposedWhitelistedIncentiveTokens.length(); i++) {
    token = _proposedWhitelistedIncentiveTokens.at(i);
    _isWhitelistedIncentiveToken[token] = true;
    emit WhitelistedIncentiveTokensAdded(_proposedWhitelistedIncentiveTokens.values());
    // Remove from proposed whitelisted tokens
    _proposedWhitelistedIncentiveTokens.remove(token);
}
```

The same issue exists in `executeRemoveIncentiveTokens()`. The code removes the token from `_proposedRemoveWhitelistedIncentiveTokens` inside the for loop.

**Recommendation:**  
Modify the function as follows:

```solidity
while (_proposedWhitelistedIncentiveTokens.length() != 0) {
    address token = _proposedWhitelistedIncentiveTokens.at(0);
    _isWhitelistedIncentiveToken[token] = true;
    _proposedWhitelistedIncentiveTokens.remove(token);
}
```

Adding test cases to test `executeRemoveIncentiveToken` and `executeAddIncentiveTokens` is also highly recommended.
