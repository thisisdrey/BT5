# State variables saleOpen & presaleOpen of Smart Contract CryptoBearWatchClub are allocated to regulate current state of contract.

## Summary
The bug report is about a smart contract called CryptoBearWatchClub.sol, which has two variables called saleOpen and presaleOpen that are causing issues. The recommendation is to use a single state variable with different values to indicate the current phase of the contract, such as PRIVATE_SALE, PRESALE, or AUCTION. This will help to simplify the code and make it easier to track the current phase of the contract.

## Details
**Description**

CryptoBearWatchClub.sol, saleOpen & presaleOpen

**Recommendation**:

use one state variable to indicate current phase, e.g. state variable with data type enum with
values (PRIVATE_SALE, PRESALE, AUCTION).
