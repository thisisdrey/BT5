# No initialization.

## Summary
The bug report is about a critical issue in a contract called TangleswapFactory. The contract has a function called initNonfungiblePositionManager, which is used to initialize a variable called `nonfungiblePositionManager`. However, this variable is not being initialized when the NonfungiblePositionManager contract is deployed. This means that the owner of the factory contract can update the value of this variable at any time, which is a risk. The recommendation is to refactor the code so that the variable is only initialized once during the deployment of the NonfungiblePositionManager contract, which will eliminate the risk of centralization.

## Details
**Severity**: Critical

**Status**:  Resolved 

**Description**

In contract TangleswapFactory, in the initNonfungiblePositionManager,`nonfungiblePositionManager` must be initialized when NonfungiblePositionManager contract is deployed, and it should be be initialized only once, but it isn’t initialized when NonfungiblePositionManager contract deployed and the owner of the factory contract can update the it anytime.

**Recommendation**: 

Refactor the code to eliminate the centralization risk of being able to set the value multiple time, one possible way to achieve it’s to refactor it so that nonFungiblePositionManager is initalized only one time in the TangleSwapFactory at the moment of deployment of the NonFungiblePositionManager.
