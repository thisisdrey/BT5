# Potential DoS in _distributeExcessIdleSafe

## Summary
This bug report discusses three different issues found in the code for HyperdriveLP, LPMath, and YieldSpaceMath. The severity of these issues is considered medium risk. The first issue is in HyperdriveLP.sol, specifically lines 527-532, where the safe version of LPMath.calculateUpdateLiquiditySafe should be used instead of the current code. The second issue is in LPMath.sol, specifically lines 759-761, where calculatePresentValueSafe should be used instead of the current code. The third issue is in YieldSpaceMath.sol, specifically line 508, where calculateEffectiveShareReservesSafe should be used and any failures should be bubbled up. These issues can potentially cause reverts in the _distributeExcessIdleSafe function due to casting or using pow, ln, and exp functions in the calculations. The report notes that these issues have been resolved in PR 906 and have been fixed.

## Description/Recommendation

- **HyperdriveLP.sol#L527-L532**: 
  Use the safe version of `LPMath.calculateUpdateLiquiditySafe` here instead:
  ```solidity
  // Remove the withdrawal pool proceeds from the reserves.
  success = _updateLiquiditySafe(-shareProceeds.toInt256()); // <--- needs to be defined
  if (!success) {
      return false;
  }
  // note that we swapped the update of these storage parameters in case
  // `_updateLiquiditySafe` fails
  // Update the withdrawal pool's state.
  _withdrawPool.readyToWithdraw += withdrawalSharesRedeemed.toUint128();
  _withdrawPool.proceeds += shareProceeds.toUint128();
  return true;
  ```

- **LPMath.sol#L759-L761**: 
  Use `calculatePresentValueSafe` since one ends up in this scope only through `_distributeExcessIdleSafe`, which is supposed to be a safe function (no reverts when underflowing).

- **YieldSpaceMath.sol#L508**: 
  Use `calculateEffectiveShareReservesSafe` and bubble up failure; otherwise, in some cases `_distributeExcessIdleSafe` would not be safe. 

**Note**: Other cases that could cause a revert in the `_distributeExcessIdleSafe` flow stem from casting or using `pow`, `ln`, and `exp` functions in the calculations.
