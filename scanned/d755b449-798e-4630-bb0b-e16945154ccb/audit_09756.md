# Funds might get stuck in the pool due to totaldeposited underﬂow 

## Summary
The bug report describes an issue with the `totalDeposited` variable in the Omnipool. This variable tracks the amount deposited and withdrawn from the pool, but it can lead to an underflow if the price of WETH increases and users try to withdraw. This can result in some users losing their funds. The recommendation is to either get rid of the variable or add a check to prevent an underflow.

## Description
The `totalDeposited` variable tracks the amount that gets deposited and withdrawn from the Omnipool. However, the amount withdrawn can be greater than the amount deposited. Consider the following scenario:

- The protocol deposits USDC into the USDC-WETH pool.
- The price of WETH increases.
- As a result, we get more USDC when exiting the pool.
- If users try to withdraw, the function at some point will revert due to the underflow.
- Once users realize that, they might all try to withdraw from the pool (in order to not be the last ones), and the last ones will lose their funds.

## Recommendation
Either discard the `totalDeposited`, or add a check and set it to zero if it's about to underflow.
