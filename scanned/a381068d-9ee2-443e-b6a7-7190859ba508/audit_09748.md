# The return value(bool) of aurapool.withdrawandunwrap() isn't checked 

## Summary
The bug report is about a function called `withdrawAndUnwrap()` in the `Omnipool.sol` file. This function returns a boolean value, but the report states that the value is not being checked. This means that the code is not verifying whether the function was successful or not. The recommendation is to check the returned boolean value and make sure that the function is successful.

## Details
## Code Review Issue

## Context
Omnipool.sol#L552

## Description
The function `auraPool.withdrawAndUnwrap()` returns a boolean, but the return value is never checked. 

```solidity
function withdrawAndUnwrap(uint256 _amount, bool _claim) external returns (bool);
```

The returned boolean from `auraPool.withdrawAndUnwrap()` signifies whether the operation was indeed successful or not. Hence, it is crucial to check this value.

## Recommendation
Please check the returned boolean of `auraPool.withdrawAndUnwrap()` and ensure it is successful.
