# M-8: Insufficient oracle validation

## Summary
This bug report is about Insufficient oracle validation found by Breeje, IllIllI, PRAISE, ShadowForce, and rvierdiiev. Prices are provided by an off-chain oracle archive via signed prices, but a Chainlink oracle is still used for index prices. These prices are insufficiently validated and there is no freshness check on the timestamp of the prices, so old prices may be used if Off-Chain Reporting was unable to push an update in time. This means traders will get wrong PnL values for their positions, leading to liquidations or getting more/less than they should, at the expense of other traders and the liquidity pools. The timestamp field is ignored which means there's no way to check whether the price is recent enough. The tool used for this bug was Manual Review and the recommendation is to add a staleness threshold number of seconds configuration parameter, and ensure that the price fetched is within that time range. The bug was fixed in a pull request.

## Found by 
Breeje, IllIllI, PRAISE, ShadowForce, rvierdiiev

## Summary

Most prices are provided by an off-chain oracle archive via signed prices, but a Chainlink oracle is still used for index prices. These prices are insufficiently validated.


## Vulnerability Detail

There is no freshness check on the timestamp of the prices, so old prices may be used if [OCR](https://docs.chain.link/architecture-overview/off-chain-reporting) was unable to push an update in time


## Impact

Old prices mean traders will get wrong PnL values for their positions, leading to liquidations or getting more/less than they should, at the expense of other traders and the liquidity pools.
