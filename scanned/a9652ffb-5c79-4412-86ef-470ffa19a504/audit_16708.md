# Unsafe ERC20 transfer in SEuroOffering `transferCollateral`.

## Summary
The bug report is about a function called transferCollateral in a smart contract called SEuroOffering.sol. This function is transferring assets (specifically tokens) without properly safeguarding the transfer or validating the return. The recommendation is to use a different function called safeTransfer, which is specifically designed for transferring ERC20 tokens. The issue has not been resolved yet.

## Details
**Description**

SEuroOffering.sol transferCollateral - function is transferring assets (i.e. tokens) without safeguarding that transfer and not validating return.

**Recommendation**

It is preferred to use safeTransfer while transferring ERC20.
