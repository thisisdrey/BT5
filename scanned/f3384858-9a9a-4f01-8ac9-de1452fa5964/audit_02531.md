# Outdated Prices May Cause Incorrect Swap Outputs or Lp Token Calculations

## Summary
This bug report discusses an issue with the system relying on outdated prices from the oracle, which could result in incorrect liquidity provision or swap amounts for users. The client has acknowledged the issue and has decided not to stop pools when the oracle data is outdated, as it could prevent users from interacting with the system during a serious oracle outage. However, this increases the risk of outdated prices being used. The report recommends storing the latest oracle update timestamp in the pool and rejecting operations if the prices have not been updated within a reasonable timeframe. This will help prevent users from receiving incorrect LP tokens or swap amounts.

## Details
**Update**
Marked as "Acknowledged" by the client. The client provided the following explanation:

> We've made the decision not to stop pools when the oracle data is outdated because in case of serious oracle outage users will not be able to interact with the system. To determine exchange rate pools also rely on the ratio of current reserves. Meaning that in oracle outage scenario the pool reserve ratio will get adjusted due to market incentives and therefore price will stabilize

**File(s) affected:**`pool.func`

**Description:** The oracle verifies timestamps before sending the `op::update_rates` message to the pools. This means the system relies on the multisig constantly calling the oracle to update the pools. This increases the risk of pools operating with outdated prices, which could lead to incorrect liquidity provision (LP tokens) or inaccurate swap amounts.

This issue is mitigated by the input variables `min_shares_out` and `min_amount_out`, but only if those values were calculated using up-to-date prices. If outdated rates were used, users may still receive incorrect amounts.

**Recommendation:** We recommend storing the latest oracle update timestamp in the pool. Add liquidity or swap operations should be rejected if the prices have not been updated within a reasonable timeframe to prevent users from receiving incorrect LP tokens or swap amounts.
