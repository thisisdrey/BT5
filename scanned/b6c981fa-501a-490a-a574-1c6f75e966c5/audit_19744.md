# Incorrect LISTING_FEE value

## Summary
This bug report highlights an issue with the conversion of SOL (a cryptocurrency) to lamport (a unit of measurement). The report states that at line 22 in the processor.rs file, the value of 1000000 is being incorrectly displayed as 0.01 SOL instead of the expected value of 10000000. The report suggests that this discrepancy may be due to the conversion formula used, which is different from the one used in Solidity (a programming language). The recommendation is to use a different formula, specifically one that involves using the uint256 data type and multiplying by a percentage instead of a decimal value.

## Details
**Description**

At
src/processor.rs:22
It says 1000000 is 0.01 SOL
But from the docs at https://docs.solana.com/terminology#lamport
1 SOL is 1000000000 lamport, so 0.01 SOL = 10000000, one zero is missing

**Recommendation**:

Use the way how it works in Solidity. All calculations are in uint256 and replace statements like
x=x*0.25 with x=x*25/100.
