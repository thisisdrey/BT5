# TRST-M-1 Vesting end time is not enforced

## Summary
This bug report is about an issue with the vestingEntryCloseTime variable, which is used to determine when vesting ends. The problem is that the transmuteInstant() and transmuteLinear() functions do not check that the current time is lower than the close time, which means that users may be able to initiate new transmutations when the desired period is over.

The recommended mitigation for this issue was to add a timestamp check in the transmuteInstant() and transmuteLinear() functions. The team responded that the issue was fixed, and the mitigation review showed that the timestamp checks were implemented successfully.

## Details
**Description:**
vestingEntryCloseTime is defined to be the time when vesting ends. However, there is a lack 
of check in **transmuteInstant()** and **transmuteLinear()**, that current time is lower than close 
time. Therefore, users may initiate new transmutations when desired period is over.

**Recommended Mitigation:**
Add a timestamp check in **transmuteInstant()** and **transmuteLinear()**

**Team Response:**
Issue was fixed.

**Mitigation review:**
Timestamp checks implemented successfully.
