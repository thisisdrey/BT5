# NMKT-6 | Reset Offers on Fee Change

## Summary
This bug report is about an issue with offers being rendered invalid when the fee percentage (`feePerc`) is changed by the owner. When the owner changes the `feePerc`, even if they lower it, all offers with a different fee will need to be reset due to a fee mismatch. This is an unexpected behavior as sellers would prefer a lower fee.

The recommendation to fix this issue is to compare the offer’s fee percentage against an upper bound rather than an exact check. This would ensure that offers remain valid even when the `feePerc` is changed.

## Details
**Description**

If the owner changes the `feePerc`, even if they lower it, all offers with a different fee will need to be reset due to a fee mismatch. A seller would prefer a lower fee so it is unexpected for their offer to be rendered invalid.

**Recommendation**

Compare the offer’s fee percentage against an upper bound rather than an exact check.
