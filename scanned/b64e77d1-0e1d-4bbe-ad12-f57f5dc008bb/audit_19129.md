# Malicious user can pay less to pool

## Summary
The bug report states that there is a vulnerability in the `CreditManager.sol` contract where a malicious user can manipulate the `paths[i]` variable to decrease profits and pay less to the pool. The report suggests that a solution to this issue would be to check the total value of the user before any transfers are made.

## Details
##### Description
Malicious user can add poisoned pool to `paths[i]` to minimize profits and pay less to pool:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L260
##### Recommendation
We recommend checking the total value of user before transfers.
