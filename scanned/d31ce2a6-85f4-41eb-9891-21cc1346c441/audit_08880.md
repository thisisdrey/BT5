# deleteToken Should Prompt User for Its Consent ✓ Fixed

## Summary
The bug report is about a missing user confirmation prompt for the `deleteToken` function in the Tezoroproject/metamask-snap code. Currently, the code only prompts for user confirmation when using the `saveToken` function, but the same should be applied to the `deleteToken` function to maintain consistency and ensure user control over their data. The recommendation is to update the code to include a prompt for user consent before deleting the token from the snap's state. This will ensure that all state-changing interactions require user confirmation. The issue has been addressed in a pull request and should be resolved once the code is updated.

## Details
#### Resolution



Addressed by [tezoroproject/metamask-snap#45](https://github.com/tezoroproject/metamask-snap/pull/45)


#### Description


As a rule of thumb, every state-changing interaction with the Snap’s state should require user confirmation, and the process should be aborted if the user does not consent. This principle is already applied to the `saveToken` RPC endpoint. To maintain consistency and ensure user control over their data, the `deleteToken` endpoint should also prompt the user for consent before proceeding to delete the token from the Snap’s state.


#### Examples


**packages/snap/src/index.ts:L85-L96**



```
case 'deleteToken': {
 await snap.request({
 method: 'snap\_manageState',

 params: {
 operation: ManageStateOperation.UpdateState,
 newState: {},
 encrypted: true,
 },
 });
 return true;
}

```
#### Recommendation


Similarly to the `saveToken` RPC endpoint, the `deleteToken` endpoint should ask the user for its consent before deleting the token from the snap’s state.
