# calling mintFounderHero() may accidentally lock user funds

## Summary
A bug has been reported in the IOTA Heroes Contracts code, which affects the user's ability to recover any additional ether sent when minting NFTs. For example, if the user wants to mint 5 NFTs and the price of founderHeroPrice is 1 ether, but the user sends 5.5 ether instead, the extra .5 ether cannot be recovered. 

To fix this, two recommendations have been made. The first is to update the condition on the Factory.sol file to ensure that the amount of ether sent matches the price of the NFTs. The second is to add a condition to check if any extra ether was sent and allow the user to access it.
