# EVM channel task may terminate early 

## Summary
The bug report discusses a problem with the codebase that has two main components: a plugin and an EVM channel. These components work together to handle requests and execute logic, but there is a bug that causes the EVM channel to terminate early. This means that the plugin will still handle requests, but there will be no EVM channel to execute the logic, resulting in a timeout. This bug affects specific notification logics in the EVM channel and can occur due to network issues. The recommendation is to remove the code that causes the EVM channel to terminate early and replace it with code that will continue the task. The bug has been fixed in the latest commit.

## Details
## Context
- **Location**: `channel_manager.rs#L827-L866`, `main.rs#L142-L150`

## Description
Currently, the codebase has two main components that interact together: the plugin and the EVM channel. The plugin provides the RPC endpoints and sends notifications to the EVM channel to execute the underlying logic. These two components are independent tasks. However, the EVM channel task can terminate early due to multiple returns in the task code.

When this occurs, the plugin component will still handle RPC requests, but there will be no EVM channel task to execute the logic. The plugin will always timeout in this condition. The impacted notification logics in `evm_channel` are:
- **TokenSettleHTLC** and **TokenForwardHTLC**: when the gas oracle fails.
- **GetBalance**: when one of the balance retrievals fails.

## Code Snippet
The EVM channel task can terminate early as there are multiple returns in the loop code. For example, the `GetBalance` logic will terminate the EVM channel when it fails to retrieve a single balance. This is likely to occur due to network issues.

```rust
Ok(Ok(Notification::GetBalance())) => {
    let eth_wallet_balance = match client.get_balance(cln_address, None).await {
        Ok(b) => b,
        Err(e) => {
            error!(
                "Connection issue on obtaining wallet ETH balance. Error {}, proceeding...",
                e
            );
            return Ok(()); // @POC: return will terminate the task
        }
    };
    
    let eth_forwarder_balance = match client.get_balance(htlc_address, None).await {
        Ok(b) => b,
        Err(e) => {
            error!(
                "Connection issue on obtaining forwarder ETH balance. Error {}, proceeding...",
                e
            );
            return Ok(()); // @POC: return will terminate the task
        }
    };
}
```

## Recommendation
Consider removing all the returns that will terminate the EVM channel task from the `evm_channel` function. Those will likely be replaced with `continue`. Moreover, consider adding logic in `main.rs` to automatically restart the EVM channel task when it terminates.
