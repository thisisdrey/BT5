# The return value is not processed when transferring tokens

## Summary
This bug report describes an issue with the smart contract, Grandthrax/yearnV2-generic-lender-strat, which is not processing the return value of the `transfer()` function. This function is part of the ERC-20 specification and is used to transfer tokens between two accounts. The issue is that the return value of the `transfer()` function is not being processed, which can lead to incorrect operation of the smart contract. The recommendation is to add handling of the value returned by the `transfer()` function in order to ensure that the smart contract is operating correctly.

## Details
##### Description
According to the ERC-20 specification, the `transfer()` function returns a variable of the `bool` type.
At the line https://github.com/Grandthrax/yearnV2-generic-lender-strat/blob/979ef2f0e5da39ca59a5907c37ba2064fcd6be82/contracts/GenericLender/GenericLenderBase.sol#L56 there is a call to the `transfer()` function.
But the return value is not processed. This can lead to incorrect operation of the smart contract.

##### Recommendation
It is necessary to add handling of the value returned by the `transfer()` function.
