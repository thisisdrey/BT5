# Minter Privilege Not Withdrawable In MintbaseStore

## Summary
The withdraw_minter function in the mb-store/src/minting.rs file is causing an error because it is not marked as payable. This means that users are unable to attach any yNEAR to the transaction, making it impossible to pass the check. To fix this, the function needs to be marked as #[payable]. This issue has been resolved in the db7ef02 patch.

## Details
## Withdraw Minter Function

`withdraw_minter` asserts that a transaction is signed with a full access key by calling `assert_one_yocto`. However, since it is not marked as `#[payable]`, the function does not allow the caller to attach any yNEAR to the transaction, thus making it impossible to pass the check.

## Remediation
Mark `withdraw_minter` as `#[payable]`.

### mb-store/src/minting.rs DIFF
```rust
+ #[payable]
pub fn withdraw_minter(&mut self) {
    assert_one_yocto();
    self.revoke_minter_internal(&env::predecessor_account_id())
}
```

## Patch
Resolved in `db7ef02`.
