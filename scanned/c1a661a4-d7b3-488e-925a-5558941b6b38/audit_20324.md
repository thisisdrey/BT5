# Router - Cancel is not implemented

## Summary
This bug report is about a problem with canceling failed/expired swaps on the router. This issue could potentially allow a user to trick the router into preparing all its funds which will not be automatically reclaimed after expiration, leading to a router DoS. The cancelExpired method is never called and there is disabled code which could be used to implement the cancel flow. The recommendation is to implement the cancel flow.

## Details
#### Description


Canceling of failed/expired swaps does not seem to be implemented in the router. This may allow a user to trick the router into preparing all its funds which will not automatically be reclaimed after expiration (router DoS).
