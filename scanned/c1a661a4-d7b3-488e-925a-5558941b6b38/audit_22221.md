# Fund receiving address can be changed by owner for a brief period

## Summary
This bug report is about the [`setMultisig`](https://github.com/CoinFabrik/ico/blob/142f43f3bec4afd0ffff3fce06946cf67cb0272f/contracts/Crowdsale.sol#L417) function in the Crowdsale.sol file. This function allows the owner to change the address where funds are sent. It is intended to be used for a brief period after the crowdsale is deployed, before the investor count reaches 5. However, the contract does not enforce this and if the crowdsale is already running, with less than 5 investors, the owner can still change the address. This can be dangerous as if an invalid address is set, it will be impossible to change once the investor count reaches 5. To fix this issue, the function should be removed or the `onlyInEmergency` modifier should be added. Hubii responded and the issue was fixed in [`5a90957abf3f792951b3aad66d6eb0317f0500ca`](https://github.com/CoinFabrik/ico/commit/5a90957abf3f792951b3aad66d6eb0317f0500ca) by declaring the function as `internal` to only allow modifications when the contract is being deployed.

## Details
The [`setMultisig`](https://github.com/CoinFabrik/ico/blob/142f43f3bec4afd0ffff3fce06946cf67cb0272f/contracts/Crowdsale.sol#L417) function allows the owner to change the address where funds are sent. It can be used for a brief period after the crowdsale is deployed, before the investor count reaches 5. The purpose stated in the comments is to be able to fix it before the crowdsale has begun, but this is not enforced by the contract.


If the crowdsale is already running, with less than 5 investors, the owner can still change the address. If by error or malice he sets an invalid one, soon afterwards it will be impossible to change, once investor count reaches 5. For this reason, it’s not a good safety measure as it stands. Consider removing the function, or adding the `onlyInEmergency` modifier.


**Update:** Hubii replied *“The function to change the receiving address was declared `internal` to only allow modifications when the contract is being deployed.”* Fixed in [`5a90957abf3f792951b3aad66d6eb0317f0500ca`](https://github.com/CoinFabrik/ico/commit/5a90957abf3f792951b3aad66d6eb0317f0500ca).
