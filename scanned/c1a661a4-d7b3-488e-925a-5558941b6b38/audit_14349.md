# M-7: `EmptyFeedPrice` will cause orders to be canceled

## Summary
This bug report is about an issue found in the GMX-Synthetics codebase. The issue is that when orders are submitted with invalid oracle prices, the check for `isEmptyPriceError()` returns true, and the order execution is allowed to revert, rather than canceling the order. However, the issue arises when the price reaches zero, and `EmptyFeedPrice` is not counted as one of the errors, which causes any outstanding orders to be canceled. This can lead to losses as orders to close positions will be canceled. The tool used to find this issue was manual review. The recommendation to fix this issue is to include `EmptyFeedPrice` in the list of `OracleUtils.isEmptyPriceError()` errors. The fix was made in a commit to the GMX-Synthetics repository.

## Summary

In most cases where orders are submitted using invalid oracle prices, the check for [`isEmptyPriceError()`](https://github.com/sherlock-audit/2023-02-gmx/blob/main/gmx-synthetics/contracts/oracle/OracleUtils.sol#L292-L310) returns true, and the order execution is allowed to revert, rather than canceling the order. 


## Vulnerability Detail

[`EmptyFeedPrice`](https://github.com/sherlock-audit/2023-02-gmx/blob/main/gmx-synthetics/contracts/oracle/Oracle.sol#L584-L586) isn't counted as one of these errors, and so if the price reaches zero, any outstanding order will be canceled.


## Impact

Orders to close positions will be canceled, leading to losses.
