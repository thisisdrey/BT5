# M-4: no validation to ensure the arbitrum sequencer is down

## Summary
This bug report is about a vulnerability in the Index Coop smart contracts that could be leveraged by malicious actors to take advantage of the sequencer downtime. Chainlink is used in L2 chains such as Arbitrum. When the sequencer is down, stale prices are used for the oracle and the borrow value and collateral value are calculated, which could force the protocol into a rebalance position that would result in a loss. The code snippet provided can be found in the AaveLeverageStrategyExtension.sol file. Manual review was used to identify the issue. The recommendation is to add checks to ensure the sequencer is not down. This vulnerability is specific to L2 and not relevant for the current deployment strategy on Ethereum.

## Vulnerability Detail
```solidity
 int256 rawCollateralPrice = strategy.collateralPriceOracle.latestAnswer();
        rebalanceInfo.collateralPrice = rawCollateralPrice.toUint256().mul(10 ** strategy.collateralDecimalAdjustment);
        int256 rawBorrowPrice = strategy.borrowPriceOracle.latestAnswer();
        rebalanceInfo.borrowPrice = rawBorrowPrice.toUint256().mul(10 ** strategy.borrowDecimalAdjustment);
```

Using Chainlink in L2 chains such as Arbitrum requires to check if the sequencer is down to avoid prices from looking like they are fresh although they are not.

The bug could be leveraged by malicious actors to take advantage of the sequencer downtime.
## Impact
when sequencer is down, stale price is used for oracle and the borrow value and collateral value is calculated and the protocol can be forced to rebalance in a loss position
