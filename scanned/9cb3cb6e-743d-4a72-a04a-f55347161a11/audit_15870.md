# M-5: The ````Stream```` contract is designed to receive ETH but not implement function for withdrawal

## Summary
The Stream contract is a part of the nounsDAO project and is designed to receive ETH but not implement a function for withdrawal. This means that ETH occasionally sent by users will be stuck in these contracts. This issue was found by KingNFT and rvierdiiev, who tested the Stream contract and found that it could receive ETH normally. To fix this issue, a rescueETH() function should be added, similar to the existing rescueERC20() function. A Fix PR was made by eladmallel and can be found at https://github.com/nounsDAO/streamer/pull/10. The Stream contract is important for the nounsDAO project, as it is used to receive ETH from users. Therefore, this issue should be addressed and the recommended rescueETH() function should be added to ensure ETH sent by users is not stuck in the contract.

## Summary

## Vulnerability Detail
Shown as the test case, it can receive ETH normally.
```solidity
contract StreamReceiveETHTest is StreamTest {
    function setUp() public override {
        super.setUp();
    }

    function test_receiveETH() public {
        s = Stream(
            factory.createStream(
                payer, recipient, STREAM_AMOUNT, address(token), startTime, stopTime
            )
        );

        vm.deal(payer, 10 ether);
        vm.prank(payer);
        (bool success, ) = address(s).call{value: 1 ether}("");
        assertEq(success, true);
        assertEq(address(s).balance, 1 ether);
    }
}
```
