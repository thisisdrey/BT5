# Lack of checks on the payed amount in `buyNFT` and `buyMultipleNFT` functions if `paymentToken` is `WETH`

## Summary
This bug report is about the `buyNFT` and `buyMultipleNFT` functions in the Vibe contract. Currently, there are no checks to verify that the amount of ether sent (`msg.value`) matches the expected payment amount when the `paymentToken` is set to `WETH`. This means that if a user sends too much ether, the extra funds will be locked in the contract forever, as there is no way to redeem them.

To fix this issue, the report recommends adding checks to the `buyNFT` and `buyMultipleNFT` functions to ensure that the amount of ether sent matches the expected payment amount. If the payment token is set to `WETH`, the report suggests using the code `require(msg.value == payedAmount)`, where `payedAmount` is the sum of prices for the tokens being purchased. If the payment token is not `WETH`, the report suggests using the code `require(msg.value == 0)`. 

By adding these checks, it will ensure that users are only sending the correct amount of ether to the contract, and will prevent any extra funds from being locked in the contract.

## Details
##### Description
The `buyNFT` and `buyMultipleNFT` functions currently lack checks to verify that `msg.value` matches the expected payment amount when the `paymentToken` is set to `WETH`. If users accidentally send more ether than the intended payment amount, the funds that exceed the payment amount will be locked within the contract forever, as there is no mechanism to redeem ether from the contract.

##### Recommendation
We recommend adding the following checks to `buyNFT` (https://github.com/vibexyz/vibe-contract/blob/d08057edbaf83b00d94dcaca2a05e3c44a45e4d9/contracts/mint/NFTMintSale.sol#L69,
 https://github.com/vibexyz/vibe-contract/blob/d08057edbaf83b00d94dcaca2a05e3c44a45e4d9/contracts/mint/NFTMintSaleMultiple.sol#L76) 
and `buyMultipleNFT` (https://github.com/vibexyz/vibe-contract/blob/d08057edbaf83b00d94dcaca2a05e3c44a45e4d9/contracts/mint/NFTMintSale.sol#L80,
https://github.com/vibexyz/vibe-contract/blob/d08057edbaf83b00d94dcaca2a05e3c44a45e4d9/contracts/mint/NFTMintSaleMultiple.sol#L86) functions:
```solidity=
if (paymentToken == WETH) {
    require(msg.value == payedAmount)
} else {
    require(msg.value == 0)
}
```
Here, the `payedAmount` is the sum of prices for the tokens being purchased.
