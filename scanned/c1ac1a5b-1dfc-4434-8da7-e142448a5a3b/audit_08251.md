# LACK OF RESOURCES AND RATE LIMITING

## Summary
The report describes a vulnerability in an API that can be exploited by an attacker to overload the system. This can happen when too many requests are sent at the same time and the API does not have enough resources to handle them. This can lead to the API becoming unresponsive and third-party API tokens being depleted. The impact of this vulnerability is rated as 3 out of 5 and the likelihood of it occurring is 4 out of 5.

## Details
##### Description

API requests consume resources such as network, CPU, memory, and storage. This vulnerability occurs when too many requests arrive simultaneously and the API does not have enough compute resources to handle those requests. An attacker could exploit this vulnerability to overload the `https://wallet-backend.mainnet.anatha.net/api/v1` API by sending more requests than it can handle. As a result, the API becomes unavailable or unresponsive to new requests, and there can be a depletion of third-party API tokens.
