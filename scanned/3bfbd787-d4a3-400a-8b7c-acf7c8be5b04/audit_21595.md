# Fairswap interfaces are inconsistent ✓ Fixed

## Summary
This bug report is about inconsistencies between three Fairswap contract interfaces which may cause issues with external contracts. These inconsistencies are in the name of the function used to submit orders between the base and settlement currency. This function is called `orderEThToToken()` in `Fairswap_iDOLvsETH`, `OrderBaseToSettlement()` (capitalized) in `Fairswap_iDOLvsLien`, and `orderBaseToSettlement()` in `Fairswap_iDOLvsImmmortalOptions`. To fix this issue, the desired interface should be implemented in a separate file, and inherited on the exchange contracts to ensure they are implemented as intended.

#### Description


There are unexpected inconsistencies between the three Fairswap contract interfaces, which may cause issues for composability with external contracts.


#### Examples


The function used to submit orders between the base and settlement currency has a different name across the three exchanges:


1. In `Fairswap_iDOLvsETH` it is called: `orderEThToToken()`.
2. In `Fairswap_iDOLvsLien` it is called: `OrderBaseToSettlement()` (capitalized).
3. In `Fairswap_iDOLvsImmmortalOptions` it is called: `orderBaseToSettlement()`.


#### Recommendation


Implement the desired interface in a separate file, and inherit it on the exchange contracts to ensure they are implemented as intended.
