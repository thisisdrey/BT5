# [M-11] No slippage protection on Uniswap swap

## Summary
This bug report is about a vulnerability found in a contract called `BuyBackBurnerBalancer.sol` in the `autonolas-tokenomics` project. The function that is affected is called `_performSwap` and it is used to perform swaps on the Uniswap V3 platform. The issue is that the function sets a parameter called `amountOutMinimum` to a fixed value of `1` instead of using a configurable or validated value. This means that the contract is vulnerable to price manipulation or front-running, which could result in a loss of funds for the project.

## Details
*Found by [v12](https://v12.zellic.io)*

`autonolas-tokenomics/../contracts/utils/BuyBackBurnerBalancer.sol` [# L106](https://github.com/valory-xyz/autonolas-tokenomics/blob/bbec5ac12721a62672fb7a5ffba5c40f5a46d8cb/contracts/utils/BuyBackBurnerBalancer.sol# L106)

### Targets

* \_performSwap (BuyBackBurnerBalancer)

### Description

The function unconditionally sets `amountOutMinimum` to `1` when performing the Uniswap V3 swap. This allows the swap to execute even if the output amount is almost zero, leaving the contract unprotected against slippage or price manipulation.

### Root cause

`amountOutMinimum` parameter in the Uniswap router call is hard-coded to `1` instead of deriving from configuration or validating the expected output.

### Impact

An attacker can front-run or manipulate pool prices so that the contract receives almost no OLAS tokens in exchange for the input, leading to a loss of funds for the protocol.
