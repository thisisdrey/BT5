# Inproper copy-paste in `rebalance()`

## Summary
A bug was recently reported in the Mellow Finance contracts repository, where the amount1Min value was copy-pasted from the previous live version without proper modification. This bug was found in the LiquidityVault.sol file, line 433. The recommendation is to fix the amount1Min value. 

In the code, the ClosePositionArgs function was being used, which includes the strategyId, amount0Min, amount1Min, and deadline. The amount0Min and amount1Min were both set to the same value, args.token0MinClose, instead of two different values. This could cause issues in the code or create unintended behavior. 

It is important to fix this bug as soon as possible as it could lead to unexpected results in the code. It is recommended to modify the amount1Min value to ensure that the code works as intended.
