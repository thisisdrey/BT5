# Arithemetic underflow leading to unexpected revert and loss of funds in Receiver contract.

## Summary
A bug has been found in the Receiver contract, which is designed to gracefully return funds to users. This bug can be exploited by an attacker to cause token loss in the Receiver contract. 

The logic of reserving gas for recovering funds is implemented at Receiver.sol#L236-L258. The gasleft() function returns the remaining gas of a call, which is continuously decreasing. The second query of gasleft() is smaller than the first query. If the attacker tries to relay the transaction with a carefully crafted gas where gasleft() >= _recoverGas at the first quiry and gasleft() - _recoverGas reverts, the bug will be exploited. 

The recommended solution to this bug is to cache the gasleft() by adding a uint256 variable called cacheGasleft, and replacing gasleft() with cacheGasleft in the code. This will prevent the second query of gasleft() from being smaller than the first query, and the bug will be resolved. 

In conclusion, the bug found in the Receiver contract can be exploited by an attacker to cause token loss. To prevent this, it is recommended to cache the gasleft() variable.

## Details
## Severity: Medium Risk

## Context
- **Files:** `Receiver.sol#L254`, `Receiver.sol#L282`

## Description
The Receiver contract is designed to gracefully return the funds to users. It reserves the gas for recovering gas before doing swaps via `executor.swapAndCompleteBridgeTokens`. The logic of reserving gas for recovering funds is implemented at `Receiver.sol#L236-L258`.

```solidity
contract Receiver is ILiFi, ReentrancyGuard, TransferrableOwnership {
    // ...
    if (reserveRecoverGas && gasleft() < _recoverGas) {
        // case 1a: not enough gas left to execute calls
        receiver.call{ value: amount }("");
        // ...
    }
    // case 1b: enough gas left to execute calls
    try
        executor.swapAndCompleteBridgeTokens{
            value: amount,
            gas: gasleft() - _recoverGas
        }(_transactionId, _swapData, assetId, receiver) 
    {} catch {
        receiver.call{ value: amount }("");
    }
    // ...
}
```

The `gasleft()` function returns the remaining gas of a call. It is continuously decreasing. The second query of `gasleft()` is smaller than the first query. Hence, if the attacker tries to relay the transaction with a carefully crafted gas where `gasleft() >= _recoverGas` at the first query and `gasleft() - _recoverGas` reverts, this results in the token loss in the Receiver contract.
