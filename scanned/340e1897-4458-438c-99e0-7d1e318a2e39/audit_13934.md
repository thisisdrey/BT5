# Return value of low-level .call() not checked

## Summary
This bug report is related to the Receiver.sol contract. The bug has a high risk severity and can be found at lines 209, 238, and 257. The bug occurs when a low-level primitive .call() is used to transfer a native token to a receiver. If the receiver reverts, this can lead to locked ETH in the Receiver contract. The bug can be resolved by checking the return value and reverting if false is returned. To do this, the code should be changed from receiver.call{ value: amount }(""); to (bool success, ) = receiver.call{ value: amount }(""); and require(success). This bug was fixed in PR 244 and verified by Spearbit.

## Details
## Vulnerability Report

## Severity
**High Risk**

## Context
Locations:
- Receiver.sol#L209
- Receiver.sol#L238
- Receiver.sol#L257

## Description
The low-level primitive `.call()` doesn't revert in the caller's context when the callee reverts. If its return value is not checked, it can lead the caller to falsely believe that the call was successful. The `Receiver.sol` contract uses `.call()` to transfer the native token to the receiver. If the receiver reverts, this can lead to locked ETH in the `Receiver` contract.

## Recommendation
Check the return value and revert if false is returned:

```solidity
-receiver.call{ value: amount }("");
+(bool success, ) = receiver.call{ value: amount }("");
+require(success);
```
