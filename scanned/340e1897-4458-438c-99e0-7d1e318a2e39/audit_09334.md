# Inaccurate Fee Accrual When Fee Params Are Set To New Values

## Summary
The bug report states that there is an issue with the `setManagementFeeInfo` function in the GlmVault.sol file. This function allows the owner to update the Management Fee Info. However, the current implementation does not take into account any fees that may have accrued before the update. This could result in incorrect fee calculation. The report suggests that before updating the management fee parameters, the `keeperManagementFeeClaim()` function should be called to claim any pending fees. This will prevent the loss of any unaccrued fees. The bug has been resolved and the recommendation should be followed to avoid any issues.

## Details
**Severity** - Medium

**Status** - Resolved

**Description**

The owner can update the Management Fee Info with the `setManagementFeeInfo` function in the GlmVault.sol . 
However, the current implementation does not account for accrued fees before the update, potentially leading to incorrect fee calculation.
Keeper fee might have been pending for the last set of management fee params and needs to be claimed before new params are set , else fee not accrued is lost forever.

**Recommendation**:

Call the `keeperManagementFeeClaim()` before updating the management fee params.
