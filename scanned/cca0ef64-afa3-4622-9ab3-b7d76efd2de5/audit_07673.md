# Incorrect Computation Of claimIntervalsPassed

## Summary
This bug report is about a problem in calculating intervals since the last claim period. The current code divides the interval start time by the claim interval duration and subtracts it from the current block timestamp. However, it should actually be the difference between the current block timestamp and the claim interval start, divided by the interval duration. The report recommends modifying the code on line 79 to fix this issue. The bug has been resolved in commit `f8a658c`.

## Details
## Description
Intervals since last claim period are incorrectly computed. Currently, the interval start time is divided by claim interval duration, and subtracted from current block timestamp:

```plaintext
return block.timestamp - (getClaimIntervalTimeStart() / getClaimIntervalTime())
```

However, it should actually be the difference between current block’s timestamp and claim interval start, divided by the interval duration.

## Recommendations
Modify code on line [79] to correctly calculate intervals since last claim period, i.e.

```plaintext
return (block.timestamp - getClaimIntervalTimeStart()) / getClaimIntervalTime()
```

## Resolution
The issue has been addressed in commit `f8a658c` as per the recommendations.
