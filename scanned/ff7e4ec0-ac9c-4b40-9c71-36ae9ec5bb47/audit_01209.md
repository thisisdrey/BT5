# Key Stored Alongside Encrypted Data

## Summary
The client has marked the bug as "fixed" and provided an explanation for the update. The bug involves storing encryption keys and data together, making it easy for anyone who can access the data to also decrypt it. The recommendation is to use a library like `react-native-keychain` to separate the keys from the data and ensure that the OS is unlocked before accessing the keychain. This will improve the security of the encrypted data.

**Update**
A minor optimization would be to clear the key from memory immediately after use rather than leaving it in scope until function exit, reducing the window of key exposure before storing the encrypted data.

**File(s) affected:**`utils/cryptoUtils.ts`

**Description:** Currently, the `interface EncryptedData` stores the decryption key alongside the encrypted data, allowing anyone who can access the `EncryptedData` the immediate ability to decrypt this data. We recognize that the `EncryptedData` is indeed only stored on the device's secure storage, however fundamentally, this is a flawed mechanism to secure encrypted data. This mechanism is akin to storing both a safe with funds, and a key to unlock it, in a safebox at a bank. If the Safebox is opened, the attacker can immediately access the locked funds.

**Recommendation:** Use a library such as `react-native-keychain` to not only separate the keys from the encrypted data but also to ensure that when accessed, the OS is in a state that has been unlocked through the native keychain.
