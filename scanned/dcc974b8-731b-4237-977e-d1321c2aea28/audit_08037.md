# The contract will be paused after `withdrawAllGTH`

## Summary
A medium severity bug has been acknowledged in the LPMigration contract's withdrawAllGTH function. On line #150, the `IPausableERC20.pause()` function is called without a condition, which means that even if the contract was not initially paused, it will be paused after `withdrawAllGTH()` is called. To fix this, it is recommended to add a local variable and assign it to the value of `IPausableERC20.paused` before line #140. Then, this value can be used as a condition for both the unpause and pause function calls.

## Details
**Severity**: Medium

**Status**: Acknowledged

**Contract**: LPMigration

**Function**: withdrawAllGTH

**Details**: 

one line #150 there is a call to the `IPausableERC20.pause()` function. It is an unconditioned call so even if the contract was not initially paused, it will be paused after `withdrawAllGTH()` is called.

**Recommendations**: 

add a local variable and assign it to the value of `IPausableERC20.paused` before line #140. Then use this value for a condition for both unpause/pause function calls
