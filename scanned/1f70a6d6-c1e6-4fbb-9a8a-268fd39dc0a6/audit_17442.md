# [M-11] addFeedFor should check if inverse feed already exists

## Summary
This bug report is about a vulnerability in the code of the Juice-Contracts-V2-Code4Rena repository on Github. The vulnerability can lead to inconsistent currency conversions. The bug occurs when the addFeedFor function is called, as it does not check if the inverse of the currency already exists. This can lead to different prices being used for currency to base vs. base to currency conversion, which could lead to inconsistent conversion ratios. To mitigate this issue, the code should be changed to include a check for the inverse of the currency.

## Details
_Submitted by 0x52, also found by DimitarDimitrov_

Potentially inconsistent currency conversions.

### Proof of Concept

`addFeedFor` requires that a price feed for the \_currency \_base doesn't exist when adding a new price feed but doesn't check if the inverse already exists. This means that two different oracles (potentially with different prices) could be used for \_currency -> \_base vs. \_base -> \_currency. Different prices would lead to inconsistent between conversion ratios depending on the direction of the conversion.

### Recommended Mitigation Steps

Change L115 to:
`if (feedFor\[\_currency]\[\_base] != IJBPriceFeed(address(0)) || feedFor\[\_base]\[\_currency] != IJBPriceFeed(address(0))) revert PRICE_FEED_ALREADY_EXISTS()`
