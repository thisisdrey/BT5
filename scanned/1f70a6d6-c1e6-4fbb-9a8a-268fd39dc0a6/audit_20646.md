# Incorrect tokens transfer

## Summary
This bug report is about an issue with the Compound Protocol, specifically the CCollateralCapErc20.sol file. The issue is that when a user tries to send more tokens than they have, the transaction fails. To fix this issue, the report recommends adding a require statement to the code. This statement will save the user gas and give them information about why the transaction failed. The statement reads, "require(accountTokens[src] >= tokens, "Insufficient balance");".

## Details
##### Description
Tx fail if user tries to send more tokens than he has:
https://github.com/CreamFi/compound-protocol/blob/23a4ae93adc70334553f5a83429a4e967c1eefaa/contracts/CCollateralCapErc20.sol#L339

##### Recommendation
We recommend to add simple require to save gas for user and give him some information about why tx failed:
```solidity=
require(accountTokens[src] >= tokens, "Insufficient balance");
```
