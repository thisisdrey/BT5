# particlepositionmanager fails to implement pausableupgradeable properly 

## Summary
The ParticlePositionManager contract inherits the PausableUpgradeable contract from the Openzeppelin library, which allows for pausing the contract in emergency scenarios. However, the contract fails to actually implement this functionality, leading to a false sense of security. This bug is considered high priority as it can result in unexpected behavior and the inability to pause the contract during critical attacks. To fix this, the internal _pause() and _unpause() methods in the inherited contract should be overridden and the pause functionality should be fully implemented. This issue has been resolved by removing the use of the PausableUpgradeable contract.

## Context
**File:** ParticlePositionManager.sol#L27

## Description
The contract `ParticlePositionManager` inherits the `PausableUpgradeable` contract from the OpenZeppelin library and uses the `whenNotPaused` modifier across multiple instances in the contract. However, the contract fails to implement the actual ability to pause the contract, which gives a false sense of security regarding its ability to be paused in an emergency scenario.

This issue is classified as **HIGH** as it can lead to unexpected protocol behavior and the inability to pause the protocol under critical attacks.

## Recommendation
Override the internal `_pause()` and `_unpause()` methods in the inherited contract and implement the pause functionality completely. For more info, please read the OpenZeppelin documentation [here](https://docs.openzeppelin.com). The exact impact is detailed in the notes.

```solidity
function pause() onlyOwner {
    _pause();
}

function unpause() onlyOwner {
    _unpause();
}
```

## Particle
**Status:** Fixed. We don’t use pausable anymore.
