# Restrict deflationary tokens

## Summary
This bug report is about a contract that is not working properly when used with deflationary tokens. The contract receives tokens by 'safeTransferFrom' and increases the user's amount by the requested amount, however, this approach does not work with deflationary tokens as the real received amount will be less than the requested amount. This will break the logic of the contract if someone creates a pool with a deflationary token. The recommendation is to support such tokens or restrict the contract from being used with them.

## Details
##### Description
At line https://github.com/tosdis/TosDisFinance/blob/be50dbf8a52a8f919694498bf30394d328d88fbb/StakeMaster/contracts/StakingPool.sol#L124 contract receives token by `safeTransferFrom`, and increases `user.amount` by `_amountToStake`, however that approach doesn't work with deflationary tokens because in that case real received amount will be less than requested. That will break the whole logic of contract if somebody creates pool with deflationary token.

##### Recommendation
We recommend to support such tokens or restrict contract usage with them
