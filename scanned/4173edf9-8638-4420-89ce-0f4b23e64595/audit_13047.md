# `AggregateStablePrice` can be manipulated

## Summary
This bug report discusses an issue with the AggregateStablePrice contract in the Curve-Stablecoin project. If there is not enough liquidity in the pools or there are no pools, then 10**18 is returned as the price. This is supposed to be used to manipulate the price, and could be significant at an early stage of the project. The recommendation is to take these conditions into account when deploying contracts.

## Details
##### Description

If there is not enough liquidity in the pools or there are no pools, then `10**18` is returned as the price in `AggregateStablePrice`.

- https://github.com/curvefi/curve-stablecoin/blob/0d9265cc2dbd221b0f27f880fac1c590e1f12d28/contracts/price_oracles/AggregateStablePrice.vy#L151

```
if Dsum == 0:
    return 10**18
```

It is supposed to be used to manipulate the price. At an early stage of the project, this can be significant.

##### Recommendation
We recommend taking these conditions into account when deploying contracts.
