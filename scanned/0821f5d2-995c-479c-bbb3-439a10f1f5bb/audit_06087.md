# Unbounded gas limit in the simulation & precompile

## Summary
A bug has been reported in the simulation code of Ethereum, which affects the maximum gas limit for transactions. This bug has been classified as a medium risk and is located in the simulation.rs and modexp.rs files. The issue is that there is a discrepancy between the defined maximum gas limit and the default gas limit used in the run function of the Evaluation struct. The MAXIMUM_GAS_LIMIT constant, defined in the constants module, sets the maximum gas limit to 30 million gas units. However, in the run function, if no gas limit is provided, it defaults to u64::MAX. This means that the gas limit is not consistent with the defined maximum gas limit. The recommendation is to use the MAXIMUM_GAS_LIMIT constant as the default gas limit. The bug has been fixed by MR 13729 and verified by Spearbit.

## Details
## Severity: Medium Risk

## Context 
- `simulation.rs#L275`
- `modexp.rs#L57`

## Description 
In the simulations, there is an inconsistency between the defined maximum gas limit for Ethereum transactions and the usage of `u64::MAX` as the default gas limit in the `run` function of the `Evaluation` struct. 

In the constants module, the `MAXIMUM_GAS_LIMIT` constant is defined as follows:
- `tick_model.rs#L21`:
```rust
pub const MAXIMUM_GAS_LIMIT: u64 = 30_000_000;
```
This constant sets the maximum gas limit for Ethereum transactions to 30 million gas units. However, in the `run` function of the `Evaluation` struct, when calling the `run_transaction` function, the gas limit is set to `u64::MAX` if no gas limit is provided in the `Evaluation` instance:
```rust
self.gas.or(Some(u64::MAX))
```

## Recommendation 
Use the `MAXIMUM_GAS_LIMIT` constant as the default gas limit.

## Etherlink 
Fixed by MR 13729.

## Spearbit 
Fix is verified.
