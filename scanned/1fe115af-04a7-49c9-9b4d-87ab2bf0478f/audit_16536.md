# ORDH-2 | Phantom Decrease

## Summary
This bug report is about an issue with the `orderStore` in which it is possible to have a decrease order when no corresponding position is present. This can happen when orders are not cancelled upon failure with the `EMPTY_POSITION_ERROR_KEY`. Consider the following scenario: User A creates a position, sends a `LimitDecreaseOrder` and a `MarketDecreaseOrder` in the same block. The `LimitDecreaseOrder` gets executed first, but the `MarketDecreaseOrder` is left floating on an empty position and it never gets canceled. User A then opens a new position with an increase order of the same market, collateral token, and directionality. The market decrease order unexpectedly affects User A’s new position as it was waiting to be executed in the `orderStore`.

To resolve this issue, the recommendation is to cancel the order upon failed execution in the case of `EMPTY_POSITION_ERROR_KEY`. The GMX Team has acknowledged this report and is working on a fix.
