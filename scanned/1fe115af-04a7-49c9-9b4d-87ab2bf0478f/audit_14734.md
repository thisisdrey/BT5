# M-5: Operator can cause fee shares to be minted to address(0)

## Summary
This bug report was found by GimelSec, Dug, Aymen0909, bin2chen, slvDev, ast3ros, 0x52, BTK, 0xAgro, SovaSlava, favelanky, gogo, and ctf\_sec. It was discovered through the use of a Solidity YouTube Tutorial. The bug is related to the setting of a fee rate when the fee recipient is address(0). Currently, when setting the fee it is required that if the fee is not 0 then the fee recipient is not address(0). However, when setting the fee recipient there is no similar check. This means that an operator can bypass the check in setFeeMantissa by setting the fee recipient to address(0) after setting a nonzero fee value. The impact of this bug is that the operator can bypass the fee recipient check. It is recommended that a check similar to the one in setFeeMantissa is implemented that does not allow a nonzero fee when the fee recipient is address(0).
