# No Validation on the feeWallet Address

## Summary
This bug report discusses an issue with the `_updateFeeWallet` function in a code. This function allows for the setting of a `feeWallet` address without any validation, which could lead to potential problems. The report recommends implementing checks to ensure that the provided `feeWallet` address is valid before setting it. The bug has been resolved and is considered low severity.

## Details
**Severity**: Low

**Status**: Resolved

**Description**: 

The `_updateFeeWallet` function allows setting the `feeWallet` without any validation if it's a valid address or not. It's possible to set it to the zero address or another potentially harmful address.
An incorrect fee wallet address, like the zero address, could cause funds to become irretrievable.

**Recommendation**: 

Implement checks to validate the provided `feeWallet` address before setting it.
