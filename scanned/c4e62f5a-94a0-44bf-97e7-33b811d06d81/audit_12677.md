# M-13: When swapping 18-decimal token to 8-decimal token , user could buy decimal-18-token with 0 amount of decimal-8-token

## Summary
This bug report is about an issue found in the D3Trading.sol contract. When a user tries to swap an 18-decimal token to an 8-decimal token, they can buy the 18-decimal token with 0 amount of the 8-decimal token. This could lead to unexpected loss. The issue was found by the Protocol Team through manual review. The recommendation is to add a rule to the buyToken() function in D3Trading.sol so that if the payFromAmount is 0, it is set to 1. This bug report is not part of the contest submissions and is not eligible for contest rewards.

## Details
Source: https://github.com/sherlock-audit/2023-06-dodo-judging/issues/245 

## Found by 
Protocol Team

## Vulnerability Detail
Here is the poc:
```
uint256 payFromToken = d3Proxy.buyTokens(
            address(d3MM),
            user1,
            address(token1),
            address(token2),
            10000000,
            0,
            abi.encode(swapData),
            block.timestamp + 1000
        );
assertEq(payFromToken, 0);
```
## Impact
It may cause unexpected loss

## Tool Used
Manual Review

## Recommendation
In buyToken() of D3Trading.sol, add this rule:
```
if(payFromAmount == 0) { // value too small
            payFromAmount = 1;
        }
```
