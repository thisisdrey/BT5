# Beneﬁcary will lose funds on contract termination 

## Summary
This bug report discusses an issue with the SmartEscrow contract. If the contract is terminated while there are still pending funds to be released to the beneficiary, the beneficiary will not receive those funds. The report provides a proof of concept and recommends a solution to transfer the funds to the beneficiary before terminating the contract. The report confirms that a fix will be made for this issue.

## Details
## Context
SmartEscrow.sol#L174-L175

## Description
If the contract is terminated in between schedule, then the beneficiary will not receive already pending vested funds.

## Proof of Concept
1. Let's say the beneficiary's releasable amount was **X**.
2. Before they could call the `releasable()` function to get this amount, the benefactor side called the `terminate()` function, which sets `contractTerminated = true`.
3. Due to this, the `releasable()` function starts failing due to the below condition:
   ```solidity
   function release() public {
       if (contractTerminated) revert ContractIsTerminated();
       // ...
   }
   ```
4. This means that the beneficiary cannot receive the pending releasable amount **X**, which will go to the benefactor instead.

## Recommendation
On calling the `terminate()` function, the releasable amount should be calculated and transferred to the beneficiary first, and then `contractTerminated = true` should happen.

## Base
The finding is confirmed and we will be making a fix.
