# Unchecked vesting contract address

## Summary
This bug report is about a lack of sanity checks in a contract called VestingSplitter.sol. This contract is part of the bondappetit-protocol repository on github. At line 111 of the contract, it accepts a vesting contract address. However, there is no sanity check in place to make sure the address is valid. This means that anyone can easily ask the contract to call another contract.

To fix this issue, we recommend adding a sanity check for the vesting contract address. This would help ensure that only valid addresses are accepted, and prevent malicious actors from attempting to exploit the contract.
