# Mathematical Errors from Incorrect Operations

## Summary
The bug report discusses issues with incorrect fee calculations and maximum value representation in a financial market software. The problem lies in the calculation of the fee, where it is being scaled up instead of down due to a multiplication error. Additionally, a rounding error is also present in the calculation of the magnitude, resulting in fewer units being allocated than intended. The root cause of these issues is a mistake in the code, where the exchange rate is being multiplied instead of divided, and the use of an incorrect maximum value for unsigned integers. The report suggests a solution to fix these issues by dividing the fee by the exchange rate and applying the correct rounding method. The report also mentions that the issues have been resolved in the latest patches for the software.

## Details
## Issues and Remediation in MarketFinancials and Dec_Num Library

## Issue Summary

In `MarketFinancials::trade_pt`, the `sy_fee` calculation currently multiplies `asset_fee` by the `sy_exchange_rate`. This multiplication incorrectly scales up the fee, leading to an incorrect fee calculation. Instead, it should be divided by the exchange rate to convert the asset fee into SY terms, effectively scaling down the fee according to how many base assets are equivalent to one SY.

Additionally, in `sy_magnitude_from_net_trader_asset`, `ceil` should be applied to `sy_magnitude` when `net_trader_asset` is negative. Using `floor` will round down, potentially resulting in the allocation of fewer SY than intended.

### Code Snippet

> **File**: `exponent_core/src/state/market_two.rs`  
> **Language**: Rust

```rust
fn sy_magnitude_from_net_trader_asset(net_trader_asset: DNum, sy_exchange_rate: Number) -> u64 {
    let asset_magnitude: u64 = net_trader_asset.value.floor().abs().try_into().unwrap();
    let sy_magnitude = Number::from_natural_u64(asset_magnitude) / sy_exchange_rate;
    let sy_magnitude = sy_magnitude.floor_u64();
    sy_magnitude
}
```

Additionally, in the `dec_num` library, `MAX_U96` is incorrect. The maximum value for an unsigned integer with `n` bits is \(2^n - 1\). For 96 bits, the maximum value is \(2^{96} - 1\). Utilizing `1 << 96` gives \(2^{96}\), which is one more than this maximum value, resulting in an off-by-one error in representing the maximum value for 96-bit unsigned integers.

## Remediation

1. Ensure that the `sy_fee` is calculated by **dividing** the `asset_fee` by the `sy_exchange_rate` to convert the fee from the base asset to SY units correctly.
2. Apply `ceil` to `sy_magnitude` when `net_trader_asset` is negative.

## Patch

1. The issue in `MarketFinancials::trade_pt` was resolved in **PR#559**.
2. The issue in `sy_magnitude_from_net_trader_asset` was resolved in **PR#560**.
3. The issue in `dec_num` was resolved in **PR#553**.
