# Code commentary doesn't comply with real implementation

## Summary
This bug report concerns the Yearn Vaults project, a decentralized finance (DeFi) project. The bug is related to the rate limit function, which is defined as having a “tokens per block” dimension in the code, but the rate limit checker code assumes it as a variable with “tokens per second” dimension. This inconsistency can lead to errors and incorrect calculations.

The recommendation is to keep the commentaries consistent with the implementation. This means that the code should be updated so that the rate limit function is consistently defined as having a “tokens per block” dimension. This will ensure that the code is accurate and that no errors occur.

## Details
##### Description
At [line](https://github.com/iearn-finance/yearn-vaults/blob/054034304c7912d227d460feadc23177103de0b9/contracts/Vault.vy#L94) it is defined that rate limit has "tokens per block" dimension:
> "Increase/decrease per block"
But in rate limit checker code https://github.com/iearn-finance/yearn-vaults/blob/054034304c7912d227d460feadc23177103de0b9/contracts/Vault.vy#L1124 `strategy_rateLimit` assumed as variable with "tokens per second" dimension

##### Recommendation
We recommend keeping commentaries consistent with implementation
