# M-22: Lost funds on ZetaChain due to unprocessed ZEVM logs

## Summary
This is a bug report about an issue with the ZetaChain cross-chain judging system. The problem was found by a user named berndartmueller. In some cases, when users send ZETA tokens from an external chain to ZetaChain, their funds are burned on ZetaChain without being withdrawn on the external chain. This happens when the ZEVM logs that signal the creation of outbound CCTXs are not processed. This can occur when certain functions are called, such as `processFailedZETAOutboundOnZEVM()` and `HandleEVMDeposit()`. These functions interact with ZetaChain's ZEVM gateway contract, but the logs are not processed, causing the funds to be lost. This is different from other situations where logs are processed, such as when depositing gas or ERC-20 tokens from an external chain to ZetaChain. The impact of this bug is that funds are burned on ZetaChain without creating an outbound CCTX. The team has acknowledged the issue but will not be fixing it at this time. A possible solution is to always process ZEVM logs to create outbound CCTXs.

## Summary

In some instances, ZEVM logs that are emitted to signal the creation of outbound CCTXs are not processed, burning the user's funds on ZetaChain while not withdrawing them on the external chain.

## Root Cause

In the following two instances,

- [`processFailedZETAOutboundOnZEVM()#L335-344`](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/node/x/crosschain/keeper/cctx_orchestrator_validate_outbound.go#L335-L344) - Used to revert a failed ZETA outbound that errored on an external chain and which was initiated on ZetaChain (ZEVM)
- [`HandleEVMDeposit()#L58-71`](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/node/x/crosschain/keeper/evm_deposit.go#L58-L71) - Handles a ZETA deposit from an external EVM chain to ZetaChain (ZEVM)

the ZEVM on ZetaChain is invoked, and a target contract is called.

This contract might as well interact with ZetaChain's ZEVM gateway contract to initiate subsequent outbound CCTXs. For example, calling [`GatewayZEVM.withdraw()`](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/protocol-contracts/contracts/zevm/GatewayZEVM.sol#L183-L211), which burns the user's ZRC-20 tokens and emits `Withdrawn`.

However, logs are not processed in those two instances, meaning no outbound CCTX is created, and the burned funds are not withdrawn on the external chain; they remain burned on ZetaChain. The user loses their funds.

This is in contrast to other situations where ZEVM logs are processed. [For example, when depositing gas or ERC-20 tokens from an external chain to ZetaChain](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/node/x/crosschain/keeper/evm_deposit.go#L108-L158).

## Attack Path

1. [User sends ZETA tokens from an external chain (e.g., Ethereum) to ZetaChain (ZEVM) with a contract call on ZetaChain](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/protocol-contracts/contracts/evm/legacy/ZetaConnector.eth.sol#L30-L48)
2. On ZetaChain, the user's target contract interacts with the ZEVM gateway contract to initiate an outbound CCTX. ZRC-20 tokens are burned
3. The emitted log is not processed, preventing the outbound CCTX from being created
4. The user/contract on ZEVM loses their funds

## Impact

Funds are burned on ZetaChain while not creating an outbound CCTX.
