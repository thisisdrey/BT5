# TRST-H-1 Incorrect implementation of getProfitSharingE18() greatly reduces Lender's yield

## Summary
This bug report is about the wrong implementation of the `getProfitSharingE18()` function in the ProfitSharingModel. This function calculates the share of profit that Lender gets based on the APR of the position. According to the formula, the higher the APR, the lower the share of profit the Lender gets. However, the bug was that if the APR was smaller than the MAX_ANNUALIZED_YEILD, the base share of 25% was returned instead of the calculated share of profit. 

For example, considering an APR of 5%, the Lender's share of the profit should be 77%, while getProfitSharingE18() returned 25%, which greatly reduced the Lender's share of the profit.

The team recommended to modify `getProfitSharingE18()` as follows:
```solidity
            - if (_annualizedYieldE18 < MAX_ANNUALIZED_YEILD) {
            + if (_annualizedYieldE18 >= MAX_ANNUALIZED_YEILD) { 
            return 0.25e18;
            }
```

The team has fixed the bug as recommended to make the logic correct.
