# Loading Arbitrary Data as Wire Commitments  Acknowledged

## Summary
This bug report is about a function called `load_wire_commitments_commit_api` which is used to load wire commitments from a proof into an array called `wire_commitments`. The array is designed to hold 2 values per commitment, or the size of the array should be 2 * `vk_nb_commitments_commit_api`. The function should run the loop `vk_nb_commitments_commit_api` times. However, the loop is running twice as many times, which means that if there is 1 commitment, it will run two times. The first iteration will pick up the actual coordinates and the second one can pick any arbitrary data from the proof and load it into memory. This can add an unnecessary overhead for the processing.

The recommendation is to switch from running the loop `mul(vk_nb_commitments_commit_api,2)` times to `vk_nb_commitments_commit_api` times. This will ensure that the loop runs the correct number of iterations and will help reduce the overhead.

## Details
#### Description


Function `load_wire_commitments_commit_api` as the name suggests, loads wire commitments from the proof into the memory array `wire_commitments`. The array is made to hold 2 values per commitment or the size of the array is 2 \* `vk_nb_commitments_commit_api`, which makes sense as these 2 values are the **x** & **y** co-ordinates of the commitments.


**contracts/Verifier.sol:L453-L454**



```
uint256[] memory wire\_committed\_commitments = new uint256[](2\*vk\_nb\_commitments\_commit\_api);
load\_wire\_commitments\_commit\_api(wire\_committed\_commitments, proof);

```
Coming back to the function`load_wire_commitments_commit_api`, it extracts both the **x** & **y** coordinates of a commitment in a single iteration. However, the loop runs `2 * vk_nb_commitments_commit_api`, or in other words, twice as many of the required iterations. For instance, if there is 1 commitment, it will run two times. The first iteration will pick up the actual coordinates and the second one can pick any arbitrary data from the proof(if passed) and load it into memory. Although, this data which has been loaded in an extra iteration seems harmless but still adds an overhead for the processing.


**contracts/Verifier.sol:L307**



```
for {let i:=0} lt(i, mul(vk\_nb\_commitments\_commit\_api,2)) {i:=add(i,1)}

```
#### Recommendation


The number of iterations should be equal to the size of commitments, i.e., `vk_nb_commitments_commit_api`.
So consider switching from:



```
for {let i:=0} lt(i, mul(vk_nb_commitments_commit_api,2)) {i:=add(i,1)}

```
to:



```
for {let i:=0} lt(i, vk_nb_commitments_commit_api) {i:=add(i,1)}

```
