# `takeClaimingSnapshot()` call will lead to an inconsistent state

## Summary
This bug report is about a function in the FantiumClaimingV1.sol file that can block the ongoing distribution event until the funds are paid or the event parameters are returned back. The bug report recommends removing this function or adding a `topUpDistributionEvent()` call at the end of the function in order to fix the issue. This will ensure that the distribution event is not blocked and can complete successfully.

## Details
##### Description
This function can block the ongoing distribution event until the funds are paid or the event parameters are returned back.
https://github.com/FantiumAG/smart-contracts/blob/b4acd978e25289becc640f75eb5921e760b83720/contracts/claiming/FantiumClaimingV1.sol#L689

##### Recommendation
We recommend removing this function or adding a `topUpDistributionEvent()` call at the end of the function.
