# `maxInvocations` should be limited

## Summary
A bug report has been filed regarding the TokenId in the FantiumNFTV3.sol file. The TokenId contains collactionId, versionId and tokenNumber. The parameter `maxInvocations` determines the max tokenNumber. However, due to using versionId, tokenNumber should be less than 10_000, otherwise the versionId logic would be broken. To fix this issue, it is recommended to add a check that `maxInvocations < 10_000`.
