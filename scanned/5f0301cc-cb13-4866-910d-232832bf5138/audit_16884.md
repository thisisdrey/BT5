# Executor Cache Inconsistency

## Summary
This bug report discusses an issue with executors retaining cached VerifiedExecutables even after certain features are activated. This can lead to the execution of invalid executables on some executors while others reject them. The suggested solution is to flush executor caches when features affecting ELF parsing and verification are enabled. A patch has been created to address this issue.

## Details
## Stricter Rules on ELF Parsing and Validation

Stricter rules on ELF parsing and validation are periodically added through feature gates. However, executors will retain cached VerifiedExecutables even after features are activated that would otherwise cause an executable to fail verification.

A VerifiedExecutable could be created and cached by some executors before a feature that would prevent its verification (e.g., `reject_callx_r10`) is enabled. After the feature is enabled, the now-invalid executable will still be run on executors that cached it before the feature was activated, but it will be rejected by executors that have not cached it.

## Remediation

Flush executor caches when features affecting ELF parsing and verification are enabled.
