# createPot() reverts due to incorrect condition on fee 

## Summary
The bug report is about a smart contract called QuailFinance, specifically in two lines of code (L90 and L97). The contract has a function called "createPot()" that requires a certain amount of money (fee) to be sent along with the call. However, there is a mistake in the code that is causing the function to fail. The mistake is that the function is checking if the amount of money sent (msg.value) is less than the required fee, instead of checking if it is equal to or greater than the fee. This is causing the function to revert and not create the intended "pot". The recommendation is to fix this mistake by updating the code to check if msg.value is equal to or greater than the fee.

## Details
## Context
- **QuailFinance.sol#L90-L90**
- **QuailFinance.sol#L97-L97**

## Description
The `entropy` function requires the fee amount to be sent as `msg.value`, but the `createPot()` function doesn't allow the caller to send the fee value along with the call due to the following require condition:

```solidity
require(msg.value < fee, "Insufficient fee");
```

As a result, the call reverts, blocking the creation of a pot.

## Recommendation
Update the require condition to:

```solidity
require(msg.value == fee, "Insufficient fee");
```

Alternatively, you may consider using `msg.value >= fee`, but be aware that this can lead to overcharging the user.
