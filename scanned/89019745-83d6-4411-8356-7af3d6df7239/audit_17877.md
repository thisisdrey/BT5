# [M-33] RubiconMarketAddress in BathPair can’t be updated

## Summary
This bug report is about a vulnerability in the RubiconMarketAddress code in the BathPair and BathHouse contracts. The RubiconMarketAddress in the BathPair is initialized only once, however, the market can change in the BathHouse. This means that the RubiconMarketAddress will get out of sync between the two contracts. The comment in the code says that it changes the Bath Token’s target Rubicon market, but actually it updates its own instance variable. The recommended mitigation steps for this vulnerability is to sync the RubiconMarketAddress between the BathPair and BathHouse contracts.

## Vulnerability details

## Impact
RubiconMarketAddress in BathPair is initialized only once:
```solidity
  RubiconMarketAddress = IBathHouse(_bathHouse).getMarket();
```
but market can change in Bath house:
```solidity
    /// @notice Admin-only function to set a Bath Token's target Rubicon Market
    function setMarket(address newMarket) external onlyAdmin {
        RubiconMarketAddress = newMarket;
    }
```
Thus it will get out of sync. Also, the comment says that it changes Bath Token's target Rubicon market but actually it updates its own instance variable.

## Recommended Mitigation Steps
I think RubiconMarketAddress should sync between BathPair and BathHouse.
