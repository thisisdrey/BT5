# Anyone can reset the emergencyWithdrawable ﬂag to false to block emergency action in SuperVault-

## Summary
This bug report discusses a problem with a code in SuperVaultStrategy.sol. The severity of the bug is considered medium risk and it affects lines 814 to 820 of the code. The issue is that the condition `timestamp >= emergencyWithdrawableEffectiveTime = 0` is always true if there is no proposal, which allows anyone to reset the emergency withdrawable feature to false. The recommendation to fix this bug is to add a check in the code that will prevent this from happening.

## Details
## Strategy

**Severity:** Medium Risk  
**Context:** SuperVaultStrategy.sol#L814-L820  
**Description:** `timestamp >= emergencyWithdrawableEffectiveTime = 0` is always true if there is no proposal, so anyone can reset `emergencyWithdrawable = false`.  
**Recommendation:** Add the check:  
```solidity
if (emergencyWithdrawableEffectiveTime == 0) revert NoProposal();
```
