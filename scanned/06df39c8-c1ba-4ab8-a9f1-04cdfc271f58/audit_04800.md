# Missing Account Seed Verification

## Summary
This bug report describes an issue with the OpenPositionShort structure in the code. The lp_staking account seed is not being checked against the lp_token_mint, allowing an attacker to manipulate fee calculations and potentially cause financial losses. The same issue exists in the lm_staking account. The suggested solution is to check the seed against the correct account and the bug has been resolved in a recent update.

## Details
## Vulnerability Report on OpenPositionShort Structure

In the current implementation of the OpenPositionShort structure within `open_position_short`, the `lp_staking` account seed is the static byte string `b"staking"` referencing the `staked_token_mint` field within the `lm_staking` account. However, this seed is not checked against `lp_token_mint`. Consequently, it allows an attacker to provide an arbitrary staking account. As a result, this disrupts and manipulates fee calculations and distribution, potentially resulting in financial losses. A similar issue exists in the `lm_staking` account.

> _position/open_position_short.rs_
>
> ```rust
> #[derive(Accounts)]
> pub struct OpenPositionShort<'info> {
> /// #6
> #[account(
> mut,
> seeds = [b"staking", lp_staking.load()?.staked_token_mint.as_ref()],
> bump = lp_staking.load()?.bump,
> constraint = lp_staking.load()?.is_initialized() @AdrenaError::InvalidStakingState
> )]
> pub lp_staking: AccountLoader<'info, Staking>,
> [...]
> }
> ```

## Remediation

Ensure to check the seed against `lp_token_mint` to verify that the `lp_staking` account is the actual LP staking account associated with the given pool. Similarly, verify `lm_staking` account's seed.

## Patch

Resolved in 8749738.
