# totalMembers Can Be Manipulated

## Summary
This bug report is about the lockTokens() and withdrawTokens() functions in the ERC20Guild. These functions are used by users to lock tokens and gain voting power, but a malicious user can use them to manipulate the totalMembers variable by locking or withdrawing zero amount. The testing team recommended preventing zero amount as an input on these functions and the issue has been fixed in PR#180. A user can no longer lock zero tokens.

## Details
## Description
The `lockTokens()` function is used by a user to lock tokens and gain voting power. This function, however, can be used to manipulate the `totalMembers` variable because it accepts zero amount as an input. A malicious user can call `lockTokens()` as many times as they want to increase the `totalMembers` variable. Similarly, the `withdrawTokens()` function can be used to decrease the `totalMembers` by withdrawing a zero amount (see DXD-01).

## Recommendations
The testing team recommends preventing a zero amount as an input on `lockTokens()` and `withdrawTokens()`.

## Resolution
The issue has been fixed in PR#180. A user can no longer lock zero tokens.
