# Subsequent rounds of winner selection is blocked for a pot 

## Summary
The bug report is about a problem with the QuailFinance contract, specifically in lines 145-146. The issue is that when the function rotateLiquidity() is called for the second time for a specific pot, it fails because it calls the function entropy.reveal(). This function cannot be called again with the same sequence number, causing the pot to not be able to select a winner again. The recommendation is to either use a new sequence number for each round of winner selection or to only allow a pot to choose a winner once and not be able to join again.

## Details
## QuailFinance.sol - Issue Overview

## Context
- **File**: QuailFinance.sol
- **Lines**: 145-146

## Description
The second call to `rotateLiquidity()` for a pot reverts due to the call to `entropy.reveal()`. Once a `sequenceNumber` is used, `entropy` reverts if it’s used again as it clears the request. Hence, a pot cannot be used to select a winner again.

## Recommendation
- Consider selecting a new `sequenceNumber` for each round of winner selection.
- Alternatively, discard the concept of rounds for a pot, choose a winner only once, and ensure that pot cannot be joined again.
