# [Oval] Attempts to Push Price to the CoinbaseOracle Will Always Fail

## Summary
The `CoinbaseOracle` contract is used to provide price data for assets. To do this, users must call the `pushPrice` function which updates the price data for a given asset. However, the contract currently has a bug where it always reverts when the `kind` field is not equal to "price". This is because the data signed by the Coinbase oracle always has `kind` equal to "prices". This makes it impossible to provide accurate asset prices. The bug has been fixed in a recent update and users should verify that the `kind` value is equal to "prices" when using the `pushPrice` function.

## Details
The [`CoinbaseOracle` contract](https://github.com/UMAprotocol/oval-contracts/blob/297ab25a914171aedaeb01dabf4a829a304cce08/src/oracles/CoinbaseOracle.sol#L10-L116) serves as an oracle for price data reported and signed by Coinbase. In order to supply such price data to the contract, users have to call the [`pushPrice`](https://github.com/UMAprotocol/oval-contracts/blob/297ab25a914171aedaeb01dabf4a829a304cce08/src/oracles/CoinbaseOracle.sol#L81) function which, in case of the correct data passed, [updates](https://github.com/UMAprotocol/oval-contracts/blob/297ab25a914171aedaeb01dabf4a829a304cce08/src/oracles/CoinbaseOracle.sol#L98-L100) the price data for a given asset. In order to verify the correctness of provided data, [several checks](https://github.com/UMAprotocol/oval-contracts/blob/297ab25a914171aedaeb01dabf4a829a304cce08/src/oracles/CoinbaseOracle.sol#L89-L96) are made, including the [check for a valid `kind` field](https://github.com/UMAprotocol/oval-contracts/blob/297ab25a914171aedaeb01dabf4a829a304cce08/src/oracles/CoinbaseOracle.sol#L89). However, this check reverts when `kind` is different than "price". The data signed by the Coinbase oracle [will always contain `kind` equal to "prices"](https://docs.cdp.coinbase.com/exchange/reference/exchangerestapi_getcoinbasepriceoracle/), resulting in the `pushPrice` function always reverting, making it impossible to provide asset prices.


Consider verifying that the provided `kind` value is equal to "prices".


***Update:** Resolved in [pull request \#18](https://github.com/UMAprotocol/oval-contracts/pull/18) at commit [ff31b1b](https://github.com/UMAprotocol/oval-contracts/commit/ff31b1bdd52da472fed40f93970e1886f4f27cef).*
