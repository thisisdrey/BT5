# Malicious Canceller Can Lock the Safe

## Summary
The bug report describes an issue where an address with the `CANCELLER_ROLE` can stop any proposals from being executed. If the address is compromised while the `DEFAULT_ADMIN_ROLE` is held by the Safe, it becomes difficult to revoke the role because it requires a time-locked transaction that the attacker can cancel repeatedly. This can result in the Safe being permanently unable to execute any operations, which is a medium severity issue. The recommendation is to allow an authorized admin to call `revokeRole` without delay. The client has fixed the issue and the fix can be found at the provided link.

## Details
##### Description
An address with `CANCELLER_ROLE` can veto any queued proposal. If that address becomes compromised while `DEFAULT_ADMIN_ROLE` is held by the Safe, revoking the role requires a timelocked transaction that the attacker can cancel repeatedly. The Safe becomes permanently unable to execute any operation, including the role-revocation and disabling the guard contract, if they were not whitelisted preemptively. This places overall control at risk, giving the issue **Medium** severity. 
<br/>
##### Recommendation
We recommend allowing an authorised admin to call `revokeRole` without delay.
