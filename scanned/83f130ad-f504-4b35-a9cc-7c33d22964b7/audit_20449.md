# Max amount may exceed totalSupply

## Summary
The bug report is about a problem with the maxAmounts function, which can only be used by authorized people. The function should calculate the total amount of maxAmounts to make sure it does not exceed the totalSupply or balance of a specific address. However, this is not currently happening and it is recommended to add an internal function to calculate the sum of max amounts and prevent it from going over the limit. This function should be called in two other functions, createLGEWhitelist and modifyLGEWhitelist, after new values have been set. Alternatively, the current functionality and calculations should be confirmed.

## Details
**Description**

Although these functions can be called only by an authorized person, the total of
maxAmounts should be calculated in order not to exceed totalSupply or balance of
_lgePairAddress .
Also notice, that max amount for each round is a limit for each recipient, but not a total limit
for a whole round. Consider adding more checks.

**Recommendation**:

Add an internal function which calculates sum of max amounts and reverts if total is
exceeded. Call this function in both createLGEWhitelist and modifyLGEWhitelist functions after
setting new values to amountsMax. Or confirm the functionality and calculations.
