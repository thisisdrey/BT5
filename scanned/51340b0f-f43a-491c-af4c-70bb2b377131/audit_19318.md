# [M-02] Unchecked return value from low-level call()

## Summary
This bug report describes a vulnerability in the Ether contract, which has the potential to lock the Ether if the low-level call fails. The vulnerability was discovered through manual review and the proof of concept can be found at the given GitHub link. The recommended mitigation step is to add a condition to check the return value of the low-level call. This will ensure that the Ether is not locked in the contract if the call fails.

## Details
_Submitted by JMukesh, also found by certora_

#### Impact

The return value of the low-level call is not checked, so if the call fails, the Ether will be locked in the contract. If the low level is used to prevent blocking operations, consider logging failed calls.

#### Proof of Concept

<https://github.com/code-423n4/2021-12-amun/blob/98f6e2ff91f5fcebc0489f5871183566feaec307/contracts/basket/contracts/singleJoinExit/EthSingleTokenJoinV2.sol#L26>

#### Recommended Mitigation Steps

add condition to check return value
