# CurveProxy does not support chains other than Ethereum mainnet 

## Summary
The report explains that there is an issue with Bima's CurveProxy code, which was copied from Prisma Finance. The problem is that the code only works with Ethereum mainnet, and not with other chains like Arbitrum. The reviewer tried to make assumptions based on Bima's website, but there was no response from the Bima sponsor team. The code needs to be updated to use the correct API for minting CRV on sidechains, which is IGauge(gauge).factory().mint(gauge). This will ensure that the CurveProxy works with sidechains.

## Description
Bima's CurveProxy is copied from Prisma Finance. However, Prisma Finance only supports Ethereum mainnet, so it uses the Curve API of mainnet. But this is not compatible with other chains (e.g. Arbitrum).

**Note:** Bima sponsor team did not answer which chains this would be deployed on, so I made an assumption from their website (bima.money), since it says to support on Arbitrum Sepolia right now. 

See Curve Gauge's sidechain implementation differences with mainnet (see the curve docs). One of the major differences is when minting CRV, the correct API to use is `ChildGaugeFactory.mint(gauge: address)`. However, this is not the case in current CurveProxy code, because it calls `Minter.mint(gauge)` instead.

## Code Snippet
```solidity
// CurveProxy.sol#L168:
function mintCRV(address gauge, address receiver) external onlyApprovedGauge(gauge) returns (uint256 amount) {
    uint256 initial = CRV.balanceOf(address(this));
    minter.mint(gauge); // <<<
    amount = CRV.balanceOf(address(this)) - initial;

    // apply fee prior to transfer
    uint256 fee = (amount * crvFeePct) / BIMA_100_PCT;
    amount -= fee;

    CRV.transfer(receiver, amount);
    
    // lock and extend if needed
    uint256 unlock = unlockTime;
    uint256 maxUnlock = ((block.timestamp / WEEK) * WEEK) + MAX_LOCK_DURATION;
    if (unlock < maxUnlock) {
        _updateLock(initial + fee, unlock, maxUnlock);
    }
}
```

## Impact
CurveProxy does not work with sidechains.

## Recommendation
Use `IGauge(gauge).factory().mint(gauge)` for sidechain CRV minting.
