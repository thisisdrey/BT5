# UNTRUSTED TOKENS NOT CHECKING BEFORE AND AFTER BALANCE

## Summary
This bug report is about a problem in the code for a project called MoonscapeDefi. The code is located in the "contracts/defi/MoonscapeDefi.sol" file. The function "addTokenStaking" is not working properly and may cause issues for users. The impact of this bug is rated as 5 out of 10, and the likelihood of it happening is 2 out of 10. The recommended solution is to check the post balance and compare it to the sum of the pre-balance and the requested amount. This issue has been marked as "solved."

        if (_token != address(0)) {

            uint256 _balance = _rewardToken.balanceOf(address(this));
            if (_amount > _balance) {
                _rewardToken.transfer(_to, _balance);
            } else {
                _rewardToken.transfer(_to, _amount);
            }
        } else {

                payable(_to).transfer(_balance);
            } else {
                payable(_to).transfer(_amount);

        }
    }
