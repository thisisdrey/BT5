# Incorrect Token Addresses Passed For Quote Price

## Summary
The bug report discusses an issue with the calculation of amountOut in the UniswapOracle contract. It is caused by a mistake in passing the arguments for the getQuoteAtTick function, resulting in an unintended swap of the argument order. This leads to incorrect prices being returned for the quoteToken, causing further issues when the getTokenPrice function is called. The report recommends fixing this by ensuring the correct order of arguments is passed to getQuoteAtTick. This has been patched in the latest version of the contract.

## Details
## UniswapOracle Token Price Calculation Issue

The calculation of `amountOut` for the given amount of the base token is performed incorrectly in `getTokenPrice` inside `UniswapOracle`, where the quote token is USDC.

## UniswapOracle.sol (SOLIDITY)

```solidity
function getQuoteAtTick(
    int24 tick,
    uint128 baseAmount,
    address baseToken,
    address quoteToken
) internal pure returns (uint256 quoteAmount) {
    [...]
}
```

As highlighted above, `getQuoteAtTick` requires two token addresses as arguments: the `baseToken` address followed by the `quoteToken` address. However, in `getTokenPrice`, a call to `getQuoteAtTick` occurs with `baseToken` set to the USDC token address, and the intended base token is mistakenly passed as the `quoteToken` argument, resulting in an unintended swap of the argument order.

## UniswapOracle.sol (SOLIDITY)

```solidity
function getTokenPrice(
    uint128 amount,
    uint32 secondsAgo
) public view returns (uint amountOut) {
    (int24 tick, ) = consult(secondsAgo);
    amountOut = OracleLibrary.getQuoteAtTick(
        tick,
        amount,
        USDC_ADDRESS,
        address(tokenAddress)
    );
    [...]
}
```

This will return an incorrect price for the `quoteToken`, resulting in further issues whenever `getTokenPrice` is called.

## VTVL Audit 04 | Vulnerabilities

### Remediation

Ensure passing the correct order of arguments to `getQuoteAtTick`, where the `baseToken` address is passed first and then the `quoteToken` address.

## UniswapOracle.sol (SOLIDITY)

```solidity
function getTokenPrice(
    uint128 amount,
    uint32 secondsAgo
) public view returns (uint amountOut) {
    (int24 tick, ) = consult(secondsAgo);
    amountOut = OracleLibrary.getQuoteAtTick(
        tick,
        amount,
        address(tokenAddress),
        USDC_ADDRESS,
    );
    [...]
}
```

### Patch

Fixed in `cf25628` by ensuring that the arguments are passed in the correct order to `getQuoteAtTick`.
