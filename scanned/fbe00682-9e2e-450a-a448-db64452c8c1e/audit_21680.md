# [H01] Liquidation process could push the protocol into insolvency

## Summary
This bug report is about the liquidation process of a protocol potentially leading to its insolvency. This can happen when the collateral to oToken price fluctuation causes the collateralization ratio to drop, allowing someone to proceed with the liquidation. The liquidation process incentivizes liquidators by providing them with a liquidation bonus, but this can push the protocol further into insolvency when it is already under collateralization stress. A chart is included to demonstrate how the protocol's insolvency state is affected during a liquidation event, with 3 variables: `collateralizationRatio`, `liquidationIncentive`, and `liquidationFee`. It is suggested to set up an offline observation mechanism to ensure liquidation events happen as quickly as possible before the price gets too close to the mentioned value. The Opyn team is also implementing a liquidator bot to help solve this issue.

## Details
Under certain asset price change conditions, the liquidation process could push the protocol into insolvency. The design of the current liquidation process incentivizes liquidators by providing them with a liquidation bonus. However, at times when the protocol is already under collateralization stress, offering a liquidation bonus plus, in OpynŌĆÖs case, a protocol fee, will push further the particular Option into insolvency. Essentially these actions work against the original purpose of the liquidation function.


In the following chart we explore how the protocolŌĆÖs insolvency state is affected during a liquidation event, generated by the collateral to oToken price fluctuation, together with 3 variables: `collateralizationRatio`, `liquidationIncentive`, and `liquidationFee`.


![](https://i0.wp.com/blog.openzeppelin.com/wp-content/uploads/2020/01/liquidation.png?resize=840%2C336&ssl=1)


On the left, we issue `Vo` value of oTokens with collateral value of `Vc`:  

`Vc = Vo * CollateralizationRatio`


On the right, when total collateral value drops to `Vc1`, collateralization ratio drops, allowing someone to proceed with the liquidation. Assume that a user provides `Lo` value of oTokens for liquidation and that the `Repo` is under `liquidationFactor`. Then, the amount of collateral that will be deducted is:


`Lo * (1 + liquidationIncentive + liquidationFee)`


After this liquidation event, looking at the leftover oToken value and the leftover collateral value, if the leftover the collateral is less than the leftover the oToken value then the protocol is insolvent. Which means:


`Vc1 - Lo(1 + liquidationIncentive + liquidationFee) < Vo - Lo`


together with `Vc = Vo * collateralizationRatio` from the left part of the chart, we can get:


`Vc1 < Vc/collateralizationRatio + Lo(liquidationIncentive + liquidationFee)`


Which basically means that when the new value of collateral drops to this level, the liquidation process will push the protocol into insolvency. Plus from this moment on, and because the collateralization ratio is still low, further liquidation events are still allowed. Such events will further push the protocol deeper into insolvency.


Consider setting up an offline observation mechanism to ensure liquidation events happen as fast as they can before the price gets closer to the mentioned value.


**Update**: *The Opyn team is implementing a liquidator bot to help solve this issue.*
