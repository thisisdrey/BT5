# tokenToXtz sends the tokens to the user

## Summary
This bug report is about the RemoveLiquidity.hs file in Dexter, a tool for token conversion from tokens to tezos. The bug allows an attacker to drain all the assets from Dexter by calling the tokenToXtz function and receiving tezos while keeping its tokens. The vulnerability is caused by the tokens being sent to the user-controlled field, rather than the contract. 

The recommendation is to send the tokens to the Dexter contract in the short-term, and in the long-term create thorough unit tests and test each function using QuickCheck.

## Details
## Type: Undefined Behavior
**Target:** RemoveLiquidity.hs

**Difficulty:** Low

## Description
`tokenToXtz` sends the received tokens to a user-controlled address, which allows an attacker to drain all the assets from Dexter. `tokenToXtz` allows the conversion from tokens to Tezos. The user specifies how many tokens they want to convert. The tokens are supposed to be sent to the contract, but they are actually sent to the user-controlled `to` field:

```
dip ( do dip ( do getField # tokensSold; dip ( do getField # owner; dip (toField # to))); transferOwnerTokensTo)
```

**Figure 2.1:** TokenToXtz.hs#L58.

```
transferOwnerTokensTo(tokensSold, owner, to)
```

**Figure 2.2:** High-level representation of Figure 2.1.

As a result, anyone can call `tokenToXtz` and receive Tezos while keeping their tokens.

## Exploit Scenario
Dexter holds $1,000,000 worth of assets. Eve uses `tokenToXtz` to drain the exchange’s assets.

## Recommendation
**Short term:** Send the tokens to the Dexter contract.  
**Long term:** Create thorough unit tests and test each function. Consider using QuickCheck for every feature.
