# [M-04] Users cannot receive rewards from `ConcentratedLiquidityPoolManager` if their liquidity is too large

## Summary
This bug report is about an integer underflow error that could occur when a user claims an incentive reward. The problem is that the rewards that the user wants to claim exceed the incentive.rewardsUnclaimed, which causes an integer underflow error and prevents the user from getting the rewards. The bug is caused by the fact that the unclaimed reward is proportional to the `secondsInside`, which is proportional to the position's liquidity. If the liquidity is too large, then `secondsInside` can be larger than `secondsUnclaimed` and this causes the integer underflow error.

The recommended mitigation steps for this bug are to check whether the `rewards` exceeds the `incentive.rewardsUnclaimed`. If it does, then only send the `incentive.rewardsUnclaimed` amount of rewards to the user. The bug was acknowledged by Sushi, but they requested a proof of concept to be provided.

## Details
_Submitted by broccoli_

#### Impact
There could be an integer underflow error when the reward of an incentive is claimed, forcing users to wait for a sufficient period or reduce their liquidity to claim the rewards.

#### Proof of Concept
The unclaimed reward that a user could claim is proportional to the `secondsInside`, which is, in fact, proportional to the position's liquidity. It is possible that the liquidity is too large and causes `secondsInside` to be larger than `secondsUnclaimed`. As a result, the rewards that the user wants to claim exceed the `incentive.rewardsUnclaimed` and causes an integer underflow error, which prevents him from getting the rewards.

#### Recommended Mitigation Steps
Check whether the `rewards` exceeds the `incentive.rewardsUnclaimed`. If so, then send only `incentive.rewardsUnclaimed` amount of rewards to the user.
