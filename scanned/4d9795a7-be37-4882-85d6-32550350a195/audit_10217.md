# [H-01] Missing `payable` function

## Summary
This bug report states that there is a problem with a function called `_call()` in a contract called Calls.sol. The severity of this bug is medium, meaning it can have a significant impact on the wallet, and the likelihood of it occurring is high because the use of msg.value and sending native tokens is a common operation. Currently, none of the contracts, including Main.sol, PreauthorizedCalls.sol, SessionCalls.sol, Calls.sol, and Controllers.sol, can receive native tokens even though the function is supposed to be able to do so. The report recommends adding a payable modifier to related functions and adding a `receive()` function to fix the issue.

## Details
**Severity**

**Impact:** Medium, wallet will not receive native tokens but expected

**Likelihood:** High, because the use of msg.value and sending native token is a frequent operation

**Description**

`_call()` could send native token via `.call{ value: _callRequest.value }`, however, none of the contracts can receive native token, including Main.sol, PreauthorizedCalls.sol, SessionCalls.sol, Calls.sol, Controllers.sol.

```solidity
File: contracts\modules\Calls\Calls.sol
80:     function _call(CallsStructs.CallRequest calldata _callRequest) internal returns (bytes memory) {
81:         (bool success, bytes memory result) = _callRequest.target.call{ value: _callRequest.value }(_callRequest.data);
82:
```

**Recommendations**

- Add payable modifier to the related functions
- Add `receive()` function
