# Loss of Precision in `Rewarder._calibrateReward()`

## Summary
The client has marked a bug as "Fixed" in the file `Rewarder.sol` and provided an explanation for the fix. The bug was caused by a calculation that divided before multiplying, resulting in truncated values. The recommendation is to restructure the expression so that the multiplications occur before the divisions to avoid precision loss. The updated calculation is `rewardRate = (farmRewardConfig.apr * totalValue * priceData.precision) / (APR_PRECISION * DENOMINATOR * ONE_YEAR * priceData.price)`.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `bfbbd5b6e7134244a360115032c9ee3fad3cef85`. The client provided the following explanation:

> Implemented the recommended fix. Updated the rewardRate calculation statement.

**File(s) affected:**`Rewarder.sol`

**Description:** The calculation `(((farmRewardConfig.apr * totalValue) / (APR_PRECISION * DENOMINATOR)) / ONE_YEAR) * priceData.precision` divides before multiplying with the defined precision. This truncates the values in the divide step.

**Recommendation:** We recommend restructuring this expression such that the multiplications occur before the divisions to mitigate precision loss, i.e.:

```
rewardRate = (farmRewardConfig.apr * totalValue * priceData.precision) / (APR_PRECISION * DENOMINATOR * ONE_YEAR * priceData.price)
```
