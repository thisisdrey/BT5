# Off-by-One Possible in deployFunds

## Summary
The report addresses a bug in the `_deployFunds` function of the YRizStrategy.sol file. The bug is caused by an off-by-one error due to rounding, which can occur when two assets have a bps allocation of 5000 and the amount being deposited is 10001. This results in an incorrect amount being calculated for the second deposit, causing the remaining funds to be insufficient. The suggested solution is to transfer the remaining balance for the final asset instead of computing the amount using bps, as distributions should be descending. The bug has been resolved in a recent pull request.

## Details
In the [`_deployFunds` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L214) there could be an off\-by\-one error due to [rounding](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L222-L223). Consider the case where two assets have `5000` as their `bps` allocation and the [`_amount` is `10001`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L214). In this case, the [`amount`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L222) for both will be `5001`, which will fail upon the second deposit as the remaining funds will be `5000`.


Since [distributions should be descending](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L142), consider transferring the remaining balance for the final asset instead of computing the amount using `bps`.


***Update:** Resolved in [pull request \#94](https://github.com/radiant-capital/riz/pull/94) at commit [f8d4889](https://github.com/radiant-capital/riz/pull/94/commits/f8d4889840b34d42b3c646d2ceb80f2d9db292d4).*
