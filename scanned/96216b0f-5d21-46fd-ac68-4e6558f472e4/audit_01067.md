# _toggleYieldSourceActivation can DoSed via 1 wei transfer

## Summary
This bug report discusses a medium risk bug found in the code of SuperVaultStrategy.sol. The code currently has a strict requirement that the TVL (total value locked) owner share must be 0. However, a user can send 1 share to the contract, which can cause a denial of service (DOS) attack on the `_toggleYieldSourceActivation` function. The recommendation is to remove the `getTVLByOwnerOfShares` check to prevent this attack.

## Details
## Severity: Medium Risk

## Context
SuperVaultStrategy.sol#L787-L802

## Description
The code strictly requires that the TVL owner share is 0:

```solidity
if (IYieldSourceOracle(yieldSource.oracle).getTVLByOwnerOfShares(source, address(this)) > 0) {
    revert INVALID_AMOUNT();
}
```

Then a user can send 1 share to this contract to DOS the `_toggleYieldSourceActivation`.

## Recommendation
Consider removing the `getTVLByOwnerOfShares` check.
