# Incorrect logic in the `onlyAdminOrOperator()` modifier

## Summary
This bug report is about a medium severity issue that has already been resolved. The problem was with a modifier called `onlyAdminOrOperator()` in the `TokenPrice` contract. This modifier was supposed to only allow callers with either the `DEFAULT_ADMIN_ROLE` or `OPERATOR_ROLE` to pass, but the current logic only allows callers with both roles. The recommendation is to replace the `||` with `&&` to fix this issue.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

`onlyAdminOrOperator()` modifier in the `TokenPrice` contract is intended to be only passed when the caller has either `DEFAULT_ADMIN_ROLE` or `OPERATOR_ROLE`.
But the current logic of the modifier is only passed when the caller has both roles.

**Recommendation**: 

Replace || with &&.
