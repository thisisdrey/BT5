# Missing access control check

## Summary
Severity: High
Status: Resolved

Description:
The Contract VaultETH_V2 has a method called RM_UpdateReward(...) that is meant to only be accessible to accounts with the RISK_MANAGER role. However, this method currently does not have an access control check, which means that anyone can call it. This can lead to a Denial of Service (DoS) attack, as the method can mint unlimited ZeUSD tokens by calling the `SellETH()` method. This is a problem because the `SellETH()` method has no limit on the amount of tokens it can mint.

Recommendation:
To fix this issue, the developers should add a check in the RM_UpdateReward() method to ensure that only accounts with the RISK_MANAGER role can call it. This can be done by adding the following line of code: `assert(hasRole(RISK_MANAGER, msg.sender));`

## Details
**Severity**: High

**Status**: Resolved 

**Description**

In Contract VaultETH_V2, the method RM_UpdateReward(...) should only be callable by an account that has the RISK_MANAGER  role as per the comment similar to the method RM_UpdateDeposit(...).
Since this method has no access control check, malicious users can call this method which can cause DoS for other users to mint ZeUSD tokens as this method internally calls the `SellETH()` method which can mint max amount of ZeUSD as the `SellETH()` method allows this contract to mint as many tokens as possible with no upper limit. 

**Recommendation**: 

Add the following check in the method RM_UpdateReward().

```solidity
       assert(hasRole(RISK_MANAGER, msg.sender));
```
