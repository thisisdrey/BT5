# Function might revert due to underflow.

## Summary
This bug report describes an issue with a function called _settleStrike(). The function is supposed to perform a specific action, but when it is called during the first time period (epoch 0), it causes an error and reverts back to a previous state. The error is caused by an arithmetic operation that results in an underflow, which means the value becomes smaller than the minimum allowed value. The recommendation is to check for potential underflow/overflow before performing the operation. After further investigation, it was determined that this behavior is intentional and the function will always cause an error during epoch 0 because of a specific condition. This was confirmed by the client and no further action is needed.

## Details
**Description**

In case, function _settleStrike() is called during epoch 0, function will revert with underflow in
line 136 in 'epoch - 1'.

**Recommendation**:

Validate that arithmetic operation won’t revert due to overflow/underflow.

**Post-audit**:

Was verified after the consultation with the Client to be working as intended. _settleStrike will
always revert in epoch 0, as expiry will always be 0 in epoch 0, and will revert on require(expiry
> 0) check.
