# Undeclared identifier

## Summary
The bug report states that there is a critical issue in the code where a variable called `_decimals` is not defined but is being used in the constructor method. This can cause unexpected behavior and needs to be fixed. The recommendation is to declare the variable as `uint8 immutable private _decimals` in the contract and override the `decimals` function to return its value. This will resolve the issue and prevent any further problems.

## Details
**Severity**: Critical

**Status**: Resolved

**Description**

Variable `_decimals` is not defined but is set in the constructor method

**Recommendation**: 

declare the variable as `uint8 immutable private _decimals` in the contract and override the `decimals`

function to return it's value:

```solidity
    function decimals() public view virtual override returns (uint8) {
        return _decimals;
    }
```
