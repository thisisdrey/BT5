# Safe math library not used

## Summary
This bug report is about a smart contract that is not using the library for safe math, which can lead to an arithmetic overflow. This can cause incorrect operation of the smart contract. The contract in question is located at https://github.com/Grandthrax/yearnV2-generic-lender-strat/blob/979ef2f0e5da39ca59a5907c37ba2064fcd6be82/contracts/Strategy.sol and the arithmetic operations that need to be redone are on lines: 136, 155, 180, 206, 213, 287, 430, 464, 543, 547.

To fix the problem, the recommendation is to use the Safe math library for all arithmetic operations in the contract. This library will help to prevent arithmetic overflows and ensure that the smart contract runs correctly.

## Details
##### Description
If you do not use the library for safe math, then an arithmetic overflow may occur, which will lead to incorrect operation of smart contracts.
In the contract https://github.com/Grandthrax/yearnV2-generic-lender-strat/blob/979ef2f0e5da39ca59a5907c37ba2064fcd6be82/contracts/Strategy.sol on lines: 136, 155, 180, 206, 213, 287, 430, 464, 543, 547 calculations are without safe mathematics.

##### Recommendation
All arithmetic operations need to be redone using the Safe math library.
