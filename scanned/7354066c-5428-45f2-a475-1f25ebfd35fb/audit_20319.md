# TribalChief - Setting the totalAllocPoint to zero shouldn’t be allowed

## Summary
This bug report outlines an issue with the `TribalChief.updatePool` function. This function will revert when `totalAllocPoint = 0`, which will lock up user funds and rewards. The recommendation is to add an assert to `TribalChief.add` and `TribalChief.set` that `totalAllocPoint > 0`, as well as adding a similar validation check to `TribalChief.updatePool`. This will ensure that the function does not revert when `totalAllocPoint = 0`.

## Details
#### Description


`TribalChief.updatePool` will revert in the case `totalAllocPoint = 0`, which will essentially cause users' funds and rewards to be locked.


#### Recommendation


`TribalChief.add` and `TribalChief.set` should assert that `totalAllocPoint > 0`. A similar validation check should be added to `TribalChief.updatePool` as well.
