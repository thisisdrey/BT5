# No rate limiting for (s *DispersalServer) GetBlobStatus API requests

## Summary
The report highlights a medium risk bug in the disperser server code. The specific issue is located in `disperser/apiserver/server.go` at line 530. The bug is related to the `GetBlobStatus` function, which does not have a limit on the number of requests it can handle. This can lead to errors and excessive disk usage due to mutexes being held on blob metadata. The recommendation is to implement rate limiting for the RetrieveBlob API call to prevent these issues.

## Details
## Severity: Medium Risk

## Context
`disperser/apiserver/server.go#L530`

## Description
*(s *DispersalServer) GetBlobStatus does not enforce rate limiting.* This request holds mutexes on blob metadata and will cause error logging when requests fail, filling up disk space. Grieving these shared resources could be detrimental to proper disperser operation.

## Recommendation
Implement rate limiting for the RetrieveBlob API call.
