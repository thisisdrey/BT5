# [M-10] Initial deploy won't succeed because of too high `initialMintAmount` for USDC market

## Summary
This bug report concerns a problem with the deployment process for Moonwell, a token market. The issue is that the `initialMintAmount` is hardcoded to `1e18` which is appropriate for WETH (1,876 USD) but not for USDC (1,000,000,000,000 USD). This means that the deploy will fail.

The problem lies in the code where `initialMintAmount = 1 ether` is set. This amount is then approved to the MToken contract and supplied to mint MToken.

ElliotFriedman (Moonwell) confirmed the issue but disagreed with the severity, as the deploy script was only configured to work on local testnets where they control the ERC20 tokens. Alcueca (Judge) commented that the issue is still valid as Medium with the code and knowledge available to the wardens.

The recommended mitigation step is to specify `initialMintAmount` for every token separately in the config.

## Details
At the time of deploy, deployer initializes token markets with initial amount to prevent exploit. At least USDC and WETH markets will be initialized during deploy.
But `initialMintAmount` is hardcoded to `1e18` which is of for WETH (1,876 USD), but unrealistic for USDC (1,000,000,000,000 USD). Therefore deploy will fail.

```solidity
    /// @notice initial mToken mint amount
    uint256 public constant initialMintAmount = 1 ether;
```

Here this amount is approved to MToken contract and supplied to mint MToken: <br><https://github.com/code-423n4/2023-07-moonwell/blob/fced18035107a345c31c9a9497d0da09105df4df/test/proposals/mips/mip00.sol#L334-L379>

```solidity
            for (uint256 i = 0; i < cTokenConfigs.length; i++) {
                Configs.CTokenConfiguration memory config = cTokenConfigs[i];

                address cTokenAddress = addresses.getAddress(
                    config.addressesString
                );

               ...

                /// Approvals
                _pushCrossChainAction(

                    "Approve underlying token to be spent by market"

                /// Initialize markets
                _pushCrossChainAction(

                    abi.encodeWithSignature("mint(uint256)", initialMintAmount),

                ...
            }
```

### Recommended Mitigation Steps

Specify `initialMintAmount` for every token separately in config
