# Remove duplicate code

## Summary
This bug report is about duplicate code in the Ether-Camp Virtual Accelerator. Duplicate code is code that is repeated multiple times in a program, which can lead to confusion and hidden bugs. The report recommends six ways to remove the duplicate code, which includes extracting code into a generic function, removing the first check inside the fallback function, combining two functions, replacing one function with another, and removing a duplicate check. Following these steps will help reduce the risk of introducing hidden bugs and make the code easier to understand.

## Details
Duplicate code makes it harder to understand the code’s intention and thus, auditing the code correctly. It also increases the risk of introducing hidden bugs when modifying one of the copies of some code and not the others. We recommend the following to remove duplicate code:


* `submitEtherProposal` and `submitHKGProposal` share a lot of code, which could be extracted into a generic function.
* [The first check inside the fallback function](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L131-L135) could be removed by adding the `onlyAfterEnd` modifier.
* `onlyBeforeEnd` and `onlyAfterEnd` are declared both in [VirtualExchange.sol](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/VirtualExchange.sol#L168) and in [DSTContract.sol](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L704).
* `convert` is repeated in both [VirtualExchange.sol](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/VirtualExchange.sol#L112) and [DSTContract.sol](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L688).
* [`isExistByString`](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/VirtualExchange.sol#L136) could be implemented by calling `isExistByBytes`.
* [This check](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L193-L195) in `issuePreferredTokens` is a duplicate of the `onlyIfAbleToIssueTokens` function modifier.
