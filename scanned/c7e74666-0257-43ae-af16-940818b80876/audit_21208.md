# BondingCurve allows users to acquire FEI before launch

## Summary
A bug was discovered in the Fei Protocol Core that allowed anyone to mint themselves Fei each time the timer expired. This was due to the fact that `allocate` could be called before genesis launch, as long as the contract held some nonzero PCV. This could be done by force-sending the contract 1 wei, which allowed users to bypass the majority of checks and actions in `allocate`. The bug was addressed in fei-protocol/fei-protocol-core#59, and it was recommended that `allocate` be prevented from being called before genesis launch.

## Details
#### Resolution



This was addressed in [fei-protocol/fei-protocol-core#59](https://github.com/fei-protocol/fei-protocol-core/pull/59)


#### Description


`BondingCurve.allocate` allocates the protocol’s held PCV, then calls `_incentivize`, which rewards the caller with FEI if a certain amount of time has passed:


**code-update/contracts/bondingcurve/BondingCurve.sol:L180-L186**



```
/// @notice if window has passed, reward caller and reset window
function \_incentivize() internal virtual {
    if (isTimeEnded()) {
        \_initTimed(); // reset window
        fei().mint(msg.sender, incentiveAmount);
    }
}

```
`allocate` can be called before genesis launch, as long as the contract holds some nonzero PCV. By force-sending the contract 1 wei, anyone can bypass the majority of checks and actions in `allocate`, and mint themselves FEI each time the timer expires.


#### Recommendation


Prevent `allocate` from being called before genesis launch.
