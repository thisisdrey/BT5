# Absence of Verification of Issuance Account Index

## Summary
The report discusses a bug in the purchase_order::process_redeem_purchase_order function that allows an attacker to make a profit by providing an old issuance account with a lower bond price. This is because the function does not verify the index of the issuance account against the current issuance number. The report also mentions a missing account check in the process_create_purchase_order function, which could result in the user's payment token account being set to another user's account. The recommended solution is to verify the issuance index and add account checks for the user's payment token account. The bug has been resolved in a recent patch.

## Details
## Vulnerability Report: Purchase Order Processing

## Overview

In the function `purchase_order::process_redeem_purchase_order`, there is a critical issue where the provided issuance account is not verified to correspond with a valid and current issuance. Specifically, it fails to check if the index of the issuance account matches the expected value derived from deterministic seeds.

## Vulnerability Details

An attacker could exploit this vulnerability by supplying an old issuance account. This would allow them to take advantage of a lower bond price due to previous market conditions or earlier issuance terms. Since the issuance index is not validated against the current issuance number stored in `bond`, the system will incorrectly accept this outdated issuance as valid. Consequently, purchasing the bonds at a lower price would enable the attacker to realize an instant profit.

### Code Snippet

```rust
pub fn process_redeem_purchase_order(
    _program_id: &Pubkey,
    accounts: &[AccountInfo],
) -> ProgramResult {
    [...]
    // #[account(3, name = "issuance_account")]
    validate!(
        issuance_account_info.owner == &crate::ID,
        ErrorCode::InvalidIssuanceOwner
    )?;
    let issuance = Issuance::unpack(&issuance_account_info.data.borrow())?;
    validate!(
        issuance.status == IssuanceStatus::Started || issuance.status == IssuanceStatus::Matured,
        ErrorCode::InvalidIssuanceStatus
    )?;
    validate!(
        issuance.parent_bond == *bond_account_info.key,
        ErrorCode::InvalidIssuance
    )?;
    Ok(())
}
```

Additionally, the function `process_create_purchase_order` is missing account checks for the `user_payment_token_account_info`. This oversight could allow `user_payment_token_account_info` to be set to another user’s account.

## Remediation

- Verify the issuance index with `bond.issuance_number`.
- Add account checks for `user_payment_token_account_info`.

## Patch

This issue was resolved in commit [6850535](https://github.com/your-repo/commit/6850535).
