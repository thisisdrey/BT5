# Improper Full Access Key Fallback Design

## Summary
This bug report is about a Full Access Key that has total control over an account. Full Access Keys have more privileges than the owner and can do more than the owner. The bug states that the current design of the contract allows the owner to add a new Full Access Key and gain more privileges than expected. This is a security issue because the contract should not allow the owner to gain more privileges than expected. To fix this issue, the contractor should limit the privileges of the Full Access Key and ensure that the owner does not gain more privileges than expected.

## Details
**Description:** 

A Full Access Key could have total control over the account. It can transfer NEAR, deploy a smart contract and so on. Full access keys have more privileges than the owner and can do more things than the owner. In the current design, the owner of the contract can attach a new full-access key and can gain more privileges than expected.
