# M-3: Vulnerability related to ‘Optimizer Bug Regarding Memory Side Effects of Inline Assembly’

## Summary
This bug report is related to a vulnerability in Solidity versions 0.8.13 and 0.8.14 related to an optimizer bug regarding memory side effects of inline assembly. The bug was found by 0xSmartContract and 0xheynacho and can occur under specific conditions, such as the legacy optimizer being enabled and the assembly blocks not referring to any local Solidity variables. The code snippet and tool used to find this bug was manual review. The recommendation is to use the recent version 0.8.15 which has the fix for these issues. The discussion section includes a fix by jparklev and verification of the fix by aktech297. To prevent this vulnerability, it is recommended to use the latest version of Solidity, 0.8.15, and to ensure that the legacy optimizer is not enabled and that assembly blocks do not refer to any local Solidity variables.

## Summary
Solidity versions 0.8.13 and 0.8.14 are vulnerable to a recently reported [optimizer bug](https://blog.soliditylang.org/2022/06/15/inline-assembly-memory-side-effects-bug/) related to inline assembly. Solidity 0.8.15 has been released with a fix.

## Vulnerability Detail
AutoRoller.sol inherits solidity contracts from  solmate FixedPointMathLib and ERC4626 , and both these uses inline assembly, and optimization is enabled while compiling.


## Impact
This bug only occurs under very specific conditions: the legacy optimizer must be enabled rather than the IR pipeline (true for the current project configuration), and the affected assembly blocks must not refer to any local Solidity variables.
