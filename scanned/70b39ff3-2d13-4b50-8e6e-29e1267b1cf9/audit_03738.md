# In _getSwingFactor(): reserveImpact >= targetReserveRatio not handled correctly 

## Summary
The bug report describes an issue in the code for QuoterV1.sol, specifically in the functions _getSwingFactor() and calculateDepositBonus(). The problem occurs when the reserveImpact value is equal to or greater than the targetReserveRatio, resulting in a 100% bonus instead of the intended behavior. The report suggests changing the code to handle this situation correctly, and also mentions that the issue has been fixed in a pull request (PR 231) and verified by Cantina Managed.
