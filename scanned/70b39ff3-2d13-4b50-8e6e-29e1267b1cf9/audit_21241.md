# [M02] Casting between types without overflow checks

## Summary
This bug report highlights the potential for overflow errors when downcasting or casting between `int` and `uint` types in the codebase. It provides three examples of such behavior, which can be found on lines 382 and 466 of `FutureCash.sol`, and line 1098 of `Escrow.sol`. OpenZeppelin's `SafeCast` library is suggested to be used when casting between types to prevent overflow errors. Partially fixed in pull request #54, however line 1002 of `Escrow.sol` still uses casting that allows for overflows. OpenZeppelin will verify this in an upcoming 2nd audit.

## Details
Throughout the codebase, many instances of both implicit and explicit casting between types exist, with both having the potential to impart overflow errors when downcasting or casting between `int` and `uint`.


Some examples of such behavior can be found on:  

– [line 382 of `FutureCash.sol`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L382) where a `uint256` value is downcast to `uint128`.  

– [line 466 of `FutureCash.sol`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L466) where a `uint256` value is downcast to `uint128`.  

– [line 1098 of `Escrow.sol`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1098) where a `uint256` is cast to `int256`.


OpenZeppelin’s [`SafeCast`](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/release-v3.1.0/contracts/utils/SafeCast.sol) library provides overflow checking when casting from one type of number to another. Consider using `SafeCast` when casting between types to prevent overflow errors.


***Update:** Partially fixed in [pull request #54](https://github.com/swapnet-protocol/swapnet-lite/pull/54). [Line 1002 of `Escrow.sol`](https://github.com/swapnet-protocol/swapnet-lite/blob/63f015aefd8bdb6a106f4a1dce9ad072f3ec279e/packages/contracts/contracts/Escrow.sol#L1002) still uses casting in a way that allows for overflows, however Notional state this logic is removed. OpenZeppelin will verify this in an upcoming 2nd audit.*
