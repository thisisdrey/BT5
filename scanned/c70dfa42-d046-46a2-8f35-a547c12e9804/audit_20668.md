# zBanc - DynamicLiquidTokenConverter ineffective reentrancy protection ✓ Fixed

## Summary
This bug report is about a potential for direct reentrancy when an ERC-777 token is used as reserve in zBanc's DynamicLiquidTokenConverter.sol contract. The issue is that the `reduceWeight` function calls `_protected()` in an attempt to protect against reentrant calls, but this check is insufficient as it only checks for the `locked` statevar but never sets it. To fix the issue, the recommendation is to remove the call to `_protected()` and decorate the function with `protected` instead, which will properly set the lock before executing the function body and reject reentrant calls.

## Details
#### Resolution



Fixed with [zer0-os/[email protected]`ff3d913`](https://github.com/zer0-os/zBanc/commit/ff3d91390099a4f729fe50c846485589de4f8173) by following the recommendation.


#### Description


`reduceWeight` calls `_protected()` in an attempt to protect from reentrant calls but this check is insufficient as it will only check for the `locked` statevar but never set it. A potential for direct reentrancy might be present when an erc-777 token is used as reserve.


It is assumed that the developer actually wanted to use the `protected` modifier that sets the lock before continuing with the method.

```
contract ReentrancyGuard {
    // true while protected code is being executed, false otherwise
    bool private locked = false;

    /\*\*
 \* @dev ensures instantiation only by sub-contracts
 \*/
    constructor() internal {}

    // protects a function against reentrancy attacks
    modifier protected() {
        \_protected();
        locked = true;
        \_;
        locked = false;
    }

    // error message binary size optimization
    function \_protected() internal view {
        require(!locked, "ERR\_REENTRANCY");
    }
}

```
#### Recommendation


To mitigate potential attack vectors from reentrant calls remove the call to `_protected()` and decorate the function with `protected` instead. This will properly set the lock before executing the function body rejecting reentrant calls.
