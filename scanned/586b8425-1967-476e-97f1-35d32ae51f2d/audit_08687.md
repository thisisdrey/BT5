# Partial Withdrawal Requests of `ZCTokens` Results in Withdrawing More than What Is Requested

## Summary
This report discusses a bug found in the `LendingMarketUserLogic.sol` file, where users are unable to make partial withdrawals of ZC tokens. This is due to the `isAll` flag, which determines whether a user can withdraw their entire position for a specific maturity. This means that if the flag is set to `true`, even if a user intends to withdraw only a portion of their tokens, all of their tokens will be locked. This could affect the user's risk management strategies. The client has marked this bug as "Fixed" and the code has been updated to allow for partial withdrawals of ZC tokens.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `3e8bda875a75cd8991ede8d0c35dff322166aad5`.

**File(s) affected:**`LendingMarketUserLogic.sol`

**Description:** Users have the capability to withdraw a specified `_amount` of ZC tokens using the `_withdrawZCToken()` and `_withdrawZCPerpetualToken()` functions. However, the presence of the `isAll` flag, as determined by the `_getWithdrawableZCTokenAmount()` function, indicates that users can withdraw their entire position for a specific maturity. This functionality effectively eliminates the option for partial withdrawals. Specifically, based on the `isAll` value, either `0` or the `_amount` specified will be forwarded to the `lock()` function. The `lock()` function interprets an amount of `0` as an instruction to lock the user's total balance or the specified amount. Consequently, if `isAll` is set to `true` for a user intending to withdraw only `8` out of `10` tokens, all `10` tokens will be locked, not just the `8` requested. This could inadvertently impact the user's risk management strategies, as it differs from their initial withdrawal intent.

**Note:** This issue was raised by the client during the audit process.

**Recommendation:** Consider updating the code to support partial withdrawals of ZC Tokens.
