# Unrestricted approval to hardcoded address

## Summary
This bug report is about the risk of giving OpenSea full control over users' tokens. It is noted that the address 0x53d7...96ad is an autoapproved operator, meaning it can take action on any user NFT. However, it is not possible to know if the code contains genuine code as it is unverified. 

The report recommends removing the hardcoded address and letting the user decide if they want to approve an address to take actions on behalf of their NFT. Alternatively, deploying a ContractRegistry to handle hardcoded addresses dynamically can also be done, keeping in mind that the OpenSea address is different on Mumbai.
