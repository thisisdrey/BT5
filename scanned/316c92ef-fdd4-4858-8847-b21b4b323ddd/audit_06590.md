# Order of Operations Will Reduce Output Precision

## Summary
The code in the `EmergencyWithdraw` library has a bug that affects the precision of the result. The code first divides the value by the price and then multiplies it by `DEFAULT_DENOMINATOR`, causing the result to lose some precision. This can be fixed by swapping the operations in the code. The bug has been resolved in a recent update.

## Details
In the `EmergencyWithdraw` library, the code [gets the amount of tokens](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/libraries/EmergencyWithdraw.sol#L84) owed to the user for each reserve in the market. The mentioned operation results in:


`(valueInUSD[i] / priceAsset[i]) * DEFAULT_DENOMINATOR`


However, because the division operation comes first, the value in assets will be rounded down by the price and then the result will be multiplied up by `DEFAULT_DENOMINATOR` (1e18\). This means that it will have 18 ending zeros and part of the precision of the operation will be lost.


Consider swapping the operations to improve the precision of the result.


***Update:** Resolved in [pull request \#83](https://github.com/radiant-capital/riz/pull/83) at commit [c92e990](https://github.com/radiant-capital/riz/pull/83/commits/c92e9908e159a1ea234bfa1b9f6cc3fd46e6a4fa).*
