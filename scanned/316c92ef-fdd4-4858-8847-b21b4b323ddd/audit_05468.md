# Missing call to computeCollateralOrlvlUSDAmount() in LevelMinting.mintDefault() 

## Summary
The bug report discusses an issue in the code for LevelMinting.sol, specifically in the mintDefault() function. This function is used to mint a specific amount of a currency called lvlUSD, but it does not properly validate the input from the user. This means that users can enter any value for lvlUSD and set the collateral amount to 1, which allows them to mint an infinite amount of lvlUSD. The recommendation is to add a new function called computeCollateralOrlvlUSDAmount() to properly validate the input before minting. This bug has been fixed in a recent commit and has been verified by the Cantina team.

## Details
## Code Review Summary

## Context
- **File:** LevelMinting.sol
- **Lines:** 226-233

## Description
The function `Bridge.mintDefault()` directly passes the order struct specified by the user to `_mint()`:

```solidity
function mintDefault(
    Order memory order
) external virtual onlyMinterWhenEnabled {
    if (msg.sender != order.benefactor) {
        revert MsgSenderIsNotBenefactor();
    }
    _mint(order, _route);
}
```

However, this means that `lvlusd_amount` and `collateral_amount` do not have any input validation. Therefore, users can specify `lvlusd_amount` as anything and `collateral_amount = 1` to mint infinite `lvlUSD`.

## Recommendation
Add the following call to `computeCollateralOrlvlUSDAmount()`:

```solidity
function mintDefault(
    Order memory order
) external virtual onlyMinterWhenEnabled {
    if (msg.sender != order.benefactor) {
        revert MsgSenderIsNotBenefactor();
    }
    // Before
    _mint(order, _route);
    
    // After
    Order memory _order = computeCollateralOrlvlUSDAmount(order);
    _mint(_order, _route);
}
```

## Status
- **Level Money:** Fixed in commit `532771c9`.
- **Cantina:** Verified, the missing call was added.
