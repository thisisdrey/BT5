# Low accuracy. After division, loss of accuracy is possible.

## Summary
The bug report suggests using a specific method to improve the accuracy of calculations in Solidity, a programming language used for creating smart contracts on the Ethereum blockchain. This method involves assigning a percentage value to a large number, such as 10^27, and then using that value to calculate smaller percentages. This is recommended because Solidity does not support fractional numbers, and this method can improve the accuracy of calculations. An example is provided to demonstrate how this method can be used.

**Recommendation**:
Since there are no fractional numbers in solidity, one method is used. We can take any
number as one hundred percent. For example, let's say we have 100% = 10 ^ 27, then 1% = 10
^ 25.
