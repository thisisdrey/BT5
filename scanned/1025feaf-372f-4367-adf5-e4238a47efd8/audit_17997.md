# Order book arithmetic overflow

## Summary
The bug report states that there is an issue with Aob's fixed point operations, specifically with the functions fp32_div and fp32_mul. These functions may overflow while casting down to u64, resulting in unexpected behavior. This can be seen in the OrderBookState::new_order method, where fp32_div may overflow when limit_price is less than one, causing base_qty_to_post to be unexpectedly constrained. To fix this issue, Aob's fixed point operations should be rewritten with checked arithmetic and casting, similar to what Dex already uses. This has already been addressed in a patch.

## Details
## Aob's Fixed Point Operations

Aob's fixed point operations can overflow, resulting in unexpected behavior. In particular, `fp32_div` and `fp32_mul` may overflow while casting down to `u64`.

```rust
// agnostic-orderbook/program/src/utils.rs:L76-L84
/// a is fp0, b is fp32 and result is a/b fp0
pub(crate) fn fp32_div(a: u64, b_fp32: u64) -> u64 {
    (((a as u128) << 32) / (b_fp32 as u128)) as u64
}

/// a is fp0, b is fp32 and result is a*b fp0
pub(crate) fn fp32_mul(a: u64, b_fp32: u64) -> u64 {
    (((a as u128) * (b_fp32 as u128)) >> 32) as u64
}
```

As an example, consider the `OrderBookState::new_order` method.

```rust
// agnostic-orderbook/program/src/orderbook.rs:L255-L258
let base_qty_to_post = std::cmp::min(
    fp32_div(quote_qty_remaining, limit_price),
    base_qty_remaining,
);
```

Notice that `fp32_div` may overflow when `limit_price` is less than one. If the result is smaller than `base_qty_remaining`, this will unexpectedly constrain `base_qty_to_post`.

## Remediation

Aob's fixed point operations should be rewritten with checked arithmetic and casting, identical to what Dex already uses.

```rust
// dex-v4/program/src/utils.rs:L39-L60
/// a is fp0, b is fp32 and result is a/b fp0
pub(crate) fn fp32_div(a: u64, b_fp32: u64) -> Option<u64> {
    ((a as u128) << 32)
        .checked_div(b_fp32 as u128)
        .and_then(safe_downcast)
}

/// a is fp0, b is fp32 and result is a*b fp0
pub(crate) fn fp32_mul(a: u64, b_fp32: u64) -> Option<u64> {
    (a as u128)
        .checked_mul(b_fp32 as u128)
        .and_then(|e| safe_downcast(e >> 32))
}

fn safe_downcast(n: u128) -> Option<u64> {
    static BOUND: u128 = u64::MAX as u128;
    if n > BOUND {
        None
    } else {
        Some(n as u64)
    }
}
```
