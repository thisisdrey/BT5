# H-1: All fund from Teller contract can be drained because a malicious receiver can call reclaim repeatedly

## Summary
This bug report concerns the Teller contract of the 2023-06-bond-judging project. The issue is that all funds from the Teller contract can be drained because a malicious receiver can call reclaim repeatedly. The bug was found by 0xcrunch, Jiamin, Juntao, Kow, Musaka, Sm4rty, Yuki, berndartmueller, bin2chen, circlelooper, ctf_sec, kutugu, stuxy, tnquanghuy0512, and tvdung94. 

The vulnerability detail is that when minting an option token, the user is required to transfer the payout token for a call option or quote token for a put option. If after the expiration, the receiver can call reclaim to claim the payout token if the option type is call or claim the quote token if the option type is put. However, the root cause is when reclaiming the token, the corresponding option is not burnt. As a result, the Teller contract is likely to hold funds from multiple option tokens and a malicious actor can create call Teller#deploy and set a receiver address that can be controlled by himself and then wait for the option expiry and repeated call reclaim to steal the fund from the Teller contract.

The impact of this bug is that all funds from the Teller contract can be drained because a malicious receiver can call reclaim repeatedly. The code snippet associated with this bug is linked to in the report. The tool used to find this bug was manual review. The recommendation for fixing this bug is to burn the corresponding option when reclaiming the fund.

Oighty and ctf_sec discussed the fix, agreeing that a simpler solution would be to store whether a specific option token has been reclaimed in a mapping and not allow that token to be reclaimed again. The fix was implemented at the linked Github page and ctf_sec confirmed that it looks good.

In conclusion, this bug report is about the Teller contract of the 2023-06-bond-judging project. All funds from the Teller contract can be drained because a malicious receiver can call reclaim repeatedly. The fix for this bug is to burn the corresponding option when reclaiming the fund.
