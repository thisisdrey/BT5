# Unclear whether a finalize agent is required for a crowdsale

## Summary
This bug report focuses on the `Crowdsale` contract of the CoinFabrik/ico repository on GitHub. The code of the finalize function states that the “finalizing is optional”, however, the `Crowdsale` can never move out of the `Preparing` state if no finalize agent is set.

The report suggests that if the intention is for the finalize agent to be mandatory, it should be accepted as part of the constructor parameters, and the method to change it on the fly should be removed. On the other hand, if the intention is to support no finalize agent, an alternative way to release the tokens should be provided.

Hubii replied that the `Preparing` stage was removed and the `finalizeAgent` is now mandatory.

## Details
Although the code of the finalize function [points out](https://github.com/CoinFabrik/ico/blob/142f43f3bec4afd0ffff3fce06946cf67cb0272f/contracts/Crowdsale.sol#L319) that *“Finalizing is optional. We only call it if we are given a finalizing agent”*, the `Crowdsale` can never move out of the `Preparing` state as per [L481](https://github.com/CoinFabrik/ico/blob/142f43f3bec4afd0ffff3fce06946cf67cb0272f/contracts/Crowdsale.sol#L481) if no finalize agent is set.


If the intention is for the finalize agent to be mandatory, consider accepting it as part of the constructor parameters, and remove the method to change it on the fly. On the other hand, if the intention is to support no finalize agent, ensure there is an alternative way to release the tokens, which is currently responsibility of the finalize agent.


**Update:** Hubii replied *“As part of another issue the `Preparing` stage was removed, and now the `finalizeAgent` is mandatory.”*
