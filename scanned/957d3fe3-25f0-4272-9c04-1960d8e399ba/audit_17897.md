# [M-13] Early funds withdrawers can get bonus in multiples of vested bonus tokens (e.g. 2-times, 3-times, etc.)

## Summary
This bug report is about a vulnerability in the code of the Rubicon project, which is a decentralized finance platform. The vulnerability is present in the function setBonusToken, which allows the same BonusToken to be added more than once to the array bonusTokens. This can lead to early withdrawers getting Bonus in multiples of what they actually have right to, while late withdrawers might not get any Bonus due to shortage.

The bug was identified by manual review of the code, and the recommended mitigation step is to add validations to avoid duplicate additions of bonus tokens. This can be done by adding a require statement to the setBonusToken function, which checks if the newBonusERC20 is already present in the bonusTokens array. If it is, the require statement will throw an error.

### Vulnerability details

The function setBonusToken allows the same BonusToken to be added more than once to the array bonusTokens.

      function setBonusToken(address newBonusERC20) external onlyBathHouse {
        bonusTokens.push(newBonusERC20);
      }

### Impact

If that happens, early withdrawers can get Bonus in multiples of what they actually have the right to. Late withdrawers, might not get any Bonus due to shortage.

### Recommended Mitigation Steps

Add the required validations to avoid duplicate additions of bonus tokens.

      function setBonusToken(address newBonusERC20) external onlyBathHouse {
        require(newBonusERC20 != address(0), "invalid_addr");
        if (bonusTokens.length > 0) {
          for (uint256 index = 0; index < bonusTokens.length; index++) {
            require (token != newBonusERC20, "token already exists")
          }
        }
        bonusTokens.push(newBonusERC20);
      }
