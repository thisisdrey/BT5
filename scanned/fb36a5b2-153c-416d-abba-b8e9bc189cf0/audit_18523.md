# Unimplemented getAmountsOut function in Balancer V2

## Summary
This bug report is about the getAmountsOut function in the BalancerVaultV2Wrapper contract which is unimplemented. The purpose of this function is to allow users to know the amount of funds they will receive when executing a swap. Because the function does not invoke any functions on the Balancer Vault, a user must actually perform a swap to determine the amount of funds they will receive. 

For example, Alice, a user of the Composable Finance vaults, wants to swap 100 USDC for DAI on Balancer. Because the getAmountsOut function is not implemented, she is unable to determine how much DAI she will receive before executing the swap.

The short-term recommendation to fix this issue is to implement the getAmountsOut function and have it call the queryBatchSwap function on the Balancer Vault. The long-term recommendation is to add unit tests for all functions to test all flows. This will detect incorrect function behavior.
