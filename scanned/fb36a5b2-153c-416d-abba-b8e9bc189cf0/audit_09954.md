# Collector Role Is Not Relinquished During Lock Transferral

## Summary
The bug report states that there is an issue with the `setCollectAddress` and `setAdditionalCollector` functions in the liquidity locker contract for UNCX, a cryptocurrency. These functions allow the owner of the lock to set addresses for collecting fees. However, when the lock is transferred to a new owner, these addresses are not reset. This means that the previous owner can still collect fees even after transferring the lock. The report suggests either resetting the addresses or forcing the new owner to set them when the lock is transferred. The bug has been resolved in a recent update.

## Details
Owners of a lock can set collector addresses through the [`setCollectAddress`](https://github.com/uncx-private-repos/liquidity-locker-univ3-contracts/blob/342c621cc93a13882601b30e547907877f3e3f86/contracts/UNCX_ProofOfReservesV2_UniV3.sol#L414) and [`setAdditionalCollector`](https://github.com/uncx-private-repos/liquidity-locker-univ3-contracts/blob/342c621cc93a13882601b30e547907877f3e3f86/contracts/UNCX_ProofOfReservesV2_UniV3.sol#L403) functions. However, these addresses are not reset when the lock is transferred. This means that the previous owner can still call `collect` on a `lock` that they previously owned and collect the market fees.


Consider resetting the previous `collector` and `additionalCollector` addresses or forcing the new owner to set them whenever a `lock` is transferred.


***Update**: Resolved in [pull request #1](https://github.com/uncx-private-repos/liquidity-locker-univ3-contracts/pull/1) at commit [c1e2718](https://github.com/uncx-private-repos/liquidity-locker-univ3-contracts/pull/1/commits/c1e2718fb4bd9590d690c698926d0c502c87c292).*
