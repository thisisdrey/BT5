# M-34: Lack of error handling when making blockless api call

## Summary
The bug report discusses an issue with error handling when making a blockless API call. This means that when the API call is made, if it fails for any reason, the code does not handle the error and continues as if the call was successful. This can lead to incorrect data being used and can cause problems with the system. The code snippet provided shows where the error handling is missing and how it can affect the data being used. The bug was discovered through manual review and it is recommended that the error handling is implemented to prevent any potential issues. The team responsible for the protocol has fixed the issue in a recent update.
