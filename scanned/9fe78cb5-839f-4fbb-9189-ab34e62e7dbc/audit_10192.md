# Lack of Timestamp Sanitization

## Summary
The set_price function is missing a check to ensure that the timestamp being input is valid, which could potentially lead to problems in the protocol. To fix this, a thorough check should be implemented for the timestamp. This issue has been resolved in a recent patch by adding an assertion that the timestamp is normalized and not set to a future date. This change can be seen in the code DIFF, where a new condition has been added to check for a valid timestamp before executing the function.

## Details
## Sanity Check for set_price Protocol

The absence of a sanity check in `set_price` exposes the protocol to potential issues by permitting the insertion of invalid timestamps.

## Remediation

Implement a thorough sanity check for the input timestamp within the `set_price` entry point.

## Patch

Fixed in commit `3716f44` by asserting that the input timestamp is both normalized and not set to a future date.
