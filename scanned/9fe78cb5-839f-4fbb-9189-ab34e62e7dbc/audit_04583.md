# `timeoutPacket()` and `timeoutOnClose()` Incorrectly Enforce an Open Channel State

## Summary
The client has marked an issue as "Fixed" and it has been addressed in a specific file called `IBCChannelPacketTimeout.sol`. The issue is related to the protocol's support for optimistic packet sends, where packets are sent on channels that may not open in the future or may never open. The problem is that two functions in the file incorrectly enforce an open channel state, causing issues with timing out packets. The recommendation is to allow packets to be timed out even when the channel is not in an "OPEN" state.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `3b309dd6b4fd6339973c47979db114754387ef2b`.

**File(s) affected:**`IBCChannelPacketTimeout.sol`

**Description:** As the protocol is intended to support optimistic packet sends, that will be sent on channels that will either open in the future, or never open, there should be support for the timing out of packets on channels that are in a state different than `OPEN`. However, these two functions incorrectly enforce an open channel state.

**Recommendation:** Allow packets to be timed out when the Channel is in a state other than OPEN.
