# [M-09] Users of Migration.sol may forfeit rebase rewards

## Summary
This bug report is about a vulnerability in the moveFundsToUpgradedContract() function in the Migration.sol contract. The vulnerability may cause users to forfeit rebase rewards. The proof of concept showed that the function calls instantUnstake(false) which skips the optional rebase. This means that if there is a pending rebase, the user calling the function will not get this rebase and miss out on potential rewards. To mitigate this issue, it is recommended to add an input bool _trigger and add the code to the start of the function that allows users to optionally call rebase if they are concerned about losing pending rebase rewards.

## Details
_Submitted by 0x52, also found by berndartmueller_

Users of `moveFundsToUpgradedContract()` in migration.sol may forfeit rebase rewards.

### Proof of Concept

L54 calls instantUnstake(false) meaning that it skips the optional rebase. If there is a pending rebase then the user calling the function will not get this rebase and miss out on potential rewards.

### Recommended Mitigation Steps

Add an input bool `\_trigger` then add the following code to the start of the function:

if (\_trigger) {
rebase();
}

This allows users to optionally call rebase if they are concerned about losing pending rebase rewards.
