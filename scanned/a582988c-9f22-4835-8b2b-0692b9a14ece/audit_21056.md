## Summary

This function is used to purchase a new Senior Bond and performs a number of calculations and checks before it either reverts or mints a new Senior Bond NFT to the user. ## Details
When purchasing a new Senior Bond, users call the [`buyBond` function](https://github.com/BarnBridge/BarnBridge-SmartYieldBonds/blob/943df3a8fcd8dd128af3beb0c85a0480c0e95ead/contracts/SmartYield.sol#L179), which performs a number of calculations and checks before it either reverts or mints a new Senior Bond NFT to the user. One of the checks performed is that the new Senior Bond’s [`gain` is less than the protocol’s loanable amount](https://github.com/BarnBridge/BarnBridge-SmartYieldBonds/blob/943df3a8fcd8dd128af3beb0c85a0480c0e95ead/contracts/SmartYield.sol#L217).


However, the function that calculates the protocol’s loanable amount – [`underlyingLoanable`](https://github.com/BarnBridge/BarnBridge-SmartYieldBonds/blob/943df3a8fcd8dd128af3beb0c85a0480c0e95ead/contracts/SmartYield.sol#L400) – is incorrect and can underflow. This is due to the fact that the calculation of the Junior Token price via the [`price` function](https://github.com/BarnBridge/BarnBridge-SmartYieldBonds/blob/943df3a8fcd8dd128af3beb0c85a0480c0e95ead/contracts/SmartYield.sol#L378) uses [`abondPaid`](https://github.com/BarnBridge/BarnBridge-SmartYieldBonds/blob/943df3a8fcd8dd128af3beb0c85a0480c0e95ead/contracts/SmartYield.sol#L415), which is the total `abond.gain` accrued so far, whereas `underlyingLoanable` subtracts the *entire* `abond.gain`. This underflow effectively leads to *any amount* of Senior Bond `gain` being approved, as `underlyingLoanable` outputs values around 1e77.


To illustrate this issue empirically, we have provided a test case in [this secret gist](https://gist.github.com/hensha256/e0799ae33fcab8270714a23f1b0b5a4f). This test can be run after pasting it into [line 79 of `jbond.test.ts`](https://github.com/BarnBridge/BarnBridge-SmartYieldBonds/blob/943df3a8fcd8dd128af3beb0c85a0480c0e95ead/test/jbond.test.ts#L79).


Consider updating the calculation within [`underlyingLoanable`](https://github.com/BarnBridge/BarnBridge-SmartYieldBonds/blob/943df3a8fcd8dd128af3beb0c85a0480c0e95ead/contracts/SmartYield.sol#L400), so that the return value aligns with protocol intentions. Additionally, consider this an excellent example of how utilizing [SafeMath](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/release-v3.4/contracts/math/SafeMath.sol) could prevent unintentional overflows and underflows that can cause unexpected or undesirable protocol behavior.


**Update**: *Fixed in [PR#42](https://github.com/BarnBridge/BarnBridge-SmartYieldBonds/pull/42). The referenced PR is based on commits that include various other changes to the codebase which have not been reviewed by OpenZeppelin.*
