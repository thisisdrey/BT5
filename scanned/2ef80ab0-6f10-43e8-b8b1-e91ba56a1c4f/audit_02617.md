# [M-02] Block gas utilization is slightly lower than the actual gas utilization due to the gas snapshot being taken too early, resulting in an inaccurate base fee calculation

## Summary
The xfeemarket implementation has a bug that causes the block utilization for the current height to be inaccurate, leading to a lower base fee than it should be. This is because the gas snapshot that is provided is not completely accurate, as it does not account for any gas consumed by subsequent actions. This can result in underestimated base fees and can have a moderate impact on the chain's performance. A proof of concept has been provided and it is recommended to take a snapshot right before calling the state update function to mitigate this issue. The severity of this bug has been raised to Medium.

## Details
### Finding description and impact

`xfeemarket` implements a custom `PostHandler()` for skip-mev’s feemarket. At almost the very end of the execution, the feemarket’s state is updated with the consumed gas amount so that the current block’s gas utilization and, thus, the base fee can be calculated.

However, the `gas` snapshot that is provided is not 100% accurate. It is taken in [line 81](https://github.com/code-423n4/2024-11-mantra/blob/37dedf312c3ea161792b8e1278978db23bb5a8b0/x/xfeemarket/post/fee.go# L81). But any gas consumed by the subsequent actions, e.g., `GetMinGasPrice()` in [line 101](https://github.com/code-423n4/2024-11-mantra/blob/37dedf312c3ea161792b8e1278978db23bb5a8b0/x/xfeemarket/post/fee.go# L101), or `ante.CheckTxFee()` in [line 112](https://github.com/code-423n4/2024-11-mantra/blob/37dedf312c3ea161792b8e1278978db23bb5a8b0/x/xfeemarket/post/fee.go# L112), are not accounted for in the `gas` snapshot.

As a result, the block utilization for the current height is inaccurate, leading to a slightly lower base fee than it should be.

### Proof of Concept

[`x/xfeemarket/post/fee.go# L129`](https://github.com/code-423n4/2024-11-mantra/blob/37dedf312c3ea161792b8e1278978db23bb5a8b0/x/xfeemarket/post/fee.go# L129)

In line 129, the skip-mev’s feemarket state is updated with the currently used gas so that the block utilization for the current height is calculated correctly.
```

129: err = state.Update(gas, params)
```

<https://github.com/skip-mev/feemarket/blob/f1f216e15aa6bcc05075a7079496764820a0f545/x/feemarket/types/state.go# L26-L36>
```

// Update updates the block utilization for the current height with the given
// transaction utilization i.e. gas limit.
func (s *State) Update(gas uint64, params Params) error {
	update := s.Window[s.Index] + gas
	if update > params.MaxBlockUtilization {
		return fmt.Errorf("block utilization of %d cannot exceed max block utilization of %d", update, params.MaxBlockUtilization)
	}

	s.Window[s.Index] = update
	return nil
}
```

However, the `gas` snapshot is taken in [line 81](https://github.com/code-423n4/2024-11-mantra/blob/37dedf312c3ea161792b8e1278978db23bb5a8b0/x/xfeemarket/post/fee.go# L81), which does not include any gas consumed in [lines 101-127](https://github.com/code-423n4/2024-11-mantra/blob/37dedf312c3ea161792b8e1278978db23bb5a8b0/x/xfeemarket/post/fee.go# L101-L127) up until the state update.
```

81: 	gas := ctx.GasMeter().GasConsumed() // use context gas consumed
```

### Recommended mitigation steps

Consider taking the snapshot right before calling `state.Update()`.

> To put this into context: [A simple bank send tx](https://explorer.mantrachain.io/MANTRA-1/tx/FBA240100B1D4115C2D2D53A23AC0036787EE241E30C9AA10441B3400380E668), consumed 151,517 gas, that’s `31,368 / 151,517 = ~21%` of the gas of such a tx.
>
> When Mantra chain is heavily utilized, this gas inaccuracy is not negligible and results in underestimated base fees, failing to properly increase gas fees to counteract high utilization (and e.g. spam). Therefore, I kindly ask the judge to reconsider the severity and consider Medium appropriate.
