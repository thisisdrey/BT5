# Incorrect Token Refunded

## Summary
The report states that there was a bug in the `FinanceToken.sol` file where the function `purchaseNMTWithToken()` was not returning the correct amount of `_paymentToken` to the caller. Instead, it was always returning the `nmtToken`. The client fixed the issue by removing a `transfer()` call in commit `7dc6cc7d1f09d688a6af848a655ed4345c86bd02`. The recommendation is to remove the `transfer()` call on line 158.

## Details
**Update**
The client fixed the issue in commit `7dc6cc7d1f09d688a6af848a655ed4345c86bd02` by removing the `transfer()` call.

**File(s) affected:**`FinanceToken.sol`

**Description:** The function `FinanceToken.purchaseNMTWithToken()` intends to return any excess `_paymentToken` to the caller, but `nmtToken` is always returned instead. There is no need to refund any payment as only the `useAmount` is transferred from the caller, not the full `_paymentAmount`.

**Recommendation:** Remove the `transfer()` call on #L158.
