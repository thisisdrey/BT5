# M-6: Pending Owner cannot accept ownership because of wrong implementation of `transferOwnership` and `acceptOwnership` functions

## Summary
The bug report is about a problem found by some users when trying to transfer ownership of a contract. The issue is that the pending owner is unable to accept ownership even after the required delay has passed. This is due to a mistake in the code, specifically in the functions responsible for transferring and accepting ownership. This bug can prevent the ownership handover process from completing as intended, which can cause problems in the contract's functionality. The root cause of the issue is that the wrong function is called in the acceptOwnership function, which leads to an error when the pending owner tries to accept ownership. The report also includes information about the internal and external pre-conditions, the attack path, and the impact of the bug. To fix this issue, the report suggests calling the correct function in the transferOwnership function.

### Summary

When `admin`(`owner`) try to transfer ownership, `pending owner` is unable to accept ownership even after the required delay has elapsed because of wrong implementation of `transferOwnership` and `acceptOwnership` functions in `TimedAdminFacet` which is the base contract of `AdminFacet`.

This prevents the ownership handover process from completing as intended, potentially leaving the contract without an effective admin and blocking any actions that require ownership privileges.

### Root Cause

It calls `AdminLib.reassignOwner` function in [`acceptOwnership` function](https://github.com/itos-finance/Commons/blob/621c2337f7acbf2a537500cddb82e3216adf5aa6/src/Util/TimedAdmin.sol#L29-L35) called by pending owner instead of calling it in [`transferOwnership` function](https://github.com/itos-finance/Commons/blob/621c2337f7acbf2a537500cddb82e3216adf5aa6/src/Util/TimedAdmin.sol#L23-L26) called by current owner.

```solidity
/* Owner changes */

function transferOwnership(address _newOwner) external virtual override {
    AdminLib.validateOwner();
    Timed.memoryPrecommit(OWNER_USE_ID, abi.encode(_newOwner));
}

/// The pending owner can accept their ownership rights.
function acceptOwnership() external virtual override {
    bytes memory entry = Timed.fetchPrecommit(OWNER_USE_ID, getDelay(true));
    address _newOwner = abi.decode(entry, (address));
    AdminLib.reassignOwner(_newOwner);
    AdminLib.acceptOwnership();
    emit IERC173.OwnershipTransferred(AdminLib.getOwner(), msg.sender);
}
```

In [`reassignOwner` function](https://github.com/itos-finance/Commons/blob/621c2337f7acbf2a537500cddb82e3216adf5aa6/src/Util/Admin.sol#L112-L115), it validates owner to check if `msg.sender` is current owner.

```solidity
function reassignOwner(address newOwner) internal {
    validateOwner(); // This is the only validate we do internal to AdminLib because it's so important.
    adminStore().pendingOwner = newOwner;
}

function validateOwner() internal view {
    if (msg.sender != adminStore().owner) {
        revert NotOwner();
    }
}
```

So it will be reverted with `NotOwner` error as the caller of `acceptOwnership`(`msg.sender`) is `pending owner`, not the `owner`.

### Internal Pre-conditions

1. Admin try to transfer ownership to a new owner by calling `transferOwnership`.
2. After delay(3 days according to codebase), the new owner try to accept ownership by calling `acceptOwnership`.

### External Pre-conditions

None

### Attack Path

1. Admin try to transfer ownership to a new owner by calling `transferOwnership`.
2. After delay(3 days according to codebase), the new owner try to accept ownership by calling `acceptOwnership`.

### Impact

The pending admin (owner) is unable to accept ownership even after the required delay has elapsed since the current admin’s call to `transferOwnership`.

This prevents the ownership handover process from completing as intended.

### PoC

Add below test function to `Ammplify\test\facets\Admin.t.sol` and run `forge test --mt testPendingOwnerCannotAcceptOwnership -vvvv` in terminal.

```solidity
function testPendingOwnerCannotAcceptOwnership() public {
    address pendingOwner = makeAddr("pendingOwner");
        
    adminFacet.transferOwnership(pendingOwner);

    // Skip the time delay for testing (3 days as per AdminFacet.getDelay implementation)
    vm.warp(block.timestamp + 3 days);
        
    vm.startPrank(pendingOwner);

    vm.expectRevert(AdminLib.NotOwner.selector);
    adminFacet.acceptOwnership();

    vm.stopPrank();
}
```


### Mitigation

Call `AdminLib.reassignOwner` in `transferOwnership` function.

```solidity
function transferOwnership(address _newOwner) external virtual override {
    AdminLib.validateOwner();
    AdminLib.reassignOwner(_newOwner);  // @audit: call it here
    Timed.memoryPrecommit(OWNER_USE_ID, abi.encode(_newOwner));
}

function acceptOwnership() external virtual override {
    // @audit: You should leave this as is even though you try to remove below condition check because need to check delay is passed or not
    bytes memory entry = Timed.fetchPrecommit(OWNER_USE_ID, getDelay(true));
    address _newOwner = abi.decode(entry, (address));
    // @audit: Just check if _newOwer is msg.sender. You can skip this check because AdminLib.acceptOwnership checks it
    if (_newOwner != msg.sender) {
        revert AdminLib.ImproperOwnershipAcceptance();
    }
    AdminLib.acceptOwnership();
    emit IERC173.OwnershipTransferred(AdminLib.getOwner(), msg.sender);
}
```
