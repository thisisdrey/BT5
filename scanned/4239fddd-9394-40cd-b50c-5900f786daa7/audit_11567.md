# Accidentally sent ETH can be stuck in protocol resolver contract 

## Summary
The report is about a security issue in a protocol called SchemaResolverUpgradeable.sol. The issue is that the protocol does not intend to support ETH transfers to resolvers, but the `attest` and `revoke` functions have a payable specifier which allows for ETH transfers. This means that if someone accidentally sends ETH to the protocol, it could get stuck in the resolver contracts. The recommendation is to prevent ETH transfers if the protocol does not support them. The bug has been fixed in PR 44.

## Description
The protocol currently does not intend to support ETH transfers to resolvers. However, the `attest` and `revoke` functions of `SchemaResolverUpgradeable.sol` have the payable specifier, which allows ETH transfers to underlying resolvers via these functions.

## Impact
Accidentally sent ETH can be stuck in resolver contracts.

## Recommendation
Prevent ETH transfers if the protocol currently does not intend to support ETH transfers to resolvers.

## Coinbase
Fixed by PR 44.

## Cantina
Reviewed that PR 44 fixes the issue by adding a `notPayable` modifier to these functions, which reverts when `msg.value != 0`.
