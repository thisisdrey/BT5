# Silent Fails Ignored for Transfers of Funds via `transfer()` and `transferFrom()`

## Summary
The client has reported a bug in the code, which has been marked as "Fixed". The bug affects three files: `PheasantNetworkBridgeChild.sol`, `PolygonPheasantNetworkBridgeChild.sol`, and `BondManager.sol`. The bug occurs when using the `transfer()` and `transferFrom()` functions for ERC-20 tokens. These functions may fail if the token implementation deviates from the standard, such as not returning a value (e.g. USDT). This affects several functions, including `tokenTransfer()`, `tokenTransferFrom()`, `tokenReceive()`, `slash()`, `deposit()`, and `finalizeWithdrawalBond()`. As a result, transfers of funds may fail without any indication, causing issues in validating operations that involve receiving or transferring funds. To fix this bug, it is recommended to use the `SafeERC20` library from OpenZeppelin to ensure that unsuccessful transfers with any ERC20 contract will revert.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `ac05853c5e6832f718622a6c0c4b02ada20bb4d9`.

**File(s) affected:**`PheasantNetworkBridgeChild.sol`, `PolygonPheasantNetworkBridgeChild.sol`, `BondManager.sol`

**Description:** ERC-20 `transfer()` and `transferFrom()` can fail if the ERC-20 token implementation deviates from the standard in some ways, such as if the return value is omitted (e.g. USDT). The return values are not handled correctly in the following functions:

*   `PheasantNetworkBridgeChild.tokenTransfer()`
*   `PheasantNetworkBridgeChild.tokenTransferFrom()`
*   `PheasantNetworkBridgeChild.tokenReceive()`
*   `PolygonPheasantNetworkBridgeChild.tokenTransfer()`
*   `PolygonPheasantNetworkBridgeChild.tokenTransferFrom()`
*   `PolygonPheasantNetworkBridgeChild.tokenReceive()`
*   `BondManager.slash()`
*   `BondManager.deposit()`
*   `BondManager.finalizeWithdrawalBond()`

As a result, transfer of funds can silently fail in the following functions, ultimately leading to issues in validating operations where funds are received/transferred:

*   `PheasantNetworkBridgeChild.newTrade()`
*   `PheasantNetworkBridgeChild.cancelTrade()`
*   `PheasantNetworkBridgeChild.withdraw()`
*   `PheasantNetworkBridgeChild.cancelTradeByRelayer()`
*   `PheasantNetworkBridgeChild.withdrawAssetFromSlashedTrade()`
*   `PheasantNetworkBridgeChild.acceptUpwardTrade()`

**Recommendation:** Use OpenZeppelin's `SafeERC20` library to assure that non-successful transfers with any ERC20 contract revert.
