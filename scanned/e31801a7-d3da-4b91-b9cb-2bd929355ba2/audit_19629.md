# [M-07] Missing hasStarted modifier, can lead to user vesting before the owner begin the vesting

## Summary
This bug report concerns the claimConverted() function in the LinearVesting.sol smart contract. This function allows a user to vest their vader token for a certain amount of time. However, the hasStarted modifier is missing, which means that the function can be called by anyone, allowing them to claim the token even if the vesting has not been started by the owner. The proof of concept for this bug is available on GitHub. To mitigate this issue, it is recommended to add the hasStarted modifier.

## Details
_Submitted by rfa_

#### Impact

In the `claimConverted()` function, the user can vest their vader token for a certain amount of time, but `hasStarted` modifier is missing,
this can lead to `claimConverted()` function is callable by anyone, and the user can claim eventhough the vesting havent been started by the owner.
