# [M-31] execute in VoteProxy should be payable

## Summary
This bug report is about a vulnerability in the VoteProxy.sol contract, which is part of the 2022-02-concur repository on GitHub. The vulnerability is that the `execute` function will revert when `msg.value > 0`. This means that the function will not work as expected when a user attempts to transfer a value greater than 0. The bug can be traced to the code snippet provided in the report, which lacks the `payable` mutability specifier. The recommended mitigation step is to add the `payable` mutability specifier to the code. This should fix the bug and allow the `execute` function to work as expected.

## Details
_Submitted by wuwe1_

`execute` will revert when `msg.value > 0`

### Proof of Concept

Lacking `payable` mutability specifier.

[VoteProxy.sol#L28-L35](https://github.com/code-423n4/2022-02-concur/blob/main/contracts/VoteProxy.sol#L28-L35)<br>

```solidity
    function execute(
        address _to,
        uint256 _value,
        bytes calldata _data
    ) external onlyOwner returns (bool, bytes memory) {
        (bool success, bytes memory result) = _to.call{value: _value}(_data);
        return (success, result);
    }
```

### Recommended Mitigation Steps

Add `payable` mutability specifier.
