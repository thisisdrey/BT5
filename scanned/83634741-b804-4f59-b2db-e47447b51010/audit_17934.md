# PAIR-1 | Duplicate Dividends

## Summary
A bug was found in the `mint` function, where an address can continually `mint` and transfer/burn its tokens to enter the `users` list multiple times. This can result in multiple dividends being paid out in the `distributeDividends` function. To resolve this issue, the Bridges Team refactored the dividends to a dividendsPerShare implementation. Additionally, a check for addresses that are already in the `users` list has been added.

## Details
**Description**

In the `mint` function, the only precondition for adding an address to the `users` list is if the balance of the address is 0. Additionally, an address is not removed from the `users` list if it transfers it‚Äôs balance of the BridgesPair token.

**Recommendation**

**Resolution**

Bridges Team:

- Dividends have been refactored to a dividendsPerShare implementation.
