# REW-6 | Lost Rewards Upon Withdrawal

## Summary
This bug report is about an issue with claiming a deposit bonus. When a user deposits money, but then withdraws even a tiny amount before the cadence is reached, the lastDepositTime is reset to 0, and the user must deposit again in order to be able to claim a deposit bonus. The recommendation is to verify whether this is expected behavior. If not, either a weighted bonus should be provided or the withdraw function should delete the lastDepositTime only if _claimDepositBonus returns true or the user’s new position amount is less than the minimum.

## Details
**Description**

If you deposit but then withdraw even a tiny amount before the `cadence` is reached, the `lastDepositTime` is reset to 0 and you must deposit again in order to be able to claim a deposit bonus.

**Recommendation**

Verify whether this is expected behavior. If not, either provide a weighted bonus or in the withdraw function `delete` the `lastDepositTime` only if `_claimDepositBonus` returns true or the user’s new position amount is less than the minimum.
