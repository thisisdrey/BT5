# Fees can be sent to the zero address 

## Summary
Bug report: The RecipeOrderbook contract has a bug where if the protocolFee is set to a non-zero amount and an IP order is created without setting the protocolFeeRecipient, the fees will be sent to the zero address. This means that the fees will be lost and cannot be retrieved. The recommended solution is to assign a value to protocolFeeRecipient in the constructor, so that when protocolFee is set to a non-zero value, protocolFeeRecipient is also set in the same transaction. This bug has been fixed by assigning the protocolFeeRecipient to the owner as the default in the constructor, until it is manually replaced by the owner. The fix has been implemented by Cantina Managed.
