# Paused Token Contract Does Not Pause Allowance Changes

## Summary
The AukiToken contract has a bug where allowance changes can still occur while the contract is paused, even though no tokens can be transferred, minted, or burned. This goes against the expected behavior of pausing all operations. The Auki Labs team has suggested adding a modifier to prevent allowance changes while the contract is paused. This bug has been resolved in a recent pull request.

## Details
The `AukiToken` contract can be [paused and unpaused](https://github.com/aukilabs/auki-contracts/blob/841aa7df56619dcec2af75efcb5ec7faeeaa955c/src/AukiToken.sol#L50-L56) by an address that has been granted the `PAUSER_ROLE`. Through the use of the [`whenNotPaused` modifier on the `_beforeTokenTransfer` function](https://github.com/aukilabs/auki-contracts/blob/841aa7df56619dcec2af75efcb5ec7faeeaa955c/src/AukiToken.sol#L62-L64), when the `AukiToken` contract is paused, no tokens can be transferred, minted, or burned. However, this does not prevent approvals, permits, and/or any other allowance changes from taking place while the contract is paused.


According to the Auki Labs team, "*the expected behavior when [Auki] token is paused is to pause all operations.*". As such, consider adding the `whenNotPaused` modifier to all corresponding allowance change functions to prevent this from occurring while the contract is paused.


***Update:** Resolved in [pull request #48](https://github.com/aukilabs/auki-contracts/pull/48) at commit [5c27964](https://github.com/aukilabs/auki-contracts/pull/48/commits/5c279646014a97c2a122da890137f4f60136b3e4).*
