# Redefine the Private Key Regexes

## Summary
A bug has been reported in the `wallet/index.ts` file, where the regex validation for private keys does not support those prefixed with `0x`. This can cause issues with wallet validation. The client has marked it as "Fixed" and provided a solution to consolidate the ECDSA and Ed25519 private key patterns into a single regex. The recommendation is to redefine the regex to be agnostic of the `0x` prefix and delete any unused regex.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `49ec016c124930295e2b04bf204fe59320703be3`. The client provided the following explanation:

> Simplify regex validation by consolidating ECDSA and Ed25519 private key patterns

**File(s) affected:**`wallet/index.ts`

**Description:** In `wallet/index.ts` the `ED25519_OR_ECDSA_REGEX` does not support private keys that are prefixed with `0x`. Furthermore `ECDSA_REGEX` and `ED25519_OR_ECDSA_REGEX`, can be merged to be a single regex. Failing to support private keys with this prefix jeopardizes wallet validation.

**Recommendation:** Redefine a single regex that is agnostic of the `0x` prefix, deleting any unused regex.
