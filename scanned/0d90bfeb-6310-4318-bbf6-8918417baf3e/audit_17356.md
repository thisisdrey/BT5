# Iteration through the whole storage array.

## Summary
The bug report discusses an issue with the TokensFarm.sol function_removeParticipant() at line 332, which is called when a user withdraws from the system. This function checks if the user should be removed from the list of participants by iterating through their stakes. However, if the user has too many stakes, it can use more gas than allowed in one transaction. Additionally, fully withdrawn stakes are not being removed from the array, which also increases gas usage. The recommendation is to remove fully withdrawn stakes and consider adding a limit on the number of stakes per user or creating a mapping to track active staking. The bug has been resolved.

## Details
**Description**

TokensFarm.sol: function_remove Participant(), line 419.

Perpetual TokensFarm.sol: function_removeParticipant(), line 332.

When a user performs withdrawal,_removeParticipant() is called to verify whether the user should be removed from participants. An iteration through all user's stakes is performed to determine this. In case, user has too many stakes, iteration will consume more gas than allowed per one transaction. Also, fully withdrawn stakes are not removed from the array, which also increases gas spendings.

**Recommendation**

Remove stakes, which are fully withdrawn. Consider adding a limitation on the number of stakes, allowed per one user. Or add a mapping with the amount of active staking.

**Re-audit comment**

Resolved
