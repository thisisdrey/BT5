# Overlooking the Launch Fee in Coin Distribution

## Summary
The CoinDistributeToHolders instruction is not deducting the launch fee when distributing lamports to holders. This means that the amount of lamports distributed may not match the intended amount. The function calculates the distribution based on the coin's liquidity balance without considering the launch fee. This can result in distributing more lamports than are actually available. To fix this, the launch fee should be deducted before calculating the distribution amount. This issue has been resolved in version 58c4974.

## Details
## CoinDistributeToHolders Instruction Overview

The `CoinDistributeToHolders` instruction does not account for the launch fee when distributing lamports to holders. This omission may result in discrepancies in the amount of lamports actually distributed and the intended amount. 

## Current Mechanism

Currently, `CoinDistributeToHolders` calculates the total lamports to be distributed (`total_lamports_to_distribute_to_all`) utilizing `coin.liquidity_balance`. It determines the amount each holder should receive based on their share and transfers the calculated amounts to each holder’s account. After these transactions, it updates the coin account’s balance and status without considering the launch fees.

```rust
> _ src/ixns/coin_distribute_to_holders.rs rust
pub fn handler(
    ctx: Context<CoinDistributeToHoldersCtx>,
    _args: CoinDistributeToHoldersArgs
) -> Result<()> {
    let coin: &mut Account<Coin> = &mut ctx.accounts.coin;
    let config: &Account<Config> = &ctx.accounts.config;

    // Calculate distributions
    let total_lamports_to_distribute_to_all: u64 = coin.liquidity_balance;
    let actual_number_of_holders = ctx.remaining_accounts.len();
    
    if actual_number_of_holders > config.top_holder_shares_per_million.len() {
        return Err(ErrorCode::MismatchInDistributionAccountsAndDistributionShares.into())
    }
    [...]
}
```

The launch fee is typically deducted from the total amount of funds before distribution. If the launch fee is not deducted before calculating the distribution amounts, the function may end up distributing more lamports than are actually available after accounting for the launch fee. Thus, the expected balance of the coin account may not align with the actual balance after the distribution.

## Remediation

To remedy this issue, the launch fee should be deducted from the total available lamports before calculating the amount to distribute to holders.

## Patch

This issue has been resolved in commit `58c4974`.
