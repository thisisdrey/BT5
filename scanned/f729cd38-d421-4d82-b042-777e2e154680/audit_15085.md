# Explicate Overflow Boundaries

## Summary
The report discusses a bug in the Phoenix software related to calculating adjusted quote lots. This occurs in the matching engine and is written in the RUST programming language. The bug is caused by the maximum size of adjusted quote lots exceeding the representable limit for u64, which can result in the calculation aborting. The report suggests either increasing the internal representation to u128 or setting constraints on lot sizes as a potential solution. The Ellipsis team has acknowledged the issue and plans to carefully select parameters to mitigate this issue in the future.

## Details
## Phoenix Numerical Calculation in the Matching Engine

Throughout Phoenix, the largest numerical calculation occurs in the matching engine when calculating adjusted quote lots.

## Code Snippet

```rust
inflight_order.adjusted_quote_lot_budget =
    inflight_order.adjusted_quote_lot_budget.saturating_sub(
        self.tick_size_in_quote_lots_per_base_unit
        * order_id.price_in_ticks
        * num_base_lots_quoted,
    );
```

**Note:** Adjusted quote lots are declared as a `basic_u64_struct` with an internal maximum `u64` representation.

## Code Definition

```rust
macro_rules! basic_u64_struct {
    ($type_name:ident) => {
        #[derive(Debug, Clone, Copy, PartialOrd, Ord, Zeroable, Pod)]
        #[repr(transparent)]
        pub struct $type_name {
            inner: u64,
        }
        basic_u64!($type_name);
    };
}
```

## Variables

More concretely, let:
1. **q** be the number of quote atoms transacted
2. **b** be the number of decimals in the base token
3. **lotsq** be quote atoms per lot
4. **lotsb** be base atoms per lot

## Abort Condition

This calculation will abort if:

\[ q \cdot lotsq \cdot lotsb \cdot 10^b \geq 2^{64} \]

## Ellipsis Labs Audit

## Vulnerabilities

### Remediation

Because adjusted quote lots are multiplied by an additional factor of `base_lots_per_base_unit`, the maximum size can exceed the representable limit for `u64`. 

Consider either increasing the internal representation for adjusted quote lots to `u128` or explicating constraints on lot sizes.

### Patch

The Ellipsis team acknowledges the issue and agrees to select parameters carefully with these constraints in mind. In particular, they note that upon large price fluctuations, new markets will likely be created, mitigating this issue for most practical use cases.
