# [M-14] Halting the protocol should be onlyGovernance and notonlyStrategist

## Summary
This bug report is about a vulnerability in a protocol that could allow a malicious strategist to halt the entire protocol and force a permanent shutdown. The vulnerability is caused by the access control on setHalted() being onlyStrategist, meaning that anyone who is a strategist can halt the protocol. The proof of concept for this vulnerability was provided in the form of two links to two sections of code in a GitHub repository. The tools used to identify this vulnerability were manual analysis. The recommended mitigation step is to change the access control to onlyGovernance from onlyStrategist for setHalted().

## Vulnerability details

## Impact

A malicious strategist can halt the entire protocol and force a permanent shutdown once they observe that governance is trying to set a new strategist and they do not agree with that decision. They may use the 7 day window to halt the protocol. The access control on setHalted() should be onlyGovernance.
