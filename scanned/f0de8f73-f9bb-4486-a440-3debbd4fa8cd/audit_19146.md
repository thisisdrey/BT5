# Incorrect taking out of `tail` account

## Summary
The bug report states that there is an issue with the code in the "AccountFactory.sol" file, specifically in line 260. The problem occurs when the "creditAccount" is equal to the "tail" and as a result, the "tail" is not properly updated. This causes the list of accounts to break. The recommendation is to update the "tail" if it has been taken out.

## Details
##### Description
If `creditAccount == tail` then `tail` is not updated properly and this will break the list of accounts:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/core/AccountFactory.sol#L260
##### Recommendation
We recommend updating `tail` if it was taken out.
