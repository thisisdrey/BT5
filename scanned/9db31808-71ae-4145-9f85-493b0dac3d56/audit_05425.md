# [M-03] Using address rather than bytes32 for destinationAddress in wstUsdcBridge#bridgeWstUsdc causes incompatibility with chains that utilize 32 byte addresses

## Summary
The bug report is about an issue with the WstUsdcBridge contract in the stakeup-contracts repository. The function bridgeWstUsdc is using the type "address" for the destinationAddress parameter, which is not compatible with chains that use a 32 byte address instead of a 20 byte address, like evm chains. The report recommends changing the type to "bytes32" to fix the compatibility issue. This has been addressed in a pull request and the issue has been resolved.
