# M-3: ADL borrow deleverage triggers on global debt instead of per-group debt, force-liquidating healthy positions

## Summary
The bug report is about a mistake in the code that could cause borrowers to lose their collateral when they shouldn't. The mistake is in a function called `handle_debt_auto_deleverage` and it causes the system to use the wrong amount of debt when deciding whether to liquidate a borrower's position. This means that borrowers could have their positions force-liquidated even though they are not at risk. The bug was found by a group of people and the cause has been identified. The bug can be reproduced using a specific test and the suggested solution is to replace one line of code with another.

### Summary

The wrong debt metric in `handle_debt_auto_deleverage` will cause loss of collateral for borrowers in unrelated emode groups as ADL liquidators will force-liquidate their positions based on the global reserve debt instead of the per-group borrow amount.

### Root Cause

In [`market.move:580`](https://github.com/sherlock-audit/2026-03-currentsui-contest-march-2026/blob/main/sui-move-contract/contracts/protocol/sources/internal/market/market.move#L580), the ADL borrow trigger reads `reserve.debt()` which is the total debt for a coin type across all emode groups. It should read `emode_group.borrow_amount(debt_type)` which is the debt for that specific group. The stop condition at [`market.move:686`](https://github.com/sherlock-audit/2026-03-currentsui-contest-march-2026/blob/main/sui-move-contract/contracts/protocol/sources/internal/market/market.move#L686) already uses the correct per-group value, confirming this is a mistake.

```move
// market.move:580 - BUG: checks global debt across all groups
let total_debt = (*self.reserves.load_by_type(debt_type).debt()).floor();
let debt_params = timelock.inner();
debt_params.ensure_limit_breached(total_debt);
```

```move
// market.move:686 - CORRECT: checks per-group borrow
let total_borrow = emode_group.borrow_amount(debt_type).floor();
let adl = dynamic_field::borrow_mut<ADLRegistryKey, AutoDeleverageRegistry>(market_id, ADLRegistryKey {});
adl.try_stop_borrow_deleverage<DebtType>(type_name::with_defining_ids<MarketType>(), emode_group_id, total_borrow);
```

### Internal Pre-conditions

1. Admin configures via `enable_debt_adl` ADL on any emode group with a `target_amount` for a given coin type.
2. At least two emode groups need to have borrows in that same coin type.

### External Pre-conditions

None

### Attack Path

1. Admin enables debt ADL for group 0 with a USDC target of 500K. Group 0 has 100K USDC borrowed. Group 1 has 900K USDC borrowed. Global reserve debt is 1M USDC.
2. An ADL liquidator calls `liquidate_adl_borrow` targeting a group 0 position.
3. The code checks `reserve.debt()` = 1M > 500K target and allows the liquidation to proceed.
4. The position gets force-liquidated. The borrower's collateral is seized at a discount even though group 0's actual borrow (100K) is well below the 500K threshold.
5. The stop check at L686 correctly reads group 0's borrow and disables ADL. But the next `liquidate_adl_borrow` call re-triggers it because the start check still sees global debt > 500K.
6. This repeats - group 0 positions keep getting picked off one by one until global debt (driven mostly by group 1) drops below 500K.

### Impact

Borrowers in the targeted emode group lose collateral through force-liquidation when their group was never at risk.

The ADL mechanism includes a time-degrading LTV threshold that starts at some configured value and drops toward zero over hours. Once it reaches zero, every single position in the group becomes liquidatable regardless of health. The liquidator extracts the ADL liquidation incentive on each liquidation. The loss scales with the total collateral held by group members.

### PoC

Place in `protocol/tests/integration/test_cases/poc_adl_global_debt.move` and run with:
```sh
sui move test -i 50000000000 poc_adl_triggers_on_wrong_emode_group
```

The test sets up two emode groups:
- Group 0 borrows 100 USDC (below the 500 ADL target)
- Group 1 borrows 900 USDC. Global debt is 1000 USDC

The ADL liquidation of a group 0 position succeeds, proving the bug. It should have reverted because group 0's borrow is below the threshold.

```move
/// H-01 PoC: ADL borrow trigger uses global reserve.debt instead of per-emode borrow.
///
/// Group 0 borrows:  100 USDC  (below 500 target -> ADL should NOT trigger)
/// Group 1 borrows:  900 USDC
/// Global debt:     1000 USDC  (above 500 target -> ADL DOES trigger)
///
/// This test PASSES, proving the bug: a healthy group-0 position gets
/// force-liquidated because handle_debt_auto_deleverage (market.move:L580)
/// checks reserve.debt() instead of emode_group.borrow_amount().
#[test_only]
module protocol::poc_adl_global_debt {

  use sui::test_scenario;
  use sui::clock;

  use protocol::market_t::MainMarket;
  use protocol::oracle_t;
  use test_coin::eth::ETH;
  use test_coin::usdc::USDC;
  use protocol::market_t::default_eth_decimal_places;
  use protocol::market_t::default_stable_decimal_places;

  const ADMIN: address = @0xAD;
  const LIM: u64 = 9223372036854775808; // 2^63

  #[test]
  public fun poc_adl_triggers_on_wrong_emode_group() {
    let borrower_0 = @0xB0;
    let borrower_1 = @0xB1;
    let liquidator = @0xCC;

    let mut scenario = test_scenario::begin(ADMIN);
    let mut clock = clock::create_for_testing(scenario.ctx());

    // 1. Default market (group 0 with ETH, USDC, ...)
    let (admin_cap, mut app, mut market, coin_registry) =
      protocol::app_t::default_app_init<MainMarket>(&mut scenario, &mut clock, ADMIN);
    let mut x_oracle = oracle_t::init_t(&mut scenario);

    // 2. Create group 1, onboard ETH + USDC to it
    protocol::emode_admin::onboard_new_emode_group<MainMarket>(
      &admin_cap, &app, &mut market, 0, &clock, scenario.ctx(),
    );

    let dl1 = protocol::emode_admin::create_limiter(&admin_cap, &app, LIM, 10000000, 10000000);
    let bl1 = protocol::emode_admin::create_limiter(&admin_cap, &app, LIM, 10000000, 10000000);
    let p1 = protocol::emode_admin::create_emode_params_test(
      &admin_cap, 7000, 7000, 500, 1_000_000_000_000, 10000, 1, dl1, bl1,
    );
    protocol::emode_admin::onboard_asset_to_emode_group<MainMarket, ETH>(
      &admin_cap, &app, &mut market, 1, p1, scenario.ctx(),
    );

    let dl2 = protocol::emode_admin::create_limiter(&admin_cap, &app, LIM, 10000000, 10000000);
    let bl2 = protocol::emode_admin::create_limiter(&admin_cap, &app, LIM, 10000000, 10000000);
    let p2 = protocol::emode_admin::create_emode_params_test(
      &admin_cap, 7000, 7000, 500, 1_000_000_000_000, 10000, 1, dl2, bl2,
    );
    protocol::emode_admin::onboard_asset_to_emode_group<MainMarket, USDC>(
      &admin_cap, &app, &mut market, 1, p2, scenario.ctx(),
    );

    // 3. Set prices
    clock.set_for_testing(200 * 1000);
    x_oracle.update_price<ETH>(&clock, oracle_t::calc_scaled_price(1000, 0));
    x_oracle.update_price<USDC>(&clock, oracle_t::calc_scaled_price(1, 0));

    // 4. Borrower 0 (group 0): deposit 1 ETH ($1000), borrow 100 USDC
    scenario.next_tx(borrower_0);
    let ob0 = protocol::open_obligation_t::open_obligation_t<MainMarket>(
      &mut scenario, &app, &mut market,
    );
    let eth0 = sui::coin::mint_for_testing<ETH>(
      1 * 10u64.pow(default_eth_decimal_places()), scenario.ctx(),
    );
    protocol::deposit::deposit<MainMarket, ETH>(
      &app, &mut market, &ob0, eth0, &clock, scenario.ctx(),
    );
    let b0 = protocol::borrow::borrow<MainMarket, USDC>(
      &app, &ob0, &mut market, &coin_registry,
      100 * 10u64.pow(default_stable_decimal_places()),
      &x_oracle, &clock, scenario.ctx(),
    );
    std::unit_test::destroy(b0);

    // 5. Borrower 1 (group 1): deposit 10 ETH ($10k), borrow 900 USDC
    scenario.next_tx(borrower_1);
    let ob1 = protocol::enter_market::create_obligation_with_group<MainMarket>(
      &app, &mut market, 1, scenario.ctx(),
    );
    let eth1 = sui::coin::mint_for_testing<ETH>(
      10 * 10u64.pow(default_eth_decimal_places()), scenario.ctx(),
    );
    protocol::deposit::deposit<MainMarket, ETH>(
      &app, &mut market, &ob1, eth1, &clock, scenario.ctx(),
    );
    let b1 = protocol::borrow::borrow<MainMarket, USDC>(
      &app, &ob1, &mut market, &coin_registry,
      900 * 10u64.pow(default_stable_decimal_places()),
      &x_oracle, &clock, scenario.ctx(),
    );
    std::unit_test::destroy(b1);

    // State: group-0 borrow = 100 USDC, group-1 borrow = 900, global = 1000

    // 6. Admin: enable debt ADL for group 0, target = 500 USDC
    //    Group 0 borrows (100) < 500 -> should NOT trigger
    scenario.next_tx(ADMIN);
    let adl_params = protocol::adl_admin::new_adl_params<MainMarket>(
      &admin_cap, &app, &market,
      500 * 10u64.pow(default_stable_decimal_places()),
      1,     // liquidation_factor_base: 0.01% -> any LTV qualifies
      0,     // hourly drop
      100,   // incentive base: 1%
      0,     // incentive daily penalty
      10000, // close factor: 100%
    );
    protocol::adl_admin::enable_debt_adl<MainMarket, USDC>(
      &admin_cap, &app, &mut market,
      0, adl_params, 0, &clock, scenario.ctx(),
    );

    // 7. Liquidator: ADL-liquidate borrower_0
    //    BUG: succeeds because reserve.debt()=1000 > 500
    //    FIX: should abort because emode_group.borrow_amount()=100 < 500
    scenario.next_tx(liquidator);
    let permit = protocol::whitelist_admin::mint_new_whitelist(
      &admin_cap, &mut app, scenario.ctx(),
    );
    protocol::whitelist_admin::update_permission(
      &admin_cap, &mut app, object::id(&permit),
      protocol::whitelist_admin::adl(), true,
    );
    let repay = sui::coin::mint_for_testing<USDC>(
      50 * 10u64.pow(default_stable_decimal_places()), scenario.ctx(),
    );
    protocol::liquidate::liquidate_adl_borrow<MainMarket, USDC, ETH>(
      &app, &permit, ob0.id(), &mut market,
      repay, &coin_registry, &x_oracle, &clock, scenario.ctx(),
    );
    // ^^^ This succeeds, proving the bug.

    // Cleanup
    clock::destroy_for_testing(clock);
    test_scenario::return_shared(x_oracle);
    test_scenario::return_shared(market);
    std::unit_test::destroy(admin_cap);
    std::unit_test::destroy(ob0);
    std::unit_test::destroy(ob1);
    std::unit_test::destroy(app);
    std::unit_test::destroy(permit);
    std::unit_test::destroy(coin_registry);
    test_scenario::end(scenario);
  }
}
```

### Mitigation

Replace `reserve.debt()` with `emode_group.borrow_amount(debt_type)` at `market.move:580`:

```move
let total_debt = emode_group.borrow_amount(debt_type).floor();
```

This makes the start check consistent with the stop check at L686 which already uses the per-group value.
