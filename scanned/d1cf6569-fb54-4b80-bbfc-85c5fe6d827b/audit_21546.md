# Potentially inconsistent input validation ✓ Fixed

## Summary
This bug report is related to the functions `Amp.transferWithData`, `Amp.authorizeOperatorByPartition`, and `Amp.revokeOperatorByPartition` in the Amp.sol file. The `msg.sender` cannot be authorized or revoked from being an operator for itself. The `transferWithData` function was removed as a resolution of another filed issue, while the other two functions have been documented properly. The bug report recommends adding additional input validation to these functions, similar to what is done in other functions. This could help prevent the `msg.sender` from being authorized or revoked from being an operator for itself.

## Details
#### Resolution



`transferWithData` was removed as a resolution of another filed issue, the rest are documented properly.



> 
> The `msg.sender` cannot be authorized or revoked from being an operator for itself. This should also be clear from the natspec comments now.
