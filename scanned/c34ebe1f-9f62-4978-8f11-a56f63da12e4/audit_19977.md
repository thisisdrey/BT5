# [M-09] Fee calculation is potentially incorrect

## Summary
This bug report is about a vulnerability found in the Basket.sol file of the 2021-09-defiProtocol repository. The impact of this vulnerability is that more fees are being charged than intended. The mitigation steps involve the replacement of `BASE - feePct` in the denominator with `BASE` at line 118 of the Basket.sol file. This will ensure that the correct fee is charged. To explain this in more detail, assume that the license fee is 10%, which is equivalent to 1e17, and that the time difference is half a year. When calculating feePct, the expected amount is 5e16 since that is 5%, and the actual amount of fee to be charged should be totalSupply * feePct (5) / BASE (100). However, on line 118, we are dividing by BASE - feePct, which is 95. This results in 5 / 95 = 0.052 instead of the intended 0.05. Replacing `BASE - feePct` in the denominator with `BASE` will ensure that the correct fee is charged.
