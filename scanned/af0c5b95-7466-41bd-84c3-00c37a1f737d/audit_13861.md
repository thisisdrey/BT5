# Calls to Set.previewRedeem() will revert if a market of the set is frozen 

## Summary
This bug report is about an error in the DelayLib.sol file, specifically on line 76. The error occurs when the function `Set.previewRedeem(<redemptionId>)` is called after a market in the set has been frozen with a FAIL. The reason for the error is an arithmetic over/underflow. The bug was tested and debugged using a test file and the parameters `now_`, `startTime_`, and `currentInactiveDuration_`. The overflow occurs in the line mentioned above. The recommendation to fix this bug is to update the set's inactive period data when a market transitions to or from a frozen state. The bug has been addressed in PR 133 for the Cozy platform and has been fixed for the Cantina platform.

## Details
## Context
**File:** DelayLib.sol#L76  
**Description:** Any call to `Set.previewRedeem(<redemptionId>)` will revert after a market of the set is frozen with a **FAIL. Reason:** Arithmetic over/underflow error.

## Debugged Parameters
- `now_`: 1382401
- `startTime_`: 1209601
- `currentInactiveDuration_`: 1382401

**Overflow occurs in this line:** `DelayLib.sol#L76`:
```solidity
return now_ - startTime_ - inactiveDurationAfterStartTime_;
```

## Recommendation
In order to enforce that redemption delay does not accrue when the set is frozen, update the set's inactive period data in the logic for when a market transitions to frozen, or out of frozen.

- **Cozy:** Addressed in PR 133.
- **Cantina:** Fixed.
