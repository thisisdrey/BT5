# Undefined Order For Token Distribution

## Summary
The bug report discusses an issue with the function responsible for distributing tokens to bidders after an auction has ended. The problem arises when there is a limit on the number of tokens that can be minted and the order in which token receivers are specified becomes important. Currently, this order is not enforced on-chain, which means that a user who placed a bid early may not receive their tokens if the limit is reached before their tokens are minted. The report suggests implementing the ordering of token minting on-chain or ensuring proper execution of the ordering off-chain. The issue has been fixed in the latest version by transferring the refund functionality to the owner and using a modifier to ensure off-chain ordering.

## Details
## Token Distribution in Mythos.sol

The `sendTokensAndRefund` function in `Mythos.sol` is responsible for distributing tokens to the bidders based on the price fixed after an auction has ended. This function is called by a user with the `REFUND_ROLE`. As there is a limit on the number of tokens that can be minted, the order in which token receivers are specified becomes significant. It is important that the tokens are distributed in the same order as the bidding process, taking into account each user’s last bidding time for sorting.

However, this order is not enforced on-chain, which means that a user who placed a bid early may not receive their tokens if the limit is reached before their tokens are minted.

## Proof of Concept

Consider the following scenario:

1. An auction is created with predefined parameters.
2. The auction starts, and a user (`user1`) bids a certain amount.
3. Another user (`user2`) bids an amount after `user1`.
4. Once the auction ends, if `user2`’s tokens are distributed first, then it is possible that `user1` is denied minting new tokens due to the token mint limit.

## Remediation

Implement the ordering of the token minting on-chain or by ensuring that the ordering of token receivers is properly executed off-chain.

## Patch

Fixed in `1fc5784` by transferring the refund functionality to the owner using the `onlyOwner` modifier. The owner should ensure off-chain that the bids are in proper order.
