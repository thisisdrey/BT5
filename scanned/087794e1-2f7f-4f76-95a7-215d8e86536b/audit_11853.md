# A guardian cannot cancel a malicious proposal in AdminVoting

## Summary
A bug report has been submitted that states the issue of a user being able to create a new proposal with a payload of actions, but a guardian is unable to use the `cancelProposal()` function on the proposal if the first action is a call to `IPrismaCore.setGuardian()`. This means that if the proposal contains a malicious action, the guardian is unable to cancel it. The recommendation is to allow the cancellation of a proposal if its `payload.length > 1`. This would allow guardians to cancel proposals if they contain malicious actions and prevent the malicious action from being executed.

## Details
##### Description
https://github.com/prisma-fi/prisma-contracts/blob/c0122d27677cd4e1aaee7f1e21f807ccadf46ac8/contracts/dao/AdminVoting.sol#L186

A user can create a new proposal via `createNewProposal()` with a payload of actions. A guardian, however, is prohibited from using `cancelProposal()` on a proposal, if the first action is a call to `IPrismaCore.setGuardian()`. Therefore, if the proposal contains a malicious action, the guardian is unable to cancel it.

##### Recommendation

Our recommendation is to allow the cancellation of a proposal if its `payload.length > 1`.
