# External Call Reverts if Period Has Not Elapsed

## Summary
A bug was discovered in the function notifyRewardAmount() which would cause the Synthetix.mint() function to fail if block.timestamp >= periodFinish. This would halt the mint process as Synthetix.mint() cannot be successfully called until enough time has elapsed for the period to finish. The development team has resolved this issue by changing the code in the commit 4feb1cb. This bug was identified as UNI-04 Gap Between Periods Can Lead to Erroneous Rewards in the Solidity Security Review Detailed Findings.

## Details
## Description
The function `notifyRewardAmount()` will revert if `block.timestamp >= periodFinish`. However, this function is called indirectly via the `Synthetix.mint()` function. A revert here would cause the external call to fail and thereby halt the mint process. `Synthetix.mint()` cannot be successfully called until enough time has elapsed for the period to finish.

## Recommendations
Consider handling the case where the reward period has not elapsed without reverting the call.

## Resolution
The development team has resolved this issue in commit `4feb1cb`.
