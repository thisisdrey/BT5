# Blocked LP tokens on contract

## Summary
This bug report deals with a problem that occurs when a line of code in the UniswapMarketMaker.sol contract is changed. Specifically, when the `incoming` token is changed, the contract sends all remaining `incoming` tokens to `_recipient`, but it does not check for any remaining liquidity on the contract side. This means that these tokens cannot be recovered after the change has been made. 

The recommendation is to remove all liquidity before changing the `incoming` token, in order to avoid losing any tokens. This will ensure that all tokens are properly accounted for and can be recovered.

## Details
##### Description
At the line https://github.com/bondappetit/bondappetit-protocol/blob/355180f0aca0b29d60d808f761052956b7a3a159/contracts/profit/UniswapMarketMaker.sol#L85 contract changes `incoming` token to another one, while transferring contract sends all remaining `incoming` tokens to `_recipient`, but contract never check remaining incoming <> support LP tokens on contract side. That tokens cannot be rescued anymore after changing incoming.

##### Recommendation
We recommend to remove all liquidity before changing `incoming` token
