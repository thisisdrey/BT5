# Lack of High-Level Documentation

## Summary
The client has provided additional documentation for the project, including a summary of the project. After reviewing, the team has determined that the issue has been resolved. The client has marked the issue as "Acknowledged" and stated that they are currently working on producing more documentation. The issue at hand is that while the project has in-line documentation, there is no high-level documentation to explain how to use the project, the functions of the system methods, and the correct ways to interact with it. This lack of documentation can cause confusion for users and pose security risks. The team recommends that the development team create documentation for both technical and non-technical users to better understand the project.
