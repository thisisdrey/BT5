# Margin accounts can be registered with suspended agreements

## Summary
This bug report discusses a high difficulty bug in the registration logic for margin accounts. The issue is classified as undefined behavior and is caused by a lack of a check to ensure that the assigned agreement contract is not suspended. This is a problem because the agreement contract inherits a library that allows it to be paused and unpaused, but only the pause logic is exposed. This means that paused agreement contracts cannot be unpaused, which could result in critical operations being blocked on newly registered accounts. 

The bug is triggered during margin account registration when an agreement contract address is passed as a parameter to the registerMarginAccount function. The verifyThresholds function in the ThresholdVerifier contract validates the input before proceeding with registration, but it does not check if the agreement contract is suspended. This oversight could lead to critical operations being blocked on newly registered accounts if the assigned agreement contract is paused. 

The recommended solution is to include an agreement status check within the validation logic for margin account registration to ensure that the assigned agreement contract is not suspended before proceeding with the registration. In the long term, it is also suggested to expand the unit test suite to explore a wide variety of edge case scenarios and validate both happy and unhappy code execution paths within the system.

## Description
The registration logic for margin accounts lacks a check to ensure the assigned agreement contract is not suspended. The agreement contract inherits OpenZeppelin’s PausableUpgradeable library, which allows the contract to be paused and unpaused. However, only the pause logic is exposed via the `suspend` function, so paused agreement contracts cannot be unpaused.

```solidity
function suspend() external override onlyFactory {
    _pause();
}
```
*Figure 4.1: contracts/agreement/Agreement.sol#L184–L186*

During a margin account registration, an agreement contract address is passed as a parameter to the `registerMarginAccount` function in the Dispatcher contract. The `verifyThresholds` function in the ThresholdVerifier contract validates the input before proceeding with registration.

```solidity
function registerMarginAccount(
    address agreement,
    Asset[] calldata _collateral,
    Asset calldata _leverage
) external override returns (address account) {
    verifyThresholds(agreement, _collateral, _leverage);
    IAgreement(agreement).enforceCanBorrow(msg.sender);
    account = factory.borrowAccount();
    for (uint256 i; i < _collateral.length; i++) {
        _collateral[i].forward(account);
        ...
    }
}
```
*Figure 4.2: contracts/dispatcher/Dispatcher.sol#L39–L54*

However, the current logic does not check whether the agreement contract is suspended. This oversight could result in critical operations on newly registered accounts being blocked if the assigned agreement contract is paused.

```solidity
function verifyThresholds(
    address agreement,
    Asset[] calldata _collaterals,
    Asset calldata _leverage
) internal view {
    if (_leverage.amount == 0) revert AmountMustNotBeZero(_leverage.token);
    if (_collaterals.length == 0) revert ArrayMustNotBeEmpty();
    (IAgreement.Metadata memory metadata, ) = IAgreement(agreement).info();
    if (metadata.leverage != _leverage.token) revert WrongLeverage(agreement, _leverage.token);
    
    for (uint256 x; x < _collaterals.length; x++) {
        Asset calldata c = _collaterals[x];
        bool isValidCollateral = false;
        for (uint y = 0; y < metadata.collaterals.length; y++) {
            if (c.token == metadata.collaterals[y]) isValidCollateral = true;
        }
        if (!isValidCollateral) revert WrongCollateral(agreement, c.token);
        if (c.amount == 0) revert AmountMustNotBeZero(c.token);
    }
}
```
*Figure 4.3: contracts/dispatcher/base/ThresholdsVerifier.sol#L21–L42*

## Exploit Scenario
Alice registers a new margin account, unaware that the agreement contract used had been suspended. The account is created successfully, but critical operations like trading are blocked due to the contract’s paused state.

## Recommendations
**Short term:** Include an agreement status check within the validation logic for margin account registration to ensure that the assigned agreement contract is not suspended before proceeding with the registration.

**Long term:** Expand the unit test suite to explore a wide variety of edge case scenarios, validating both happy and unhappy code execution paths within the system.
