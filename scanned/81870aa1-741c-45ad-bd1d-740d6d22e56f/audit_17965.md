# [M-09] Reward can be vested even after endTime

## Summary
This bug report is related to the AuraVestedEscrow.sol#L96 contract. Currently, the reward vesting does not end once the endTime is reached. To demonstrate this, the fund function was observed and it was found that there is no check to disallow funding once endTime has been reached. 

A mitigation step was proposed to add a check requiring that the block timestamp must be less than or equal to the endTime, i.e. `require(block.timestamp<=endTime, "Reward vesting period over");`. This was acknowledged by 0xMaharishi (Aura Finance), however, they disagreed with the severity of the issue. LSDan (judge) commented that this was totally valid and would cause a loss of funds if not addressed. 

In the end, the issue was resolved with the implementation of the proposed mitigation step in the code4rena aurafinance/aura-contracts#84 pull request and the All code4rena fixes code-423n4/2022-05-aura#6 pull request.
