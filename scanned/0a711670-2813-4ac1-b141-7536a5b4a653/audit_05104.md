# Lack of Validation Checks

## Summary
The trove-manager's function called "batch_liquidate_troves" has some missing checks that are causing problems with the liquidation process. This function should only work on active troves and sorting is important for it to work properly. The issue is also causing duplicate entries, which can lead to incorrect results and unexpected behavior. To fix this, checks need to be implemented to make sure the troves are active, sorted correctly, and to identify any duplicate entries. This issue has been resolved in version 2eff674.

## Details
## Batch Liquidation of Troves in Trove Manager

The `batch_liquidate_troves` function in the trove-manager is missing checks for active troves, sorted order, and duplicate entries, which affects the proper functioning of the liquidation process. 

## Issues Identified

1. **Active Troves**: Liquidation should only occur on active troves.
2. **Sorting**: Proper sorting is crucial for efficient liquidation.
3. **Duplicate Entries**: Duplicate entries result in redundant operations, skewed liquidation results, and unexpected behavior, such as attempting to liquidate the same trove multiple times.
