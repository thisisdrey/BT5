# Should return zero. But instead the transaction crashes

## Summary
A bug has been identified in the code of a project called CoverFlashLoan, which is used to calculate the number of tokens. In some cases, an arithmetic overflow occurs and the transaction is reverted. To fix the bug, two lines in the code should be modified. 

At line 163, the code should be modified to: 
>`if (amountDaiNeeded + flashFee < daiReceivedFromSwap) {`
>`            totalCost = 0;`
>`        } else {`
>`            totalCost =  amountDaiNeeded - daiReceivedFromSwap + flashFee;`
>`        }`

At line 186, the code should be modified to: 
>`if (daiReceivedFromRedeem < amountDaiNeeded + flashFee) {`
>`            totalReturn = 0;`
>`        } else {`
>`            totalReturn =  daiReceivedFromRedeem - amountDaiNeeded - flashFee;`
>`        }`

It is recommended to fix the code in order to avoid the arithmetic overflow and revert of transactions.
