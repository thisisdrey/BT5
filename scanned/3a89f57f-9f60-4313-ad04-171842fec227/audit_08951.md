# Lack of Reentrancy Protection in `redeem()`, `depositAndVote()`, and `withdraw()` Functions

## Summary
A recent update to the file `Pool.sol` has fixed a bug that left three functions vulnerable to re-entrancy attacks. These functions, `Pool.redeem()`, `Pool.depositAndVote()`, and `Pool.withdraw()`, did not have proper protection against re-entrancy, which can allow attackers to manipulate the contract's state and potentially drain funds or cause other unintended issues. It is important for these functions to have re-entrancy safeguards in place to ensure the contract's security and functionality.

## Details
**Update**
Fixed in `3bbf73b91fab903820af21c6f037d81c2026949a`.

**File(s) affected:**`Pool.sol`

**Description:** The functions `Pool.redeem()`, `Pool.depositAndVote()`, and `Pool.withdraw()` lack re-entrancy protection. Re-entrancy, a prevalent vulnerability in smart contracts, enables attackers to re-invoke a contract's functions before it finalizes its state, potentially leading to unintended behaviors like fund drainage, state manipulation, or logical execution errors.

**Recommendation:** It is crucial to implement re-entrancy safeguards for these functions to ensure the integrity and security of the contract's operations.
