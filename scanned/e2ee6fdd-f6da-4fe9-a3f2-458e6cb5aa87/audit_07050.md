# DeBridgeValidator::validateTxData wrong destination chain for Debridge/DLN order rescuer 

## Summary
Summary:

The bug report discusses an issue with the DeBridgeValidator and DeBridgeForwarderValidator contracts when bridging liquidity via DLN. If the bridged order is not taken or needs to be cancelled, a specific address is supposed to have the authority to cancel the order. However, this address is set incorrectly on the wrong chain, potentially causing funds to become unavailable. The recommendation is to change the chain ID to the correct one to fix the issue. The bug has been fixed in a recent commit. The risk level of this bug is considered low.

## Details
## Bridging Liquidity Issue

## Context
- **File Locations**: 
  - `DeBridgeForwarderValidator.sol#L83`
  - `DeBridgeValidator.sol#L58`

## Description
When liquidity is bridged via DLN, an order is created on the destination chain. If the order is never taken or needs to be canceled for other reasons, a superform controlled address (role `CORE_STATE_REGISTRY_RESCUER_ROLE`) is set to be able to cancel the order.

In `DeBridgeValidator::validateTxData`, it is enforced for the order `orderAuthorityAddressDst` to be the `CORE_STATE_REGISTRY_RESCUER_ROLE` but on the wrong chain (see `DeBridgeValidator.sol#L58`). 

Indeed, in the case of withdrawals, `dstChainId`, which is the chain on which the acknowledgment must be received, can be different from `liqDstChainId`, which is the chain on which the order is created. 

In that case, funds may end up unavailable since nobody has the authority to cancel the order on the destination chain.

## Recommendation
Change `dstChainId` to `liqDstChainId` (in `DeBridgeValidator` and `DeBridgeForwarderValidator`):

```solidity
if (
- superRegistry.getAddressByChainId(keccak256("CORE_STATE_REGISTRY_RESCUER_ROLE"), args_.dstChainId)
+ superRegistry.getAddressByChainId(keccak256("CORE_STATE_REGISTRY_RESCUER_ROLE"), args_.liqDstChainId)
!= _castToAddress(deBridgeQuote.orderAuthorityAddressDst)
) revert DeBridgeError.INVALID_DEBRIDGE_AUTHORITY();
```

Fixed in commit `a554840b`.
