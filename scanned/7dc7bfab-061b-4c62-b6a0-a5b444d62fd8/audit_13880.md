# Incorrect value is emitted during fulfillTradeOer for trade.askValue and for trade.oerValue when there are fee is deducted

## Summary
This bug report is about an issue in the TradingPost.sol file of the IOTA-Heroes-Contracts repository. During the fulfillTradeOer function, a fee is deducted from the askValue and oerValue. However, when the event is emitted at the end of the function, it still uses the old value and does not take into account the fee that was deducted. 

To resolve this issue, it is recommended to use the correct askValue and oerValue if there is a fee deducted.
