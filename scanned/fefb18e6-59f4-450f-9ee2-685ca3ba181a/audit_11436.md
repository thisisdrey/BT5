# Incorrect Usage of Uniswap V2 Oracle

## Summary
The bug report is about a discrepancy between the example code provided by Uniswap for their oracle and the implementation of the `UniV2TwapOracle` used by the `v2-core` project. The issue arises because Uniswap's code relies on unchecked arithmetic, which is only available in Solidity version `0.6.6`, while the `UniV2TwapOracle` uses Solidity version `0.8.12` which has default checks to prevent overflow. This may cause the code to unexpectedly revert and the suggestion is to add `unchecked` blocks to the parts of the code that are expected to overflow. The bug has been resolved in a recent pull request by wrapping the expected overflow parts in `unchecked` blocks.

## Details
Uniswap provides [example code](https://github.com/Uniswap/v2-periphery/blob/0335e8f7e1bd1e8d8329fd300aea2ef2f36dd19f/contracts/examples/ExampleOracleSimple.sol) for a Uniswap V2 oracle and `UniV2TwapOracle` [follows the example implementation](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/oracles/UniV2TwapOracle.sol#L128-L161) to retrieve oracle price data. However, Uniswap's implementation actually relies on the unchecked arithmetic in Solidity `0.6.6` in order to work correctly. In contrast, the Solidity version `0.8.12` used by [`UniV2TwapOracle`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/oracles/UniV2TwapOracle.sol) contains default checks to prevent overflow from occurring. Since the Uniswap implementation [expects overflows](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/oracles/UniV2TwapOracle.sol#L131-L139) as part of normal operation, the use of modern Solidity `>=0.8.0` may eventually cause the code to revert unexpectedly.


Consider adding `unchecked` to the parts of the `UniV2TwapOracle` code that are expected to overflow.


***Update:** Resolved in [pull request #204](https://github.com/radiant-capital/v2-core/pull/204) at commit [6e67d1c](https://github.com/radiant-capital/v2-core/commit/6e67d1c2f9232f29285fe11df2dd1e49972dca05). All parts that are expected to overflow have been wrapped in `unchecked` blocks.*
