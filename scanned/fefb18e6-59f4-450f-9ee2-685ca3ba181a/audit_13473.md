# Empty Listings Lead to Several Issues

## Summary
The bug report is about a problem in the code for creating listings on a marketplace. The problem is that there is no check to make sure the listing has at least one item for sale. This can lead to potential issues, such as someone tricking users into paying for empty listings or creating fake listings to make the marketplace seem more active. The recommendation is to add a check to make sure there is at least one item for sale before creating a listing. This has been addressed in a recent update to the code.

## Details
**Update**
Addressed in: `89f0c9820af7e6d9005ab1a031e788301f5d49dd`. The client added a check for `listing.tokenIds > 0` in `_createListing()`.

**File(s) affected:**`Listings.sol`, `Marketplace.sol`

**Description:** In `Listings._createListing()`, there is no validation done on `tokenIds` being a non-empty array. As a result of this it is possible to create a valid empty listing, which can result in the following issues ranging from High to Medium severity:

*   A malicious actor can intentionally flood the marketplace with a series of empty listings with arbitrary listing amounts and trick users into accidentally paying for these empty listings. (High Severity)
*   A malicious actor can spoof the marketplace with fraudulent listings, creating artificial marketplace activity (Medium Severity)

**Recommendation:**`_createListing()` should validate that `tokenIds` is a non-empty array.
