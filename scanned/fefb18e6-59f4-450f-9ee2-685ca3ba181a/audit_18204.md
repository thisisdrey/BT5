# Incorrect Ether transfer

## Summary
The bug report is about a problem in the ShyftStaking.sol code, specifically in the unstake(), notifyRewardAmount(), finishPrePurchasersMode(), and _getReward() functions. This issue occurred after the Istanbul and Berlin updates, which made it impossible to send Ethereum using the transfer() and send() methods. This is because these methods may fail during a fallback function call due to not enough forwarded gas. The recommended solution is to use the call() method instead for sending Ethereum.

## Details
**Description**

ShyftStaking.sol, unstake() (line 267), notifyRewardAmount() (line 316),
finishPrePurchasersMode() (line 424), _getReward() (line457)
After the Istanbul and Berlin updates, sending Eth with transfer() and send() methods is
forbidden since it may fail during fallback function call because of the not enough forwarded
gas.
Thus, the only correct for m of Eth sending is with the call() method, for example:
(bool success,) = addr.call{value: val}(””);
require(success, “Failed Eth transfer”);

**Recommendation**

Use the call() method for Eth sending.
