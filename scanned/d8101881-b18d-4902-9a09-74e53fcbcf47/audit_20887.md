# RocketDAONodeTrustedActions - member cannot be kicked if the vault does not hold enough RPL to cover the bond ✓ Fixed

## Summary
This bug report deals with a scenario in which a DAO member is proposed to be evicted from the DAO, but if the RocketVault does not hold enough RPL to pay back the DAO member bond, the actionKick will throw and the node is not evicted. The recommendation is to ensure that there is no way a node can influence a succeeded kick proposal to fail, either by burning the bond or allowing the node to redeem it in a separate step. The bug has been addressed by returning the bond if enough RPL is in the treasury, and if not, continuing without returning the bond, so the member kick action does not block and the member can be kicked regardless of the RPL balance.

## Details
#### Resolution



Addressed in branch `rp3.0-updates` ([rocket-pool/[email protected]`b424ca1`](https://github.com/rocket-pool/rocketpool/commit/b424ca1ae1d0c1c5f3fa3f7f6f36051f8e57bbc2)) by returning the bond if enough RPL is in the treasury and else continue without returning the bond. This way the member kick action does not block and the member can be kicked regardless of the RPL balance.


#### Description


If a DAO member behaves badly other DAO members may propose the node be evicted from the DAO. If for some reason, `RocketVault` does not hold enough `RPL` to pay back the DAO member bond `actionKick` will throw. The node is not evicted.


Now this is a somewhat exotic scenario as the vault should always hold the bond for the members in the system. However, if the node was kicked for stealing RPL (e.g. passing an upgrade proposal to perform an attack) it might be impossible to execute the eviction.


#### Recommendation


Ensure that there is no way a node can influence a succeeded `kick` proposal to fail. Consider burning the bond (by keeping it) as there is a reason for evicting the node or allow them to redeem it in a separate step.
