# DoSing the launch ﬁnalization process 

## Summary
The bug report is about a problem in a code called TruglyMemeception.sol. The issue is that a function called UniswapV3Pool.initialize() can be called before it is supposed to, which can cause issues with the finalization process. The recommendation is to initialize the pool during its creation instead. The bug has been fixed in a commit by Trugly Labs and also by Cantina Managed. The risk level of this bug is considered medium.

## Details
## Security Advisory

## Context
TruglyMemeception.sol#L326

## Description
The `UniswapV3Pool.initialize()` function can be called before the launch is complete, potentially causing a denial of service (DOS) in the finalization process.

## Recommendation
Initialize the pool during its creation.

## Status
- **Trugly Labs:** Fixed in commit [2199fab3](https://github.com/TruglyLabs/TruglyMemeception/commit/2199fab3).
- **Cantina Managed:** Fixed.
