# Iteration through all locks, including expired locks.

## Summary
The report describes a bug in the Expo.sol code, specifically in the functions transfer(), send(), burn(), and transferWithLock(). These functions check if a user is transferring more tokens than they have locked for a certain period of time. However, the code also checks expired locks, which is unnecessary and can use up more gas than allowed. This can prevent users from transferring their tokens if they have a lot of locks. The bug has been marked as medium severity since only the owner can create locks for users. The recommendation is to remove expired locks after a successful transfer. The bug has been resolved.

## Details
**Description**

Expo.sol: functions transfer(), send(), burn(), transferWithLock(). Functions iterate through all user's locks to verify that he doesn't transfer more than locked until some period of time. Iteration is performed through expired locks as well, performing unnecessary actions and increasing gas spendings. Also, in case there are a lot of locks for the user, iterating through all of them might consume more gas than allowed per transaction, preventing user from transferring his tokens. Issue is marked as medium, since only the owner can create locks for user.

**Recommendation**

Remove locks, which have expired after successful transfer.

**Re-audit comment**

Resolved
