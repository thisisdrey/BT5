# [M-09] BathBuddy locks up Ether it receives

## Summary
This bug report is about a vulnerability in the BathBuddy contract, which is able to receive ETH, but there is no way of ever retrieving that ETH from the contract. This means that the funds will be locked up, and the bug is considered a Medium issue. The proof of concept is the `receive()` function, and there are no tools used. To fix this vulnerability, the recommended mitigation steps are to either remove the `receive()` function if the contract isn't supposed to handle ETH, or to add the necessary logic to release the ETH it gets.

## Details
_Submitted by Ruhum, also found by 0x1f8b and pauliax_

The BathBuddy contract is able to receive ETH. But, there's no way of ever retrieving that ETH from the contract. The funds will be locked up.

Currently, there seems to be no logic in the protocol where ETH is sent to the contract. But, it might happen in the future. So I'd say it's a MED issue.

### Proof of Concept

`receive()`  function: <https://github.com/code-423n4/2022-05-rubicon/blob/main/contracts/peripheral_contracts/BathBuddy.sol#L69>

### Recommended Mitigation Steps

Remove the `receive()` function if the contract isn't supposed to handle ETH. Otherwise, add the necessary logic to release the ETH it gets.
