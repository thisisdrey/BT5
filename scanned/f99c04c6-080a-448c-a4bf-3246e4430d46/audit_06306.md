# Recent Solidity Versions May Not Be Supported By Layer 2 Systems

## Summary
Summary:

The bug report states that Solidity versions starting from 0.8.25 use an op-code called MCOPY by default. This op-code can cause issues on chains that have not upgraded to the EVM version Cancun. The report recommends using an earlier EVM version or a compiler version less than 0.8.25 to avoid this issue. The development team is resolving the issue by setting the EVM version to Paris during compilation.

## Details
## Description

Solidity versions starting from 0.8.25 use the `MCOPY` op-code by default. The op-code `MCOPY` will cause a revert if it is called on chains that are not upgraded to the EVM version Cancun. The Solidity operations that include `MCOPY` by default:

- The helper function `abi.encode()`;
- Functions which return byte arrays;
- Functions which return string types.

The testing team has compiled the contract and discovered occurrences of `MCOPY` in the bytecode. The impact of deploying the contract on chains that do not upgrade their EVM to Cancun is high. This is because the funds can be stuck. `MCOPY` will not be used when depositing funds, allowing funds to enter the protocol. However, when trying to redeem, the transaction will revert since the function `_registerRedeemRequest()` contains an `abi.encode()` instruction which uses `MCOPY` by default.

## Recommendations

Consider setting the Solidity compile flag `--evm-version` such that it is an earlier EVM version. Paris is a good choice for the EVM version as it does not contain either `PUSH0` or `MCOPY` opcodes.

Alternatively, use a compiler version which is strictly less than `0.8.25`.

Note that Shanghai (the default in Solidity from `0.8.20` to `0.8.24`) introduces the opcode `PUSH0`, which may or may not be supported by other chains.

Furthermore, validate the EVM version of each chain before compiling contracts and deploying to that chain.

## Resolution

The development team is resolving the issue by setting the EVM version to Paris during compilation.
