# Tree does not extend as expected

## Summary
The bug report is about a problem with a code called "phenomanelTree.sol". The problem was that when testing the code, it was found that the tree of a given referrer did not get filled as expected. This means that the code did not work properly and did not add the correct number of nodes. The expected number of nodes was 265720, but only 1093 were added. The recommendation is to adjust the algorithm and parameters used in the code, as it was noticed that the capacity of nodes decreased in a certain pattern to fit a smaller depth. The bug was fixed in a commit called "05ec745" and the tree of each individual user was shown to be filled up to the expected capacity after the fix.

## Details
**Severity**: High

**Status**: Resolved

**Description**

phenomanelTree.sol - Starting from positionUser, testing showed that the tree of a given referrer does not get filled as expected (i.e. according to given depth). For instance, if depth is 11 we expect to have 265720 nodes but only 1093 are added.

**Recommendation** 

Algorithm and parameters used in checks might need to be readjusted. It is noticed that the capacity of nodes gets decreased according to a pattern to fit a smaller depth.
**Fix** - Tree of each individual user is shown in test to be filled up to the expected capacity after fix is carried out on commit 05ec745 .
