# Incorrect update of `totalLiquidity`

## Summary
This bug report describes an issue with the Keep3r Network's StakingRewardsV3 contract. The bug occurs when a user calls the `deposit()`, `withdraw()`, and `getReward()` functions in succession. This causes the contract to incorrectly calculate the `totalLiquidity` value, which in turn leads to incorrect rewards calculations for users.

The recommendation for fixing this bug is to change the logic of the `update` modificator so that the `totalLiquidity` value will only update when an NFT is transferred to the contract. This should prevent the incorrect calculations of `totalLiquidity` and ensure that rewards are correctly calculated for users.

## Details
