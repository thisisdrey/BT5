# Lack of Broker Address Validation in MsgUpdateBrokerAddress's ValidateBasic Function

## Summary
In this bug report, it is mentioned that there is an issue with the **MsgUpdateBrokerAddress** function in the given code. This function is designed to update the broker address in an app chain, but there is a problem with the associated ValidateBasic method. This method does not validate the **BrokerAddress**, which can lead to swap operations failing. The report recommends enhancing the ValidateBasic function to include validation for the BrokerAddress to ensure it is correctly formatted and adheres to expected standards. The Elys Network team has already solved the issue and a remediation plan has been provided.

## Details
##### Description

The **MsgUpdateBrokerAddress** function in the given code is designed to update the broker address in app chain. This function creates an instance of MsgUpdateBrokerAddress with a specified creator and broker address. However, a crucial issue is identified in the associated ValidateBasic method: it does not validate the **BrokerAddress**. Currently, the **ValidateBasic** method only validates the Creator field, overlooking the validation of BrokerAddress. For instance, some swap operations can fail.

##### Recommendation

Enhance the **ValidateBasic** function to include validation for the BrokerAddress. This validation should ensure that the BrokerAddress is correctly formatted and adheres to the expected address standards.
