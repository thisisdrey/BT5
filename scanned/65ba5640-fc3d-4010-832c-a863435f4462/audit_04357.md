# ClaggSyncAdapter._addLiquidity() wrongly calls approve() instead of safeApprove() 

## Summary
The report describes a bug in the ClaggSyncAdapter code, specifically in the _addLiquidity() function. The function is incorrectly calling the approve() function instead of the safeApprove() function when depositing tokens as liquidity. This could cause issues because the token being used may not comply with the ERC20 specification. The recommendation is to call the safeApprove() function instead. This bug has been fixed in the commit 84e5179f and has been verified by Cantina Managed.

## Details
## ClaggSyncAdapter Issue Report

## Context
`ClaggSyncAdapter.sol#L186-L196`

## Description
In `ClaggSyncAdapter._addLiquidity()`, `approve()` is called instead of `safeApprove()` when depositing token as liquidity:

```solidity
IERC20(token).approve(address(syncRouter), amount);
liquidity = syncRouter.addLiquidity2(
    pool,
    inputs,
    abi.encode(address(this)),
    0,
    address(0),
    new bytes(0),
    address(0)
);
```

However, `safeApprove()` should be called instead as the token could be a token that does not comply with the ERC20 specification.

## Recommendation
Call `safeApprove()` instead:

```solidity
- IERC20(token).approve(address(syncRouter), amount);
+ IERC20(token).safeApprove(address(syncRouter), amount);
```

## Clave
Fixed in commit `84e5179f`.

## Cantina Managed
Verified, the recommendation was implemented.
