# Broken `operatorFilterRegistry` may lead to DOS

## Summary
This bug report is about the `operatorFilterRegistry` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/openseaFilterUpgradeable/OperatorFiltererUpgradeable.sol#L10) in the Fantium Smart Contracts. If this registry is broken and reverts on every transaction, all transfers with the `modifier onlyAllowedOperator` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/openseaFilterUpgradeable/OperatorFiltererUpgradeable.sol#L32) will be locked.

The recommendation is for the platform manager to update the `operatorFilterRegistry` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/openseaFilterUpgradeable/OperatorFiltererUpgradeable.sol#L10). This will help to prevent the issue of all transfers being locked if the registry is broken.

## Details
##### Description

If the `operatorFilterRegistry` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/openseaFilterUpgradeable/OperatorFiltererUpgradeable.sol#L10) is broken and reverts on every transaction, then all transfers with `modifier onlyAllowedOperator` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/openseaFilterUpgradeable/OperatorFiltererUpgradeable.sol#L32) will be locked.

##### Recommendation

It is recommended to allow the platform manager to update `operatorFilterRegistry` (https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/openseaFilterUpgradeable/OperatorFiltererUpgradeable.sol#L10).
