# A Transaction May Not Distribute All Funds

## Summary
A bug was found in the code of `Hinkal.sol` where the contract was able to keep more tokens than expected. This could potentially lead to incomplete withdrawals. The client has marked the bug as "Fixed" and provided a solution by changing the balancing inequality to an equality. They also recommended using a different verification method when external integrations are involved in transactions.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `b9979de6ba2d63e833d96b153477580b1c6d4e6f`. The client provided the following explanation:

> Changed the balancing inequality to the equality

**File(s) affected:**`Hinkal.sol`

**Description:** After completing the transactions, `_transact()` loops through the exchanged tokens to compare the changes in the token balance seen to the changes in the token balance expected by the provided proof. It then checks to confirm that the contract has not distributed more tokens than expected:

```
require(
    balanceDif >= circomData.amountChanges[i],
    "Inbalance in token detected"
);
```

This creates an imbalance where the contract can keep more tokens than expected. Though this might be in place due to unforeseen transfer amounts from integrations with external protocols, this still allows for incomplete withdrawals.

**Recommendation:** Require a strict equality between the expected balance changes and the actual balance changes. Ideally, a different verification method should be used when a transaction involves external integrations.
