# [M-02] Marketplace calls unimplemented function

## Summary
This bug report was submitted by 0x52 and relates to the implementation of the transferFrom function in the Marketplace.sol contract. This function is not implemented, which means that the mint and mintWithUnderlying functions will not function correctly. The recommended mitigation step is to create an implementation for transferFrom. JTraversa (Illuminate) disagreed with the severity of the bug, claiming that no value would be at risk. gzeoneth (judge) decreased the severity of the bug to Medium, also claiming that no value would be at risk and that the function of the protocol would be impacted.

### Vulnerability details

While safe.sol isn't explicitly listed in scope it is listed as a library for all in scope contracts so I believe it should be in scope

### Impact

mint and mintWithUnderlying won't function.

### Proof of Concept

`safe.sol` never implements a transferFrom function meaning it will revert whenever a user calls either function.

### Recommended Mitigation Steps

Create an implementation for transferFrom.
