# Insufficient slippage protection can lead to displaced transactions

## Summary
The report is about a bug in a contract called `CurveLiquidityWarehouse.sol`. The bug is that the `_withdrawFromTarget` function does not have a parameter for a deadline, which is the last moment when the transaction must be accepted. This can cause the transaction to be executed at a later time, resulting in a worse price. The recommendation is to add a deadline parameter and check if the transaction is executed after the deadline, and if so, revert the transaction. This bug is considered to be of high severity and has been acknowledged.

## Details
**Severity**: High	

**Status**: Acknowledged

**Description**

The `_withdrawFromTarget` function in the `CurveLiquidityWarehouse.sol` contract does not receive a deadline argument representing the last moment at which the transaction must be accepted. This can lead to a scenario where the transaction is displaced and executed in later blocks obtaining a worse price.

**Recommendation**:

Add a `deadline` parameter and check when the transaction is executed, if `block.timestamp` is greater than deadline, revert.
