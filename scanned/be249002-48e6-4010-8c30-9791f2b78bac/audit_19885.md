# No validation of the value of the variable `msg.value`

## Summary
A bug report has been written regarding the code found at lines 57-68 of the 1inch-contract repository on GitHub. The issue is that when processing input data, `ETH` is not required to work with `WETH` and regular tokens, but the user can inadvertently transfer it and lose the `ETH` as a result. To prevent this from happening, the code should be updated with additional checks before lines 60 and 67. The recommended change is to add the following code: `require(msg.value == 0, "CL1IN: wrong msg.value");`

## Details
##### Description
At the lines https://github.com/1inch/1inch-contract/blob/93868c483180cf74fc2551568f0396938b3eeaa8/contracts/ClipperRouter.sol#L57-L68 
processing of the input data is done before the exchange procedure.
`ETH` is not required to work with `WETH` and regular tokens. But the user can inadvertently transfer it.
In this case, the user will lose these `ETH`.
To prevent this from happening, you need to add checks before lines 60 and 67:
```solidity=
 require(msg.value == 0, "CL1IN: wrong msg.value");
```

##### Recommendation
Additional checks need to be added.
