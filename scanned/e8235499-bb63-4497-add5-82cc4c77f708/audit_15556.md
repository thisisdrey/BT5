# TRST-M-4 hedgeDelta(0) doesn’t behave properly

## Summary
This bug report is about an issue with the function `hedgeDelta()` in a pool. When the pool wishes to reset the delta to zero, the function `hedgeDelta(0)` is called. However, the flow will enter the sell wETH branch and call `_createUniswapRangeOrder()` with 0 delta, which results in the minting of a UniswapV3 position with 0 liquidity, which reverts at the Uniswap level. As a result, the previous exposure remains as `_yankRangeOrderLiquidity()` is not called.

The team recommended to add branching logic for hedgeDelta. If delta is 0, the team suggested to do nothing. The team fixed the issue and hedgeDelta() now correctly implements an early-exit in case delta is 0.

## Details
**Description:**
`hedgeDelta()` is called again by the pool when the exposure to underlying asset needs to 
change. If it was previously non-zero and the pool wishes to reset the delta to zero, 
`hedgeDelta(0)` would be called. Unfortunately, it will never execute.

Flow will enter the sell wETH branch and call `_createUniswapRangeOrder()` with 0 delta. 
Eventually it will try minting a UniswapV3 position with 0 liquidity, which reverts at the 
Uniswap level.

As a result, the previous exposure remains as `_yankRangeOrderLiquidity()` is not called.

**Recommendation:**
Add branching logic for hedgeDelta. If delta is 0, do nothing.

**Team response**
Fixed

**Mitigation Review**
hedgeDelta() now correctly implements an early-exit in case _delta is 0.
