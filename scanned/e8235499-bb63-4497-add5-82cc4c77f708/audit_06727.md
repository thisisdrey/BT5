# M-24: The SelectTopNWorkerNonces function lacks a sorting algorithm internally.

## Summary
The issue reported is that the SelectTopNWorkerNonces function in the Allora blockchain lacks a sorting algorithm and only selects a certain number of workers from the array, instead of selecting the top N latest workers based on the Nonce. This can result in choosing workers that are already offline or unable to provide accurate data. The impact of this bug is that it does not select the intended workers and can lead to incorrect data being used. A manual review was used to identify this issue and the recommendation is to add a sorting function within the SelectTopNWorkerNonces function. The protocol team has fixed this issue in a recent PR/commit.

## Impact

It does not select the top N latest worker nonces as intended. Moreover, selecting workers not within the top might result in choosing workers that are already offline or not able to return the desired data correctly.
