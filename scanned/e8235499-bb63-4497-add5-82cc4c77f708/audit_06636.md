# ChainID should not be 1

## Summary
A bug has been reported in the codebase of a project called Overprotocol. This bug is considered to be of medium risk and it affects several files in the project, namely params/config.go, params/bootnodes.go, cmd/clef/main.go, and cmd/clef/README.md. The issue is related to the use of default values for the ChainID and references to Ethereum bootnodes in the code. This could potentially cause problems with cross pollination between Overprotocol and Ethereum nodes, as well as the ability for transactions to be replayed between the two chains. To fix this bug, it is recommended that the default ChainID for Overprotocol Mainnet be changed to something unique and all references to Ethereum bootnodes be removed from the codebase. This will prevent any potential issues and ensure the stability of the project.

## Description
There are references in the codebase to Ethereum related bootnode ENRs as well as chainID defaults that are equal to Ethereum mainnet and various testnets.

## Recommendation
The default ChainID for Overprotocol Mainnet should be changed to something unique, and ENRs of all Ethereum related bootnodes should be removed from the codebase. This will prevent cross-pollination with Ethereum nodes in mempools and during syncing. More importantly, this will prevent the ability for transaction replays between chains.
