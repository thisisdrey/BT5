# It is possible to go beyond the boundaries of the array

## Summary
Bug report summary: In the line https://github.com/iearn-finance/yoracle.link/blob/faf1309cbe7a05f70b338351315039eb8e5b9c09/contracts/Keep3rV1Oracle.sol#L685 of the 'sample' function, there is no check for a valid 'observations[pair]' array element number. This means that on lines 700, 701, 709, and 710, a call is made to the array element with the 'nextIndex' index, but there is no check that the 'nextIndex' value is less than and equal to the value of the variable 'length'. To fix this bug, it is recommended to add such a check.

## Details
##### Description
At the line https://github.com/iearn-finance/yoracle.link/blob/faf1309cbe7a05f70b338351315039eb8e5b9c09/contracts/Keep3rV1Oracle.sol#L685 the `sample` function does not check for a valid `observations[pair]` array element number.
On lines 700, 701, 709, 710 there is a call to the array element with the `nextIndex` index. But there is nowhere a check that the `nextIndex` value is less than and equal to the value of the variable `length`.

##### Recommendation
We recommend to add such a check.
