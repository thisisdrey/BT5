# Insuffisient quality of the `_collateralERC20` check in BAMM

## Summary
The report is about a bug in the BAMM constructor that allows users to input an incorrect address in the `_collateralERC20` parameter. This can cause issues such as using the wrong token or a token with the wrong decimal value. The recommendation is to either add a check for the address or remove the parameter and retrieve the address from the `StabilityPool` contract instead.

## Details
##### Description
It is possible to provide the wrong address in the `_collateralERC20` parameter's value to the BAMM constructor:
- value that is different from StabilityPool,
- value that is not an ERC20 token,
- address of a token with the wrong decimals value (not 18)
##### Recommendation
We recommend adding a check of the `_collateralAddress` address parameter or removing this parameter from the constructor and getting it from the `StabilityPool` contract.
