# Incorrect input data could break the collection

## Summary
A bug report has been submitted regarding the FantiumNFTV3.sol contract on GitHub. The report suggests that a check should be added to ensure that the `_athletePrimarySalesBPS` is less than or equal to 10_000, as if it is greater than 10_000, the collection mint would revert due to an underflow. It is recommended that the following check be added: `_athletePrimarySalesBPS <= 10_000`. This check should be added to the line 524-525 of the FantiumNFTV3.sol contract.

## Details
##### Description
Here https://github.com/FantiumAG/smart-contracts/blob/a2d126453c1105028f12277b8f342d2cdbf01a77/contracts/FantiumNFTV3.sol#L524-L525 the `_athletePrimarySalesBPS <= 10_000` check should be added. If `_athletePrimarySalesBPS` is > 10_000, then the collection mint would revert because of the underflow.

##### Recommendation
We recommend adding the following check: `_athletePrimarySalesBPS <= 10_000`.
