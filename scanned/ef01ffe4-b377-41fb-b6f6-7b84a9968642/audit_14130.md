# Changing the Reward Pool Ratios Can Lead to Locked Funds

## Summary
This bug report is about an issue with the `distribute()` function in the `rewards/VaultRewardDistributor.sol` file. The function assumes that two ratios, `supplyRewardPoolRatio` and `borrowRewardPoolRatio`, will always add up to `1000`. However, if the values of these ratios are changed using the `setSupplyRewardPoolRatio()` or `setBorrowRewardPoolRatio()` functions, the sum may be different and not all rewards will be distributed. This can cause the rewards to become locked. The recommendation is to calculate the distribution based on the sum of the two ratios and to possibly create a new function to update both ratios at once to avoid any issues.

## Details
**Update**
The team has fixed the issue by following the recommendation to update both ratios in the functions.

**File(s) affected:**`rewards/VaultRewardDistributor.sol`

**Description:**`distribute()` assumes that `supplyRewardPoolRatio` and `borrowRewardPoolRatio` will sum to `1000`. However, this may not be the case if `setSupplyRewardPoolRatio()` or `setBorrowRewardPoolRatio()` modify the values to sum to a different number. If the sum of the ratios is less than `1000`, not all of the rewards will be distributed and they will become locked.

**Recommendation:** Calculate the distribution based on the sum of `supplyRewardPoolRatio` and `borrowRewardPoolRatio`. Consider creating a function that allows updating both ratios at once to avoid any undesired ratios in between calls.
