# H-2: Bull can `transferPosition()` to `address(0)` and the original order can be matched again

## Summary
This bug report is about the BullvBear Protocol, a decentralized trading platform. The bug is that the bull can transferPosition() to address(0) and the original order can be matched again. This is a vulnerability because an order should not be matched more than once.

The vulnerability was found by GimelSec, dipp, aviggiano, Bahurum, bin2chen, curiousapple, imare, KingNFT, rvierdiiev, 0x52, carrot, hansfriese, WATCHPUG, and neumo. The code snippet relevant to the issue is located at BvbProtocol.sol#L734-L761 and BvbProtocol.sol#L521-L538. The tool used to find the issue was manual review.

The impact of this vulnerability is that the attacker can match the orders of the bear makers multiple times, pulling order.premium + bearFees from the victims' wallet as many times as they want.

The recommendation to fix this issue is to consider using matchedOrders[contractId] to check if the order has been matched or not. Additionally, consider disallowing transferPosition() to address(0).

The discussion section of the report includes a link to a pull request fixing the issue, and a link to a pull request fixing the transfer to 0x0.

## Summary

Using `bulls[uint(orderHash)] == address(0)` to check whether the order is matched is insufficient, the bull can `transferPosition` to `address(0)` and the order can be matched again.

## Vulnerability Detail

An order must not be matched more than once.

There is a check presented in the current implementation to prevent that: L760 `require(bulls[uint(orderHash)] == address(0), "ORDER_ALREADY_MATCHED");`.

However, this check can be easily bypassed by the bull, as they can `transferPosition()` to `address(0)` anytime.

Then the original order can be matched again.

## Impact

Attacker can match the orders by bear makers multiple times, pulling `order.premium + bearFees` from the victims' wallet as many times as they want.

## Recommendation

Consider using `matchedOrders[contractId]` to check if the order has been matched or not. Also, consider disallowing `transferPosition()` to `address(0)`.
