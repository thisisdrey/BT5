# Add preconditions to important functions

## Summary
This bug report is about a function called getAddress in line 20 of bitcoin.js of fundraiser-lib. This function does not perform any checks on the argument 'pub'. If an invalid value is sent, an address will be returned, corresponding to no public key. This can lead to critical bugs from programming or user input handling errors. To prevent this, more precondition checks should be added to the function.

## Details
Some functions could use more precondition checking. This helps prevent critical bugs coming from programming or user input handling errors. For example, [the function getAddress in line 20 of bitcoin.js of fundraiser-lib](https://github.com/cosmos/fundraiser-lib/blob/426425dfc296060a9b87830e69e19ae8a6d444c0/src/bitcoin.js#L20) performs no checks on the `pub` argument. If there’s a bug in the caller code and and invalid value is sent, an address will be returned, corresponding to no public key. Example:
