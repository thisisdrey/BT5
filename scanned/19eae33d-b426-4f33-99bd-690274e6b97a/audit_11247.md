# Meta_pool doesn't allow base_pools with len(coins) > 3

## Summary
A bug has been identified in the current implementation of the meta_pool, which does not allow base_pools with more than 3 tokens. This means that if a meta_pool is created with a base_pool with more than 3 tokens, part of the meta_pool functionality will not work. The bug has been identified in the code at https://github.com/curvefi/stableswap-ng/blob/8c78731ed43c22e6bcdcb5d39b0a7d02f8cb0386/contracts/main/CurveStableSwapMetaNG.vy#L64-L68.

The recommended solution to this bug is to add a check in the constructor to ensure that the number of tokens (BASE_N_COINS) is less than 4. This will ensure that the meta_pool functionality works correctly.

## Details
##### Description
The current implementation of the meta_pool doesn't allow base_pools with more than 3 tokens https://github.com/curvefi/stableswap-ng/blob/8c78731ed43c22e6bcdcb5d39b0a7d02f8cb0386/contracts/main/CurveStableSwapMetaNG.vy#L64-L68 but such base_pools can be added in the factory. If meta_pool will be created with base_pool with more than 3 tokens, part of the meta_pool functionality will not work.

##### Recommendation
We recommend adding a check in the constructor that `BASE_N_COINS < 4`.
