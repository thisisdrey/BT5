# EOPBCTemplate - permission documentation inconsistencies ✓ Fixed

## Summary
This bug report is about the permissions set up by the template contract in the externally owned presale bonding curve (EOPBC) not being documented in the accompanying README.md. The template sets up two permissions that are not documented in the README.md: one in the TokenManager and one in the Controller. In the TokenManager, the owner is granted the MINT_ROLE and the marketMaker is granted the BURN_ROLE. In the Controller, the owner is granted the UPDATE_BENEFICIARY, UPDATE_FEES, ADD_COLLATERAL_TOKEN, REMOVE_COLLATERAL_TOKEN, UPDATE_COLLATERAL_TOKEN, UPDATE_MAXIMUM_TAP_RATE_INCREASE_PCT, UPDATE_MAXIMUM_TAP_FLOOR_DECREASE_PCT, ADD_TOKEN_TAP, UPDATE_TOKEN_TAP, OPEN_PRESALE, OPEN_TRADING, CONTRIBUTE, OPEN_BUY_ORDER, OPEN_SELL_ORDER, and WITHDRAW permissions.

The bug was fixed by adding the undocumented and deviating permissions to the documentation. For transparency, all permissions set-up by the template must be documented.
