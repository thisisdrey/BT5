# max_supply will not be reached as there is an early return that does set ismintingfinished to early 

## Summary
The report states that there is a bug in the publicMint() function of DCT. The function has a check that stops the minting of tokens when the total supply reaches a certain limit. However, the function does not take into account the pendingA() function, which calculates the amount of tokens to be minted per second. This means that even when the minting is stopped, there could still be a large number of tokens left to be minted if the protocol is not widely used. The report recommends normalizing the minting amount to ensure that all tokens are minted before the total supply reaches its limit. The bug has been fixed in the code by adding a line that mints the remaining tokens before setting the isMintingFinished flag to true.

## Description
Currently in the `publicMint()` function in DCT, there is a check that if the `totalSupply()` of tokens plus the minting amount:

```solidity
uint mintingA = pendingA();
```

is bigger than `MAX_SUPPLY`:

```solidity
if (totalSupply() + mintingA > MAX_SUPPLY) {
    isMintingFinished = true;
    return;
}
```

The minting of DCT will be finished and no more GOAT will be minted. The issue here is that there will still be GOAT left to be minted even when `isMintingFinished` is true. This is because the `pendingA()` function:

```solidity
function pendingA() public view returns (uint) {
    if (isMintingFinished || _lastMintAt == 0) {
        return 0;
    }
    uint pastTime = block.timestamp - _lastMintAt;
    return _tps * pastTime;
}
```

calculates the amount of tokens to be minted per second multiplied by the past time from the last mint. This will only increment with the increase of time; therefore, the amount of tokens left to mint after setting `isMintingFinished = true` could be a huge amount if the protocol is not the most used one in the space.

Just to make things clearer, currently 7 tokens per second should be the minting rate:

```solidity
uint private _tps = 7 ether; // 7 tokens per second.
```

If `MAX_SUPPLY = 3111666666` and there have been `totalSupply() = 3111660000` tokens minted, meaning there are still 6666 tokens left to be minted. If 953 seconds go by without anyone calling `publicMint()`, the tokens will never be minted. This is less than 15 minutes without the function being called, which is extremely likely; it could be much more.

## Recommendation
If `totalSupply() + mintingA > MAX_SUPPLY`, normalize the `mintingA` amount to mint the last tokens before `MAX_SUPPLY` is reached:

```solidity
if (totalSupply() + mintingA > MAX_SUPPLY) {
    mintingA = MAX_SUPPLY - totalSupply();
    isMintingFinished = true;
    return;
}
```

Goat: fixed 

```solidity
if (totalSupply() + mintingA > MAX_SUPPLY) {
    isMintingFinished = true;
    _mint(_rewardPool, MAX_SUPPLY - totalSupply());
    _lastMintAt = block.timestamp;
    return;
}
```
