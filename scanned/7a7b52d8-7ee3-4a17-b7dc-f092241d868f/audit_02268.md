# M-2: Jailed validators are able to participate in voting

## Summary
This bug report discusses an issue found by BrothersInCode and forgebyola regarding the voting process on the Zetachain cross-chain platform. The current code only prevents tombstoned validators from voting, but does not prevent jailed validators from doing so. This can lead to misbehaving validators being able to vote. The recommendation is to add a jailed validator check during the voting process to ensure that only active and non-jailed validators can vote. The issue has been fixed by replacing instances of `IsNonTombstonedObserver()` with `CheckObserverCanVote()`, which also checks if a validator is jailed.

## Details
Source: https://github.com/sherlock-audit/2025-04-zetachain-cross-chain-judging/issues/46 

## Found by 
BrothersInCode, forgebyola

### Description

When voting on an inbound or outbound ballot, the following check is performed:

```go
func (k Keeper) VoteOnInboundBallot(
	ctx sdk.Context,
	senderChainID int64,
	receiverChainID int64,
	coinType coin.CoinType,
	voter string,
	ballotIndex string,
	inboundHash string,
) (isFinalized bool, isNew bool, err error) {
	// ... code

	// Ensure the voter is an active (non-tombstoned) observer
	if ok := k.IsNonTombstonedObserver(ctx, voter); !ok {
		return false, false, types.ErrNotObserver
	}

	// ... code
}
```

This prevents tombstoned validators from voting on the observation chain. However, the current logic does **not** prevent jailed validators from voting.

In contrast, a jailed check is correctly performed during observer updates:

```go
func (k msgServer) UpdateObserver(
	goCtx context.Context,
	msg *types.MsgUpdateObserver,
) (*types.MsgUpdateObserverResponse, error) {
	// ... code

	// New observer must be a bonded, non-jailed validator
	err = k.IsValidator(ctx, msg.NewObserverAddress)
	if err != nil {
		return nil, errorsmod.Wrap(types.ErrUpdateObserver, err.Error())
	}
}
```

Within [`IsValidator`](https://github.com/zeta-chain/node/blob/a32ffd88ea9957c8f3424466bb358b4197d41889/x/observer/keeper/voting.go#L94-L108):

```go
if validator.Jailed || !validator.IsBonded() {
	return types.ErrValidatorStatus
}
```

This ensures that only active, non-jailed validators can be assigned as observers.

### Impact

Since no such check exists during voting, jailed validators are still able to vote, even though they may have misbehaved.

### Recommendation

Add a jailed validator check during the voting process to ensure that only bonded and non-jailed validators are allowed to vote.
