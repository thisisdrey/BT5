# Treasury withdrawals will revert for non-18 token decimals

## Summary
The bug report describes an issue with the `treasuryCash` variable in the `Storage.sol` file. This variable is not being properly unscaled for withdrawals, causing it to revert for tokens with a `scalingFactor` greater than 1. This can affect tokens with less than 18 decimals, such as USDC. The report includes a proof of concept and a recommendation to unscale the amount to the raw amount. The bug has been fixed in a recent commit by Pendle Finance and verified by Spearbit.

## Details
## Severity: Medium Risk

## Context
**File:** Storage.sol  
**Lines:** 147-151

## Description
The `treasuryCash` is in scaled amounts, but it isn't unscaled to the raw amounts for withdrawals. As a result, it will revert for tokens with `scalingFactor > 1`, such as tokens that have less than 18 decimals (e.g., USDC).

## Recommendation
Unscale the amount to the raw amount:
```solidity
- IERC20(data.token).safeTransfer(TREASURY, amount);
+ IERC20(data.token).safeTransfer(TREASURY, amount / data.scalingFactor);
```
