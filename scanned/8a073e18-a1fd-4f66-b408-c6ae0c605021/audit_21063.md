# Incorrect check of `timeWindow`

## Summary
This bug report is about an error in the code of the Cover Protocol's core version 2. The code in question is located at the line https://github.com/CoverProtocol/cover-core-v2/blob/513f5e502a8e8a623729c2c3480fca4e80fdef39/contracts/ClaimConfig.sol#L50. The error is that there is an incorrect check of `_newTimeWindow`. 

The recommendation is to change the code to `require(_newTimeWindow >= 3 days, "CC: window too short");`. This means that the code should be updated to ensure that the time window is greater than or equal to 3 days. This will help to ensure that the code is functioning properly and that the Cover Protocol's core version 2 is running correctly.
