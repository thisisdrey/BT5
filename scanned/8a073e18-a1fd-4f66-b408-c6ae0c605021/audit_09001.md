# User funds can be withdraw without their permission 

## Summary
The bug report is about a function called "claimFunds()" in the QuailFinance smart contract. This function allows the owner of the contract to withdraw all USDB funds without the permission of the participants. This is because the function creates a merkle tree with leaves that add up to the total amount of deposits made by the participants, allowing the owner to withdraw all funds at once. The recommendation is to ensure that only the yield can be claimed and not the entire amount.

## Details
## QuailFinance.sol#L229-L229

## Description
The `claimFunds()` function enables the owner to withdraw all USDB funds in the contract. This allows pot participants' funds to be withdrawn without their permission:

- The owner creates a Merkle tree with leaves such that the total `claimAmount` in the leaves equals the deposits made by pot participants.
- The `claimFunds()` function is called, withdrawing all funds.

## Recommendation
Ensure that only the yield can be claimed programmatically, and nothing more.
