# TODO comments should be resolved

## Summary
This bug report is about ensuring that the application is ready to be put into production. It states that any comments made by developers, such as TODOs, should be resolved before the application is ready for production. These comments often indicate a missing functionality or missing validation checks that need to be addressed. This report is important because it ensures that the application is ready to be used in a production environment, with all necessary functions and validations present.

## Details
#### Description


As part of the process of bringing the application to production readiness, dev comments (especially TODOs) should be resolved. In many cases, these comments indicate a missing functionality that should be implemented, or some missing necessary validation checks.
