# [M-08] `WeightedIndex` creation could always fail if pool is already created

## Summary
This bug report is about the `WeightedIndex` contract which creates a V2 pool with pod and index tokens. An attacker can predict the address of the `WeightedIndex` and cause the `createV2Pool` function to revert because the pool already exists. The severity of this bug is high as it can impact the functioning of the contract. The likelihood of this bug occurring is low. The recommendation is to only call the `createV2Pool` function if the pool does not already exist.

## Details
## Severity

**Impact:** HIgh

**Likelihood:** Low

However, it is possible that an attacker that can predict the address of `WeightedIndex` front runs the creation of the pool and causes the `createV2Pool` to revert because the pool already exists.

## Recommendations

Consider calling `createV2Pool` only if a pool does not yet exist.
