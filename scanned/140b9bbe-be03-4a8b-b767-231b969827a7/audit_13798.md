# Incorrect Update of `_withdraws` Mapping in `_transfer()`

## Summary
The Fabric team recently fixed an issue where the `_withdraws` mapping was not being updated correctly when contributions were transferred between accounts. This caused incorrect values for `_withdraws[from]` and `_withdraws[to]` when the `withdrawYieldBalance()` function was called. The formula used to calculate `withdrawAmount` was incorrect and should have been `(amount * fromWithdraws) / fromBalance` instead of `(fromBalance - amount) * fromWithdraws) / fromBalance`. This bug could be exploited by a malicious user to withdraw all of the available yield balance from the contract by transferring a contribution of zero value to a random address and then repeatedly calling `withdrawYieldBalance()`. The Fabric team has fixed this issue in their code by implementing the recommended formula.

## Details
**Update**
The Fabric team fixed the issue in commit `5b5df31` by implementing our recommendation.

**Description:** When contributions are transferred between accounts, the `_withdraws` mapping must be updated to accurately reflect the yield balance when the `withdrawYieldBalance()` function is called. However, the current implementation is flawed, resulting in incorrect values for `_withdraws[from]` and `_withdraws[to]`. Specifically, the formula used to calculate `withdrawAmount` is incorrect, and should be updated from:

```
withdrawAmount = (fromBalance - amount) * fromWithdraws) / fromBalance;
```

To:

```
withdrawAmount = (amount * fromWithdraws) / fromBalance;
```

This error can be exploited by a malicious user to withdraw all of the available yield balance from the contract by executing the following steps multiple times until the contract balance is empty:

1.   The malicious user transfers a contribution of zero value to a random address, setting its `_withdraws[from]` value to zero.
2.   The malicious user executes `withdrawYieldBalance()` to withdraw its generated yield again.

**Recommendation:** Change `withdrawAmount` computation to:

```
withdrawAmount = (amount * fromWithdraws) / fromBalance
```
