# Lack of Validation for valid\_till\_block\_height on FastBridge Service

## Summary
This bug report concerns the FastBridge Service, which is responsible for managing transfers between the NEAR and Ethereum networks. The issue is that the valid_till_block_height parameter, which is used to set an expiration block height for the transfer, is not being validated properly. This could lead to transfers being processed after they have expired. To fix this issue, it is recommended that the FastBridge Service implement proper validation for the valid_till_block_height parameter. This could include checks to ensure that the current block height is not greater than the valid_till_block_height value.
