# Users Can Submit to Bounties They Funded

## Summary
The client has acknowledged a bug where users are able to submit to bounties that they have funded themselves. This is because the functions `approveSubmission()` and `makeSubmission()` do not check if the user is the funder of the bounty. This allows users to receive back their initial bounty amount and all the submission stakes from failed submissions. It is recommended to review this behavior and prevent users from submitting to their own bounties if it is not intended.

## Details
**Update**
Marked as "Acknowledged" by the client. The client provided the following explanation:

> We are going to allow this behavior to account for the possibility that someone discovers the solution to their own bounty. Since this is costly in fees to the poster, we don't expect it to happen often. Also, while it is possible that bounty posters might close down bounties early to collect extra submission stakes, the incentives don't work out to encourage people to post deliberately tricky bounties to try to catch as many failed submissions as possible.

**File(s) affected:**`BountyV1.sol`

**Description:** It is possible for a user to submit to bounties they funded. That is because the functions `approveSubmission()` and `makeSubmission` do not check if `msg.sender` is the `funder` of a given bounty.

The benefit users will gain in such scenarios is to get back a bounty's initial amount + all the `submissionStake` of all failed submissions.

**Recommendation:** Review if this is intended. If not, then prevent users from submitting to their own bounties.
