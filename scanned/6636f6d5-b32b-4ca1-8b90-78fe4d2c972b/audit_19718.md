# [M-01] Unable to transfer `WrappedIbbtc` if Oracle go down

## Summary
The user gzeon submitted a bug report about a problem with the `WrappedIbbtc` function. They found that if the `oracle.pricePerShare()` reverts, users will not be able to transfer. This is because the function `balanceToShares()` is called in both transfer and `transferFrom`, which includes a call to `pricePerShare()`. Another issue is that the `WrappedIbbtcEth` function behaves differently, as it uses a cached value instead of calling `updatePricePerShare()`. The recommended solution is to change one of the functions depending on the specification. The team at BadgerDAO has confirmed this issue.

## Details
_Submitted by gzeon_

#### Impact
In `WrappedIbbtc`, user will not be able to transfer if `oracle.pricePerShare()` (L124) revert. This is because `balanceToShares()` is called in both transfer and `transferFrom`, which included a call to `pricePerShare()`.

If this is the expected behavior, note that `WrappedIbbtcEth` is behaving the opposite as it uses the cached value in a local variable `pricePerShare`, which is only updated upon call to `updatePricePerShare()`.

#### Recommended Mitigation Steps
Depending on the specification, one of them need to be changed.

**[dapp-whisperer (BadgerDAO) confirmed](https://github.com/code-423n4/2021-10-badgerdao-findings/issues/20)**
