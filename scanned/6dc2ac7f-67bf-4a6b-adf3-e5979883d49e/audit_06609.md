# Old Distributions May Still Exist After Updating

## Summary
There is a bug in the YRizStrategy contract where the setFullPoolDistributions function can cause unexpected behavior if the number of distributions decreases. This is because the _setPoolDistribution function does not delete or overwrite old distributions if the new number is less than the previous number. This can cause issues with functions like getUiKeeperPoolBalances, getAllPoolDistributions, and _deployFunds. The solution is to delete old distributions that are no longer in use and set the pool and bps values to address(0) and 0, respectively. This will prevent invalid pools from receiving distributions and help detect them early. The bug has been resolved in a recent pull request.

## Details
Within the [`YRizStrategy` contract](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol), the [`setFullPoolDistributions` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L133-L149) changes the content of the `_distributions` mapping via the [`_setPoolDistribution` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L144).


However, in the event that [`newDistros.length`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L134) is less than the number of previous elements in the `_distributions` mapping, the elements which exceed the length of `newDistros` will never be overwritten nor deleted in the [`_setPoolDistribution` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L144).


This means that if there is a reconfiguration of distributions where the number of distributions decreases, some of these old distributions may still appear valid, and functions such as [`getUiKeeperPoolBalances`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L96-L113), [`getAllPoolDistributions`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L178), and [`_deployFunds`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L214-L225) may behave unintuitively and may move funds in unexpected ways.


Consider deleting `Distributions` which are no longer used, and setting both the `pool` and `bps` values to `address(0)` and `0`, respectively. This will prevent old invalid pools from receiving distributions, and will help [detect invalid pools early in loops](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/vaults/YRizStrategy.sol#L219-L220).


***Update:** Resolved in [pull request \#100](https://github.com/radiant-capital/riz/pull/100) at commit [e15ba56](https://github.com/radiant-capital/riz/pull/100/commits/e15ba565f3ca3647b4d87e389ad2fcd9949fb8a3).*
