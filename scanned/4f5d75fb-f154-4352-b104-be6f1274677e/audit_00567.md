# M-3: EIP violation for `totalAssets()` in the `Vault`

## Summary
This is a bug report for a stablecoin factory judging issue on GitHub. The issue has been acknowledged by the team but will not be fixed at this time. The bug was found by a group of users and it involves the `totalAssets()` function calling `getPendingInterest()` which can cause reverts. This violates EIP4626 which states that `totalAssets()` should not revert. The bug is caused by the implementation not respecting this statement. The attack path involves the `totalAssets()` function calling `calculateInterest()` which can cause reverts due to different errors. The impact of this bug is considered medium severity due to the EIP violation. A proof of concept has been provided in the report and the recommended mitigation is to change the implementation so that `totalAssets()` cannot revert.

### Summary

According to the contest README:

```solidity
Issues that break the EIP's MUST statements may be deemed valid Medium severity (even if the violation is in view functions and doesn't impact state functions) even if the impact is low/info, unless they conflict with common sense.
```

But `totalAssets()` can revert because of its call to `getPendingInterest()` function.

### Root Cause

[EIP4626](https://eips.ethereum.org/EIPS/eip-4626) has the following statement regarding `totalAssets()`:

```solidity
 MUST NOT revert
```

However, it's not respected in the current implementation.

### Attack Path

`totalAssets()` function in the `Vault` calls `getPendingInterest()` in the `Lender` contract that, in its turn, calls `calculateInterest()` inside of the try/catch mechanism meaning the function is prone to reverts due to different errors (like division by zero or some other type of error) and, inside of the catch block, the function checks whether the call was with sufficient gas:

https://github.com/sherlock-audit/2025-12-monolith-stablecoin-factory/blob/main/Monolith/src/Lender.sol#L908-910
```solidity
  } catch {
            require(gasBefore >= INTEREST_CALCULATION_GAS_REQUIREMENT, "Not enough gas for accrueInterest");
        }
```

The value is set to 40000 gas and it won't have to require that amount if it didn't happen to revert in the first place. So if users doesn't provide more gas than estimated by RPC, the call will revert.

### Impact

EIP violation -> medium severity.

### PoC

Provided in the attack path.

### Mitigation

Change the implementation so that `totalAssets()` can't revert.
