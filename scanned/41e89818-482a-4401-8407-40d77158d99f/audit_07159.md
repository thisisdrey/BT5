# Missing Associated Token Account Check

## Summary
This bug report discusses an issue with the stake::process_stake function in the Rust programming language. This function loads various accounts, including the treasury_tokens_info account, which is supposed to be associated with the treasury token account. However, without a strict equality check, there is no guarantee that the provided treasury_tokens_info account is actually the correct associated token account. This could allow an attacker to redirect staked tokens to an account they control instead of the intended treasury account. The report suggests implementing a strict equality check to ensure the correct associated token account is used. This issue has been resolved in version 3b1039b of the program.

## Details
## Stake Process

`stake::process_stake` loads various accounts, including the `treasury_tokens_info` account, which is intended to be the associated token account of the treasury. However, without a strict quality check, there is no assurance that the provided `treasury_tokens_info` account is indeed the correct associated token account. Thus, an attacker may provide a different token account for `treasury_tokens_info`, thereby redirecting the staked tokens to an account they control instead of the intended treasury account.

## Remediation

Implement a strict quality check to ensure that `treasury_tokens_info` matches the expected associated token account for the treasury. Similarly, do this in the claim and rest instructions.
