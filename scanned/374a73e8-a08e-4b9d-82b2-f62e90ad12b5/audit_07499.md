# Premature Fund Claim

## Summary
This bug report discusses a vulnerability in the "keeper" function that allows users to prematurely claim funds intended for future periods. This is due to an incorrect update of the "NextClaimFrom" field, which tracks the time from which the next claim can be made. The current implementation only adds one period to the previous value, instead of accounting for the actual number of periods that have passed. This allows recipients to exploit the vulnerability by repeatedly calling the function and claiming funds before the intended time. The suggested solution is to update the "NextClaimFrom" field by adding the total elapsed periods to the last claim time. This issue has been resolved in the latest patch.

## Details
## Vulnerability in `keeper::calculateClaimableFunds`

The vulnerability in `keeper::calculateClaimableFunds` arises from the incorrect update of the `NextClaimFrom` field, which tracks the time from which the next claim may be made. This miscalculation allows users to prematurely claim funds that are intended for future periods. In the current implementation, `NextClaimFrom` is updated by adding a single period to the previous `NextClaimFrom` value rather than accounting for the actual number of periods that have passed.

```go
// Location of the function in the codebase
// _x/protocolpool/keeper/keeper.go
func (k Keeper) calculateClaimableFunds(ctx context.Context, recipient sdk.AccAddress, budget types.Budget, currentTime time.Time) (amount sdk.Coin, err error) {
    [...]
    // Update the ClaimedAmount
    claimedAmount := budget.ClaimedAmount.Add(amount)
    budget.ClaimedAmount = &claimedAmount
    // Update the last claim time for the budget
    nextClaimFrom := budget.NextClaimFrom.Add(*budget.Period)
    budget.NextClaimFrom = &nextClaimFrom
    [...]
    return amount, nil
}
```

As a result, the user may claim funds for periods that have not yet elapsed. A recipient may exploit this vulnerability by repeatedly calling the `calculateClaimableFunds` and prematurely claiming funds.

## Remediation

Update `NextClaimFrom` by adding the total elapsed periods (`Period * periodsPassed`) to the last claim time.

## Patch

Resolved in commit `03d70aa`.
