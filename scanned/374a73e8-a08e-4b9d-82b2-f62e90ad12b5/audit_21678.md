# [M01][Fixed] Potential race condition with Repo ownership transfer

## Summary
This bug report is about a potential security vulnerability in the `Repo` ownership transfer process. The current system allows the original owner to call a function called `transferRepoOwnership` and transfer the ownership to a new owner. However, a malicious original owner could front run this transaction with another one that puts the `Repo` in a worse collateralization status, for example, minting more oTokens and removing collateral. This could potentially harm the new owner.

The team suggested solutions such as implementing a time lock and allowing the proposed new owner to accept the ownership, state the risk clearly to users in documentation or remove this function all together. The bug has been fixed in the follow-up commit where the function has been removed.

## Details
Currently a `Repo` owner can transfer the ownership by calling [`transferRepoOwnership`](https://github.com/opynfinance/Convexity-Protocol/blob/c34598565cba2bfcf824eb2da63d95c7f5dda4fa/contracts/OptionsContract.sol#L429). A malicious original owner could front run this transaction with another one that puts the `Repo` in a worse collateralization status, For example, mint more oTokens and remove collateral. This could potentially harm the new owner.


Depending on how the arrangement for the ownership transfer has been done and how important this function is and the risk it presents, we recommend the team to consider solutions accordingly such as: implementing a time lock and allow the proposed new owner to accept the ownership, state the risk clearly to users in documentation or remove this function all together.


**Update**: *Fixed in the follow-up commit [`3adfd9afa6d463869d9e0a78cc7f316ae34eb89e`](https://github.com/opynfinance/Convexity-Protocol/commit/3adfd9afa6d463869d9e0a78cc7f316ae34eb89e) where this function is removed.*
