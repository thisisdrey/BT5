# Unsafe transfer

## Summary
This bug report is about tokens not reverting when an unsuccessful transfer occurs. The report suggests using the OpenZeppelin safeTransfer library instead of a basic transfer method. This library is a set of functions that help ensure the security of the transfer process. It is recommended to use this library to avoid any issues with tokens not reverting back when an unsuccessful transfer occurs.

## Details
##### Description
Some of the tokens do not revert on an unsuccessful transfer. We recommend using the OZ safeTrasnfer method instead of a basic transfer method.
