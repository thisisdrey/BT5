# Transfer of 0 funds

## Summary
The bug report describes an issue with the `withdrawFee` function in the Gearbox protocol. When the `withdrawFee` is set to 0, no assets are being transferred, which is not the intended behavior. The report suggests adding a check to prevent this from happening.

## Details
##### Description
If `withdrawFee == 0`, then here 0 amount of asset transferred:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/pool/PoolService.sol#L191
##### Recommendation
We recommend adding a check, so 0 funds wouldn't be transferred.
