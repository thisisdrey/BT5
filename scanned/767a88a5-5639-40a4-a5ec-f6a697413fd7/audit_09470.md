# Insufficient Price Feed Validation

## Summary
The Hifi protocol relies on Chainlink price feeds, but there is a bug in the `ChainlinkOperator` contract that does not check the freshness of the price. This means that old prices can be used to manipulate the protocol. To fix this, a new state variable `priceStalenessThreshold` has been added and the `getPrice()` function now checks that the returned price is within this threshold. It is recommended to also check the `updatedAt` value to ensure that the price is not too old. This bug has been fixed in commit [4ddb307](https://github.com/hifi-finance/hifi/commit/4ddb307).

## Details
**Update**
Fixed in commit [4ddb307](https://github.com/hifi-finance/hifi/commit/4ddb307) by a new state variable `priceStalenessThreshold` and checking the returned price freshness to be within its value.

**File(s) affected:**`packages/protocol/contracts/oracles/ChainlinkOperator.sol`

**Description:** The Hifi protocol is highly dependent on Chainlink price feeds, which are read from the `ChainlinkOperator` contract. While the `getPrice()` function validates that the retrieved price exceeds zero, it does not validate the staleness of the price. Since an attacker could use the predictability of old prices to manipulate the protocol, it is critical that the protocol validates that all prices are fresh.

**Recommendation:** Validate that the `updatedAt` value returned by `IAggregatorV3.latestRoundData()` indicates a timestamp that is not too old.
