# Missing Sanitization of URLs Passed to AngularJS `bypassSecurityTrustResourceUrl`

## Summary
The client has marked a bug as "Fixed" in the AngularJS framework. This bug involves a security vulnerability that can be exploited by disabling certain security policies. This can lead to cross-site scripting (XSS) attacks, such as the identified self-XSS vulnerability in the HashPack application. The recommendation is to apply a sanitizer or validation mechanism before using the `bypassSecurityTrustResourceUrl` function to only allow 'http://' or 'https://' protocols.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `ff13da4`. The client provided the following explanation: Have implemented global URL sanitization function

**Description:** AngularJS is designed to treat all values as untrusted by default, effectively safeguarding against common cross-site scripting (XSS) injection attacks. However, certain security policies can be violated or disabled, potentially exposing vulnerabilities. For instance, the HashPack application utilizes the `bypassSecurityTrustResourceUrl` security configuration, which disables security mechanisms when handling URLs. Consequently, this creates an opportunity for an attacker to execute an XSS attack.

As an example we identified a self-XSS vulnerability where the primary cause of the security weakness was the enabled `bypassSecurityTrustResourceUrl` policy (see [HAS-4](https://certificate.quantstamp.com/full/hash-pack/95a96750-4624-412c-876e-5965dc021e70/index.html#findings-qs4)).

**Recommendation:** Before passing URLs into the `bypassSecurityTrustResourceUrl` function, apply a sanitizer or validation mechanism that permits only 'http://' or 'https://' protocol schemes.
