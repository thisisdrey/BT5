# [M-01] sellPrincipalToken, buyPrincipalToken, sellUnderlying, buyUnderlying uses pool funds but pays msg.sender

## Summary
This bug report is about the potential for funds to be lost from the marketplace. A group of users (0x52, datapunk, kenzo, kirk-baird, and pashov) have identified that the functions sellPrincipalToken, buyPrincipalToken, sellUnderlying, buyUnderlying do not require permission and use marketplace funds to complete the action, but send the resulting tokens to msg.sender. This means that any address can call these functions and steal the resulting funds.

JTraversa (Illuminate) confirmed the issue but disagreed with the severity, noting that none of these methods would work at all. gzeoneth (judge) decreased the severity to medium, noting that it's unclear how funds would end up in the contract and it doesn't seem to be possible during normal operation.

The recommended mitigation steps for this issue are that all functions should use safetransfer to get funds from msg.sender instead of from the marketplace. This would ensure that only authorized addresses can access the funds.

## Details
_Submitted by 0x52, also found by datapunk, kenzo, kirk-baird, and pashov_

Fund loss from marketplace.

### Proof of Concept

sellPrincipalToken, buyPrincipalToken, sellUnderlying, buyUnderlying are all unpermissioned and use marketplace funds to complete the action but send the resulting tokens to msg.sender. This means that any address can call these functions and steal the resulting funds.

### Recommended Mitigation Steps

All functions should use safetransfer to get funds from msg.sender not from marketplace.
