# Transfer/TransferFrom are used instead of their counterparts from SafeERC20

## Summary
This bug report discusses an issue with the ERC20.transfer(), ERC20.transferFrom(), and ERC20.approve() functions, which are used in Ethereum smart contracts. These functions are supposed to return a boolean value to indicate whether the transfer was successful or not. However, some ERC20 tokens do not follow this specification and may return a false value even if the transfer fails. This can cause problems for the calling contract if it does not check the return value. The report recommends using a solution called SafeERC20 to avoid this issue. The bug has been resolved.

## Details
**Severity**: Medium

**Status**:  Resolved

**Description**

The ERC20.transfer(), ERC20.transferFrom() and ERC20.approve() functions return a boolean value indicating success. Some ERC20 tokens that are not compliant with the specification could return false from the transfer function call to indicate that the transfer fails, but the calling contract would not notice the failure if the return value is not checked.The EIP-20 specification requires checking the return value.

**Recommendation**: 

Consider implementing OpenZeppelin's SafeERC20.
