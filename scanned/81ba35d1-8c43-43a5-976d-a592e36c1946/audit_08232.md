# CRAFTS COOLDOWN TIME ARE ALWAYS ZERO

## Summary
The CraftingSystem contract has a bug where the last completion time is not being updated in the account data mapping. This means that the cooldown value is not being considered, resulting in crafts having no cooldown. The impact of this bug is rated as 2 out of 5, and the likelihood of it occurring is 5 out of 5. The Proof of Play team has partially solved this issue by updating the last completion time every time a recipe is completed. However, the value is being updated even if no crafts are completed successfully, which should not be the case. The recommendation is to only update the last completion time if at least one craft was completed successfully.

## Details
##### Description

In the `CraftingSystem` contract, when a craft is completed, the `lastCompletionTime` in the `_accountData` mapping is not updated. This means that the `cooldown` value is never considered, so all crafts have no cooldown.

##### Score

Impact: 2  
Likelihood: 5

##### Recommendation

**PARTIALLY SOLVED**: The `Proof of Play team` now updates the `lastCompletionTime` value every time `_completeRecipe()` is called. Although, this value will be updated even if no crafts are completed successfully. This value should only be updated if at least one craft was completed successfully.
