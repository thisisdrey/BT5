# [M-04] Attacker can exploit oracle price differences to arbitrage

## Summary
Severity: Medium
Impact: Medium
Likelihood: Medium

Description: The chainlink oracle only updates the price when it reaches a certain threshold, making it vulnerable to arbitrage attacks. Attackers can deposit and withdraw tokens in the same block to take advantage of the price difference and make a profit. This can be seen in the provided Proof of Concept (PoC).

Recommendations: Charge fees for the withdraw function to prevent this type of attack.

Appendix: The PoC shows how an attacker can exploit the price difference between WETH and USDC on the chainlink oracle by depositing and withdrawing tokens in the same block. This allows them to buy WETH at a lower price and sell it on another exchange for a profit.

## Details
**Severity**

**Impact:** Medium

**Likelihood:** Medium

**Description**

The chainlink oracle will submit a new price only when the price fluctuation reaches the deviation threshold. For example, the deviation threshold of ETH/USD is `0.5%`. Attackers can use this to arbitrage. The specific scenario is:

1. Attacker1 deposits some USDC
2. The oracle updates the ETH/USD price to increase by `0.5%`
3. Attacker1 sends all LPs to Attacker2 to bypass `checkLastBlockAction`
4. Attacker2 withdraws all LPs and get ETH + USDC
5. Attacker2 returns the tokens to Attacker1

The above steps can be completed in the same block. The attacker bought ETH at a lower price, and he can sell ETH on another DEX. For more detailed examples, please see PoC.

**Recommendations**

Let the withdraw function charge some fees
