# Guardians Can Prematurely End a Sale

## Summary
The client reported a bug where the `early_purchase::end_sale::handler()` function does not check if the sale end time has passed before allowing guardians to end the sale. This means that guardians can end a sale even before it has officially ended. The client has suggested verifying the sale end time before allowing guardians to end the sale. The bug has been marked as "Fixed" by the client and the fix can be found in `c349ac074b252fd2f1ad52055be09972e4a64163` in the `programs/early-purchase/src/instructions/end_sale.rs` file.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `c349ac074b252fd2f1ad52055be09972e4a64163`. The client provided the following explanation:

> A check that a Sale’s end timestamp has been reached has been included in the end_sale instruction.

**File(s) affected:**`programs/early-purchase/src/instructions/end_sale.rs`

**Description:** The `early_purchase::end_sale::handler()`function is used by a guardian to end a specific sale. However, the this function does not check that the referenced sale end time has passed (`sale.is_end_time_reached()`), allowing guardians to end any sale even before it has officially ended.

**Recommendation:** Consider verifying that the sale end time has passed before allowing guardians to end the sale.
