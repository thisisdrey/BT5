# Anyone can burn tokens

## Summary
The bug report is regarding the `CentrallyIssuedToken` which inherits from `BurnableToken` and provides a function which allows anyone to burn a certain token amount, reducing the `totalSupply`. This is a problem as it allows anyone to burn their tokens, which is not the intended behavior. Storj needs this to burn part of their token holdings according to their crowdsale plans.

It is suggested that the `burn` function should only be callable from a Storj address. It should be noted that any ERC20 token allows owners to burn tokens by transferring them to the zero address or any random one, but this does not modify the `totalSupply`.

The report was edited to indicate that this was intended behavior.

## Details
`CentrallyIssuedToken` inherits from `BurnableToken`, which provides [a function](https://github.com/Storj/storj-contracts/blob/2bdeb27c0216d2f0889b6e7363d8a84b54cd7f39/contracts/BurnableToken.sol) allowing anyone to burn a certain token amount, destroying them and reducing `totalSupply`. Storj needs this to burn part of their token holdings, according to their crowdsale plans, but as a side effect anyone can burn their tokens.


Please make sure this is expected, or consider making the `burn` function only callable from a Storj address. Note that any ERC20 token allows owners to burn tokens by transferring them to the zero address or any random one, but this doesn’t modify the `totalSupply`.


**EDIT**: This was intended behavior.
