# Using floats for financial calculations is dangerous

## Summary
The report highlights potential issues with using floating point numbers in financial applications. These numbers can sometimes have small errors that can lead to unexpected results. The report suggests using a different approach, similar to how it is done in Solidity, where all calculations are done using a specific data type (uint256) and statements involving multiplication by a decimal number are replaced with multiplication and division by integers. This can help avoid errors and ensure more accurate calculations.
