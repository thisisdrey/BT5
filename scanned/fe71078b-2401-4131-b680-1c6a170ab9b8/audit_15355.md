# Incorrect BorrowRate Updating In UpdateStates

## Summary
The bug report is about a function called \spverb|_updateState| in a software called InSpotEngineStates. This function is not working correctly in some cases. The problem is related to the equation used for \spverb|utilRatio > interestInflection|. The equation is causing issues because it is not taking into account certain scenarios. The report suggests that the calculation should be changed to ensure that the interest (borrowRate) increases with the utilization ratio. The bug has been fixed in a patch called #98.

## Details
## InSpotEngineStates's `_updateState` Function

For the `utilRatio > interestInflection`, the equation is acting improperly in certain cases.

**File:** contracts/SpotEngineStates.sol (SOLIDITY)

```solidity
borrowerRateX18 +=
    config.interestSmallCapX18 +
    config.interestLargeCapX18.mul(
        (
            (ONE - utilizationRatioX18).div(
                ONE - config.interestInflectionUtilX18
            )
        )
    );
borrowRate = interestFloor + interestSmallCap + interestLargeCap ×
    1 − utilRatio
    1 − interestInflection
```

Since `utilRatio = borrow / deposit`, it may equal or be greater than one, which will decrease the `borrowRate`. Also, if `utilRatio` equals two or three, the `borrowRate` may become negative, resulting in profit for the borrowers when borrowing money.

## Remediation

Change the calculation to ensure that interest (`borrowRate`) increases with the utilization ratio.

## Patch

The `borrowRate` calculation has been changed. Fixed in #98
