# [H-01] Lack of withdrawal limits check

## Summary
The report states that there is a bug in the code that manages withdrawal limits for a cryptocurrency platform. The bug is classified as medium severity and has a high likelihood of occurring. The update to the code has removed the checks for withdrawal limits, which means that users can bypass these limits and withdraw more than they should. This issue is present in two functions, `AccountCollateral.transferFunds` and `ExposedModule.updateCollateralShares`, as they do not have the necessary checks in place. The report recommends implementing these checks to prevent users from bypassing withdrawal limits.

## Details
## Severity

**Impact:** Medium

**Likelihood:** High

## Description

The provided update fully excludes withdrawal limits checks from the `CollateralConfiguration.checkWithdrawLimits` function and puts them into the `AccountCollateral.updateBalance` function. So `AccountCollateral.transferFunds` and `ExposedModule.updateCollateralShares` functions have neither `GlobalCollateralConfiguration.checkWithdrawLimits` nor `CollateralConfiguration.checkWithdrawLimits` check since they bypass the `updateBalance` function.
