# Possible filling of already filled order

## Summary
This bug report is about a mistake that a user can make when trying to fill an already filled order. The recommendation is to add a check to prevent this mistake. Specifically, the following code should be added: `require(remainingMakerAmount > 0, "LOP: already filled");` This code will ensure that an order cannot be filled if it has already been filled, thus preventing the mistake from occurring. This bug report and recommendation are important for users of the limit order protocol, as it prevents them from making a mistake that could have unintended consequences.

## Details
##### Description
User can try to fill already filled order by mistake:
https://github.com/1inch/limit-order-protocol/blob/a14bde6a260458de5083cee117d734221e1cbc05/contracts/LimitOrderProtocol.sol#L294
##### Recommendation
We recommend to add the following check:
```solidity=
require(remainingMakerAmount > 0, "LOP: already filled");
```
