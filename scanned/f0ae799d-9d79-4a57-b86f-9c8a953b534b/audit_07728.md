# Unstoppable Proposal

## Summary
The client has fixed a bug in their code and provided an explanation for the fix. The bug was found in the `Web3MusicNativeTokenDAO.sol` file and it allowed anyone to create a proposal that would pass the quorum and vote checks, even if no members were whitelisted. This could lead to malicious proposals being executed. The recommended solution is to add checks in two functions to prevent this scenario from happening.

## Details
**Update**
The client fixed the issue in commit `0b6cd1e719d9628cff535ad53da7d4bc5dfc4fd4` and provided the following explanation:

> A check has been added on the _reachedQuorum and _votePassed functions to avoid the scenario described in the audit.

**File(s) affected:**`Web3MusicNativeTokenDAO.sol`

**Description:** When the `whitelistEnabled` flag is set to `true`, but there are no whitelisted members, anyone can create a proposal guaranteed to pass the quorum and vote checks once the voting period has expired.

**Exploit Scenario:**

1.   `Web3MusicNativeTokenDAO` is deployed with `whitelistEnabled` set to `true`, `_quorum` set to 5e8 (50%), and `_majority` set to 5e8 (50%).
2.   No members are whitelisted; therefore, the state variable `_membersNumber` is zero.
3.   A malicious proposal is created with a `maxProposalMembers` of zero since there are no whitelisted members.
4.   The voting period expires without any votes cast.
5.   The proposal quorum is reached as the following is evaluated as `true`.

```
_proposals[proposalId].proposalVoters >= 
    uint256(_quorum).mulDiv(_proposals[proposalId].maxProposalMembers, 10e8)

--> 0 >= (5e8 * 0) / 10e8 --> 0 >= 0 --> true
```

1.   The proposal vote is reached as the following is evaluated as `true`.

```
_proposals[proposalId].votesFor >=
    uint256(_majority).mulDiv(
        (_proposals[proposalId].votesFor +
            _proposals[proposalId].votesAgainst),
        10e8
    )

--> 0 >= (5e8 * (0 + 0)) / 10e8 --> 0 >= 0 --> true
```

1.   The proposal succeeds, and the malicious calls are executed.

**Recommendation:** Add a check in `_reachedQuorum()` that returns `false` if a proposal's `proposalVoters` is zero. Similarly, add a check to `_votePassed()` that returns `false` when the sum of `votesFor` and `votesAgainst` is zero.
