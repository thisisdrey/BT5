# Anyone is able to upgrade implementation of the contract.

## Summary
Report Summary:

The report discusses a bug found in the EXPO.sol and EXPOVO.sol files related to the function_authorizeUpgrade(). This function is necessary for authorizing valid upgrades, but it was not properly checking the msg.sender of the upgradeTo() function. The recommendation is to validate the msg.sender in_authorizeUpgrade() using the onlyOwner modifier from OZ OwnableUpgradeable. The bug has been resolved and additional unit tests have been added to cover this scenario.

## Details
**Description**

EXPO.sol and EXPOVO.sol: function_authorizeUpgrade(). Function_authorizeUpgrade() is necessary in order to authorize that upgrading implementation is valid, thus this function should always be implemented and check that msg.sender of upgradeTo() function is valid(Either owner, admin or authorized user).

**Recommendation**

Validate msg.sender in_authorizeUpgrade(). For example, onlyOwner modifier from OZ OwnableUpgradeable can be used.

**Re-audit comment**

Resolved.

Post-audit:

OnlyOwner modifier is used and scenario was also covered with additional unit-tests.
