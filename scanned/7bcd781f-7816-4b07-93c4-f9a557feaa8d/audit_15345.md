# Incorrect CanSocialize Condition In GetLiquidationStatus

## Summary
The getLiquidationStatus function in the clearingHouse contract has a mistake in its updating condition for the variable canSocialize. According to the documentation, if there is a basis liability for any product, canSocialize should be set to false. However, the code is currently using the opposite condition, causing canSocialize to become false if any product has a basis amount of 0. To fix this, the condition should be changed to use the == operator instead of !=. This issue has been resolved in patch #116.

## Details
## Liquidation Status Function Update

In the `getLiquidationStatus` function of the ClearingHouse contract, the updating condition for `canSocialize` appears incorrect. Based on the documentation comments, if the basis liability exists for any `productId`, then `canSocialize` must be false.

## Current Code
```solidity
canSocialize = canSocialize && (summary.basisAmountX18 != 0);
```

As the condition for updating `canSocialize` appears to be the opposite, it becomes false if any of the product's `basisAmount` is equal to 0.

## Remediation
Change the above condition to use `==` instead of `!=`.

## Updated Code
```solidity
canSocialize = canSocialize && (summary.basisAmountX18 == 0);
```

## Patch
Fixed in #116
