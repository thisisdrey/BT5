# SimplePriceOracle.price is susceptible to integer overflow ✓ Fixed

## Summary
A bug was discovered in the `SimplePriceOracle.price` code of the ethregistrar repository. The code was vulnerable to an overflow attack, where a user could set the duration to `2**256/rentPrice` and get a price of 0. The severity of the bug is assumed to be high, as the code may be used somewhere. The issue was resolved by using `SafeMath` or explicitly checking for the overflow.

## Details
#### Resolution



Issue has been closed in [ensdomains/ethregistrar#17](https://github.com/ensdomains/ethregistrar/pull/17) by using `SafeMath`.


#### Description


`SimplePriceOracle.price` is as follows:


**ethregistrar/contracts/SimplePriceOracle.sol:L26-L28**



```
function price(string calldata /\*name\*/, uint /\*expires\*/, uint duration) external view returns(uint) {
    return duration \* rentPrice;
}

```
This is susceptible to a simple overflow attack, e.g. setting the duration to `2**256/rentPrice` to give yourself a price of 0.


Severity note: It’s unclear whether the `SimplePriceOracle` is expected to be used in practice, but the severity is set here under the assumption that the code may be used somewhere.


#### Remediation


Use `SafeMath` or explicitly check for the overflow.
