# Various NatSpec issues

## Summary
This bug report is about NatSpec documentation in the code. NatSpec is a form of documentation which is part of the contract’s public API, and is therefore assigned a higher severity than other code-comment related issues. The report lists several issues, such as empty tags, incorrect/misleading tags, and missing @return tags. It also suggests to remove empty tags, add @return tags where needed, use @dev tags to describe the intended use of contracts/functions, and otherwise address the issues. The Solidity Documentation provides more information about NatSpec documentation.

## Details
There are various issues related to the NatSpec documentation in the code. We list several of them here as a single issue. We consider the NatSpec documentation to be part of the contract’s public API, so NatSpec-related issues are assigned a higher severity than other code-comment related issues.


* The [@notice and @dev tags](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/InstaRegistry.sol#L7-L8) are empty on the `AddressRegistry` contract.
* The [@notice tag](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/InstaRegistry.sol#L45) is empty on the `LogicRegistry` contract.
* The [@dev tag](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/InstaRegistry.sol#L46) repeats the [@title tag](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/InstaRegistry.sol#L44) on the `LogicRegistry` contract.
* The [@dev tag](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/InstaRegistry.sol#L59) of the `logic` function is empty / does not explain what the `logic` function does.
* The [@dev tag](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/InstaRegistry.sol#L69) of the `logicStatic` function is empty / does not explain what the `logicStatic` function does.
* The [@dev tag of the user wallet `constructor` function](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/UserWallet.sol#L115) is incorrect / misleading.
* The [@dev tag](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/InstaRegistry.sol#L134) of the `record` function contains a misspelling.
* The NatSpec for the [`isAuth` modifier](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/UserWallet.sol#L63) should have a @return tag.
* The NatSpec for the [`execute` function](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/UserWallet.sol#L131) should have a @return tag.


Consider removing empty tags, adding @return tags where needed, using @dev tags to describe the intended use of contracts/functions, and otherwise addressing the above-listed issues. See the [Solidity Documentation](https://solidity.readthedocs.io/en/v0.5.8/natspec-format.html) for more information.
