# Restoration Server result.RestoredBalance nil pointer dereference DoS

## Summary
A bug has been found in the restoration server of the gethclient software. This bug can cause a medium risk issue, as it can lead to a denial of service (DOS) in the restoration server. The bug occurs when the server receives a null json from its gethclient, specifically in the restorationProofResult.RestoredBalance field. This results in a nilpointer dereference panic, which can crash the server. To fix this issue, it is recommended to add a check in the code to ensure that the pointer is not nil before proceeding, and to gracefully handle the error if it is. It is also recommended for all restoration servers to run their own gethclient, but the server currently allows for remote connections to this service.

## Details
## Severity: Medium Risk

## Context
`gethclient/gethclient.go#L159`

## Description
The restoration server is vulnerable to a nil pointer dereference panic (`panic: runtime error: invalid memory address or nil pointer dereference`) if its `gethclient` returns a null `json` for `restorationProofResult.RestoredBalance`. This will cause a Denial of Service (DoS) in the restoration server. While the ideal situation is that every restoration server also runs its own `gethclient`, the restoration server does allow remote connections for this service.

## Recommendation
Add an `if restorationProofResult.RestoredBalance == nil` check in `(ec *Client) GetRestorationProof()` and error gracefully if the pointer is nil.
