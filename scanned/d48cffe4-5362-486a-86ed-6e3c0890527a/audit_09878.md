# The `Trade` function doesn't use `minCollateralAmount`

## Summary
The bug report discusses a function in a code that can be accidentally used and result in loss of money. The issue is caused by a parameter that is always set to zero, leading to incorrect calculations. The recommendation is to add a restriction to prevent this function from being used by anyone other than the Kyber protocol contracts.

## Details
##### Description
[`This function`](https://github.com/Threshold-USD/dev/blob/800c6c19e44628dfda3cecaea6eedcb498bf0bf3/packages/contracts/contracts/B.Protocol/BAMM.sol#L268) can be used by mistake by someone and according to the M-1 it leads to losing money because the `minCollateralAmount` parameter is always zero.
##### Recommendation
We recommend adding a restriction that `trade` is called only by the Kyber protocol contracts.
