# M-5: stakingContext.auraBooster.deposit boolean return value not handled in Boosted3TokenPoolUtils.sol

## Summary
This bug report is about an issue in the Boosted3TokenPoolUtils.sol file in the Notional project. It was found by ctf_sec and is labelled as Issue M-5. The issue is that the function _joinPoolAndStake in Boosted3TokenPoolUtils.sol does not handle the boolean return value from stakingContext.auraBooster.deposit. This value is returned by the AuraBooster implementation when a token is deposited for boosted staking.

The risk of this issue is that if the return value is not handled, the transaction could fail silently. This could have an impact on the Notional -> AuraBooster -> BaseRewardPool chain.

The code snippets associated with this issue can be found in the Leveraged Vaults contracts in the Notional project.

The recommended solution to this issue is to handle the stakingContext.auraBooster.deposit boolean return value explicitly in the code. This can be done by requiring that the staked value is true.

The discussion for this issue has been handled by jeffywu and weitianjie2000.

However, when entering the stake and interacting with external contract, the logic does not handle the returned boolean value in the code below

```solidity
        // Transfer token to Aura protocol for boosted staking
        stakingContext.auraBooster.deposit(stakingContext.auraPoolId, bptMinted, true); // stake = true
```

## Recommendation

We recommend the project handle the stakingContext.auraBooster.deposit boolean return value explicitly.

```solidity
  // Transfer token to Aura protocol for boosted staking
    bool staked = stakingContext.auraBooster.deposit(stakingContext.auraPoolId, bptMinted, true); // stake = true
    require(staked, 'stake failed');
```
