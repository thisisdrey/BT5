# NMKT-7 | Unable To Withdraw Bid

## Summary
This bug report is about a potential issue where a user is unable to withdraw their bid value after receiving an NFT (non-fungible token) named “Alice”. This could happen if the user bids for the name “Alice”, and then receives the NFT named “Alice” through secondary markets or a direct transfer.

The recommendation is to consider whether it is necessary to check for ownership when withdrawing a bid for a name and, if needed, properly document such behavior for users. This will ensure that users understand the process and can make informed decisions about their bids.

## Details
**Description**

There is potential for a user to be unable to withdraw their bid.

Consider the following scenario:

1. Bob bids for the name “Alice”.
2. He then receives the NFT named “Alice” through secondary markets or a direct transfer.
3. Bob is unable to withdraw his bid value because he now owns the NFT named “Alice”.

**Recommendation**

Consider whether it is necessary to check for ownership when withdrawing a bid for a name.
If necessary, properly document such behavior for users.
