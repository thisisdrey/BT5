# Deleverage will revert for markets in closed hours 

## Summary
This bug report is about a problem in the TradeService.sol file on line 626. The issue is with the deleveraging safety mechanism, which is meant to automatically close positions in a certain order. However, if a position is opened in a market that is currently closed, the mechanism will not work properly. The recommendation is to allow the mechanism to work in closed markets. This bug has been fixed in a recent commit and has been verified by the Cantina team.

## Details
## Context
**File:** TradeService.sol  
**Line:** 626  

## Description
The deleveraging safety mechanism is designed to automatically close positions in a specific order, prioritising the most profitable positions first. However, if a position is opened in a market that is currently closed, the deleveraging flow will be reverted.

## Recommendation
Allow deleveraging positions in markets that are in closed hours.
