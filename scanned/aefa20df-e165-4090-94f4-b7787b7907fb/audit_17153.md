# Increment positionNonce in transferFrom in Vault . . . . . .

## Summary
The bug report is about an issue in the Vault contract's transferFrom function, which is used to transfer a trader's profile to another account. The problem is that the profile is moved to a specific "nonce" (a unique identifier), but this nonce is not incremented. This means that if the recipient performs a purchase, the trader's details will be overwritten on that nonce, and if the transferFrom function is performed twice on the same recipient, the first transferred profile will be completely erased. To reproduce the issue, a series of steps are provided. The solution is to increment the nonce in the transferFrom function, which has already been fixed in the code.

## Details
## Vault Contract Transfer Function

In the Vault contract, the `transferFrom` function is used to transfer a trader profile to another account.

## Solidity Code
```solidity
function transferFrom(
    address from,
    address to,
    uint256 nonce
) external nonReentrant returns(bool) {
    require(startTime != 0);
    require(
        allowanceTracker[from][msg.sender][nonce]
        || msg.sender == from
    );
    require(adjustmentsMade[from][nonce] == adjustmentsRequired);
    traderProfile[to][positionNonce[to]] = traderProfile[from][nonce];
    delete traderProfile[from][nonce];
    return true;
}
```

The profile is moved to the `positionNonce[to]` nonce, but this nonce is not incremented. If the recipient performed a purchase, the trader’s details (excluding `ethLocked`) will be overwritten on that nonce.

If `transferFrom` is performed twice on a recipient, then the first transferred profile will be erased completely along with `ethLocked`.

## Proof of Concept
To reproduce the issue, follow the steps below:

1. User A performs a purchase (A0).
2. User B performs a purchase (B0).
3. User B performs another purchase (B1).
4. User A transfers their profile (A0) to User C (C0).
5. User B transfers their profile (B0) to User C (C0).
6. User B transfers their profile (B1) to User C (C0).

Now User C has one profile with nonce (0). The transferred profiles of A0 and B0 are overwritten by the most recent profile, B1.

## Remediation
Increment `positionNonce[to]` in the `transferFrom` function.

## Updated Solidity Code
```solidity
function transferFrom(
    address from,
    address to,
    uint256 nonce
) external nonReentrant returns(bool) {
    require(startTime != 0);
    require(
        allowanceTracker[from][msg.sender][nonce]
        || msg.sender == from
    );
    require(adjustmentsMade[from][nonce] == adjustmentsRequired);
    traderProfile[to][positionNonce[to]] = traderProfile[from][nonce];
    positionNonce[to]++;
    delete traderProfile[from][nonce];
    return true;
}
```

## Patch
Incremented `positionNonce[to]` in the `transferFrom` function. Fixed in commit `473728d`.
