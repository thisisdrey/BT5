# [M-02] Insufficient input validation opens up multiple attack vectors

## Summary
This bug report describes a vulnerability in the `adminTransferFrom` method which does not validate that the `from` argument should not have a value of `address(0)`. This vulnerability has high impact as it can overflow a balance and re-mint burned NFTs. The likelihood of this vulnerability is low as it requires a malicious/compromised owner account or an owner input error. There are three attack vectors that open when `from == address(0)`: re-minting of burned NFTs, underflow of `_balanceOf[from]`, and the 0 ID token (which shouldn't exist) can be "minted" with this method. The recommendation to address this vulnerability is to add a check and assert that the `from` argument is not `address(0)`.

## Details
**Impact:**
High, as it can overflow a balance and re-mint burned NFTs

**Likelihood:**
Low, as it requires a malicious/compromised owner account or an owner input error

**Description**

The `adminTransferFrom` method does not validate that the `from` argument shouldn't have a value of `address(0)`. Now if `from == address(0)` multiple attack vectors open:

1. Burned NFTs can be "re-minted", and also that happens without changing `totalSupply`
2. If `from` balance is zero, this will underflow `_balanceOf[from]`
3. The 0 ID token (which shouldn't exist) can be "minted" with this method

**Recommendations**

Add a check and assert that the `from` argument is not `address(0)`.
