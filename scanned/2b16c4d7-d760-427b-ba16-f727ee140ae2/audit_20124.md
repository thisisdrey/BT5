# Useless check

## Summary
The bug report is about an issue in a file called RUSD.sol, specifically on line 67. The statement "require(_balanceFrom.sub(_amount) >= 0)" is not necessary because it does not serve any purpose. This is because the overflow check will always revert incorrect math, making the statement always true. The recommendation is to verify and correct the functionality, as there will be no helpful error message due to the overflow. The statement should be rephrased to "require(_balanceFrom >= _amount, "Insufficient balance for non network member") to better guard the logic.
