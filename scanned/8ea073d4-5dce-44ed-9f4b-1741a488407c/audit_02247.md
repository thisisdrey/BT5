# M-23: Impartially aborted CCTXs are incorrectly marked as refunded, resulting in a loss of funds

## Summary
The bug report discusses an issue found by multiple users where inbound transactions on ZetaChain may not be refunded to the user if the transaction is aborted. This can result in a loss of funds for the user. The root cause of this issue is a function that marks the transaction as refunded regardless of whether the refund was successful or not. This can happen if the transaction involves a temporarily paused ZRC20 token or if the transaction exceeds the liquidity cap. This issue cannot be manually fixed by a ZetaChain admin. The bug has been fixed by the protocol team in a recent PR.

## Details
Source: https://github.com/sherlock-audit/2025-04-zetachain-cross-chain-judging/issues/320 

## Found by 
0xAlix2, BrothersInCode, berndartmueller

## Summary

Inbound CCTXs for either paused ZRC20 tokens or those that exceed the ZRC20 liquidity cap during the abort process will not be refunded to the user, manifesting as a loss of funds.

## Root Cause

In the case of a CCTX that gets aborted on ZetaChain, the parent caller function `ProcessAbort()` [sets `cctx.CctxStatus.IsAbortRefunded = true`](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/node/x/crosschain/keeper/abort.go#L102), regardless of whether the [abort deposit succeeded or not](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/node/x/fungible/keeper/deposits.go#L203-L234).

For example, if [`getAndCheckZRC20()`](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/node/x/fungible/keeper/deposits.go#L202-L212) errors, due to

- temporarily paused ZRC20 token, or, more likely,
- ZRC20 liquidity cap reached (especially problematic if a user sent an inbound with a large ZRC20 amount, which exceeds the liquidity cap)

the CCTX amount is not refunded.

Additionally, a ZetaChain admin is also unable to manually refund such an aborted CCTX via the `MsgRefundAbort` message [due to the `CctxStatus.IsAbortRefunded` flag being set to `true`](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/node/x/crosschain/keeper/msg_server_refund_aborted_tx.go#L43-L46).

## Internal Pre-conditions

- CCTX needs to be aborted on ZetaChain
- Inbound with a large ZRC20 amount is sent to ZetaChain, which errors during the [ZEVM deposit](https://github.com/sherlock-audit/2025-04-zetachain-cross-chain/blob/main/node/x/crosschain/keeper/evm_deposit.go#L27) and the amount exceeds the liquidity cap

## Mitigation

Ensure that the CCTX is only marked as `IsAbortRefunded = true` if the refund was successful.
