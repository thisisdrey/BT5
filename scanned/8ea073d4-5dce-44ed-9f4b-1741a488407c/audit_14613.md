# Modifier onlyAllowed potentially causes DoS

## Summary
The bug report states that there is a problem with the code in the file "phenomanelTree.sol". The code has a function called "onlyAllowed()" that checks if a contract is allowed to use certain methods. However, the way the code is written can cause a problem because it has to search through a list of allowed contracts every time the function is used. This means that if the list of allowed contracts gets too long, it could cause the program to crash and prevent the function from working. The report recommends using a different method, called a "map", to keep track of allowed contracts instead. This will make the program run more efficiently and prevent any potential crashes. The bug has been fixed and the issue is now resolved.

## Details
**Severity**: High

**Status**: Resolved

**Description**

phenomanelTree.sol - modifier onlyAllowed() is applying an exhaustive search on array elements of allowedContracts . Given that allowedContracts can be appended indefinitely without even an option to remove elements, this can potentially lead to a denial of service on the methods using the modifier.

**Recommendation** 

use a map for allowedContracts
mapping(address => bool) allowedContractsMap;
