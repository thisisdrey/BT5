# Users Would Lose Rewards If A Huge Majority Bets On The Winning Side Due To Rounding

## Summary
This bug report highlights an issue with the UpDown game, where the winning side (down side) is not receiving the correct amount of rewards. The problem lies in the calculation of the rate, which is used to distribute the rewards to the players. Due to rounding, the rate is calculated as 0, resulting in players receiving less tokens than they initially deposited. This issue also affects the setup game. The recommendation is to ensure that the rate is calculated with enough precision to avoid this problem.

## Details
**Severity** - High

**Status** - Resolved

**Description**

totalDepositsDown = 100 tokens

totalDepositsUp = 100000000 tokens

A similar issue would arise for the setup game where rate is calculated via calculateSetupRate().

**Recommendation**:

Ensure there is sufficient precision when calculating the rate.
