# Possible reentrancy

## Summary
This bug report states that there is a possibility of reentrancy occurring in several functions within the gearbox-contracts code. Reentrancy is a type of bug where a function can be called multiple times before the first call is completed, potentially causing unexpected behavior. The report recommends adding a `nonReentrant` modifier to prevent this issue from happening.
