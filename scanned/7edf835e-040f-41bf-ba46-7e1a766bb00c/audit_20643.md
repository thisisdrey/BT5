# Allowed zero `redeemFresh`

## Summary
This bug report is about the `redeemTokensIn` and `redeemAmountIn` functions in the CCollateralCapErc20.sol file on the compound-protocol GitHub repository. The bug is that both of these functions may be zero, which should not be allowed. 

The recommendation is to either append an existing require statement to check if one of the functions is zero, or to quit the function if both are zero. The code for the require statement would be: 
```solidity=
require(redeemTokensIn == 0 && redeemAmountIn > 0 ||
redeemAmountIn == 0 && redeemTokensIn > 0, "one of redeemTokensIn or redeemAmountIn must be zero");
```
The code for quitting the function if both are zero would be:
```solidity=
if (redeemTokensIn == 0 && redeemAmountIn == 0) {
    return uint(Error.NO_ERROR);
}
```
This bug report recommends adding a check to the code of the compound-protocol GitHub repository to ensure that the `redeemTokensIn` and `redeemAmountIn` functions are not both zero.

## Details
##### Description
`redeemTokensIn` and `redeemAmountIn` may be both zero: https://github.com/CreamFi/compound-protocol/blob/23a4ae93adc70334553f5a83429a4e967c1eefaa/contracts/CCollateralCapErc20.sol#L559
