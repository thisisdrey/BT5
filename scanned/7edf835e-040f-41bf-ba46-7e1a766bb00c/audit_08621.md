# State Change Capability In Staticcall

## Summary
This bug report discusses an issue with the staticcall operation in the Ethereum protocol. This operation is intended to only execute code in a read-only manner, but it has been found that some pre-compiled contracts, which are native contracts implemented in the Ethereum protocol, contain methods that can modify the state. This means that executing state-changing actions of these pre-compiled contracts using staticcall bypasses the intended restrictions on state modification. This could lead to unexpected behavior or vulnerabilities if contracts rely on the immutability of staticcall operations for security or correctness. To fix this issue, developers should check against staticcalls before performing state-changing actions of pre-compiled contracts. This bug has been fixed in versions 9205987 and 8e39a8c.

## Details
## Staticcall in Ethereum Contracts

`staticcall` executes code in a read-only manner. However, some precompiled contracts, which are native contracts implemented in the Ethereum protocol, contain methods that may modify the state. Executing state-changing actions of pre-compiled contracts utilizing `staticcall` bypasses the intended restriction on state modification. This may result in unexpected behavior or vulnerabilities if contracts rely on the immutability of `staticcall` operations for security or correctness.

## Remediation

Check against `staticcalls` before performing state-changing actions of pre-compiled contracts.

## Patch

Fixed in `9205987` and `8e39a8c`.
