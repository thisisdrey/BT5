# Wrong stake amount is stored.

## Summary
A bug has been found in the Perpetual TokensFarm contract, specifically in the function_deposit() on line 1083. The issue is that the 'stake.amount' parameter is not correctly storing the amount being deposited. This can result in a larger amount being stored if there is a stake fee. The recommendation is to store the 'stakedAmount' in the 'stake.amount' parameter to resolve this issue. This bug has been resolved.

## Details
**Description**

Perpetual TokensFarm.sol: function_deposit(), line 1083. The_amount parameter is stored in 'stake.amount'. In case there is a stake fee, a greater amount would be stored instead of the value after taking the fee.

**Recommendation**

Store 'stakedAmount` in `stake.amount`.

**Re-audit comment**

Resolved
