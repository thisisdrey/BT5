# Fees applied twice

## Summary
This bug report is about the CurveStableSwapNGViews.vy file in the stableswap-ng repository on GitHub. The fees for a transaction are already accounted for in the `_base_calc_token_amounts` function, so there is no need to account for them again. To fix the issue, it is recommended to remove the fees applying in the `get_dy_underlying` function.

## Details
##### Description
Fees are already accounted for in the `_base_calc_token_amounts`, so there is no need to account them twice. https://github.com/curvefi/stableswap-ng/blob/8c78731ed43c22e6bcdcb5d39b0a7d02f8cb0386/contracts/main/CurveStableSwapNGViews.vy#L184

##### Recommendation
We recommend removing fees applying in the `get_dy_underlying` function.
