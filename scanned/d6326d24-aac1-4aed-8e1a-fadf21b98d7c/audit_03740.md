# _getInvestmentSize() doesn't retrieve the asset balance correctly 

## Summary
A function in ERC4626Router is not retrieving the asset balance correctly, causing issues with investments. The balanceOf() value is in shares and needs to be converted to assets, similar to another function in QuoterV1. The recommendation is to change the code and combine the function with another one. The bug has been fixed in a pull request and verified by Cantina.

## Details
## Issue Report

## Context
- **QuoterV1.sol**: Lines 242-246
- **ERC4626Router.sol**: Lines 179-192

## Description
The function `_getInvestmentSize()` in `ERC4626Router` doesn't retrieve the asset balance correctly. This means investments can't be done effectively. The `balanceOf()` value is in shares and needs to be converted to assets, similar to `QuoterV1::_getErc4626Assets()`.

## Recommendation
Consider changing the code to:
```solidity
uint256 currentBalance = IERC20(component).balanceOf(address(node));
```
to:
```solidity
uint256 currentBalance = IERC4626(component).convertToAssets(IERC20(component).balanceOf(address(node)));
```
Also, consider combining this function with `QuoterV1::_getErc4626Assets()`.
