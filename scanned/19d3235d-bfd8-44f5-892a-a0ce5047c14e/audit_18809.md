# [M-16] Rewards get diluted because `totalAllocPoint` can only increase.

## Summary
This bug report describes an issue with the MasterChef contract from the github repository https://github.com/code-423n4/2022-02-concur/blob/main/contracts/MasterChef.sol. The vulnerability is that there is no functionality for removing pools or setting pool's allocPoints, so the totalAllocPoint only increases and rewards for the pool decreases. 

To demonstrate the vulnerability, the bug report outlines a scenario in which an owner adds a new pool for staking with points of 900 (totalAllocPoint=900). After a week passes, the staking period ends and the owner adds a second pool with points of 100 (totalAllocPoint=1000). Then, Alice stakes 10 tokens in the second pool. After another week passes, Alice claims rewards, but she is only eligible for 10% of the rewards while the other 90% goes to the unused pool.

The bug was discovered through manual review. To mitigate the issue, the report recommends adding functionality for removing pool or functionality for setting pool's totalAllocPoint param.

There is no functionality for removing pools/setting pool's allocPoints. Therefore `totalAllocPoint` only increases and rewards for pool decreases.

### Proof of Concept

Scenario:

1.  Owner adds new pool (first pool) for staking with points = 900 (totalAllocPoint=900).
2.  1 week passes.
3.  First pool staking period ends (or for other reasons that pool is not meaningfully anymore).
4.  Owner adds new pool (second pool) for staking with points = 100 (totalAllocPoint=1000).
5.  1 block later Alice stake 10 tokens there (at the same time).
6.  1 week passes.
7.  After some time Alice claims rewards. But she is eligible only for 10% of the rewards. 90% goes to unused pool.

### Recommended Mitigation Steps

Add functionality for removing pool or functionality for setting pool's `totalAllocPoint` param.
