# Wrongly used `safeApprove`

## Summary
This bug report is about a problem in the code of the BondAppetit Protocol on GitHub. At line 51, the contract is attempting to call the `safeApprove` method, but it fails if the account has remaining allowed tokens. The recommendation is to reset the approval by calling the ERC20 token's `safeApprove` method with the recipient and 0 as parameters before setting a new approval. This should resolve the bug and allow the code to function properly.

## Details
##### Description
At line https://github.com/bondappetit/bondappetit-protocol/blob/88680691fe8d872c5fc26e9500d19cf7caaa9861/contracts/Treasury.sol#L51 contract call `safeApprove` method, however that method fails if account have remaining allowed tokens.

##### Recommendation
We suggest to reset approval calling
```soldity
ERC20(token).safeApprove(recipient, 0);
```
before setting new approval
