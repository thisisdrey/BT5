# [M-06] pendingWithdrawals just increments

## Summary
This bug report is about an issue found in the Withdrawable.sol contract. The code in this contract is used when a user wants to withdraw an amount. The code for the _increaseWithdrawal function adds the amount to the pendingWithdrawals state variable, but the code for the withdraw function does not reduce this variable. This has an impact on the getReserveBalance function, which is used to calculate the number of tokens to burn/mint and to calculate the price of the tokens using the ABC.sol contract. As a result, the pendingWithdrawals variable will only increase, causing incorrect calculations.
