# [M-13] lack of checks in Factory::setBondPercentDiv allow owner to prevent bonding in Auction::bondForRebalance()

## Summary
A bug report has been submitted regarding the `setBondPercentDiv` function in the Auction contract. This function has no checks for minimum and maximum values, which can lead to the `Auction::bondForRebalance()` function being reverted. This can be exploited by the owner to prevent bonding by setting the `bondPercentDiv` to 0.

The recommended mitigation steps are to refactor the code to include checks for minimum and maximum values. This can be done by adding `require` statements to the function, such as `require(newBondPercentDiv > AMOUNT)` and `require(newBondPercentDiv <= AMOUNT_2)`. This will ensure that the `bondPercentDiv` is set to a value within the expected range.

## Vulnerability details

## Impact
`setBondPercentDiv` has no checks for min and max

Setting `bondPercentDiv` to 0 will cause `Auction::bondForRebalance()` to revert

This allows the owner to prevent bonding by setting the `bondPercentDiv` to 0


## Recommended Mitigation Steps
Refactor to
``` 
    function setBondPercentDiv(uint256 newBondPercentDiv) public override onlyOwner {
        require(newBondPercentDiv > AMOUNT);
        require(newBondPercentDiv <= AMOUNT_2);
        bondPercentDiv = newBondPercentDiv;
    }
```
