# Incorrect assumptions about bptindex may lead to incorrect input amounts in _deposittoau- rapool 

## Summary
The report discusses a bug in the Omnipool.sol code that can potentially lead to loss of funds. The issue occurs in the Omnipool._depositToAuraPool function, where there is a missing check for bptIndex < assetIndex. This causes the userDataAmountsIn array to be formatted incorrectly, resulting in wrong amounts being sent to the Pool for deposits. The bug can be demonstrated with an example, where the vulnerable if statement is triggered and the userDataAmountsIn array is formatted incorrectly. The recommendation is to ensure that the assetIndex is the expected one when bptIndex > 0 and bptIndex > assetIndex. This is already being done in the _withdrawFromAuraPool function.

## Details
## Vulnerability Report: Omnipool._depositToAuraPool Function

## Context 
- **Source**: Omnipool.sol#L445-L448, Omnipool.sol#L557

## Description
The `Omnipool._depositToAuraPool` function checks if the assets include BPT by assessing if `_pool.bptIndex > 0`. If this condition is met, it decrements `_pool.assetIndex` by 1 before setting the value. 

The issue arises because there is no validation to ensure that `bptIndex < assetIndex`. In cases where this condition holds true, the `assetIndex` should not be decremented by 1. This flaw causes improper formatting of the `userDataAmountsIn` array prior to executing the `JoinPoolRequest`.

## Impact
The incorrect amounts may be sent to the Pool for deposits, which can potentially lead to a loss of funds for users.

## Proof of Concept
To illustrate the vulnerability, consider the following example:

- `_underlyingAmountIn = 123`
- `_pool.assetIndex = 1`
- `_pool.bptIndex = 2`

- **Before bug**: `userDataAmountsIn = [0, 123, 0]`
- When the vulnerable if statement is triggered, `userDataAmountsIn = [123, 0]`. 

However, the expected format is `[0, 123]` since the BPT amount should follow the asset amount.

## Recommendation
To rectify this issue, ensure that the `assetIndex` is set correctly when `bptIndex > 0` and `bptIndex > assetIndex`. 

This validation is already present in the `_withdrawFromAuraPool` function:

```solidity
// BPT not being in the assets array, we need to adjust the index
if (_pool.bptIndex > 0 && _pool.bptIndex < assetIndex) {
    assetIndex = assetIndex - 1;
}
```
