# [M-08] Unrestricted access in `OmoRouter.transfer()` function

## Summary
The report highlights a bug in the `OmoRouter.transfer()` function, which can potentially lead to token theft. This function allows any user to transfer any amount of vault shares from the router balance to themselves, which could result in loss of tokens if the router has any balance. The severity and likelihood of this bug are both classified as medium. The report recommends restricting the `transfer()` function to the contract owner or manager, or removing it entirely to prevent potential token theft.

## Details
## Severity

**Impact:** Medium

**Likelihood:** Medium

## Description

The `OmoRouter.transfer()` function allows any user to transfer any amount of vault shares from the router balance to themselves, which could lead to token theft if the router has any balance:

```javascript
    function transfer(
        uint256 vaultId,
        address to,
        uint256 amount
    ) external returns (bool) {
        address vault = vaultFactory.getVault(vaultId);
        return IOmoVault(vault).transfer(to, amount);
    }
```

## Recommendations

Restrict the `transfer()` function to the contract owner or manager, or it can be removed.
