# Users can pool SHEEP to reducesheepDogfee 

## Summary
The report is about a bug in the sheepDog contract where the rent for protecting sheep is charged as 10 wGAS per user per day, regardless of the number of sheep deposited. This means that a group of users can pool their sheep together and only pay 10 wGAS per day, making it cheaper for them compared to a single user with a smaller amount of sheep. The recommendation is to charge a fee per sheep per day instead of per user. The bug has been fixed in the code and the fix has been approved.

## Description
Rent for protecting SHEEP by depositing them into the `sheepDog` contract is charged at **10 wGAS** per user per day, regardless of how many SHEEP are deposited. A group of users can pool their sheep in a contract, letting `sheepDog` protect all of them while only paying **10 wGAS** per day for all users combined. This model is also significantly cheaper for a user holding a large amount of sheep compared to one that holds a smaller amount.

## Recommendation
Although this could be regarded as part of the gamification, consider charging a fee per SHEEP per day rather than per user.
