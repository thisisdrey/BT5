# Signature can be reused 

## Summary
The bug report states that there is a problem with the QuailFinance.sol code, specifically in lines 127-128. The issue is that an authenticated address, referred to as A, is able to join multiple times using a signature replay attack. This means that A can use the same signature and nonce (a unique number) to join the same round more than once. This causes two problems: first, it prevents other authenticated addresses from participating, and second, A can potentially enter the round so many times that it becomes the majority participant, guaranteeing a win and taking contributions from other participants. The recommendation is to fix the code so that each authenticated address can only join a round once.

## Details
## QuailFinance Vulnerability Report

## Context
QuailFinance.sol#L127-L128

## Description
An authenticated address (say A) can join multiple times due to a signature replay attack. The nonce used here also comes as an argument, hence can be reused. This has two issues:

- It blocks other authenticated addresses from participating.
- A can enter into the pot so many times that it becomes the majority of participants (up to `numParticipants - 1` times). So with very high probability, it wins the pot, effectively guaranteeing a win and taking other participants' contributions.

## Recommendation
Ensure that each authenticated address joins a round at most once.
