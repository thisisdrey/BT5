# Incorrect usage of the parameter

## Summary
This bug report describes an issue with the `decreaseLimitOrder` function in the LimitOrderManager.sol contract. The issue is that there are not enough checks on the amount of liquidity that can be removed from the limit order. In this case, the liquidity can be greater than the available liquidity on the limit order, which will cause the transaction to revert.

The recommendation to fix this issue is to decrease the `liquidity` to `cache.liquidityLast` if `liquidity > cache.liquidityLast`. This will ensure that the amount of liquidity removed from the limit order does not exceed the available liquidity.
