# [H-01] If a STRATEGY TOKEN is "Toggled off" STRATEGIES will still be able to withdraw, but returning of tokens with `replenishReserves` will be disabled.

## Summary
A bug has been discovered in the code for the Ulysses Omnichain BranchPort.sol contract. The `BranchPort.manage` function allows a registered Strategy to withdraw certain amounts of enabled strategy tokens, but it does not check to see if the token itself is currently enabled. Conversely, the `BranchPort.replenishTokens` function allows a forced withdrawal of managed tokens from a strategy, but it performs a check to see if the token is currently an active strategy token. This means that a strategy token may be disabled by `toggleStrategyToken()` even if there are active strategies managing it actively, and these strategies will still be able to withdraw the tokens with calls to `manage()` while `replenishTokens` will not be callable on them; thus, tokens won't be forced as returnable.

The bug has been confirmed by 0xBugsy (Maia) and 0xLightt (Maia) has commented that it has been addressed. The recommended mitigation steps are to add a check on the enabled strategy token in `manage()` and to validate `getPortStrategyTokenDebt[_strategy][_token] > 0` instead of `!isStrategyToken[_token]` in `replenishReserves()`. This bug has been identified as an Access Control issue.

### Impact

`BranchPort.manage` allows a registered Strategy to withdraw certain amounts of enabled strategy tokens. It validates access rights; i.e. if called by a strategy registered for the requested token. However, it doesn't check to see if the token itself is currently enabled.

Conversely, `BranchPort.replenishTokens` allows a forced withdrawal of managed tokens from a strategy. However, it performs a check to see if the token is currently an active strategy token.

A strategy token may be disabled by `toggleStrategyToken()` even if there are active strategies managing it actively. In such cases, these strategies will still be able to withdraw the tokens with calls to `manage()` while `replenishTokens` will not be callable on them; thus, tokens won't be forced as returnable.

### Recommended Mitigation Steps

1.  Add a check on the enabled strategy token in `manage()`.
2.  Validate `getPortStrategyTokenDebt[_strategy][_token] > 0` instead of `!isStrategyToken[_token]` in `replenishReserves()`.
