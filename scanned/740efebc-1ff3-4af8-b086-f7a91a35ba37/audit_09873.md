# A cross-check of contract parameters

## Summary
The bug report states that if certain contracts, specifically `ActivePool`, `CollSurplusPool`, `DefaultPool`, `StabilityPool`, and `BorrowerOperations`, are deployed with a misconfigured `_collateralAddress` parameter, users will not be able to withdraw their deposited collateral. The report recommends enforcing equality of the `_collateralAddress` parameter in these system contracts to prevent this issue. The specific lines of code that should be changed are also provided for reference.

## Details
##### Description
If contracts `ActivePool`, `CollSurplusPool`, `DefaultPool`, `StabilityPool` and `BorrowerOperations` are deployed with the misconfigured `_collateralAddress` parameter, they will be unable to withdraw the deposited collateral.

##### Recommendation
It is recommended to enforce equality of the `_collateralAddress` parameter in the system contracts:
