# Unrestricted Access Control Rule for Internal Transfers From Objects' Children

## Summary
A bug was found in the `TokenizedVaultFacet.sol` file, specifically in the `internalTransferFromEntity()` function. This function is no longer protected by a security measure that ensures only authorized entity administrators can trigger a transfer of tokens. This means that any address that is a child of an object id can transfer tokens to another internal address, which could cause issues in the current and future versions of the system. It is recommended that a privilege check be added to this function to prevent unauthorized transfers. The bug has been marked as "Fixed" by the client and the fix can be found in the `31bff1268847de2df699b37d4881ee8718e10989` address.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `31bff1268847de2df699b37d4881ee8718e10989`.

**File(s) affected:**`contracts-v3/TokenizedVaultFacet.sol`

**Description:** The function `TokenizedVaultFacet.internalTransferFromEntity()` is no longer protected by an "onlyEntityAdmin" modifier. Since the fact that the parent of `msg.sender` is not checked to be an entity, any address being the child of an object id owning tokens can trigger a transfer of any token to an arbitrary other internal address. This could lead to unexpected consequences in the current version of the system and future ones as well.

**Recommendation:** Consider adding a privilege check to this function.
