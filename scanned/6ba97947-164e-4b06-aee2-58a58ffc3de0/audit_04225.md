# marketData handling in getSwapData and verifyQuote misaligned 

## Summary
The bug report highlights a discrepancy between two functions, runTask and verifyQuote, which are used to process data and validate quotes. While runTask includes marketData as a potential source of data, verifyQuote only considers limitOrderData and dcaData. This can lead to inconsistencies and ambiguity in task handling. The recommendation is to either remove marketData from runTask or update verifyQuote to include validation for it. The bug has been fixed by ensuring that only one type of data is used in running tasks.

## Description
The `runTask` function uses the `getSwapData` and `verifyQuote` functions to process task data and validate quotes. 

In `getSwapData`, `marketData` is included as a potential source of swap data:

```javascript
const data = task?.marketData || task?.limitOrderData || task?.dcaData;
if (!data) throw new Error("Unsupported intent type!");
```

However, in `verifyQuote`, only `limitOrderData` and `dcaData` are considered, while `marketData` is excluded from validation:

```javascript
const data = task?.limitOrderData || task?.dcaData;
if (!data) throw new Error("Could not get task data");
```

This creates a misalignment between `getSwapData`, which supports `marketData`, and `verifyQuote`, which does not. If `marketData` is present, it can proceed through `getSwapData`, but it is neither validated nor handled in `verifyQuote`. This discrepancy introduces ambiguity and could result in inconsistencies in task handling.

## Recommendation
If `marketData` is not intended to be used, its handling should be removed from `getSwapData` to ensure consistency and avoid ambiguity. For example:

```javascript
const data = task?.limitOrderData || task?.dcaData;
if (!data) throw new Error("Unsupported intent type!");
```

Alternatively, if `marketData` is intended to be supported, update `verifyQuote` to include validation for `marketData`:

```javascript
const data = task?.marketData || task?.limitOrderData || task?.dcaData;
if (!data) throw new Error("Could not get task data");
```

Ensuring alignment between `getSwapData` and `verifyQuote` will improve consistency and make the intent of the code clear.
