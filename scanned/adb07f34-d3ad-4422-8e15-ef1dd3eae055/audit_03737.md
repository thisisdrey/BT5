# _enforceLiquidationQueue() checks shares in the wrong location 

## Summary
The bug report is about a function called _enforceLiquidationQueue() in a contract called BaseRouter.sol. This function is supposed to check the amount of shares in the router contract, but it is currently not working because the shares are actually located in a different place called the node. This means that the entire liquidation mechanism is disabled. 

To fix this, the code needs to be changed so that the shares are checked in the correct location. The recommended change is to replace "address(this)" with "address(node)" in the code. It is also mentioned that it is important not to use the same candidate in multiple routers, which is currently not the case. 

This bug has already been fixed in a pull request (PR 241) and has been verified by Cantina Managed.

## Details
## BaseRouter.sol Review

## Context
`BaseRouter.sol#L184-L190`

## Description
The function `_enforceLiquidationQueue()` checks the amount of shares in the router contract. However, the shares are located in the node, which disables the entire `liquidationQueue` mechanism.

## Recommendation
Change the code to:
```solidity
- uint256 candidateShares = IERC20(candidate).balanceOf(address(this));
+ uint256 candidateShares = IERC20(candidate).balanceOf(address(node));
```

## Note
It is important that the same candidate is not used in multiple routers. This is currently not the case.
