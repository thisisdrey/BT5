# [M01] Lack of event emission after sensitive action

## Summary
This bug report concerns the `_initialize` function of the `Governed` contract. The `_initialize` function does not emit the `NewOwnership` event after setting the value of the `governor` to be the `_initGovernor`. This event is important for tracking changes to the contract and notifying off-chain clients of activity. This issue was fixed by a pull request (PR462) and a commit (1714b78d1243a824f36106539b34f8a79ebf14f3). This ensures that the `NewOwnership` event is emitted when the `governor` is set to the `_initGovernor` in the `_initialize` function. This allows for tracking changes to the contract and notifying off-chain clients of the activity.

## Details
The [`_initialize` function](https://github.com/graphprotocol/contracts/blob/d51553b3d70c61108852f0ffe1ed249254c91e9a/contracts/governance/Governed.sol#L31) of the `Governed` contract does not emit the [`NewOwnership` event](https://github.com/graphprotocol/contracts/blob/d51553b3d70c61108852f0ffe1ed249254c91e9a/contracts/governance/Governed.sol#L18) after setting the value of the `governor` to be the `_initGovernor`.


Consider emitting events after sensitive changes occur to facilitate tracking and notify off-chain clients following the contracts’ activity.


**Update:** *Fixed in [PR462](https://github.com/graphprotocol/contracts/pull/463) at commit [`1714b78d1243a824f36106539b34f8a79ebf14f3`](https://github.com/graphprotocol/contracts/pull/463/commits/1714b78d1243a824f36106539b34f8a79ebf14f3).*
