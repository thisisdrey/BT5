# H-1: All funds can be stolen from JOJODealer

## Summary
The report describes a bug in the JOJODealer smart contract where all funds can be stolen by a malicious user. The bug was found by a group of people and it involves the `Funding._withdraw()` function which allows arbitrary calls with user specified parameters. This means that a user can make a call to transfer ERC20 tokens to themselves and steal funds. The impact of this bug is that all funds in the JOJODealer contract can be stolen. The code snippet and tool used for the review are also mentioned. The recommendation is to not make arbitrary calls with user specified parameters. The bug has been fixed in a new pull request and the fix has been validated by another user.

## Summary
`Funding._withdraw()` makes arbitrary call with user specified params. User can for example make ERC20 to himself and steal funds. 

## Vulnerability Detail
User can specify parameters `param` and `to` when withdraws:
```solidity
    function executeWithdraw(address from, address to, bool isInternal, bytes memory param) external nonReentrant {
        Funding.executeWithdraw(state, from, to, isInternal, param);
    }
```

In the end of `_withdraw()` function address `to` is called with that `bytes param`:
```solidity
    function _withdraw(
        Types.State storage state,
        address spender,
        address from,
        address to,
        uint256 primaryAmount,
        uint256 secondaryAmount,
        bool isInternal,
        bytes memory param
    )
        private
    {
        ...

        if (param.length != 0) {
@>          require(Address.isContract(to), "target is not a contract");
            (bool success,) = to.call(param);

                assembly {
                    let ptr := mload(0x40)

```

As an attack vector attacker can execute withdrawal of 1 wei to USDC contract and pass calldata to transfer arbitrary USDC amount to himself via USDC contract.
