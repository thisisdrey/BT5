# Allowing last-minute bidding cancellations will always make the auction settle at the re- serve price 

## Summary
Summary: The bug report discusses an issue with the KimNFTMarketplace protocol that allows users to cancel their bids at the last minute, potentially manipulating the auction process. The report recommends disallowing bid cancellations before the auction closes and implementing a buffer period. The bug has been fixed in a recent update.

## Details
## Auction Cancellation Vulnerability

## Context

- **File**: KimNFTMarketplace.sol
- **Lines**: 252-L264

## Description

The protocol allows users to cancel their bids at the last minute, even at the timestamp `== listingEnd` (when `settleAuction()` is also allowed). 

For English auctions, an attacker could use a high bid to discourage other users from bidding, then cancel their bids at the last minute (or last block time) and place a lower one. Another exploit could involve frontrunning the `settleAuction()` call to cancel their bid at `timestamp == listingEnd`, resulting in the auction concluding as null. 

Since bids cannot be lower than the reserve price, the attacker can either force the English auction to settle at the reserve price or cause it to conclude as null.

## Recommendation

It is recommended to disallow users from cancelling their bids before the auction closes for a certain buffer period.

### KIM Exchange

- Fixed in PR 8.

### Cantina Managed

The new implementation prohibits canceling bids during the `MIN_LENGTH_BID_EXPIRY` period before the auction ends. However, bids made with `BUY_IT_NOW_WITH_BIDS` can still be canceled at any time. It is recommended that cancellation of `BUY_IT_NOW_WITH_BIDS` bids should only be allowed after they expire.

```solidity
uint256 listingEnd = listings[listingId].end;
if (listingEnd > 0 && listings[listingId].listingType != ListingType.BUY_IT_NOW_WITH_BIDS) {
    // Ensure a buffer exists near the end of the auction where cancellations cannot happen,
    // but also gives a window for snipe bids to enter, which is good for sellers.
    bool isEnded = (block.timestamp + MIN_LENGTH_BID_EXPIRY) > listingEnd;
    if (isEnded) revert AuctionEnded();
}
```

### Conclusion

The fix has been implemented as per the recommendation.
