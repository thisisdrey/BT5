# CONTEXT ISOLATION DISABLED

## Summary
The bug report discusses a problem with context isolation in Electron, which is a feature that allows developers to run code in a separate JavaScript context. This feature is important for preventing malicious code from accessing sensitive information. However, if the contextIsolation setting is not used, there is a risk that malicious code can manipulate JavaScript functions and preload scripts. The issue is located in the main.dev.js file and has a high impact and likelihood of occurring.

## Details
##### Description

Context isolation is an Electron feature that allows developers to run code in preload scripts and in Electron APIs in a dedicated JavaScript context. Even when `nodeIntegration: false` is used, to truly enforce strong isolation and prevent the use of Node primitives contextIsolation must also be used.

If contextIsolation is not used, malicious JS code can tamper with JavaScript native functions as well as preload script code via prototype pollution.

Code Location
-------------

#### /wallet-desktop/app/main.dev.js

```
      webPreferences: {
      nodeIntegration: true,
      nodeIntegrationInWorker: true,
      enableRemoteModule: true,
      contextIsolation: false,
      devTools: IS_DEV_QA_MODE,
    },
  });

```
