# Incorrect initialization

## Summary
The bug report is about an issue with a specific line of code in a file called PontoonTokenVesting.sol. The problem is that incorrect parameters are being passed to a function called _addTeam() for the Ecosystem and Community vesting. This means that the timing and number of vestings are not in the correct order. The same issue also applies to other similar functions called _addRound(). 

The recommendation is to fix the order of the parameters and double check that everything is initialized correctly. After discussing the issue with the Pontoon team and verifying the changes, the bug was marked as resolved.

## Details
**Description**

PontoonTokenVesting.sol, line 113, 126, 151, 154, 157, constructor()
Incorrect parameters are passed to the _addTeam() for Ecosystem and for Community -
vesting time and N of vestings are at the wrong places.
The same is applicable for all _addRound() calls.

**Recommendation**:

Correct the order of the parameters and verify the initialization.
Post-audit: marked as resolved after the conversation with Pontoon team and verification.
