# Signature Omits Remaining Amount

## Summary
The bug report addresses an issue with the withdraw function in SyncSpace. The function verifies the admin's signature and then transfers tokens. However, because the final amount is not included in the signature, users can manipulate the remaining amount by passing their preferred final amount along with the signature. The suggested solution is to include the final amount as part of the data signed by the admin to prevent this manipulation. The bug has been fixed in a recent patch.

## Details
## Withdraw Function in SyncSpace

The `withdraw` function in SyncSpace verifies the validity of the admin signature utilizing the `requireAdminSignature` modifier, then transfers tokens. However, since `final_amount` is not included in the signature, users may pass their preferred `final_amount` value along with the admin signature, manipulating the remaining amount.

## SnycSpace.sol (SOLIDITY)

```solidity
function withdraw(
    uint256 id,
    address tokenAddress,
    uint256 amount,
    uint256 finalAmount,
    uint256 nonce,
    SignerData memory sd
)
public
whenNotPaused
requireNonce(nonce)
requireAdminSignature(sd, id, tokenAddress, amount, 1)
```

## Remediation

Include the `final_amount` parameter as part of the data signed by the admin during signature verification to prevent manipulation of the remaining amount.

## Patch

Fixed in commit `7126c8d` by including the `final_amount` parameter in the signed data from the admin.
