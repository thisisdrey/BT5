# Functions may become unexecutable due to potentially huge amount of allTokens array

## Summary
The bug report describes an issue in the Incentives.sol contract where four functions (withdrawAllIncentives, putIncentivesTokens, userState, and availableTokens) have a cycle that may cause problems if the array of tokens becomes too large (more than 500). This can result in high gas fees and hitting gas limits. The recommendation is to add a comment if less than 100 tokens are expected, and to perform additional tests if more than 100 tokens are expected. The partner has responded by commenting that they expect only 10-20 tokens, which is less than the recommended 100, but the issue is still unresolved. The report concludes by stating that the issue has been fixed and a re-audit is recommended.

## Details
**Description**

At contract Incentives.sol there are 4 functions with this cycle: withdrawAllIncentives,
putIncentivesTokens, userState, availableTokens.
If the array of tokens becomes significant in size (more than 500, need to perform additional
tests to get exact array size), first it will take a lot of gas to execute, then it will hit gas limits.

**Recommendation**:

If no significant amount of tokens is expected (less than 100 for example), add a comment
that confirms this. If more than 100, additional tests are required to develop a suitable
solution.
