# [M-10] `UserLock` information can be found during emergency mode

## Summary
This bug report is about the HolyPaladinToken.sol contract. When the contract is in blocked state (emergency mode), the protocol wants to return an empty UserLock info, on calling the function getUserLock. However, there is another way, by which the users can find the same information. This is because the function getUserPastLock is not protected when in emergency mode, and users can use this alternative.

The impact of this bug is that there is no loss of funds, however the intention to block information (return empty lock info) is defeated, because not all functions are protected. There is inconsistency in implementing the emergency mode check.

The recommended mitigation steps are to add checking for emergency mode for this function getUserPastLock, and to add additional user access check, so that the function returns correct value when the caller (msg.sender) is admin or owner.

Kogaroshi (Paladin) confirmed, resolved, and commented on the issue, and instead of reverting the call, they returned an empty Lock (as for getUserLock()). The changes were made in the PaladinFinance/Paladin-Tokenomics#13 pull request.
