# Bptoracle.bptpriceweightpool() tries to use totalsupply() to get the total supply of the pools, resulting in inaccurate bptprice 

## Summary
This bug report discusses an issue with Balancer pools, specifically related to the calculation of the total supply of minted LP tokens. The documentation states that the totalSupply method should only be used for older stable and weighted pools, and that the newer pools should use the getActualSupply method. However, it has been found that some newer weighted pools also use the getActualSupply method, resulting in inaccurate prices and potential losses for users. The recommendation is to check which method is used by the specific pool being interacted with to ensure accurate calculations.

## Details
## Context
- **File Locations:** 
  - BPTOracle.sol#L163
  - Omnipool.sol#L186

## Description
Balancer pools have different methods to get their total supply of minted LP tokens, which is also specified in the docs here.

The docs specify that `totalSupply` is only used for older stable and weighted pools and should not be used without checking it first, since the newer pools have pre-minted BPT and `getActualSupply` should be used in that case. 

Most of the time, the assumption would be that only the new composable stable pools use `getActualSupply`, but that is not the case, as even the newer weighted pools utilize `getActualSupply`.

To provide a few examples of newer weighted pools that use `getActualSupply`, check addresses:
- `0x9f9d...42f298`
- `0x3ff3...c56a2e`
- `0xcf7b...a1d52a` (the last one also being on Aura finance).

Because of that, attempting `BptPriceWeightPool()` on newer weighted pools and future weighted pools would result in inaccurate prices since the wrong total supply value is used when calculating price:

```solidity
// 5. BPT Price (USD) = TVL / totalSupply
uint256 bptPrice = uint256((numerator.toInt().div(totalSupply)));
```

This will cause losses in `Omnipool._withdrawFromAuraPool()`. The `_bptPrice` won't be accurate, and `Omnipool.getPoolTvl()` will also use the wrong `bptPrice`.

## Recommendation
Try to check first if the weighted pool you are interacting with is a newer one and uses `getActualSupply`, or if it is an older one and uses `totalSupply`. This way, the protocol could interact with multiple pools in the long run.
