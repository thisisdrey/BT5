# M-1: Unnecessary precision loss in `_recipientBalance()`

## Summary
This bug report concerns the precision loss in the `_recipientBalance()` function of the Stream.sol file. The `_recipientBalance()` function calculates the vested amount (`balance`) from the `ratePerSecond()` function. The current formula in `_recipientBalance()` incurs an unnecessary precision loss as it includes div before mul. This can be avoided and can save some gas. The precision loss in `_recipientBalance()` can be fixed by changing the formula to `balance = elapsedTime_ * tokenAmount_ / duration`. This was found by WATCHPUG and was fixed by PR: https://github.com/nounsDAO/streamer/pull/7.

## Summary

Using `ratePerSecond()` to calculate the `_recipientBalance()` incurs an unnecessary precision loss.

## Vulnerability Detail

The current formula in `_recipientBalance()` to calculate the vested amount (`balance`) incurs an unnecessary precision loss, as it includes div before mul:

```solidity
balance = elapsedTime_ * (RATE_DECIMALS_MULTIPLIER * tokenAmount_ / duration) / RATE_DECIMALS_MULTIPLIER
```

This can be avoided and the improved formula can also save some gas.

## Impact

Precision loss in `_recipientBalance()`.
