# State transition to “successful” doesn’t check raised amount

## Summary
This bug report is about a state transition from `ICOclosed` to `ICOSuccessful` which doesn't check if the crowdsale succeeded or not. To fix this, the suggestion is to add a line `require(raisedWei &gt;= minRaiseWei)` in the `Milestone_ICOSuccessful` code. Additionally, a similar requirement should be added to `Milestone_ICOFailed` with the line `require(raisedWei &lt; minRaiseWei)`. This report has been fixed as suggested in the `55f9886` commit.
