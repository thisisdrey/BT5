# DefaultOperatorFiltererUpgradeable initializer is not called

## Summary
This bug report is about the *FantiumNFTV1* class which inherits *DefaultOperatorFiltererUpgradeable*. In the `initialize()` method at https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L168, the initializer of *FantiumNFTV1* is not called together with other base classes initializers. This causes the full functionality related to OperatorFilterRegistry to become unavailable.

The recommendation is to add the `DefaultOperatorFilterer_init()` call to the *FantiumNFTV1* initializer. This will ensure that all the base classes initializers are called together and the full functionality related to OperatorFilterRegistry is available.

## Details
##### Description
*FantiumNFTV1* class inherits *DefaultOperatorFiltererUpgradeable* but it's initializer is not called together with other base classes initializers in the `initialize()` method at https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L168.
Due to this, the full functionality related to OperatorFilterRegistry becomes unavailable.
##### Recommendation
We recommend adding the `DefaultOperatorFilterer_init()` call to *FantiumNFTV1* initializer.
