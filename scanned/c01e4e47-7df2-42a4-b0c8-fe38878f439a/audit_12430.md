# Integrity Check Bypass

## Summary
The Gramine manifest file has a bug that allows critical files in the /etc directory to be accessed and manipulated by an attacker. This is because the manifest file includes the /etc directory in the allowed_files list, but also has critical files in the trusted_files list that reside in /etc. This means that the integrity of these files is not being verified, which can lead to security vulnerabilities. To fix this issue, the /etc directory should be removed from the allowed_files list.

## Details
## Gramine Manifest File Security

Gramine’s manifest file allows the filtering of files or directories readable by the enclave program via the `allowed_files` field. Additionally, it ensures the integrity of files through the `trusted_files` field. Any files in this list are hashed at compile time, and the hash will be later verified when the program attempts to load the file.

## Issue with quote-verification-oracle

The `quote-verification-oracle`’s Gramine manifest has the `/etc` directory in the `allowed_files` list; however, it also contains critical files in `trusted_files` that reside in `/etc`.

```
quote-verification-oracle/configs/qvn.manifest.template 
sgx.trusted_files = [
  [...]
  "file:/etc/ssl/certs/",
  [...]
]
sgx.allowed_files = [
  "file:/etc",
  [...]
]
```

Consequently, all `/etc` files or subdirectories in `trusted_files` will not have their hashes verified. This affects SSL certificate files in the `/etc/ssl/certs` subdirectory, allowing an attacker to manipulate network data protected by TLS encryption.

## Remediation

- Remove the `/etc` directory from `allowed_files`.
