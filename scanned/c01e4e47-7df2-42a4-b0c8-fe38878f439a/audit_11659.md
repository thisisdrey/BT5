# Farming can be deactivated by accident

## Summary
A bug has been found in the current check for farming deactivation. This bug can be triggered by accident if the `tick` hasn't changed during the swap and `zeroToOne` is false. If this bug is triggered, it will disconnect farming from the pool and have a negative effect on incentive distribution for users. To fix this bug, it is recommended to use a different check before setting the `deactive` state to the virtual pool.

##### Description
The current check for farming deactivation can be triggered by accident if `tick` hasn't changed during the swap and `zeroToOne` is false https://github.com/cryptoalgebra/Algebra/blob/6f57b3e218630106a4d41aedefd38f9e83b41e2b/src/farming/contracts/farmings/EternalVirtualPool.sol#L131-L134. This action will disconnect farming from the pool and will affect incentive distribution for users.
