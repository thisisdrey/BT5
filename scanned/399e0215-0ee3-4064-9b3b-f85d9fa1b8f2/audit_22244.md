# Avoid duplicate code

## Summary
This bug report addresses the issue of code repetition in the contracts `Token`, `StandardToken`, and `SafeMath`. Such code repetition can lead to unexpected bugs and should be avoided. The bug report suggests using the standard modules from OpenZeppelin, which is a library of reusable smart contracts for Ethereum. Additionally, the bug report suggests extracting repeated conditional checks into reusable internal functions with declarative names. This would make the code more readable and easier to understand.

## Details
The contracts `Token`, `StandardToken`, and `SafeMath` are very similar to OpenZeppelin library contracts `ERC20Token`, `StandardToken`, and `SafeMath`, respectively. Consider avoiding code repetition, which can bring regression problems and [introduce unexpected bugs](https://medium.com/zeppelin-blog/hackergold-bug-analysis-68d893cad738). Consider using the standard modules from [OpenZeppelin](https://github.com/OpenZeppelin/zeppelin-solidity).


Several conditional checks, such as `block.number &lt; fundingEndBlock`, are repeated throughout the code ([line 51](https://github.com/brave-intl/basic-attention-token-crowdsale/blob/17a5f8440a256a6dc5d8dd894b9615182c2901b2/contracts/BAToken.sol#L51), [line 68](https://github.com/brave-intl/basic-attention-token-crowdsale/blob/17a5f8440a256a6dc5d8dd894b9615182c2901b2/contracts/BAToken.sol#L68), [line 79](https://github.com/brave-intl/basic-attention-token-crowdsale/blob/17a5f8440a256a6dc5d8dd894b9615182c2901b2/contracts/BAToken.sol#L79)). Consider extracting these to reusable internal functions with declarative names (for example `isWithinFundingPeriod`) for better code readability.
