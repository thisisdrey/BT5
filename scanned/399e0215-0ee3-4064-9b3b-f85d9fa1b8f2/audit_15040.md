# potentialDebt is not compared against a new lien's maxPotentialDebt in_appendStack

## Summary
This bug report discusses an issue in the LienToken.sol code. Specifically, the code is not performing a comparison between newSlot.lien.details.maxPotentialDebt and potentialDebt when stack.length is 0. This means that the input params is not being fully checked for misconfiguration.

The recommendation is to make sure to perform the check in either _createLien or _appendStack by removing the stack.length > 0 condition. This would ensure that the comparison between newSlot.lien.details.maxPotentialDebt and potentialDebt is always performed, regardless of the stack.length value. Astaria and Spearbit have acknowledged the recommendation.

## Details
## Severity: Medium Risk

## Context
`LienToken.sol#L435-L439`

## Description
In `_appendStack`, we have the following block:

```solidity
newStack = new Stack[](stack.length + 1);
newStack[stack.length] = newSlot;
uint256 potentialDebt = _getOwed(newSlot, newSlot.point.end);
...
if (
    stack.length > 0 && potentialDebt > newSlot.lien.details.maxPotentialDebt
) {
    revert InvalidState(InvalidStates.DEBT_LIMIT);
}
```

Note, we are only performing a comparison between `newSlot.lien.details.maxPotentialDebt` and `potentialDebt` when `stack.length > 0`. If `_createLien` is called with `params.stack.length == 0`, we would not perform this check and thus the input params are not fully checked for misconfiguration.

## Recommendation
Make sure to perform this check in either `_createLien` or here in `_appendStack` by removing the `stack.length > 0` condition:

```solidity
// `potentialDebt` needs to be calculated in `_createLien`
if (potentialDebt > params.lien.details.maxPotentialDebt) {
    revert InvalidState(InvalidStates.DEBT_LIMIT);
}
```
