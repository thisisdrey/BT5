# [M-06] `StakingRewards.sol` `recoverERC20()` can be used as a backdoor by the `owner` to retrieve `rewardsToken`

## Summary
This bug report concerns a vulnerability in the code for the StakingRewards smart contract. The vulnerability allows users to lose all of their rewards to a malicious or compromised owner. The code snippet provided shows the current code that is vulnerable to this exploit. The recommendation is to change the code to a new snippet that requires the token address to be different from both the staking token and rewards token. This would prevent the malicious owner from accessing the rewards.

## Details
_Submitted by WatchPug, also found by cmichel_

[StakingRewards.sol#L166-L176](https://github.com/code-423n4/2022-02-concur/blob/72b5216bfeaa7c52983060ebfc56e72e0aa8e3b0/contracts/StakingRewards.sol#L166-L176)<br>

```solidity
    function recoverERC20(address tokenAddress, uint256 tokenAmount)
        external
        onlyOwner
    {
        require(
            tokenAddress != address(stakingToken),
            "Cannot withdraw the staking token"
        );
        IERC20(tokenAddress).safeTransfer(owner(), tokenAmount);
        emit Recovered(tokenAddress, tokenAmount);
    }
```

### Impact

Users can lose all the rewards to the malicious/compromised `owner`.

### Recommended Mitigation Steps

Change to:

```solidity
 function recoverERC20(
    address tokenAddress,
    address to,
    uint256 amount
) external onlyOwner {
    require(tokenAddress != address(stakingToken) && tokenAddress != address(rewardsToken), "20");

    IERC20(tokenAddress).safeTransfer(to, amount);
    emit Recovered(tokenAddress, to, amount);
}
```
