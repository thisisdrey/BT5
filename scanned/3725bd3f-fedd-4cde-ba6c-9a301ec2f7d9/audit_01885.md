# Non whitelisted caller can queue withdrawal

## Summary
This bug report discusses a medium risk issue found in the StakingManager contracts within the system. These contracts are used by institutions, and only whitelisted addresses should be able to stake and withdraw from them. However, the `_whitelist` variable is only checked in the `stake()` function and not in the `queueWithdrawal()` function, allowing non-whitelisted users to interact with the StakingManager. The recommendation is to implement the check in the `queueWithdrawal()` function to prevent unauthorized access. The issue has been fixed in the code by implementing the recommendation.

## Description
There will be several StakingManager contracts in the system as some of them will be deployed for institutions. Only whitelisted addresses should be allowed to stake and withdraw from the institution's Staking-Manager. It's possible to configure whitelisted addresses through the `_whitelist` variable. The problem is that it's only checked in the `stake()` function that the caller is whitelisted, but there is no same check in the `queueWithdrawal()`. Thus, a non-allowed actor can interact with the StakingManager of the institution.

## Recommendation
It's recommended to check that the caller is whitelisted in the `queueWithdrawal()` function.
