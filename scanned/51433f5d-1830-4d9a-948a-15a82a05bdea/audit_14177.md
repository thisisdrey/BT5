# [M-04] Users won't be able to claim vested tokens when contract is paused

## Summary
This bug report describes a potential attack that can be executed by the owner of a contract. The attack involves the owner calling the `setPaused` function with `paused == true`, which pauses the contract and prevents users from claiming vested tokens via the `releaseAvailableTokensForHolder` function. This is a centralization issue, meaning that the contract owner can "rug" users by preventing them from claiming their tokens. The likelihood of this attack is low, as it requires malicious or a compromised owner. The impact of this attack is high, as it gives the owner the power to prevent users from claiming vested tokens. The recommendation for this bug is to remove the `whenNotPaused` modifier from `releaseAvailableTokensForHolder`, so that users can claim their vested tokens even if the contract is paused by the owner.

## Details
**Impact:**
High, as owner has the power to make it so that users can't claim any vested tokens

**Likelihood:**
Low, as it requires a malicious or a compromised owner

**Description**

The owner can currently execute the following attack:

1. Call `setPaused` with `paused == true`, so pause the contract
2. Now all user calls to `releaseAvailableTokensForHolder` will fail, since it has the `whenNotPaused` modifier
3. He can not unpause the contract forever or even renounce ownership

This is a common centralization problem which means the contract owner can "rug" users.

**Recommendations**

Remove the `whenNotPaused` modifier from `releaseAvailableTokensForHolder`, so users can claim vested tokens even if admin pauses the contract.
