# NMKT-4 | Inaccurate Event Data

## Summary
This report is about an issue with the event not emitting the token name that has been transferred. The `string memory name = toLower(nftr.tokenName(collectionFrom, tokenFrom))` is performed after the transfer is already made, which is causing the event to emit the empty string instead of the token name.

In order to fix this issue, it is recommended to either grab the name from the `toToken` or cache the name prior to transfer. This should ensure that the event emits the correct token name.

## Details
**Description**

`string memory name = toLower(nftr.tokenName(collectionFrom, tokenFrom))` is performed after the transfer is already made. As a result, the event doesn’t emit the name that has been transferred but the empty string.

**Recommendation**

Grab the name from the `toToken` or cache the name prior to transfer.
