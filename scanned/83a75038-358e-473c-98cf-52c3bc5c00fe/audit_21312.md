# Code does not match

## Summary
This bug report is about an undefined behavior in the app-vault.teal file. The current development practice involves copying code from string variables in vault.js into the vault.teal.tmpl file, which is error-prone as it requires manual copying and pasting of the TEAL code. This code needs to be copied into the vault.teal.tmpl file, but if this is not done, it results in a mismatch between the two files, including omission of an important Rekey check.

The exploit scenario is that Bob rewrites the deployment script, but the new script opens vault.teal.tmpl which was not updated, so the issues remain. Eve exploits one of the vault’s flaws to drain all the funds.

The recommendations to address this issue are to maintain one version of the vault contract and use JavaScript to parse the file, to ensure that a single standard codebase is updated. Additionally, it is recommended to minimize the amount of manual copying and pasting required in order to run the tests, by listing which hardcoded parameters must be updated, and either use a template or additional checks to keep them up to date.

## Details
## Type: Undefined Behavior
**Target:** app-vault.teal  

**Difficulty:** High  

## Description
The current development practice involves copying code from string variables in `vault.js` into the `vault.teal.tmpl` file. This practice is error-prone, as it requires manual copying and pasting of the TEAL code.

Figure 7.1 shows the string variable `vaultTEAL`, which represents the code for the vault:

```javascript
var vaultTEAL =
`#pragma version 2
addr TMPL_USER_ADDRESS
pop
gtxn 0 ApplicationID
int TMPL_APP_ID
==
// do not allow to call the App from the Vault
txn GroupIndex
int 0
!=
&&
gtxn 0 Accounts 1
txn Sender
==
&&
txn RekeyTo
global ZeroAddress
==
&&
`
```

**Figure 7.1:** vaultTEAL variable in `vault.js#L35-L60`.  

However, this code needs to be copied into the `vault.teal.tmpl` file. Failure to do so results in a mismatch between the two files, including omission of an important Rekey check:

```teal
#pragma version 2
addr TMPL_USER_ADDRESS
pop
gtxn 0 ApplicationID
int TMPL_APP_ID
==
// do not allow to call the App from the Vault
txn GroupIndex
int 0
!=
&&
gtxn 0 Accounts 1
txn Sender
==
&&
```

**Figure 7.2:** `vault.teal.tmpl`.  

## Exploit Scenario
Bob rewrites the deployment script. The new script opens `vault.teal.tmpl`. Because `vault.teal.tmpl` was not updated, the issues shown in the report are still present. Eve exploits one of the vault’s flaws to drain all the funds.

## Recommendations
- **Short term:** Maintain one version of the vault contract and use JavaScript to parse the file. This ensures that a single standard codebase is updated.
  
- **Long term:** Minimize the amount of manual copying and pasting required in order to run the tests. List which hardcoded parameters must be updated, and either use a template or additional checks to keep them up to date.
