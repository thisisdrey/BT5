# Reward durations of reward tokens in existing vaults can never be updated 

## Summary
The bug report is about a missing feature in the InfraredVault smart contract. The contract has a function called `updateRewardsDuration()` but there is no way to call it from the Infrared contract. This means that the reward durations of tokens in existing vaults cannot be updated, which will impact the value captured by the vaults. The likelihood of this feature being implemented is medium and the recommended solution is to add a function that allows for fine-grained control over setting reward durations for specific vaults or all reward tokens of a specific vault.

## Details
## Infrared Vault Analysis

## Context
- **Files**: InfraredVault.sol#L128-L131, Infrared.sol#L276-L288

## Description
While `InfraredVault.updateRewardsDuration()` has an `onlyInfrared` modifier, `Infrared` does not have any function to call it. Therefore, `Infrared` is missing wrapper-function logic to call `updateRewardsDuration()` on any of its existing vaults to update the rewards duration of their reward tokens. Whenever `Infrared` governance updates `rewardsDuration` via `updateRewardsDuration()`, this will only apply to reward tokens of new vaults created or new reward tokens sent.

## Impact
**Medium**: The reward durations of reward tokens in existing vaults can never be updated. This limitation will prevent `Infrared` governance from effectively adjusting the reward rates of existing vaults to enforce fine-grained control over their incentives. Given that enabling incentive flywheels is a primary objective of `Infrared`, this missing feature will impact the value captured by its vaults.

## Likelihood
**Medium**: Updating reward durations of tokens in existing vaults is anticipated to be implemented in `Infrared` vault's `updateRewardsDuration()` (which has an `onlyInfrared` modifier) and `setRewardsDuration()` logic, as communicated by the project team.

## Recommendation
Given that this is an `Infrared` governance-driven action, one option is to enable fine granularity for setting reward durations of tokens in existing `Infrared` vaults, for example:
- For a specific vault+token combination: 
  ```solidity
  function updateRewardsDurationForVaultToken(address _vault, address _rewardsToken, uint256 _rewardsDuration)
  ```
- For all reward tokens of a specific vault: 
  ```solidity
  function updateRewardsDurationForVault(address _vault, uint256 _rewardsDuration)
  ```
