# Genesis config checks

## Summary
The reported bug occurs in a specific part of the code called "Basilisk-node/pallets/asset-registry/src/lib.rs" and can cause storage corruption if there is a mistake in the genesis configuration. The recommended solution is to use a function called "assert!()" to check for consistency in three specific values. This will help prevent the bug from happening.

## Details
**Occurs**:

Basilisk-node/pallets/asset-registry/src/lib.rs:94-98
Absence of genesis config consistency can lead to storage corruption in case when a mistake
was made in genesis config.

**Recommendation**:

Use assert!() to check that CoreAssetId, NextAssetId, AssetIds are consistent.
