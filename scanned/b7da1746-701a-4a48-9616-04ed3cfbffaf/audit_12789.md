# Missing signextend when dealing with non-full word signed integers

## Summary
This bug report is about a medium risk issue in the AssemblyLib.sol code on line 83. It is related to the use of non-full word signed integers operations, which may result in the code reverting. The Solidity compiler can help with this issue, but it is not guaranteed. The team decided to remove the Assembly code and use Solidity instead, as it was more cost effective in the end. The issue was addressed in PR 319 and has been fixed.

## Details
## Security Report

## Severity
**Medium Risk**

## Context
AssemblyLib.sol#L83

## Description
The AssemblyLib is using non-full word signed integers operations. If the signed data in the stack has not been sign-extended, the two’s complement arithmetic will not work properly, most probably reverting. The Solidity compiler does this cleanup, but this cleanup is not guaranteed to be done while using the inline assembly.

## Recommendation
Consider sign-extending the data before it is used, e.g. `delta := signextend(15, delta)` or just use Solidity instead of inline assembly.

## Primitive
The team decided to remove the Assembly code and simply use Solidity. A lot of different cases were possible, and covering all of them in Assembly was eventually costing more in the end. Addressed in PR 319.

## Spearbit
Fixed.
