# Bypass Pool Ratio Check

## Summary
The report states that there is a bug in the protocol that allows users to manipulate the pool ratio by quickly depositing and then withdrawing assets. This can result in an elevated ratio that enables the user to perform a swap. To fix this issue, a time lock or cool-down period should be implemented after a deposit to prevent users from depositing and withdrawing within a short time frame. This bug has been fixed in version 46201b2.

## Details
## Liquidity Provider Protocol

The protocol allows liquidity providers to execute deposits and withdraw transactions in quick succession. This enables a user to manipulate the pool ratio by executing a deposit to change the ratio and immediately following it with a withdrawal.

## Example Scenario

1. A user deposits a significant amount of assets into pool A, thereby increasing the ratio of the pair A/B.
2. Immediately after the deposit, the user initiates a swap. The elevated ratio resulting from the recent deposit enables the swap to occur.
3. Following the swap, the user quickly withdraws their assets from the pool.

## Remediation

Implement a time lock or cool-down period after a deposit to prevent users from depositing and withdrawing within a short time frame.

## Patch

Fixed in `46201b2`.
