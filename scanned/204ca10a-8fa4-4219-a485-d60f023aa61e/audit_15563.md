# Transaction validation is out of the scope

## Summary
The bug report is about a medium severity issue in a file called RailgunLogic.sol. The specific function that is affected is called validateTransaction. The problem is that the current implementation of this function does not properly check for a correct value in the unshieldPreimage. This means that anyone can call the function using their own address and receive all the tokens. The recommendation is to make sure that the Verifier.verify function is audited and does all the necessary checks and validations. This bug has been resolved.

## Details
**Severity**: Medium

**Status**: Resolved

**File**: RailgunLogic.sol

**Function**: validateTransaction

**Details**:

The current implementation of the in-scope functions does not validate unshieldPreimage to contain a correct value. Without such validation, anyone could call a function providing its own address and get all tokens.

**Recommendation**: 

make sure Verifier.verify function is audited and does all necessary verifications and validations."
