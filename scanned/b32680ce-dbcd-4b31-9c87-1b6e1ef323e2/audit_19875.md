# Missed interface import at SafeERC20.sol

## Summary
The bug report states that the SafeERC20.sol contract has an import statement that is not included in the repository. This import is necessary for the contract to function properly. The recommendation is to either add the missing interface to the contracts folder or remove the contract if it is not needed. This will ensure that the contract can be compiled correctly.

## Details
**Description**

Contract SafeERC20.sol has import import "../interfaces/IERC20Burnable.sol" which not
included in repo. In all other contracts where needed SafeErc20 you use it from openzeppelin
library.

**Recommendation**:

If needed add an interface to the contracts folder, so the contract can be compiled or remove
the contract if you don’t need to use it.
