# [M-09] Debug Logging Leaks Sensitive Data in Production

## Summary
The bug report describes a problem where sensitive information is being leaked in production logs. This is a medium risk issue, meaning it could potentially cause problems but is not immediately critical. The report identifies two specific locations where this is happening: `src/middleware/auth.ts` and `src/infrastructure/database/mongodb-client.ts`. In the first location, the API key type result and allowed types are being logged on every authentication check. In the second location, the full MongoDB connection URI is being logged, which can contain credentials.

The report gives an example of how this vulnerability could be exploited, with a compromised log aggregator, a developer with read-only log access, or a log export to a third-party SIEM being able to expose database credentials and internal auth configuration. This could allow an attacker to connect directly to the database.

The report recommends removing all `console.log()` statements from `auth.ts`, redacting credentials from the MongoDB URI before logging, and auditing all `logInfo()`/`logError()` calls for sensitive data. The team has already fixed the issue.

## Details
## Severity

Medium Risk

## Description

Multiple locations contain console.log statements that leak sensitive information in production logs. Most critically, `src/middleware/auth.ts` logs the API key type result and allowed types on every authentication check, and `src/infrastructure/database/mongodb-client.ts` logs the full MongoDB connection URI (which typically contains credentials).

 **Vulnerable Scenario:**
CloudWatch/Datadog logs and sees:

```bash
INFO: Connected to MongoDB { uri: "mongodb://admin:SuperSecretP4ss@10.0.1.50:27017/crush-wallet" }
INFO: result TokenRadar
INFO: allowedTypes ["TokenRadar", "ATS"]
```

A compromised log aggregator, a developer with read-only log access, or a log export to a third-party SIEM now exposes database credentials and internal auth configuration. An attacker uses the MongoDB URI to connect directly.

## Affected Files

`src/middleware/auth.ts`

## Impact

- Log aggregation systems contain MongoDB credentials, API key metadata, and internal configuration details.

## Recommendations

- Remove all `console.log()` debug statements from `auth.ts`
- Redact credentials from MongoDB URI before logging (only log the host/db name)
- Audit all `logInfo()`/`logError()` calls for sensitive data in their context objects

## Team Response

Fixed.
