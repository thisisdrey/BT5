# Lack of Signature Dialog Context and RPC Origin ✓ Fixed

## Summary
A bug was found in the FilSnap signing dialog that caused it to not indicate which user account was used for signing. This could lead to users being misled into providing unintended signatures, as they may think they are signing with one account, when in fact, another is being used. To address this issue, two commits were made to the FilSnap repository, which display the user account used for signing and the origin site that triggered the process. Furthermore, details such as the RPC URL and the network name are displayed. This ensures users have full context before confirming any transactions.

## Details
#### Resolution



The signature UI dialogue now displays the user account used for signing and the origin site that triggered the process. Furthermore, details such as the RPC URL and the network name are displayed. This issue has been fully addressed in [`1a8715f42cfc9f721e8faab8a7a2610f53592f94`](https://github.com/filecoin-project/filsnap/commit/1a8715f42cfc9f721e8faab8a7a2610f53592f94) and [`6ddb227738ed3aa041c18131eee65a98e17acdf4`](https://github.com/filecoin-project/filsnap/commit/6ddb227738ed3aa041c18131eee65a98e17acdf4).


#### Description


The FilSnap signing dialog doesn’t indicate which user account is used for signing. This omission can be exploited by malicious dapps, leading users to believe they’re signing with one account when in fact, another is being used. Such transparency gaps, especially in an environment integrating multiple dapps and accounts, can mislead users into providing unintended signatures.


For reference, MetaMask displays these details during its signing requests, a practice FilSnap should adopt.

#### Recommendation


Display the user account used for signing and the origin domain of the RPC call during the signing dialog, ensuring users have full context before confirming any transactions.
