# Users' Native Tokens Can Get Stuck

## Summary
The client has marked a bug in the code as "fixed". The affected file is BreederTimeLockPool.sol. The bug occurs in the function crossChainClaimReward(), where it accepts more native tokens than necessary for bridging. This results in the excess tokens being stuck in the contract. The recommendation is to include a refund mechanism to address this issue.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `01de7717898b17b22fd13167b09b91d7c3f44040`.

A refund mechanism was included.

**File(s) affected:**`BreederTimeLockPool.sol`

**Description:**`BreederTimeLockPool.crossChainClaimReward()` will accept native tokens that exceed the `nativeFee` estimated by the bridge contract. In the case where a user sends more native tokens than needed to cover the bridging, their funds will remain in the contract.

**Recommendation:** Include a refund mechanism to deal with the case where users may send more native token than needed.
