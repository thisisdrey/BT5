# Incorrect Token Amounts Used in add_liquidity()

## Summary
A bug was found in the add_liquidity() function on line 108 of the code, which incorrectly references the amount of token1 to transfer to the gateway for further processing. This could result in incorrect amounts being transferred out of the borrower's address and may cause reverts due to insufficient balance or exceeding approved transfer amount. To fix this, the development team changed token1 amount on line 108 from amounts[0] to amounts[1] in commit 5d506f0. This resolved the issue.

## Details
## Description

Incorrect amount of `token1` is used to transfer funds to the `CurveV1StETHPoolGateway`, resulting in unexpected behaviour. The `add_liquidity()` function on line 108 incorrectly references the amount of `token1` to transfer to the gateway for further processing. This will result in an incorrect amount being transferred out of the borrower’s address and may also cause reverts due to insufficient balance or exceeding the approved transfer amount.

## Affected Code:
```solidity
function add_liquidity(
    uint256[N_COINS] memory amounts,
    uint256 min_mint_amount
) external {
    if (amounts[0] > 0) {
        IERC20(token0).safeTransferFrom(
            msg.sender,
            address(this),
            amounts[0]
        );
        IWETH(token0).withdraw(amounts[0]);
    }
    if (amounts[1] > 0) {
        IERC20(token1).safeTransferFrom(
            msg.sender,
            address(this),
            amounts[0] // note, 'amounts[0]' is referenced, instead of 'amounts[1]'
        );
    }
}
```

## Recommendations
Change the `token1` amount on line 108 of the `add_liquidity()` function from `amounts[0]` to `amounts[1]`.

## Resolution
The development team mitigated the issue in commit `5d506f0`, implementing the recommendations above.
