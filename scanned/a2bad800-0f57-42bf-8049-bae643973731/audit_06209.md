# Insecure handling of admin signing key 

## Summary
A bug has been identified in the current implementation of a system. The admin signing key, which is important for authenticating administrative actions, is being exposed in an insecure manner. This can potentially compromise the security of the entire system. The key is being passed as a command-line argument or environment variable, making it visible in process lists, logs, and command history. To fix this issue, it is recommended to implement a secure key management system or integrate with a Hardware Security Module (HSM). If local storage is necessary, the key should be encrypted at rest and decrypted only when in use. The team has acknowledged this issue and plans to implement a solution that will allow the admin key to be revoked in the future. The Cantina Managed team has also acknowledged the issue. This bug is considered to have a low risk level.
