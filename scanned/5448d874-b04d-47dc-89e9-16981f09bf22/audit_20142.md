# No check for zero address

## Summary
This bug report concerns the line of code https://github.com/aave/governance-crosschain-bridges/blob/763ef5da8befff3a129443a3ff4ef7ca4d3bb446/contracts/BridgeExecutorBase.sol#L207, which assigns a value to the variable `targets` of type `address`. The issue is that there is no check for the value of this variable, so if it is equal to zero, the transaction will fail.

The recommendation is to add a check for a zero address. This will ensure that the transaction will not fail if the `targets` variable is equal to zero. This will help to prevent any issues caused by the transaction failing due to a zero address.

## Details
##### Description
At the line https://github.com/aave/governance-crosschain-bridges/blob/763ef5da8befff3a129443a3ff4ef7ca4d3bb446/contracts/BridgeExecutorBase.sol#L207 is assigned to the variable `targets` of type `address`.
But there is no check for the value of this variable. If this variable is equal to zero, the transaction will fail.

##### Recommendation
It is recommended to add a check for a zero address.
