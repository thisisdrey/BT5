# [M-19] Staking: the rebase function needs to be called before calling the function in the Yieldy contract that uses the rebasingCreditsPerToken variable

## Summary
A bug has been identified in the Yieldy contract, which is a part of the Staking contract. This bug can have an impact on functions such as balanceOf/creditsForTokenBalance/tokenBalanceForCredits/transfer/transferFrom/burn/mint. The bug occurs when the rebasingCreditsPerToken variable is not updated before calling these functions. Therefore, the rebase function should be called in the unstake/claim/claimWithdraw function of the Staking contract. A proof of concept has been provided to demonstrate the bug. No tools were used in the identification of the bug. To mitigate the bug, the rebase() function should be added to the claim() and claimWithdraw() functions of the Staking contract. The rebase() function should also be removed from the unstake() function to prevent any vulnerabilities.

## Details
_Submitted by cccz_

In the Yieldy contract, functions such as balanceOf/creditsForTokenBalance/tokenBalanceForCredits/transfer/transferFrom/burn/mint will use the rebasingCreditsPerToken variable, so before calling these functions in the Staking contract, make sure that the rebase of this epoch has occurred. Therefore, the rebase function should also be called in the unstake/claim/claimWithdraw function of the Staking contract.
