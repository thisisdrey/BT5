# [H01] Fei reweight and stabilization may not react to price swings

## Summary
This bug report is about two functions in two different contracts of the Fei Protocol - TribeReserveStabilizer and UniswapPCVController. In both cases, the functions read the current price from an oracle, but the oracle is not updated before reading the price. This means that a stale price may be used to determine whether an exchange or reweight is allowed. 

The issue has been partially fixed in PR#70, where the reweight issue has been fixed in UniswapPCVController, but the exchangeFei issue has not been fixed in TribeReserveStabilizer. The Fei Protocol team stated that this is because the feiOracle will be a chainlink oracle without the need for update. 

To avoid making incorrect decisions in critical protocol operations, the Fei Protocol team should always update oracle prices before reading their values.

## Details
In the [`TribeReserveStabilizer`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/stabilizer/TribeReserveStabilizer.sol#L14) contract, the [`exchangeFei` function](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/stabilizer/TribeReserveStabilizer.sol#L42) determines whether to exchange a users FEI for TRIBE by calling the [`isFeiBelowThreshold` function](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/stabilizer/TribeReserveStabilizer.sol#L59). This function reads the current FEI price from an oracle, however this price is not ever updated by this contract, so a stale price is used to determine whether the exchange can occur.


Similarly, in the [`UniswapPCVController`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/UniswapPCVController.sol), the [`reweight` function](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/UniswapPCVController.sol#L57) determines whether a reweight is allowed by immediately calling the [`reweightEligible` function](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/UniswapPCVController.sol#L105). This fetches the current uniswap price, and compares it to a price that it reads from an oracle. If the difference between the 2 prices is large enough, a reweight is permitted to occur. However the `update` function of the oracle is not called prior to reading the price, meaning that stale oracle prices can be used without detection.


Consider always updating oracle prices before reading their values, to avoid making incorrect decisions in critical protocol operations.
