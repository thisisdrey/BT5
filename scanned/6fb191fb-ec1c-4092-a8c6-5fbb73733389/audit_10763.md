# Signature is replayable across domains

## Summary
The bug report is about a medium severity issue in the CPortModule.sol contract. The problem is that the signature, which contains information about the contract's name, version, address, and chain, is not properly checked to match the contract's deployment state. This can lead to security vulnerabilities, especially if the contract is deployed on different chains. The recommendation is to add a domainSeparator, which is a unique identifier, to the contract's deployment information to prevent replay attacks. The fix for this issue was implemented in the PaymentProcessor.sol contract, which was previously known as CPort.sol. The fix involves adding the domainSeparator to the calldata in a newly implemented modifier called delegateCallReplacementDomainSeparator. This fix is used in functions related to trading.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In CPortModule.sol - The domain of the signature (i.e. Contract name, version, address, chain) is not asserted to be conforming with the deployment state of the contract. A common vulnerable scenario that results from this issue is taking place if the contract is deployed across chains. Signature can be taken from one chain to be replayed on another.
Recommendation - The domainSeparator (i.e. which can be fed arbitrarily by taker) needs to be asserted with the current domain of contract deployment (i.e. chain, address, ...etc).
