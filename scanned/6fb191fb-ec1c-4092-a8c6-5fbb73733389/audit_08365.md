# PROPOSAL CAN BE DEFEATED IF THERE IS NO MEMBER

## Summary
This bug report is about a contract called GovHub, which is used for governance in CoreDAO. The problem is that if all the members are deleted during the creation of the contract, all proposals can be defeated. This is because the contract marks a proposal as 'DEFEATED' if the number of votes in favor is less than half of the total votes. The code location of this issue is in the GovHub.sol file, specifically on line 139. The code snippet provided shows a function called removeMember, which is used to remove a member from the contract. The report also includes a proof-of-concept image showing how the bug can be exploited. The impact and likelihood of this bug are both rated as 3, and the recommendation is to add a minimum threshold requirement to prevent this issue. The CoreDAO team has already solved this issue in a recent commit.

## Details
##### Description

GovHub is the governance contract for **CoreDAO**, a contract that allows any member to submit proposals with the initiator address stored in the proposer field. On the **GovHub** contract, the proposal is marked as **DEFEATED** If **forVotes** is smaller than **totalVotes/2**. However, if all the members are deleted during the genesis with **governance**, all proposals can be defeated in the system. The system should be ensured that member's length have a minimum threshold.

#### GovHub.sol

```
  function removeMember(address member) external onlyInit onlyGov {
    uint256 index = members[member];
    require(index > 0, "member does not exist");
    if (index != memberSet.length) {
      address addr = memberSet[memberSet.length - 1];
      memberSet[index - 1] = addr;
      members[addr] = index;
    }
    memberSet.pop();
    delete members[member];
    emit MemberDeleted(member);
  }
```
