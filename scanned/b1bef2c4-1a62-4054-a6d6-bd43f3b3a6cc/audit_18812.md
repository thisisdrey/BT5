# [M-13] `StakingRewards.recoverERC20` allows owner to rug the `rewardsToken`

## Summary
This bug report is about an issue in the StakingRewards.sol code of the 2022-02-concur repository on GitHub. The vulnerability allows the owner of the contract to sweep away the rewards tokens over time, without any check. This is an admin privilege that could be used to rug depositors. To prove this, the code for the StakingRewards.recoverERC20 function was provided, and calling it with the rewardsToken and its balanceOf value would enable the owner to sweep the token. To mitigate this issue, the code should be updated with an additional check that prevents the owner from withdrawing the rewards token.

## Details
_Submitted by Alex the Entreprenerd, also found by pauliax_

[StakingRewards.sol#L166](https://github.com/code-423n4/2022-02-concur/blob/main/contracts/StakingRewards.sol#L166)<br>

`StakingRewards.recoverERC20` rightfully checks against the `stakingToken` being sweeped away.<br>
However, there's no check against the `rewardsToken` which over time will sit in this contract.

This is the case of an admin privilege, which allows the owner to sweep the rewards tokens, perhaps as a way to rug depositors.

### Proof of Concept

Calling `StakingRewards.recoverERC20(rewardsToken, rewardsToken.balanceOf(this))` enables the `owner` to sweep the token.

### Recommended Mitigation Steps

Add an additional check

            require(
                tokenAddress != address(rewardsToken),
                "Cannot withdraw the rewards token"
            );
