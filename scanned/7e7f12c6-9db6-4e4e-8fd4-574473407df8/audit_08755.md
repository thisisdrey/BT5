# Stake function can be inhibited 

## Summary
This bug report discusses a vulnerability in the Controller.sol file, specifically on line 330. The report explains that in the `_stake` function, there is a conditional statement that can be exploited by an attacker. The condition requires the value to be either 0 or greater than or equal to a minimum stake amount. However, an attacker can send a small token to the address, causing the condition to always fail and allowing them to interfere with the function. The report recommends stopping the calculation of the value based on the address balance to mitigate this vulnerability. The status of the bug is marked as fixed.

## Details
## Vulnerability Analysis

## Context
**File:** Controller.sol  
**Line:** 330  

## Description
In the `_stake` function, there is a conditional statement:

```solidity
require(value == 0 || value >= minStakeAmount, "amount too small");
```

Where `value` is defined as follows:

```solidity
uint256 value = isEth_ ? _geth.balanceOf(address(this)) : _dct.balanceOf(address(this));
```

Let's assume that the person who wants to `_stake` intends to satisfy the condition `value == 0`. However, an attacker can front-run this and send a small token (where `0 < value < minStakeAmount`) to this address. This will always cause the conditional statement to fail, allowing the attacker to interfere with `_stake`, which is a critical function at the core of the product.

## Recommendation
To mitigate this vulnerability, you should stop calculating the `value` based on the address balance. At the very least, it should be considered that outsiders can increase the balances of addresses.

## Goat
**Status:** Fixed
