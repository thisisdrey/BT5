# Function consolidatePendingState() can be executed during emergency state

## Summary
This bug report is about a vulnerability in the PolygonZkEVM.sol code. The function consolidatePendingState() can be executed by anyone, even when the contract is in an emergency state, which could interfere with cleaning up the emergency. This is a medium risk issue.

The code in question is from lines 783 to 793 of the PolygonZkEVM.sol code. It looks like this: 

function consolidatePendingState(uint64 pendingStateNum) public {
if (msg.sender != trustedAggregator) {
require(isPendingStateConsolidable(pendingStateNum),...);
}
_consolidatePendingState(pendingStateNum);
}

The recommendation was to add a line to the code that would require that the contract not be in an emergency state before the function can be executed. This would improve consistency and prevent interference with cleaning up the emergency. The change was implemented in PR 87 and verified by Spearbit.

## Details
## Severity: Medium Risk

## Context
`PolygonZkEVM.sol#L783-L793`

## Description
The function `consolidatePendingState()` can be executed by everyone even when the contract is in an emergency state. This might interfere with cleaning up the emergency. Most other functions are disallowed during an emergency state.

```solidity
function consolidatePendingState(uint64 pendingStateNum) public {
    if (msg.sender != trustedAggregator) {
        require(isPendingStateConsolidable(pendingStateNum),...);
    }
    _consolidatePendingState(pendingStateNum);
}
```

## Recommendation
Consider adding the following, which also improves consistency:

```solidity
function consolidatePendingState(uint64 pendingStateNum) public {
    if (msg.sender != trustedAggregator) {
        + require(!isEmergencyState,...);
        require(isPendingStateConsolidable(pendingStateNum),...);
    }
    _consolidatePendingState(pendingStateNum);
}
```
