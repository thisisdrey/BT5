# Improper Authorization Login in Farming Instructions

## Summary
The report states that there is a bug in the code related to the add_farm and send_farm_tokens instructions in the exponent_core::market_two function. These instructions do not have proper access control checks, which means that any user can call them and potentially manipulate the farm emissions data. This can lead to unauthorized addition or modification of farms in the emissions array, which can cause unnecessary consumption of storage resources. Additionally, the authority of the token_farm account is incorrectly set to market.token_sy_escrow instead of market, which can cause issues with other instruction calls. To fix this bug, proper access control should be enforced and the authority of the token_farm account should be set to market. This issue has been resolved in PR#526 and PR#527.

## Details
## Security Review: Access Control in Exponent Core

## Overview

The `add_farm` and `send_farm_tokens` instructions in `exponent_core::market_two` lack sufficient access control checks. Without proper access controls, any user may call these functions, potentially manipulating the farm emissions data, including adding or modifying farms in the emissions array. 

## Risks

Without restricted access, any user may repeatedly call the `add_farm` instruction to add new entries to the `farm_emissions` array, bloating the array and unnecessarily consuming storage resources.

### Code Snippet

```rust
// market_two/admin/send_farm_tokens.rs
#[derive(Accounts)]
pub struct SendFarmTokens<'info> {
    [...]
    #[account(
        mut,
        associated_token::mint = mint,
        associated_token::authority = market.token_sy_escrow,
    )]
    pub token_farm: InterfaceAccount<'info, TokenAccount>,
    pub token_program: Interface<'info, TokenInterface>,
}
```

Furthermore, in the `SendFarmTokens` structure in `send_farm_tokens`, the authority of the `token_farm` account is incorrectly set to `market.token_sy_escrow` instead of `market` itself. This would lead to subsequent instruction calls being reverted.

## Remediation

- Enforce proper access control to ensure that only authorized users are able to invoke `add_farm` and `send_farm_tokens`.
- Additionally, set the authority to `market` for the `token_farm` account.

## Patch

Resolved in PR#526 and PR#527.
