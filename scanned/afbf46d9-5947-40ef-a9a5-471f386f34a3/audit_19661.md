# [H-08] USDV and VADER rate can be wrong

## Summary
This bug report is about a vulnerability in the TwapOracle.sol contract, which could lead to wrong rate calculations. The code in question is located at https://github.com/code-423n4/2021-11-vader/blob/main/contracts/twap/TwapOracle.sol#L166. The vulnerability occurs when tUSDInUSDV is smaller than tUSDInVader, and then getRate returns 0. The bug was discovered manually. The recommended mitigation step is to multiple enough decimals before division.

## Details
_Submitted by xYrYuYx_

#### Impact

<https://github.com/code-423n4/2021-11-vader/blob/main/contracts/twap/TwapOracle.sol#L166>

`tUSDInUSDV` can be smaller than `tUSDInVader`, and then `getRate` will return 0.
This will lead wrong rate calculation.

#### Tools Used

Manually

#### Recommended Mitigation Steps

Multiple enough decimals before division

**[SamSteinGG (Vader) confirmed](https://github.com/code-423n4/2021-11-vader-findings/issues/50)**
>The TWAP oracle module has been completely removed and redesigned from scratch as LBTwap that is subject of the new audit.
