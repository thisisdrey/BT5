# the killed gauge collect claim amount

## Summary
This bug report is about a smart contract that is not working properly on the Alchemix protocol. The contract is called Voter.sol and can be found on the GitHub repository for Alchemix. The bug is causing the contract to not deliver the expected returns, but it does not lose any value. This report is being submitted as insights for the Alchemix protocol.

The bug is related to a function called _updateFor, which is not preventing the update of claimable amounts for a specific type of gauge, even after it has been killed. This means that the gauge can still accrue claimable amounts even after it has been killed. This function is taken from another contract, but the Alchemix protocol may not want it to behave this way. If the gauge is revived, the claimable amount can become very large until then.

The impact of this bug is that the _updateFor function is updating the claimable amount for the killed gauge, which is not intended. This can cause issues with the gauge's functionality.

To demonstrate this bug, a proof of concept has been provided in the form of a code snippet. This code shows how the gauge can still accrue claimable amounts even after it has been killed. This bug can be reproduced by following the steps outlined in the code.

In conclusion, this bug is causing the Voter.sol contract to not function properly on the Alchemix protocol. It is important to fix this bug to ensure the correct functioning of the protocol.

## Vulnerability Details
the _updateFor did not prevent updating claimable amount for the killed gauge:

```solidity 
    function _updateFor(address _gauge) internal {
        require(isGauge[_gauge], "invalid gauge");

        address _pool = poolForGauge[_gauge];
        uint256 _supplied = weights[_pool]; 
        if (_supplied > 0) {
            uint256 _supplyIndex = supplyIndex[_gauge];
            uint256 _index = index; // get global index0 for accumulated distro
            supplyIndex[_gauge] = _index; // update _gauge current position to global position
            uint256 _delta = _index - _supplyIndex; // see if there is any difference that need to be accrued
            if (_delta > 0) {
                uint256 _share = (uint256(_supplied) * _delta) / 1e18; // add accrued difference for each supplied token
                claimable[_gauge] += _share;
            }
        } else {
            supplyIndex[_gauge] = index;
        }
    }


```

while the function is taken from the velo voter contract, the velo protocol set the claimable to zero when it kill it, but if this is not what the alchemix protocol want to do then the updatefor should not update the claimable amount for the killed gauge, in case if the gauge revived then the claimable amount can be huge amount till then.

## Impact Details
`updateFor` updates the claimable to killed gauge.
