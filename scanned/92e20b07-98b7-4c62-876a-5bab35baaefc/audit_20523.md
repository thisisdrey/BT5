# Probably incorrect using of `safeApprove`

## Summary
This bug report concerns a problem with the code at line 129 in the Strategy.sol file of the yearnV2-strat-SNX-staking repository. The code at this line does not set the approvement to zero before new approving. To fix this issue, the code should be changed to include the line of code `IERC20(susd).safeApprove(address(newSusdVault), 0);` which will set the approvement to zero before new approving. This will ensure that the code is working as intended.

## Details
##### Description
At line https://github.com/jmonteer/yearnV2-strat-SNX-staking/blob/91b839df4a350d80cb583795bccafe0836fdb732/contracts/Strategy.sol#L129 we see the single `safeApprove` without setting to zero. 

##### Recommendation
Set approvement to zero before new approving
```solidity
IERC20(susd).safeApprove(address(newSusdVault), 0);
```
