# `OPERATOR_FILTER_REGISTRY` can be missing in a new network

## Summary
A bug has been identified in the `VibeERC721` contract, which uses `onlyAllowedOperatorApproval` and `onlyAllowedOperator` modifiers. These modifiers require the `OPERATOR_FILTER_REGISTRY` to be deployed for the contract to work correctly. However, the `OPERATOR_FILTER_REGISTRY` may not be deployed on new networks.

It is recommended that a check is added to ensure that `OPERATOR_FILTER_REGISTRY` is deployed on the network that will be used for the protocol deployment. This will ensure that the `VibeERC721` contract is able to function correctly.

## Details
##### Description
`VibeERC721` contract uses `onlyAllowedOperatorApproval` and `onlyAllowedOperator` modifiers that required deployed `OPERATOR_FILTER_REGISTRY` for correct work https://github.com/vibexyz/vibe-contract/blob/d08057edbaf83b00d94dcaca2a05e3c44a45e4d9/contracts/tokens/VibeERC721.sol#L132 But it can be not deployed on a new networks.

##### Recommendation
We recommend adding a check that `OPERATOR_FILTER_REGISTRY` is deployed on the network that will be used for the protocol deployment.
