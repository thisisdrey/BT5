# Missing setter for cooldownDuration in LevelMinting 

## Summary
The LevelMinting contract has a bug where the cooldown duration, which is set to 7 days, cannot be updated by the protocol admin. This means that the withdrawal period for users cannot be changed to match the withdrawal period for vaults in LevelReserveManager. Additionally, the redeem() function cannot be called. To fix this, a new function should be added to update the cooldown duration. This bug has been fixed in the Level Money commit 5afc1851, but there is still an issue where the new duration cannot exceed the maximum duration. This has also been fixed in the Level Money commit 628bf291.

## Details
## Issue Report: LevelMinting Contract Cooldown Duration

## Context
- **File**: LevelMinting.sol
- **Lines**: 
  - L97
  - L133-L143

## Description
In the LevelMinting contract, `cooldownDuration` is set to 7 days in the constructor. However, the contract does not have a function to update the value of `cooldownDuration`. Therefore, the protocol admin will not be able to update the withdrawal period that users need to go through or disable it. This is problematic as the protocol’s withdrawal period cannot be adjusted to match the withdrawal period of vaults in LevelReserveManager. Additionally, `redeem()` can never be called.

## Recommendation
Add a permissioned function to update `cooldownDuration`.
