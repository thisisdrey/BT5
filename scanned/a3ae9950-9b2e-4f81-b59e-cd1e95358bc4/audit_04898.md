# Signature library does not revert on invalid signature 

## Summary
The report highlights a potential bug in the SignatureLib.sol code, where there is no check to ensure that ecrecover() does not return a zero address. This means that any invalid signature could be treated as valid, leading to potential issues such as forged data packages and incorrect price updates. While the likelihood of this happening is low, it is still a possibility and could have severe consequences. The recommended solution is to either check the return value of ecrecover or consider using a different library, such as Open Zeppelin's SignatureChecker.sol. The bug has been fixed in the Redstone and Canitna Managed commits.

## Details
## SignatureLib Vulnerability Report

## Context
SignatureLib.sol#L25

## Description
There is no check to ensure `ecrecover()` does not return zero. As such, any invalid signature is treated as valid and returned as the zero address.

## Impact
The impact of this could potentially be quite severe. Anyone could forge any data package and push a fictitious update to a price feed that could result in liquidations, improper fills, and other disastrous behavior.

## Likelihood
The likelihood of this happening is presumably low. The code comments for `RedstoneConsumerBase.getAuthorisedSignerIndex` clearly state:

```solidity
/**
 * ...
 * @dev This function must be implemented by the child consumer contract.
 * It should return a unique index for a given signer address if the signer
 * is authorised, otherwise it should revert
 * ...
 */
```

The integrator would have to overlook this or make a mistake in order for this to have an impact. However, there is also no guarantee how the SignatureLib will be used in the future which increases the chance of unexpected behavior.

## Recommendation
Check the return value of `ecrecover` and revert on address zero. Alternatively, consider replacing the library with Open Zeppelin's `SignatureChecker.sol`.

## Status
- **Redstone**: Fixed in commit 06557458.
- **Canitna Managed**: Fixed.
