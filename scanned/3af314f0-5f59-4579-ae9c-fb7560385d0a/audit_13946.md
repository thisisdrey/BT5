# Incorrect newSpotPrice and newDelta may be obtained due to unsafe downcasts

## Summary
This bug report details an issue with the XykCurve.sol file, specifically around lines 83 and 130. The issue is that when calculating newSpotPrice and newDelta in the functions getBuyInfo() and getSellInfo(), respectively, an unsafe downcast from uint256 to uint128 may occur and silently overflow, leading to much less value for newSpotPrice and newDelta than expected. A proof of concept is provided to demonstrate how this would not revert, but would instead silently overflow. 

The recommendation is to check if the value would overflow before casting, as is done in other places in the code. This can also be done with libraries such as OpenZeppelin SafeCast. A code snippet is provided to demonstrate the proposed solution. Sudorandom Labs and Spearbit have both reported that this issue has been solved in PR#42 and verified that this is fixed.

## Details
## Severity: Medium Risk

## Context
- `XykCurve.sol#L83`
- `XykCurve.sol#L130`

## Description
When calculating `newSpotPrice` in `getBuyInfo()`, an unsafe downcast from `uint256` to `uint128` may occur and silently overflow, leading to much less value for `newSpotPrice` than expected.

```solidity
function getBuyInfo(
    uint128 spotPrice, 
    uint128 delta, 
    uint256 numItems,
    uint256 feeMultiplier, 
    uint256 protocolFeeMultiplier
) external pure override returns (
    Error error, 
    uint128 newSpotPrice, 
    uint128 newDelta,
    uint256 inputValue, 
    uint256 tradeFee, 
    uint256 protocolFee
) {
    ...
    // get the pair's virtual nft and token reserves
    uint256 tokenBalance = spotPrice;
    uint256 nftBalance = delta;
    ...
    // calculate the amount to send in
    uint256 inputValueWithoutFee = (numItems * tokenBalance) / (nftBalance - numItems);
    ...
    // set the new virtual reserves
    newSpotPrice = uint128(spotPrice + inputValueWithoutFee); // token reserve
    ...
}
```

The same happens when calculating `newDelta` in `getSellInfo()`:

```solidity
function getSellInfo(
    uint128 spotPrice, 
    uint128 delta, 
    uint256 numItems,
    uint256 feeMultiplier, 
    uint256 protocolFeeMultiplier
) external pure override returns (
    Error error, 
    uint128 newSpotPrice, 
    uint128 newDelta,
    uint256 outputValue, 
    uint256 tradeFee, 
    uint256 protocolFee
) {
    ...
    // get the pair's virtual nft and eth/erc20 balance
    uint256 tokenBalance = spotPrice;
    uint256 nftBalance = delta;
    ...
    // set the new virtual reserves
    newDelta = uint128(nftBalance + numItems); // nft reserve
    ...
}
```

## PoC
Proof of concept about how this wouldn't revert but silently overflow:

```solidity
import "hardhat/console.sol";

contract test {
    constructor() {
        uint256 a = type(uint128).max;
        uint256 b = 2;
        uint128 c = uint128(a + b);
        console.log(c); // c == 1, no error
    }
}
```

## Recommendation
Check if the value would overflow before casting as is already done in other places. This can also be done with libraries such as OpenZeppelin SafeCast.

```solidity
// set the new virtual reserves
+ uint256 _newDelta = nftBalance + numItems
+ if (_newDelta > type(uint128).max) {
+     return (Error.SPOT_PRICE_OVERFLOW, 0, 0, 0, 0, 0);
+ }
- newDelta = uint128(nftBalance + numItems); // nft reserve
+ newDelta = uint128(_newDelta ); // nft reserve
```
