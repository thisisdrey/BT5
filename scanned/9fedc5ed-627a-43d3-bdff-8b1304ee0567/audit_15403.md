# Missing Owner Check In Partial Close Position

## Summary
The partial_close_position instruction is not properly checking for the owner of the position, allowing anyone to close it using this instruction. This can be fixed by implementing the same checks used in close_amount and calling the validate_close_position function. The issue has been addressed in the code by using validate_close_position.

## Details
## Partial Close Position Vulnerability

The `partial_close_position` instruction is used to partially close positions. This instruction needs to implement all the checks that are implemented by `close_amount`. 

In particular, `validate_partial_close_position` is used to validate the instruction parameters and accounts. This validation was missing from partial closes.

## Relevant Code Snippet

```rust
pub fn validate_partial_close_position(&self, amount: u64) -> Result<()> {
    require!(
        amount <= self.position.collateral_amount,
        CoreError::InvalidPartialClosePositionAmount
    );
    validate_amount(amount)
}
```

## Proof of Concept

An attacker can close anyone’s position using the `partial_close_position` instruction since the owner checks are not in place.

## Remediation

This can be remediated by calling the `validate_close_position` function to invalidate the `validate_partial_close_position` function.

## Patch

The `validate_close_position` was used and fixed in commit `8783032`.

```diff
@@ -120,7 +120,8 @@ impl<'info> ModifyPosition<'info> {
```
