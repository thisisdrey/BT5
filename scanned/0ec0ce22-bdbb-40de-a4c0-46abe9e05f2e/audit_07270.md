# Governor Proposal Creation Is Vulnerable to Front Running in OpenZeppelin Contracts - Phase 1

## Summary
This bug report discusses a vulnerability in the Taiko governance system that allows for a front running attack. This means that an attacker can manipulate the system by taking control of a proposal and potentially preventing legitimate proposals from being submitted. The vulnerability can be fixed by updating the OpenZeppelin contracts library to version 4.9.1 or later. The Taiko team has already resolved the issue by upgrading to version 4.9.6. This update will improve the security and integrity of the governance process.

## Details
The Taiko governance employs the [`GovernorCompatibilityBravoUpgradeable`](https://github.com/taikoxyz/taiko-mono/blob/b47fc34cb0e7fe9b7ebd9416b3051a067483b860/packages/protocol/contracts/L1/gov/TaikoGovernor.sol#L5-L6) contract from OpenZeppelin version [4.8.2](https://github.com/taikoxyz/taiko-mono/blob/b47fc34cb0e7fe9b7ebd9416b3051a067483b860/packages/protocol/package.json#L38-L39) which is susceptible to a [front running attack](https://github.com/OpenZeppelin/openzeppelin-contracts/security/advisories/GHSA-5h3x-9wvq-w4m2). By front running the creation of a proposal, an attacker can take control as the proposer and have the option to cancel it. This vulnerability can be leveraged to indefinitely prevent the submission of legitimate proposals, posing a significant risk to the governance process.


Consider updating the OpenZeppelin contracts library to version [4.9.1](https://github.com/OpenZeppelin/openzeppelin-contracts/tree/v4.9.1) or later as these versions contain the patch for this issue. This will enhance the security and integrity of the governor proposal mechanism.


***Update:** Resolved in [pull request #16360](https://github.com/taikoxyz/taiko-mono/pull/16360) at commit [2a0fe95](https://github.com/taikoxyz/taiko-mono/commit/2a0fe9526718bdf799874c7f2b0968f3dda7b6f2). The Taiko team stated:*



> *Thank you for reporting this bug. We have upgraded @openzeppelin library to "4.9.6".*
