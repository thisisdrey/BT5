# ECDSA library has a vulnerability for signature malleability of EIP-2098 compact signatures

## Summary
This bug report is about a malleability attack vector found in the 1inch ECDSA library. This library supports several types of signatures and forms in which they could be provided, but for compact signatures there is a recently discovered vulnerability. This could be exploited if contracts use transaction replay protection through signature uniqueness. To fix this issue, the 1inch team added a warning note in the comments of the library code. For more information, an advisory notice and fix can be found in the OpenZeppelin’s ECDSA library.

## Details
#### Resolution



Remediated as per the 1inch team in [1inch/[email protected]`166353b`](https://github.com/1inch/solidity-utils/commit/166353b6d225b0ae830f9d9d76be17444c8c2178) by adding a warning note in the comments of the library code.


The 1inch ECDSA library supports several types of signatures and forms in which they could be provided. However, for compact signatures there is a recently found malleability attack vector. Specifically, the issue arises when contracts use transaction replay protection through signature uniqueness (i.e. by marking it as used). While this may not be the case in the scope of other contracts of this audit, this ECDSA library is meant to be a general use library so it should be fixed so as to not mislead others who might use this.
