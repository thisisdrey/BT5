# User Can Receive Excessive Funds

## Summary
The code in the file `PositionSubMgrLib.sol` is not working properly. The function `calculateTransToUser()` is supposed to return a value of zero when a certain condition is met, but it only does so when the value is less than zero. This means that in some cases, users are receiving more money than they should be when closing a position. To fix this, the code should be modified to include an `<=` check instead of just `<`.

## Details
**File(s) affected:**`./contracts/market/PositionSubMgrLib.sol`

**Description:**`PositionSubMgrLib.calculateTransToUser()` does not return a zero value if `_params.collateralDelta.toInt256() + dPNL - fees == 0`, only if it is `< 0`. In the case where this value equals zero, the user will still get transferred `_params.collateralDelta.toInt256() + dPNL`, which is more than they should have received as that amount should just be covering the fees associated with closing this position.

**Recommendation:** Modify the line of code `if (_params.collateralDelta.toInt256() + dPNL - fees < 0) return 0;` to `if (_params.collateralDelta.toInt256() + dPNL - fees <= 0) return 0;`, including the `<=` check instead of just `<`.
