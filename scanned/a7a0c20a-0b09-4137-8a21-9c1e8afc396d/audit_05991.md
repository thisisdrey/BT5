# Wrong ERC20 token transferred

## Summary
This bug report is about a high-risk issue in a code called SwapIntent.sol. The problem is that the tokenUserBuys ERC20 token is being transferred instead of the intended auctionBaseCurrency. This is happening because the balance amount is corresponding to auctionBaseCurrency. The recommendation is to make a change in the code so that the correct token is transferred. This bug has been solved in PR 165 and has been verified by Spearbit.

## Details
## Security Report

## Severity
**High Risk**

## Context
*SwapIntent.sol#L129*

## Description
The `tokenUserBuys` ERC20 token is transferred here when it's meant to be `auctionBaseCurrency` since the balance amount transferred is corresponding to `auctionBaseCurrency`:

```solidity
if (auctionBaseCurrencyBalance > 0) {
    ERC20(swapIntent.tokenUserBuys).safeTransfer(user, auctionBaseCurrencyBalance);
}
```

## Recommendation
- Use:
  ```solidity
  ERC20(swapIntent.tokenUserBuys).safeTransfer(user, auctionBaseCurrencyBalance);
  ```
- Instead, use:
  ```solidity
  ERC20(swapIntent.auctionBaseCurrency).safeTransfer(user, auctionBaseCurrencyBalance);
  ```
