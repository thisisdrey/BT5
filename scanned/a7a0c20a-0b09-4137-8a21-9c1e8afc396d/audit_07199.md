# User can bypass the max amount restriction

## Summary
Severity: Medium

Status: Resolved

Description: 

A function called `upgradeStakeTokens()` has a restriction that limits the amount of tokens a user can stake to 1000 when the lockType is set to 1. However, this restriction is not present in another function called `stakeTokens`, allowing users to stake more than 1000 tokens for lockType 1. 

Recommendation: 

To fix this issue, the same restriction should be added to the `stakeTokens()` function.

## Details
**Severity**: Medium	

**Status**: Resolved

**Description**

The function `upgradeStakeTokens()` implements a `require` statement to ensure that the total staked amount by the user is less or equal to 1000 tokens when the lockType is 1.
```solidity
if(_locktype == 1){
               require(totalamount<=1000*10**18 ,"max amount reached"); 
}
```

This restriction is missing in the `stakeTokens` function, meaning that users can stake more than 1000 tokens for lockType 1 directly by the `stakeTokens` function.

**Recommendation**:

Add the same `require` statement to the `stakeTokens()` function.
