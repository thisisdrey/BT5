# Always failing check inside the `checkCondition` for `chainlinkOracleCondition`

## Summary
The bug report discusses an issue with a specific line of code in a project. The problem is that the `updatedAt` value being fetched from a Chainlink function is always in the past, which causes a condition to fail and the program to stop working. The report suggests a solution to fix this issue by changing a check in the code to compare `block.timestamp` with `updatedAt` and a specified duration. This will prevent the condition from failing and allow the program to run properly.

## Details
##### Description
There is an issue at [the following line](https://github.com/MetaLex-Tech/borg-core/blob/b89ff5b7796047bbb6123c97192cfcff33e6b7f5/src/libs/conditions/chainlinkOracleCondition.sol#L59). `updatedAt` value fetched from the Chainlink `latestRoundData()` is the time in the past when the price was reported. `block.timestamp` would always be bigger than that value, which will lead to condition revert.

##### Recommendation
We recommend changing the mentioned check to `block.timestamp - updatedAt > acceptableDuration`.
