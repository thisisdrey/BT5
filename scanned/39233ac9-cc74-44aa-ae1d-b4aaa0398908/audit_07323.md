# Incorrect check in the `createMinipool()` function

## Summary
A bug has been found in the MinipoolManager smart contract, which is used for creating minipools. The bug affects early stakers who are trying to contribute to the minipool. Currently, the function only allows contributions that are equal to the DAO's minimum requirement and the maximum PLS assignment. This means that some early stakers may not be able to contribute to the minipool. To fix this, the if condition in line 244 of the code needs to be changed to allow for a combination of contributions and PLS assignment that is within the DAO's minimum and maximum requirements. This bug has been marked as resolved, so it should be fixed in the next update.

## Details
**Severity**: High

**Status**: Resolved

**Description**

MinipoolManager.sol L244
For the early stakers in the `createMinipool()` function, the function only accepts the combined contribution equal to the DAO’s minimum requirement and `plsAssignmentRequest` equal to the DAO’s max PLS assignment.

**Recommendation**: 
Change the if condition to like this.
```solidity
if (msg.value + plsAssignmentRequest < dao.getMinipoolMinPLSStakingAmt() || plsAssignmentRequest > dao.getMinipoolMaxPLSAssignment())
```
