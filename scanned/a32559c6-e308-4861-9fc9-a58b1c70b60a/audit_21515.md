# Sensitive data is archived in iCloud and iTunes backups

## Summary
This bug report is about the data exposure vulnerability in MagmaWallet.ReceiveViewController and MagmaWallet.RecoveryPhraseDisplayViewController. This vulnerability is considered to be of low difficulty. It is possible for sensitive data to be exported to iTunes and iCloud backups, which can be accessed by Apple and any attackers with access to a user's iTunes or iCloud backup. 

Exploit scenarios include Alice gaining physical access to Bob's phone and initiating a backup of Bob's phone to iTunes from which she is able to extract all sensitive keychain data. Alternatively, Mallory can identify Magma user email addresses, then use a previously disclosed password database to guess their current iCloud passwords and retrieve iCloud backups that contain sensitive Magma keychain data from a large number of users. 

The recommendation to short term prevent keychain data from being migrated to iTunes and iCloud backups is to explicitly set a ThisDeviceOnly accessibility class for all keychain items. Long term, it is suggested to empirically validate that no sensitive data is stored to a backup of the Magma Wallet iOS application and consider uniform usage of a wrapper, such as Square’s Valet, to simplify storage and retrieval of data from the keychain. References include Apple Developer Documentation: Keychain Services and Square Valet.

## Details
## Type: Data Exposure

## Target:

- MagmaWallet.ReceiveViewController
- MagmaWallet.RecoveryPhraseDisplayViewController

## Difficulty: Low

## Description
Magma does not explicitly exclude sensitive data from device backups, which will allow sensitive data to get exported to iTunes and iCloud backups. Apple and any attackers with access to a user’s iTunes or iCloud backup will have access to a user’s private data.

## Exploit Scenario
Alice gains physical access to Bob’s phone and knows his passcode. She initiates a backup of Bob’s phone to iTunes from which she is able to extract all sensitive keychain data. Alternatively, Mallory identifies Magma user email addresses, then uses a previously disclosed password database to guess their current iCloud passwords. She retrieves iCloud backups that contain sensitive Magma keychain data from a large number of users.

## Recommendation
Short term, explicitly set a `kSecAttrAccessibleWhenUnlockedThisDeviceOnly` accessibility class (such as `ThisDeviceOnly`) for all keychain items. This should prevent keychain data from being migrated to iTunes and iCloud backups.

Long term, empirically validate that no sensitive data is stored to a backup of the Magma Wallet iOS application. Consider uniform usage of a wrapper, such as Square’s **Valet**, to simplify storage and retrieval of data from the keychain.
