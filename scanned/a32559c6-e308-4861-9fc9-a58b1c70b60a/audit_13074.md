# LlamaPolicy.hasRole doesn't check if a policyholder holds a token

## Summary
This bug report is about the LlamaPolicy.sol file at line 250. It states that if the revokePolicy function is used incorrectly, a policyholder could still have a role in the Llama system, even after their token has been burned. To fix this issue, the hasRole function should include an if statement that checks if the policyholder still holds the token in order to be active. This bug has been resolved by commit ad0391 and PR 294.

## Details
## Security Report

## Severity
**Medium Risk**

## Context
LlamaPolicy.sol#L250

## Description
Incorrect usage of the `revokePolicy` function can result in a case where the token of a policyholder is already burned but still holds a role. The `hasRole` function doesn't check if, in addition to the role, the policyholder still holds the token to be active. This means the role could still be used in the Llama system.

## Recommendation
Add the following line to the `hasRole` function:
```solidity
if (balanceOf(policyholder) == 0) return false;
```

## Resolution
- **Llama**: Resolved by commit ad0391 and PR 294.
