# Hardcoded values for amounts of ERC20 tokens

## Summary
This bug report concerns a hardcoded value in a smart contract. At the given line in the code, the value of 10 billion payout tokens is hardcoded as $10_000_000_000_000_000, which is equal to 10 billion multiplied by 10^6. This means that if the `payoutToken` is changed in the future with a different `decimals` value, then the contract will need to be redeployed with a new hardcoded value. 

The recommendation in the report is to use `ERC20(payoutToken).decimals()` to obtain the decimal multiplier instead of a hardcoded value for more flexibility. This will enable the contract to adjust to changes in the `payoutToken` without needing to be redeployed. 

In summary, this bug report concerns a hardcoded value in a smart contract and recommends using `ERC20(payoutToken).decimals()` to obtain the decimal multiplier instead of a hardcoded value for more flexibility.
