# Self Bidding On Auctions

## Summary
The bug report discusses an issue in the Auction contract where users are able to bid on their own auctions, which can lead to unfair bidding and difficulty for other users to participate. This is due to the lack of a check in the _placeBidOnAuction function to prevent the auction owner from bidding. A proof of concept is provided, and the suggested solution is to implement a check to ensure the bidder is not the owner of the auction. The bug has been fixed in the latest version of the contract.

## Details
## Auction Vulnerability Report

In Auction, `ownerInitAuctions` is the entry point for the owner to create auctions for a specified set of NFTs within the collection. Consequently, `placeBidOnAuction` allows users to place bids on active auctions for specific NFTs within a collection.

## Auction.sol Solidity Code

```solidity
function _placeBidOnAuction(
    CollectionState storage collectionState,
    BidParam memory param
) public {
    [...]
    {
        (bidToken, prevBidAmount, prevBidder, activityId) =
            (auctionInfo.bidTokenAddress, auctionInfo.lastBidAmount,
            auctionInfo.lastBidder, auctionInfo.activityId);
        // param check
        if (endTime == 0) revert Errors.AuctionNotExist();
        if (endTime <= block.timestamp) revert Errors.AuctionHasExpire();
        if (prevBidAmount >= param.bidAmount || auctionInfo.minimumBid >
            param.bidAmount) {
            revert Errors.AuctionBidIsNotHighEnough();
        }
        if (prevBidder == param.bidder) revert Errors.AuctionSelfBid();
        if (prevBidAmount > 0 && !isValidNewBid(param.bidAmount, prevBidAmount,
            param.minIncrPct)) {
            revert Errors.AuctionInvalidBidAmount();
        }
    }
    [...]
}
```

However, the issue arises as `_placeBidOnAuction` does not check to ensure that the user placing a bid on an auction is not the owner of that auction. This omission may result in a scenario where users bid on their own auctions, which is typically not the desired behavior for an auction system, as this could result in an instance where the user continuously outbids others, leaving it difficult for others to bid.

## Proof of Concept

1. User A owns an NFT within a collection and initiates an auction utilizing `ownerInitAuctions`. They provide details such as the NFT, minimum bid, and other auction parameters.
2. The auction is successfully created with User A as the owner.
3. At a later point, User A decides to participate in their own auction by placing a bid through `placeBidOnAuction`, utilizing the same account and token they used to create the auction.
4. The contract permits User A to place the bid without any checks to prevent the auction owner from bidding.
5. User A’s bid becomes the new highest bid in the auction, and the auction’s end time may be extended based on the bid extension logic.
6. As a result, other potential bidders interested in the auction may find it challenging to place higher bids because User A continues to bid on their own auction, increasing the bid price.

## Remediation

Ensure to implement a check to ensure the bidder of an auction is not the owner of that auction.

## Patch

Fixed in commit `22a7c3d`.
