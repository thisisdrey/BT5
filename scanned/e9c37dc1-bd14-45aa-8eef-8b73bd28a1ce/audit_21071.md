# Add additional check for `addLender`

## Summary
This bug report is about the `addLender` method in the `Strategy.sol` file located on the GitHub page https://github.com/Grandthrax/yearnV2-generic-lender-strat/blob/979ef2f0e5da39ca59a5907c37ba2064fcd6be82/contracts/Strategy.sol#L67. The bug is that there are no checks for `want` in this method.

The recommendation is to add a check to ensure that the `want` token of the Strategy is equal to the `want` token of the Lender. This will help to ensure that the Strategy is using the correct token. 

Overall, this bug report is suggesting that a check should be added to the `addLender` method in the `Strategy.sol` file in order to verify that the correct token is being used. This will help to ensure that the Strategy is functioning as intended and that the correct token is being used.

## Details
##### Description
At the line https://github.com/Grandthrax/yearnV2-generic-lender-strat/blob/979ef2f0e5da39ca59a5907c37ba2064fcd6be82/contracts/Strategy.sol#L67 in method `addLender` there are no checks for `want`.

##### Recommendation
It is recommended to check that `want` token of Strategy equals `want` token of Lender.
