# `UnikuraPhygitalCollection.mint()` Accepts Payment More than Required

## Summary
The client has fixed an issue in the `UnikuraPhygitalCollection` file where the `mint()` function was accepting more payment than required. This was due to a problem with the payment check logic, which did not return or revert extra payments. The team has recommended either changing the logic to strictly check for the required payment or clearly documenting the current logic for users. They also suggest adding a function for the admin to withdraw extra payments.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `4160633e1ffeb48582bd3d313b92e47352863333`. The client has fixed the issue by enforcing the check of `msg.value` with strict equality.

**File(s) affected:**`UnikuraPhygitalCollection.sol`

**Description:** In `UnikuraPhygitalCollection.mint()`, the function accepts payment more than required. The function accepts `msg.value` as payment, and the payment check logic is `require(msg.value >= (totalPrice + totalFee), "Insufficient payment");`. The function does not revert or return the extra payment to the sender when `msg.value` is greater than `totalPrice + totalFee`. Since there is no function for the admin to withdraw the extra payment, the extra payment will be locked in the contract forever.

**Recommendation:** We recommend the team to re-consider the payment logic. Our recommendations:

*   If the extra payment is intended, we recommend to perform strict equality for the payment check logic, i.e., `require(msg.value == (totalPrice + totalFee), "Insufficient payment");`. This will ensure that the payment is exactly the required amount.
*   Otherwise, clearly define this in the user-facing documentation to ensure that the users are aware of the payment logic. Also, consider adding a function for the admin to withdraw the extra payment.
