# Misaligned Memory Access

## Summary
The current implementation of the code has a bug where function arguments are not properly aligned in memory, causing undefined behavior. This is due to the use of #[repr(C, packed)] in the structures, which prevents the compiler from inserting padding for alignment. To fix this, the types need to be stored as byte arrays and manually converted when needed. This issue has been resolved in PR#6.

## Details
## Function Argument Alignment Issue

In the current implementation, function arguments are referenced with their type values without ensuring proper memory alignment. The structures utilize `#[repr(C, packed)]`, preventing the compiler from inserting padding for alignment. This creates misaligned references during argument parsing in most functions, resulting in undefined behavior.

## Example Struct

```rust
#[repr(C, packed)]
#[derive(Clone, Copy, Debug, Pod, Zeroable)]
pub struct DepositIx {
    pub account_index: u16,
    pub amount: u64,
    pub bump: u8,
}
```

## Remediation

Store the types as a byte array (for example, `u64` may be stored as `[u8; 8]` as demonstrated in ore) and manually convert (as done here) when needed, ensuring proper alignment.

## Patch

Resolved in PR#6.
