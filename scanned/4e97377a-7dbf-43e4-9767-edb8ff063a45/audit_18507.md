# Tokens are not approved to a new pool helper.

## Summary
The LockZap.sol contract has a function called setPoolHelper() that is used to set up new pools. However, after the contract is initialized, it requires an unlimited allowance to be granted to the pool helper in two types of tokens. This allowance is not being granted when a new pool is set, causing the protocol to not work with any new pool helpers. The recommendation is to either grant allowance before each transfer or to grant unlimited allowance in the setter function. After an audit, it has been found that approval is now being done before each transfer.

## Details
**Description**

LockZap.sol: setPoolHelper(). 
After the LockZap.sol initialization, it requires granting an unlimited allowance to_poolHelper in WETH and RDNT tokens. However, when a new pool is set with the function setPoolHelper(), allowance is not granted to a new pool helper. Thus the protocol couldn't operate with any new pool helper due to the lack of allowance. 

**Recommendation**: 

Consider granting allowance before each transfer OR grant unlimited allowance in the setter as well. 

**Post-audit**: 
Approval is performed before each transfer now.
