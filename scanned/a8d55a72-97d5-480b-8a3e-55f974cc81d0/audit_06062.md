# >= Instead Of >

## Summary
This bug report is about a feature called "auto deleveraging" in a compartment. Auto deleveraging is when the compartment's profit and loss (Pnl) is higher than a certain ratio, the balance is automatically reduced. However, there is an error in the code that calculates the ADLValue, which is the value at which auto deleveraging should take place. The code is currently set to trigger auto deleveraging when the global Pnl is above the ADLValue, but it should actually trigger when the global Pnl is equal to or above the ADLValue. The recommendation is to make this change in the code. The bug has been resolved.

## Details
**Severity** - Medium

**Status** - Resolved

**Description**

Auto deleveraging is a feature where if the compartment’s Pnl is higher than the ADL ratio then the balance is auto deleveraged  , ADLValue is calculated as follows →
```solidity
uint256 ADLValue = (s.ADLPercentage[_address] * (compartment.balance)) / BASIS_POINTS_DIVISOR;
```

According to the AppStorage definition of ADLPercentage , it is the percentage at which the ADL will hit , meaning when global Pnl equals (or above) the ADLValue then ADL should take place , therefore instead of 
```solidity
if (uint256(globalPnL) > ADLValue) { 
            return true;
        }
```
It should be 
```solidity
if (uint256(globalPnL) >= ADLValue) {
            return true;
```
**Recommendation**:

Make the suggested change.
