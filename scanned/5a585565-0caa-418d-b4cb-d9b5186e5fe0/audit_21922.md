# Admin/Owner roles may be burned by mistake

## Summary
This bug report is about the InstaRegistry contract, which is used to manage user accounts. The issue is that if the admin or owner of the contract calls the setAddress function without entering an input for the _userAddress parameter, some front-ends may interpret the missing address parameter as the zero address. This would result in the admin or owner mistakenly burning their admin/owner rights.

To fix this, the report suggests adding a require statement in the setAddress function that would revert if the _userAddress parameter is the zero address. This would protect against the accidental burning of admin/owner rights, while still allowing intentional burning of these rights.

## Details
If the admin or owner of the InstaRegistry contract mistakenly calls the [`setAddress`](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/InstaRegistry.sol#L37) function without explicitly entering an input for the `_userAddress` parameter (a common user error when interacting with contracts), some front-ends may interpret the missing `address` parameter as the zero address having been passed as the `_userAddress` parameter. This would result in the admin or owner mistakenly burning their admin/owner rights.


Consider adding a `require` statement in the `setAddress` function that would revert if the `_userAddress` parameter is the zero address. This would protect against the accidental burning of admin/owner rights. (Intentional burning of admin/owner rights would still be possible.)
