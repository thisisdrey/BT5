# Zero Token

## Summary
A bug report has been submitted regarding the constructor of the FarmingPool contract on GitHub. The constructor has no address checking for tokens params, meaning that a zero address can be used. This can lead to potential security risks, as it is possible to send funds to a zero address, which would be lost. It is recommended to add a check for a non-zero address to the constructor in order to prevent such problems.

## Details
##### Description
There is no address checking for tokens params in constructor:
https://github.com/1inch/farming/blob/7a007ec7784cca2899889e99e46cf06d5788a7d9/contracts/FarmingPool.sol#L35-L36
##### Recommendation
It is recommended to add a check for non-zero address.
