# `deposit` function is not working if `totalSupply()` is greater than zero

## Summary
This bug report is about a problem with a function called `deposit` in a program. The severity of this issue is high, but it has been resolved. The problem occurs when the `totalSupply()` is greater than zero and the `deposit` function is called by a user. The program will not work and give an error message saying "ReentrancyGuard: reentrant call" because it is trying to access the `update()` function. To fix this issue, the `nonReentrant` modifiers should be used correctly.

## Details
**Severity**: High

**Status**: Resolved

**Description**

if the ``totalSupply()` is greater than zero and user calls `deposit` function, the call will be reverted with the message: "ReentrancyGuard: reentrant call" because trying to access the `update()` function

**Recommendations**: 

fix the `nonReentrant` modifiers usage
