# Rounding down caused leftover funds in the contract

## Summary
This bug report is about a problem with the `claimPayment()` function. This function is responsible for calculating payments to parties and it is currently using rounding down division. This could potentially result in some token weis being left over after the payment is made. 

The recommendation is to consider returning the last payment with the remaining amount without calculating it using rounding division. This way, all token weis can be accounted for. 

In conclusion, this bug report is about an issue with the `claimPayment()` function which is resulting in some token weis being left over after payment. The recommended solution is to return the last payment with the remaining amount without calculating it using rounding division.

## Details
**Description:** 

In `claimPayment()`,all payments to parties are calculated with rouding down division. It could potentially have some token weis leftover after `claimPayment()`. 

**Recommendations:** 

Consider returning the last payments the remaining amount without calculating it using rounding division.
