# [M03] Math issues in SafeInt256.sol

## Summary
This bug report is about the SafeInt256.sol file, which is a library of functions that do not return the expected result for a given input. The mul, abs and div functions all have issues. 

The mul function does not check for overflowing results, meaning that mul(-1, -2**255) returns -2**255 instead of 2**255, which is too large for an int256 and should revert. The abs function has the same issue, and returns -2**255 instead of 2**255. 

The div function has a check of y>0 for the second parameter of the function, meaning that for any negative integer, the function would revert with error DIVIDE_BY_ZERO. Expected behaviour would be for div to be able to divide by a negative number without throwing.

To fix the issues, OpenZeppelin’s SignedSafeMath library should be used for mul and div, and line 49 of the SafeInt256.sol file should be changed to use the neg function, which includes an overflow check. The bug has since been fixed in pull request #50.

## Details
There are a number of issues in [`SafeInt256.sol`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/lib/SafeInt256.sol), whereby the functions do not return the expected result for a given input. A few examples of such issues are outlined below.


#### [`function mul`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/lib/SafeInt256.sol#L32)


`mul` does not correctly check for overflowing results, meaning that `mul(-1, -2**255)` returns `-2**255`. The expected result of this should be `2**255`, which is too large for an `int256`, and therefore it should revert.


#### [`function abs`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/lib/SafeInt256.sol#L48)


Similarly to `mul`, `abs` does not check for an overflowing value, meaning that `abs(-2**255)` returns `-2**255`. The expected result of this should be `2**255`, which is too large for an `int256`, and therefore it should revert.


#### [`function div`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/lib/SafeInt256.sol#L43)


`div` has a check `y>0` for the second parameter of the function, meaning that for any negative integer, the function would revert with error `DIVIDE_BY_ZERO`. Expected behaviour would be for `div` to be able to divide by a negative number without throwing.


Consider using OpenZeppelin’s [`SignedSafeMath` library](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/release-v3.1.0/contracts/math/SignedSafeMath.sol) for `mul` and `div` instead of the versions implemented in `SafeInt256.sol`. To correct a potential overflow in `abs`, consider changing [line 49](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/lib/SafeInt256.sol#L49) to use the [`neg`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/lib/SafeInt256.sol#L53) function, which includes an [overflow check](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/lib/SafeInt256.sol#L54).


***Update:** Fixed in [pull request #50](https://github.com/swapnet-protocol/swapnet-lite/pull/50).*
