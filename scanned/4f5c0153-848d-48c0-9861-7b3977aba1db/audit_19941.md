# [M-07] `HybridPool`'s `flashSwap` sends entire fee to `barFeeTo`

## Summary
A bug was found in the `HybridPool.flashSwap` function of the SushiSwap project. This function sends the entire trade fees `fee` to the `barFeeTo` address, instead of only sending `barFee * fee` as it should. This bug has had a significant impact on liquidity providers (LPs) as they are not receiving any fees at all when this function is used. Without the incentive of receiving fees, there is no incentive to provide liquidity.

The recommended mitigation step is to use the same fee mechanism as `swap` and only send `barFee * fee / MAX_FEE` to the `barFeeTo`. This can be seen in the `_handleFee` function. This bug has been confirmed by maxsam4 (Sushi).

## Details
_Submitted by cmichel, also found by 0xsanson_

The `HybridPool.flashSwap` function sends the entire trade fees `fee` to the `barFeeTo`.
It should only send `barFee * fee` to the `barFeeTo` address.

#### Impact

#### Recommended Mitigation Steps
The `flashSwap` function should use the same fee mechanism as `swap` and only send `barFee * fee / MAX_FEE` to the `barFeeTo`. See `_handleFee` function.
