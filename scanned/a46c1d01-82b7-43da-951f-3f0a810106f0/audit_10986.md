# Macro Implementations Missing Adequate Docstrings

## Summary
The report states that there is a problem with the code in the `codegen` folder. It is missing proper explanations called docstrings, which can make it difficult for people to understand the code. The team responsible for the code has acknowledged the issue and will work on fixing it. The progress can be tracked on the issue tracker of the code repository.

## Details
In the `codegen` folder, the files that hold the implementations for each macro are missing proper docstrings. Given the complexity and length of the code, adding detailed docstrings is strongly recommended. This will make it easier for readers and developers to understand the inner workings of the codebase.


***Update:** Acknowledged, will resolve. The Brushfam team expressed that they will resolve the issue. Progress can be tracked on [issue #155](https://github.com/Brushfam/openbrush-contracts/issues/155) in the repository's issue tracker.*
