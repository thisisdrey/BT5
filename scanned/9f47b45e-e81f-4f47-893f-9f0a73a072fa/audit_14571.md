# M-20: Malicious users could set allocations to a blacklist Protocol and break the rebalancing logic

## Summary
This bug report is about a vulnerability in the `game.sol` contract which could be exploited by malicious users. The vulnerability allows malicious users to set allocations to a blacklist Protocol, which would break the rebalancing logic. This would cause the `game.sol` contract to lose the value of the `deltas` and the whole system to be down.

The vulnerability was found by Met, saidam017, bin2chen, Ch\_301, and immeas, and was identified as Issue M-20. It was found through manual review.

The code snippet that is vulnerable is in the `receiveProtocolAllocations()` and `setDeltaAllocationsInt()` functions. The `receiveProtocolAllocations()` function checks if the Protocol is on the blacklist, and if it is, the `setDeltaAllocationsInt()` function will revert the `receiveProtocolAllocations().receiveProtocolAllocationsInt().setDeltaAllocationsInt()` call. This will prevent the `rebalance()` function from being executed.

The impact of this vulnerability is that the guardian is not able to restart the protocol manually, and the `game.sol` contract loses the value of the `deltas`. The whole system is down.

The recommendation to fix this vulnerability is to check if the Protocol is on the blacklist when Game players `rebalanceBasket()`.

Finally, sjoerdsommen noted that this issue is a duplicate of Issue #168.

## Summary
`game.sol` pushes `deltaAllocations` to vaults by [pushAllocationsToVaults()](https://github.com/sherlock-audit/2023-01-derby/blob/main/derby-yield-optimiser/contracts/Game.sol#L465-L477) and it deletes all the value of the `deltas`

```solidity
vaults[_vaultNumber].deltaAllocationProtocol[_chainId][i] = 0;
```

## Vulnerability Detail
Malicious users could set allocations to a blacklist Protocol. 
If only one of the `Baskets` has a non-zero value to a **Protocol on blacklist**
[receiveProtocolAllocations()](https://github.com/sherlock-audit/2023-01-derby/blob/main/derby-yield-optimiser/contracts/MainVault.sol#L343-L345) will revert 
`receiveProtocolAllocations().receiveProtocolAllocationsInt().setDeltaAllocationsInt()`
```solidity
  function setDeltaAllocationsInt(uint256 _protocolNum, int256 _allocation) internal {
    require(!controller.getProtocolBlacklist(vaultNumber, _protocolNum), "Protocol on blacklist");
    deltaAllocations[_protocolNum] += _allocation;
    deltaAllocatedTokens += _allocation;
  }
```
and You won't be able to execute [rebalance()](https://github.com/sherlock-audit/2023-01-derby/blob/main/derby-yield-optimiser/contracts/Vault.sol#L135-L154) 


## Impact
The guardian isn't able to restart the protocol manually. 
`game.sol` loses the value of the `deltas`.
The whole system is down.
