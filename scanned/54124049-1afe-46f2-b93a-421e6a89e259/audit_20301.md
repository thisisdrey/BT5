# [M04] ReserveStabilizer performs unsafe token transfer

## Summary
The `exchangeFei` function of the `ReserveStabilizer` contract is designed to burn a user’s FEI and in exchange transfer them tokens at a discounted rate. However, the code does not check the boolean return value of the token transfer, which can cause problems if the token transfer fails. This means that if the token transfer fails, such as with the ERC20-compatible token ZRX, the user will not receive any tokens for their FEI.

The issue can be fixed by updating the token transfer to use OpenZeppelin’s SafeERC20, which will handle all edge cases of token transfers. This issue has now been fixed, as seen in PR#71.

## Details
The [`exchangeFei` function of the `ReserveStabilizer`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/stabilizer/ReserveStabilizer.sol#L43) contract burns a user’s FEI and in exchange transfers them tokens at a discounted rate. However, the boolean return value of [the token transfer](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/stabilizer/ReserveStabilizer.sol#L101) performed is not checked. This means that ERC20-compatible tokens that return false on failure, such as ZRX, will not cause the call to revert. In turn the user will receive no tokens for their FEI.


Consider updating [the token transfer](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/stabilizer/ReserveStabilizer.sol#L101) to instead use OpenZeppelin’s SafeERC20, which will handle all edge cases of token transfers for you.


***Update:** Fixed in [PR#71](https://github.com/fei-protocol/fei-protocol-core-internal/pull/71).*
