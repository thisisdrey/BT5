# [M-04] Invalid Nodes can be registered due to an incorrect check

## Summary
The report discusses a bug in the DivReducer node, which is a type of code used in smart contracts. This bug affects the functionality of the node and has a medium impact on the overall system. The likelihood of encountering this bug is also medium. 

The description of the bug explains that the DivReducer node is supposed to have 2 parent nodes, but during registration, it does not properly check if the parents are valid. This means that even if the node is registered, it may not work correctly if the parents are not valid. 

The code snippet provided in the report shows that the function responsible for checking if the node is valid does not return anything if the parents are invalid. This means that the node will still be registered even if the parents are not valid, which can cause issues in the system. 

The recommendation provided in the report suggests adding a line of code to the function to return false if the parents are not valid. This will prevent the node from being registered and potentially causing problems in the system. 

To summarize, the bug in the DivReducer node can cause issues in the system if the parents are not valid. To fix this, the code should be modified to properly check the validity of the parents before registering the node.
