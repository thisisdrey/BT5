# MKTF-1 | Same Short and Long Token

## Summary
This bug report is about a problem with creating new markets. It was discovered that nothing prevents the long token from being set to the same address as the short token. This would cause funds to be double counted when deposits are made to the pool. The recommendation was to validate that the long token is not equal to the short token. The GMX Team implemented this recommendation and the bug was resolved.

## Details
**Description**

When creating a new market, nothing prevents the long token from being set to the same address as the short token.

In such a case, deposits would double count the funds you are sending to the pool. Since the creation of markets will eventually be unpermissioned, this validation is paramount.

**Recommendation**

Validate that the long token is not equal to the short token.

**Resolution**

GMX Team: The recommendation was implemented.
