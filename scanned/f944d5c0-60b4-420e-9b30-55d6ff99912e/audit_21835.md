# Validator should be able to remove delegator ✓ Addressed

## Summary
This bug report is about a situation where a validator cannot remove a delegator for the next period. The code added in SKALE-2162 allows both the validator and the delegator to request an undelegation, provided the delegation is not in the 'DELEGATED' state. The recommendation is that either the validator should be allowed to undelegate delegators for the next period, or the acceptance mechanism should be removed if it is not needed.

## Details
#### Resolution



Code added in [SKALE-2162](https://github.com/skalenetwork/skale-manager/pull/107), If the delegation is not in `DELEGATED` state, both validator and the delegator can request `undelegation`.


#### Description


In order to delegate tokens to a validator, the validator should accept the delegation request, however it’s not possible to remove the delegator for the next period.


#### Recommendation


For consistency, either allow a validator to `undelegate` delegators for the next period or remove acceptance mechanism if it’s not needed.
