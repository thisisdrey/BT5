# Repeated Identical Range Rewards Will Be Allowed In Separate Transactions

## Summary
The bug report states that there is an issue with the `submissionNonce` variable, which increments with every range reward and is used in calculating reward hashes. This results in all generated reward hashes being unique, even if the range reward values are the same. This means that the checks in the code will not catch if a range reward has been submitted before. The report recommends not using nonces in the calculation of reward hashes to fix this issue.

## Details
## Description

As `submissionNonce[msg.sender]` increments with every range reward and its value is used in the calculation of `rewardsSubmissionHash` and `rewardsSubmissionForAllHash`, all of the generated range reward hashes will be unique, regardless of their other values. This means that the checks on line [136] and line [164] will not catch whether a range reward has been submitted before. Instead, if a range reward is submitted twice, it will be fully processed with a new and unique reward hash, which will also allow it to be successfully paid out to stakers and operators through the reward calculation system.

## Recommendations

Consider not using nonces in the calculation of the reward hashes.
