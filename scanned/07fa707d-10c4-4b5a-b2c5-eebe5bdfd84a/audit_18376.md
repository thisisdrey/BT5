# Subtraction might revert.

## Summary
This bug report is about a problem with a storage variable called 'eqFeePool' in a function called 'swapRemote'. The variable collects fees in one function and subtracts them in another, but if the second function is called before the first one, the subtraction may cause an error. The recommendation is to fix this issue so that the function can be called without any problems.

## Details
**Description**

Line 181. Storage variable ‘eqFeePool’ collects fees in function swapRemote() and subtracts
fees in function swap(). However, in case swap() is executed before swapRemote() has been
executed, subtraction ‘eqFeePool = eqFeePool.sub(s.eqReward);’ might revert.

**Recommendation**:

Make sure that subtraction won’t revert and prevent function call.
