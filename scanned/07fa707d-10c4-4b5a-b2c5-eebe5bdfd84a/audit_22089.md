# SimpleShares contract allows execution during pause

## Summary
A bug was found in the MultiDistributor.sol (v1) contract, which is a low difficulty issue. The SimpleShares contract has functions that are not restricted to prevent execution during a paused contract state, which leads to successful function invocation during a paused state. The SimpleShares implementation of the distribute function allows for execution during a paused state, invoking the MultiDistributor implementation of distribute. Since neither the SimpleShares or MultiDistributor implementations are modified to restrict execution during a paused state, successful execution can occur. 

The exploit scenario is that the SimpleShares contract enters a paused state, and Bob invokes the SimpleShares distribute function, which executes successfully despite the contract being paused. 

The recommendation is to apply appropriate modifiers to the SimpleShares contract functions to prevent successful invocations during a paused state in the short term. In the long term, modify the MultiDistributor contract to take into consideration if the base contract is paused.

## Details
## Type: Access Controls  
**Target:** MultiDistributor.sol (v1)

## Difficulty: Low

### Description
Some functions defined in the **SimpleShares** contract are not restricted to prevent execution during a paused contract state. This leads to successful function invocation during a paused state.

The **SimpleShares** implementation of `distribute` allows for execution during a paused state, invoking the **MultiDistributor** implementation of `distribute`.

```solidity
// Distribute all unallocated balance of token to current token holders.
function distribute(address token) public returns (bool) {
    return super.distribute(token, totalSupply);
}
```
*Figure 1: SimpleShares implementation of distribute*

Since neither the **SimpleShares** nor **MultiDistributor** implementations are modified to restrict execution during a paused state, successful execution can occur.

```solidity
function distribute(address token, uint256 totalSupply) internal returns (bool)
```
*Figure 2: MultiDistributor function definition of distribute*

### Exploit Scenario
The **SimpleShares** contract enters a paused state. Bob invokes the **SimpleShares** `distribute` function, which executes successfully despite the contract being paused.

### Recommendation
In the short term, ensure appropriate modifiers are applied to the **SimpleShares** contract functions to prevent successful invocations during a paused state.

In the long term, modify the **MultiDistributor** contract to take into consideration if the base contract is paused.
