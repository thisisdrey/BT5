# rotateLiquidity() can revert if all participants don 't join a pot 

## Summary
This bug report is about a potential issue in the code for QuailFinance, specifically in line 146. The problem is that not all participants may join a pot, which can cause the winnerIndex to go out of bounds. The recommendation is to change how the winnerIndex is selected to prevent this issue.

## Details
## Context
**File:** QuailFinance.sol  
**Line:** L146  

## Description
It's possible that not all joiners join a pot. Therefore, `pot.participants.length < pot.numParticipants`, which can cause `winnerIndex` to go out of bounds:

```solidity
uint256 winnerIndex = uint256(randomNumber) % pot.numParticipants;
address winner = pot.participants[winnerIndex];
```

## Recommendation
Consider selecting `winnerIndex` as follows:

```solidity
uint256 winnerIndex = uint256(randomNumber) % pot.participants.length;
```
