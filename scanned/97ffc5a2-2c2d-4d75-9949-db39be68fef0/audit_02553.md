# Contract Balance Does Not Adequately Separate Commission

## Summary
This bug report is about a problem in the `Financial` contract. The issue is that when processing an `OP::UNSTAKE` request, the contract does not take into account the commission funds that are in the contract. This means that the funds reserved for commission could be used to fulfill an unstaking request. The same issue occurs when processing an `OP::SEND_TON_FROM_FINANCIAL` request. The recommendation is to modify the code to include the commission funds in the condition.

## Details
**Update**
Marked as "Acknowledged" by the client. The client provided the following explanation:

> We decided not to implement a fix because, during the OP::SEND_TON_FROM_FINANCIAL operation, we send both the main funds and the commission funds to the validation process to ensure that they are not left idle within the contract. Additionally, we allow users to unstake their funds without accounting for our commission, which we consider acceptable in this situation. However, when it comes to withdrawing the commission, we do take into consideration the user funds that are committed for withdrawals.

**File(s) affected:**`Financial.fc`

**Description:** In the `Financial` contract, when processing `OP::UNSTAKE`, this condition `if (balance >= withdraw_ton_amount + msg_value)` does not consider the commission funds in the contract. Therefore, there is a chance that funds allocated for commission, that are sitting in the contract, could be used to honor an unstaking request. Similarly, the accounting for the reserved commission amount is also missing when processing `OP::SEND_TON_FROM_FINANCIAL`.

**Recommendation:** Modify the condition to consider the commission funds.
