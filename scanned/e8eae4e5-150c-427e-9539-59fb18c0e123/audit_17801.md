# [M-15] Owner should be allowed to change feeManager

## Summary
This bug report is about the code found in the GitHub repository at the link provided. The vulnerability of this code is that, once the Fee Manager has been set by the owner, the owner has no power to change it. This could be problematic if the owner feels the current fee manager is behaving maliciously. The proof of concept for this vulnerability is that the setFeeManager function only allows the feeManager to change it once it has been set initially. The recommended mitigation step for this vulnerability is to change the setFeeManager function so that the owner is allowed to make the changes, as is done with other important functionality involving setArbitrator and setVoteDelegate.

## Details
