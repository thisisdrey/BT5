# [M-05] Possible DOS (out-of-gas) on loops.

## Summary
This bug report is about a possible out of gas issue while iterating a for loop in the BatchRequests.sol file in the code-423n4/2022-06-yieldy repository. A manual revision was done to identify the issue, and it was found that if there are a lot of contracts pending for withdrawal, the function on BatchRequests.sol#L14 could cause an out of gas issue. 

The impact of this issue is that a transaction could fail due to lack of gas, resulting in a loss of funds. A proof of concept was provided to demonstrate how this issue could occur, and a recommended mitigation step was also provided. The suggested mitigation step is to use a specific pattern of code to avoid the out of gas issue. 

Overall, this bug report provides important information about a potential out of gas issue in the BatchRequests.sol file, and provides a suggested mitigation step to avoid this issue.

### Impact

It is possible to get an out-of-gas issue while iterating the for loop.

Please take a look at [this link](https://github.com/wissalHaji/solidity-coding-advices/blob/master/best-practices/be-careful-with-loops.md).

### Proof of Concept

Let's say I want to run the function on [`BatchRequests.sol#L14`](https://github.com/code-423n4/2022-06-yieldy/blob/main/src/contracts/BatchRequests.sol#L14) and I got a lot of [`contracts`](https://github.com/code-423n4/2022-06-yieldy/blob/main/src/contracts/BatchRequests.sol#L9) pending for withdrawal.

### Recommended Mitigation Steps

Use this pattern;

```solidity
    /**
        @notice sendWithdrawalRequests on all addresses in contracts
     */
    function sendWithdrawalRequests(uint256 from, uint256 to) external {
        uint256 contractsLength = contracts.length;
        require(from < contractsLength, "Invalid from");
        require(to <= contractsLength, "Invalid to");
        for (uint256 i = from; i < to; ) {
            if (
                contracts[i] != address(0) &&
                IStaking(contracts[i]).canBatchTransactions()
            ) {
                IStaking(contracts[i]).sendWithdrawalRequests();
            }
            unchecked {
                ++i;
            }
        }
    }
```
