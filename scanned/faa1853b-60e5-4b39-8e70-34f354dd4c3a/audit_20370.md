# Immutable variable in access modifier

## Summary
This bug report is about the variable `_guardian` in the access modifier at line https://github.com/aave/governance-crosschain-bridges/blob/9fd0609a2e14d546885f76211961f251d2e15cb9/contracts/BridgeExecutorBase.sol#L22. This variable is only initialized in the constructor and there is no other functionality to change it. This can lead to unavailability to use the `cancel()` method. To fix this, it is recommended to add a tool to change the `_guardian` variable.

## Details
##### Description
At the line https://github.com/aave/governance-crosschain-bridges/blob/9fd0609a2e14d546885f76211961f251d2e15cb9/contracts/BridgeExecutorBase.sol#L22 used the `_guardian` variable in the access modifier. But it is initialized only once in the constructor and there is no other functionality to change. This can lead to unavailability to use `cancel()` method.

##### Recommendation
It is recommended to add tool to change `_guardian` variable.
