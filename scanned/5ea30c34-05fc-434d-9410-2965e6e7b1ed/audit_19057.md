# Variable ‘TOTAL_SUPPLY’ has no decimals

## Summary
The report states that there is a bug in the Standard ERC20 token, where the decimals for the 'TOTAL_SUPPLY' variable are missing. This means that the token does not have the correct number of decimals, which can cause issues with its functionality. The recommendation is to add the decimals for the variable by setting it to 100000000e18. This will fix the bug and ensure that the token works properly.

## Details
**Description**


Standard ERC20 token has 18 decimals, however decimals of ‘TOTAL_SUPPLY’ are missing.

**Recommendation**:

Add decimals for variable(e.g. TOTAL_SUPPLY = 100000000e18).
