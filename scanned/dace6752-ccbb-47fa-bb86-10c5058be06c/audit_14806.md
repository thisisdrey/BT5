# Open Redirect In Native WebView

## Summary
The report states that a malicious URL with a specific code in the URL fragment can cause an open redirect vulnerability. This means that a hacker can potentially redirect users to a harmful website. The QR code used for this URL is checked for validation, but the specific code is not, allowing the vulnerability to occur. This can lead to various types of attacks, such as phishing and cross-site scripting, which can harm security and privacy. The issue only affects Android devices. To fix this, validation should be performed on URLs with the specific code to ensure they come from trusted sources. The problem has been fixed in a recent update by checking the code's origin and only allowing trusted sources.

## Details
## Open Redirect Vulnerability in URL Fragment

A malicious URL containing `S.browser_fallback_url=<evil_site>` in a URL fragment may cause an open redirect vulnerability. Despite strict validation being performed on the host in the QR code, there is no validation being performed on `S.browser_fallback_url`. This URL is loaded directly into the WebView without any checks, leading to an open redirect vulnerability.

## DART Code

```dart
onLoadStart: (controller, url) {
    String newUrl;
    if (Platform.isAndroid) {
        if ('$url'.startsWith('intent://${QoinServices.qrQoinLounge}')) {
            webView.stopLoading();
            try {
                newUrl = url.fragment.split(";").firstWhereOrNull((element) =>
                    element.split("=")[0] == "S.browser_fallback_url").split("=")[1];
            } catch (e) {}
            if (newUrl != null) {
                newUrl = Uri.decodeFull(newUrl) +
                    '&source=qoinpay&data=${QoinAccount.userData.fullName};...';
                controller.loadUrl(urlRequest: URLRequest(url: Uri.parse(newUrl)));
                return null;
            }
        }
    }
}
```

A malicious actor may launch various types of attacks, such as phishing, cross-site scripting, and others. These attacks potentially result in significant harm to both security and privacy. Note that this attack only works on Android devices.

## Remediation

Perform validation on URLs that contain the `S.browser_fallback_url` fragment to ensure that they originate from trusted sources. Specifically, URLs that come from QR code data should not be allowed directly.

## Patch

Fixed in `37a0889` by parsing the `S.browser_fallback_url` and checking its host against the allowed host.
