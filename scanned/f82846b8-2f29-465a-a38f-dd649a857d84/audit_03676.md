# Pools are not always replenished

## Summary
Report Summary:

The client has marked a bug as "Fixed" in the file `src/Pool.sol`. The function `_deltaSourceUpdate()` is supposed to replenish the soft partitions of the liquidity pool by applying `deltaCredit` according to the weights of each pool. However, if `fullMode` is set to true and the `totalDiff` is less than or equal to `deltaCredit`, the `diff` is not applied to `amtToCredit`. This results in only the `excessCredit` being distributed to the pools, which is not the intended behavior. The recommendation is to add `diff` to `amtToCredit` when `totalDiff` is less than or equal to `deltaCredit` in `fullMode`.

**Update**

**Description:**`_deltaSourceUpdate()` is designed to apply the `deltaCredit` to replenish the soft partitions of the liquidity pool. The `diff` between the target total balance and current balance (the deficit) is calculated for each pool. `deltaCredit` is then applied to replenish each pool according to their weights.

However, if `fullMode == true` and `totalDiff <= deltaCredit`, the `diff[i]` is never applied to `amtToCredit`. In this scenario, only the `excessCredit` is distributed to the pools. The inline and NatSpec comments further validate that this is a missing requirement.

**Recommendation:** Add `diff[i]` to `amtToCredit` when `totalDiff <= deltaCredit` in `fullMode`.
