# The `RedemptionOffer.requested_redemptions` Field Not Updated upon a Successful Redemption Request Fulfillment

## Summary
The client has marked a bug as "Fixed" in the code. The bug is related to the `RedemptionOffer` account not subtracting the requested redemption amount when a request is fulfilled. This causes incorrect accounting and makes it difficult for the protocol team to provide the correct amount of tokens. The recommendation is to deduct the `token_in_amount` from the `requested_redemptions` field in the `fulfill_redemption_request()` function.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `84fd65216f0a6133ada8eaa65625fdb9ddf0deee`. The client provided the following explanation:

> Added subtracting requested redemptions when fulfilling a redemption request

**File(s) affected:**`programs/onreapp/src/instructions/redemption/fulfill_redemption_request.rs`

**Description:** The `RedemptionOffer` account holds the state related to a given redemption offer, including the current `requested_redemptions` amount. This value is added to when a user creates a redemption request; however, it is not subtracted from when their redemption request is fulfilled. The inline documentation above the `fulfill_redemption_request()` function suggests that this action should take place, but the necessary logic is missing in the current implementation. This causes inconsistent accounting, making it more difficult for the protocol team to correctly ensure they provide the necessary amount of tokens to fulfill all requests.

**Recommendation:** Ensure that the `requested_redemptions` field has `token_in_amount` deducted from it within the logic of the `fulfill_redemption_request()` function.
