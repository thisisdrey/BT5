# Missing Transaction Information

## Summary
A bug has been reported where the amount of AE coins being sent is not shown in the confirmation dialog when calling a payable function of a smart contract. The bug has been identified in the code for the contactCallPanel function in the components.ts file. The suggested solution is to add an entry showing the amount of AE being sent on the contract call transaction and the contract create transaction. This information can be retrieved from the serialized transaction according to the documentation. A patch has been implemented to fix this issue by adding the amount of the transactions in the prompt message.

## Details
## Payable Function Call in Smart Contract

When calling a payable function of a smart contract, tokens are sent alongside the call data. However, the amount of AE coins being sent is not shown in the confirmation dialog.

```typescript
export const contactCallPanel = (args?: any) => {
  return panel([
    divider(),
    text("Contract call"),
    divider(),
    labelPanel({ label: "Address from", value: args.callerId }),
    labelPanel({ label: "Address to", value: args.contractId }),
    labelPanel({
      label: "Fee",
      value: args.fee ? `**${args.fee} AE** ${args.feeInUsd}` : undefined,
      isCustomValue: true,
    }),
  ]);
};
```

## Remediation

Add an entry showing the amount of AE being sent on the contract call transaction (Tag 43) and the contract create transaction (Tag 42). This information can be retrieved from the serialized transaction according to the documentation.

## Patch

The fix was implemented in `0b08937` by adding the amount of the transactions in the prompt message:

```typescript
export const contactCallPanel = (args?: any) => {
  return panel([
    divider(),
    text("Contract call"),
    divider(),
    labelPanel({ label: "Address from", value: args.callerId }),
    labelPanel({ label: "Address to", value: args.contractId }),
    + labelPanel({ label: "Amount", value: args.amount }),
  ]);
};
```
