# Sales Cannot Be Frozen

## Summary
The bug report is about a missing feature in a software that allows the freezing and unfreezing of sales. The client has marked this issue as "Fixed" and provided an explanation for the new feature. The affected file is `programs/early-purchase/src/state/sale.rs` and the recommendation is to implement the freezing/unfreezing logic so that the authorized users can perform this action.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `3f9b681e745166219241db82648032e2dbd94cd1`. The client provided the following explanation:

> A new instruction freeze_sale was added that allows an a Guardian with the update_sale permission to toggle the frozen state on an active Sale

**NOTE** In the latest implementation only the purchasing tokens operations can be frozen and users can still redeem receipts.

**File(s) affected:**`programs/early-purchase/src/state/sale.rs`

**Description:**`Sale` accounts are supposedly freezable by guardians/admins as they have an `is_frozen()` function that checks if `self.state == SaleState::Frozen`, which is called in the `PurchaseTokens` instruction handler. However there is no mechanism allowing authority to freeze and unfreeze sales as no instruction implements such a logic.

**Recommendation:** Consider implementing the freezing/unfreezing logic so the authority can perform the action.
