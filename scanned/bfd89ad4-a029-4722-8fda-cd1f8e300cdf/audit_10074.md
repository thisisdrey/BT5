# Critical Functions Are Public and Without Access Control ✓ Fixed

## Summary
The bug report highlights a security issue in the `RescuableV1` and `BlacklistableV1` contracts. These contracts have public and unauthenticated functions that can be exploited by anyone to steal funds and blacklist other accounts. Although the child contract `FiatTokenV1` has protected these functions, other contracts that inherit from `RescuableV1` and `BlacklistableV1` may still be vulnerable. The recommendation is to make these functions internal and add a public function with authentication in the child contract to call the inherited functions. This will ensure that only authorized users can access these critical functions.

## Details
#### Description


Critical functions in `RescuableV1`(`rescue`) and `BlacklistableV1` `(blacklist,unblacklist)` are public and unauthenticated, any one can call these function to steal funds and blacklist other accounts. Although the child contract `FiatTokenV1` has authenticated the overridden functions and protected them from public access, other contracts inheriting `RescuableV1` and `BlacklistableV1` might have risks from the unauthenticated public functions
