# Issue in generic withdrawal of Tokens

## Summary
The bug report is about a problem with a code called Vester.sol/VesterNLP.sol. The problem is that when trying to withdraw a token, any token can be withdrawn, including one called esToken. This can cause issues with the Vester balances, as the amount of esToken transferred is not reflected in the amount of Vester minted. The recommendation is to exclude esToken from being transferred. The issue has been resolved by adding a requirement that checks if the token being transferred is not esToken.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

Vester.sol/VesterNLP.sol - In withdrawToken, it withdraws any token including esToken. This can mess up the Vester balances as it is shown that amount of esToken transferred corresponds to a related amount of Vester minted back.

**Recommendation** 

exclude esToken from transfer.

**Fix**:  Require statement added that asserts token being transferred is not indeed the esToken.
