# There is no check on the result of the function

## Summary
The bug report is about an issue with the `approve()` function in a contract. According to the ERC20 standard, the `approve()` function should return a boolean value. However, the contract in question does not process this value after the call to the `_approveAll()` function. This can lead to a situation where a False value is returned.

To fix this issue, it is recommended to add a check of the return value. This will ensure that the contract follows the ERC20 standard and that the boolean value is processed correctly.

## Details
##### Description
According to the ERC20 standard, the `approve()` function returns a boolean value. But in the contract on lines https://github.com/orbxball/liquity-stability-pool/blob/c3fa76af0a4e2d5fd7132b8e24361d5b7439a75d/contracts/Strategy.sol#L120-L123, after the call to the `_approveAll()` function, this values are not processed.     A situation may arise that a False will return. 

##### Recommendation
It is recommended to add a check of the return value.
