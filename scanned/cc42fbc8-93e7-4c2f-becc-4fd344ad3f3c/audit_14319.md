# M-37: There is no market enabled validation in Swap and CreateAdl activities

## Summary
This bug report was found by IllIllI and caventa and refers to the issue M-37. This issue is related to the lack of market enabled validation in Swap and CreateAdl activities. This vulnerability allows controller to execute activities for disabled market in Swaphandler and CreateAdlHandler. The code snippets that are related to this issue can be found in the following links: https://github.com/sherlock-audit/2023-02-gmx/blob/main/gmx-synthetics/contracts/swap/SwapUtils.sol#L98-L149, https://github.com/sherlock-audit/2023-02-gmx/blob/main/gmx-synthetics/contracts/swap/SwapUtils.sol#L158-L318 and https://github.com/sherlock-audit/2023-02-gmx/blob/main/gmx-synthetics/contracts/adl/AdlUtils.sol#L125-L173. The manual review was the tool used to find this issue. The recommendation is to add the following code: ```solidity
Market.Props memory _market = MarketUtils.getEnabledMarket(dataStore, market);``` to swap and createAdl. According to xvi10, the market is validated in _getExecuteOrderParams called after AdlUtils.createAdlOrder for ADL, and the swap path is fetched with getEnabledMarkets before calling the swap function for swaps.
