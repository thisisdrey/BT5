# Function totalVaultBalance() doesn't use firstVaultId variable

## Summary
The bug report states that there is an error in the code where a cycle always starts from 0, but it should start from a different variable. The recommendation is to change the starting point of the cycle from 0 to the variable specified. The issue has been fixed after a re-audit.

## Details
**Description**

Cycle “for (uint256 i = 0; i <= _lastVaultId; i++)” always starts from 0 but must start from the
firstVaultId variable.

**Recommendation**:
Change “uint256 i = 0” to “uint256 i = firstVaultId”

**Re-audit**:
Fixed.
