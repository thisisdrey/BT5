# ArbitrumBridgeFacet does not check if msg.value is enough to cover the cost

## Summary
A bug was found in the ArbitrumBridgeFacet.sol#L97-L121 code that does not check if the user provided ether (msg.value) is enough to cover the amount and cost of the transaction. This could allow exploiters to set a large cost and sweep the ether from LiFi's LibDiamond address. A recommendation was given to always check that the inbound ether is enough to cover the outbound ether. This can be done by checking the msg.value is greater than the cost for startBridgeTokensViaArbitrumBridge and assetId not equal to NATIVE_ASSET, and checking the msg.value is greater than the cost plus amount for startBridgeTokensViaArbitrumBridge and assetId equal to NATIVE_ASSET. For swapAndStartBridgeTokensViaArbitrumBridge and assetId not equal to NATIVE_ASSET, the received ether should be calculated and checked to be greater than the cost. Lastly, for swapAndStartBridgeTokensViaArbitrumBridge and assetId equal to NATIVE_ASSET, the received ether should be calculated and checked to be greater than the amount plus cost. LiFi has fixed the issue with PR #104 and Spearbit has verified it.

## Context
`ArbitrumBridgeFacet.sol#L97-L121`

## Description
The `ArbitrumBridgeFacet` does not check whether the users’ provided ether (`msg.value`) is enough to cover `_amount + cost`. If there are remaining ethers in LiFi’s LibDiamond address, exploiters can set a large cost and sweep the ether.

```solidity
function _startBridge(
...
) private {
...
    uint256 cost = _bridgeData.maxSubmissionCost + _bridgeData.maxGas * _bridgeData.maxGasPrice;
    if (LibAsset.isNativeAsset(_bridgeData.assetId)) {
        gatewayRouter.createRetryableTicketNoRefundAliasRewrite{ value: _amount + cost }(
        ...
        );
    } else {
        gatewayRouter.outboundTransfer{ value: cost }(
        ...
        );
    }
}
```

## Recommendation
Always check the inbound ether is enough to cover the outbound ether. There are different possible cases:

- **`startBridgeTokensViaArbitrumBridge` and `assetId != NATIVE_ASSET`**
  - Check `msg.value > cost`
  
- **`startBridgeTokensViaArbitrumBridge` and `assetId == NATIVE_ASSET`**
  - Check `msg.value > cost + amount`
  
- **`swapAndStartBridgeTokensViaArbitrumBridge` and `assetId != NATIVE_ASSET`**
  - Calculate `received _ether = post_swap_ether - pre_swap_ether` and check `received _ether > cost`
  
- **`swapAndStartBridgeTokensViaArbitrumBridge` and `assetId == NATIVE_ASSET`**
  - Calculate `received _ether = post_swap_ether - pre_swap_ether` and check `received _ether > amount + cost`

Fixed with PR #104.
