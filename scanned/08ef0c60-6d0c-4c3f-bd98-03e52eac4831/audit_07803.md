# Authorized function is never used

## Summary
This bug report is about a function called `resetTokenStateByArbManager()` in the `LiquidityManager.sol` file. The function is not being used, even though it is supposed to be called by the `ArbitrageManager` contract. The recommendation is to check and fix the logic of the function. The Chainport team has confirmed that the function should be called periodically to reset the state of the arb manager withdrawals and deposits, and they have changed the access control from the arbManager to Authorized. The bug has been resolved.
