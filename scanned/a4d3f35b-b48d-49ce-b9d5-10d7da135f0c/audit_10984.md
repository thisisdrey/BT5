# No Access Control on Setters Produced by accessors Macro

## Summary
The `accessors` macro in the openbrush-contracts project allows users to easily create getters and setters for struct fields in smart contracts. However, there is currently a bug where modifiers cannot be attached to the setters, making it difficult to control access to sensitive information. The team behind the project has acknowledged the issue and plans to fix it in a future update.

## Details
The `accessors` macro implements [getters](https://github.com/Brushfam/openbrush-contracts/blob/553347354de6d0ead6a335ba4c197bdd01fb2d12/lang/codegen/src/accessors.rs#L29) and [setters](https://github.com/Brushfam/openbrush-contracts/blob/553347354de6d0ead6a335ba4c197bdd01fb2d12/lang/codegen/src/accessors.rs#L45) for struct fields, which can be called externally in a smart contract context since they are annotated with `#[ink(message)]`. It is vital to ensure that any sensitive or critical state variables cannot be trivially accessible or modifiable using these generated functions, as it could expose vulnerabilities in the contract.


Currently, modifiers cannot be attached to these setters, and there is inadequate documentation cautioning users about this macro's use.


Consider modifying the macro setter attribute to incorporate access control capabilities. Furthermore, it would be ideal for the setters to come with access control mechanisms enabled by default and only allow them to be disabled if users intentionally choose to do so.


***Update:** Acknowledged, will resolve. The Brushfam team stated that they will resolve the issue:*



> *The macro is going to be reworked soon, the suggestion will be implemented there, progress can be tracked here: https://github.com/Brushfam/openbrush-contracts/issues/135.*
