# getAmountOut Calculates an Output Value That Sets the Invariant to Zero, Instead of Preserving Its Value

## Summary
This bug report is about the swap function in the RMM01Lib.sol code. The swap function is designed to ensure that the pool's invariant value does not decrease. However, the getAmountOut function computes an expected swap output based on setting the pool's invariant to zero, which is not accurate if the initial value of the invariant was not already zero. This could lead to either disadvantageous swaps or swaps that revert, depending on the current pool invariant value.

The recommended fix is to add another field to the Bisection struct for the initial invariant value and modify optimizeDependentReserve to return the difference between the value it currently returns and the initial invariant value. This fix has been implemented in commit cbec75 and has been confirmed as fixed.

## Details
## Severity: Medium Risk

**Context:** [1] RMM01Lib.sol#L84, [2] RMM01Lib.sol#L215, [3] RMM01Lib.sol#L231

**Description:**  
The swap function enforces that the pool's invariant value does not decrease; however, the `getAmountOut` function computes an expected swap output based on setting the pool's invariant to zero. This is only equivalent if the initial value of the invariant was already zero—something that will generally not be the case as fees accrue and time passes. 

This is highlighted in the `computeSwapStep` function (invoked by `getAmountOut` [1]), where the function `optimizeDependentReserve` passed [2] to the bisection algorithm for root finding returns just the invariant evaluated on the current arguments [3] instead of the difference between the evaluated and original invariant. As a consequence, `getAmountOut` will return an inaccurate result when the starting value of the invariant is non-zero, leading to either disadvantageous swaps or swaps that revert, depending on whether the current pool invariant value is less than or greater than zero.

**Recommendation:**  
Add another field to the Bisection struct for the initial invariant value and modify `optimizeDependentReserve` to return the difference between the value it currently returns and the initial invariant value.

**Primitive:** Fixed in commit `cbec75`.

**Spearbit:** Fixed confirmed.
