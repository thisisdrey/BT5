# NODE INTEGRATION ENABLED

## Summary
The report discusses a bug in Electron, a software used for creating desktop applications. The bug is related to a feature called "nodeIntegration" which allows JavaScript to use Node.js functions and modules. This feature can be potentially dangerous as it can lead to remote system compromise if untrusted content is rendered. This means that a malicious website or code can use this feature to access and execute code on a user's computer. The code responsible for this bug is located in the "/wallet-desktop/app/main.dev.js" file. The impact of this bug is rated 4 out of 5, and the likelihood of it occurring is rated 3 out of 5. This bug can be exploited through Cross-Site Scripting (XSS) attacks or by rendering untrusted content in a browser window.

## Details
##### Description

In Electron, nodeIntegration is a boolean option that can be used to determine whether node integration is enabled, allows JavaScript to leverage Node.js primitives and modules.

As a result, this could lead to full remote system compromise if you are rendering untrusted content.

For instance, a remote untrusted domain rendered in a browser window could invoke Node.js APIs to execute native code on the user’s machine. Similarly, a Cross-Site Scripting (XSS) vulnerability on a website can lead to remote code execution.
