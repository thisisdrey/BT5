# Incorrect planning of orders to ﬁll can crash the nord server 

## Summary
The report discusses a bug in the `slab.rs` file that can cause the nord server to crash. The bug occurs when the `limit_by_quote_size` function is used to limit the size of an order, but it uses two different calculations that can lead to a mismatch in the order's quote value. This can result in an unexpected additional order being added to the collection of orders to fill, causing the server to crash. The report includes a proof of concept and a recommendation to fix the bug by resetting the limit sizes to zero when one of them is reached. The bug has been fixed in PR 999 and is now managed by Cantina.

## Order Filling Issue in `slab.rs`

## Description
During the planning of orders to fill in `plan_fills`, the `limit_by_quote_size` function can be used to limit the size of an order through both the `limit_size` and the `limit_quote` arguments. However, this function uses both `quote2size` and `size2quote` on the same operator. As the `quote2size` division loses precision, the conversion `quote → size → quote2` will lead to `quote != quote2`.

When the difference `quote - quote2` is greater than the price of the next order to fill, the calculation `quote2size` will be greater than 0 (`limit_quote / price >= 1`). This will add an unexpected additional order to fill in the collection of orders to fill. Then, this unexpected element in the orders to fill will trigger an assertion failure in `apply_fills`, which will crash the nord server. A user can crash the nord server through a well-crafted order with a quote limit.

## Proof of Concept
The `limit_by_size_quote` function can limit the fill value through `q_size` calculations. In this case, `limit_quote` will be decreased by `delta_q`. This function is used in `plan_fills_impl` to know if an order should be fully filled or partially filled.

```rust
fn limit_by_size_quote<'a>(
    limit_size: &'a mut u64,
    limit_quote: &'a mut u128, // 30440155
    used_size: &'a mut u64,
    used_quote: &'a mut u128,
) -> impl FnMut(u64, u64) -> u64 + 'a {
    move |size, price| { // size = 200; price = 249970
        let q_size = quote2size(*limit_quote, price); // 30440155 / 249970 = 121
        let fill = size.min(q_size).min(*limit_size); // fill = 121
        let delta_q = size2quote(fill, price); // delta_q = fill * price = 121 * 249970 = 30246370
        *limit_size -= fill;
        *limit_quote -= delta_q; // limit_quote = 30440155 - 30246370 = 193785
        *used_size += fill;
        *used_quote += delta_q;
        fill // fill = 121
    }
}
```

Let's say that the order that was tested for filling in `plan_fills_impl` has size 200. `limit_by_quote_size` indicates that 121 should be taken from this order.

```rust
fn plan_fills_impl(
    &self,
    mut order_price_matched: impl FnMut(NonZeroPriceMantissa) -> bool,
    mut compute_fill_amount: impl FnMut(u64, u64) -> u64,
) -> Vec<Fill> {
    let mut fills = vec![];
    // $O(log(K))$ operation per each entry in map.
    for (order_price, orders) in &self.orders {
        // If limit price is "less" than order one, i.e. can't fill it up, break
        if !order_price_matched((*order_price).into()) {
            break;
        }
        let mut maybe_fills = orders.iter().map_while(|order| {
            let fill_amount = compute_fill_amount(order.size, u64::from(*order_price)); // @POC
            ` limit_by_quote_size ` is called here,→
            if !fill_amount.is_zero() { // fill = 121, include the order.
                Some(Fill::new(
                    Order {
                        order_id: order.order_id,
                        user_id: order.user_id,
                        is_reduce_only: order.is_reduce_only,
                        limit_price: order_price.inner_value(),
                        size: order.size,
                    },
                    fill_amount,
                ))
            } else {
                None
            }
        }).peekable();
        if maybe_fills.peek().is_none() {
            break;
        }
        fills.extend(&mut maybe_fills);
    }
    fills
}
```

Then, `plan_fills_impl` does another loop round. It calls `limit_by_size_quote`, but now the price for the order to potentially fill has dropped such that `limit_quote > price`.

```rust
move |size, price| { // size = 1234; price = 167329
    let q_size = quote2size(*limit_quote, price); // 193785 / 167329 = 1 => HERE THE DIVISION RESULT IN 1!!!
    let fill = size.min(q_size).min(*limit_size); // fill = 1
    let delta_q = size2quote(fill, price);
    *limit_size -= fill;
    *limit_quote -= delta_q;
    *used_size += fill;
    *used_quote += delta_q;
    fill // fill = 1
}
```

As `limit_by_size_quote` returns 1, `plan_fills_impl` will include the current order. However, it should not have included it as the previous order was not fully filled. This will lead to keeping one element in the fills iterator in `apply_fills`, leading to breaking the assertion that the iterator is empty.

```rust

        while let Some(order) = price_orders.get_mut().peek_mut() {
            let Some(fill) = fills.next() else {
                // No more fills
                break 'outer;
            };
            // Assert order id is valid
            assert_eq!(fill.updated_order.order_id, order.order_id);
            assert_eq!(fill.updated_order.side(), S::side());
            let new_size = fill.updated_order.size;
            let remove_id = order.order_id;
            if !new_size.is_zero() { // @POC: will exit at first order we show, as 200 - 121 != 0. But there is still one element in `fills`,→
                order.size = new_size;

        }
        // If price slot is empty, erase it

            price_orders.remove();
        }

    assert!(fills.next().is_none(), "all fills were traversed"); // @POC: will break the assertion
}
```

## Recommendation
The assertion in `apply_fills` is correctly written and should be kept as is. `plan_fills_impl` must not continue testing orders to fill when an order has been partially filled. A way to obtain this behavior is to set `limit_size`, `limit_by_size`, and `limit_quote` to zero when an order is partially filled. This may be done through the following code.

```rust
fn limit_by_quote<'a>(
    limit_quote: &'a mut u128,
    used_size: &'a mut u64,
    used_quote: &'a mut u128,
) -> impl FnMut(u64, PriceMantissa) -> u64 + 'a {
    move |size, price| {
        let q_size = quote2size(*limit_quote, price);
        let fill = size.min(q_size);
        let delta_q = size2quote(fill, price);
        *limit_quote -= delta_q;
        *used_size += fill;
        *used_quote += delta_q;
        if size > fill { // if size is greater than fill, then one of the quote limit was triggered so we don't accept any more orders.,→
            *limit_quote = 0;
        }
        fill
    }
}
```
