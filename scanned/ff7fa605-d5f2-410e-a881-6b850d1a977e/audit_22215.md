# Token ownership can be transferred while minting

## Summary
This bug report is about the KinTokenSale contract which allows the owner to transfer the token ownership to a different address at any moment. This could be a problem as it would allow the owner to mint themselves an arbitrary number of tokens, which is against what was established in paper. The comment next to the function stated that transferring token ownership would prevent the token sale from continuing, and that the function acceptKinTokenOwnership would allow the sale to become the token owner again. The bug report recommends to remove the functions requestKinTokenOwnershipTransfer and acceptKinTokenOwnership, as they are not needed and compromise the correctness and security of the sale. However, the team replied that the owner will be a multisig wallet which mitigates the risk, so due to time constraints, the functions will be kept as the only method to pause the sale in an emergency.

## Details
The `KinTokenSale` contract allows the owner to transfer the token ownership to a different address at any moment, using the function [`requestKinTokenOwnershipTransfer`](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L310). Since the token is in a *minting* state during the sale (i.e. new tokens can be created by the owner), this harms the trustlessness of the contract, as it would allow the owner to mint themselves an arbitrary number of tokens, against what was established in paper.


A [comment](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L308-L309) next to the function correctly states that transferring token ownership would prevent the token sale to continue operating. If such a thing happened, the function [`acceptKinTokenOwnership`](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L316) would allow the sale to become the token owner again, and continue operating. However, in the meantime, an owner could [end the minting phase](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinToken.sol#L51) of the token manually, violating the [explicit requirement](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L257) that grants be distributed before minting is ended, and that this be done after the sale period.


We recommend to remove the functions [`requestKinTokenOwnershipTransfer`](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L310) and [`acceptKinTokenOwnership`](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L316), as they are not needed and compromise the correctness and security of the sale.


***Update:** The team replied that the owner will be a multisig wallet, which mitigates the risk. Due to time constraints, the functions will be kept as the only method to pause the sale in an emergency.*
