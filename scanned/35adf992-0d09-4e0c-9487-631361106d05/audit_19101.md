# `expectedLiquidityLimit` can be equal to zero

## Summary
This bug report states that there is an issue where the variable `expectedLiquidityLimit` can sometimes have a value of zero. The recommendation is to set a value for this variable in the constructor of the code.

## Details
##### Description
`expectedLiquidityLimit` can be equal to zero:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/pool/PoolService.sol#L145
##### Recommendation
We recommend to set value to `expectedLiquidityLimit` in constructor.
