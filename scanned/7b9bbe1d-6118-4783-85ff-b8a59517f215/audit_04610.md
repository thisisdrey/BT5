# Buyer can grief attack seller in english and blind auction 

## Summary
This bug report is about a problem with a smart contract called KimNFTMarketplace. The issue occurs after an auction for a non-fungible token (NFT) ends. If there are any bidders, the seller is unable to cancel the auction and the NFT can be sent to the bidder without their consent. This allows a malicious bidder to refuse to receive the NFT and potentially extort the seller. The report recommends adding a function for the bidder to claim the NFT at a later time if the transfer fails. The bug has been fixed in a recent update, but there is still a risk of the NFT being sent to the bidder without their consent. The fix will now return the NFT to the seller if the bidder rejects it.

## Details
## KimNFTMarketplace.sol Auction Behavior

## Context
KimNFTMarketplace.sol#L443-L455

## Description
After the English auction or Blind auction ends, the following occurs:

- If there are any bidders, the auction **cannot** be cancelled by the seller.
- Anyone can call `settleAuction()` to send the NFT to the bidder and the tokens to the seller.

When the NFT is sent, the bidder's callback function will be called, which gives the bidder the opportunity to "grief" the seller by refusing to receive the NFT. This means a malicious bidder can "extort" the seller.

Consider a scenario where the bidder is a contract that refuses to receive NFTs by default. Only by sending 1 ETH to the contract will the key variables of the contract be changed so that it can receive NFTs.

## Recommendation
If the NFT transfer fails, `settleAuction()` should ensure the payment is sent to the seller while leaving the NFT escrowed. It is recommended to provide a public function that allows the bidder to claim the NFT later.

## KIM Exchange
This issue has been fixed in PR 8.

## Cantina Managed
The fix will return the NFT to the seller if the bidder rejects the NFT, addressing the issue with the following code snippet:

```solidity
bool nftTransferSucceeded = _concludeSale(bestBid.user, bestBid.amount, true, listingId, listing, true);
if (!nftTransferSucceeded) {
    // Here the buyer refused to accept the NFT. We will protect the seller's NFT
    // by allowing the transaction to conclude with the seller safely getting their NFT back
    isNoLongerEscrowed[listingId] = true;

    _transferNFT(
        listing.nft,
        listing.maker,
        listing.tokenId,
        listing.amount,
        listing.isSemiFungible,
        false // Reentrancy protection from _transferNFT
    );
    
    // Try to refund the user
    _refundUserBid(bestBid.amount, bestBid.user, listing.currency, listingId);
}
```
