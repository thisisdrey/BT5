# amountOut Not Converted to shares

## Summary
This bug report is about the function swap() in the BentoBox application. This function is used to swap amountIn of tokenIn to amountOut of tokenOut. The amountIn is calculated by deducting the current balances (balance0 and balance1) from the previous recorded balances (reserve0 and reserve1). All these values are expressed in BentoBox’s amount (elastic). The problem is that when the amountOut is used in the function _transfer(), it requires shares instead of amount. This means that the user may receive more tokens than expected, especially when BentoBox receives significant profits from its strategy contracts. 

The testing team recommended converting amountOut to shares before transferring tokens through function _transfer(). The issue was fixed in commit ec9c147. The shares in function _transfer() were replaced with (or converted to) amount.

## Details
## Description

The function `swap()` swaps `amountIn` of `tokenIn` to `amountOut` of `tokenOut`. To calculate `amountIn`, this function deducts the current balances (`balance0` and `balance1`) with the previous recorded balances (stored in variable `reserve0` and `reserve1`). According to internal functions `_getReservesAndBalances()` and `_balance()`, all of these values (`balance0`, `balance1`, `reserve0`, and `reserve1`) are expressed in BentoBox’s amount (or elastic). This means that `amountIn` is also in amount.

The function `swap()` further uses `amountIn` to calculate `amountOut` through internal function `_getAmountOut()`. It is safe to assume that `amountOut` is also in amount, because there is no amount-shares conversion in the `_getAmountOut()` function.

The problem is that `amountOut` is used in function `_transfer()` that requires shares and not amount. This means that the user may receive more tokens than expected, especially in cases where BentoBox receives significant profits from its strategy contracts.

Function `getAmountOut()` shows the correct process, where the outcome of function `_getAmountOut()` is converted to shares to get `finalAmountOut`.

## Recommendations

The testing team recommends converting `amountOut` to shares before transferring tokens through function `_transfer()`.

## Resolution

The issue has been fixed in commit `ec9c147`. The shares in function `_transfer()` are replaced with (or converted to) amount.
