# RLQ-4 | Incorrect Truncation Avoidance

## Summary
This bug report is about the inaccuracy of the truncation in a weight calculation logic. Lines 508 and 511-512 are mathematically equivalent but should be swapped in order to avoid the truncation inaccuracy. An example is given to demonstrate this. If `oldValue` was 1 wei but `addedValue` was 1 ether, the `else if` branch on line 510 would be entered. `weightOld` would be rounded down to 0 and `weightNew` would be exactly 1e18. However, if the branch on line 508 were to be entered instead, you would get slightly less than 1e18 as the `weightNew` - providing a more accurate result. Vice versa, if `oldValue` was 1 ether but `addedValue` was 1 wei, the `if` branch on line 508 would be entered. In this case, the `weightNew` produced would be 0 as the denominator is larger than the numerator. However, if line 510 was entered, the `weightNew` produced would be 1 - providing a more accurate result.

The recommendation is to swap the weight calculation logic between lines 508 and 511-512 in order to avoid the truncation inaccuracy.
