# Cancel of already cancelled order

## Summary
This bug report is about an issue with the LimitOrderProtocol.sol contract. When a user tries to cancel an already filled order, an incorrect event is emitted. To fix this, the report recommends adding a check to the contract that requires the remaining order to not equal 1. This should prevent the incorrect event from being emitted.

## Details
##### Description
User can cancel already filled order, so incorrect event would emit:
https://github.com/1inch/limit-order-protocol/tree/a14bde6a260458de5083cee117d734221e1cbc05/contracts/LimitOrderProtocol.sol#L161
##### Recommendation
We recommend to add the following check:
```solidity=
require(_remaining[orderHash] != 1, "LOP: already filled");
```
