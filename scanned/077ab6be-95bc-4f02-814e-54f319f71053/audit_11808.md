# ETH value above limited value can be lost

## Summary
This bug report concerns the SwapExecutor contract, which has a line of code that limits the amount of ETH that can be sent to the contract. If the amount sent is above the limit, the contract will only use the limited amount, and any excess will be lost and can be withdrawn by anyone. It is recommended that the contract be redesigned so that no funds are lost in edge situations.

## Details
##### Description

SwapExecutor has the following lines for ETH:

```
if (value > valueLimit) {
    value = valueLimit;
}
(success, result) = target.call{ value: value }(data);
```

- https://github.com/BarterLab/argon/blob/22239aaccdbb78d4aa7ac5c4d0859a9b31c0fc00/contracts/SwapExecutor.sol#L116-L120

So, only a limited amount will be used if sent ETH is above valueLimit.
This ETH will be lost on the contract and anyone can withdraw these exceeded amounts.

##### Recommendation
We recommend redesigning valueLimit purpose so that no funds are lost in edge situations.
