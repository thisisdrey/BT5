# Hardcoded address

## Summary
This bug report highlights an issue with the StableXPair.sol contract, specifically on line 189. The governance address has special permissions to change the setter and collect fees, but there is no way to change this address if there is a governance change or compromise. The recommendation is to add the governance address as an argument in the initialize function or to allow the StableXFactory.sol contract to change the address. This will ensure that the contract has the ability to update the governance address if needed.

## Details
**Description**

StableXPair.sol, line 189
The governance address has unique permissions to change the setter and collect the fees.
Though there is no way to change that address and the address is hardcoded, so the contract
has no ability to change that permission in case of governance change or compromise.

**Recommendation**:

Add the governance address to the arguments of the initialize function and/or add the ability
in the StableXFactory.sol contract to change this address and pass the required one.
