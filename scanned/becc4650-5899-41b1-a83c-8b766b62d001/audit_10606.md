# DoS Through Exposed API Keys

## Summary
The bug report states that there was a security vulnerability in a file called `hardhat.config.ts` which contained important API keys for the team's mainnet, Sepolia, and Goerli networks, as well as the etherscan API key. This vulnerability could potentially lead to financial loss if an attacker gained access to the keys and used them to consume the team's Infura account balance. The recommendation is to keep the keys in a hidden file, not exposed to the public, and to use tools like "dotenv" to fetch them instead of directly exposing them in the code. It is also advised to create new keys since the existing ones were already exposed.

## Details
**Update**
Fixed in commit `e87feb8b41f62fb76158dd706dfae5206ed6391f` but new keys need to be generated.

**Description:** The `hardhat.config.ts` file exposes the team's API keys for mainnet, Sepolia, and Goerli as well as the etherscan's API key.

The impact could include a DoS through financial loss since an attacker could use the keys to consume the team's Infura account balance.

**Recommendation:** Keep the keys in a hidden file that is not exposed to the public (i.e. `.env`) and fetch them using known tools like "dotenv" so that instead of doing this `url: https://mainnet.infura.io/v3/KeyValues`, do this `url: process.env.Mainnet_Key`. Additionally, since the repo is public and the keys were already exposed, the team should create new keys. [See here for more information](https://rules.sonarsource.com/secrets/RSPEC-6783/)
