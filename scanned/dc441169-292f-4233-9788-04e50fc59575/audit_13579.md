# `maxInvocations` cannot be reduced during an update

## Summary
This bug report is about the `maxInvocations` parameter in the FantiumNFTV3.sol smart contract. This parameter is used to limit the number of invocations that can be made to the smart contract. Currently, it cannot be reduced during an update, which can lead to insolvency in the protocol.

To fix this issue, the developers recommend adding a check to ensure that `_maxInvocations > collections[_collectionId].maxInvocations`. This check will ensure that the `maxInvocations` parameter cannot be reduced during an update, thus avoiding the risk of insolvency in the protocol.

## Details
##### Description
`maxInvocations` cannot be reduced during an update because it will lead to insolvency in the protocol.

##### Recommendation
We recommend adding the following check:
`_maxInvocations > collections[_collectionId].maxInvocations`.
