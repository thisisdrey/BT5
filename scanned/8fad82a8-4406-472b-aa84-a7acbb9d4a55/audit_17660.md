# [H-14] `WETH.allowance()` returns wrong result

## Summary
A bug was reported in the WETH.allowance() function of the Plex Engineer lending-market repository. The issue is that the "return" keyword is missing from the function, causing it to always output 0. This bug was found by hansfriese and 0xf15ers and confirmed by nivasan1 (Canto). Alex the Entreprenerd (judge) increased the severity of the bug to High and commented that breaking of a core contract such as WETH is a non-starter. The recommended mitigation step is to change line 104 of the WETH.sol file to include the return keyword: `return _allowance[owner][spender];`.

## Details
_Submitted by hansfriese, also found by 0xf15ers_

WETH.allowance() returns wrong result.<br>
I can't find other contracts that use this function but WETH.sol is a base contract and it should be fixed properly.

### Proof of Concept

In this function, the "return" keyword is missing and it will always output 0 in this case.

### Tools Used

### Recommended Mitigation Steps

L104 should be changed like below.

    return _allowance[owner][spender];
