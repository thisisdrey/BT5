# [M01] ERC777 tokens may not implement ERC20 functions

## Summary
This bug report is about the Swapnet Lite Protocol and its implementation of ERC777 tokens. EIP 777 states that ERC777 tokens may implement both ERC20 and ERC777 in parallel, but this is not a requirement. The `withdraw` function in the `Escrow.sol` file uses a call to `transfer` to send tokens to a withdrawing user. However, if the ERC777 token does not implement a `transfer` function, the user will not be able to extract their tokens. 

A fix was implemented in pull request #49. The `withdraw` function now calls `send` for ERC777 tokens. This allows users to withdraw ERC777 tokens even if they do not implement a `transfer` function. Alternatively, it is suggested to consider adding a process to vet ERC777 tokens to ensure that they implement a `transfer` function which behaves like an ERC20 transfer.

## Details
[EIP 777](https://eips.ethereum.org/EIPS/eip-777) states that ERC777 tokens [“MAY implement both ERC20 and ERC777 in parallel”](https://eips.ethereum.org/EIPS/eip-777#backward-compatibility), but not that they must do so. Thus, some ERC777 tokens may not implement ERC20 standard functions such as `transfer` or `transferFrom`.


Within `Escrow.sol`, the [`withdraw` function](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L464) uses a call to [`transfer`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L473) to send tokens to a withdrawing user. There is no other way for a user to withdraw tokens aside from this function. If an ERC777 token does not implement a `transfer` function, the call to `transfer` within `withdraw` will fail (or possibly behave unexpectedly, if a `fallback` function exists in the ERC777 contract) and the user will not be able to extract their tokens.


Consider adding a process to vet ERC777 tokens to ensure that they implement a `transfer` function which behaves like an ERC20 transfer. Alternatively, consider implementing separate functionality to call the ERC777 `send` function when withdrawing ERC777 tokens.


***Update:** Fixed in [pull request #49](https://github.com/swapnet-protocol/swapnet-lite/pull/49/files#diff-f689c7b6227a1a7456ed8178ba6c25c9R495-R496). The `withdraw` function now calls `send` for ERC777 tokens.*
