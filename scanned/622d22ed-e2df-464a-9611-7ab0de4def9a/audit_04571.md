# An update to StakingRewards.rewardToken allows the owner to drain all rewardTokens

## Summary
The report discusses a bug that has been found in the code for the StakingRewards contract. This bug has been classified as a medium risk and is located in two specific lines of code. The issue is that the contract allows for certain variables to be changed, specifically the rewardToken, which can then be used to transfer unclaimed rewards to the owner. The recommendation is to not allow for the updating of reward tokens to prevent this issue from occurring. The bug has been fixed in the Berachain commit 534a0eba and has also been addressed by implementing the recommendation from the auditor in Spearbit.

## Details
Severity: Medium Risk
Context: /src/base/StakingRewards.sol#L38-L41, /src/pol/BGTStaker.sol#L81-L85
Description: StakingRewards contract leaves a number of variables mutable, namely therewardToken which has
an exposed setter in BGTStaker.sol .
After updating rewardToken , all unclaimed rewards become transferable to owner by calling
BGTStaker.rewardToken .
Recommendation: Do not allow updating of reward tokens.
Berachain: Fixed in commit 534a0eba.
Spearbit: Fixed by implementing the auditor's recommendation.
