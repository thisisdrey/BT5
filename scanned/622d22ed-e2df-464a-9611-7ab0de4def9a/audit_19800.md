# [M-03] Validator can fail to receive commission reward in `redeemAllRewards`

## Summary
The bug report submitted by jonah1005 states that there is a problem with the `redeemAllRewards` function in the `DelegatedStaking.sol` contract. This function is used by validators to receive their commission rewards, but there is a check in the code that can cause the transaction to fail even if the validator has rewards to redeem. This can be triggered by setting the `commisionRate` to a specific value. The report suggests that this may not happen often, but it could cause issues when the validator is managed by a contract. The reporter recommends calling the `redeemRewards` function within `redeemAllRewards` as a solution. The Covalent team has acknowledged the issue and the judge has classified it as a medium severity bug. They suggest limiting the `commisionRate` to prevent this issue from occurring.

## Details
_Submitted by jonah1005_

#### Impact
Validator can fail to receive commission reward by calling `redeemAllRewards`.
There's a check in `redeemAllRewards`

```solidity
uint128 rewards = sharesToTokens(s.shares, v.exchangeRate) - s.staked;
require(rewards > 0, "Nothing to redeem");
```

The validator's tx might be reverted here even if he got some commission reward to receive.

#### Proof of Concept
We can trigger the bug by setting `commisionRate` to `1e18 - 1` ([DelegatedStaking.sol L275-L276](https://github.com/code-423n4/2021-10-covalent/blob/main/contracts/DelegatedStaking.sol#L275-L276))

#### Recommended Mitigation Steps
Though this may rarely happen and the validator can redeem the reward through `redeemRewards`, this may cause some issues when the validator is handled by a contract.

I consider calling `redeemRewards` in `redeemAllReawards` as a more succinct way to do this.
