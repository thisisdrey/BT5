# Email Address Doesn’t Support Some Special Characters

## Summary
The `parse_email_address` function in `email_address.nr` does not properly handle email addresses with spaces between the right angle bracket and the address. This means that certain email addresses, such as those used with Microsoft Exchange mail servers, will not be recognized correctly. Additionally, the function does not support commonly used special characters like `+` or `_` in the address. A recommendation is to update the function to support these cases.

## Details
#### Description

The `parse_email_address` function in `email_address.nr` doesn’t support email addresses where there are spaces between the right angle bracket and the address, e.g. `<[[email protected]](/cdn-cgi/l/email-protection) >` , `< [[email protected]](/cdn-cgi/l/email-protection) >` that are compatible with the [Microsoft Exchange mail server standard](https://learn.microsoft.com/en-us/defender-office-365/anti-phishing-from-email-address-validation) . The following assert would fail when the ending character is space. The accepted ending character are `>`, `r` and `n`.

```
if email_address_sequence.end_index() < header.len() {
        assert(
            EMAIL_ADDRESS_CHAR_TABLE[header.get_unchecked(email_address_sequence.index + email_address_sequence.length)]
            == 3, "Email address must end with an acceptable character"
        );
    }

```

Additionally, the function also doesn’t support some commonly used special characters such as `+` or `_` in the address

#### Recommendation

Add support for spaces between the right angle brackets and the email address and special characters in the address body
