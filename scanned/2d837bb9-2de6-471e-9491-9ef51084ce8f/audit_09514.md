# USDB and WETH addresses are misconﬁgured in blast.sol 

## Summary
The report states that there is a bug in the Blast.sol contract file where the addresses for USDB and WETH are declared incorrectly. The correct addresses should be 0x4300000000000000000000000000000000000003 and 0x4300000000000000000000000000000000000004 respectively, as mentioned in the documentation and seen in the blast node code. This bug can cause unexpected behavior in the protocol, which may affect integrators. The recommendation is to update the addresses to the correct ones. The bug has been fixed and the addresses on the mainnet differ from those on the testnet.

## Details
## Issue Report: Blast.sol Contract

## Context
Blast.sol#L8-L9

## Description
The addresses of USDB and WETH are constants in the Blast.sol contract file; however, these variables are declared wrongly. As per the documentation, the addresses of USDB and WETH are `0x4300000000000000000000000000000000000003` and `0x4300000000000000000000000000000000000004` respectively. This value can also be seen in the precompiles of the blast node code.

The impact would be unexpected protocol behavior, which is not ideal for integrators, thus classifying it as a medium issue.

## Recommendation
Update the correct addresses for USDC and WETH accordingly.

```solidity
- address constant WETHB = 0x4200000000000000000000000000000000000022;
- address constant USDB = 0x4200000000000000000000000000000000000023;
+ address constant WETHB = 0x4300000000000000000000000000000000000004;
+ address constant USDB = 0x4300000000000000000000000000000000000003;
```

## Particle
Fixed. Mainnet addresses also differ from testnet.
