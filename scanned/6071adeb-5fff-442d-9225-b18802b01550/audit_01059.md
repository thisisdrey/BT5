# Validators can fake the number of participants for a PPS transaction

## Summary
This bug report discusses a security issue in two smart contracts called `ECDSAPPSOracle.sol` and `SuperVaultAggregator.sol`. The problem is that the function `ECDSAPPSOracle._validateProofs()` does not properly check the number of validators that have signed for a transaction and the total number of validators. This means that if enough validators collude, they can bypass a check in the function `SuperVaultAggregator._forwardPPS()`. The report recommends adding a check to ensure that the number of validators matches the number of valid signatures, and suggests fetching the total number of validators from another contract called `SuperGovernor`. This issue has been classified as a medium risk.

**Medium Risk**

- `ECDSAPPSOracle.sol` lines 174-176
- `SuperVaultAggregator.sol` lines 875-883

## Description
`ECDSAPPSOracle._validateProofs()` does not check `validatorSet` and `totalValidators`, which represent the number of validators that signed for a PPS transaction and the total number of validators. As such, validators can collude and pass incorrect data for `validatorSet` and `totalValidators` to bypass the participation rate check in `SuperVaultAggregator._forwardPPS()`:

```solidity
// C2.3) M/N Check: Check if enough validators participated
if (args.totalValidators > 0 && _strategyData[args.strategy].mnThreshold > 0) {
    // Calculate participation rate, scaled by 1e18
    uint256 participationRate = (args.validatorSet * 1e18) / args.totalValidators;
    if (participationRate < _strategyData[args.strategy].mnThreshold) {
        checksFailed = true;
        emit StrategyCheckFailed(args.strategy, "INSUFFICIENT_VALIDATOR_PARTICIPATION");
    }
}
```

The only requirement is that a sufficient number of validators must collude in order to pass quorum.

## Recommendation
Add a `validatorSet == validSignatureCount` check in `ECDSAPPSOracle._validateProofs()`. Additionally, the total number of validators can be fetched from the number of validators registered in `SuperGovernor` (i.e. `_validatorsList.length`).
