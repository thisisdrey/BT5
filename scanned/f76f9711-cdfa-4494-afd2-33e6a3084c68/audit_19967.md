# [M-19] Scoop ERC20 tokens from basket contract

## Summary
This bug report details a vulnerability in a basket contract that could be exploited to retrieve unrelated ERC20 tokens. The exploit is possible because the function settleAuction() allows users to specify an outputToken, which could be a token that is not part of the basket. Proof of concept code is provided in the report. The recommended mitigation step is to check that the outputTokens are part of the basket's tokens.

## Vulnerability details

## Impact
Suppose some unrelated ERC20 tokens end up in the basket contract  (via an airdrop, a user mistake etc)

Then anyone can do a bondForRebalance() and settleAuction() to scoop these tokens.

The function settleAuction() allows you to specify an outputToken, so also completely unrelated tokens.
Thus you can retrieve additional tokens with  settleAuction()
