# Duplicated Elements in Configuration Could Return the Wrong Amount During Swaps

## Summary
The `RevenueManagement` contract has a function that allows users to swap tokens for a specific list of output tokens with predefined ratios. However, the function does not take into account the possibility of having repeated elements in the input array, which can result in two problems: the configuration of the repeated element will be the same as its last instance, and the calculations performed during the swap may include the same element twice with the configuration of the last instance. This can potentially cause incorrect transactions and skew the 100% limitation. To fix this issue, a check should be added to ensure there are no repeated elements in the input array. This bug has been resolved in a recent update.

## Details
The `RevenueManagement` contract implements functionality which allows users to [swap](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L224) the tokens deposited in the contract for a specific list of output tokens with pre\-defined ratios. The [sum of such ratios](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L151) must equal `10_000` (or 100%) during the setup.


However, when computing that sum, the `setOutputTokensConfig` function does not take into consideration the possibility of having repeated elements in the `_outputTokens` input array. As a result, two problems arise:


* The [configuration of the repeated element](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L147) will be same as that of its last repeated instance.
* The calculations performed during the swap will include the same element twice but with the configuration of the last repeated instance. This can potentially skew the 100% limitation.


To prevent erroneous transactions during the swap after a faulty setup is not caught by the current validation, consider checking that there are no repeated elements in the `_outputTokens` input array.


***Update:** Resolved in [pull request \#63](https://github.com/radiant-capital/riz/pull/63) at commit [a98e99e](https://github.com/radiant-capital/riz/pull/63/commits/a98e99e8a7fd30982981de69f318172e1a91455d).*
