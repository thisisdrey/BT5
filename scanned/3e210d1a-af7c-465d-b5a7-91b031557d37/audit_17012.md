# Exchange rate vulnerability

## Summary
This bug report is about an exchange rate issue for new and empty pools in CToken contracts. The bug occurs when a user mints and redeems cToken, and then transfers the underlying asset from one user to the market. In this case, the user who redeems the cToken receives more tokens than expected. To prevent this bug, the report recommends checking the exchange rate before the first mint or using the 'internalCash' value for all CToken contracts.

## Details
##### Description
An exchange rate bug for new pools and empty pools (without borrowers and suppliers) for CToken contracts without the 'internalCash' variable.
Flow:
1. Create cToken
2. Mint cToken by user1 (1,000,000)
3. Redeem cToken by user1 (999,999.999999)
4. Transfer underlying (1,000,000) from user1 to market
5. Mint cToken by user2 (1,000,000)
6. Redeem cToken by user1 (user1 receive extra tokens)

##### Recommendation
We recommend checking the exchange rate before the first mint or using the 'internalCash' value for all CToken contracts.
