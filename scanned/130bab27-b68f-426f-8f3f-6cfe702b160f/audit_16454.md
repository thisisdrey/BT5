# M-7: UserManager.sol#L438-L466 : getFrozenInfo could revert due to out of gas when the vouchees array size is large

## Summary
This bug report is about an issue found in the function getFrozenInfo in the file UserManager.sol. The function traverses the vouchees array and calculates two uint256 variables, memberTotalFrozen and memberFrozenCoinAge. When the vouchees array is large, calling this function could cause the transaction to revert due to out of gas. This affects the functions where getFrozenInfo is called, such as the functions at lines 863, 881 and 889 of the same file. 

The bug was found manually by ak1 and the code snippet of the vulnerable function is available at lines 438-466 of the same file. The recommended solution is to put a cap on the number of vouchees, do operations only up to a certain index and not for the full length of the array in one shot. 

The discussion section of the report states that the vouchees array does have a cap on the length, and that the name needs to be corrected to maxVouchees. This is valid for the introduction of an index.
