# Method `getEarnings()` underflows

## Summary
The bug report describes a problem where the principalAssetDeposited is increasing for a receiver who has no shares or LP tokens. This is due to a method called getEarning(...) returning an unexpected result. A test was conducted and it showed that the earnings for the receiver after redemption was a very large value, which is caused by an underflow. It is recommended to update the logic to avoid this misleading scenario and maintain consistency between the real shares balance and the mapping for principalAssetsDeposited.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

As we notice in the above findings, the principalAssetDeposited is being increased for a receiver who has 0 LP tokens or no shares. The method getEarning(...) returns an unexpected result if it is checked for the same receiver.
