# Function in `BridgeFallback` Reverts During Retry Due to `onlySelf` Modifier

## Summary
The client has marked a bug as "Fixed" in the `BridgeFallback.sol` file. The `BridgeFallback` contract has a modifier called `onlySelf` which is meant to limit access to its `addRevert*()` functions to only be called internally. However, the `retryOnReceive` function, which is supposed to retry failed cross-chain operations, tries to call these functions from an external caller instead of the `BridgeFallback` contract itself. This results in a revert error. The recommendation is to change the call to `this.addRevertRefuel()` in the `retryOnReceive()` function to avoid this issue.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `3257348e22e6f3c1e637b08a4700ea657a211d44`.

**File(s) affected:**`src/BridgeFallback.sol`

**Description:** The `BridgeFallback` contract uses the `onlySelf` modifier to restrict access to its `addRevert*()` functions, intending for these functions to be called only internally within the `BridgeFallback` contract itself. However, the `retryOnReceive` function, designed to retry failed cross-chain operations, attempts to call these `addRevert*()` functions when a retry fails, using a `try-catch` block. In that case, the `msg.sender` will be the external caller of `retryOnReceive`, not the `BridgeFallback` contract itself. This causes a revert, because the `onlySelf` modifier evaluates as false.

**Recommendation:** Consider changing the call to be `this.addRevertRefuel()`in the `retryOnReceive()` function to avoid issues with the `onlySelf` modifier.
