# [M12] There is no security policy

## Summary
This bug report is about the [`celo-monorepo` project](https://github.com/celo-org/celo-monorepo/) not having a security policy. A security policy includes instructions for the responsible disclosure of any security vulnerabilities found in the project. It was suggested to add a policy to the repository and add links to it in the [`README.md` file](https://github.com/celo-org/celo-monorepo/blob/7be22605e172ca536c028e408b147aab83202e4a/README.md) and in the [Contributing guide](https://docs.celo.org/community/contributing). Additionally, it was suggested to add a method for secure and encrypted communication with the team, like an email address with its GPG key. This bug report has been fixed with [pull request #2906](https://github.com/celo-org/celo-monorepo/pull/2906). The [security policy document has been added](https://github.com/celo-org/celo-monorepo/pull/2906/files#diff-f1a97ccd40c28e5c0945852559d3903c).

## Details
The [`celo-monorepo` project has no security policy](https://github.com/celo-org/celo-monorepo/security/policy).


The security policy includes instructions for the responsible disclosure of any security vulnerabilities found in the project.


Consider adding a policy to the repository, and adding links to it in the [`README.md` file](https://github.com/celo-org/celo-monorepo/blob/7be22605e172ca536c028e408b147aab83202e4a/README.md) and in the [Contributing guide](https://docs.celo.org/community/contributing).


Consider adding a method for secure and encrypted communication with the team, like an email address with its GPG key.
