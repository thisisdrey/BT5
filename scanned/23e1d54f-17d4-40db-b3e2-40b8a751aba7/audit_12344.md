# Inconsistency with the documentation

## Summary
The bug report is about a discrepancy in the provided documentation for a contract called Sake. The documentation states that when a position is created, the contract will automatically stake an equivalent amount of SAKE tokens in the MasterChef contract. However, the code does not actually do this and only calls a function to unstake and liquidate tokens if the user has already staked them in the MasterChef contract. The recommendation is to update either the documentation or the code to be consistent. The client has commented that this is not actually an issue and is by design, as users are able to stake tokens themselves if they choose to. The code also includes an if statement to check if the user is already staked in the MasterChef contract before unstaking and burning tokens. The report notes that this is still inconsistent and requests an update to the documentation. Overall, the bug report is about a discrepancy between the documentation and the actual code of the Sake contract.
