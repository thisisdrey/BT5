# [M09] Owner of Portfolios can disable markets

## Summary
This bug report concerns Swapnet-lite, a protocol for trading digital assets. A bug was discovered in the Portfolios.sol contract where the owner of the contract was able to set the value of numPeriods to 0, which would cause the FutureCash.setParameters to set G_NUM_PERIODS to 0 for the FutureCash market. This would then cause every call to _isValidBlock to revert, preventing successful calls to addLiquidity, removeLiquidity, takeCollateral, and takeFutureCash. 

To fix this bug, a pull request was made to modify the removeLiquidity function so that it cannot be shut down. This bug was fixed in pull request #60. As a precaution, it was suggested that a time-delay mechanism be added between the signaled intent to close the market and the actual closing of the market. Alternatively, if the ability to immediately close the market is desired, users should be explicitly informed of this admin power in the documentation, and a means to allow users to recover their funds should be provided.

## Details
By calling [`updateFutureCashGroup`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Portfolios.sol#L129) with a [`numPeriods`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Portfolios.sol#L131) value of `0`, the `owner` of `Portfolio.sol` can cause the call to [`FutureCash.setParameters`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Portfolios.sol#L113) to set [`G_NUM_PERIODS` to `0`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L122) for that FutureCash market.


This will then cause every call to [`_isValidBlock`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L892) to [`revert`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L898), preventing successful calls to [`addLiquidity`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L266), [`removeLiquidity`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L378), [`takeCollateral`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L542), and [`takeFutureCash`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L633).


Since this change effectively shuts down the market instantly, consider adding a time-delay mechanism which allows users a grace period between the signaled intent to close the market, and the actual closing of the market. Alternatively, if the ability to immediately close the market is desired, consider informing users expicitly of this admin power in the documentation, and providing a means to allow users to recover their funds


***Update:** Fixed in [pull request #60](https://github.com/swapnet-protocol/swapnet-lite/pull/60). The `removeLiquidity` function has been modified so it cannot be shut down.*
