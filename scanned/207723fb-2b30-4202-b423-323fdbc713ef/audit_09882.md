# Subscription Loss Due To Upgrade Flaw

## Summary
The bug report describes an issue in the CreateSubscription function within the subscription module. If a user holds an active subscription plan that spans multiple months and another user upgrades it to a higher-priced plan, the original user may suffer a loss. This can happen if the upgrade involves switching to a new plan or plan block, as the original user may forfeit their prepaid benefits without adequate compensation. The report provides an example where User A has an active subscription for Plan A priced at 1000 ulava per month. User A has prepaid for 12 months, totaling to 12000 ulava. User B initiates an upgrade to User A's subscription, changing it from Plan A to Plan B for only one month at a cost of 2000 ulava. However, the total value of User A's original subscription to Plan A (12 months * 1000 ulava) exceeds the cost of one month of Plan B (2000 ulava). This results in User A losing the value of their prepaid months under Plan A when upgrading to Plan B for just one month. The report recommends ensuring that only the consumer or buyer can upgrade the plan and states that the issue has been fixed in a patch.

## CreateSubscription in the Subscription Module

In `CreateSubscription` within the subscription module, if a user holds an active subscription plans spanning multiple months and another user upgrades it to a higher-priced plan, the original user may suffer a loss. If the upgrade entails switching to a new plan or plan block, the original user may forfeit prepaid benefits without receiving adequate compensation.

## Proof of Concept

1. User A currently holds an active subscription for Plan A, priced at `1000 ulava` per month.
2. User A has prepaid their subscription to Plan A for twelve months, amounting to `1000 ulava * 12 = 12000 ulava`.
3. User B initiates an upgrade to User A’s subscription, changing it from Plan A to Plan B for only one month at the cost of `2000 ulava`.
4. However, the total value of the original subscription to Plan A (12 months * `1000 ulava`) exceeds the cost of one month of Plan B (`2000 ulava`).
5. Therefore, User A effectively loses the value of the prepaid months under Plan A when upgrading to Plan B for just one month.
