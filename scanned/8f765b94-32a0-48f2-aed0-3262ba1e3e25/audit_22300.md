# [DLT-06] Uninitialized bridge proxy implementation can be initialized and destroyed to freeze all bridged funds

## Summary
An upgradeable bridge can suffer permanent fund freezing if the implementation contract behind a proxy is left uninitialized and exposes upgrade, initialization, or self-destruct functionality. An attacker can initialize the implementation directly, gain implementation-level privileges, and destroy or corrupt code used by the proxy.

This is a critical bounty pattern because bridge proxies often custody large assets while delegating execution to shared implementation contracts.

## Details
The vulnerable pattern is:

```solidity
contract BridgeImplementation {
    address owner;

    function initialize(address newOwner) external {
        require(owner == address(0), "initialized");
        owner = newOwner;
    }

    function destruct() external onlyOwner {
        selfdestruct(payable(owner));
    }
}
```

The proxy may be initialized correctly, but the implementation contract itself remains callable. If an attacker calls `initialize()` on the implementation address directly, they become owner of the implementation. Any dangerous implementation-only function, upgrade hook, delegatecall target, or self-destruct path can then be abused.

If the implementation bytecode is destroyed or corrupted, every proxy delegating to that implementation can become unusable. Users may be unable to transfer, redeem, or withdraw bridged assets.

## Impact
- Permanent freezing of bridged funds.
- Loss of bridge availability across all proxies using the implementation.
- Emergency upgrade or migration required to recover.

## Recommendation
- Disable initializers in implementation constructors.
- Remove self-destruct and arbitrary delegatecall functionality from implementations.
- Verify implementation contracts are initialized or locked immediately on deployment.
- Monitor for direct calls to implementation addresses.
- Add deployment tests that assert implementation initialization cannot be called post-deploy.
