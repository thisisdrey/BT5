# Insecure Encryption On Seed

## Summary
The SnapAlgo application uses a method from the CryptoJS library to encrypt sensitive information, but this method lacks authentication. This makes it vulnerable to attacks where an attacker can decrypt the information by repeatedly trying different methods and observing the results. To fix this issue, the application needs to switch to a more secure cipher mode called GCM. This has been done in a recent update to the application by replacing the old library with a new one called tweetnacl, which uses a different encryption algorithm that is not susceptible to these types of attacks.

## Details
## SnapAlgo Security Overview

SnapAlgo uses the CryptoJS library’s encryption method in its default mode to safeguard sensitive information, such as the seed, while an account is imported. When the library is utilized in its default mode, the PKCS7 and AES-CBC padding schemes are employed. However, this mode lacks authentication, implying that the ciphertext may be manipulated to a certain extent in a predictable and intelligible manner.

Consequently, it is well-known that the PKCS7 padding scheme is susceptible to oracle attacks. More specifically, if an attacker can decrypt the ciphertext repeatedly and monitor the outcome of whether the decryption succeeds or fails, the attacker could be enabled to acquire the plaintext message.

## Code Example

```javascript
const address = algo.encodeAddress(keys.publicKey);
let b64Seed = Buffer.from(seed).toString('base64');
const key = await this.#getencryptionKey();
const encryptedSeed = AES.encrypt(b64Seed, key).toString();
```

## Remediation

Migrate to an authenticated cipher mode such as GCM instead of utilizing the CBC cipher mode. As a modern cipher mode, GCM does not require a padding scheme and is not vulnerable to oracle attacks.

## Patch

Fixed in `81414a9` by introducing the tweetnacl library. This library is a JavaScript implementation of the NaCl cryptographic library, which provides authenticated encryption using the XSalsa20-Poly1305 secret-key algorithm.

## Updated Code Example

```javascript
function encryptNACL(secretKey, message) {
  const nonce = nacl.randomBytes(24);
  const secretKeyUint8Array = new Uint8Array(secretKey);
  const messageUint8Array = new TextEncoder().encode(message);
  const encryptedMessage = nacl.secretbox(messageUint8Array, nonce, secretKeyUint8Array);
  
  const fullMessage = new Uint8Array(nonce.length + encryptedMessage.length);
  fullMessage.set(nonce);
  fullMessage.set(encryptedMessage, nonce.length);
  
  const base64Encoded = Buffer.from(fullMessage).toString('base64');
  return base64Encoded;
}
```
