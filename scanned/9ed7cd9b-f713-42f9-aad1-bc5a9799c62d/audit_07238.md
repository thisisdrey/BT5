# Possible to Have `signatures_required == 0` for a Multi-Sig Auth Descriptor

## Summary
The client has marked a bug as "fixed" in the file `rell/src/lib/ft4/core/accounts/module.rell`. The issue is related to a function called `add_auth_descriptor()` which does not properly validate the input it receives. This allows for a potential vulnerability where the number of required signatures can be set to zero, making it possible for anyone to use the authentication descriptor. The report includes an example of how this vulnerability can be exploited and recommends updating the function to validate that the number of required signatures is greater than zero.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `d3bde59b3461ec6ede29afdd9bbade01bb54686e`.

**File(s) affected:**`rell/src/lib/ft4/core/accounts/module.rell`

**Description:**`add_auth_descriptor()` makes a call to `validate_auth_descriptor_args()`. Unlike `validate_auth_descriptor_rules()`, `validate_auth_descriptor_args()` doesn't perform any validation on the auth descriptor. This makes it possible to set `signatures_required == 0` for the `multi_sig_args`.

In the case of `ft_auth`: Given how the `for` loop in `check_multi_sig_auth()` works, at least one signature is required to reach the `if` statement checking whether the number of signatures has been reached. However, should this implementation ever change, the vulnerability may leak through.

In the case of `evm_auth`: `signatures_required == 0` allows **anyone** to use the auth descriptor.

**Exploit Scenario:**

```
function test_should_not_allow_zero_signatures_required() {
  val auth_descriptor = test_utils.create_auth_descriptor(alice.pub, ["A", "T"]);
  admin.external.register_account(auth_descriptor)
    .sign(asset_utils.admin_priv_key())
    .run();

  val auth_descriptor2 = accounts.auth_descriptor(
    accounts.auth_type.M,
    [
        ["T"].to_gtv(),
        0x0.to_gtv(),
        [
          bob.pub,
          alice.pub
        ].to_gtv()
    ],
    rules = null.to_gtv()
  );

  rell.test.tx()
      .op(test_utils.ft_auth_operation_for(alice.pub))
      .op(accounts.external.add_auth_descriptor(auth_descriptor2))
      .sign(alice)
      .sign(bob)
      .run();

  assert_null(accounts.account_auth_descriptor @? { .id == auth_descriptor2.hash() });
}

function test_multisig_external_signature_should_require_signatures() {
  val alice_address = crypto.eth_privkey_to_address(rell.test.keypairs.alice.priv);
  val bob_address = crypto.eth_privkey_to_address(rell.test.keypairs.bob.priv);
  val desc = accounts.auth_descriptor(
    accounts.auth_type.M,
    [
      set(["A", "T"]).to_gtv(),
      (0).to_gtv(),
      [alice_address, bob_address].to_gtv()
    ],
    rules = null.to_gtv()
  );

  utils._create_test_acc_from_auth_desc_single_sig(rell.test.keypairs.bob, desc);

  val op = test_operations.test_auth.test_perform_large_transfer(1000, to_addr);

  val account_id = [alice_address, bob_address].hash();
  val auth_descriptor_id = desc.hash();
  rell.test.tx()
    .op(auth.external.evm_auth(account_id, auth_descriptor_id, []))
    .op(op)
    .run_must_fail();
}
```

**Recommendation:** Update `validate_auth_descriptor_args()` to validate that `signatures_required > 0`.
