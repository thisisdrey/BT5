# Non-forfeitable wallets can be Forfeited 

## Summary
This bug report describes an issue with the WeirollWallet contract where the forfeit() function can be called on non-forfeitable wallets. The contract includes a field meant to determine if the function can be called, but it is not checked in the forfeit() function and an associated event is not emitted. The recommendation is to add code to the forfeit() function to check if the wallet is forfeitable before allowing it to be forfeited. The bug has been fixed by Canitna Managed.

## Description
The `WeirollWallet` contract includes an immutable field (`WeirollWallet.sol#L86`) meant to determine whether the `forfeit()` function can be called or not. However, this field is not checked in the `forfeit()` function, and an associated event, `WalletNotForfeitable`, is never emitted. As a consequence, non-forfeitable wallets can be forfeited.

## Recommendation
Add appropriate code to the `forfeit()` function, e.g.:
```solidity
function forfeit() public onlyOrderbook {
    if (!isForfeitable()) {
        revert WalletNotForfeitable();
    }
    forfeited = true;
}
```
