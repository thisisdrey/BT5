# `merkleProof` length not checked

## Summary
The report states that there is a bug in the `merkleProof` function of the Gearbox protocol. This function does not check the length of the `merkleProof` input, which can cause problems. The recommendation is to add a check that ensures the `merkleProof` input has a length greater than 0. This will prevent any potential issues from occurring.

## Details
##### Description
`merkleProof` length is not checked:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/core/AccountMining.sol#L64
##### Recommendation
We recommend adding a check that `merkleProof.length > 0`.
