# [H-02] `Basket.sol#auctionBurn()` A failed auction will freeze part of the funds

## Summary
The bug report submitted by WatchPug is related to a problem in the `auctionBurn()` function of the `Basket.sol` contract. This function is responsible for burning the auction bond, but it does not update the `ibRatio` value. As a result, when the bond of a failed auction is burned, the proportional underlying tokens won't be able to be withdrawn, which means they are frozen in the contract.

To demonstrate the issue, WatchPug provided a proof of concept with a specific configuration of the contract. It involves creating an auction, bonding with 1 BASKET TOKEN, and waiting for 24 hrs before calling the `auctionBurn()` function. The result is that the `basket.ibRatio` remains at 1e18, while the basket.totalSupply is reduced to 399. This means that only 1 BTC and 1 ETH are returned, leaving 1 BTC and 1 ETH frozen in the contract.

WatchPug recommended a mitigation step to fix the problem, which involves changing the code of the `auctionBurn()` function. This code update ensures that the `ibRatio` is updated when the bond is burned, thus allowing the proportional underlying tokens to be withdrawn.

The severity of the issue was confirmed by frank-beard (Kuiper) and Alex the Entreprenerd (judge) commented that it is a high severity finding as it is a bug in the logic of the protocol that can lead to funds being irrevocably lost.

## Details
_Submitted by WatchPug_

<https://github.com/code-423n4/2021-09-defiProtocol/blob/main/contracts/contracts/Basket.sol#L102-L108>

Given the `auctionBurn()` function will `_burn()` the auction bond without updating the `ibRatio`. Once the bond of a failed auction is burned, the proportional underlying tokens won't be able to be withdrawn, in other words, being frozen in the contract.

#### Proof of Concept

With the configuration of:

basket.ibRatio = 1e18
factory.bondPercentDiv = 400
basket.totalSupply = 400
basket.tokens = \[BTC, ETH]
basket.weights = \[1, 1]

1.  Create an auction;
2.  Bond with 1 BASKET TOKEN;
3.  Wait for 24 hrs and call `auctionBurn()`;

`basket.ibRatio` remains to be 1e18; basket.totalSupply = 399.

Burn 1 BASKET TOKEN will only get back 1 BTC and 1 ETH, which means, there are 1 BTC and 1 ETH frozen in the contract.

#### Recommended Mitigation Steps

Change to:

```solidity
function auctionBurn(uint256 amount) onlyAuction external override {
    handleFees();
    uint256 startSupply = totalSupply();
    _burn(msg.sender, amount);

    uint256 newIbRatio = ibRatio * startSupply / (startSupply - amount);
    ibRatio = newIbRatio;

    emit NewIBRatio(newIbRatio);
    emit Burned(msg.sender, amount);
}
```
