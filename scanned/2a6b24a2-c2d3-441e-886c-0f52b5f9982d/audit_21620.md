# Re-entrancy attack allows to buy EternalHeroes cheaper ✓ Addressed

## Summary
This bug report is about the `_buy` function of the `EternalHeroesFactory` contract. The issue is that buyers can do re-entracy before items are minted, which allows them to buy more items with the old price. To solve this issue, the recommendation is to add re-entrancy protection or mint items before sending the refund. This was addressed in the pull request [horizon-games/SkyWeaver-contracts#4](https://github.com/horizon-games/SkyWeaver-contracts/pull/4), and re-entrancy guard was added to [horizon-games/SkyWeaver-contracts#10](https://github.com/horizon-games/SkyWeaver-contracts/pull/10).

#### Description


When buying eternal heroes in `_buy`  function of `EternalHeroesFactory` contract, a buyer can do re-entracy before items are minted.


**code/contracts/shop/EternalHeroesFactory.sol:L278-L284**



```
uint256 refundAmount = \_arcAmount.sub(total\_cost);
if (refundAmount > 0) {
  arcadeumCoin.safeTransferFrom(address(this), \_recipient, arcadeumCoinID, refundAmount, "");
}

// Mint tokens to recipient
factoryManager.batchMint(\_recipient, \_ids, amounts\_to\_mint, "");

```
Since price should increase after every `N` items are minted, it’s possible to buy more items with the old price.


#### Recommendation


Add re-entrancy protection or mint items before sending the refund.
