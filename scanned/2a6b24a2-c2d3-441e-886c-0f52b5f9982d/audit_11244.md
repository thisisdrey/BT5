# An incorrect oracle update

## Summary
The bug report describes an issue with the EMA oracle for the `D` value in the CurveStableSwapNG.vy code. The oracle is currently updated with the `D2` value which accounts for all fees, but it should be updated with a value that accounts only for admin fees. To fix this issue, the report recommends calculating the `D3` value that accounts only for admin fees and using this value for the oracle update.

## Details
##### Description
EMA oracle for `D` is updated with the `D2` value here https://github.com/curvefi/stableswap-ng/blob/8c78731ed43c22e6bcdcb5d39b0a7d02f8cb0386/contracts/main/CurveStableSwapNG.vy#L598 but it should be updated with a slightly different value since `D2` accounts for all fees, but the oracle should be updated with the value that accounts only admin fees.

##### Recommendation
We recommend calculating the `D3` value that accounts only for admin fees and using this value for the oracle update.
