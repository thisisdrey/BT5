# [M-02] Excessive msg.value lost when buying

## Summary
This bug report highlights a potential issue with the `msg.value` equation, which can result in users losing their funds forever. The severity of this bug is high, but the likelihood of it occurring is low, as it requires a mistake from the user. The problem arises when the `msg.value` is considered incorrect in the equation `msg.value < price + protocolFee + creatorFee`. This means that if a user accidentally provides a higher `msg.value` than the expected price plus fees, the excess funds will not be returned and will be lost. This mistake can happen due to incorrect fee calculations or if the user has performed sell operations before making the transaction. To prevent this issue, it is recommended to either strictly require the `msg.value` to be the exact price plus fees, or to return any excess funds back to the user, although this option may be riskier.

## Details
**Severity**

**Impact:** High, funds lost forever

**Likelihood:** Low, requires a mistake from a user

**Description**

`msg.value` is considered incorrect only in this equation:

```
msg.value < price + protocolFee + creatorFee
```

As a result, `msg.value` above price+fees will pass, but never return and thus lost.

It can happen in case of a mistake from users if they provide the wrong `msg.value`.

This mistake can be either from fee calculations, or it can be some sell operations before the user, so the total price is less than expected.

**Recommendations**

Consider either having a strict requirement to have msg.value the exact price+fees (better), or return excessive funds back to the user (more risky).
