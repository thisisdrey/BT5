# Node Operators ETH Can Be Mistakenly Sent To The Deposit Pool

## Summary
This bug report is about a problem with the Node Operator deposit process. If a Node Operator has sufficient deposit credit but also sends ETH when depositing, they lose their additional ETH. The problem is that the function does not check the msg.value to see if the Node Operator also sent ETH. This can happen if the Node Operator forgets or inadvertently sends a deposit value when they have enough credit for a deposit bond. The recommendation is to either return the excess ETH to the Node Operator or revert the transaction if msg.value != 0 to prevent excess ETH being lost to the network.

## Details
## Description

Node operators that have a sufficient deposit credit and additionally send ETH when depositing, lose their additional ETH. On line [72], the conditional case that an operator has sufficient credit to create a minipool does not check `msg.value` to see if the node is also sending additional ETH.

It can be the case that a Node Operator forgets or inadvertently sends a deposit value when it has sufficient credit for a deposit bond. The function accepts the ETH without returning the excess to the node operator.

## Recommendations

Either return the excess ETH to the node operator or revert the transaction if `msg.value != 0` to prevent excess ETH from being lost to the network.
