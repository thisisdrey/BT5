# Users wouldn't cancel their proposals due to the increased `proposalThreshold`.

## Summary
This bug report describes a problem where users are unable to cancel their proposals due to a change in the `proposalThreshold` parameter. This parameter can be changed by a specific function, but when a user tries to cancel their proposal, it checks their voting power against the new `proposalThreshold` and reverts if they do not have enough voting power. This means that users are unable to cancel their proposals if the `proposalThreshold` has been increased since they made their proposal. 

The impact of this bug is that users are unable to cancel their proposals, which can be frustrating and limit their ability to participate in the platform. To mitigate this issue, it is recommended to cache the `proposalThreshold` as a proposal state. The client has acknowledged this issue and will ensure that parameters are only changed when there are no pending proposals.

## Details
**Severity:** Medium

**Description:** When users call `cancel()`, it validates the caller's voting power with `proposalThreshold` which can be changed using `setProposalThreshold()`.

```solidity
  function setProposalThreshold(uint256 newProposalThreshold) external {
    if(msg.sender != multisig) revert NotMultisig();
    if(newProposalThreshold < MIN_PROPOSAL_THRESHOLD || newProposalThreshold > MAX_PROPOSAL_THRESHOLD) revert InvalidVotingParameter();
    uint256 oldProposalThreshold = proposalThreshold;
    proposalThreshold = newProposalThreshold;
    emit ProposalThresholdSet(oldProposalThreshold, proposalThreshold);
  }
```

Here is a possible scenario.
- Let's assume `proposalThreshold = 100` and a user has 100 voting power.
- The user has proposed a proposal using `propose()`.
- After that, `proposalThreshold` was increased to 150 by `multisig`.
- When the user calls `cancel()`, it will revert as he doesn't have enough voting power.

**Impact:** Users wouldn't cancel their proposals due to the increased `proposalThreshold`.

**Recommended Mitigation:** It would be good to cache `proposalThreshold` as a proposal state.

**Client:** Acknowledged, we will ensure to only change parameters while there are no pending proposals.

**Cyfrin:** Acknowledged.
