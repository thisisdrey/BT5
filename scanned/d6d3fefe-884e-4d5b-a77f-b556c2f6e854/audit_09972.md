# Inconsistency With Documentation

## Summary
The bug report discusses an issue with the Tensor website's documentation regarding NFT orders. The website claims that the order's funder will receive either their original NFT or the minimum amount of SOL specified. However, the code allows for the taker to swap out the collateral NFT with a different one from the same collection, resulting in the maker receiving a different NFT than expected. The report provides a proof of concept and suggests remediation by ensuring that the order type matches with what the taker receives or explicitly stating on the website that the maker may receive a random NFT from the same collection. The issue has been patched by updating the website.

## Details
## Documentation Overview

The documentation on Tensor’s website assures the order’s funder that they will receive either their original NFT or the minimum amount of SOL the funder specified. However, according to the code, the maker may end up with a different NFT from the same collection after executing the order. 

The taker may swap out the order’s collateral NFT with a different one from the same collection by depositing the amount of Lamports required for the new NFT into the `order_vault` and purchasing the new NFT. At this point, there are two NFTs in the `order_vault`: one is the original, and the second is the one the taker transfers. Then, the taker calls another trade function to receive the original NFT and provide SOL. As a result, the taker obtains the order’s collateral NFT, while the Maker receives the random NFT provided by the taker.

## Proof of Concept

1. The maker initializes the order with `OrderType NFT` and transfers NFT A to the order.
2. The taker transfers an amount of Lamports equal to `order_state.price` directly to `order_vault`.
3. The taker calls `claim_compressed_receive_sol` with NFT B. (The taker may transfer an NFT to an order with NFT collateral).
4. The taker calls `claim_compressed_receive_nft` with NFT A. `order_vault` receives `order_state.price` SOL. Now `order_state` has NFT B and `order_state.price` SOL.

## Remediation

Ensure either that `order_state.order_type` matches with `taker_receives_nft` or explicitly state on the website that the maker agrees to get back one of two: either a random NFT from the same collection or the minimum amount of SOL they specified.

## Patch

Fixed by updating Tensor’s website.
