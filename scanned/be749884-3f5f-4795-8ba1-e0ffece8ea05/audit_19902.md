# [M-01] Incentive should check that it hasn't started yet

## Summary
This bug report concerns the `ConcentratedLiquidityPoolManager.addIncentive` function, which can add an incentive that already has a non-zero `incentive.secondsClaimed`. This would cause rewards to be calculated incorrectly, leading to assets being at risk after a wrong governance action - a severity 2 issue.

The recommended mitigation step is to add a check into the function, requiring that `incentive.secondsClaimed` is equal to zero. This was confirmed by sarangparikh22 (Sushi) and commented on by alcueca (judge).

## Details
_Submitted by cmichel, also found by pauliax_

The `ConcentratedLiquidityPoolManager.addIncentive` function can add an incentive that already has a non-zero `incentive.secondsClaimed`.

#### Impact
Rewards will be wrong.

#### Recommended Mitigation Steps
Add a check: `require(incentive.secondsClaimed == 0, "!secondsClaimed")`.
