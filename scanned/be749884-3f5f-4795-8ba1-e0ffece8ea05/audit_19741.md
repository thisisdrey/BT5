# Invalid floats comparison

## Summary
The bug report is about a statement in a file called "processor.rs" that is causing problems. The statement is "ui_amount != 1.0" and there are two issues with it. First, it should be written as "ui_amount != 1.0f64" to be explicit. Second, comparing float numbers using strict comparison is not safe. The report suggests using "(ui_amount - 1.0f64).abs() > error_margin" instead.
