# DEPOSITING TO SECONDARY TOKENS DOES CAUSE THE CONTRACT TO LOCK

## Summary
The bug report describes an issue with the code in the WeightedFarmingPool.sol file. The code is assuming that the user's amount will contain an array for all newly added tokens to the pool. However, this is not always the case when a user makes their first deposit on a token with an index that is not equal to zero. This can cause problems with the calculations and potentially lead to incorrect results. The bug has been given a score of 5 for both impact and likelihood, meaning it is a significant issue that is likely to occur. 

The recommended solution for this bug is to check the length of the user's amount array and compare it to the index. If there is a difference, empty values will be pushed to ensure that the arrays are the same length. This will prevent any errors from occurring and ensure that the calculations are accurate.

## Details
##### Description

The code under `_deposit` is assuming that `user.amount` will contain an array for all newly added tokens to the pool, which is not the case if the user performs a deposit for the first time on a token whose index is `!=0`.

Code Location
-------------

#### src/reward/WeightedFarmingPool.sol

```
// check if current index exists for user
if (user.amount.length < index + 1) {
    user.amount.push(0);
}
if (pool.amount.length < index + 1) {
    pool.amount.push(0);
}

user.amount[index] += _amount;
user.share += _amount * pool.weight[index];

pool.amount[index] += _amount;
pool.shares += _amount * pool.weight[index];

user.rewardDebt = (user.share * pool.accRewardPerShare) / SCALE;

```

##### Recommendation
