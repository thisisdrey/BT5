# Front-running Can Lead to Stolen Validator Rewards

## Summary
The report describes a bug in the `UpdateManager.sol` file where the `update()` function can be called by anyone, potentially allowing an attacker to claim a reward intended for validators. The client has acknowledged the bug and recommends restricting the function to only be called by validator nodes or making users aware of the risk.

## Details
**Update**
Marked as "Acknowledged" by the client. The client provided the following explanation:

> This is intended, the reward is intended for anybody who send the update. The reward should cover the update gas fee, should not be set much higher than it.

**File(s) affected:**`UpdateManager.sol`

**Description:** The `update()` function is callable by anyone, allowing an attacker that monitors the transaction pool to submit a transaction calling the function with correct parameters and a higher gas price. As the function transfers the contract balance to the function caller, the attacker may be able to claim a reward intended for validators.

**Recommendation:** Restrict the function to only be called by validator nodes, or make clear of this inherent risk to users.
