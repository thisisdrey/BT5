# Possible overflow in loop index variable

## Summary
This bug report is about the `airdropTokens` function of the `PolyDistribution` contract. This function takes an array of addresses as an argument and “airdrops” tokens to each of the addresses. To do this, a `for` loop is used with an index variable of type `uint8`. This will cause an overflow for arrays which have more than 255 addresses, resulting in an out of gas error and wasting gas. 

The suggested solution is to use a `uint256` variable for the index. This has been fixed in commit `0b47ae4`.

## Details
The [`airdropTokens`](https://github.com/PolymathNetwork/polymath-token-distribution/blob/672fabe081e8f90ea025252d92c2eb247d60010e/contracts/PolyDistribution.sol#L117) function of the [`PolyDistribution`](https://github.com/PolymathNetwork/polymath-token-distribution/blob/672fabe081e8f90ea025252d92c2eb247d60010e/contracts/PolyDistribution.sol) contract takes an array of addresses as a parameter in order to “airdrop” tokens to each of them. To do so, a `for` [loop](https://github.com/PolymathNetwork/polymath-token-distribution/blob/672fabe081e8f90ea025252d92c2eb247d60010e/contracts/PolyDistribution.sol#L120) is used, with an index variable of type `uint8`. This will result in an overflow for arrays which have more than 255 addresses. The loop will indefinitely iterate over the first 255 addresses, eventually failing with an out of gas error, wasting gas.  

Consider using a `uint256` variable for the index.  

***Update**: Fixed in commit [`0b47ae4`](https://github.com/PolymathNetwork/polymath-token-distribution/commit/0b47ae467f95a02c6b71421e5816b5d50b698158#diff-894d6a69e0d58851828ece9513b36f5fR119).*
