# Unchecked Return Value of transferFrom in requestRedemption

## Summary
The bug report states that there is an issue with the transferFrom function in the Hub.sol file. This function is used to transfer tokens from the user to the operatorBurner address. However, the return value of this function is not being checked to confirm if the transfer was successful. This is important because the function returns a boolean value, which must be checked to ensure the transfer was completed correctly. The bug has been resolved and the recommendation is to always check the return value of the transferFrom function to avoid any issues.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

The transferFrom function on Hub.sol is called to transfer tokens from the user to the operatorBurner address. However, this function’s return value is not checked to confirm that the transfer was successful and since the function returns a boolean, the return value must be checked.

**Recommendation:**

Check the return value of transferFrom to ensure the transfer was successful.
