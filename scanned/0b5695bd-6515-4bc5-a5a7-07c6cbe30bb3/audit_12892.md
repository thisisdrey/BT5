# PBKDF2 `keySize` and `iterations` Insufficiently Large

## Summary
The client has marked a bug as "Fixed" and provided an explanation for the fix. The bug was related to the use of PBDKF2 to encrypt and decrypt secret information, which was using a relatively low number of iterations and producing a key with a small size. This could potentially make it easier to crack the hash. The report recommends following the OWASP recommendation for using a higher number of iterations depending on the hash size.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `8b90784`. The client provided the following explanation: This is fixed by [HAS-1](https://certificate.quantstamp.com/full/hash-pack/95a96750-4624-412c-876e-5965dc021e70/index.html#findings-qs1)

**Description:** The password for encrypting and decrypting secret information is hashes using PBDKF2 with 10000 iterations and producing a key with 128 bits. However, in the event that such a hash is uncovered, it may be easy to crack given the relatively low key size and few iterations used to generate it.

As of 2023, [OWASP recommends](https://cheatsheetseries.owasp.org/cheatsheets/Password_Storage_Cheat_Sheet.html#pbkdf2) that

*   for a 128-bit hash, 1300000 iterations should be used;
*   for a 256-bit hash, 600000 iterations should be used;
*   for a 512-bit hash, 210000 iterations should be used.

**Recommendation:** We recommend abiding by the OWASP recommendation described above.
