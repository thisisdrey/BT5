# Inconsistent Withdrawal Fee Accounting

## Summary
The `Collateral` contract has a bug where the calculation of the `amountWithFee` in the `claimCollateral` function is not consistent with the formula used in the `getClaimableAmount` function. This results in a slight discrepancy in the amount of collateral that can be claimed due to fees. The bug has been fixed in a recent update.

## Details
In the `Collateral` contract, the calculation of the [`amountWithFee`](https://github.com/AmperaFoundation/sol-contracts/blob/4c4423791b3427153937881fc5287a81283ee141/contracts/Collateral.sol#L337) amount within the `claimCollateral` function is not consistent with the formula of the `getClaimableAmount` function. More specifically, the formula in the `claimCollateral` function is currently:


`amountWithFee = (_amountToReceive * (10_000 + feeBasisPoint)) / 10_000`


Whereas according to [`getClaimableAmount`](https://github.com/AmperaFoundation/sol-contracts/blob/4c4423791b3427153937881fc5287a81283ee141/contracts/Collateral.sol#L730) it should be:


`amountWithFee = (_amountToReceive * 10_000) / (10_000 - feeBasisPoint)`


The latter formula results in a slightly greater `amountWithFee` and is also used in [`reserveClaimableCollateral`](https://github.com/AmperaFoundation/sol-contracts/blob/4c4423791b3427153937881fc5287a81283ee141/contracts/Collateral.sol#L311). This results in slightly more collateral being reserved to account for the fee, compared to the calculation in `claimCollateral`, which will calculate less fee. Hence, it is possible to claim slightly more collateral due to the fee discrepancy.


Consider using the same formula consistently when calculating the claimable or withdrawal fee amount. This will help avoid accounting errors in the contract.


***Update:** Resolved in [pull request \#151](https://github.com/AmperaFoundation/sol-contracts/pull/151).*
