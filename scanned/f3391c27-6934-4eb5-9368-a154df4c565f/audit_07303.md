# Any covenant committee member can prevent all BTC stakers from successfully constructing a valid Unbonding Transaction 

## Summary
This bug report discusses two separate issues in the CLI Tools project related to constructing Unbonding Transactions in the Babylon blockchain. The first issue is that the BTC staker does not verify the correctness of signatures from covenant committee members, which could result in an invalid Unbonding Transaction being uploaded to the blockchain. This could be exploited by a malicious committee member and prevent all BTC stakers from successfully creating valid Unbonding Transactions.

The second issue is related to staking transactions and the system's configuration. If a staker creates a transaction with the current configuration, but the configuration changes before the transaction is included in a block, the transaction may be considered invalid and the staker's funds will be locked for the entire staking period. This could happen frequently as there is no mechanism in place to ensure backward compatibility in the configuration.

Another issue raised in the report is that the withdrawal transaction output value can go below the dust limit and even become negative. This could lead to errors and break the protocol's functionality. The recommended solution is to validate the output value after subtracting the estimated fee to ensure it can cover the fee and avoid negative values.

To address these issues, the Babylon team has fixed the bugs in the CLI Tools and BTC staking code. They have also recommended solutions to prevent similar issues in the future, such as ensuring backward compatibility in the configuration and properly validating output values.

## Description
1. In the CLI Tools project, a BTC staker needs to request all covenant committee members to sign the transaction to construct an Unbonding Transaction. The BTC staker collects the required quorum of signatures based on the order of arrival of the responses.
2. In the function `requestSigFromCovenant`, the BTC staker does not verify the correctness of the signatures. If a malicious covenant committee member returns an incorrect signature, the BTC staker will include it in the Unbonding Transaction and upload it to the Babylon blockchain. The Babylon blockchain will reject this Unbonding Transaction.
3. Since a malicious covenant committee member does not need to spend time computing the signature, its response will return early to the BTC staker. This invalid response will enter the quorum signature collection, causing the BTC staker's Unbonding Transaction to remain invalid.
4. The location where the responses from covenant committee members are not verified: `cli-tools/internal/services/unbonding_pipeline.go#186`.

## Impact
Prevents all BTC stakers from successfully constructing a valid Unbonding Transaction.

## Likelihood
Only requires one malicious covenant committee member, and the covenant committee member will not be punished.

## Recommendation
It is recommended that the BTC staker verify the signatures returned by the covenant committee members.
