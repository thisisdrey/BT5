# Non-Comprehensive Test Suite

## Summary
This bug report highlights that the current test suite for the `BalancerMetaPoolStrategy` is incomplete and does not cover all the functionality. Some important features, such as `deposit` and `safeApproveAllTokens`, do not have tests, which could lead to potential issues and incorrect results. Additionally, the existing tests do not cover certain corner cases, such as stale oracle prices and different deposit/withdrawal scenarios. The report suggests that increasing test coverage would not only help maintain the correctness of the code but also serve as documentation for developers and auditors. The bug has been resolved in a recent pull request, where the Origin team has added tests for `safeApproveAllTokens`, `BAL` and `AURA` token rewards, and improved testing for `maxDepositSlippage` and the underlying value of the Strategy. However, some tests, such as `deposit` and `withdrawAll` on a subset of assets, are still missing.

## Details
The current test suite does not cover all the functionality in the `BalancerMetaPoolStrategy`.


* There are no tests for [`deposit`](https://github.com/OriginProtocol/origin-dollar/blob/eb11498c376b65696c90981757221b076d6226aa/contracts/contracts/strategies/balancer/BalancerMetaPoolStrategy.sol#L36), only for [`depositAll`](https://github.com/OriginProtocol/origin-dollar/blob/eb11498c376b65696c90981757221b076d6226aa/contracts/contracts/strategies/balancer/BalancerMetaPoolStrategy.sol#L67). This can be problematic, as supplying only one token at a time will incur more slippage and potentially return fewer BPTs than expected, as opposed to joining the Balancer pool with multiple supported assets at the same time.
* There are no tests for [`safeApproveAllTokens`](https://github.com/OriginProtocol/origin-dollar/blob/eb11498c376b65696c90981757221b076d6226aa/contracts/contracts/strategies/balancer/BalancerMetaPoolStrategy.sol#L421).
* There are no tests to ensure that `BAL` and `AURA` tokens are actually rewarded when calling [`collectRewardTokens`](https://github.com/OriginProtocol/origin-dollar/blob/eb11498c376b65696c90981757221b076d6226aa/contracts/contracts/strategies/balancer/BaseAuraStrategy.sol#L76).


Additionally, because of the significant number of external dependencies, there are multiple corner cases that are not covered in the existing test suite:


* If price oracles become stale and report outdated prices
* If `withdrawAll` is called when supported strategy assets are a subset of the pool's assets
* Based on different deposited/withdrawn values as well as oracle staleness, what `maxDepositSlippage` and `maxWithdrawalSlippage` should be in order to prevent DoS while depositing and withdrawing
* Whether the underlying value of the Strategy will be reported correctly irrespective of the liquidity pool being balanced


Additional tests would not only help maintain the correctness of the codebase when changes are implemented but would also serve as documentation for developers and auditors, aiding them in understanding multiple possible scenarios. Consider increasing the test coverage, especially around scenarios that deviate from normal operation.


***Update:** Resolved in [pull request #1780](https://github.com/OriginProtocol/origin-dollar/pull/1780). The Origin team stated:*



> *1. The `deposit` test has not been added because single-asset deposits have been disabled.  
> 2. The test for `safeApproveAllTokens` has been added.  
> 3. The BAL and AURA harvesting test has been added.  
> 4. `maxDepositSlippage` and how it affects oracle prices: We are no longer using oracles so this portion is skipped. Changing `maxDepositSlippage` (now `maxDepositDeviation`) is tested in H-03. Also, the `maxDepositDeviation` is much more lenient now as `VaultValueChecker` has become the line of defense against MEV.  
> 5. Whether the underlying value of the Strategy will be reported correctly, irrespective of the liquidity pool being balanced: initial test added in H-03's pull request and enhanced via [a separate commit.](https://github.com/OriginProtocol/origin-dollar/pull/1697/commits/3fd8f23efb1cb80041f2c373ee6f2c74e5523734)  
> 6. Stale oracle prices test: skipped since we do not use oracles.  
> 7. `withdrawAll` on a subset of assets: no test since this functionality is not yet supported.*
