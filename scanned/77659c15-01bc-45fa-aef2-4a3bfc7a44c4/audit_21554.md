# Should check if the asset already exists when adding a new asset ✓ Fixed

## Summary
This bug report is about a public function called `includeAsset` which is located in `src/Loihi.sol:L128-L130`. This function calls an internal implementation of the same name located in `src/Controller.sol:L72`. This implementation does not check to see if the asset already exists in the list, which can cause `shell.numeraires` to contain multiple identical instances. The development team has decided not to have dynamic adding/removing of assets in this release, but the recommendation is to check if the `_numeraire` already exists before invoking `includeAsset`.

## Details
#### Resolution



*Comment from the development team*:


We have decided not to have dynamic adding/removing of assets in this release.
