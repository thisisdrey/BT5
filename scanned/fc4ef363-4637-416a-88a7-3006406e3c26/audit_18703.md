# [M-16] `DEPOSITOR_ROLE` can manipulate `b.amount` value

## Summary
This bug report is about a vulnerability in the BribeVault.sol contract, which is located at https://github.com/code-423n4/2022-02-redacted-cartel/blob/main/contracts/BribeVault.sol#L187. The vulnerability allows a malicious DEPOSITOR_ROLE to manipulate b.amount without any cost. This is done by inputing a WETH address and setting the briber argument to address(this) in the safeTransferFrom function. This triggers a self transfer, which allows the malicious actor to increase b.amount without any cost. 

To mitigate the vulnerability, it is recommended to validate that the briber argument is not equal to Address(this). This will prevent the malicious actor from triggering a self transfer.

## Details
_Submitted by rfa, also found by Omik_

[BribeVault.sol#L187](https://github.com/code-423n4/2022-02-redacted-cartel/blob/main/contracts/BribeVault.sol#L187)<br>

Malicious `DEPOSITOR_ROLE` can doing self transfer and manipulate `b.amount`

### Proof of Concept

In case malicious `DEPOSITOR_ROLE` inputing `WETH` address and putting `briber` == address(this) in `safeTransferFrom` argument (which is self transfering). Therefore, it is posible to increase `b.amount` without any cost.

WETH token contract:

    //Line 62 WETH contract
    function transferFrom(address src, address dst, uint wad)
            public
            returns (bool)
        {
            require(balanceOf[src] >= wad);

            if (src != msg.sender && allowance[src][msg.sender] != uint(-1)) { 
                require(allowance[src][msg.sender] >= wad); 
                allowance[src][msg.sender] -= wad; // <----------- this line won't executed
            }

            balanceOf[src] -= wad;
            balanceOf[dst] += wad;

            Transfer(src, dst, wad);

            return true;
        }

If the condition didn't pass (in this case msg.sender != `src` ), the transaction will treated like a transfer (doesn't need an allowance), Therefore its possible to do self transfer

### Recommended Mitigation Steps

I recommend to validate that bribe != Address(this)
