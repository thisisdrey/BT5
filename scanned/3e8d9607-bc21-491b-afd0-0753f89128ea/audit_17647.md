# Potential reentrancy attack to claim rewards multiple times.

## Summary
This report is about a bug in a function called "claim RewardsAsFNDZ" in a file called FNDZStaking.sol. This function swaps rewards to a token called FNDZ using routes that are chosen by the user. However, there is a risk that a malicious token could be used in the swap route, which could lead to problems. To prevent this, the report recommends updating certain variables before making external calls. Specifically, the "stakeUpdatedAt" and "hasEarned Rewards" variables should be updated before the call. This will prevent users from claiming rewards multiple times if a malicious token is used in the swap route. The report suggests making this update before a certain line in the code, which is line 462.

## Details
**Description**

FNDZStaking.sol: function _ claim RewardsAsFNDZ(). 
Function performs swap of rewards to FNDZ token with swap routes which are passed by user. Since, anyone can create a pair on Uniswap factory, a path with malicious token might be passed, which is why all all storage variables should be changed before external calls.Currently, stakeHolder info "stakeUpdatedAt" and "hasEarned Rewards" are updated after the call. In case, a malicous token is used in a swap route with reentrancy to claiming rewards, "stakeUpdatedAt" should be already updated, so that user doesn't claim rewards multiple times. 

**Recommendation**: 

Update stakeHolderInfo (Currently on lines 483, 484) before the loop (Before line 462).
