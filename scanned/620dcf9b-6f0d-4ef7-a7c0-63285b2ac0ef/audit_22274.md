# Remove unnecessary code

## Summary
This bug report is related to the Ether Camp Virtual Accelerator. It points out four issues that can be improved in the code. 

The first issue is that the convert function in the DSTContract.sol can be replaced by simply casting strings to bytes32. The second issue is that the two separate transfers in the buy function of VirtualExchange.sol can be replaced by a single call of hackerGold.transferFrom(msg.sender, dstContract.getAddress()). The same idea can be applied to the lines in DSTContract.sol. 

The third issue is that the selfAddress in line 33 of DSTContract is unnecessary and can be replaced with ‘this’. The fourth issue is that some constant getter functions can be removed if the variables are made public. 

Overall, this bug report is useful for understanding the issues in the Ether Camp Virtual Accelerator code and how they can be improved.

## Details
* [convert function](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L688-L696) could be replaced by simply casting strings to bytes32.
* The two separate transfers in [the buy function of VirtualExchange.sol](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/VirtualExchange.sol#L101-L105) could be replaced by a single call of `hackerGold.transferFrom(msg.sender, dstContract.getAddress())` . Same idea can be applied to [these lines](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L250-L252) in DSTContract.sol.
* selfAddress in [line 33](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L33) of DSTContract is unnecessary. Use `this`.
* [Some constant getter functions](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L627-L686) can be removed if variables are made public.
