# No validation of apy

## Summary
This bug report is about a problem in the setApys function. The issue is that there is no validation check for the input value, which can cause problems for the tokenomics of the ecosystem. The severity of this issue is considered medium and it has been resolved by adding a require statement to check the input value.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In setApys, there is no validation check for the input value. This issue severity is higher than it is usually due to the impact a mistake in new value can have on the tokenomics of the ecosystem.

**Recommendation**: 

require statement to check input new value.
