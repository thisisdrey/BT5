# buyItNow() can be used to conclude sale of a BUY_IT_NOW_WITH_BIDS listing at a price lower than reserve 

## Summary
The bug report discusses an issue with the logic in a code for a marketplace that allows users to buy and sell NFTs (non-fungible tokens). The current logic allows for two types of listings, BUY_IT_NOW and BUY_IT_NOW_WITH_BIDS, to be settled. However, there are problems with the BUY_IT_NOW_WITH_BIDS type. This type needs to check if the sale price is above the reserve price, but this check is bypassed if the function buyItNow() is called. This means that an attacker can potentially buy an NFT for a price lower than the reserve. The recommendation is to modify the code to ensure that the reserve price is always checked for BUY_IT_NOW_WITH_BIDS listings and to update the documentation accordingly. The bug has been fixed in a recent update.

## Description
The current logic in `buyItNow()` allows both `BUY_IT_NOW` and `BUY_IT_NOW_WITH_BIDS` listing types to be settled. This flow performs checks on the sale window, refunds any existing bids, and concludes the sale by transferring payment and the NFT to the seller and buyer, respectively.

This approach works well for a `BUY_IT_NOW` listing. However, for `BUY_IT_NOW_WITH_BIDS`, it introduces several problems:

- The `BUY_IT_NOW_WITH_BIDS` type needs to check if the sale price (bid) is at least above the reserve price. This reserve check is bypassed if someone calls `buyItNow()`.
- According to the specifications, the price parameter for a `BUY_IT_NOW_WITH_BIDS` listing must be greater than the reserve. The issue arises because, if `buyItNow()` is called, the sale is concluded with `listing.price` as the payment, which can be lower than the reserve since this is not validated in `listNFT()`.

The impact is that an attacker could potentially acquire the NFT for a payment lower than the reserve.

## Recommendation
In `listNFT()`, ensure that `BUY_IT_NOW_WITH_BIDS` type listings always have a price greater than or equal to the reserve. Modify the documentation to reflect this change.
