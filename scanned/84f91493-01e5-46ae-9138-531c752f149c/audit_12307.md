# Inaccurate Price Calculation Formula

## Summary
The getTokenPrice function in UniswapOracle is supposed to provide the price of one base token in USD with two decimal precision. However, there is a bug in the formula used for calculation, resulting in inaccurate price values. The formula is incorrect as the quantity 10 ** (decimal - 6) is used as the denominator instead of being multiplied to the amountOut. This causes an underflow error for tokens with a decimal value less than ten. To fix this, the price of only one token should be passed in as the baseAmount and the result should be divided by 10 ** 4 for two decimal precision. The bug has been fixed in the latest commit.

## Details
## Token Price Calculation Issue in UniswapOracle

`getTokenPrice` in UniswapOracle provides the price of one base token (10**decimal) in USD with two decimal precision. However, there is an error in the formula for the calculation, resulting in inaccurate price values.

## Current Formula

The following is the conversion formula used:

```
((amountOut * 100) / 10 ** (decimal - 6)) / amount
```

This formula is incorrect because the quantity `10 ** (decimal - 6)` acts as the denominator when it should have been multiplied to `(amountOut * 100)`. Moreover, due to the presence of `10 ** (decimal - 6)`, a token with a decimal value less than ten will fail to calculate the price as an underflow error will occur.

## Code Example

In `UnisapOracle.sol`, the function `getTokenPrice` looks like this:

```solidity
function getTokenPrice(
    uint128 amount,
    uint32 secondsAgo
) public view returns (uint amountOut) {
    [...]
    // calculate the price with 100 times
    uint256 decimal = IERC20Extented(tokenAddress).decimals();
    return ((amountOut * 100) / 10 ** (decimal - 6)) / amount;
}
```

## Proof of Concept

- Consider a token with `decimal = 8` and a price of `20 USD` per token.
- Let the amount of the base token be `2 * 10**8`, thus, `amountOut = 40 * 10 ** 6`.
- Substituting in the formula, we get: 

```
((40 * 10**6 * 100) / 10 ** (8 - 6)) / (2 * 10 ** 8)
```

This amounts to `1/5`, resulting in zero. 

- This final result is incorrect as it should have been `2000` (since the price of one token is `20 USD`).

## Remediation

To remediate the issue, pass in the price of only one token (i.e., `10**decimal`) as the `baseAmount` to `getQuoteAtTick`, which will then result in `amountOut` being the USD price of one token in six decimals. 

To ensure the result converts to a precision of two decimal places, divide this `amountOut` by `10**4`. This will also lead to a more optimized approach by reducing the number of calculations performed.

## Audit and Patch

This vulnerability was addressed in VTVL Audit 04. 

### Patch

Fixed in commit `a3d984d` by correcting and simplifying the token price calculation formula in `getTokenPrice`.
