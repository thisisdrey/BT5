# Improper Usage Of Variable For State Updating

## Summary
The report describes a bug in the SpotEngineState contract where the quoteState variable is being used instead of the state variable in the updateStates function. This is causing unexpected behaviors within the protocol. The suggested solution is to change the quoteState variable to the state variable in the highlighted line of code. This bug has been fixed in the latest update of the code.

## Details
## Code Update in SpotEngineState Contract

In the `updateStates` function of the **SpotEngineState** contract, the `quoteState` variable is used instead of the `state` variable.

```solidity
contracts/SpotEngineState.sol
for (uint32 i = 0; i < productIds.length; i++) {
    uint32 productId = productIds[i];
    State memory state = states[productId];
    _updateState(productId, quoteState, dt);
    if (productId != QUOTE_PRODUCT_ID) {
        LpState memory lpState = lpStates[productId];
        _updateBalance(state, lpState.base, 0);
        _updateBalance(quoteState, lpState.quote, 0);
        lpStates[productId] = lpState;
        states[productId] = state;
    } else {
        quoteState = state;
    }
}
```

Updates will apply to respective states since updates are applied to `quoteState` instead of `state`, leading to unexpected behaviors within the protocol.

## Remediation
Change the `quoteState` variable to the `state` variable in the highlighted line of the code snippet.

## Patch
The code was modified to pass the `state` variable to `updateStates`. Fixed in **#101**.
