# Possible Revert In _depositToCoreLendingPool

## Summary
The bug report discusses an issue with the `_depositToCoreLendingPool` function in the RevenueManagement.sol file. The call to `lendingPool.deposit` may potentially pass a value of 0, causing the `validateDeposit` function to fail. The report suggests implementing a check to skip the entire `_depositToCoreLendingPool` call if the value of `actualTokens` is 0. The bug has been resolved in a recent pull request.

## Details
Within the [`_depositToCoreLendingPool` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L302), the [call to `lendingPool.deposit`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L305) could potentially pass an `_amount` of 0, causing [the `validateDeposit` function to fail](https://github.com/radiant-capital/v2-core/blob/43bb77f155c679c35dd30652b6c984c5ad0215f4/contracts/main/lending/libraries/logic/ValidationLogic.sol#L46).


Consider implementing a check to skip the [entire `_depositToCoreLendingPool` call](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L275) if `actualTokens == 0`.


***Update:** Resolved in [pull request \#84](https://github.com/radiant-capital/riz/pull/84) at commit [b25bc7a](https://github.com/radiant-capital/riz/pull/84/commits/b25bc7aaca610cba63914bcc1d848eda5a768767).*
