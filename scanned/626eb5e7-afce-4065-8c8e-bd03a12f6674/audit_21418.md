# Attacker can abuse swapLiquidity function to drain users’ funds ✓ Fixed

## Summary
A bug was discovered in the `swapLiquidity` function of the `LendingPoolCollateralManager.sol` contract. This function allows liquidity providers to atomically swap their collateral and takes a `receiverAddress` argument that normally points to an `ISwapAdapter` implementation trusted by the user. However, since an attacker can pass any address as the `receiverAddress`, they can arbitrarily transfer funds from other contracts that have given allowances to the `LendingPool` contract. The `amountToSwap` is defined by the caller and can be very small. The attacker gets the difference between `IERC20(toAsset).balanceOf(receiverAddress)` value of `toAsset` and the `amountToSwap` of `fromToken`. The bug was solved by removing the `swapLiquidity` functionality and ensuring that no funds can be stolen from contracts that have granted allowances to the `LendingPool` contract.

## Details
#### Resolution



Solved by removing `swapLiquidity` functionality.


#### Description


The `swapLiquidity` function allows liquidity providers to atomically swap their collateral. The function takes
a `receiverAddress`argument that normally points to an `ISwapAdapter` implementation trusted by the user.

However, since an attacker can pass any address as the `receiverAddress`, they can arbitrarily transfer funds from other contracts that have given allowances to the `LendingPool` contract (for example, another `ISwapAdapter`).

#### Remediation


Ensure that no funds can be stolen from contracts that have granted allowances to the `LendingPool` contract.
