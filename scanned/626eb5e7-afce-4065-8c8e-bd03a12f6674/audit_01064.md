# owner == controller should be enforced when handling redemption

## Summary
The bug report is about a medium risk issue in the code of a contract called SuperVault. The problem occurs when someone tries to redeem their shares from the contract. The contract owner transfers the shares to an escrow, but the code also tries to reduce the shares of the controller. This causes the transaction to fail if the controller does not have any shares. The recommendation is to fix this by only allowing the contract owner to redeem shares.

## Details
## Severity: Medium Risk

## Context
`SuperVault.sol#L186-L187`

## Description
When redeeming, the owner transfers the shares escrow. However, the fulfillment request tries to reduce the controller's `accumulatorShares`. If the controller does not have shares, the transaction reverts.

## Recommendation
Enforce `owner == controller` when redeeming.
