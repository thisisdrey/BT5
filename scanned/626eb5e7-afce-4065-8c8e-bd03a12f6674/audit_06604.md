# Native Asset Not Accounted for When Swapping

## Summary
The `swapTokens` function in the `RevenueManagement` contract does not include the value of native assets when calculating the total input value for a swap. This means that the `DexSwapStrategy` contract, which can perform arbitrary calls, could potentially receive ETH and send it to a different address. To fix this, the `RevenueManagement` contract should have a `receive` function and include native assets in the total input value calculation. Alternatively, the ability to send native assets in the `DexSwapStrategy` contract could be removed. This issue has been resolved in a recent pull request.

## Details
In the [`swapTokens` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L224), the [`computeInputValue` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L226C35-L226C52) only totals the values of swap inputs which are ERC20 tokens. It does not include the `value` field of data, which will be sent when performing the swap.


Since the call within [`DexSwapStrategy`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/libraries/swap-strategies/DexSwapStrategy.sol) performs an arbitrary call, it can have any target and any parameters. So, it easily could be used to receive ETH and send it to a different address.


However, this is complicated by the fact that the [`RevenueManagement` contract](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol) has no `payable` functions.


Consider including a `receive` function within `RevenueManagement` and including native assets within the total input value computations. Alternatively, consider entirely removing the ability to send the native asset within the `DexSwapStrategy` contract.


***Update:** Resolved in [pull request \#53](https://github.com/radiant-capital/riz/pull/53). The Radiant team stated:*



> *Removed the ability to send the native asset in the swap and added an `if` check to make sure the value `!= 0`.*
