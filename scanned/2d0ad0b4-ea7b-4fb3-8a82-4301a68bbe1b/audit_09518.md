# If the Blast admin calls adminClaimGas on particlePositionManager , the latter won 't be able to claim gas anymore 

## Summary
The report describes a bug in a contract called Blast, which has a function called adminClaimGas that can only be called by the admin of the contract. This function transfers all the gas accrued by a contract to a specific address and changes the gas mode to Void. This means that if the function is called for a specific contract, it will no longer receive gas rewards and will lose all future gas related income until it is upgraded. The recommendation is to add a function in the contract called ParticlePositionManager to allow it to set the gas mode to Claimable, and the bug has been fixed by adding this function.

## Description
The Blast contract has a function called `adminClaimGas`, which is only callable by Blast's admin (see Gas.sol#L128-L133).

```solidity
// ...
function adminClaimGas(address contractAddress) external onlyAdmin returns (uint256) {
    (, uint256 etherBalance,,) = readGasParams(contractAddress);
    _updateGasParams(contractAddress, 0, 0, GasMode.VOID);
    SafeTransferLib.safeTransferETH(blastFeeVault, etherBalance);
    return etherBalance;
}
// ...
```

Once called, this function transfers all the gas accrued by a contract to the `blastFeeVault`, and sets the gas parameters to 0 balance, 0 seconds, and changes the Gas mode to Void. This means if the admin calls the function for the `ParticlePositionManager`, it will stop accruing gas rewards and will lose all future gas-related income from that point, until the contract is upgraded to be able to change the gas mode again.

## Recommendation
There should be a function in the `ParticlePositionManager` to allow the contract to set the Gas mode to Claimable (to be put in the Blast library):

```solidity
// ...
function setGasMode() external {
    IBlast(BLAST).configureClaimableGas();
}
```

**Particle:** Fixed. Added `setGasMode()`.
