# libsecp256k1 allows overflowing signatures

## Summary
This bug report is about a problem with a file called Cargo.lock. The issue is that a certain library called libsecp256k1 is accepting signatures that are too big, which can cause problems when verifying them. The bug has been acknowledged and a solution has been implemented in version 0.5.0 by adding a check_overflow flag. The recommendation is to update the library to version 0.5.0 or higher.

## Details
**Severity**: High

**Status**: Acknowledged

**File**: Cargo.lock

**Details**:

libsecp256k1 accepts signatures whose R or S parameter is larger than the secp256k1 curve order, which differs from other implementations. This could lead to invalid signatures being verified.
The error is resolved in 0.5.0 by adding a check_overflow flag.


**Recommendation**: 

Update libsecp256k1 library to the version >=0.5.0
