# Investors will not be able to withdraw tokens after the end of the vesting duration for Ecosystem in UnmarshalTokenVesting

## Summary
The report describes a bug in the Array ecosystemVesting length, which is set to 50. This means that if more than 50 months have passed since the initial time, the function _calculateAvailablePercentage will result in an error. The code snippet provided shows that the function is not properly checking for this condition and is causing the error. The recommendation is to add a check for the number of months being greater than 50 and if it is, then return the last value of ecosystemVesting. This will fix the bug and prevent the error from occurring.

## Details
**Description**

Array ecosystemVesting length is equal to 50. If more than 50 months have passed since the
initial time function _calculateAvailablePercentage will lead to error.

**Snippet**:
```solidity
if (currentTimeStamp > _initialTimestamp) {
if (distributionType == DistributionType.ECOSYSTEM) {
return ecosystemVesting[noOfMonths];
```

**Recommendation**:

Add check for noOfMonths > 50, if it evaluates to true, then returns last value of
ecosystemVesting.
