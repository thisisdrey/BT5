# [M09] Untested custom SafeMath library in use

## Summary
This bug report is about the lack of testing for custom mathematical operations functions based on OpenZeppelin's SafeMath library, which are used in several contracts. These functions are used for `uint256` and `uint96` operations in the GovernorAlpha and Stake contracts respectively. The lack of tests for these functions means that any changes made to the library in the future won't be detected by the current test suite, leaving the business logic depending on it vulnerable to potential security issues. The EmptySetSquad team has acknowledged this issue but has stated that they won't fix it as the code is already tested externally.

## Details
Several contracts implement arithmetical operation functions based on [OpenZeppelin’s SafeMath](https://github.com/OpenZeppelin/openzeppelin-solidity/blob/v2.3.0/contracts/math/SafeMath.sol) library, customized to support safe mathematical operations for `uint256`, such as the [`GovernorAlpha` contract](https://github.com/emptysetsquad/emptyset/blob/bf9753ef9cd5b17236036257f290e0d0b850a029/protocol/contracts/src/governance/GovernerAlpha.sol#L333-L348), `uint96`, such as the [`Stake` contract](https://github.com/emptysetsquad/emptyset/blob/bf9753ef9cd5b17236036257f290e0d0b850a029/protocol/contracts/src/governance/Stake.sol#L348-L367). Yet, none of the functions are tested, which may hinder users and developers’ trust in this custom library. Moreover, any change in the library is not going to be detected by the current test suite, rendering all business logic depending on it vulnerable to potential security issues introduced by these future modifications.


Consider including thorough and extensive unit tests for those functions.
