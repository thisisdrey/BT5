# The ability to increase winChance without participating in the lottery.

## Summary
The report discusses a bug in the WinChancesContract.sol file, specifically in the updateAfterLoss() function. The bug allows users who joined the lottery late to increase their chances of winning as if they had lost in each round. The recommendation is to check the functionality of late users joining the lottery. The bug has been resolved by adding contingency functions and manually stopping the increase in winChance after a certain amount of time. It is important to note that the lottery runs at the city level, not the building level, so if the protocol blocks the lottery claim, a new lottery may open access for new users to increase their winChance based on the number of buildings sold in that city.

## Details
**Description**

WinChancesContract.sol, updateAfterLoss()

If the user was not a lottery participant (for example, he joined later), he can still raise his winChance in the same way as if he lost in each of them.

**Recommendation**

Verify the functionality of late users joined to the lottery.

**Re-audit comment**

Resolved.

From client:

Added contingency functions for this. So the team can manually stop the increasemet when we feel enough time has passed.

Post-audit:

Note that the protocol runs the lottery at the city level - not the buildings. That is, if the protocol blocks the lottery claim, and holds some kind of draw after a long time again in this city, this new lottery will open access to winChance increasement for new users by the amount equal to the number of buildings already sold in this city.
