# Incorrect withdraw logic for team distribution type

## Summary
This bug report is about a function called _calculateAvailablePercentage used for team distribution. The problem is that the function will always return 100% if the current timestamp is before the initial cliff (240 days). This means that the function is not properly calculating the available percentage for team members. The code snippet provided shows that there is a missing condition in the "else" block that checks if the current timestamp is greater than the initial cliff. The recommendation is to add this condition to ensure the correct calculation of the available percentage.

## Details
**Description**

Function _calculateAvailablePercentage for team distribution will always return 100% if current
timestamp is before of initial cliff (240 days).

**Snippet**:
```solidity
if (currentTimeStamp > initialCliff && currentTimeStamp < vestingDuration) {
uint256 noOfDays = BokkyPooBahsDateTimeLibrary.diffDays(initialCliff, currentTimeStamp);
if (noOfDays == 1) {
return uint256(15).mul(1e18);
} else if (noOfDays > 1) {
uint256 currentUnlockedPercentage = noOfDays.mul(everyDayReleasePercentage);
// console.log("Current Unlock %: %s", currentUnlockedPercentage);
return uint256(15).add(currentUnlockedPercentage);
}
} else {
return uint256(100).mul(1e18);
}
```

**Recommendation**:

Extend `else` block with the condition to check if the current timestamp is greater than initial
cliff.
