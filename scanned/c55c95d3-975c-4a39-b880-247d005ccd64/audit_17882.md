# [M-28] RubiconRouter maxSellAllAmount does not transfer user’s fund into its address, causing calls to always revert

## Summary
A bug has been reported in the RubiconRouter function maxSellAllAmount, which does not transfer user's fund into its address, causing the function to always revert. This means that when RubiconMarket tries to take tokens from it, the function will always fail. The recommended mitigation step is to add a transfer of fund to the function. This can be done by adding the following code to the function: 
```solidity
    /// @dev this function takes a user's entire balance for the trade in case they want to do a max trade so there's no leftover dust
    function maxSellAllAmount(
        ERC20 pay_gem,
        ERC20 buy_gem,
        uint256 min_fill_amount
    ) external returns (uint256 fill) {
        //swaps msg.sender entire balance in the trade
        uint256 maxAmount = ERC20(buy_gem).balanceOf(msg.sender);
        ERC20(buy_gem).safeTransferFrom(msg.sender, address(this), maxAmount);
        fill = RubiconMarket(RubiconMarketAddress).sellAllAmount(
            pay_gem,
            maxAmount,
            buy_gem,
            min_fill_amount
        );
        ERC20(buy_gem).transfer(msg.sender, fill);
    }
```
This code will transfer a user's entire balance for the trade in case they want to do a max trade, so there's no leftover dust. The code will also enable the RubiconMarket to take tokens from the function, thus mitigating the bug.
