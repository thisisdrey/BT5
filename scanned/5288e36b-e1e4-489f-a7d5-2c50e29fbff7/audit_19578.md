# [M-04] FeeSplitter: Unbounded number of shareholders can cause DOS

## Summary
This bug report is about a vulnerability in the NestedFactory functions, which can cause a denial of service. This vulnerability is caused by the lack of a limit on the number of shareholders that can be added. When a large number of shareholders are added, the `_sendFees()` function runs out of gas and prevents the NestedFactory functions from running, especially the ones that remove funds like `withdraw()` and `destroy()`. The recommended mitigation steps for this vulnerability is to set a sanity maximum number of shareholders that can be added.

## Details
_Submitted by GreyArt_

#### Impact
There is no limit to the number of shareholders. It is therefore possible to set a large number of shareholders such that `_sendFees()` will run out of gas when adding shares to each shareholder.
This will cause denial of service to all NestedFactory functions, especially the ones that will remove funds like `withdraw()` and `destroy()`.

#### Recommended Mitigation Steps
It would be best to set a sanity maximum number of shareholders that can be added.
