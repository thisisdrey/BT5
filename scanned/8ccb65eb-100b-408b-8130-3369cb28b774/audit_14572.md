# M-19: Manipulate Allocations from game using flashloans

## Summary
This bug report is about a vulnerability found in the game's code that allows a malicious user to manipulate DeltaAllocations and change the weight of some protocol inside the vault. This is possible because the user can call pushAllocationsToController and pushAllocationsToVaults in the same transaction. To do this, the user takes a flashloan of USDC, swaps the USDC into derby token on some dex, rebalances their basket using these tokens, calls pushAllocationsToController, calls pushAllocationsToVaults, rebalances their basket to get tokens back, swaps tokens to USDC, and pays back the loan. This vulnerability could lead to users staking tokens into bad performing protocols, leading to possible loss of funds or unexpected behavior. The bug was found by oot2k and the recommendation is to not allow to call pushAllocationsToController and pushAllocationsToVaults in the same transaction.
