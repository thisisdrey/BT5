# [H02] Chainlink rates may need to be inverted

## Summary
A bug was reported in the `Escrow.sol` file of the Swapnet-Lite protocol. The bug was related to the function `_checkUniswapRateDifference`, which compares the rate given from a chainlink oracle with the rate implied by a Uniswap trade.

The bug occurs when the oracle does not exist for a given pair. For instance, if a Uniswap trade of WBTC for DAI is made, the `localCurrencyPurchased` value would be 11000 DAI for 1 WBTC sold, and `collateralSold` should equal 1 WBTC token. When these values are passed into `_checkUniswapRateDifference`, `uniswapImpliedRate` will be calculated as `1e8 * Common.DECIMALS / 11000e18`, where `Common.DECIMALS = 1e18`. This value represents the number of WBTC units that can be purchased for 1 DAI.

This value will be compared with the `answer` from the chainlink BTC-USD feed, which provides `latestAnswer` in terms of the USD price of BTC. This is very different in its order of magnitude versus the `uniswapImpliedRate`. Both the difference in the measured units and the difference in order of magnitude should be taken into account, or the comparison is likely to revert.

The bug was fixed in pull request #48.

## Details
Within [`Escrow.sol`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1087), the function [`_checkUniswapRateDifference`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1087) compares the rate given from a chainlink oracle with the rate [implied by a uniswap trade](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1093).


In some cases, the oracle may not exist for a given pair. For instance, imagine a uniswap trade of WBTC for DAI. This makes sense because WBTC is a collateral, and DAI is a cash or `localCurrency` token. The value `localCurrencyPurchased` is [the amount of DAI tokens received](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1043) from a Uniswap trade, and at a price of roughly 11000 USD/BTC, would be 11000 DAI for 1 WBTC sold. So, `localCurrencyPurchased = 11000e18`. Meanwhile, [`collateralSold`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1046) should equal `1e8` to represent 1 WBTC token. Both of these values will be passed into a call to [`_checkUniswapRateDifference`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1059).


Within `_checkUniswapRateDifference`, [`uniswapImpliedRate` will be calculated](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1093) as `1e8 * Common.DECIMALS / 11000e18`, where [`Common.DECIMALS = 1e18`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/utils/Common.sol#L22). So, the calculation result is `9090`. Intuitively, this value represents the number of WBTC units (â€śW-satoshisâ€ť) which can be purchased for 1 DAI. This value will be [compared](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1098) with the chainlink oracleâ€™s [`answer`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1095), which will come from the [chainlink BTC-USD feed](https://feeds.chain.link/btc-usd). This feed provides [`latestAnswer` in terms of the USD price of BTC](https://etherscan.io/address/0xf5fff180082d6017036b771ba883025c654bc935#readContract), scaled up by a factor of 1e8. So, for a BTC price of `11000 USD`, the `answer` will be `11000e8`. This is very different in its order of magnitude versus the `uniswapImpliedRate`, which is discussed in more detail in issue [**[H01] Uniswap-Chainlink rate difference always reverts for certain currencies**](#h01). It also has the important difference that instead of measuring WBTC units per DAI, it is measuring DAI units per WBTC (scaled by some orders of magnitude).


Both the difference in the measured units (measuring WBTC in DAI units versus measuring DAI in WBTC units) and the difference in order of magnitude should be taken into account, or the comparison is [likely to revert](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1098-L1104). Even if the comparison does not revert, it is useless in principle to compare two values which do not represent the same thing. Since a chainlink oracle does not exist for USD in terms of BTC, the [`answer`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L1095) from the oracle must be processed to invert the ratio (disregarding the fact that the correct scaling will need to be applied as well). Consider adding a `bool` to the [`ExchangeRate` struct](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/storage/EscrowStorage.sol#L17-L25) which flags whether a chainlink rate will need to be inverted. Considering the recommendation of **[H01]**, make sure to define the oracleâ€™s decimals as the number of decimals either before or after the inversion, and clearly state this in the documentation and comments. Before adding a new oracle, consider implementing thorough testing with real-world values for the pairs that will be created.


***Update:** Fixed in [pull request #48](https://github.com/swapnet-protocol/swapnet-lite/pull/48).*
