# Possible integer overflow allows invalid SecurityParams

## Summary
This bug report discusses a potential issue in the `core/data.go` file, specifically on line 80. The severity of this bug is considered to be medium risk. The problem occurs when checking two variables, `QuorumThreshold` and `AdversaryThreshold`, which are both of type `uint8`. Since these variables are of the same type, it is possible for an unsafe addition to occur, leading to an overflow. This means that the values of these variables could become larger than the maximum value allowed for a `uint8` type. This could potentially be exploited by a malicious actor. The recommendation is to add another check to prevent this overflow from happening.

## Details
## Issue Report

## Severity
Medium Risk

## Context
`core/data.go#L80`

## Description
When checking `QuorumThreshold` and `AdversaryThreshold`, there's an unsafe addition which could overflow and be abused. These are both `uint8` types, so it is easy to overflow. For example, if `QuorumThreshold` is 5 and `AdversaryThreshold` is 255, this will pass the check and an invalid situation will be allowed.

## Recommendation
Add another check to see if the overflow is possible.
