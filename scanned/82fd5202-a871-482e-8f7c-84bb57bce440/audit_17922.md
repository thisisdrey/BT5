# UF-3 | Random Manipulation

## Summary
This bug report is about the `random` function in the codebase of Ultimate Fantoms. The `random` function relies on weak sources of pseudo-randomness from on-chain attributes, which can be manipulated by a validator node. This manipulation can lead to the `_sendTo` address being manipulated in favor of the validator.

To resolve this issue, Ultimate Fantoms has recommended utilizing the Randomness pattern to obtain on-chain randomness and avoid validator manipulation, or obtaining random numbers off-chain through an oracle. This recommendation has been acknowledged in the source code of Ultimate Fantoms.

## Details
**Description**

The `random` function relies on weak sources of pseudo-randomness from only on-chain attributes. A
validator node can manipulate the `block.timestamp` and therefore the random number. Therefore, the
`_sendTo` address can be manipulated in favor of the validator.

**Recommendation**

Utilize the Randomness pattern to obtain on-chain randomness and avoid validator manipulation or
obtain random numbers off-chain through an oracle.

**Resolution**

Ultimate Fantoms: Acknowledged in source code.
