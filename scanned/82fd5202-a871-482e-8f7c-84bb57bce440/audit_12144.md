# User Interface Message Spoofing

## Summary
The bug report discusses an issue with the prettyPrint function in a JavaScript object. The function does not validate attributes, which can be exploited by attackers to manipulate the content of an approval dialog and trick users into accepting malicious transactions. This can occur when requesting recurring transfers or when executing the default fallback of switch(keys[0]). The report suggests blocking unknown commands or separating critical attributes to prevent confusion. The issue has been fixed in a recent commit by converting the untrusted transaction object to a protobuf format and strictly checking each input.

## Details
## Vulnerability Overview

**prettyPrint** prints a JavaScript object in the snap dialog. As no attribute validation occurs in the printable object, the attacker may create an object with any attribute name. An attacker may utilize this to spoof the content of the approval dialog, tricking the user into accepting a malicious transaction. This function may be reached when requesting recurring transfers or when executing the default fallback.

## Code Snippet

```typescript
switch(keys[0]) {
    case 'batchMarketInstructions':
        return prettyPrintBatchMarketInstructions(txContent);
    case 'orderSubmission':
        return prettyPrintOrderSubmission(txContent, textFn);
    case 'orderCancellation':
        return prettyPrintCancelOrder(txContent, textFn);
    case 'orderAmendment':
        return prettyPrintOrderAmendment(txContent, textFn);
    case 'withdrawSubmission':
        return prettyPrintWithdrawSubmission(txContent, textFn);
    case 'transfer':
        return prettyPrintTransferFunds(txContent, textFn);
    default:
        return prettyPrint(txContent);
}
```

## Remediation

- Block unknown commands by removing the default handler. Create **prettyPrint** exclusively for recurring transfers.
- If blocking unknown commands is not an option, consider separating the default attributes, for example, `market_id`, from the rest so that the critical attributes will not be confused.

## Patch

Fixed in commit `001e207` by converting the untrusted transaction object to a protobuf format, which strictly checks each input, and then converts it again to a JavaScript object.
