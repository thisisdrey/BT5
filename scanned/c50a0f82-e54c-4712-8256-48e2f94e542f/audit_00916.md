# M-5: First chunk having cpu chip is incorrectly checked in convert circuit

## Summary
This bug report discusses an issue found by user "cergyk" in the "convert recursion circuit" of the Pico project. The circuit contains a check that prevents the first chunk from having no CPU chip, but due to a coding error, the check is always true. This means that even if the first chunk has no CPU chip, the check will still pass. This can potentially impact the completeness of the recursion proof. The recommendation is to either move the check before the increment or modify it to fix the issue. The issue has been fixed by the protocol team in a recent PR/commit.

## Details
Source: https://github.com/sherlock-audit/2025-08-pico-judging/issues/142 

## Found by 
cergyk

### Description

In the convert recursion circuit, there's a check preventing the first chunk from having no CPU chip:

[convert/builder.rs#L220-L221](https://github.com/brevis-network/pico/blob/3046e89b1c7bf065b7932d859a06a24920d57531/vm/src/instances/compiler/riscv_circuit/convert/builder.rs#L220-L221):
```rust
    if !flag_cpu {
        builder.assert_felt_ne(current_chunk, CC::F::ONE);
```

Unfortunately, the current_chunk is already incremented to be "expected next chunk", meaning that even if the currently converted proof has chunk number 1, the check always checks `2 != 1`, and thus is always true; 

[convert/builder.rs#L199-L213](https://github.com/brevis-network/pico/blob/3046e89b1c7bf065b7932d859a06a24920d57531/vm/src/instances/compiler/riscv_circuit/convert/builder.rs#L199-L213):
```rust
    /*
    Chunk number constraints and updates
        */
    {
        // range check
        CC::range_check_felt(builder, public_values.chunk, MAX_LOG_NUMBER_OF_CHUNKS);


        // current chunk is incremented by 1
        //@audit current_chunk is incremented before the check equal to one
        current_chunk = builder.eval(current_chunk + CC::F::ONE);


        // If the chunk has a "CPU" chip, then the execution chunk should be incremented by 1.
        if flag_cpu {
            current_execution_chunk = builder.eval(current_execution_chunk + CC::F::ONE);
        }
    }
```

### Impact

Impact is not clear, since it allows for one chunk to be converted first without having a cpu. The resulting recursion proof can be used in `combine` circuit, but will not fulfill the completeness condition (`contains_execution_chunk`).

### Recommendation

Either move the check before the increment or modify as follows:


[convert/builder.rs#L220-L221](https://github.com/brevis-network/pico/blob/3046e89b1c7bf065b7932d859a06a24920d57531/vm/src/instances/compiler/riscv_circuit/convert/builder.rs#L220-L221):
```diff
    if !flag_cpu {
-        builder.assert_felt_ne(current_chunk, CC::F::ONE);
+        builder.assert_felt_ne(current_chunk - CC::F::ONE, CC::F::ONE);
```
