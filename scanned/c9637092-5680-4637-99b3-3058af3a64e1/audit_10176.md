# Possible underﬂow of uint128 

## Summary
The bug report discusses an issue with the implementation of a function in the FeeDistributor.sol file, which is a direct copy of Curve's FeeDistributor.vy. The function in question, _checkpointTotalSupply, has a potential problem with converting values to uint128 before performing a max comparison. This can result in a large uint128 value being returned in cases where the result should be 0. The recommendation is to perform the max comparison before converting to uint128 to avoid this issue.

## Overview of FeeDistributor.sol
`FeeDistributor.sol` is a direct fork of Curve's `FeeDistributor.vy`.

### Zerolend Implementation
```solidity
function _checkpointTotalSupply(uint256 timestamp) internal {
    // ...
    veSupply[t] = Math.max(uint128(pt.bias - pt.slope * dt), 0); //@audit
}

function _claim(
    uint256 nftId,
    uint256 _lastTokenTime
) internal returns (uint256) {
    // ...
    uint256 balanceOf = Math.max(
        uint128(oldUserPoint.bias - dt * oldUserPoint.slope),
    );
}
```
The implementation by Zerolend performs a conversion to `uint128` first before executing `Math.max`, while Curve performs the maximum calculation prior to the conversion.

### Curve Implementation
```solidity
self.ve_supply[t] = convert(max(pt.bias - pt.slope * dt, 0), uint256);
balance_of: uint256 = convert(max(old_user_point.bias - dt * old_user_point.slope, 0), uint256);
```

### Potential Issue
The problem with performing a conversion first is that the calculation `(bias - slope * dt)`, which involves signed integers, may yield a negative result (i.e., less than 0). Converting to `uint128` at this stage can lead to underflow:

```solidity
Math.max(uint128(-1), 0) // 340282366920938463463374607431768211455
```

## Recommendation
To avoid the potential underflow issue, it is advisable to perform the maximum comparison before converting to `uint128`:

```solidity
veSupply[t] = uint128(int128(Math.maxInt((pt.bias - pt.slope * dt), 0)));
uint256 balanceOf = uint128(int128(Math.maxInt((oldUserPoint.bias - dt * oldUserPoint.slope), 0)));
```
