# UF-5 | Price Inconsistency

## Summary
This bug report describes an issue with the `getPrice` function in the code, which causes the stepwise price jumps to not account for certain `tokenIds`. This is because each if statement uses '>' instead of '>=' when referring to these `tokenIds`, resulting in a mint for one of these `tokenIds` being charged 6 `FTM`. The recommendation is to use '>=' or decrement the lower boundaries by one, and this has been successfully resolved.

## Details
**Description**

In the `getPrice` function the stepwise price jumps do not account for the following `tokenIds`: 101,
301, 601, 1001, 1501, and 2301.
This is because each if statement utilizes > instead of >= when referring to these tokenIds.Therefore
a mint for one of these `tokenIds` will mistakenly go to the else branch and charge 6 `FTM`.

**Recommendation**

Use `>=` or decrement the lower boundaries by one.

**Resolution**

Ultimate Fantoms: Resolved, applied suggestion.
