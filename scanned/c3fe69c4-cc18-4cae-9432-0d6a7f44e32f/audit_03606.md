# Broker's Publish, Broadcast & Shutdown are unrealiable for delivery guarantees and prone to vari-

## Summary
A bug has been reported in the mod/async/pkg/broker/broker.go file where the Publish/Broadcast function and shutdown can happen at the same time, causing unpredictable and incorrect behavior. This can result in errors, messages not being delivered to all subscribers, and panics. The only desired outcome is to return an error, while the rest can cause serious issues. It is recommended to protect the map with a mutex or other synchronization methods and ensure that subscribers and publishers have idempotent behavior. The bug has been fixed in PR 2225 for Berachain and has been verified by Spearbit.

## Details
## ous race panics

**Severity:** Medium Risk  
**Context:** mod/async/pkg/broker/broker.go  

**Description:**  
Publish/Broadcast and shutdown (due to `<- ctx.done()` in start) can happen concurrently. This results in non-deterministic and unsound behavior, resulting in the following scenarios:
- Return `ctx.Canceled` or `ctx.DeadlineExceeded` error.
- Message delivered to some subscribers but not to others.
- Panic: closing already closed channel when `ctx` cancels/timeout from multiple places.
- Panic: send on closed channel when broadcast is trying to iterate and shutdown has already closed some.

Here, only returning an error is desirable, and the rest are unwanted side effects that can cause severe damage. These effects are observed when node operations deviate from the happy path.

**Recommendation:**
- Protect map by mutex or any sync primitive.
- Ensure idempotent behavior for subscribers and publishers.

**Berachain:** Fixed in PR 2225.  
**Spearbit:** Fix verified.
