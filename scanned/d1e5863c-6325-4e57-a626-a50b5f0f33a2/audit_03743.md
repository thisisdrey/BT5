# Circular dependencies in traits would crash the Tact compiler

## Summary
This bug report discusses a low difficulty bug in the Tact compiler related to access controls. The bug occurs when there are circular dependencies in traits, causing the compiler to crash. The root cause of the bug is a missing line of code in the processType function, which leads to an infinite recursive loop. This can be exploited by pushing code with circular trait dependencies, causing the compiler to crash. The report recommends a short-term fix to add the missing line of code and a long-term solution to improve the test suite.

## Details
## Report: Access Control Vulnerability

## Difficulty: Low

## Type: Access Controls

## Target: `src/types/resolveDescriptors.ts`

### Description
The contract and traits dependency resolver crashes the compiler when there are circular dependencies in traits. The `processType` function in the `resolveDescriptors` function checks whether there are any circular dependencies in traits inherited by a contract by keeping track of intermediate traits in the `processing` variable:

```typescript
function processType(name: string) {
    // Check if processed
    if (processed.has(name)) {
        return;
    }
    if (processing.has(name)) {
        throwCompilationError(
            `Circular dependency detected for type "${name}"`,
            types.get(name)!.ast.loc,
        );
    }
    processing.has(name);
    // Process dependencies first
    const dependencies = Array.from(types.values()).filter((v) =>
        v.traits.find((v2) => v2.name === name),
    );
    for (const d of dependencies) {
        processType(d.name);
    }
    // Copy traits
    copyTraits(types.get(name)!);
    // Mark as processed
    processed.add(name);
    processing.delete(name);
}
```
*Figure 2.1: The `processType` function (tact/src/types/resolveDescriptors.ts#L1773–L1800)*

However, as shown in the highlighted line in figure 2.1, the trait being processed is not added to the `processing` list, and because of this, the `processType` function keeps calling itself recursively indefinitely if there are circular dependencies in the traits. The Tact compiler eventually crashes with the "Maximum call stack size exceeded" error.

### Exploit Scenario
Alice pushes the following Tact code to her remote Git repository, and it crashes her CI/CD pipeline:

```tact
trait A with B {}
trait B with A {}
contract Test with A {}
```
*Figure 2.2: A contract with a circular trait dependency*

### Recommendations
- **Short term:** Replace the highlighted line in figure 2.1 with the `processing.add(name)` statement to track intermediate traits correctly and detect circular dependencies.
- **Long term:** Improve the test suite to include test cases for all the errors thrown by the compiler.
