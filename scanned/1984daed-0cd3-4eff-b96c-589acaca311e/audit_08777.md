# Halving interval is 7 days contrary to the documentation of 24 months 

## Summary
The bug report states that the halving interval for the GOAT protocol is set to 7 days, which is different from the documented interval of 24 months. This means that the emission of GOAT tokens will be slower than intended. The reviewer recommends increasing the halving interval to ensure that the maximum number of tokens emitted is more than half of the total emission. They also note that the documentation is not ideal. The protocol team acknowledges that this is only for the staging version, but the reviewer believes it should still be documented. The bug is considered valid by the judge.

## Description
The halving interval is set to **7 days**, which is contrary to the documentation of **24 months**.

- **DCT.sol#L14**:
  ```solidity
  uint constant public HALVING_INTERVAL = 7 days;
  ```

The documentation states it should be **24 months**:
- **Halving**: Emission is reduced by 50% every 24 months.

As such, the GOAT emissions will be slower than intended.

## Recommendation
The maximum number of tokens that can be emitted is always double the total GOAT emission for the first interval. Therefore, it would be beneficial if the halving interval is increased so that the total GOAT emitted in the first halving interval is more than half of the maximum number of tokens that can be emitted.

As noted, the documentation is also not ideal.
