# Not deployable contract

## Summary
The bug report is about a critical issue in a function called `_batchInitialApprovals` which is causing contract deployment to fail. This is because the function is trying to approve spending of two tokens, `disposeToken` and `wrappedNative`, to two addresses, `curveRouter` and `uniswapRouter`, but these addresses are not set yet. The recommended solution is to remove the function call from the constructor and place it in the address set function instead. This will resolve the issue and allow for successful contract deployment.

## Details
**Severity**: Critical

**Status**: Resolved

**Description**

The function `_batchInitialApprovals`, which is called from the constructor, tries to approve spending of the `disposeToken` to the `curveRouter` and also the `wrappedNative` to the `uniswapRouter`. Unfortunately, both `curveRouter` and `uniswapRouter` are not set at the moment, that's why contract deployment is reverted.

**Recommendations**: 

remove the `_batchInitialApprovals` call from the constructor method, and place it in the address set function.
