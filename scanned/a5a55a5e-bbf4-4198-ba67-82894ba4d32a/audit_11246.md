# Possible DoS of `exchange_received`

## Summary
This bug report is about the `exchange_received` function in the CurveStableSwapMetaNG.vy contract. It is possible for someone to DoS (Denial of Service) the function by sending 1 wei of one of the base_pool tokens to the meta_pool.

To fix this, it is recommended to change the strict check `dx == _dx` to `dx >= _dx`. This will ensure that the function is not vulnerable to DoS attacks.

## Details
##### Description
`exchange_received` can be DoSed if someone sends 1 wei of one of the base_pool tokens to the meta_pool https://github.com/curvefi/stableswap-ng/blob/8c78731ed43c22e6bcdcb5d39b0a7d02f8cb0386/contracts/main/CurveStableSwapMetaNG.vy#L389.

##### Recommendation
We recommend changing the strict check `dx == _dx` to `dx >= _dx`.
