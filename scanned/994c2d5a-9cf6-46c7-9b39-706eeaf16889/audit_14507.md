# M-24: Manipulate Allocations from game using flashloans

## Summary
This bug report is about a vulnerability found in the Game.sol contract of the Sherlock-Audit/2023-01-derby-judging project. It was found by oot2k and allows a malicious user to manipulate DeltaAllocations to change the weight of some protocol inside vault. The malicious user can do this by taking a flashloan of USDC, swapping it into derby token on some dex, rebalancing their basket using these tokens, calling pushAllocationsToController and pushAllocationsToVaults in the same transaction, rebalancing their basket to get tokens back, swapping tokens to USDC, and paying back the loan. This vulnerability can lead to a user staking tokens into bad performing protocols, resulting in possible loss of funds or unexpected behavior. The recommendation is to not allow to call pushAllocationsToController and pushAllocationsToVaults in the same transaction.
