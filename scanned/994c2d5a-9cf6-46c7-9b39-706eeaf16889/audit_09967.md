# Centralized Power / THe Owner's Power Over the Admins

## Summary
The client has reported a bug in the contract where the owner has the ability to add or remove any number of admins without any restrictions. This means that if the owner is malicious or gets hacked, they can easily remove all protection from the admins. The client suggests implementing a grace period, as seen in the example at [Time Lock | Solidity by Example | 0.8.20](https://solidity-by-example.org/app/time-lock/), where the owner must wait before being able to change the set of admins. This grace period should be added to both the `addAdmin()` and `removeAdmin()` functions. It is important to address this issue to prevent potential security breaches.
