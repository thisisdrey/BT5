# Lack Of Chain ID Validation

## Summary
The bug report discusses an issue with the "DelegationKeyDecode" and "CuTrackerKey" functions using a space character as a delimiter to split a composite key into its parts. However, the "chainID" used in these functions is not validated in the "ValidateSpec" function. This can potentially cause problems as the "chainID" should only contain letters or digits. The report suggests validating the "chainID" in the "ValidateSpec" function and mentions that this issue has been fixed in a recent patch.

## Details
## DelegationKey

`DelegationKey` and `CuTrackerKey` utilize a space character (`" "`) as a delimiter to split the composite key into its constituent parts. However, the `spec.Index` (which is the chainID used for delegation key) is used in these functions without any validation within `ValidateSpec`.

> _dualsstaking/types/keys.gogo_
>
> // DelegationKey returns the key/prefix for the Delegation entry in fixation store.
> // Using " " (space) as separator is safe because Bech32 forbids its use as part of
> // the address (and is the only visible character that can be safely used).
> // (reference https://en.bitcoin.it/wiki/BIP_0173#Specification)

```go
func DelegationKey(provider, delegator, chainID string) string {
    return provider + " " + delegator + " " + chainID
}
```

## Remediation

Validate the chainID in `ValidateSpec`, ensuring it only contains letters or digits.

## Patch

Fixed in PR #1218.
