# _userInfo.liquidated is not set when liquidatePosition is a success

## Summary
The `liquidatePosition` function is not working properly because it is not setting the `_userInfo.liquidated` variable to true after a successful liquidation. This means that the same position can be liquidated multiple times, causing potential issues. The recommendation is to fix this by setting `_userInfo.liquidated` to true.

## Details
**Severity**: High

**Status**: Resolved

**Description**

When the `liquidatePosition` is successful the _userInfo.liquidated is not set to true which will lead to repeated liquidations of the same position

**Recommendation**: 

set _userInfo.liquidated to true
