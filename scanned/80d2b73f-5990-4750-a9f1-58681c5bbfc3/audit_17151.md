# DoS in Governance with Empty Collections Array . . . . . . .

## Summary
The Governor contract has a bug where the voting process can get stuck due to a mutex being locked. This happens because the proposeNewCollection and proposeRemoveCollection functions allow for an empty list to be proposed, but the accepting and rejecting functions do not allow it. As a result, the mutex cannot be unlocked, causing a denial of service. To fix this, a length check has been added to the proposeNewCollection and proposeRemoveCollection functions. The bug has been fixed in version 9d73ec2.

## Details
## Governor Contract Mutex Issue

In the Governor contract, voting uses the `changeLive` value in `AbacusController` as a mutex. The proposing function locks the mutex, meaning that only the accepting and rejecting functions can remove the mutex after voting.

The `proposeNewCollection` and `proposeRemoveCollection` functions allow proposing an empty list, but the accepting and rejecting functions will not allow an empty list of collections.

## Code Snippet

```solidity
// contracts/Governor.sol
function newCollectionAcceptance() external {
    require(controller.changeLive());
    require(controller.pendingWLAdditions(0) != address(0));
    ...
}

function newCollectionRejection() external {
    require(controller.changeLive());
    require(controller.pendingWLAdditions(0) != address(0));
    ...
}
```

Since these functions return an error in the case of a proposed empty list, the `changeLive` mutex cannot get unlocked, leading to a denial of service during the voting process of the Governor contract.

## Proof of Concept

An attacker calls `proposeNewCollection` with an empty list of collections, locking the mutex. The `newCollectionAcceptance` and `newCollectionRejection` fail with an error, preventing the mutex from being unlocked.

## Remediation

Add a check for the length of the array in the `proposeNewCollection` and `proposeRemoveCollection` functions.

### Patch

Length check was added in `proposeNewCollection` and `proposeRemoveCollection` functions. Fixed in commit `9d73ec2`.
