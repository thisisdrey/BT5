# Participation caps can be altered mid-crowdsale

## Summary
This bug report is about the functions setTier1Participants, setTier2Participants, and setHardParticipationCap in the KinTokenSale contract. These functions allow the owner of the token sale contract to modify the cap for any participant at any point in time, even after the crowdsale has started. This could lead to a lack of public trust on the sale mechanics. To improve the trust, the team suggests adding an onlyBeforeSale modifier to these functions so that the caps are frozen once the sale starts and cannot be modified. However, the team explained that this is intentional so that potential errors in the KYC process can be corrected by changing a participant’s tier at any moment.

## Details
The functions [`setTier1Participants`](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L192), [`setTier2Participants`](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L198), and [`setHardParticipationCap`](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L204) allow the owner of the token sale contract to modify the cap for any participant at any point in time, even after the crowdsale has started. To improve the public trust on the sale mechanics, it is recommended that once the sale starts, all caps are frozen and cannot be modified. Consider adding an `onlyBeforeSale` modifier to these functions.


***Update:** The team explained that this is intentional so that potential errors in the KYC process can be corrected by changing a participant’s tier at any moment.*
