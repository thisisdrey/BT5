# Index Value Mismatch

## Summary
The bug report discusses an issue with the initAuctionOnVault function in the Auction.sol code. This function is responsible for setting up auctions for non-fungible tokens stored in a vault. However, there is a problem with the for loop within the function, which is causing the incorrect token to be removed from the freeTokenIds array. This results in the wrong token being associated with the auction, leading to unexpected behavior during the auction process. To fix this issue, the code needs to be modified to use the correct index when updating the freeTokenIds array. This bug has been fixed in the latest version of the code.

## Details
## Vulnerability Report: Auction Initialization in Vault

While initializing an auction for a specific non-fungible token within the vault, `initAuctionOnVault` removes the incorrect free one. `initAuctionOnVault` sets up the auction for non-fungible tokens stored in a vault, calculates the safe box expiry timestamps, and initializes auction parameters for each one. 

However, in the for loop, when it updates the `freeTokenIds` array by replacing the token’s ID at the current index (`i`) with the last token’s ID in the array, the index utilized is incorrect.

## Code Snippet

```solidity
// Auction.sol
function initAuctionOnVault(
    CollectionState storage collection,
    mapping(address => UserFloorAccount) storage userAccounts,
    address creditToken,
    address collectionId,
    uint256[] memory vaultIdx,
    address bidToken,
    uint96 bidAmount
) public {
    [...]
    /// vaultIdx keeps asc order
    for (uint256 i = vaultIdx.length; i > 0;) {
        unchecked {
            --i;
        }
        [...]
        auctionTemplate.activityId = collection.generateNextActivityId();
        collection.activeAuctions[nftId] = auctionTemplate;
        activityIds[i] = auctionTemplate.activityId;
        collection.freeTokenIds[i] = collection.freeTokenIds[collection.freeTokenIds.length - 1];
        collection.freeTokenIds.pop();
    }
    [...]
}
```

The program replaces `collection.freeTokenIds[i]` with the last element of the array while removing a non-fungible token from the `freeTokenIds` array. Thus, an incorrect index will result in the deletion of the incorrect free one, associating the wrong token with the auction, as the one that should have been removed may remain in the `freeTokenIds` array. The one intended to stay may be removed, causing inconsistency between the actual and intended states of the `freeTokenIds` array, resulting in unexpected behavior during the auction process.

## Remediation

Ensure the code is modified to utilize `vaultIdx[i]` as the index instead of `i`.

## Patch

Fixed in commit `d57c194`.
