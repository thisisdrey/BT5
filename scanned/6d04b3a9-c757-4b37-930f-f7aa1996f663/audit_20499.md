# Error in condition

## Summary
This bug report is about an error in a code that updates token shares. On line 615, there is a mistake in the condition "value >= 0". It should actually be "value > 0" because the data type used is unsigned and the intention is to prevent transactions with a value of 0. The recommendation is to fix this condition.

## Details
**Description**
updateTokenShares(uint8), line 615
Condition “_value >= 0” contains a mistake - it should be “_value > 0”, because uint8 is an
unsigned type and the purpose of the condition is to prevent 0 amount transactions.
**Recommendation**:
Correct the condition.
