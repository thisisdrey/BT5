# Killed Gauge Can Be Voted For

## Summary
The VoterV3 contract has a safety feature that allows for gauges to be temporarily or permanently removed if they are found to be malicious. However, during the voting phase, there is a check to see if the address is a gauge, but it does not take into account if the gauge is still alive. This means that users can vote for gauges that have been temporarily removed, which can result in locking funds on the VoterV3 contract. To fix this issue, the ability to vote for killed gauges should be removed. This issue has been resolved in a recent pull request.

## Details
The `VoterV3` contract has safety measures put in place, allowing for the [temporary](https://github.com/ThenafiBNB/THENA-Contracts/blob/2bb84cf6bb429e9bac8ac8464f4546da855ef4ca/contracts/VoterV3.sol#L298-L303) and [permanent](https://github.com/ThenafiBNB/THENA-Contracts/blob/2bb84cf6bb429e9bac8ac8464f4546da855ef4ca/contracts/VoterV3.sol#L316-L332) removal of gauges in the case they are malicious. In order to track the state of a gauge, the [`isAlive` mapping](https://github.com/ThenafiBNB/THENA-Contracts/blob/2bb84cf6bb429e9bac8ac8464f4546da855ef4ca/contracts/VoterV3.sol#L65) will be set to either `true` or `false` depending on whether a gauge is alive or killed respectively. Both the temporary and permanent removal of gauges sets the `isAlive` mapping to `false`.


During the voting phase, there only exists a [check](https://github.com/ThenafiBNB/THENA-Contracts/blob/2bb84cf6bb429e9bac8ac8464f4546da855ef4ca/contracts/VoterV3.sol#L435) to see if the address is a gauge, which doesn't take into account if the gauge is still alive. This check will pass for any gauge temporarily killed, as a temporarily killed gauge will still reside in the `isGauge` mapping. This means users can vote for killed gauges.


Voting for killed gauges should not be allowed, as it results in reward tokens being allocated towards the gauge without any way to claim them. This means any votes for temporarily killed gauges will lock funds on the `VoterV3` contract.


Consider removing the ability to vote from a killed gauge in order to avoid locking funds on the `VoterV3` contract.


***Update:** Resolved in [pull request #3](https://github.com/ThenafiBNB/THENA-Contracts/pull/3) at commit [64bf6c5](https://github.com/ThenafiBNB/THENA-Contracts/pull/3/commits/64bf6c5da73f396fd4f0880508fd42c0b1344827).*
