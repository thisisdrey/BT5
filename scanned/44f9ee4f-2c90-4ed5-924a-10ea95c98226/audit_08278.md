# MANAGER ADDRESS CANNOT BE TRANSFERRED

## Summary
This bug report discusses a problem with updating the configuration in certain contracts. The issue is that there is no option to change the privileged address, which can cause problems if the current address is compromised or needs to be changed for operational reasons. The affected functions are located in the airdrops-registry, reward, and staking contracts. The impact of this bug is rated as a 4 out of 5, with a likelihood of 2 out of 5. The issue has been solved with a recommended fix in a specific commit.

## Details
##### Description

The functions to update the configuration in the contracts within scope lacked the option of setting a new `manager` as a privileged address.
If the keys of the `manager` account were suspected to be compromised, or the development team needed to change the address for an operational reason, a sizable portion of the contract's functionality will be rendered unusable.
