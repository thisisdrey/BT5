# Potential Performance Issue with Unbounded Loops.

## Summary
This is a bug report about a medium severity issue that has been resolved. The report mentions two functions, `leavePool` and `redeem`, in the PROPCStakingV2.sol file that have a problem with looping through user stakes. This means that as a user has more stakes, the functions may require more gas to execute, which could cause them to exceed the block gas limit and become unexecutable. This could potentially lead to a denial of service. The recommendation is to implement a mechanism to limit the number of active stakes per user, and after the audit, a limit of 10 stakes per user was put in place.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

PROPCStakingV2.sol: leavePool(), redeem().

Several functions iterate over arrays without explicit bounds, specifically when looping through user stakes. As a user's number of stakes increases, functions like `leavePool` and `redeem` require more gas to execute, potentially reaching block gas limit constraints. This situation could lead to denial of service if the functions become unexecutable due to exceeding gas limits.

**Recommendation:**

Implement a mechanism to limit the number of active stakes per user.

**Post audit.**

A limit of 10 stakes per user was introduced.
