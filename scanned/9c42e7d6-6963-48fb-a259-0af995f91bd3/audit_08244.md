# POSSIBLE LOSS OF OWNERSHIP

## Summary
This bug report is about a problem with transferring ownership of a protocol. The report says that there is no check to make sure the new address is valid and active. This means that if there is a mistake in the transfer, the whole protocol can be locked out from its important functions. The code location for this problem is in a file called Ownable.sol. The report gives a code snippet that shows where the issue is. The report also gives a score for the impact and likelihood of this problem, with a high impact score of 5 and a low likelihood score of 1. The recommendation from the team is to accept the risk for now, but they plan to fix it in a future upgrade.

## Details
##### Description

When transferring ownership of the protocol, no checks are performed on whether the new address is valid and active. In case there is a mistake when transferring the ownership, the whole protocol is locked out of its permissioned functionalities.

Code Location
-------------

#### Ownable.sol

```
    function transferOwnership(address _newOwner) public virtual onlyOwner {
        require(_newOwner != address(0), "Ownable: new owner is the zero address");
        _setOwner(_newOwner);
    }

```

##### Score

Impact: 5  
Likelihood: 1

##### Recommendation

**RISK ACCEPTED**: The `NFTfi team` accepted the risk of this finding. The team most likely will implement the remediation when they upgrade the platform.
