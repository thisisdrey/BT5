# Using Storage Root Value After Reset

## Summary
The report describes a bug in the reset_active_merkle_tree instruction, which is used to close and activate new Merkle trees. The bug occurs because the storage root value is taken after resetting the storage, which returns a default value instead of the actual root value. This can be seen in the test case provided, where the get_root function fails to give the correct value after the reset. The recommended fix is to fetch the storage root before resetting.

## Details
## reset_active_merkle_tree Instruction

The `reset_active_merkle_tree` instruction is used to close the active Merkle tree by storing its root value in a nullifier account and activating a new one. However, the storage root value is taken after resetting the storage, which returns a default value.

## Code Snippet 1
```rust
src/processor/accounts.rs
```
```rust
storage_account.reset();
storage_account.set_trees_count(&(active_merkle_tree_index.checked_add(1).ok_or(MATH_ERR)));
active_nullifier_account.set_root(&storage_account.get_root());
```

The reset function updates the `next_commitment_ptr` to `0`, which was used in the `get_node` function invoked by the `get_root` function.

## Code Snippet 2
```rust
src/processor/accounts.rs
```
```rust
let ptr = self.get_next_commitment_ptr() as usize;
// Accessing a node, that is non-existent (yet) -> we use the default value

    EMPTY_TREE[MT_HEIGHT as usize - level]

```

Since the `ptr` value is zero, `use_default_value` returns true. The `get_root` function returns the default value instead of the actual root value.

## Proof of Concept

Here is a test case for proving `get_root` fails to give the actual value after a reset.

## Code Snippet 3
```rust
src/processor/accounts.rs
```
```rust
#[test]
fn test_reset_root() {
    let mut data = vec![0; StorageAccount::SIZE];
    generate_storage_accounts_valid_size!(accounts);
    let mut storage_account = StorageAccount::new(&mut data, accounts).unwrap();

    let temp = Fr::new(BigInteger256::new([
        11638450640257199038,
        7595865483997155469,
        13070293587080331272,
        1693899333659075201,
    ]));

    let test_root = fr_to_u256_le(&temp);
    assert_ne!(storage_account.get_root(), test_root);
    storage_account.set_node(&test_root, 0, 0);
    storage_account.set_next_commitment_ptr(&1);
    assert_eq!(storage_account.get_root(), test_root); // Temp root
    storage_account.reset();
    assert_eq!(storage_account.get_root(), test_root); // Fails
}
```

## Remediation

Use the root value of storage before the reset.

## Patch

Fixed in commit `d5c6f1d` by fetching the storage root before resetting.
