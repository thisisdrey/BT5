# INCORRECT INVARIANT FACTOR CALCULATION MAY LEAD TO A LOSS OF ACCRUED FUNDS

## Summary
The GammaSwap protocol has a bug in its invariant factor formula, which calculates a variable used in multiple places across the contracts. The calculation only considers the number of decimals of the first token in a token pair, instead of both tokens. This affects borrowed and repaid liquidity, as well as storage variables in the protocol. A proof of concept has been provided to demonstrate the issue and a screenshot of the code location is included. The impact and likelihood of this bug have been rated as 4 and 3 respectively. A recommendation has been made to solve this issue by calculating the invariant factor as 10^18 instead of just using the first token's decimals. The bug has been confirmed and solved by the GammaSwap and Halborn teams.

## Details
##### Description

The invariant factor formula makes use of the number of decimals of the first token in token pair. The result of this calculation is further used in multiple places across the contracts. For example, when you borrow liquidity from a GammaSwap pool, the protocol calculates that factor to update some storage variables such as `lastFeeIndex` and `BORROWED_INVARIANT`. Instead of focusing on `s.decimal[0]` only, both decimals should be considered.

As a result, calculating this variable based on a wrong number of decimals affects borrowed/repaid liquidity and many storage variables in the protocol.

function getInvariantFactor() internal virtual override view returns(uint256) {
    return 10 ** s.decimals[0];
}

#### AbstractRateModel.sol

```
function calcUtilizationRate(uint256 lpInvariant, uint256 borrowedInvariant) internal virtual view returns(uint256) {
    uint256 totalInvariant = lpInvariant + borrowedInvariant;
    if(totalInvariant == 0)
        return 0;

    return borrowedInvariant * getInvariantFactor() / totalInvariant;
}

```

##### Recommendation

**SOLVED:** This finding was identified in a code walkthrough jointly by the `GammaSwap team` and the `Halborn team`, and the existence of the finding was confirmed by the `Halborn team`.
The invariant factor is now calculated as `10**18`.
