# [M-02] Missing payable

## Summary
This bug report is about a vulnerability in the LPool.sol contract. The function addReserves is not payable, but it uses msg.value. This means that it can lead to undesired behavior. To fix the vulnerability, the function should be made payable. This means that anyone calling the function should be able to pay for it. This will ensure that the function is not misused.
