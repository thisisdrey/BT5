# Wrong reward calculation of balance < 100

## Summary
A bug has been discovered in the Bondappetit Protocol code. It is found in the VestingSplitter.sol contract, at line 126. The code is meant to calculate the reward for an account, but it always returns zero if the balance is less than 100.

The recommendation is to perform division after multiplication. This should fix the bug and ensure that the reward calculation works correctly.

## Details
##### Description
At the line https://github.com/bondappetit/bondappetit-protocol/blob/355180f0aca0b29d60d808f761052956b7a3a159/contracts/VestingSplitter.sol#L126 contract calculate `reward` for account, however that calculation always return zero if `balance < 100`

##### Recommendation
We suggest to perform division after multiplication
