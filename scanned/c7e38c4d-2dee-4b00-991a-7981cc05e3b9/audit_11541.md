# Division before multiplication results in lower `dittoMatchedShares` distributed to users

## Summary
This bug report is about a vulnerability in the code for a dittoMatchedShares feature. It is of medium risk, and can cause up to 7.1% of user's shares to be truncated. The issue is caused by the division before multiplication in the code, which results in a lower amount of shares distributed to users. The manual review was used to identify the issue. The recommendation to fix the bug is to replace the code with a new code that has the multiplication before division.

## Summary
Shares amount is rounded down to number of days staked. Max truncation is 1 day, min time is 14 days. At most 1 / 14 * 100% = 7.1% of accrued shares will be truncated.

## Vulnerability Details
Division before multiplication
```solidity
            uint88 shares = eth * (timeTillMatch / 1 days);
```
Suppose `timeTillMatch = 14.99 days`, `eth = 1e18`. Expected result is `14.99 * 1e18 / 1 = 14.99e18 shares`. Actual result is `1e18 * (14.99 / 1) = 14e18 shares`

## Impact
Up to 7.1% of user's shares will be truncated

## Tools Used
Manual Review

## Recommendations
```solidity
-           uint88 shares = eth * (timeTillMatch / 1 days);
+           uint88 shares = uint88(uint256(eth * timeTillMatch) / 1 days);
```
