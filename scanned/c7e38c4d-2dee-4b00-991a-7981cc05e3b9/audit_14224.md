# Storage Fee Uncollected When Listing NFTs

## Summary
The bug report states that there is a problem with the nft_on_approve function in the Mintbase marketplace. This function allows NFTs to be added to listings and increases storage usage, but the issue is that storage fees are not collected from the caller. This means that an attacker could repeatedly approve NFTs and lock up all the funds in the marketplace. The code responsible for this is listed in the report and a possible solution is suggested, which is to require approvals to properly fund potential storage usage. The report also mentions that Mintbase has acknowledged the issue and will refund the marketplace if necessary.

## Details
## NFT Approval Vulnerability

## Overview
The `nft_on_approve` function adds NFTs to listings and increases storage usage. Since storage fees are not collected from the caller, it is possible for an attacker to lock up all funds in the Marketplace by repeatedly approving NFTs to it.

```rust
pub(crate) fn listing_insert_internal(
    &mut self,
    token_id: U64,
    approval_id: U64,
    owner_id: &AccountId,
    sale_args: &mb_sdk::data::market_v1::SaleArgs,
) -> TokenListing {
    let approval_id: u64 = approval_id.into();
    // Create the tokens. Skip any tokens that are already listed.
    let key = TokenKey {
        token_id: token_id.0,
        account_id: env::predecessor_account_id().to_string(),
    };
    let token = TokenListing::new(
        owner_id.clone(),
        env::predecessor_account_id(),
        token_id.into(),
        approval_id,
        sale_args.autotransfer,
        sale_args.price,
    );
    match self.listings.get(&key) {
        None => {
            self.listings.insert(&key, &token);
        }
        Some(old_token) => {
            // token has been relisted, handle old token data and reinsert.
            self.delist_internal(&key, old_token);
            self.listings.insert(&key, &token);
        }
    }
    token
}
```

## Remediation
Require approvals to properly fund potential storage usage.

## Patch
Mintbase acknowledged this issue and decided to keep the original implementation for NEP compatibility reasons. Marketplace will be refunded by Mintbase to account for additional locked storage fees if necessary.
