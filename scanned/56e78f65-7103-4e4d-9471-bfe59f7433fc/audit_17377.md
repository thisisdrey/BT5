# [M-09] VaultTracker has the wrong admin

## Summary
This bug report is about an issue with the Marketplace, Creator, and VaultTracker contracts which are part of the code-423n4/2022-07-swivel repository on GitHub. The Marketplace contract calls the Creator contract, which creates an instance of the ZcToken and a VaultTracker. The VaultTracker takes the msg.sender as the admin, which in this case is the address of the Creator contract. This means that the Creator contract is not able to interact with the VaultTracker, which is not intended. Manual analysis was used to identify this issue. To fix this issue, it is recommended to modify the constructor of the VaultTracker contract so that the Creator contract can pass in the Marketplace's address to be used as admin.

### Description

`MarketPlace.createMarket()` calls `Creator.create()` which creates an instance of `ZcToken` and a `VaultTracker`. `VaultTracker` takes `msg.sender` as the admin. We know that if contract A calls contract B which calls contract C, `msg.sender` in contract C is the address of B i.e. the `msg.sender` in VaultTracker is the address of the creator contract. However, the creator contract is not able (and not supposed to) interact with the VaultTracker unlike the marketplace contract.

### Recommended Mitigation Steps

Modify the constructor of the VaultTracker contract so that the creator contract can pass in msg.sender (MarketPlace’s address) to be used as admin.
