# Division by zero is possible

## Summary
This bug report is about a problem in the code of the SushiSwap Bentobox project. At certain lines of code, it is possible that the variables `_totalBorrow.fraction`, `_totalAsset.amount`, `_totalBorrow.amount`, `_totalAsset.fraction`, `_totalBorrow.fraction`, `info[i].totalAssetFraction`, `info[i].totalBorrowFraction`, `priceC`, `decimals`, `division`, `_getPrice(collateralSymbol)`, `_peekPrice(collateralSymbol)`, `timeElapsed`, and `denominator` can be equal to zero. This can cause a division by zero error.

To fix this problem, the recommendation is to use the SafeMath Library to redo the division operation. The SafeMath Library provides functions that check for division by zero errors and other arithmetic errors, and will throw an error if any of these occur. This will prevent the division by zero error from occurring.
