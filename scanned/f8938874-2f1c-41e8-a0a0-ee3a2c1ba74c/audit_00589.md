# Insufficient Testing of Parsers

## Summary
The bug report states that there is a lack of unit tests and coverage in the `app` directory of a project that uses both official and custom parsers for transaction hashes. This is a concern as unit tests are important for ensuring correct operation, security, and results. The recommendation is to perform sufficient unit tests and aim for a coverage of at least 80% in the `app` directory. This will help ensure the quality and functionality of the project.
