# Missed depositary check

## Summary
This bug report is about a function called `buy` in the CollateralMarket.sol contract, located at https://github.com/bondappetit/bondappetit-protocol/blob/88680691fe8d872c5fc26e9500d19cf7caaa9861/contracts/CollateralMarket.sol#L120. This function exchanges collateral tokens for stable tokens, but if the wrong depositary is used, it can lead to a collateralization disbalance. This is a problem even with a manual depositary changing mechanism, as the issuer requires an exact list of depositaries and the transaction won't fail due to the fault tolerance of the rebalance call.

The recommendation is to check the depositary to ensure it is correct. This will help avoid the collateralization disbalance, and ensure that the issuer has an exact list of depositaries.

## Details
##### Description
In function `buy` defined at https://github.com/bondappetit/bondappetit-protocol/blob/88680691fe8d872c5fc26e9500d19cf7caaa9861/contracts/CollateralMarket.sol#L120 contract exchanges collateral tokens to stable tokens. But in case of wrong `depositary` that code will lead to collateralization disbalance, that is bad even you have manual `depositary` changing mechanism because `issuer` requires exact list of depositaries and transaction wont fail because `rebalance` call is fault tolerance.

##### Recommendation
We recommend check depositary
