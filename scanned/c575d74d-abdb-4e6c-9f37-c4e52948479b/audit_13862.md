# When a purchase occurs, other markets do not maintain the same utilization 

## Summary
The bug report is about an issue in the ProtectionPurchaser contract, where a certain invariant is not being held for when a purchase occurs. This is causing a discrepancy in the utilization data for different markets. The report includes a test case and notes explaining the root cause of the issue and a recommendation to fix it by calling a certain function in purchases. The bug has been addressed in the Cozy contract, but still needs to be fixed in the Cantina contract.

## Details
## ProtectionPurchaser.sol

## Description
The following invariant for when a purchase occurs was not holding: "Markets that did not have a purchase occur must have the same utilization."

1. Calling -> `MockTrigger(triggers[1]).mockState(MarketState.TRIGGERED`
2. Calling -> `contract_WETH.approve(address(contract_CozyRouter), assetsNeeded_)`
3. Calling -> `contract_CozyRouter.purchase(contract_SetWETH, 2, 1e18, buyer1, type(uint256).max)`

## Notes
As it is done in sells and deposits, it is also needed to `dripSupplierFees()` in purchases. If this is not done, the calculated drip amount is `(fees from previous purchases + fees from new purchase) * drip rate` instead of `fees from previous purchases * drip rate`. You want to ensure that the fees from previous purchases have an amount dripped, but the new fees (if no time has elapsed) should not be dripped.

### For example:
- Bob purchases protection, adding `100` to the purchaseFeePool
- Some time elapses
- Alice purchases protection, adding `50` to the purchaseFeePool
- Alice purchases some more protection

Since no time has elapsed since Alice's first purchase, none of the `50` she added to the pool should be dripped/calculated for the next drip amount yet, but some of Bob's should.

Because this is not the case before this change set, the utilization in markets that didn’t have the purchase occur in is lower than before because the `totalCollateralAvailable` calculation adds the amount of assets to be dripped that haven’t been dripped yet, which is higher than it should be since we apply `dripRate * all purchaseFeePools`.

## Recommendation
Call `dripSupplierFees()` in purchases.

- **Cantina:** Fixed.
