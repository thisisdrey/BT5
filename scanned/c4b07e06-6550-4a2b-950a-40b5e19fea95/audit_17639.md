# Old governor can call acceptGov() after renouncing its role through _abdicate()

## Summary
This bug report is regarding a function called __abdicate in an Ethereum smart contract called Gov.sol. This function is used to remove the current governor and set a new governor. The bug is that the pendingGov value is not being reset to 0 when this function is called. This means that if a pending governor is set, the user can become a governor by calling acceptGov.

To fix this bug, it is recommended to set the pendingGov to address(0) inside the __abdicate function. This will ensure that the pendingGov value is reset to 0 whenever the __abdicate function is called. This will prevent users from becoming governors without being properly accepted.

## Details
## High Risk Issue

**Severity:** High Risk  
**Context:** Gov.sol#L30  

### Description
The `__abdicate` function does not reset the `pendingGov` value to 0. Therefore, if a pending governor is set, the user can become a governor by calling `acceptGov`.

### Recommendation
Consider setting `pendingGov` to `address(0)` inside the `__abdicate` function.

```solidity
function __abdicate() governed external override {
    address old = gov;
    gov = address(0);
    + pendingGov = address(0);
    emit NewGov(old, address(0));
}
```
