# `dayCounter` Not Updated in `forceSkipEpoch()` May Lead to DoS

## Summary
The bug report is about a problem in a file called `DataMarket.sol`. The function `releaseEpoch()` is supposed to increase the `dayCounter` value when the `epochIdCounter` overflows to the next day. However, the same thing does not happen in the function `forceSkipEpoch()`, which can be used by the owner to skip a certain number of epochs. This can lead to a Denial of Service (DoS) attack when the sequencer calls `updateRewards()`. To fix this issue, it is recommended to update the day counter with the number of skipped days.

## Details
**File(s) affected:**`DataMarket.sol`

**Description:** The function `releaseEpoch()` increments the `dayCounter` value if the `epochIdCounter` overflows to the next day. The same logic does not exist in `forceSkipEpoch()` which can be called by the owner to skip a number of epochs, which may lead to a DoS when the sequencer calls `updateRewards()`.

**Recommendation:** Update the day counter with the number of skipped days.
