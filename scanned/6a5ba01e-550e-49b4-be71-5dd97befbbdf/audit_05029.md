# Incorrect Delivery Cost

## Summary
The handleInstantAction function in the SpokeOptimisticFinalityLogic.sol solidity file uses the same array indices for both the instantAction and finalizeCredit messages, which conflicts with the design of the costs array. This can lead to issues with insufficient msg.value and hinder the protocol's robustness. To fix this, the indices of costs need to be changed from 0 to 1 in the second function. This issue has been fixed in the code.

## Details
## Analysis of handleInstantAction Function

The `handleInstantAction` function uses the same array indices of the `costs` array for both the `instantAction` and `finalizeCredit` messages. This conflicts with the design of the `costs` array, which was intended to return two entries: the first for the instant action and the second for the finalize action.

## Code Reference
> _src/libraries/logic/optimisticFinality/SpokeOptimisticFinalityLogic.sol_

```solidity
function handleInstantAction(

        CommonAccountingLogic.handleInboundTokensAndAdjustAction(action, asset, amount,
        weth, totalCost);
    }
    if (CommonOptimisticFinalityLogic.getActionDirection(action) ==
        HubSpokeStructs.ActionDirection.Inbound) {
        /* ... */
        if (storedCredit.createdAt == 0) {
            /* ... */
            _sendInstantActionTunnelMessage(
                ofState,
                commState,
                action,
                storedCredit.token, // same as b32(a(asset)), but already as b32
                amount,
                costs[0],
                costForReturnDelivery[0]
            );
        } else {
            /* ... */
        }
        _sendFinalizeCreditTunnelMessage(
            commState,
            storedCredit,
            asset,
            costs[0],

        );

    /* ... */
}
```

## Impact of the Issue

The impact of this issue hinders protocol robustness. When `costs[0]` is significantly smaller than `costs[1]`, the hub would be unable to send the finalize credit response due to the depletion of `msg.value`.

## Remediation

Fix the indices of `costs` in the second function from `0` to `1`.
