# Wrong comparison

## Summary
This bug report is about a problem in the code of the WETH-maker project. The code in question is located at line 217 of the Strategy.sol file. The issue is that the left operand of the code has a unit of `1e+6` while the right operand has a unit of `1e+10`. This means that if the value of `c_safe` is set to 40000, the code will be ignored.

The recommendation is to change the value of `c_safe` after deployment. This will ensure that the code is not ignored and will make sure that the project runs as expected.

## Details
##### Description
At line: https://github.com/orbxball/weth-maker/blob/39cfbee59bcebfce19a5b9ac6f11fb84f3ab7b23/contracts/Strategy.sol#L217 left operand has unit `1e+6` and the right operand has unit `1e+10`.
```solidity=
if (_current > DENOMINATOR.mul(c_safe).mul(1e2)) {//  1e6 > 10000 * 40000 * 1e2 
    _current = DENOMINATOR.mul(c_safe).mul(1e2);

```
In this implementation where `c_safe = 40000;` this code will be ignored.

##### Recommendation
It is recommended to change `c_safe` value after deploy.
