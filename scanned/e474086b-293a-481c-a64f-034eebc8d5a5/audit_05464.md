# Removing assets from _supportedAssets bricks pending redemptions in LevelMinting 

## Summary
The report discusses a bug in the LevelMinting contract code. The _transferToBeneficiary() function is responsible for transferring collateral assets to users, but it has a check that only allows supported assets to be transferred. This means that if an asset is removed from the list of supported assets, it cannot be withdrawn by users or the protocol. The report recommends adding another set to track all past and present collateral assets and changing the check in _transferToBeneficiary() to use this new set. The bug has been fixed and verified by the Cantina team.

## Details
## LevelMinting Vulnerability Report

## Context
- **File**: LevelMinting.sol
- **Line Numbers**: `#L559`, `#L372-L377`

## Description
In `LevelMinting`, the `_transferToBeneficiary()` function checks if the asset to be transferred is in the `_supportedAssets` set:

```solidity
if (!_supportedAssets.contains(asset)) revert UnsupportedAsset();
```

`_transferToBeneficiary()` is used to transfer collateral assets to users upon withdrawal. As such, once a collateral asset is removed from `supportedAssets`, it cannot be withdrawn by users anymore as this check would fail.

This is problematic as:
- All pending redemptions with that asset as collateral cannot be completed.
- If the protocol still owns some of that collateral asset, it cannot be withdrawn forever.

This makes it such that an asset cannot be removed from `supportedAssets` unless the protocol's balance for that asset is zero.

## Recommendation
Consider adding another set that tracks all past and present collateral assets:

```solidity
function addSupportedAsset(address asset) public onlyRole(DEFAULT_ADMIN_ROLE) {
    if (asset == address(0) || asset == address(lvlusd) || !_supportedAssets.add(asset)) {
        revert InvalidAssetAddress();
    }
    _redeemableAssets.add(asset);
    emit AssetAdded(asset);
}
```

The check in `_transferToBeneficiary()` can then be changed to use `_redeemableAssets`:

```solidity
if (!_supportedAssets.contains(asset)) revert UnsupportedAsset();
```
becomes:
```solidity
if (!_redeemableAssets.contains(asset)) revert UnsupportedAsset();
```

## Status
- **Level Money**: Fixed in commit `5afc1851`.
- **Cantina**: Verified, the recommended fix was implemented.
