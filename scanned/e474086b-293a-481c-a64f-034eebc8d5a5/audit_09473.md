# Can Redeem Even when `Fintroller` Disallows Redemption

## Summary
The bug report is about a problem with the `HToken.redeem()` function and the `Fintroller` contract in the HiFi project. The `HToken.redeem()` function allows users to exchange hTokens for underlying tokens, but currently it does not properly check if the `Fintroller` allows for redemption. This means that users can redeem hTokens even if it is not allowed by the `Fintroller`. The recommendation is to fix this by adding a check in the `HToken.redeem()` function to make sure it only allows redemption if the `Fintroller` allows it. This bug has been fixed in a recent commit.

## Details
**Update**
Fixed in commit [3126a78](https://github.com/hifi-finance/hifi/commit/3126a78) by acknowledging `isRedeemHTokenAllowed` to be a left over of a previous refactoring and removing it.

**File(s) affected:**`packages/protocol/contracts/core/fintroller/Fintroller.sol`, `packages/protocol/contracts/core/h-token/HToken.sol`

**Description:** The `HToken.redeem()` function allows a user to redeem hTokens in exchange for underlying tokens after maturity. The `Fintroller` contract defines a configuration `isRedeemHTokenAllowed` for every `Bond` added which indicates whether the hToken can be redeemed. However, the `HToken.redeem()` currently allows a user to redeem hTokens even if `isRedeemHTokenAllowed` is false as defined by the `Fintroller`.

**Recommendation:** Add validation to the `HToken.redeem()` function so that it reverts if `isRedeemHTokenAllowed` is false as defined by the `Fintroller`.
