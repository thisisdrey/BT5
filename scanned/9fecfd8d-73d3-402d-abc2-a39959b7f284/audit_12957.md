# Incomplete Cancel Order Implementation

## Summary
The report states that there is a bug in the "cancelOrder" function where it does not return the collateral and execution fees paid by the creator of the order. This bug can be found in the "ExecuteOrderFacet.sol" file in the "contracts/manager/facets/order" directory. The function only cleans up certain storage variables without properly returning the fees. To fix this bug, the functionality of "cancelOrder" needs to be extended so that it also returns the collateral and fees. This bug has been fixed in the latest version of the code (274f9b0).

## Details
## cancelOrder Function Overview

The `cancelOrder` function only cleans up relevant storage variables without returning collateral and execution fees paid by the order’s creator.

## Source Code Location

`contracts/manager/facets/order/ExecuteOrderFacet.sol` (SOLIDITY)

```solidity
function cancelOrder(bytes32 key) external {
    OrderState storage orderState = LibStorage.orderState();
    Order memory order = orderState.orders[key];
    require(order.owner == msg.sender, "not owner");
    require(order.orderType != OrderType.INVALID, "not active");
    
    if (order.orderType == OrderType.PERP_INCREASE) {
        LibPerpIncreaseOrder.cleanupPerpIncreaseOrder(key);
    } else if (order.orderType == OrderType.PERP_DECREASE) {
        LibPerpDecreaseOrder.cleanupPerpDecreaseOrder(key);
    } else if (order.orderType == OrderType.POOL_DEPOSIT) {
        LibPoolDepositOrder.cleanupPoolDepositOrder(key);
    } else if (order.orderType == OrderType.POOL_REDEEM) {
        LibPoolRedeemOrder.cleanupPoolRedeemOrder(key);
    } else if (order.orderType == OrderType.POOL_SWAP) {
        LibPoolSwapOrder.cleanupPoolSwapOrder(key);
    } else if (order.orderType == OrderType.OPTION_OPEN) {
        LibOptionOpenOrder.cleanupOptionOpenOrder(key);
    } else {
        revert("unsupported order type");
    }
    
    orderState.orderKeys.remove(key);
    delete orderState.orders[key];
    emit OrderCancelled(key);
}
```

## Remediation

Extend the functionality of `cancelOrder` so that it returns collateral and execution fees.

## Patch

Fixed in commit `274f9b0`.
