# Use of Foundry 's ffi cheatcode should be avoided if possible 

## Summary
This bug report is about a function called `constructCreate2Salt()` that uses a cheat code to generate a salt for deploying contracts in a program called Foundry. This cheat code is risky and can allow anyone to run commands on devices running the program's tests. It is recommended to use safer alternatives like `vm.readFile()` and `vm.parseJson()` instead. The bug has been fixed in the projects called Sablier and Cantina Managed.

## Technical Summary

## Context
- **File Locations:** 
  - `foundry.toml#L7-L12`
  - `Base.s.sol#L86`
  
## Description
The function `constructCreate2Salt()` utilizes Foundry's FFI cheatcode to generate a salt for the deterministic deployment of contracts, which is dependent on the version of those contracts. 

**Warning:** The use of FFI is considered dangerous, as documented [here](link-to-documentation). It is generally recommended to use this cheat code as a last resort and to avoid enabling it by default. Allowing this can grant anyone who can modify the project's tests the ability to execute arbitrary commands on devices running those tests.

## Recommendation
To mitigate risks, it is advised to use `vm.readFile()` and `vm.parseJson()` to achieve similar results with reduced risk.
