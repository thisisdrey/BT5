# Uninitialized Discount Variable Severity: Medium

## Summary
This bug report concerns the payBackLoanInternal function in a smart contract, which is used to calculate a fee discount for users who hold CoinlendCredits tokens. The problem is that the coinlendCreditsDiscount variable used in this function is never initialized and does not have a setter function, so it defaults to zero. As a result, users who hold these tokens never receive a discount on their fees.

To fix this issue, two steps are recommended. First, a setter function should be added to the contract to allow the contract owner to set the coinlendCreditsDiscount variable. This function should only be callable by the contract owner to prevent unauthorized changes to the discount rate. Second, the coinlendCreditsDiscount variable should be initialized in the contract constructor to a sensible default value. This will ensure that users who hold CoinlendCredits tokens receive a discount on their fees from the moment the contract is deployed.
