# Mooniswap pairs cannot be unpaused

## Summary
This bug report is about the `MooniswapFactoryGovernance` contract, which is used to pause and prevent any future swaps. The issue is that there is no way to unpause the contract or redeploy a `Mooniswap` instance for a given pair of tokens. This means that if a `Mooniswap` contract is ever paused, it will not be possible for that pair of tokens to ever be traded on the Mooniswap platform again, unless a new factory contract is deployed.

The bug report recommends providing a way for `Mooniswap` contracts to be unpaused. This would allow users to continue trading the given pair of tokens on the Mooniswap platform, without having to deploy a new factory contract.

## Details
The `MooniswapFactoryGovernance` contract has a [`shutdown` function](https://github.com/1inch-exchange/mooniswap-v2/blob/b9c06335fb1d68b19054442547fc677f42795b44/contracts/governance/MooniswapFactoryGovernance.sol#L48) that can be used to pause the contract and prevent any future swaps. However there is no function to unpause the contract. There is also no way for the factory contract to redeploy a `Mooniswap` instance for a given pair of tokens. Therefore, if a `Mooniswap` contract is ever shutdown/paused, it will not be possible for that pair of tokens to ever be traded on the Mooniswap platform again, unless a new factory contract is deployed.


Consider providing a way for `Mooniswap` contracts to be unpaused.
