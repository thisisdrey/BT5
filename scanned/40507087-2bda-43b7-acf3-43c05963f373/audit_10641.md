# Superpositions.onlyminter() has wrong implemerantion, leading to wrong access control 

## Summary
The bug report is about a function called onlyMinter() in a contract called SuperPositions. This function is used for access control, but it has a mistake in its code that makes it insecure. The problem is that instead of comparing the correct IDs, the function is comparing the wrong ones. This can lead to anyone being able to access the function, even if they are not supposed to. The report suggests a correction to the code that will fix this issue and make the function secure.

## Details
## SuperPositions.onlyMinter() Implementation Issue

## Context
SuperPositions.sol#L69-L83

## Description
The `SuperPositions.onlyMinter()` function has a faulty implementation, which results in incorrect access control. The issue arises because the current implementation compares the `formImplementationId` instead of the state registry ID for the superform against the state registry ID of `msg.sender`.

### Problem
The correct approach should involve retrieving the state registry ID of the superform using the function `getStateRegistryId()`, and then comparing this ID to the ID of `msg.sender`. Due to this incorrect comparison, the implementation fails, leading to any function that utilizes this modifier having improper access control, thereby creating a serious security vulnerability.

A similar issue is present with the modifier `onlyBatchMinter()`.

## Recommendation
Make the necessary changes to compare the correct state registry ID as shown below:

```solidity
modifier onlyMinter(uint256 superformId) {
    address router = superRegistry.getAddress(keccak256("SUPERFORM_ROUTER"));
    // if msg.sender isn't superformRouter then it must be state registry for that superform
    if (msg.sender != router) {
        (, uint32 formBeaconId,) = DataLib.getSuperform(superformId);
        (address superform_, uint32 formImplementationId_, uint64 chainId_) = DataLib.getSuperform(superformId);
        uint8 formRegistryId = ERC4626FormImplementation(superform).getStateRegistryId();
        uint8 registryId = superRegistry.getStateRegistryId(msg.sender);
        if (registryId != formRegistryId) {
            revert Error.NOT_MINTER();
        }
    }
    _;
}
```
