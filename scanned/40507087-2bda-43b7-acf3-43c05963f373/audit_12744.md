# Lack of checks of tier range in the `init` function of the `NFTMintSaleMultiple` contract

## Summary
A bug has been identified in the `NFTMintSaleMultiple` contract's `init` function. This function is used to verify that the id ranges of different tiers do not intersect with each other and appear in increasing order. However, there is a missing check for the last tier with index `0`. To fix this bug, a corresponding check should be added to the `init` function of the `NFTMintSaleMultiple` contract. This will ensure that all tiers are verified correctly.

## Details
##### Description
The `init` function within the `NFTMintSaleMultiple` contract currently verifies that the id ranges of different tiers do not intersect with each other and appear in increasing order. However, there is a missing check for the last tier with index `0`.
https://github.com/vibexyz/vibe-contract/blob/d08057edbaf83b00d94dcaca2a05e3c44a45e4d9/contracts/mint/NFTMintSaleMultiple.sol#L62

##### Recommendation
We recommend adding a corresponding check for the last tier in the `init` function of the `NFTMintSaleMultiple` contract.
