# Possible call to zero address

## Summary
The report states that there is a bug in the GmxAccountManager contract. This bug is classified as medium severity and has been resolved. The bug is related to the _executeAccountSet function, which is used to make external calls to each account using assembly. The bug causes the calls to still be executed even if some of the accounts have not been set properly, meaning they have a zero address. This can result in incorrect behavior and the execution will not be reverted. The recommendation to fix this bug is to add a sanity check to ensure that the target address is different from address(0). This will help ensure proper behavior of the contract.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In the GmxAccountManager contract, when the _executeAccountSet function is used to make external calls to each account using assembly, the calls will still be executed even if some of the accounts have not been set properly (i.e., have a zero address). In this case, the execution will not be reverted..

**Recommendation**: 

To ensure proper behavior, add a sanity check to ensure the target address is different from address(0).
