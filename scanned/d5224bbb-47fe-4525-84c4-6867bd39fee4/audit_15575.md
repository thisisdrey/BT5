# Operators might add the same validator more than once

## Summary
This bug report discusses an issue with the OperatorsRegistry contract, specifically the addValidators function. This function allows operators to add the same validator multiple times, which can cause a pool of depositors' funds to be directed to the same address multiple times. This will cause the address to have more than 32 eth, which will damage the capital efficiency of the entire deposit pool and potentially impact the pool's APY.

The recommendation is to add a de-duplication mechanism to the OperatorsRegistry contract. This would ensure that the same validator is not added more than once, thereby avoiding the issue of the address having more than 32 eth.

Alluvial has acknowledged this bug report.

## Details
## Vulnerability Report

**Severity:** Medium Risk  
**Context:** OperatorsRegistry.1.sol#L270  

## Description
Operators can use `OperatorsRegistryV1.addValidators` to add the same validator more than once. Depositors' funds will be directed to these duplicated addresses, which in turn, will end up having more than 32 ETH. This act will damage the capital efficiency of the entire deposit pool and thus will potentially impact the pool's APY.

## Recommendation
Consider adding a de-duplication mechanism.
