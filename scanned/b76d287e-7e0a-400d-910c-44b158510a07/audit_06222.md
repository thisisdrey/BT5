# Malicious Manager Can Pass Any Proposal and Drain Funds

## Summary
The client made a fix to the code, but there is still a problem in the `LiquidityFundsManage.sol` file. The `vote(uint256 id)` function is not properly checking if a manager has already voted on a proposal. This means that a manager can vote multiple times on the same proposal, potentially causing it to pass and allowing them to take funds from the contract. The recommendation is to update the code so that it keeps track of which managers have voted on which proposals.

## Details
**Update**
The client fixed the issue in commit `b4eb42c56d6265c4a42d8867281bd48c5e2dc913` by updating the `voteRecord` mapping to reflect that a manager has voted on a proposal, preventing them from voting more than once.

**File(s) affected:**`LiquidityFundsManage.sol`

**Description:** The `vote(uint256 id)` function checks if a manager has already voted on a proposal using the `voteRecord` mapping. However, the voting record is not updated once a manager has voted. This allows a malicious manager to repeatedly vote on the same proposal, potentially reaching the threshold required to pass the proposal. Additionally, a malicious manager can propose any malicious proposal and ensure it passes, allowing them to drain funds from the contract.

**Recommendation:** Update the `voteRecord` mapping in the `vote()` function to reflect that a manager has voted on a specific proposal.
