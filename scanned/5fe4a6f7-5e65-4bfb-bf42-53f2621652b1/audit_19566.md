# Input parameters in `notify()` not checked

## Summary
This bug report is regarding the `notify()` function in the StakingRewardsV3-1.sol contract. Currently, the `notify()` function can be called with `amount` equal to zero, which is an incorrect input data. To fix this, it is recommended to add a `require` statement in the `notify()` function to ensure that the `amount` is greater than zero. This will prevent incorrect input data from being passed to the function.

## Details
##### Description
`notify()` can be called with `amount` equal to zero:
https://github.com/keep3r-network/StakingRewardsV3/blob/13ecc6966ae1a413f62224382bfd4d64b1a22351/contracts/StakingRewardsV3-1.sol#L299
##### Recommendation
It is recommended to add `require(amount > 0, "Incorrect input data")` in function `notify()`.
