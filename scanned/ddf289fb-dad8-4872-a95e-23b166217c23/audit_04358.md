# ClaggSyncAdapter._removeLiquidity() incorrectly withdraws ETH when token is WETH 

## Summary
The bug report discusses an issue with the ClaggSyncAdapter and ClaggBaseAdapter contracts where the _removeLiquidity() function is not functioning properly. When the token address is not native ETH, the function calls a burnLiquiditySingle() function with the wrong withdrawMode, resulting in the incorrect withdrawal of assets. This causes problems when trying to withdraw from a SyncSwap pool with WETH as the token, as there will be no WETH available to transfer to the user. The recommended solution is to change the withdrawMode to 2 instead of 1 when the token is not ETH. This bug has been fixed in a recent commit and has been verified by Cantina Managed.

## Details
## Context
- ClaggSyncAdapter.sol#L220-L227
- ClaggBaseAdapter.sol#L92-L93

## Description
In `ClaggSyncAdapter._removeLiquidity()`, when the token address is not native ETH, `syncRouter.burnLiquiditySingle()` is called with `withdrawMode = 1`:

```solidity
// data = abi.encode(token, receiver, withdrawMode)
// withdrawMode: 0 = withdraw to vault, 1 = ETH, 2 = WETH
```

```solidity
bytes memory data;
if (token == address(ETH_TOKEN_SYSTEM_CONTRACT)) {
    data = abi.encode(address(_weth()), address(this), 1);
} else {
    data = abi.encode(token, address(this), 1);
}
```

`withdrawMode = 1` means that when the token is either ETH or WETH, assets withdrawn will always be transferred as native ETH. Therefore, when the token is WETH, `_removeLiquidity()` will withdraw ETH instead of WETH. This is incorrect as assets withdrawn from `_removeLiquidity()` should always be the token itself. As a result, when `withdraw()` is called for a SyncSwap pool with the token as WETH, `_transferToCaller()` will revert as the contract will not have any WETH to transfer to the user:

```solidity
// Transfer the tokens to the user
_transferToCaller(poolConfig.token, tokenAmount);
```

## Recommendation
In `ClaggSyncAdapter._removeLiquidity()`, when `token != ETH_TOKEN_SYSTEM_CONTRACT`, `withdrawMode = 2` should be used instead:

```solidity
bytes memory data;
if (token == address(ETH_TOKEN_SYSTEM_CONTRACT)) {
    data = abi.encode(address(_weth()), address(this), 1);
} else {
    // - data = abi.encode(token, address(this), 1);
    + data = abi.encode(token, address(this), 2);
}
```

This will withdraw WETH instead of ETH when `token = WETH`.

## Clave
Fixed in commit `cfb8b0d2`.

## Cantina Managed
Verified, the recommendation was implemented.
