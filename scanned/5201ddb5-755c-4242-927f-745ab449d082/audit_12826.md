# GaugeExtraRewarder Accounts for Excessive Rewards

## Summary
The `GaugeExtraRewarder` contract has a function called `updatePool` that is supposed to update the pool's information every time there is a change in the contract. However, the reward rate value keeps increasing even after the distribution period is over. This is because the variable `lastRewardTime` is not being set correctly. As a result, users are receiving rewards that were meant for others and it could cause issues with deposit and withdrawal operations. A solution has been proposed in a pull request and has been resolved.

## Details
The function [`updatePool`](https://github.com/ThenafiBNB/THENA-Contracts/blob/2bb84cf6bb429e9bac8ac8464f4546da855ef4ca/contracts/GaugeExtraRewarder.sol#L138) of the `GaugeExtraRewarder` contract is supposed to update the [pool's information](https://github.com/ThenafiBNB/THENA-Contracts/blob/2bb84cf6bb429e9bac8ac8464f4546da855ef4ca/contracts/GaugeExtraRewarder.sol#L40-42) upon every state-changing interaction with the contract. The reward-per-share value is then [updated by considering the time interval since the latest update](https://github.com/ThenafiBNB/THENA-Contracts/blob/2bb84cf6bb429e9bac8ac8464f4546da855ef4ca/contracts/GaugeExtraRewarder.sol#L144-L146). However, the reward rate value keeps increasing even if the distribution period is over. In this scenario, the variable [`lastRewardTime` is assigned incorrectly](https://github.com/ThenafiBNB/THENA-Contracts/blob/2bb84cf6bb429e9bac8ac8464f4546da855ef4ca/contracts/GaugeExtraRewarder.sol#L148), as it should be set equal to the end of the distribution period.


This improper accounting results in users accruing and taking rewards that were meant for other users, and in the worst case, causing the underlying gauge to revert during deposit and withdrawal operations.


Consider updating the implementation of the `GaugeExtraRewarder` contract so as to properly account for the reward amounts after the end of the distribution period.
