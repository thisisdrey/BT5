# Missing check for the rewards available

## Summary
The bug report is about a function called finishPrePurchasersMode() in a contract called ShyftStaking.sol. This function assumes that rewards have already been distributed, but there are no security checks to confirm this. As a result, the contract may calculate incorrect amounts to be returned and withdraw staked amounts from users. The recommendation is to add checks for the lastUpdateTime to be greater than 0, indicating that rewards have been started.

## Details
**Description**

ShyftStaking.sol, finishPrePurchasersMode()
The function relies on the fact that the contract already has rewards distributed. Though there
is no security checks confirming this fact. Thus the contract may calculate amounts to be
returned incorrectly and draw staked amounts from the users.

**Recommendation**:

Consider adding checks for the lastUpdateTime to be greater than 0 (so the rewards are
started).
