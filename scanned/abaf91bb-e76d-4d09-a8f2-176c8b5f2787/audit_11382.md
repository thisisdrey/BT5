# Constructor Parameter Validation

## Summary
This bug report is about a medium severity issue that has been resolved. The report mentions a contract called GenericERC20Token.sol and recommends performing a sanity check when setting the addresses of UniV2SwapRouter and UniV2Factory in its constructor. This is important because setting a zero address for these can cause problems with the contract's functionality. The client has fixed the issue by updating the vault contract to prevent a zero address from being used.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In contract GenericERC20Token.sol, is highly recommended to perform a sanity check when setting the UniV2SwapRouter and UniV2Factory in this contract's constructor.

**Recommendation**: 

Ensure that the addresses of UniV2SwapRouter and UniV2Factory are not set to zero (address(0)). A zero address can lead to contract functionality issues and unexpected behavior.

**Comment**: The client fixed the issue in commit: c1e1fba2e1958f6a41d9b573ec6133a684260892 by updating the vault contract to never allow a zero address to be passed.
