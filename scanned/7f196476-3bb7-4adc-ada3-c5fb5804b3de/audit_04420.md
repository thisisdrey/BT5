# Recommendations Regarding the Use of Libraries

## Summary
The ZK Email Noir code uses several libraries, one of which has known issues in its current version. The library, called `noir_base64`, has been updated in later versions to fix these issues. It is recommended to actively follow new releases and bugfixes in the libraries used, use a version without known issues, and review the code for compatibility and potential bugs. It is also advised to not solely rely on the library for security and correctness, and to add extra constraints as needed.

## Details
#### Description and Recommendation

The ZK Email Noir code makes use of several libraries, and while these libraries themselves were out of scope for this review, we happened to notice in the context of [issue 5.10](#several-leniencies-in-get_body_hash) that v0.2.2 of `noir_base64` – which is used in this project – has, at the time of this review, known issues. These have been fixed in later versions.

Noir is a relatively young language, and many Noir libraries are in active development and still change frequently as well as substantially. It is hard to give a good recommendation in such an environment, but we suggest the following guidelines:

1. Actively follow new releases and bugfixes in the libraries you’re using.
2. Generally use a version without known issues, or at least make sure that the issues don’t affect your project.
3. If possible, review the code yourself for e.g. compatibility, hidden or undocumented assumptions, and potentially even bugs.
4. Don’t be over-confident in the security/corrrectness/robustness of the library. For instance, if in doubt, add extra constraints yourself instead of relying on the library to properly validate the input.
