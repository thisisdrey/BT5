# Unbonding Period Bypass

## Summary
The report is about a bug in the stFLUO protocol, which allows users to initiate the redemption of stFLUO tokens for FLUO tokens with a specified unbonding period. The problem is that users are bypassing this unbonding period by calling withdraw, which is part of the implementation of ERC-4626. This can potentially affect the stability of the protocol and reduce liquidity, as users can withdraw their tokens at any time without following the unbonding period. To fix this issue, the withdraw function needs to be disabled, similar to how the redeem function of ERC-4626 has been disabled. This bug has been fixed in version fef1670.

## Details
## stFLUO Redemption Issue

In stFLUO, `initiateRedeem` allows users to initiate the redemption of stFLUO tokens for FLUO tokens with a specified unbonding period. The issue arises due to users bypassing this unbonding period. The bypass occurs by calling `withdraw`, which is included in the implementation of ERC-4626. This, in turn, may affect the stability of the protocol and result in reduced liquidity as users may withdraw anytime without the constraint of the unbonding period.

## Remediation

Ensure that `withdraw` is disabled similar to how `redeem` of ERC-4626 has been disabled.

### stFLUO.sol Solidity Code

```solidity
/// @dev Withdraw is disabled
function withdraw(uint256 assets, address receiver, address owner) public pure override returns (uint256) {
    assets;
    receiver;
    owner;
    revert("unable to withdraw, call initiateRedeem instead");
}
```

## Patch

Fixed in `fef1670`.
