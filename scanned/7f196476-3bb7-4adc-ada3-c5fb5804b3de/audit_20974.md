# Budget payment blocking

## Summary
This bug report is about the `pay` function of the `Budget.sol` contract, which is defined at a particular link and is used to send ETH to recipients in a loop. The problem is that the `transfer` method used in the loop has a gas limit of 2300, meaning that if any single recipient has a payable fallback method, it can block the whole `pay` function from executing. 

To solve this problem, the report recommends reworking the payment scheme to a claimable model. This means that instead of sending payments in a loop, the payments would be sent to a single address, and the recipients would have to claim the payments from that address. This would prevent any single recipient from blocking the whole `pay` function.

## Details
##### Description
In `pay` function of `Budget.sol` contract defined at https://github.com/bondappetit/bondappetit-protocol/blob/355180f0aca0b29d60d808f761052956b7a3a159/contracts/Budget.sol#L109 contract sends ETH to recipients in loop using `transfer` method. As we know `transfer` method limited by 2300 gas, so any single recipient with payable fallback method can block whole `pay` function execution

##### Recommendation
We recommend to rework payment scheme to claimable model.
