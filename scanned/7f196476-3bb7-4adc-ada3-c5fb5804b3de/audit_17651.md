# [M-09] Incorrect condition always bound to fail

## Summary
A bug has been reported in the GovernorBravoDelegate.sol file on line 135. The state function check on line 115 will always fail due to an initialization in the '_initiate' function. The '_initiate' function sets initialProposalId = proposalCount, meaning that if the proposal count is 5, the initialProposalId and proposalCount will both be set to 5. If the state function is called on proposal id 2, the require condition checks proposalCount >= proposalId && proposalId > initialProposalId, which is equivalent to 5>=2 && 5>5. Since 5>5 is not true, this always fails even though the proposal id is correct.

The recommended mitigation steps are to remove initialProposalId = proposalCount; in the '_initiate' function. Initially, it was thought that this bug would lead to an attack or loss of funds, but it was later found that this will only lead to the timelock admin not being able to be set, hence the severity was decreased to medium.

1.  The `_initiate` function sets initialProposalId = proposalCount;

2.  Now lets say proposal count was 5 so initialProposalId and proposalCount are both set to 5

3.  Now lets say state function is called on proposal id 2

4.  The require condition checks proposalCount >= proposalId && proposalId > initialProposalId

5.  This is equivalent to 5>=2 && 5>5, since 5>5 is not true this always fails even though proposal id 2 is correct

### Recommended Mitigation Steps

Remove initialProposalId = proposalCount; in the `_initiate` function.
