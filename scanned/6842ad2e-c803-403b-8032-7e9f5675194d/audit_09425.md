# The Ether can't be withdrawn from the NodeDelegator contract under certain conditions

## Summary
The bug report states that there is an issue with transferring Ether to and from the NDC (Node Delegator Contract) using certain functions. Specifically, if the `ETH_TOKEN` is not included in the list of supported assets, Ether can still be transferred to the NDC using the `transferETHToNodeDelegator` function, but it cannot be transferred back using the `transferBackToLRTDepositPool` function. The recommendation is to check for the presence of `ETH_TOKEN` in the supportedAssets list when executing the `sendETHFromDepositPoolToNDC` function.

## Details
##### Description
If `ETH_TOKEN` is not included in the list of supported assets, Ether can still be [transferred](https://github.com/Kelp-DAO/LRT-rsETH/blob/e75e9ef168a7b192abf76869977cd2ac8134849c/contracts/LRTDepositPool.sol#L352) to the `NDC` using the `transferETHToNodeDelegator` function. However, it [cannot be transferred back](https://github.com/Kelp-DAO/LRT-rsETH/blob/e75e9ef168a7b192abf76869977cd2ac8134849c/contracts/NodeDelegator.sol#L103) using the `transferBackToLRTDepositPool` function.
##### Recommendation
We recommend checking the presence of `ETH_TOKEN` in the supportedAssets list during the execution of the sendETHFromDepositPoolToNDC function.
