# NMKT-2 | Overwriting Previous Name

## Summary
This bug report concerns an issue with the `acceptBidForName` and `buyName` functions in a program. When these functions are used, the program does not check whether the NFT (non-fungible token) the name is being transferred to is already named. This means that when a user uses these functions, they may unexpectedly lose their old name upon transfer.

The recommendation for this bug is to consider if owned names should be overwritten upon transfer, and if so, to clearly document this behavior. This would ensure that users are aware of the potential to lose their old name and can make an informed decision about whether to use the functions.

## Details
**Description**

When a bid is accepted through `acceptBidForName` or an offered name is purchased through `buyName`, there is no check that the NFT the name shall be transferred to is already named. As a result, a user may unexpectedly lose their old name upon transfer.

**Recommendation**

Consider if owned names should be overwritten upon transfer. If necessary, clearly document such behavior.
