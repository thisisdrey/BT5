# No storage gap for upgradeable contracts

## Summary
This bug report is about a storage gap issue in base upgradable contracts. A storage gap is used to avoid storage collisions while using inheritance. It was observed that both ERC721ACollection.sol and ERC20Token.sol were missing the __gap variable which could lead to collision.

The recommendation is to add an appropriate storage gap at the end of upgradeable contracts, such as the code example provided. This will help to prevent storage slot collisions and ensure the contracts are working as intended.

## Details
**Description:**

1. `__gap` is used in base upgradable contracts to avoid storage clashes while using inheritance.
2. It was observed that both `ERC721ACollection.sol`and `ERC20Token.sol`were missing the `\_\_gap`variable which could lead to collision.

**Recommendations:** Recommend adding appropriate storage gap at the end of upgradeable contracts such as the below

\```

\```
