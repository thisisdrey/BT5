# Improper Sale Parameter Validation

## Summary
This report is about a bug in the `early-purchase` program. The program has instructions for initializing and updating a sale, but these instructions do not have proper validation for the input parameters. This means that users can make changes that may prevent purchases or harm other users. The bug has been partially fixed, but there are still some issues that need to be addressed. The recommendation is to add more thorough input validation to prevent these issues from happening in the future.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `f36395027ea6a0a443e0f5945cb74f0fad9243ed`. The client provided the following explanation:

> Parameter validation was added to the initialize_sale and update_sale instructions. Unit tests were added to test the validation.

**NOTE** Partially Fixed since "Uninitialized purchase and payment mint accounts" & "Too permissive max price feed age" are still not validated.

**File(s) affected:**`programs/early-purchase/src/instructions/initialize_sale.rs`, `programs/early-purchase/src/instructions/update_sale.rs`

**Description:** The `early-purchase` program includes `Sale` initialization and update instructions. These instructions lack parameter validation, allowing arbitrary updates of the sale properties that may prevent purchases or harm users. For example, it is possible to set:

*   A too early or too late start date.
*   Extremely short, extremely long or instantly ending sale.
*   A Sale with impossible conditions (e.g., `max_tokens_per_user`>`max_tokens_total`).
*   Zero or too small token price.
*   Too permissive max price feed age, allowing potential stale prices.
*   Arbitrary ID values that may have any off-chain implications (e.g., long).
*   Uninitialized purchase and payment mint accounts (`is_initialized` is not checked).
*   A zero `max_price_feed_age` which will limit users to only be able to purchase tokens in the same slot a the price feed has been updated which may cause purchase transactions to revert.

**Recommendation:** Consider implementing comprehensive input validation for sale initialization.
