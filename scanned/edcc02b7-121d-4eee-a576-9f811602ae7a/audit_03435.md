# Improper Vector Initialization

## Summary
The bug report states that there is an issue with the InitCompactUnlock instruction in the programming language Swift. This instruction is supposed to initialize a vector called "items" within the CompactUnlockState account. However, instead of creating a vector with the correct length, it is only initialized with a single item containing the length value. This causes an error when trying to access items in the vector at positions 1 to 3, as the vector only has a length of 1. The report suggests that the issue lies in the Rust processor file called "init_compact_unlock.rs" and provides a code snippet as an example. It also mentions that the bug has been fixed in a specific patch. The suggested solution is to initialize the vector with the correct size equal to the computed length.

## Details
## InitCompactUnlock Instruction Issue

The `InitCompactUnlock` instruction in Swift incorrectly initializes the items vector within the `CompactUnlockState` account. Rather than creating a vector of the computed length, it is initialized with a single item containing the computed length value, resulting in an index out-of-range error when accessing `compact_unlock.items[1..3]` as the vector length remains 1.

```rust
>_ processor/compact_unlock/init_compact_unlock.rs
pub fn handle_init_compact_unlock(ctx: Context<InitCompactUnlock>, items_count: u16) -> Result<()> {
    [...]
    compact_unlock.items = vec![CompactUnlockState::compute_items_data_size(items_count) as u8];
    compact_unlock.items[0] = SwiftVaaAction::UnlockBatchCompact as u8;
    compact_unlock.items[1..3].copy_from_slice(&items_count.to_be_bytes());
    Ok(())
}
```

## Remediation

Initialize the vector with the required size equal to the computed length.

## Patch

Resolved in commit `6af103f`.
