# TRST-M-6 _vote() doesn’t check the pool a user is voting for has a valid gauge

## Summary
The bug report describes an issue in the function `_vote()` which allows veSatin holders to vote to which pool the weekly emissions should be redirected to. The bug was that the function did not check if the pool a user is voting for has a gauge associated, which could lead to emissions being lost. The recommended mitigation was to ensure that the pool a user is trying to vote for has a valid gauge by checking if isGauge[pool] is true. The team response was that the issue was fixed and the mitigation review confirmed that the issue was resolved and that the function skips the vote for pools that don’t have a valid associated gauge.

## Details
**Description:**

should be redirected to but it doesn’t check the pool a user is voting for has a gauge 
associated, which could lead to emissions being lost.

checking if isGauge[pool] it’s true:

Fixed
