# [H-05] Redeemer.redeem() for Element withdraws PT to wrong address.

## Summary
A bug has been reported for the Redeemer.redeem() function in the code-423n4/2022-06-illuminate repository. It causes external principal tokens from Lender.sol to be transferred to the wrong address, resulting in the loss of those tokens. The bug was found by multiple users, including auditor0517, 0x52, cccz, datapunk, kenzo, and pashov.

The ReadMe.md file in the repository states that Redeemer should transfer the external principal tokens to Redeemer.sol, but instead it transfers them to the "marketPlace". To fix this issue, the line of code IElementToken(principal).withdrawPrincipal(amount, marketPlace); should be modified to IElementToken(principal).withdrawPrincipal(amount, address(this));. This was confirmed by sourabhmarathe (Illuminate).

This bug can cause a loss of principal tokens, so it is important to fix it as soon as possible. If the code is not modified, users may lose their tokens.

But it transfers to the "marketPlace" and it would lose the PT.

### Recommended Mitigation Steps

Modify [IElementToken(principal).withdrawPrincipal(amount, marketPlace);](https://github.com/code-423n4/2022-06-illuminate/blob/92cbb0724e594ce025d6b6ed050d3548a38c264b/redeemer/Redeemer.sol#L187) like this.

    IElementToken(principal).withdrawPrincipal(amount, address(this));
