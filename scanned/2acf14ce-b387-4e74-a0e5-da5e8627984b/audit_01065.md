# VaultStrategy cannot be initialized because default feeConfig is used for validation

## Summary
This bug report is classified as a "Medium Risk" and affects a specific section of code in the SuperVaultStrategy.sol file. The issue is that the code is checking for a variable called `feeConfig` instead of `feeConfig_`, which causes the strategy initialization to fail. The recommendation is to update the code to check for `feeConfig_` instead, which will fix the issue.

## Details
## Medium Risk Report

**Severity:** Medium Risk  
**Context:** SuperVaultStrategy.sol#L85-L89  
**Description:** The code checks `feeConfig` instead of `feeConfig_`, which is always false and blocks the strategy initialization.  
**Recommendation:** Check `feeConfig_` instead of `feeConfig` when the strategy is initialized.
