# Possible loss of funds due to `destroyEscrow()` method

## Summary
The bug report discusses a problem with the `destroyEscrow()` method, which is used to delete escrow details. The issue is that the method does not take into account any tokens that may have been deposited by the seller or buyer. This means that if the owner calls the method for certain escrows, the tokens in the wallet address may become stuck. The bug has been resolved, but the recommendation is to check if the seller and/or buyer funds are locked before deleting an escrow to avoid this issue in the future.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

Method `destroyEscrow()` deletes the escrow details without considering the tokens passed to the wallet address if a seller or buyer has deposited tokens.
If owner calls the `destroyEscrow()` method for such escrows then there is possibility of tokens in wallet address to be stuck.

**Recommendation**: 

Consider checking if seller and/or buyer funds are locked before deleting an escrow and proceed accordingly.
