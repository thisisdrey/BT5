# Issuer can process commitments even if it is revokable 

## Summary
The bug report is about a lack of restriction in a function called "processCommitment" that allows the issuer to process revokable commitments. This means that even after a certain time period has passed, the issuer can still process commitments that were supposed to be revoked. This can lead to financial loss for the user. The report recommends adding a restriction to prevent this from happening.

## Description
Currently, there is no restriction in the `processCommitment` function to prevent the issuer from processing revokable commitments. The commitment process guarantees the user that commitments will not be processed after `commitmentHalfLife` is passed. However, the issuer can still process revokable commitments due to the lack of restriction in the `processCommitment` function.

1. Let's say a user made a commitment to register a domain.
2. The issuer didn’t process the commitment within `commitmentHalfLife` (90 mins).
3. The user might think that the chosen domain name is not available and makes a commitment for a different domain without revoking the first commitment.
4. Now the issuer processes the first commitment even after it is revokable, and then processes the new commitment too.

## Impact
User may experience financial loss.

## Recommendation
Add a restriction to prevent the issuer from processing revokable commitments.
