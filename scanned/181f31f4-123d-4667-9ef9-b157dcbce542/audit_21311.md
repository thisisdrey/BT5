# Undocumented privileged operations

## Summary
This bug report is about an Access Controls issue in the app-vault.teal and minter.teal.tmpl systems. The difficulty level is rated as low. The issue is that several admin operations are not documented, which can lead to users being misled about the system's centralization. The privileged operations that are documented include Admin setGlobalStatus, Admin setAccountStatus, Admin setMintFee, Admin setBurnFee, and Admin setCreationFee. However, there are also additional critical operations that can be performed by the admin, such as updating the app-vault program, deleting the app-vault program, and updating the mint/admin account. 

The exploit scenario is that Bob creates a vault, thinking that its funds are safe, but a malicious admin, Eve, changes the app-vault approval program and steals Bob's assets. The recommendation is to document all the privileged operations so that users are aware of the risks when depositing funds. In the long term, it is recommended to document all the actors and their associated privileges in order to properly evaluate the system.

## Access Controls

**Type:** Access Controls  
**Target:** app-vault.teal, minter.teal.tmpl  

**Difficulty:** Low  

## Description

Several admin operations are not documented. This lack of transparency can mislead users regarding the system’s centralization. The privileged operations that are documented include:

### Admin setGlobalStatus
The admin can enable or disable any vault at any time.  
- **Tx0:**
  - **Sender:** Admin
  - **arg0:** integer: new status (0 or 1)
  - **Application Call tx**

### Admin setAccountStatus
The admin can enable or disable any vault at any time.  
- **Tx0:**
  - **Sender:** Admin
  - **acc0:** User Address
  - **arg0:** integer: new status (0 or 1)
  - **Application Call tx**

### Admin setMintFee
Set the percent of paid in ALGOs on each mintwALGOs operation.  
- **Tx0:**
  - **Sender:** Admin
  - **arg0:** integer: new fee (0 to 5000 which means 0%-50%)
  - **Application Call tx**

### Admin setBurnFee
Set the percent of ALGOs reserved for the admin on each burnwALGOs operation.  
- **Tx0:**
  - **Sender:** Admin
  - **arg0:** integer: new fee (0 to 5000 which means 0%-50%)
  - **Application Call tx**

### Admin setCreationFee
Set the fee in ALGOs that is required to send to Admin to opt in to the App.  
- **Tx0:**
  - **Sender:** Admin
  - **arg0:** integer: new fee in microALGOs
  - **Application Call tx**

However, the admin can perform many additional critical operations, including:
- Updating the app-vault program.
- Deleting the app-vault program.
- Updating the mint/admin account.

## Exploit Scenario

Bob creates a vault. Bob thinks that its funds are safe, and that no one can steal its assets. Eve, a malicious admin of the app-vault, changes the app-vault approval program and steals Bob’s assets.

## Recommendation

Short term, document all the privileged operations that can be performed by the admin to ensure that users are aware of the underlying risks when depositing funds.

Long term, identify and document all the actors and their associated privileges. Documenting the system’s actors and their privileges is required to properly evaluate the system, identify and document all the actors, and their associated privileges.
