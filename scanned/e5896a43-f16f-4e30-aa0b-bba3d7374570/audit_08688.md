# 0 Slippage Protection

## Summary
A bug has been found in the Pancake Router when adding Liquidity. This bug has a medium severity and has been resolved. When adding Liquidity, the `amountAMin` and `amountBMin` parameters are being passed as 0, which means there is no protection against slippage impacts. To fix this, these values should be assigned to a non-zero safe value to ensure the slippage is within an accepted range. The recommendation is to assign these values to non-zero values.

## Details
**Severity**: Medium

**Status**: Resolved

**description**

When Liquidity is added to the pancake router , the `amountAMin` and `amountBMin` params
are being passed as 0 (L231 and L232 in TokenSaleUSDB , same for TokenSaleETH) which
essentially means 0 protection against slippage impacts , these values should be assigned
to a non-zero safe value to ensure the slippage is within accepted range.

**Recommendation:**

Assign the values to non-zero values.
