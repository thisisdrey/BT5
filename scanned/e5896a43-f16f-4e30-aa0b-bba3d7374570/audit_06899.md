# [C-01] Incorrect variable used when calculating supply

## Summary
The report states that there is a bug in the `_calculateSupply` function in the pool contract. The severity and impact of the bug are high, and the likelihood of it occurring is also high. The bug is caused by incorrect logic when calculating the intermediate `_r` variable. The comment in the code is correct, but the implementation is incorrect. The recommended solution is to fix the line of code to use `_r` instead of `s`. The report also provides a link to the yearn contract where the same bug can be seen.

## Details
**Severity**

**Impact:** High

**Likelihood:** High

**Description**

When calculating the intermediate `_r` in the `_calculateSupply` function in the pool contract, the function uses the following logic.

```solidity
_r = FixedPointMathLib.rawDiv(
    FixedPointMathLib.rawMul(_s, _sp), _s);
    // _r*_sp/_s
```

The comment is correct, but the implementation is incorrect. It should use `_r` instead of `s`.

This can also be seen in the yearn contract here:
[link](https://github.com/yearn/yETH/blob/8d831fd6b4de9f004d419f035cd2806dc8d5cf7e/contracts/Pool.vy#L1366)

**Recommendations**

Fix the line to:

```solidity
_r = FixedPointMathLib.rawDiv(
    FixedPointMathLib.rawMul(_r, _sp), _s);
    // _r*_sp/_s
```
