# Incorrect Gas Calculation

## Summary
The bug report states that the function "getRemainingGas" in the code incorrectly multiplies the remaining SEI gas (seiGasRemaining) by the gas normalizer (gasMultiplier). The correct formula should be evmgas = seigas/gasMultiplier. The report suggests replacing "Mul" with "Quo" in the gas calculation within the function. The bug has been fixed in version 8296390.

## Details
## Documentation on `getRemainingGas` Function

The function `getRemainingGas` incorrectly multiplies the remaining SEI gas (`seiGasRemaining`) by the gas normalizer (`gasMultiplier`), where the correct formula should be:

```
evmGas = seiGas / gasMultiplier
```

## Code Snippet

```go
// precompiles/wasmd/wasmd.go
func (p Precompile) getRemainingGas(ctx sdk.Context) uint64 {
    gasMultiplier := p.evmKeeper.GetPriorityNormalizer(ctx)
    seiGasRemaining := ctx.GasMeter().Limit() - ctx.GasMeter().GasConsumedToLimit()
    return new(big.Int).Mul(new(big.Int).SetUint64(seiGasRemaining),
        gasMultiplier.RoundInt().BigInt()).Uint64()
}
```

## Remediation

Replace `Mul` with `Quo` in the gas calculation within `getRemainingGas`.

## Patch

Fixed in `8296390`.
