# SOME FUNCTIONS RECEIVE MULTIPLE NATIVE COINS INSTEAD OF ONE

## Summary
The bug report describes a problem in two contracts, krp-basset-convert and swap-extension, where certain functions can receive multiple native coins. This can result in a double payment if users send the wrong type of coin. The affected functions are execute_convert_to_basset in krp-basset-convert and swap_denom in swap-extension. The code location of the issue is provided, along with a BVSS rating of 5.0, indicating a moderate severity. The recommendation states that the Kryptonite team has solved the issue in two commits.

## Details
##### Description

Some functions in the **krp-basset-convert/krp\_basset\_converter** and **swap-extension/swap\_sparrow** contracts can receive multiple native coins, which would allow a double payment if users mistakenly send native coins different from the target one. The affected functions are the following:

* **krp-basset-convert/krp\_basset\_converter:** `execute_convert_to_basset` function
* **swap-extension/swap\_sparrow:** `swap_denom` function

Code Location
-------------

Code fragment of the `execute_convert_to_basset` function in the **krp-basset-convert/krp\_basset\_converter** contract:

#### krp-basset-convert/contracts/krp\_basset\_converter/src/contract.rs

```
let coin_denom = config.native_denom.unwrap();

let coin = info
    .funds
    .iter()
    .find(|x| x.denom == coin_denom && x.amount > Uint128::zero())
    .ok_or_else(|| {
      StdError::generic_err(format!("No {} assets are provided to deposit", coin_denom));
    });

let denom_decimals = 6u8;

```
