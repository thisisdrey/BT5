# Fetching Code Directly From Repository

## Summary
The openbrush-contracts repository only allows access to its public crates through direct fetching from the repository, which can lead to potential risks and security breaches. It is recommended to use a crates registry, such as crates.io, which offers a review and verification process for crates and ensures more reliable code access. The Brushfam team has acknowledged this issue and plans to resolve it by uploading the crates to a registry in a new release. This was not possible before due to a dependency issue, but they plan to address it in future versions. The issue can be tracked at https://github.com/Brushfam/openbrush-contracts/issues/154.

## Details
The [openbrush-contracts](https://github.com/Brushfam/openbrush-contracts/tree/f5ce8f73e8b7f3471586b9d6f7900970bf265a60) repository makes its public crates available only through direct fetching from the repository. This introduces potential risks of instability and security breaches. Using a crates registry, where each crate undergoes a review and verification process, mitigates these concerns and ensures more reliable code access.


Consider using a crates registry for all the public crates within the codebase. [Crates.io](https://crates.io/), for example, offers these benefits while also providing continuous access to crates, protecting against the pitfalls of repository deletions.


***Update:** Acknowledged, will resolve. The Brushfam expressed that they will resolve the issue:*



> *We will be uploaded to registry in a new release, could not be uploaded before because OpenBrush was using `pallet-assets-chain-extension`, which is a git dependency. In new versions we are planning to move `psp22_pallet` in another repo and publish OpenBrush. Issue: https://github.com/Brushfam/openbrush-contracts/issues/154.*
