# It is necessary to check the correctness of the variable value

## Summary
A bug report has been identified in the ExperiPie code. The bug is related to the `burn()`, `approve()`, `transferFrom()` and `_transfer ()` functions. These functions are responsible for decreasing the value of the amount of tokens for the `_from` wallet. In order to prevent the value from decreasing to zero, it is recommended to add a check of the current value of the `_from` variable for zero. This check should be added to all four functions in order to ensure that the value of the tokens is not decreased to zero. The code for the check should be as follows: `require(_from != address(0), "burn from the zero address");`, `require(_spender != address(0), "approve to the zero address");`, `require(_from != address(0), "transfer from the zero address");`, and `require(_from != address(0), "transfer from the zero address");`. This check should be added to the line https://github.com/pie-dao/ExperiPie/blob/facf3c246d9c43f5b1e0bad7dc2b0a9a2a2393c5/contracts/facets/ERC20/LibERC20.sol#L25, https://github.com/pie-dao/ExperiPie/blob/facf3c246d9c43f5b1e0bad7dc2b0a9a2a2393c5/contracts/facets/ERC20/ERC20Facet.sol#L57, https://github.com/pie-dao/ExperiPie/blob/facf3c246d9c43f5b1e0bad7dc2b0a9a2a2393c5/contracts/facets/ERC20/ERC20Facet.sol#L79 and https://github.com/pie-dao/ExperiPie/blob/facf3c246d9c43f5b1e0bad7dc2b0a9a2a2393c5/contracts/facets/ERC20/ERC20Facet.sol#L115. This is to ensure that the value of the tokens is not decreased to zero.
