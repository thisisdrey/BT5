# Funding rate update calculations drop snapshots and do not compute a full TWAP 

## Summary
The `tuples()` method in the `mod.rs` file on line 174 groups items into tuples, but it drops items to make them fit. This means that if there is an odd number of items, the last one will not be included in the tuple. This causes incorrect calculations in the TWAP calculation code. The recommendation is to implement an iterator to create a list of all adjacent pairs in the list. This issue has been fixed in PR 944 and has also been addressed in Cantina Managed by switching to using `tuple_windows` from `itertools`.

## Details
## Context: mod.rs#L174

## Description
The `tuples()` method groups items into tuples but drops items to make them fit. For example:

```rust
use itertools::Itertools;
let mut v: Vec<(i32, i32)> = Vec::new();
for x in (1..6).tuples::<(_, _)>() {
    v.push(x);
}
assert_eq!(v, vec![(1, 2), (3, 4)]);
```

Note that the range `1..6` contains the element `5`, but the grouped vector does not include it in a tuple. Hence, the latest snapshot (the newly added one) is not included in the set of tuples if the number of snapshots is odd. 

The way the following code uses `tuples()` causes incorrect TWAP calculation:

```rust
let duration_times_funding_rate = snapshots.iter().tuples::<(_, _)>().fold(
    Lifted::zero(),
    |acc, (previous, current)| {
        acc + Lifted::from(current.at - previous.at) * Lifted::from(current.funding_rate)
    },
);
let result = duration_times_funding_rate / Lifted::from(now - snapshots.front().map(|x| x.at).unwrap_or_default());
```

Worse, the calculation does not calculate a weighted average. It uses only half of the snapshot price points (those bound to the current variable in the fold function). It only considers the time difference between grouped pairs, ignoring the other time differences.

If we have five snapshots, `s1` through `s5`, the calculation of the new funding rate becomes:

```
(s2.at − s1.at)(s2.funding_rate) + (s4.at − s3.at)(s4.funding_rate)
s5.at − s1.at
```

The intended calculation is:

```
∑_{i=2}^{5} (si.at − si−1.at)(si.funding_rate)
s5.at − s1.at
```

## Recommendation
Correct the algorithm by implementing an iterator to perform what you intend `tuples()` to do: create the list of all adjacent pairs in the list. There are libraries available, but implementing your own is preferable for such a simple iterator.
