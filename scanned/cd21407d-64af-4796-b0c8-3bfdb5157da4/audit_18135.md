# [M-07] Inconsistency between constructor and setting method for `slippageTolerance`

## Summary
This bug report is about a vulnerability in the code of the StrategySwapper.sol contract. The vulnerability is that the setSlippageTolerance() method has certain requirements to set slippageTolerance, but these requirements are not present in the constructor. The impact of this vulnerability is that the constructor could be used to set slippageTolerance without meeting the requirements. The recommended mitigation step to fix this vulnerability is to add the corresponding validations to the constructor.
