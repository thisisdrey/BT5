# Potential front running attack or losing of allowance

## Summary
This bug report is about a vulnerability in the '_approve' method of the 1inch-token repository on GitHub. The '_approve' method is used to increase the allowance of a receiver, however, it has two potential problems. Firstly, if a signer wants to increase the allowance from 'A' to 'B', a receiver may withdraw 'A+B' using a front-running attack. Secondly, if a signer wants to send 'A' and 'B', but a receiver forgot to withdraw 'A', the receiver will lose the ability to withdraw 'A'. 

In order to fix this vulnerability, the bug report recommends adding 'permitIncrease' and 'permitDecrease' methods and using them instead of the 'permit' method. This would ensure that the allowance is updated correctly and the receiver would not be able to withdraw more than they are supposed to.

## Details
##### Description
At the line https://github.com/1inch-exchange/1inch-token/blob/99fd056f91005ca521a02a005f7bcd8f77e06afc/contracts/ERC20Permit.sol#L53 `_approve` method replaces the allowance, so there are two potential problems here:
1. If a signer wants to increase the allowance from `A` to `B`, a receiver may withdraw `A+B` using the front-running attack.
2. If a signer wants to send `A` and `B`, but a receiver forgot to withdraw `A`, the receiver will lose ability to withdraw `A`.

##### Recommendation
We suggest to add `permitIncrease`, `permitDecrease` methods and use it instead of the `permit`.
