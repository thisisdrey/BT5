# Undocumented assembly blocks

## Summary
The UserWallet contract includes two assembly blocks which are undocumented. This may not pose a security risk, however, it is important to document each line of assembly in order to improve the readability and understandability of the code. Furthermore, the use of assembly can reduce the safety features of Solidity, making it more error-prone. It is advised to implement thorough tests to cover all potential use cases of the functions to ensure they behave as expected.

## Details
The `UserWallet` contract includes two undocumented assembly blocks. [The first one](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/UserWallet.sol#L90-L93) is in the `note` modifier and [the second one](https://github.com/InstaDApp/contract-v2/blob/4863c0c4156af7ded9cdb38b66e5f5e527c4a6d0/contracts/UserWallet.sol#L151-L165) is in the `excecute` function.


While this does not pose a security risk *per se*, these are a critical part of the system and should be better documented. For assembly blocks, we highly recommend that *every line* of assembly be explained using code comments. This helps improve the readability and understandability of these critical sections of code.


Also note that the use of assembly discards several important safety features of Solidity, which can render the code less safe and more error-prone. Consider implementing thorough tests to cover all potential use cases of these functions to ensure they behave as expected.
