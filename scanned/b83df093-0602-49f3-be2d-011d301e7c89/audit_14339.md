# M-17: Slippage is not respected if PnL swap associated with a decrease order fails

## Summary
This bug report is about an issue in the GMX-Synthetics contracts. The issue is that when a user decreases a position, they have an option to convert the PnL token to any other token via a swap. However, if the swap fails, the tokens are sent directly to the user, without checking for slippage and they may receive fewer tokens than they expect. This could happen if the market is temporarily disabled, or the swap impact is too large, or a token being swapped through is paused. The bug was found by IllIllI and the tool used was Manual Review. The recommended fix is to calculate whether the USD value of the outputToken is equivalent to the minOutputAmount expected by the order, and revert if it's less than required. The fix was implemented in a pull request by xvi10.

## Summary

After a position has been decreased, the user has an option to convert the PnL token to any other token via a swap.


## Vulnerability Detail

If the swap fails (which is where the slippage is checked), the tokens are sent directly to the user, without checking whether there was slippage


## Impact

A user will get back fewer tokens than they expect, if there was a large price impact, and the subsequent swap fails due to e.g. the market being temporarily disabled, or the swap impact being too large, or a token being swapped through is paused
