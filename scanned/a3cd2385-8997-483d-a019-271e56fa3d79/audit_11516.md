# Insufficient Block Size Check

## Summary
The MEV lane in the block-sdk does not properly account for bundled transactions, resulting in an underestimation of the total block size. This can lead to blocks being constructed that exceed the maximum block size limit. The issue has been fixed in the latest patch and bundled transaction sizes will now be properly tracked to avoid this problem.

## Details
## MEV Lane and Block Size Limits

When checking transaction sizes against block size limits, the MEV lane does not consider that bundled transactions will be unpacked in `prepareProposal` and included twice in the block proposal. Thus, it underestimates the total block size and may result in the construction of blocks that exceed block size limits.
