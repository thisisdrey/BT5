# Re-entrancy Vulnerability in ERC777 Token

## Summary
A re-entrancy attack is a type of exploit in which a malicious contract can repeatedly call a function within a single transaction, leading to an unpredictable state of the contract and potential theft of tokens. The bug report describes a potential re-entrancy attack in the CoinlendCredits contract, due to the use of an external function call (payToken.transferFrom) before updating the state of the contract (\_mint function). The recommendation is to consider implementing re-entrancy guard, and the status of the bug is resolved.

## Details
**Description:** 

The buyCredits function in the CoinlendCredits contract may lead to a potential re-entrancy attack due to the use of an external function call (payToken.transferFrom) before updating the state of the contract (\_mint function).

In a re-entrancy attack, a malicious contract can repeatedly call the buyCredits function within a single transaction, leading to an unpredictable state of the contract and potential theft of tokens.

**Recommendations:**

Consider implementing re-entrancy guard. Status: Resolved.
