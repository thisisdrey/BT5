# cronJob checkTokens Might Flood User Notifications ✓ Fixed

## Summary
This bug report is about a feature in the snap that sends multiple notifications to the user when their tokens are not backed up. This can be overwhelming for the user and may cause them to miss important notifications. The bug has been resolved by making the snap send a single notification instead of multiple ones. It is also recommended to limit the number of notifications when the list of unbacked tokens is too long. This will improve the user experience.

## Details
#### Resolution



Addressed by [tezoroproject/metamask-snap#44](https://github.com/tezoroproject/metamask-snap/pull/44). The snap now sends a single notification.


#### Description


The snap includes a cron job named `checkToken` that activates every 15 days to verify which user tokens are backed up and which are not. For each token identified as not backed up (listed in `tokenList`), the snap issues a notification to the user. If the list of unbacked tokens is extensive, the user will receive many notifications, potentially undermining the effectiveness of these alerts or causing the user to overlook other important notifications. To alleviate this concern, it is recommended to aggregate these notifications. Issuing a single notification, or capping the number of notifications when the size of `tokenList` surpasses a specific threshold (e.g., 5), could improve the user experience.

#### Recommendation
