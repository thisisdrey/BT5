# Inconsistency In Randomness Generation

## Summary
The PREVRANDAO instruction is used to generate random values based on previous block timestamps. However, the values it produces are biased and often display patterns, such as a constant upper byte and yielding the same output after a modulo-two operation. This predictable behavior compromises the randomness of the generated values, allowing attackers to exploit contracts that rely on PREVRANDAO for randomness. To fix this issue, the instruction should be modified to generate uniformly random values and this change should be clearly documented, similar to other EVM compatible chains. ## Details
## PREVRANDAO Instruction

The **PREVRANDAO** instruction produces random values using previous block timestamps. However, the generated values display a significant bias, frequently demonstrating patterns like a constant upper bound and yielding the same output after a modulo-two operation. This predictable behavior compromises the randomness of the generated values, allowing attackers to exploit contracts that depend on **PREVRANDAO** for randomness.

## Remediation

Modify the **PREVRANDAO** instruction to be uniformly random and clearly document this difference, similar to other EVM compatible chains.
