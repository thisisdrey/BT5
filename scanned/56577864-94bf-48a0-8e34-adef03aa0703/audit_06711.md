# Multiple typos make GMIZap impossible to compile

## Summary
This bug report is about an issue in the `GMIZap` file that is preventing it from being used. The compilation step keeps failing due to two specific errors. The first error is related to a missing argument in a type conversion, and the second error is caused by a mismatch in the number of arguments in a return statement and the returns declaration. The BVSS score for this issue is 5.0, which means it has a medium impact and does not require any special privileges to exploit. The recommended solution is to fix the parenthesis placement in the first error and add a `returns (uint256)` statement in the function definition for the second error. The Gloop Finance team has already solved this issue by making the recommended changes, and the remediation hash can be found on their GitHub page.

## Details
##### Description

The following errors prevent the `GMIZap` file from being used, as the compilation step always reverts:

```
Error (2558): Exactly one argument expected for explicit type conversion.
  --> src/GMIZap.sol:58:25:
   |
58 |         require(name != bytes32("", "market not found"));
   |                         ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Error (8863): Different number of arguments in return statement than in returns declaration.
  --> src/GMIZap.sol:69:9:
   |
69 |         return gmIndex.deposit(name, _recipient, _amount, _minAmountOut);
   |         ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
```

##### BVSS

[AO:A/AC:L/AX:L/C:N/I:N/A:M/D:N/Y:N/R:N/S:U (5.0)](/bvss?q=AO:A/AC:L/AX:L/C:N/I:N/A:M/D:N/Y:N/R:N/S:U)

##### Recommendation

For the first error, move the parenthesis to its correct position and for the second one, add a `returns (uint256)` statement in the function definition.
