# `rejectDepositor` can be front-run

## Summary
The bug report states that there is an issue with the `rejectDepositor` function in the `TokenLexscrow` and `EthLexscrow` contracts. This function can be manipulated by the `buyer` before the `seller` transaction is completed, causing the transaction to fail. The report suggests that the `updateBuyer` function should be paused by the seller in order to prevent this issue from occurring.

## Details
##### Description
`rejectDepositor` can be front-run by the `buyer`, so the `seller` tx will revert: https://github.com/MetaLex-Tech/LeXscrow/blob/94ca277528bb25b8421dc127941c18915144eb29/src/TokenLexscrow.sol#L455-L473, https://github.com/MetaLex-Tech/LeXscrow/blob/94ca277528bb25b8421dc127941c18915144eb29/src/EthLexscrow.sol#L311-L329.

##### Recommendation
We recommend allowing the seller to pause the `updateBuyer` function for such cases.
