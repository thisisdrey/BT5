# User should be both `keeper` and `owner` to be able to set `to_be_liquidated = true`

## Summary
This bug report describes an issue in the `request_withdraw` function in a file called `lib.rs` in a specific folder. The issue occurs when the value of "dtv" is equal to or greater than a certain limit, and the user is checked to be both the strategy keeper and the owner of the position. The bug has been resolved and the recommendation is to allow the same account to be both the owner and the keeper in this scenario.

## Details
**Severity**: Medium

**Status**: Resolved

**File**: strategy/programs/vaultka/src/lib.rs: 902 and 913

**Description**

In the `request_withdraw` function in the case when "dtv" is greater or equals
to the "strategy.dtv_limit", there is a check for the `accounts.user` to be the strategy keeper.
and right after that, on the line 913 it checks the same user to be an owner of the position

**Recommendation**

Make sure the owner and the keeper could be the same account for this case to work
