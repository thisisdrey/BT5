# Incorrect Gas Estimation

## Summary
This bug report is about a vulnerability in the send system that occurs during the gas estimation process. This can lead to incorrect gas limits being set for transactions, which can cause problems when submitting them to Ethereum. The issue is caused by a byte being appended to the beginning of the transaction data and a gas limit being added, but since the gas limit starts at zero, this may not accurately reflect the gas required for the transaction. The suggested solution is to move the gas estimation to occur just before the transaction creation, which will ensure that the gas limit is determined based on the intrinsic gas calculation. This issue has been resolved in version 1d92897.

## Details
## Vulnerability in Send Transaction

The vulnerability in `send` stems from the gas estimation process, particularly when handling transactions without sending data to Celestia (`!isCelestia`). Initially, the gas limit (`candidate.GasLimit`) is set to zero before adjustments occur. If `!isCelestia` is true, a byte is appended to the beginning of the transaction data, and `params.TxDataNonZeroGasEIP2028` is added to the gas limit. However, since the gas limit starts at zero, this addition may not accurately reflect the gas required for submitting the transaction to Ethereum.

> _op-service/txmgr/txmgr.go_  
> // send performs the actual transaction creation and sending.

```go

            defer cancel()
            if frameRefData, err := m.payForBlob(ctx, candidate.TxData); err == nil {
                intrinsicGas, err := core.IntrinsicGas(frameRefData, nil, false, true, true, false)
                [...]
            }
        }
        if !isCelestia {
            candidate.TxData = append([]byte{1}, candidate.TxData...)
            candidate.GasLimit += params.TxDataNonZeroGasEIP2028
        }
    }
    [...]
}
```

## Remediation

Move the gas estimation utilizing `core.IntrinsicGas` to occur just before the transaction creation (`craftTx`). This adjustment ensures the gas limit is determined based on the intrinsic gas calculation, which considers parameters such as transaction data and other relevant factors.

## Patch

Resolved in 1d92897.
