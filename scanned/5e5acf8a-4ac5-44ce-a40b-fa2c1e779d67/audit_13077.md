# Reducing the quantity of a policyholder results in an increase instead of a decrease in totalQuantity

## Summary
This bug report is about the LlamaPolicy.sol#L388, a file which allows policyholders to approve or disapprove actions. Each policyholder has a quantity which represents their approval casting power. The setRoleHolder function in the LlamaPolicy is used to update the quantity of individual policyholders.

The bug is that the setRoleHolder method is not handling the decrease of quantity correctly for the totalQuantity. The totalQuantity is the sum of the quantities of the individual policyholders for a specific role. The difference between the initialQuantity and the new quantity is calculated as follows: uint128 quantityDiff = initialQuantity > quantity ? initialQuantity - quantity : quantity - initialQuantity;. However, the quantityDiff is always added instead of being subtracted when the quantity is reduced. This results in an incorrect tracking of the totalQuantity.

The recommendation for resolving the bug is to handle the increase and decrease cases of quantity in separate if-conditions or using an intvariable for calculating the difference. For the decrease case, the newTotalQuantity should be calculated as currentRoleSupply.totalQuantity - quantityDiff. For the increase case, the newTotalQuantity should be calculated as currentRoleSupply.totalQuantity + quantityDiff.

The bug has been resolved by commit eb90d2 and PR 291.

## Details
## Severity: High Risk

## Context
LlamaPolicy.sol#L388

## Description
In Llama, policyholders can approve or disapprove actions. Each policyholder has a quantity that represents their approval casting power. It is possible to update the quantity of an individual policyholder with the `setRoleHolder` function in the LlamaPolicy. 

The `_setRoleHolder` method is not handling the decrease of quantity correctly for the `totalQuantity`. The `totalQuantity` describes the sum of the quantities of the individual policyholders for a specific role. 

In the case of a quantity change, the difference is calculated as follows:

```solidity
uint128 quantityDiff = initialQuantity > quantity ? initialQuantity - quantity : quantity - initialQuantity;
```

However, the `quantityDiff` is always added instead of being subtracted when the quantity is reduced. This results in an incorrect tracking of the `totalQuantity`. Adding the `quantityDiff` should only happen in the increase case.

See: LlamaPolicy.sol#L388

```solidity
// case: willHaveRole=true, hadRoleQuantity=true
newTotalQuantity = currentRoleSupply.totalQuantity + quantityDiff;
```

## Recommendation
The increase and decrease cases of quantity should be handled in separate `if` conditions or using an `int` variable for calculating the difference.

**Decrease:**
```solidity
if (hadRoleQuantity && willHaveRole && initialQuantity > quantity) {
    newTotalQuantity = currentRoleSupply.totalQuantity - quantityDiff;
}
```

**Increase:**
```solidity
if (hadRoleQuantity && willHaveRole && initialQuantity < quantity) {
    newTotalQuantity = currentRoleSupply.totalQuantity + quantityDiff;
}
```
