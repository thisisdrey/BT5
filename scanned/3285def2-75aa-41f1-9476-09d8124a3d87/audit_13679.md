# ADM-1 | Admin Role Changes Should Be Two Step

## Summary
This bug report concerns a function, `changeAdmin`, which allows the admin address to be transferred to the wrong address. This is an issue because the admin address carries numerous important abilities for the system. To resolve this, the recommendation was to implement a two step “push” and “pull” admin transfer process. This would involve a separate function to relinquish ownership if desired. The key team adopted the recommended push and pull transfer process to resolve the issue.

## Details
**Description**

As addressed in GLOBAL-1, the admin address carries numerous important abilities for the system.
However the `changeAdmin` function allows the admin address to be errantly transferred to the wrong address as it does not use a two-step transfer process.

**Recommendation**

Implement a two step “push” and “pull” admin transfer process. If it is desired to have a method to relinquish ownership, implement a separate function to do so.

**Resolution**

Key Team: The recommended push and pull transfer process was adopted.
