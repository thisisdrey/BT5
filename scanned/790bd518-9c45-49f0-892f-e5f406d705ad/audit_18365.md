# [M-05] cBridge integration fails to send native tokens

## Summary
This bug report is about a bug found in the CBridgeFacet.sol file in the 2022-03-lifinance repository. The bug was found by hickuphh3, hyh, rayn, shw, WatchPug, and wuwe1. The bug occurs when a call to the external `sendNative()` function is made, as it fails to include sending the native tokens together with it. In order to demonstrate this bug, the following test case was added to the CBridgeFacet test file:

```jsx
// TODO: update bridge address to 0x5427FEFA711Eff984124bFBB1AB6fbf5E3DA1820
it.only('reverts because ETH not sent to bridge', async () => {
  CBridgeData.token = constants.AddressZero
  CBridgeData.amount = constants.One
  await expect(lifi.connect(alice).startBridgeTokensViaCBridge(lifiData, CBridgeData, {
    value: constants.One,
    gasLimit: 500000
  })).to.be.revertedWith('Amount mismatch');
})
```

The recommended mitigation steps for this bug include adding the following code to the CBridgeFacet.sol file:

```jsx
ICBridge(bridge).sendNative{ value: _cBridgeData.amount }(
  _cBridgeData.receiver,
  _cBridgeData.amount,
  _cBridgeData.dstChainId,
  _cBridgeData.nonce,
  _cBridgeData.maxSlippage
);
```

In addition, the `payable` keyword should be added to the CBridge interface. The issue was confirmed with a fix by H3xept (Li.Fi) and gzeon (judge).
