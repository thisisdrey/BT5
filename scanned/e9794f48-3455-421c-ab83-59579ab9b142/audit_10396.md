# Potential Pool Unbalancing

## Summary
The report discusses a bug in a pool system that adjusts its composition based on a specified amount of tokens. However, if the pool's state changes between the submission and execution of a transaction, due to activities such as arbitrage bots taking advantage of price differences, it may cause the pool to become imbalanced. This is because arbitrage activities can cause rapid changes in token prices. The bug occurs when a re-balancing transaction is initiated but not executed immediately, causing the pool to re-balance based on outdated prices, resulting in an imbalance. To fix this bug, the report suggests ensuring that re-balancing is necessary to prevent changes to reserve balances. Additionally, the computation for the re-balancing amount should be done on-chain for increased accuracy and adaptability to changing market conditions. The bug has been fixed in version 0a258ce.

## Details
## Pool Composition Adjustment

The pool adjusts the pool’s composition based on a specified amount of tokens. However, if the state of the pool changes between the transaction submission and its execution due to factors such as arbitrage bots exploiting price differences, it may imbalance the pool. Arbitrage activities may result in rapid changes in token prices. If a re-balancing transaction is initiated but not executed immediately, the pool may re-balance based on outdated prices, unbalancing it.

## Remediation

Ensure that rebalancing is necessary to prevent alterations to the reserve balances. Additionally, computation for the rebalancing amount should be on-chain, providing increased accuracy and adaptability to changing market conditions.

## Patch

Fixed in `0a258ce`.
