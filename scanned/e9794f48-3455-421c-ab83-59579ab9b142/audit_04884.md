# Weiroll value calltypes will read the wrong value amount from the state array

## Summary
The report discusses a bug in the Weiroll software that supports calls with native values attached. When the calltype is FLAG_CT_VALUE-CALL, the first argument in the input list should be taken as the amount passed along with the call. However, in CommandBuilder.buildInputs(), the state is always referenced along with IDX_VALUE_MASK, which means that the correct value amount is not being forwarded. This could potentially impact deposit/withdrawal recipes in Royco, although the exact impact is unclear. The recommendation is to update the value calltype branch in _execute() to properly index into the state array. The bug has been fixed in the Royco software by swapping the weiroll library in commit 07b60a64. The fix has been reviewed and deemed satisfactory by Spearbit.
