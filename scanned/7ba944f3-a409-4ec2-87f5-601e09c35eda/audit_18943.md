# [M-10] Use safeTransfer/safeTransferFrom consistently instead of transfer/transferFrom

## Summary
This bug report is about a vulnerability in the LaunchEvent.sol file of the code-423n4/2022-01-trader-joe repository on Github. It is possible that token transfers can fail silently, which could lead to incorrect token accounting in the contract. The vulnerability can be seen in the provided links, which are lines 457, 463, 489, 513 and 537 of the LaunchEvent.sol file. The recommended mitigation steps are to use safeTransfer/safeTransferFrom or require() consistently to check the return value of token transfers.

## Details
_Submitted by cccz, also found by 0x1f8b, bobi, byterocket, Dravee, hack3r-0m, sirhashalot, TomFrenchBlockchain, UncleGrandpa925, and WatchPug_

It is good to add a require() statement that checks the return value of token transfers or to use something like OpenZeppelin’s safeTransfer/safeTransferFrom unless one is sure the given token reverts in case of a failure. Failure to do so will cause silent failures of transfers and affect token accounting in contract.
