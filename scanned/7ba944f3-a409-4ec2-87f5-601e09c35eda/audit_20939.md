# Lack of docs

## Summary
This bug report is about a contract code that is not clear in its purpose and usage. The logic of the code with the accumulated `getCumulativeReward` and `totalWithdrawnReward` needs to be described in the code and the typical scenario of the contract usage should be explained. To solve this issue, it is recommended that comprehensive docstrings be added to the code. Docstrings are comments in the code that explain the purpose and usage of the code, which will make it easier for users to understand the contract.

## Details
##### Description
The purpose of the contract and the way it will work is not clear from the code.
Also the logic with accumulated `getCumulativeReward` and `totalWithdrawnReward` must be described in the code as well as the typical scenario of the contract usage.

##### Recommendation
Add comprehensive docstrings.
