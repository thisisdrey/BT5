# Active redirection can be removed when the DistributionModule contract is paused 

## Summary
The DistributionModule contract is missing a necessary function called "removeRedirectedOffChainDistribution" which should only be available when the contract is not paused. This means that when the contract is paused, an active redirection can be removed by anyone, even if they are not the designated admin. This could lead to unintended consequences, especially if the admin address is compromised. The recommended solution is to add the "whenNotPaused" modifier to the function, which will prevent this issue from occurring. The bug has been fixed in the latest pull request and has been verified by the Cantina team.

## Details
## DistributionModule Vulnerability Report

## Context
Refer to: `DistributionModule.sol#L863-L871`

## Description
The `DistributionModule.removeRedirectedOffChainDistribution` function is missing the `whenNotPaused` modifier. Due to this omission, an active redirection can be removed by an account even when the `DistributionModule` contract is paused. 

Pausing the contract is only intended to be triggered in critical scenarios (exploit, upgrades, active bugs, etc.), and performing any user operation in a paused state can lead to unintended outcomes. One potential scenario could occur if the redirection admin address becomes compromised. In such a case, pausing the contract would not prevent the compromised redirection admin from removing redirections.

## Recommendation
The function signature should be modified as follows:

```solidity
- function removeRedirectedOffChainDistribution(address account) external {
+ function removeRedirectedOffChainDistribution(address account) external whenNotPaused {
    DistributionModuleStorageV0 storage $ = _distributionModuleStorageV0();
    if (msg.sender != account) {
        _requireOnlyRedirectionAdmin($);
    }
    delete $.activeRedirectedOffChainDistribution[account];
    emit OffChainDistributionRedirectedRemoved(account);
}
```
