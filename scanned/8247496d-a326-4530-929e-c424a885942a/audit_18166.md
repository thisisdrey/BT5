# [M-04] Wrong requirement in reweight function (`ManagedIndexReweightingLogic.sol`)

## Summary
This bug report is about a vulnerability that affects the list of assets in a managed index reweighting logic. The vulnerability occurs when the function to reweight the list of assets is not true, because the comparison of the maximum assets for an index and the assets that need to be removed does not make sense. The impact of this vulnerability is that the list of assets won't be changed after reweight due to reverted tx. The bug was detected through manual review. The recommended mitigation steps to fix this bug is to require assets.length() <= IIndexRegistry(registry).maxComponents() at the end of the function.

The list of assets won't be changed after reweight because of reverted tx.

### Proof of Concept

`require(_updatedAssets.length <= IIndexRegistry(registry).maxComponents())`
when [reweight](https://github.com/code-423n4/2022-04-phuture/blob/594459d0865fb6603ba388b53f3f01648f5bb6fb/contracts/ManagedIndexReweightingLogic.sol#L32) is not true, because as in the [doc](https://github.com/code-423n4/2022-04-phuture/blob/594459d0865fb6603ba388b53f3f01648f5bb6fb/contracts/interfaces/IIndexRegistry.sol#L19),
`maxComponent` is the maximum assets for an index, but `_updatedAssets` also contain the assets that you want to remove. So the comparision makes no sense.

### Recommended Mitigation Steps

Require `assets.length() <= IIndexRegistry(registry).maxComponents()` at the end of function instead.
