# [M05] Cash settling during low liquidity deletes cash receiver tokens

## Summary
This bug report is about the Swapnet Protocol, which is a decentralized protocol for exchanging assets. The bug occurs when the `settleCashBalance` function is executed, which can lead to the execution of `_attemptToSettleWithFutureCash`, which in turn executes `Portfolios.raiseCollateralViaCashReceiver`. This function is used to sell a full amount of fCash receiver tokens in exchange for collateral. 

However, if the FutureCash market has low liquidity, with `market.totalFutureCash` less than the fCash amount being sold, then only `market.totalFutureCash` tokens are sold, not the full amount. This is not communicated back to the calling function, and so the full amount of receiver tokens are removed from the user’s account even though not all of them were sold for collateral.

The bug was fixed by adding a second return parameter to `FutureCash.tradeCashReceiver` to return the number of fCash receivers actually sold. This was done through a pull request, and the check has been removed from `tradeCashReceiver` entirely.

## Details
During the execution of [`settleCashBalance`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L710), the flow can lead to the execution of [`_attemptToSettleWithFutureCash`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L786), which itself executes [`Portfolios.raiseCollateralViaCashReceiver`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L955). When a cash receiver token is found in the user’s portfolio, [`FutureCash.tradeCashReceiver`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Portfolios.sol#L726) is called to sell the full amount of fCash receiver tokens in exchange for collateral. The receiver tokens are then removed from the user’s portfolio by [creating the same number of payer tokens](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Portfolios.sol#L740-L750) to offset the receiver tokens.


However if the FutureCash market has low liquidity, with `market.totalFutureCash` less than the fCash amount being sold, then [only `market.totalFutureCash` tokens are sold](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L731-L734), not the full amount. This is not [communicated back to the calling function](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L752), and so the full amount of receiver tokens are removed from the user’s account even though not all of them were sold for collateral.


Consider adding a second return parameter to [`FutureCash.tradeCashReceiver`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Portfolios.sol#L726) to return the number of fCash receivers actually sold. The [number of receiver tokens removed](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Portfolios.sol#L749) from the user’s account can then be adjusted to reflect this trade accurately.


***Update:** Fixed in [pull request #53](https://github.com/swapnet-protocol/swapnet-lite/pull/53). The check has been removed from `tradeCashReceiver` entirely.*
