# [M06] Not all calculations use SafeMath

## Summary
This bug report is about the Solidity code used in the Swapnet-Lite project. It explains that there are lines of code that do not use the SafeMath library to check for under/overflow which can lead to severe issues. It then lists three lines of code which are not using the SafeMath library and recommends to use it for all calculations, except those that are already checked for errors. The bug has been fixed in pull request #54.

## Details
Calculations that do not check for under/overflow can easily cause severe issues in Solidity. Throughout the majority of the code, [SafeMath.sol](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/release-v3.1.0/contracts/math/SafeMath.sol) is used to ensure that any calculations that could face such issues revert instead of under/overflowing. However as seen in [**[C03] Borrow rates can be severely miscalculated**](#c03), and [**[H03] `_getImpliedRate` does not handle errors**](#h03), some lines of code do mathematical calculations without using SafeMath which has lead to issues.


Besides the named issues above, other lines of code that do not check for math errors and therefore risk under/overflow include:  

– [Common.sol line 155](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/utils/Common.sol#L155)  

– [FutureCash.sol line 990](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L900) and [line 1054](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L1054)  

– [Escrow.sol line 907](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L907)


Consider using using [SafeMath.sol](https://github.com/OpenZeppelin/openzeppelin-contracts/blob/release-v3.1.0/contracts/math/SafeMath.sol) for all calculations, except those that are previously checked for errors using an `if` or `require`.


***Update:** Fixed in [pull request #54](https://github.com/swapnet-protocol/swapnet-lite/pull/54).*
