# Old quests can be extended via increaseQuestDuration()

## Summary
The bug report is about the function increaseQuestDuration() in the QuestBoard.sol file. This function does not check if a quest is already in the past, which means that extending a quest from the past in duration is not useful and might require additional calls to closePartOfQuest-Period(). The recommendation was to determine what the actions should be when the quest is in the past. The Paladin suggested that when calling the function, the current period should be either the last period of the Quest, or before that last period. If not, the Quest is over and the function should revert. This was implemented in #8.

## Description
Function `increaseQuestDuration()` does not check if a quest is already in the past. Extending a quest from the past in duration is probably not useful. It also might require additional calls to `closePartOfQuest-Period()`.

```solidity
function increaseQuestDuration(...) ... {
    updatePeriod();
    ...
    uint256 lastPeriod = questPeriods[questID][questPeriods[questID].length - 1];
    ...
    uint256 periodIterator = ((lastPeriod + WEEK) / WEEK) * WEEK;
    ...
    for(uint256 i = 0; i < addedDuration;) {
        ...
        periodsByQuest[questID][periodIterator]....= ...
        periodIterator = ((periodIterator + WEEK) / WEEK) * WEEK;
        unchecked { ++i; }
    }
    ...
}
```

## Recommendation
Determine what the actions should be when the quest is in the past.

## Paladin
We check that when calling the function, the current period is either the last period of the Quest or before that last period. If not, the Quest is over, and we revert. Implemented in #8.
