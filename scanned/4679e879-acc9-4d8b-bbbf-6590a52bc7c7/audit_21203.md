# Unchecked return value for transferFrom calls

## Summary
This bug report is about two `transferFrom` calls that do not check the return value. This can be problematic as some tokens signal failure by returning false. The bug was addressed in [fei-protocol/fei-protocol-core#12](https://github.com/fei-protocol/fei-protocol-core/pull/12) and the recommendation is to consider adding a require-statement or using `safeTransferFrom`. A require-statement is a line of code that checks the return value of a function. In this case, it should be used to check the return value of the two `transferFrom` calls. `SafeTransferFrom` is a function that can be used to transfer tokens in a secure manner and it should be used in place of `transferFrom` to ensure that the tokens are transferred successfully.

## Details
#### Resolution



This was addressed in [fei-protocol/fei-protocol-core#12](https://github.com/fei-protocol/fei-protocol-core/pull/12).


#### Description


There are two `transferFrom` calls that do not check the return value (some tokens signal failure by returning false):


**code/contracts/pool/Pool.sol:L121**



```
stakedToken.transferFrom(from, address(this), amount);

```
**code/contracts/genesis/IDO.sol:L58**



```
fei().transferFrom(msg.sender, address(pair), amountFei);

```
It is usually good to add a require-statement that checks the return value or to use something like `safeTransferFrom`; unless one is sure the given token reverts in case of a failure.


#### Recommendation


Consider adding a require-statement or using `safeTransferFrom`.
