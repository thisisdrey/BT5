# `nextCollectionId` should be limited

## Summary
This bug report is about a problem with the contract `nextCollectionId` in the FantiumNFTV3.sol file. The issue is that the contract is running unnecessary checks on tokenId when claiming tokens, which can cause the claiming functionality for tokens of collection with an id greater than or equal to 100_000 to revert. To fix this issue, the bug report recommends adding a check that limits `nextCollectionId` to a value below 1_000_000.

## Details
##### Description
Due to unnecessary checks on tokenId in claiming, contract `nextCollectionId` should be limited. Otherwise, claiming functionality for tokens of collection with an id >= 100_000 will revert.
https://github.com/FantiumAG/smart-contracts/blob/a2d126453c1105028f12277b8f342d2cdbf01a77/contracts/FantiumNFTV3.sol#L522

##### Recommendation
We recommend adding the following check: `nextCollectionId < 1_000_000`.
