# Taker incurs losses when `SingleQuote.useOldAmount` is set to `true`

## Summary
The bug report describes an issue where a setting called **SingleQuote.useOldAmount** is causing problems when used with another setting called **SingleQuote.makerAmount**. This can result in a trader receiving less tokens than expected when using the `LiquoriceSettlement.settleSingle()` function. The report recommends implementing checks to prevent this issue and ensure that traders receive the correct amount of tokens.

## Details
##### Description
When **SingleQuote.useOldAmount** set to `true` and **SingleQuote.makerAmount** less than **Single.makerAmount** anyone can call `LiquoriceSettlement.settleSingle()`, as a result trader will receive less tokens.  Although this requires the taker to sign such a quote, the process functions incorrectly, potentially causing losses for the taker.

##### Recommendation
We recommend implementing checks for such cases to prevent the trader from receiving fewer tokens than expected, allowing only for extra profit.
