# `disableInitializers` not used to prevent uninitialized contracts

## Summary
This bug report discusses an issue with contracts that are designed to be upgradeable. The bug is considered medium severity and has been resolved. The problem is that implementation contracts may be left uninitialized, which can cause problems. To prevent this, the recommendation is to add a specific function to the constructor that will automatically lock the contracts when they are deployed. The recommendation is to add the function `_disableInitializers()` to the constructor.

## Details
**Severity**: Medium	

**Status**: Resolved

**Description**

All the contracts are designed to be upgradeable.

In order to prevent leaving an implementation contract uninitialized it is recommended adding the _disableInitializers() function in the constructor to lock the contracts automatically when they
are deployed.

**Recommendation**:

Add `_disableInitializers()` to the constructor.
