# Hardcoded Limit for Delegation Retrieval and Lack of Error Handling

## Summary
Bug Report Summary:

A code snippet in the Elys Network project has two issues related to delegation retrieval and error handling. The code uses a hardcoded limit of 1024 for delegations, which may lead to incomplete reward withdrawal if the delegator has more than 1024 delegations. Additionally, the code does not handle errors properly, which can result in unexpected behavior and difficulty in diagnosing issues. To address these issues, the team recommends using a dynamic or configurable limit for delegation retrieval and implementing proper error handling. The issue has been solved by the Elys Network team by iterating delegations.
