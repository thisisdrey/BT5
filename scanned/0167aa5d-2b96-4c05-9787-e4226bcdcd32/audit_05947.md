# Depositors might be unable to withdraw desired asset

## Summary
The PSM3 contract allows deposits of three assets and allows for swapping between them. However, a malicious user can exploit this by deliberately causing a denial of service for withdrawals. This can be done by swapping out the withdrawal asset and later swapping it back in, depleting the contract's balance. This can be a problem for small depositors as their share balance can be affected. The recommendation is to add a function that chooses the asset with the highest value in the contract's balance for withdrawals. The Maker team has acknowledged this issue and plans to address it by adding a check-3-asset logic in the contract. The Cantina Managed team also acknowledges the issue and suggests checking the current balances of the assets before deciding which one to withdraw. This is considered a low-risk issue.

## Details
## Context: Global Scope

## Description
The PSM3 allows deposits of three assets and enables anyone to swap between these assets. A depositor may not expect to always be immediately able to withdraw the asset they deposited (or an asset of their choice) as the contract's balance might have been depleted by a swap. A malicious user can deliberately DoS withdrawals by sandwiching the withdrawal, swapping out the withdrawal asset, and later swapping it back in. As there are no fees, only cheap L2 gas costs are to be paid. They could even leave 1 wei in the contract balance to burn a rounded-up user's share balance each time for maximum impact.

## Recommendation
Consider adding a `withdrawAnyAsset(address receiver, uint256 maxValueToWithdraw)` function that chooses the asset to withdraw based on the highest value asset in the contract's balance. This function would convert `maxValueToWithdraw` to the asset's decimals and call `withdraw(asset, receiver, maxAssetToWithdraw)`.

## Maker
Acknowledged. This is an interesting edge case. This will be a contract executing the deposit/withdrawal, so there will be no issue, as we can put the check-3-asset logic inside the spell or another contract if it becomes a problem. We don't expect third-party LPs, and there is private transaction submission in the extreme case.

## Cantina Managed
Acknowledged. If the contract can check the current balances of the three assets before deciding what asset to withdraw, then it should be fine for you. And for small depositors, I honestly don't think anyone would attempt this.
