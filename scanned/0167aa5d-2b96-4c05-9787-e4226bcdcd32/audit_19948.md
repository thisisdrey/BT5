# [H-02] Index Pool always swap to Zero

## Summary
This bug report explains how an error in the code of the IndexPool contract can prevent users from buying token A with token B when an Index pool is initiated with two tokens A: B and the weight rate = 1:2. The root cause of the issue is an error in the implementation of the Exponentiation by squaring algorithm in the IndexPool contract, specifically that there is no bracket for the 'for' statement. This is a high-risk issue as it renders the IndexPool contract non-functional. 

The bug was demonstrated by initiating the pool with a 2:1 weight rate, which prevented users from buying dai with link. This was done by encoding an ABI which included the addresses of the two tokens, the weight rate and a value of 10^13. 

Overall, this bug report explains a high-risk issue in the IndexPool contract which prevents users from buying token A with token B when an Index pool is initiated with two tokens A: B and the weight rate = 1:2. This is due to an error in the implementation of the Exponentiation by squaring algorithm, specifically that there is no bracket for the 'for' statement.

#### Impact
When an Index pool is initiated with two tokens A: B and the weight rate = 1:2, then no user can buy token A with token B.

The root cause is the error in pow. It seems like the dev tries to implement [Exponentiation by squaring](https://en.wikipedia.org/wiki/Exponentiation_by_squaring).
[IndexPool.sol#L286-L291](https://github.com/sushiswap/trident/blob/9130b10efaf9c653d74dc7a65bde788ec4b354b5/contracts/pool/IndexPool.sol#L286-L291)

```solidity
function _pow(uint256 a, uint256 n) internal pure returns (uint256 output) {
  output = n % 2 != 0 ? a : BASE;
  for (n /= 2; n != 0; n /= 2) a = a * a;
  if (n % 2 != 0) output = output * a;
}
```

There's no bracket for `for`.

The `IndexPool` is not functional. I consider this is a high-risk issue.
