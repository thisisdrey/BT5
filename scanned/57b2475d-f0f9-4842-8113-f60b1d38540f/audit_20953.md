# Forced deposit from an account exceeding available assets

## Summary
This bug report is regarding the possibility of getting an exception due to throw from another contract while depositing. The bug is present in the code for StrategyDAI, StrategyUSDC, and StrategyUSDT contracts. To fix this bug, a check should be added to the code to limit the amount that can be deposited. This check should compare the amount that is being deposited to the balance of the contract. If the amount is greater than the balance, then the amount should be set to the balance of the contract. This check will help to ensure that the exception is not thrown and the deposit can be completed successfully.

## Details
##### Description
It is possible get an exception due to throw from another contract while depositing. 
- https://github.com/orbxball/stablecoins-3pool/blob/adeb776933c6cb3b8306239cc3357d4c6239a88d/contracts/StrategyDAI.sol#L207
- https://github.com/orbxball/stablecoins-3pool/blob/adeb776933c6cb3b8306239cc3357d4c6239a88d/contracts/StrategyUSDC.sol#L207
- https://github.com/orbxball/stablecoins-3pool/blob/adeb776933c6cb3b8306239cc3357d4c6239a88d/contracts/StrategyUSDT.sol#L209

##### Recommendation
It is recommended to add the following check:
```solidity
uint256 _amt = want.balanceOf(address(this));
if (_amount > _amt) _amount = _amt;
```
