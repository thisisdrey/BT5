# Safe math is not used

## Summary
This bug report is about a potential overflow and incorrect logic of the harvestTrigger() function in the BaseStrategy.sol file at line 338. The issue is that the function does not use SafeMath, which is a library of functions that help to prevent overflows when working with numbers. As a result, there is a risk of the code producing incorrect results. 

The recommendation to fix this issue is to use SafeMath. This library provides functions such as add, sub, and mul that can be used instead of the existing functions. This will help to prevent any potential overflows and ensure the code produces the correct results.

## Details
##### Description
At this line: https://github.com/iearn-finance/yearn-vaults/blob/cff924f1894cca1820a588b14d341c4fa4f384c0/contracts/BaseStrategy.sol#L338
```solidity
return (profitFactor * callCost < credit.add(profit));
```
Due to the fact that safe math is not used, overflow and incorrect logic of the `harvestTrigger()` function may occur.

##### Recommendation
We suggest using SafeMath
