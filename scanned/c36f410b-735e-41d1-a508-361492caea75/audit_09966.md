# Users Might Expect Inexistent Rewards

## Summary
The report describes a problem with the `createDistribution()` function, which is used to distribute rewards to users. The function accepts two inputs: the `_merkleRoot` which contains the users and their reward amounts, and the `_amount` which represents the total amount to be distributed. However, the function does not properly validate the total amount of rewards, which can result in some users expecting rewards that are not available. The recommendation is to modify the function to accept two arrays instead of the current inputs, and to add additional variables to keep track of the rewards for each user.

## Details
**Update**
The dev team stated that this is intended by design and have decided not to change it.

**Description:** The function `createDistribution()` accepts two input arguments: the `_merkleRoot` containing all the users and the amount each user should get, and the `_amount` input representing the total amount to be distributed. The `_amount` value should be equal to the total amount to be distributed to all the leaves in the Merkle tree, and also less or equal to the current ETH or token balance of the contract. Otherwise, some users would be expecting rewards that are not even available.

**Recommendation:** Validating the total amount of rewards all the users should get from the Merkle root alone is not possible. A possible solution would be to modify the function to accept two arrays instead as input parameters: one with the addresses of the beneficiaries and the other with the amounts. This would also require additional state variables to keep track of the amount of which token each influencer can claim.
