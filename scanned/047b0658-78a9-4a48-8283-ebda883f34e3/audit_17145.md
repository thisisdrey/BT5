# Unchecked stake ownership in ticket casting

## Summary
The getLotTicketByStaking instruction is not verifying if the owner of a legacy Frakt stake account is actually the one obtaining the raffle tickets. This allows an attacker to get tickets by specifying any stake account, even if they don't own it. To fix this, the user obtaining the ticket should be verified as the owner of the stake account. A patch has been created to address this issue.

## Details
## getLotTicketByStaking Instruction Vulnerability

The `getLotTicketByStaking` instruction allows the owner of a legacy Frakt stake account to obtain raffle tickets. However, it does not verify that the stake account is actually owned by the invoker. This allows an attacker to obtain raffle tickets by specifying any stake account on the chain — regardless of whether or not they own it.

## Remediation

Verify that the user obtaining the ticket is the stake account’s owner.
