# [M-02] Use of `safeApprove` will always cause `approveMax` to revert

## Summary
The bug reported is related to the usage of the `safeApprove()` function in the `mStableYieldSource` contract. This function, which is provided by OpenZeppelin, is documented as "Deprecated because of approve-like race condition" and "To be used only for initial setting of allowance (current allowance == 0) or resetting to 0 because it reverts otherwise". The bug report states that the `mStableYieldSource` contract uses this function to increase the allowance to `uint256.max` when it falls low, which is intended to work similarly to the `SwappableYieldSource` contract. However, this will not work as expected because the `safeApprove()` function will always revert if the current allowance is not equal to 0. The initial allowance is already set as `uint256.max` in the constructor, and once it is reduced, it cannot be increased using this function unless it is invoked when the allowance is reduced completely to 0.

The bug report recommends using logic similar to the `SwappableYieldSource` contract instead of using the `safeApprove()` function. This issue has since been fixed in the commit https://github.com/pooltogether/pooltogether-mstable/pull/3/commits/156a990901e6ddff543897905e3ea3d09c78d817.

## Details
_Submitted by 0xRajeev, also found by pauliax, shw, and cmichel_

Unlike `SwappableYieldSource` which uses `safeIncreaseAllowance` to increase the allowance to `uint256.max`, `mStableYieldSource` uses OpenZeppelin’s `safeApprove()` which has been documented as (1) Deprecated because of approve-like race condition and (2) To be used only for initial setting of allowance `(current allowance == 0)` or resetting to 0 because it reverts otherwise.

The usage here is intended to allow increase of allowance when it falls low similar to the documented usage in `SwappableYieldSource`. Using it for that scenario will not work as expected because it will always revert if current allowance is != 0. The initial allowance is already set as `uint256.max` in constructor. And once it gets reduced, it can never be increased using this function unless it is invoked when allowance is reduced completely to 0. See issue page for referenced code.

Recommend Using logic similar to `SwappableYieldSource` instead of using `safeApprove()`.
