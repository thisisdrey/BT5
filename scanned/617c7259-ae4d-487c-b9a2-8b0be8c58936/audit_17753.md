# Actual transferred amount may differ than expected

## Summary
This bug report is about a problem with ERC20 tokens, such as USDT, having fees on transfer. This can cause an issue with reward accounting on farming, as the farm contract may receive fewer tokens than expected and start farming with an expected amount of rewards, which will result in a lack of liquidity for rewards. Additionally, the `deposit` and `withdraw` functions in the `FarmingPool` contract may not work correctly. 

The recommendation to solve this problem is to start farming/deposit in `FarmingPool`/withdraw in `FarmingPool` with the actual transferred amount and return an actual deposited/withdrawn/claimed amount.
