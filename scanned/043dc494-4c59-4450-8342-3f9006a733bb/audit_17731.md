# [M-10] User can alter amount returned by redeem function due to control transfer

## Summary
This bug report is about a problem with the ERC777 token contract. Control is transferred to the receiver when receiving the ERC777. Then, the receiver is able to transfer the ERC777 to another account, at which time the before and after balance calculation will be incorrect. This is because the assets can be as low as 0 if they have transferred the same amount out as received.

The bug was confirmed by two users, 0xDjango and Jeffywu (Notional) from the Github repository 2022-06-notional-coop-findings. Jeffywu (Notional) disputed the bug initially, stating that control of the contract is not transferred to anyone during a `balanceOf` call which is read only. FatherGoose1 (warden) then commented that the control is transferred in `_redeemInternal()` which calls `_burn()` on the ERC777. Jeffywu (Notional) confirmed the issue and the bug was marked as confirmed.
