# Exposure to revert due to wrong operation.

## Summary
This bug report is about a medium severity issue that has been resolved. The report states that in a contract called DistributionManager.sol, there is a function called distributeRewards that can cause a panic revert (which can lead to a denial of service) in certain situations where the totalAllocationPoints or totalAllocationFee is zero. The recommendation is to add a check for zero amounts in this function.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In contract DistributionManager.sol, the function  distributeRewards


is exposed to a panic revert (leads to Denial of service) in some valid cases if the one of totalAllocationPoints and totalAllocationFee  is zero.

**Recommendation**: 

Add zero amount check.
