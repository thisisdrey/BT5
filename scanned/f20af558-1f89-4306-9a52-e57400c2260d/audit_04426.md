# Update Relevant Parts of partial_hash.nr Code to the Latest sha256.nr Library by Aztec

## Summary
The Aztec library has been updated, causing changes to the `partial_hash.nr` library. One significant change is the addition of a new constraint for verifying message block padding. It is recommended to update the `partial_hash.nr` library to match the changes in the Aztec library.

## Details
#### Description

Since the beginning of this audit, Aztec has updated the [`sha256.nr` library](https://github.com/noir-lang/noir/blob/master/noir_stdlib/src/hash/sha256.nr) that is heavily utilized by the `partial_hash.nr` library of this repository. Among refactoring, some updates stand out such [as additional constraints](https://github.com/noir-lang/noir/blob/b88db67a4fa92f861329105fb732a7b1309620fe/noir_stdlib/src/hash/sha256.nr#L114-L119):

```
        if !is_unconstrained() {
            let new_msg_byte_ptr = verify_msg_block(msg, message_size, msg_block, msg_start);
            if msg_start < message_size {
                msg_byte_ptr = new_msg_byte_ptr;
                verify_msg_block_padding(msg_block, msg_byte_ptr);
            }

```

Specifically, `verify_msg_block_padding` has been added to ensure proper padding of SHA256 blocks at the end of the hash.

As the Aztec library and its updates are outside of scope of this audit, it is difficult to gauge the true impact of not having this constraint but it is recommended to err on the side of caution and update the audit repository’s library as well.

#### Recommendation

Update the `partial_hash.nr` library to be in sync with Aztec’s `sha256.nr` library. At least include the newly added constraint for verifying message block padding.
