# No integration tests for the two main components ✓ Fixed

## Summary
A recent issue in multi-contract smart contract systems has highlighted the need for a full test suite to prevent future bugs. The existing tests cover two main components and each set of tests mocks the other component. This is good for unit testing, but may miss issues without proper system/integration tests that cover all components. To address this, `amp-contracts` was added as a submodule to `collateral-manager` and full integration tests were added. It is recommended to write test suites that achieve high code coverage to prevent missing obvious bugs that tests could cover. Going forward, consider adding system/integration tests for all components to ensure that issues in the next development and deployment cycles are avoided.
