# Unsafe type cast.

## Summary
The bug report is about a function called deposit() in a code file called PerpetualMarket.sol. This function has a line of code (line 110) that is not safe and can cause problems with calculating shares. This is because users can enter a value that is larger than a specific number, which can result in incorrect shares being minted. The report recommends using a safer method called SafeCast to avoid this issue.

## Details
**Description**

PerpetualMarket.sol, function deposit(), line 110.
Such unsafe cast can also lead to incorrect shares calculation,, since user can pass a value
greater than uint128,
shares would be minted with this value, but the actual transferred amount would be lower.

**Recommendation**:

Use SafeCast.
