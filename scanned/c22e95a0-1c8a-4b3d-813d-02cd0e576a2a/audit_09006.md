# potCreator can participate in multiple rounds and lose rewards 

## Summary
The bug report is about an issue in the QuailFinance contract, specifically in lines 169-170. The team's intention was to prevent the winner from entering the round again, but the potCreator (the person who created the pot) always becomes a participant in the next round. This means that if the potCreator wins, it breaks the assumption that an address can only win once in a pot. This also causes a problem where the reward won in a future round will overwrite all the rewards won in past rounds, making it impossible to claim. The recommendation is to check if the potCreator has already won before adding them to the round.

## Description
The intention of the team is to not let the winner enter the round again. However, the **potCreator** always becomes a participant in the next round. Hence, if **potCreator** is a winner, it breaks the assumption that an address wins only once in a pot. If **potCreator** wins again, the reward won in a future round overwrites all the rewards won in past rounds, making it unclaimable.

## Recommendation
Consider checking `pot.hasWon` to ensure **potCreator** is only added if it hasn't won before.
