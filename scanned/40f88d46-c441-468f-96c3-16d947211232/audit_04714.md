# Cooling Period Reset on Multiple Requests 

## Summary
The bug is in the code for SToken.sol and affects users who make multiple withdrawal requests within a specific time period called the COOLING_PERIOD. This causes delays and resets the eligibility date for the withdrawals, preventing them from being claimed at the intended time. The bug has been fixed by the developers in two different commits, making the code more efficient and reducing the risk for users.

## Details
## Context
SToken.sol#L79

## Description/Recommendation
If a user makes multiple withdrawal requests within the COOLING_PERIOD window, the withdrawal eligibility date keeps getting pushed back, causing indefinite delays. 

For example, with COOLING_PERIOD set to 7 days:

- The user requests withdrawal **X1**, which is eligible for claiming at **T1**. 
- 6 days later, they request another withdrawal **X2**. 
- Now, the eligibility time for both **X1** and **X2** resets to **T1 + 6 days**. 

This setup prevents **X1** from being claimed at **T1** as intended.

## YieldFi
- Fixed in commit `94b8d1c2`. 
- Implemented a `WithdrawNFT` which is `ERC721Enumerable` to make it read O(1).

## Throttle
- Fixed in commits `94b8d1c2` and `0ca7f252`.
