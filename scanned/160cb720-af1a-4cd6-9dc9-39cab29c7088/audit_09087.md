# [M-05] `oracleCircuitBreaker`: Not checking if price information of asset is stale

## Summary
The bug report discusses a potential issue with the `getOraclePrice` function in the `LibOracle.sol` contract. The function does not check if the timestamp of the asset token price oracle is stale, which could result in users trading at the wrong price if outdated information is used. The `oracleCircuitBreaker` function also does not check for stale timestamps. The recommended mitigation steps suggest setting a `chainlinkStaleLimit` for each asset and checking if the price information is not outdated. The assessed type of the bug is an issue with the Oracle. The issue has been confirmed by the DittoETH team and partial credit will be given for other findings related to the same concern.

### Recommended Mitigation Steps

Set the `chainlinkStaleLimit` for each asset and check if the price information is not outdated.
