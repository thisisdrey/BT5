# Double-Counting of Dividend Payout Results in Capital Providers Not Being Able to Withdraw Dividends

## Summary
The client has marked a bug as "Fixed" in the Nayms system. The bug was found in the `LibTokenizedVault.sol` file and it allowed for the addition of more dividend token denominations than the allowed maximum. This caused a double-counting issue and could result in incorrect dividend payouts for users. The bug was caused by a check being bypassed in the `_payDividend()` function. The report recommends implementing fixes to prevent the addition of more token denominations than allowed and to prevent the same token from being added to the list again. Additionally, another issue was found that needs to be fixed for accurate accounting.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `0649519abe4bde381328143d6b502e8b4ce3b9af`. All three items were fixed.

**File(s) affected:**`LibTokenizedVault.sol`

**Description:** The Nayms system currently restricts the number of `dividendDenomiation` to be at most `maxDividendDenominations`, which is currently set as `1`. However, in `LibTokenizedVault`, this restriction is checked before the value is updated, so the check can be bypassed. Specifically, we have the following snippet in `_payDividend()`:

```
if (s.dividendDenominationIndex[_to][_dividendTokenId] == 0) {
    if (s.dividendDenominations[_to].length > LibAdmin._getMaxDividendDenominations()) {
        revert("exceeds max div denominations");
    }
    s.dividendDenominationIndex[_to][_dividendTokenId] = uint8(s.dividendDenominations[_to].length);
    s.dividendDenominationAtIndex[_to][uint8(s.dividendDenominations[_to].length)] = _dividendTokenId;
    s.dividendDenominations[_to].push(_dividendTokenId);
}
```

This means we can add more than `1` dividend token id in the list `dividendDenominations`. However, this has a harmful unintended side effect, where a same `dividendTokenId` can be added to the array. Suppose we called `_payDividend(_to = Alice, _dividendTokenId = WETH)` for the first time. Note that after the function call, `s.dividendDenominationIndex[Alice][WETH]` will still be `0` because `uint8(s.dividendDenominations[Alice].length) = 0`, and now `s.dividendDenominations[Alice] = [WETH]`. Now, if we call `_payDividend(_to = Alice, _dividendTokenId = WETH)` again, it will pass both checks in `_payDividend`, and append `WETH` to the array `s.dividendDenominations[Alice]` again.

This leads to a double-counting issue in `_normalizeDividends` as `s.withdrawnDividendPerOwner[_tokenId][dividendDenominationId][_to] += dividendDeductionIssued;` will be executed twice in the loop, adding the same value twice to the same `dividendDenominationId`. Furthermore, since this value is used to calculate the dividend payout for an entity, the entity may receive a payout lower than expected because the system mistakenly assumes that the entity withdrew more dividends than it actually had.

To illustrate the above issue, consider the following example:

1.   Alice starts with `500 WETH` in its internal balance;
2.   Alice calls `startTokenSale(ALICE, 100, 100 WETH)`, selling `100 ALICE` tokens for `100 WETH`;
3.   Now Alice owns `100 ALICE` but they are locked;
4.   Alice pays `100 WETH` as a dividend;
5.   Alice pays `100 WETH` as a dividend;
6.   Bob buys all `100 ALICE` from Alice. Here, during the transfer, Alice would have withdrawn the `200 WETH` dividend owed to her, so her balance is `600 WETH` (`300` + `200` for dividend + `100` from Bob's purchase);
7.   Bob transfers all `100 ALICE` back to Alice;
8.   Alice pays `500 WETH` as a dividend;
9.   Alice tries to withdraw the `500 WETH` dividend but can only withdraw `100 WETH` due to the double counting issue and the other issue described in this report about Entity Token Holders not receiving future dividends;

**Recommendation:** Consider implementing the following fixes:

1.   Change the strict inequality `if (s.dividendDenominations[_to].length > LibAdmin._getMaxDividendDenominations())` to `if (s.dividendDenominations[_to].length >= LibAdmin._getMaxDividendDenominations())` to prevent adding more token denomination than `maxDividendDenomination`.
2.   Change `if (s.dividendDenominationIndex[_to][_dividendTokenId] == 0)` to `if (s.dividendDenominationIndex[_to][_dividendTokenId] == 0 && s.dividendDenominationAtIndex[_to][0] != _dividendTokenId)` so that the same `tokenId` cannot be added to the list again.
3.   Also, the other issue described in this report about Entity Token Holders not receiving future dividends must be fixed for the accounting to be accurate.
