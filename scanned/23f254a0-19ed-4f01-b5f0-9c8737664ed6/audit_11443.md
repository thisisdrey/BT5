# Vested RDNT Tokens Can Be Withdrawn Early With No Penalties

## Summary
The `individualEarlyExit` function in the `MultiFeeDistribution` contract allows users to withdraw their earnings by specifying a specific `unlockTime`. However, there is a bug where if the `unlockTime` is not found, the function will continue to execute and remove the first `userEarnings` item from the list, resulting in the user being able to withdraw their full amount without any penalties. This can be exploited by a malicious user to withdraw their full earnings at any time. The bug has been fixed in a recent update and will now revert the transaction if the `unlockTime` is not found.

## Details
The [`individualEarlyExit`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/staking/MultiFeeDistribution.sol#L895) function within the `MultiFeeDistribution` contract allows withdrawing a specific user earnings entry, which is identified by passing its `unlockTime` to [`ieeWithdrawableBalances`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/staking/MultiFeeDistribution.sol#L898-L901). Once a [record is matched](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/staking/MultiFeeDistribution.sol#L879-L883), its index is assigned and returned along with the penalty and burn amounts. The `individualEarlyExit` function [contains a check](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/staking/MultiFeeDistribution.sol#L903-L905) that presumes that an out-of-bounds `index` returned by `ieeWithdrawableBalances` indicates the target `unlockTime` was not found.


However, if the specified `unlockTime` is not found, `ieeWithdrawableBalances` will return all variables set to zero, including the `index`. Instead of [halting execution](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/staking/MultiFeeDistribution.sol#L904) as intended, it will continue and `individualEarlyExit` will [remove the first `userEarnings` item from the list](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/staking/MultiFeeDistribution.sol#L907-L914), while not correctly decreasing the user's [`earned` balance](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/staking/MultiFeeDistribution.sol#L918) accordingly. This transaction will not revert and will end up transferring 0 RDNT tokens to the user and the DAO treasury.


A malicious user can call `individualEarlyExit` as many times as necessary using an invalid `unlockTime` until their `userEarnings` array is empty, and then call the [`exit`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/staking/MultiFeeDistribution.sol#L927) function to determine the withdrawable balance by looping through the `userEarnings` array. Since there are no entries (they were all removed), the penalty and burn amounts returned by `withdrawableBalance` will be zero, resulting in a withdrawable balance equal to the full `balances[user].earned` value.


The end result is that any user can withdraw their full amount of RDNT rewards at any time, bypassing any penalties for exiting early.


To ensure users cannot withdraw earnings that are not fully vested without penalties, consider reverting when the specified `unlockTime` is not found.


***Update:** Resolved in [pull request #242](https://github.com/radiant-capital/v2-core/pull/242) at commit [0752677](https://github.com/radiant-capital/v2-core/commit/075267751d5ec2b3ce7fbd5fe9ab6f28547d23b1). If the specified `unlockTime` is not found, the transaction will revert with the custom error `UnlockTimeNotFound`.*
