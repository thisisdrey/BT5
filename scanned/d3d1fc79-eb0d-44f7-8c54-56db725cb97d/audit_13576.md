# `fantiumFeeBPS` should be restricted

## Summary
This bug report is about the `fantiumFeeBPS` variable in the FantiumClaimingV1.sol contract. The variable should never be greater than 10,000, otherwise a claim call will always revert. The recommendation is to add a check to make sure that `fantiumFeeBPS` is always less than or equal to 10,000. This will ensure that the claim call does not revert and the contract runs as intended.

## Details
##### Description
`fantiumFeeBPS` should be <= 10_000, otherwise a claim call will always revert.
https://github.com/FantiumAG/smart-contracts/blob/a2d126453c1105028f12277b8f342d2cdbf01a77/contracts/claiming/FantiumClaimingV1.sol#L242

##### Recommendation
We recommend adding the following check: `fantiumFeeBPS <= 10_000`.
