# [M-05] Admin privilege actions can be risky for users

## Summary
This bug report describes a vulnerability in a protocol that could be exploited by a malicious or compromised admin. If exploited, this vulnerability could lead to a Denial of Service (DoS) attack, which would prevent users from accessing the protocol. The vulnerability is related to the methods `updateOracle`, `updateRouter`, `updateRewardRouter`, and the `withdrawable` property of `PoolInfo`, which are all admin controllable and callable anytime. A malicious or compromised admin could provide non-existing addresses or set the `withdrawable` property to false for all pools, leading to the DoS attack.

The report recommends using a role-based access control approach instead of a single admin role, as well as a timelock for important admin actions, in order to prevent exploitation of this vulnerability. This would help ensure that only authorized personnel can access and modify the protocol, and that any changes are made in a timely manner.

## Details
**Likelihood:**
Low, because it requires a malicious/compromised admin

**Impact:**
High, because it can brick the protocol

**Description**

The methods `updateOracle`, `updateRouter` and `updateRewardRouter` are admin controllable and callable anytime. Same for the `withdrawable` property of `PoolInfo`. A malicious/compromised admin can either provide non-existing addresses or set the `withdrawable` property to false for all pools, leading to a DoS for users of the protocol.

**Recommendations**

Consider using an role-based access control approach instead of a single admin role as well as a timelock for important admin actions.
