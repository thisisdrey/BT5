# Missing access control in burn

## Summary
This bug report is about a problem in the TokenStableV6 contract. The issue is that the `burn()` function does not have proper access control, which means an attacker can use it to burn tokens from any address. This is a critical problem and has been resolved by recommending to either limit the function to only the owner of the tokens or add a control to prevent unauthorized use.

## Details
**Severity**: Critical

**Status**:  Resolved

**Description**

In the TokenStableV6 contract, there is a missing access control in `burn()`. This function can be used by an attacker to burn tokens from any address. 

**Recommendation**: 

It is advised to either allow only the msg.sender to burn his own tokens, or add an appropriate access control modifier for the same.
