# [M-07] `createERC20DAO()` will always revert when `_depositChainIds` has a length bigger than 1

## Summary
The bug report discusses an issue where a function called `createERC20DAO()` is not functioning properly. The severity and likelihood of the issue are both considered medium. The report explains that there is an `if` check that always reverts when the function is called, due to a line of code that calls a function called `createCCDao()` on the `Emitter` contract. However, this function is not implemented in the `Emitter` contract and can be confirmed by searching for it using `CTRL + Shift + F`. The report recommends implementing the `createCCDao()` functionality to resolve the issue.

## Details
## Severity

**Impact:** Medium

**Likelihood:** Medium

## Description

Whenever we call `createERC20DAO()`, we have the following **if** check:

```solidity
if (_depositChainIds.length > 1)
```

If we end up in there, we will always revert. This is due to this line:

```solidity
IEmitter(emitterAddress).createCCDao(_daoAddress, _depositChainIds);
```

It calls `createCCDao()` on the `Emitter` contract however that function is not implemented there. This can easily be confirmed by searching for that function name by using `CTRL + Shift + F` and we will see that the only function with that name across all files is located in the `IEmitter` interface.

## Recommendations

Implement the `createCCDao()` functionality
