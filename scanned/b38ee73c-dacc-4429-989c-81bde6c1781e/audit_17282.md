# Duplicate of increment.

## Summary
The bug report describes an issue with a function called withdrawERC20Tokens in the Treasury Module. The problem is that in one cycle, the index is being incremented twice. This means that the function is not working as intended and may cause errors or unexpected results. The recommendation is to remove the increment from the for loop in order to fix the issue. The re-audit comment states that the problem has been resolved.

## Details
**Description**

Treasury Module function withdrawERC20Tokens

100: for (uint256 index $=0;$ index < tokenAddressesLength; index ++) {

105:

106:

unchecked {

index ++;

107:

}

In one cycle, the index is incremented twice.

**Recommendation**

Remove increment from for head part.

**Re-audit comment**

Resolved
