# Incorrect sender in Transfer events

## Summary
This bug report is about an issue with the Transfer events in functions withdrawTo() and withdrawAllTo() when the target is not the same as the message sender. The USDC are sent from the protocol to the target and USDi are sent from the message sender to the protocol, but the Transfer events from these functions specify an incorrect sender. To fix this issue, the recommendation is to ensure that Transfer events from functions _withdraw() and _withdrawAll() specify _msgSender() as the sender. Be aware that emitted Transfer events prior to the fix of this issue specify an incorrect sender and shouldn’t be used as correct data. The resolution is that the recommendation has been implemented in commit b447c19.

## Details
## Description
The Transfer events in functions `withdrawTo()` and `withdrawAllTo()` have an incorrect `from` value when the target is not the same as the message sender. When a withdrawal is performed by any of these functions, USDC are sent from the protocol to the target and USDi are sent from the message sender to the protocol.

## Recommendations
- Ensure that Transfer events from functions `_withdraw()` and `_withdrawAll()` specify `_msgSender()` as the sender.
- Be aware that emitted Transfer events prior to the fix of this issue specify an incorrect sender and shouldn’t be used as correct data.

## Resolution
The recommendation has been implemented in commit `b447c19`.
