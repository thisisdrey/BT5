# Unexpected value for MAX_TOKENS_SOLD

## Summary
This bug report is about an issue with the state variable MAX_TOKENS_SOLD in the KinTokenSale contract. According to the whitepaper, the value of this constant should be 1 trillion, however, the value in the contract is slightly above half a trillion. The bug report suggests to update the constant to reflect the value indicated in the whitepaper.

The team has pointed out that the value in the contract represents the amount that will be offered during the sale, while the remaining half of the 1 trillion offered to the public has been sold in a presale. Therefore, the bug report does not need to be addressed and can be closed.

## Details
The state variable [`MAX_TOKENS_SOLD`](https://github.com/kikinteractive/kin-token/blob/3ed3a383b9304274ec22f41769716cadb854727f/contracts/KinTokenSale.sol#L33) in `KinTokenSale` represents the maximum number of tokens to be sold during the crowdsale, which is defined to be 1 trillion according to the [whitepaper](https://kin.kik.com/papers/Kin_Whitepaper_V1_English.pdf) (chapter 6, Kin token issuance). However, the value in the contract is 512192121951 (slightly above half a trillion). Update this constant to reflect the value indicated in the whitepaper.


***Update:** The team pointed out that this number represents the amount that will be offered during the sale, while the remaining half of the 1 trillion offered to the public [has been sold in a presale](https://medium.com/kinfoundation/kin-tde-if-you-want-to-participate-you-must-register-by-september-9-9-00-a-m-et-2f1304a4aa4b)*.\_
