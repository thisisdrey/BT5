# INADEQUATE TRACKING OF SWAPPED AMOUNTS

## Summary
The `swap_denom` function in the **swap-extension/swap_sparrow** contract is not properly updating the values of `swap_info.total_amount_in` and `swap_info.total_amount_out` variables, leading to a mix of coins involved in the swap. This makes it difficult to track the amount of each coin used. The bug has been solved by the `Kryptonite team` in commit [22e954a](https://github.com/KryptoniteDAO/swap-extension/pull/6/commits/22e954aec41150b91f75d51fb1dcbf75b7d12dc3).

## Details
##### Description

The `swap_denom` function in the **swap-extension/swap\_sparrow** contract updates the values of `swap_info.total_amount_in` and `swap_info.total_amount_out` variables whenever there is a swap operation. However, those values have a mix of both coins involved in the swap, so after some operations, it won't be possible to identify how much comes from each one of the coins.

Code Location
-------------

#### swap-extension/contracts/swap\_sparrow/src/handler.rs

```
// swap
let asset = Asset {
 amount: payment.amount,
 info: asset_infos[0].clone(),
};
let pair_address = pair_config.pair_address.clone();
let offer_asset = asset.clone();
let simulation_response = query_simulation(&deps.querier, pair_address.clone().to_string(), offer_asset.clone())?;

swap_info.total_amount_out += simulation_response.return_amount;
swap_info.total_amount_in += payment.amount;

```
