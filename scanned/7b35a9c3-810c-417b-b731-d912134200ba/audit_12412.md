# Proposals can be griefed by front-running and canceling

## Summary
A bug was found in the VeloGovernor.sol file on line 19. This bug allows a griefter to front-run a valid proposal with the same settings and then immediately cancel it. To prevent this, a macro contract can be written to generate random descriptions to avoid the front-run. Additionally, adding a proposer to the proposalHash() function can also help prevent griefing. The bug was fixed in a commit b2f8f2 and verified by Spearbit.

## Details
## Vulnerability Report

## Severity
**Medium Risk**

## Context
`VeloGovernor.sol#L19`

## Description
Because the Governor uses OZ's Implementation, a griefer can front-run a valid proposal with the same settings and then immediately cancel it. You can avoid the grief by writing a macro contract that generates random descriptions to avoid the front-run. 

See: [code-423n4/2022-09-nouns-builder-findings#182](https://github.com/code-423n4/2022-09-nouns-builder-findings/issues/182).

## Recommendation
Add `proposer` to the `proposalHash()` function to avoid griefing.
