# Master Encryption Key Derived Solely From Device `RNG` Weakens Wallet Security

## Summary
The client has marked an issue as "Unresolved" and wants to remove it from the audit report. The issue involves a security risk for users with stolen phones and requires a password to fix. The client believes that these factors should exclude the issue from the report. However, it is still acknowledged that users with jailbroken devices are at the most risk. To mitigate the issue, the `generateKey` function needs to be updated to use a more secure method, such as Argon2, and incorporate user-provided entropy. The affected file is `utils/cryptoUtils.ts` and the recommendation is to prompt the user for a password and use a modern KDF to encrypt the wallet key. This will also allow for changing the password in the future. It is also suggested to implement a timeout and lockout policy for accessing the wallet to balance usability and security.

**Update**
It is possible to mitigate this issue by updating the `generateKey` function to generate random bytes using the same approach; however, Argon2 can then be applied to the random bytes to generate the `derivedEncryptionKey` in the same function, which mitigates the issue. The derived key can be directly stored in the mobile device keychain, therefore avoiding the need to generate it each time.

**File(s) affected:**`utils/cryptoUtils.ts`

**Description:** In `cryptoUtils` the `generateKey` function derives the master encryption key solely from the device's RNG. If the RNG is weak or falls back to a low-entropy source, all encrypted wallet data may be compromised. Additionally, `PBKDF2` is used without incorporating user provided entropy, limiting overall key strength.

**Recommendation:** Incorporate user derived entropy (e.g., a password) alongside device randomness using a modern KDF such as Argon2id .

For example:

```
// Generate random `pwd_random_salt`, `wk_random_bytes`, `wk_random_salt`, `wek_random_salt`

pwd_hash = Argon2id(password, salt=pwd_random_salt, tuned params)
wallet_key =  Argon2id(wk_random_bytes, salt=wk_random_salt, secret=pwd_hash, tuned params) 
wallet_enc_key = Argon2id(password, salt=wek_random_salt, tuned params) 

// Encrypt `wallet_key` using `wallet_enc_key`
encrypted_wallet_key = AES.encrypt(wallet_key, key=wallet_enc_key)

// Encrypt wallet using `wallet_key`

// Delete `wallet_key`, `wallet_enc_key`, `wk_random_bytes` and `wk_random_salt`
```

To decrypt wallet:

*   Prompt user for their password.
*   Verify the password against `pwd_hash` using `pwd_random_salt`.
*   If they match, then derive `wallet_enc_key` from password and `wek_random_salt`.
*   Obtain `wallet_key` from `wallet_enc_key` and `encrypted_wallet_key`.
*   Decrypt wallet with `wallet_key`.

If the user wants to change password:

*   Prompt user for old password and new password.
*   Verify the old password against `pwd_hash` using `pwd_random_salt`.
*   If they match, then derive `wallet_enc_key` from password and `wek_random_salt`.
*   Obtain `wallet_key` from `wallet_enc_key` and `encrypted_wallet_key`.
*   Generate new `pwd_random_salt`, `wek_random_salt`.
*   Update `wallet_enc_key` as `Argon2id(new_password, salt=new_wek_random_salt, tuned params)`.
*   Update the `encrypted_wallet_key` as `AES.encrypt(wallet_key, key=wallet_enc_key)`.
*   Update `pwd_hash` as `Argon2id(new_password, salt=pwd_random_salt, tuned params)`.

Consider also prompting for the password at wallet access with appropriate timeout and lockout policies to balance usability with security.
