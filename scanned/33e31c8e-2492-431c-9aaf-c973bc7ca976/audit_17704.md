# [M-12] Incorrect Adopted mapping on updating wrapper token

## Summary
This bug report is about the AssetFacet.sol contract in the Connext project. It was submitted by codexploder and concerns the setWrapper function. This function allows an admin to set up a new wrapper Y instead of the old wrapper X. This becomes a problem for any old asset which was setup during setupAsset call as `s.canonicalToAdopted[_canonical.id]` will still point to old wrapper X instead of Y.

To fix this bug, codexplder suggested that if the wrapper is changed, all variables storing this wrapper should also update. This was confirmed by jakekidd from Connext who pointed out that the line in `setupAsset` suggests that the wrapper for `canonicalToAdopted` should be set. However, this fix won't be straightforward as another property will have to be added to tell us the canonical ID(s?) that are currently pointing to the wrapper contract as the adopted asset. 0xleastwood from the judge team also agreed that this bug would cause compatibility issues and the admin will be unable to call `setupAsset` and update the adopted mapping correctly.
