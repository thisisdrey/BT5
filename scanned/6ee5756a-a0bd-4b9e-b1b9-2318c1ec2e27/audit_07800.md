# Signature replay with the same fee value

## Summary
This bug report is about a problem with the ChainportMainBridge and ChainportSideBridge functions. Currently, the fee calculation is delegated to an offchain system, but this process does not include a unique identifier called a nonce. This means that the same signature can be used multiple times, which could lead to issues such as replay of the same signature or human error in fee validation. The severity of this issue is considered low, as the impact is limited and only lasts until a certain deadline. The recommendation is to either add nonce validation or provide onchain fee validation to prevent these risks. The Chainport team has verified that the signature mechanism already takes care of some issues, but they are willing to accept the risk of potential replay in the short time frame. The auditors also find it worth mentioning in the report.
