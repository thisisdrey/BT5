# M-34: `liquidationType2()` will always due wrong assumption that Asset:sAsset are minted 1:1

## Summary
The bug report is about a problem with the `liquidationType2()` function in the `borrowLiquidation.sol` contract. The issue was found by three individuals, but there has been no response to the report. The root cause of the problem is that the developer assumed that ETH and sETH are minted in a 1:1 ratio, but this is not the case. This leads to an incorrect amount of sETH being received when the `wrapper::mint()` function is called, and when `synthetix.exchange()` is called, it tries to transfer more sETH than it actually has. This can lead to a denial of service attack on the `liquidationType2()` function. The report suggests caching the actual balance of sETH received from the `wrapper` contract and using it to call `synthetix.exchange()` instead.
