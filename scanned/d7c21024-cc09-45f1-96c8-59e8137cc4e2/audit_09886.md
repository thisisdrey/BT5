# Missing Range Check For Duration

## Summary
The vulnerability in this report is found in the functionality responsible for creating future subscriptions for consumers. There is a validation check for the duration range, but if a certain condition is met, the check is not performed, allowing consumers to create subscriptions with any duration. This causes downstream calculation issues, which can be fixed by ensuring the duration value is within a reasonable range. The issue has been patched in a recent pull request.

## Details
## Vulnerability Details

The vulnerability is rooted in the functionality responsible for creating a future subscription for a given consumer. Within `CreateSubscription`, there is a validation check for the duration range (between zero and twelve) via `ValidationSubscription`. However, if `AdvancePurchase` is set to true when buying a subscription, `CreateFutureSubscription` is invoked, which does not perform this range check, thus allowing consumers to create future subscriptions with any duration.

```go
// subscription/keeper/msg serverbuy.go
func (k msgServer) Buy(goCtx context.Context, msg *types.MsgBuy) (*types.MsgBuyResponse, error) {
    [...]
    if msg.AdvancePurchase {
        err = k.Keeper.CreateFutureSubscription(ctx, msg.Creator, msg.Consumer, msg.Index, msg.Duration)
        [...]
        utils.LogLavaEvent(ctx, logger, types.AdvancedBuySubscriptionEventName, details, "advanced subscription purchased")
    }
    [...]
}
```

The issue causes downstream calculation issues such as casting to `int64(duration)` for calculation in the following places.

```go
// subscription/keeper/subscription.go
func (k Keeper) CreateFutureSubscription(ctx sdk.Context, ...) error {
    [...]
    newPlanPrice := plan.GetPrice()
    newPlanPrice.Amount = newPlanPrice.Amount.MulRaw(int64(duration))
    k.applyPlanDiscountIfEligible(duration, &plan, &newPlanPrice)
    [...]
}
```

```go
// subscription/keeper/subscription.go
creditReward := sub.Credit.Amount.QuoRaw(int64(sub.DurationLeft))
fmt.Println("advance month advance month advance month advance month advance month advance month", creditReward)
```
