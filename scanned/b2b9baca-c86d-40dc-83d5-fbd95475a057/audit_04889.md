# Error in preview rate calculation results in users losing rewards or offers not being allocated

## Summary
This bug report is about a function called previewRateAfterDeposit() in a code file called WrappedVault.sol. The function is supposed to calculate the rate a user would receive in rewards after depositing assets. However, there is an error in the code that causes the function to return a zero rate for an active reward period and a non-zero rate before the period starts. This means that an attacker can exploit this error to allocate a user's assets to the incentivized vault even though the reward period has not started yet. This can result in the user not receiving any rewards at all. Additionally, the function also fails to allocate a user's offer to the vaults even though the reward period has started and the rate is high enough. The bug has been fixed in a recent commit and confirmed by the developers. To prevent this bug from happening again, the recommendation is to change the check in the code to be more precise.
