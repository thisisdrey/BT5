# [M-18] Return values of `approve()` not checked

## Summary
This bug report is about the `IERC20` implementations that do not `revert()` when there is a failure in `approve()`. The function signature has a `boolean` return value, which should be checked to indicate errors. When not checked, operations that should have marked as failed, may potentially go through without actually approving anything.

There are 5 instances of this issue present in the contracts `UniV2LiquidityAmo.sol`, `UniV3LiquidityAmo.sol` and `RdpxV2Core.sol`. The locations of these instances in the code are provided in the report.

The bug was reported via the Automated Findings report, but was declared out of scope for the audit. However, it is being included here for completeness.

The report further states that this issue can be argued to be medium although in most cases the transaction would revert later.

## Details
_Note: this finding was reported via the winning [Automated Findings report](https://gist.github.com/code423n4/3c6507b76ca940e11c287862cb4a1fd3). It was declared out of scope for the audit, but is being included here for completeness._

Not all `IERC20` implementations `revert()` when there's a failure in `approve()`. The function signature has a `boolean` return value and they indicate errors that way instead. By not checking the return value, operations that should have marked as failed, may potentially go through without actually approving anything.
