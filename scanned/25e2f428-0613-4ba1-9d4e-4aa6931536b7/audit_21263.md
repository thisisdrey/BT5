# Too open rebalance method

## Summary
This bug report is about a potential security vulnerability in the ExperiPie project, as found in the RSIManager.sol contract. The vulnerability exists because the caller account has no restrictions, which means it can be manipulated by malicious actors. To fix this, the report recommends adding restrictions to the caller account. This is important, as it will help protect the ExperiPie project from potential manipulation of the market.

## Details
##### Description
This is potentially dangerous because of potential manipulations on the market: https://github.com/pie-dao/ExperiPie/blob/0.0.2/contracts/callManagers/RSIManager.sol#L50

##### Recommendation
We suggest adding some restrictions for the caller account.
