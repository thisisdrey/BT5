# Incorrect Pool Configuration

## Summary
The Invert field in the PoolConfig structure is used to indicate whether the base and quote currencies should be swapped when processing the pair. This is important for correctly interpreting the price data. If Invert is set incorrectly, it can result in misinterpreting the price direction and lead to incorrect price calculations and potentially significant financial discrepancies. In this case, the Invert field should be set to false to fix the issue. The bug has been fixed in PR #469.

## Details
## The Invert Field in PoolConfig Structure

The **Invert** field in the `PoolConfig` structure is used to indicate whether the base and quote currencies should be swapped when processing the pair. This is crucial for correctly interpreting the price data. If **Invert** is set incorrectly, it will result in misinterpreting the price direction, leading to incorrect price calculations and potentially significant financial discrepancies. 

Thus, when the **Invert** field is set to true, it signals that the pair should be read as **ETH/DAI** instead of **DAI/ETH**, which is incorrect in this case since the pool address corresponds to a **DAI/ETH** pair.

> _whirlpool/src/util/token.rs_

```rust
var (
    [...]
    DefaultETHMarketConfig = types.CurrencyPairsToProviderTickers{
        constants.DAI_ETH: {
            OffChainTicker: constants.DAI_ETH.String(),
            JSON: PoolConfig{
                Address: "0xC2e9F25Be6257c210d7Adf0D4Cd6E3E881ba25f8",
                BaseDecimals: 18,
                QuoteDecimals: 18,
                Invert: true,
            }.MustToJSON(),
        },
        [...]
    }
)
```

## Remediation

Set **Invert** to false.

## Patch

Fixed in PR #469.
