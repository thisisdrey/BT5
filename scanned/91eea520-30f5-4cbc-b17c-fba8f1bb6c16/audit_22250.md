# Unnecessary complexity calculating token amount

## Summary
A bug was reported in the formula used to calculate the token amount in line 116 of the GitHub repository "erkmos/moeda". It was suggested that the formula could be replaced by a multiplication of the amount of tokens bought with one wei for each given tier. The token creation rates could be replaced by one ether divided by the current values, which would have a number of benefits including exact numbers with no precision lost, less risk of overflow, and simpler code. The bug has since been fixed in the GitHub repository "erkmos/moeda" in the commit a894b2cccef0236e90326b84a0b3dde485e78cd1.

## Details
The formula used to calculate the token amount in [line 116](https://github.com/erkmos/moeda/blob/b2bf23119d563e251b6f16b29b642bac43e76a64/contracts/Crowdsale.sol#L116-L117) is unnecessarily complicated. It could be replaced by a multiplication by the amount of tokens bought with one [wei](https://forum.ethereum.org/discussion/304/what-is-wei) for each given tier. Consider making this change by replacing [the token creation rates](https://github.com/erkmos/moeda/blob/b2bf23119d563e251b6f16b29b642bac43e76a64/contracts/Crowdsale.sol#L28-L30) by `1 ether` divided by the current values. This has a number of benefits:


1. Numbers are exact and there is no precision lost (current version has loss of precision due to chained integer operations).
2. There is less risk of overflow.
3. It’s simpler, which is always preferable. When code is irreducibly complex, consider using temporary variables to improve readability.


**Update**: Fixed in <https://github.com/erkmos/moeda/commit/a894b2cccef0236e90326b84a0b3dde485e78cd1>.
