# Incorrect rounding direction in SuperVault.convertToAssets()

## Summary
This bug report discusses a problem with the `convertToAssets()` function in the `SuperVault` contract. According to the EIP-4626 specification, the function should round down, but the implementation currently rounds up. This could lead to incorrect calculations and potential security issues. The report recommends modifying the function to round down and also adjusting the `previewMint()` function to continue rounding up.

## Details
## Security Analysis Report

## Severity
**Medium Risk**

## Context
**File:** SuperVault.sol  
**Lines:** 329-333

## Description
According to the EIP-4626 specification, the `convertToAssets()` function should round down. Specifically, it states that `convertToAssets` **MUST** round down towards 0. 

However, the implementation in `SuperVault.convertToAssets()` rounds up. Below is the problematic code:

```solidity
function convertToAssets(uint256 shares) 
    public 
    view 
    override 
    returns (uint256) 
{
    uint256 currentPPS = _getStoredPPS();
    if (currentPPS == 0) return shares;
    return Math.mulDiv(shares, currentPPS, PRECISION, Math.Rounding.Ceil);
}
```

## Recommendation
Modify `convertToAssets()` to round down. Additionally, the `previewMint()` function must be modified to continue rounding up, as it currently calls `convertToAssets()`.
