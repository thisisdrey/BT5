# [M04] Mismatches between contracts and interfaces

## Summary
This bug report is related to the [`Option`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Option.sol) and [`Redeem`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Redeem.sol) contracts, which have corresponding interfaces [`IOption`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/interfaces/IOption.sol) and [`IRedeem`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/interfaces/IRedeem.sol). 

The bug report states that the `IRedeem` interface is missing the `initialize` function from the `Redeem` contract, and the `IOption` interface is missing the `take`, `kill` and `update` functions from the `Option` contract. Additionally, the definition of the `getParameters` function in the `IOption` interface does not match its counterpart implementation in the `Option` contract, which returns the first two values in the wrong order. Finally, there is an inconsistency between line 22 in `ITrader.sol` and line 82 in `Trader.sol`. This can potentially lead to unexpected errors and loss of funds.

## Details
The [`IOption`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/interfaces/IOption.sol) and [`IRedeem`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/interfaces/IRedeem.sol) interfaces define the exposed functionality of the [`Option`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Option.sol) and [`Redeem`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Redeem.sol) contracts respectively. However, in both interfaces there are functions from the counterpart contracts that are not defined, as listed below.


The `IRedeem` interface is missing:


* The [`initialize` function](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Redeem.sol#L30).


The `IOption` interface is missing:


* The [`take` function](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Option.sol#L102).
* The [`kill` function](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Option.sol#L83).
* The [`update` function](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Option.sol#L91).


Moreover, the definition of the `getParameters` function in the [`IOption` interface](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/interfaces/IOption.sol#L68-L69) does not match its counterpart implementation [`getParameters`](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Option.sol#L409-L410) of the `Option` contract, which returns the first two values in the wrong order. This can potentially lead to unexpected errors and loss of funds, since anyone that uses the `IOption` interface will expect to receive token addresses in a different order.


Finally, [line 22](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/interfaces/ITrader.sol#L22) in `ITrader.sol` does not match with the returned values in [line 82](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/extensions/Trader.sol#L82) of `Trader.sol`.


Consider applying the necessary changes in the mentioned interfaces and contracts so that definitions and implementations fully match.


**Update:** *Fixed in [PR #23](https://github.com/primitivefinance/primitive-protocol/pull/23). Suggested changes have been made.*
