# The method `multiTransfer(...)` fails to transfer tokens to recipients

## Summary
The bug report is about a high severity issue in a contract called DevvE.sol. The problem occurs in a method called multiTransfer() which is supposed to transfer tokens to different recipients. However, the transfer fails because of a line of code that uses the transferFrom() method. This method is not able to transfer tokens even if the sender has given permission to the DevvE contract to do so through the approve() method. This is because the spender and owner of the tokens become the same, which is the msg.sender. The recommendation is to modify the transferFrom() method to use this.transferFrom() instead. This should resolve the issue.

## Details
**Severity**: High

**Status**: Resolved

**Description**

In Contract DevvE.sol, the method multiTransfer(...) fails to transfer tokens to recipients due to the following line:
```solidity
           transferFrom(msg.sender, recipients[i], amounts[i]);
 ```
Here even if `msg.sender` has approved the DevvE contract to transfer tokens on behalf of the `msg.sender` using the approve(...) method although this reverts as using transferFrom(...) directly makes the spender and owner as the `msg.sender` itself.

**Recommendation**: 
Modify the transferFrom(...) method as below:
```solidity
           this.transferFrom(msg.sender, recipients[i], amounts[i]); 
```
