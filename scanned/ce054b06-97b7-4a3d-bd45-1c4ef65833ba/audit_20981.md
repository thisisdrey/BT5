# Potential `safeApprove` blocking

## Summary
This bug report concerns the BondAppetit Protocol, a set of contracts used to enable transactions. At several places in the code, a function called “safeApprove” is used before calling the corresponding Uniswap function. This function requires a zero allowance, but if Uniswap doesn't use the full allowance provided, it can lead to blocking the “safeApprove” call. This issue is present in lines 182, 248, 125, 195, 204, 116, 124, 125, 151, 152, and 181 of the BondAppetit Protocol code.

The recommendation is to always reset the allowance to zero by calling “safeApprove” with an amount of zero. This will ensure that the “safeApprove” call is not blocked and the Uniswap function can be called successfully.
