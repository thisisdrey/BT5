# Require function can't throw exceptions in UnmarshalTokenVesting

## Summary
In this bug report, it is mentioned that there is a problem with the internal function _addDistribution. The issue is that all distributions, including tokens allotment and beneficiary address, are declared when the contract is deployed. This means that the require statement in the code cannot throw exceptions for the conditions _tokensAllotment > 0 and distribution.tokensAllotment == 0. The recommendation is to remove these exceptions.

## Details
**Description**

Function _addDistribution is internal. All distributions are declared when deploying the
contract including tokens allotment and beneficiary address, so require can’t throw exceptions
for _tokensAllotment > 0 and distribution.tokensAllotment == 0.

**Snippet**:
```solidity
require(_tokensAllotment > 0, "the investor allocation must be more than 0");
require(distribution.tokensAllotment == 0, "investor already added");
```
**Recommendation**:
Remove those exceptions.
