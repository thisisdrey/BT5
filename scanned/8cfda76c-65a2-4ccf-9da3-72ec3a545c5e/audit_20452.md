# Refresh _lgeTimeStamp {in case createLGEWhitelist is supposed to be called more, than once}

## Summary
The bug report is about a specific line of code (397) in a file called BEP20TokenWhitelisted.sol. The function in question is called createLGEWhitelist, and there is a variable called _lgeTimeStamp that should be set to zero. This is important because it affects the function _applyLGEWhitelist, which is responsible for sending tokens to a specific address (_lgetPairAddress). The recommendation is to make sure that _lgeTimeStamp is always set to zero when the function is refreshed.

## Details
**Description**

BEP20TokenWhitelisted.sol, line 397, createLGEWhitelist function
_lgeTimeStamp should be set to zero, so that next time tokens are sent to _lgetPairAddress,
_lgeTimeStamp would be reassigned in _applyLGEWhitelist function.

**Recommendation**:

Assign _lgeTimeStamp to 0 every time round are refreshed inside createLGEWhitelist.
