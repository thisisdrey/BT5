# Incorrect calculation of `rewardPerLiquidity`

## Summary
This bug report is about an issue with the StakingRewardsV3-1.sol contract. If the first user deposits a Non-Fungible Token (NFT) after some time from the `notify()` call, then the calculation of `(lastTimeRewardApplicable() - lastUpdateTime)` will be less than the set `DURATION` which will cause some funds to freeze on the contract. The recommended solution is to change the calculation of `rewardPerLiquidity` to fix this issue.

## Details
##### Description

We recommend to change the calculation of `rewardPerLiquidity`.
