# OracleV1.getMemberReportStatus returns true for non existing oracles

## Summary
This bug report details an issue with the Oracle.1.sol file, specifically lines 115 to 118. The issue is that the memberIndex variable will be equal to -1 for non-existing oracles, which will cause the mask to be equal to 0 and the function to return true for non-existing oracles. This could allow a not existing member to vote, which could lead to a Denial of Service (DOS) attack.

The recommendation is to either change the function to return false for memberIndex = -1, or to always check if the member does not exist by checking if memberIndex >= 0. If the latter is chosen, the code should also be documented in the natspec comment.

The bug was fixed in SPEARBIT/12 by returning false for non-existing oracles, and Spearbit acknowledged the fix.

## Details
## Severity Report

**Severity:** Medium Risk  
**Context:** `Oracle.1.sol#L115-L118`  

## Description
`memberIndex` will be equal to `-1` for non-existing oracles, which will cause the mask to be equal to `0`, leading to the function returning `true` for non-existing oracles.

## Recommendation
Consider changing the function to return `false` for `memberIndex = -1`. However, bear in mind that if this function is directly used in any part of the logic, it could allow a non-existing member to vote. The best solution is to always check if the member does not exist by verifying if `memberIndex >= 0`. 

Alternatively, the function could revert if the member does not exist and return `true`/`false` if the member does exist and has voted/not voted. Be cautious that this choice, if integrated directly into the code, could create a Denial of Service (DOS) scenario if not handled properly.

For all these reasons, consider documenting the behavior of the function and the possible side effects in the natspec comment.
