# Assignment instead of comparison in UniMexFactory.sol require checks

## Summary
The bug report is about a mistake in a contract called UniMexFactory.sol. There are two functions, setUtilizationScaled and setMaxLeverage, where the require checks are incorrect. Instead of comparing a value, they are assigning it. For example, at line 45, the value "true" is being assigned to a variable called "allowed" instead of being compared. The recommendation is to change the assignment to a comparison. The bug has been resolved.

## Details
**Description**

In contract UniMexFactory.sol in functions setUtilizationScaled and setMaxLeverage the require checks at lines 45 and 51 are assigning a value instead of comparing it. For example at line 45 the value true is assigned to allowed. require(allowed[_token] = true);

**Recommendation**

Change assignment to comparison.
