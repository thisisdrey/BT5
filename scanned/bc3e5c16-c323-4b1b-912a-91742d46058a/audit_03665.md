# Locking of Usual tokens of integrators by bypassing redirection ﬂow 

## Summary
The DistributionModule's claimOffChainDistribution function allows anyone to initiate the distribution of Usual tokens to eligible accounts. However, there is a bug in the redirection logic that allows anyone to monitor and call the function just before the redirection is activated, leading to the tokens being minted to the integrator's account instead of the intended recipient. This bug affects ~69M Usual tokens and can result in locking and loss of tokens for the integrator. To fix this, the public access to the claimOffChainDistribution function should be disabled when a redirection is initiated for an account. This bug has been acknowledged by both Usual and Cantina Managed.

## Details
## DistributionModule.sol - Vulnerability Report

## Context
Location: `DistributionModule.sol#L437`

## Description
The `DistributionModule.claimOffChainDistribution` is a publicly accessible function that allows anyone to initiate the Usual token distribution for any eligible account. 

The redirection logic has been implemented to reroute the Usual rewards of integrators to a different address in case the integrator's native address cannot handle Usual rewards. 

Anyone can monitor the `initiatedRedirectedOffChainDistribution` and `initiatedRedirectedOffChainDistributionStartingTimestamp` states for accounts and call the `claimOffChainDistribution` function just before the redirection can be activated for an integrator, rendering the entire redirection logic ineffective. As a result, the Usual tokens will be minted to the integrator's account instead of the redirected recipient, leading to the locking and loss of Usual tokens for the integrator.

Currently, there are approximately 69 million Usual tokens eligible and waiting to be claimed as off-chain distribution rewards. All Usual tokens of integrators that are planned to be redirected are susceptible to this attack.

## Recommendation
One way to resolve this issue is to disable the public access of the `claimOffChainDistribution` function for an account when a redirection is initiated for that account. Once the redirection is activated, the public accessibility of the `claimOffChainDistribution` function for that account can be re-enabled.

```solidity
function claimOffChainDistribution(address account, ...) {
    if ($.initiatedRedirectedOffChainDistribution[account] != address(0)) revert();
    // ...
}
```

Please note that anyone can still front-run the `redirectOffChainDistribution` call (which sets the `initiatedRedirectedOffChainDistribution` state) and initiate the claim for integrators.
