# Payable function without interaction with msg.value.

## Summary
The bug report is about a function called 'mint' in the SqwidMarketplace.sol file. This function is marked as 'payable', which means it is supposed to interact with cryptocurrency. However, there is no interaction with the cryptocurrency and any REEF that is accidentally sent with this function will be locked in the contract. The recommendation is to either remove the 'payable' modifier or add interaction with the cryptocurrency by updating the user's balance with the '_updateBalance()' function. It is mentioned that the team has already removed the 'payable' modifier.

## Details
**Description**

 SqwidMarketplace.sol: function mint().
 Function is marked as payable, however there is no interaction with msg.value. All REEF, 
accidentally sent with this function will be locked on contract.
