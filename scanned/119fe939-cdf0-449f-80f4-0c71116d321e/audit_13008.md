# Index returned overwrites an existing element

## Summary
This bug report discusses a medium severity bug that has been resolved. The bug was found in the TradingStorage.sol file and affected the methods "firstEmptyTradeIndex" and "firstEmptyOpenLimitIndex". These methods would return an index of 0 if all the array slots were filled, causing the first element to be overwritten. The recommendation is to revert the process if all the open limit slots are full. The bug has been fixed in the latest commit by adding a variable to check if all slots are occupied before proceeding. The issue has been resolved.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In TradingStorage.sol - Methods: firstEmptyTradeIndex, firstEmptyOpenLimitIndex() returns index = 0 if all the array slots are occupied. This in turn overrides the first element (i.e. 0).

**Recommendation** 

Revert if all occupied (i.e. 3 by default) open limit slots are full.

**Fix** -   As of  commit a72e06b ,  A variable allOccupied is added in order to revert if all slots are full. Therefore issue is resolved.
