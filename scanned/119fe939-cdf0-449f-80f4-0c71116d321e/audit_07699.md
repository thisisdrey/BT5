## Summary
The bug report is about a problem in the `SparseMerkleProof` contract, specifically in the `verifyProof` function. The function is supposed to check if the length of the raw Merkel proof is the same as the Merkel tree depth, which is 42. However, there is no check for this and if the length is less than 42, the function will fail and cause an error. The report includes examples of the code where the bug can be found and a recommendation to add a require statement to ensure the length of the raw proof is 42 in the function. The bug has been fixed in a pull request.

## Details
#### Resolution



 The problem has been fixed in the [PR\#179](https://github.com/Consensys/linea-resolver/pull/179).
 

#### Description


In the function `verifyProof` of the `SparseMerkleProof` contract, the length of the raw Merkel proof(`_rawProof`) should be same as the Merkel tree depth (TREE\_DEPTH 42\). However, there is no check of the length. If the length is less than 42, the function `_verify` would fail on an out of bounds of array access error:



```
for (uint256 height; height < TREE_DEPTH; ++height) {
            if ((currentIndex >> height) & 1 == 1)
                computedHash = Mimc.hash(abi.encodePacked(_proof[height], computedHash)
            );

```

#### Recommendation


Add a require statement to verify that the length of `_rawProof` is 42 in the `verifyProof` function.
