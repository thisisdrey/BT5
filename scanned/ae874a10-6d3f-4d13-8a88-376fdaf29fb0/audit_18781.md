# Check zeroParams and lpTokenParams Is Not Empty

## Summary
This bug report is about the PoolManager.sol#L158-197. The function addTarget() checks targetParams is not empty, however, addSeries() does not check for that and zeroParams and lpTokenParams are not empty. This means that anyone can call addSeries() and there is a possibility of a griefing attack. The bug report recommends to verify that zeroParams and lpTokenParams are not empty within the function addSeries(). The bug has been addressed in #156 and acknowledged.

## Details
## Security Analysis Report

## Severity
**High Risk**

## Context
`PoolManager.sol#L158-197`

## Situation
The function `addTarget()` checks that `targetParams` is not empty. However, `addSeries()` does not include a check for this, and both `zeroParams` and `lpTokenParams` are not verified to be empty. Since these parameters are set through `setParams()`, they may not be initialized. This creates a vulnerability as anyone can call `addSeries()`, leading to a potential griefing attack.

### Function Breakdown

```solidity
function addTarget(address target, address adapter) external requiresTrust returns (address cTarget) {
    ...
    require(targetParams.irModel != address(0), Errors.TargetParamNotSet);
    ...
    bytes memory constructorData = abi.encode(
        ...
        targetParams.irModel,
        ...
    );
    uint256 err = ComptrollerLike(comptroller)._deployMarket(false, constructorData, targetParams.collateralFactor);
    ...
}
```

```solidity
function addSeries(address adapter, uint48 maturity) external {
    ... // no checks on zeroParams
    bytes memory constructorDataZero = abi.encodePacked(
        ...
        zeroParams.irModel,
        ...
    );
    uint256 errZero = ComptrollerLike(comptroller)._deployMarket(false, constructorDataZero, zeroParams.collateralFactor);
    
    ... // no checks on lpTokenParams
    bytes memory constructorDataLpToken = abi.encodePacked(
        ...
        lpTokenParams.irModel,
        ...
    );
    uint256 errLpToken = ComptrollerLike(comptroller)._deployMarket(false, constructorDataLpToken, lpTokenParams.collateralFactor);
    ...
}
```

## Recommendation
Verify that `zeroParams` and `lpTokenParams` are not empty within the function `addSeries()`.
