# Lack of access control for FeeProvider setter functions

## Summary
This bug report is about a timing issue in the LendingPool.sol contract. It states that the setFeesCollectionAddress and setLoanOriginationFeePercentage functions in the FeeProvider contract can be called by anyone. This allows an attacker to modify the loan origination fee percentage and the address to which the fees are sent. 

The exploit scenario involves an attacker changing the feesCollectionAddress of the FeeProvider used by a LendingPool by calling the setFeesCollectionAddress function. As a result, all collected fees are now sent to their address instead of the original one. 

The recommendation is to protect the setter operations from being called by unexpected addresses, such as the setFeesCollectionAddress and setLoanOriginationFeePercentage in the FeeProvider contract. This can be done using an Ownable contract and setting the owner to a multi-signature wallet, which will avoid a single point of failure in the case of a private-key compromise. In the long term, a governance system should be implemented to agree on changes.
