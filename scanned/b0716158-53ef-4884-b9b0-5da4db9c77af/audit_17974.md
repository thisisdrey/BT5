# [H-01] User can forfeit other user rewards

## Summary
A bug has been identified in the ExtraRewardsDistributor.sol#L127 contract of the Aura Finance protocol. The bug allows a user to forfeit other user's rewards by providing a higher _startIndex in the getReward function. This is possible because the _allClaimableRewards function at ExtraRewardsDistributor.sol#L213 computes the epochIndex as 5>0?5:0 = 5, even if the user has not received any reward yet. This means that the user will be rewarded from epoch 5 to epoch 7, even though they should only be rewarded from epoch 5 to 7.

The severity of this bug was initially disputed, with 0xMaharishi (Aura Finance) disagreeing with the high risk rating. However, LSDan (the judge) commented that a third party could cause significant loss of expected reward funds for users across the entire protocol if so inclined.

The bug has since been resolved by 0xMaharishi (Aura Finance) with the help of code4rena, through the pull requests aurafinance/aura-contracts#84 and code-423n4/2022-05-aura#6.

## Details
_Submitted by csanuragjain_

[ExtraRewardsDistributor.sol#L127](https://github.com/code-423n4/2022-05-aura/blob/main/contracts/ExtraRewardsDistributor.sol#L127)<br>

User can forfeit other user rewards by giving a higher \_startIndex in getReward function.

### Proof of Concept

1.  Assume User B has not received any reward yet so that his userClaims\[\_token]\[User B]=0
2.  User A calls getReward function with \_account as User B and \_startIndex as 5
3.  This eventually calls \_allClaimableRewards at ExtraRewardsDistributor.sol#L213 which computes epochIndex =5>0?5:0 = 5
4.  Assuming tokenEpochs is 10 and latestEpoch is 8, so reward will computed from epoch 5 till epoch index 7 and \_allClaimableRewards will return index as 7
5.  \_getReward will simply update userClaims\[\_token]\[User B] with 7
6.  This is incorrect because as per contract User B has received reward from epoch 0-7 even though he only received reward for epoch 5-7

### Recommended Mitigation Steps

Do not allow users to call getReward function for other users.
