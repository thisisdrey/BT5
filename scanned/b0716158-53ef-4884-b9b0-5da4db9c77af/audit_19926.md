# [H-03] transferNotionalFrom doesn’t check from != to

## Summary
In this bug report, gpersoon has identified a vulnerability in the function transferNotionalFrom of VaultTracker.sol. This vulnerability allows users to increase the balance of both the "from" and "to" address without decreasing any balances, effectively printing money. This function can be called via transferVaultNotional by everyone, making it possible for anyone to exploit this bug.

The Proof of Concept provided shows that the bug is caused by the fact that the same address is used for both the "from" and "to" variables, and thus the balance of "from" is overwritten by the balance of "to".

The recommended mitigation step for this bug is to add a requirement that the "from" and "to" addresses must be different. This will ensure that the bug can no longer be exploited.

## Vulnerability details

## Impact
The function transferNotionalFrom of VaultTracker.sol uses temporary variables to store the balances.
If the "from" and "to" address are the same then the balance of "from" is overwritten by the balance of "to".
This means the balance of "from" and "to" are increased and no balances are decreased, effectively printing money.

Note: transferNotionalFrom can be called via transferVaultNotional by everyone.
