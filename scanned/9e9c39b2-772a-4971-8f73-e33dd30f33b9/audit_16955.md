# NFT withdrawal vulnerability in MarinateV2.sol.

## Summary
The bug report states that there is an issue with the MarinateV2.sol code. When a user stakes their NFT with a certain token ID, while another user stakes the same NFT with a different token ID, the first user is able to withdraw the second user's token ID. This results in a loss of funds for the second user. The recommendation is to keep track of the token IDs of the NFTs to prevent this issue. The bug has been resolved by removing parts of the code related to NFTs.

## Details
**Description**

MarinateV2.sol - withdraw Multiplier & stake Multipler, marinator stakes NFT with tokenld $=x$ while another marinator stakes tokenld = yof same NFT. The former marinator is able to withdraw the tokenId=y. NFT tokenlds are not holding same value, therefore that leads to loss of funds for some stakers.

**Recommendation**

Keep track of token Ids of NFTs

Resolved.
