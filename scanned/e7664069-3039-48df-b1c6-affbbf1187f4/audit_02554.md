# Deploying New Jetton Contracts Requires Non-Bouncable Message

## Summary
The client has marked a bug as fixed in the `financial.fc` file. The issue was in the `mint()` function, where transferring to a user's Jetton wallet could create the wallet if it didn't already exist. However, according to TON documentation, creating new accounts is not allowed with a bouncable message. The recommendation is to make this a `NON-BOUNCABLE` message.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `1b6ce8b4d7d210bad51c00730ee2c98a070f1dae`.

**File(s) affected:**`financial.fc`

**Description:** In the `mint()` function, the transfer to the user Jetton wallet may create this wallet if it does not exist yet. However, as per TON [documentation](https://ton-blockchain.github.io/docs/#/howto/smart-contract-guidelines?id=using-non-bounceable-messages), creating new accounts cannot be done with a bouncable message.

**Recommendation:** We recommend making this a `NON-BOUNCABLE` message.
