# [M-05] Incorrect amount taken

## Summary
A bug was recently discovered in the code of the Lending Market's CNote.sol contract. It was observed that in the repayBorrowFresh function, the contract asks the user to send the repayAmount instead of the repayAmountFinal. This can lead to the user potentially losing funds as they could be sending more than required.

The proof of concept for this bug is as follows:

1. The user makes a repayment which calls the repayBorrowFresh function. 
2. Assuming the repayAmount is equal to the maximum value of its type (uint).max, the repayAmountFinal then becomes equal to the accountBorrowsPrev. 
3. This means that the user should only transfer the accountBorrowsPrev instead of the repayAmount, but the contract is instead transferring the repayAmount instead of the repayAmountFinal. 

The recommended mitigation steps for this bug are to revise the CNote.sol#L129 code to use the actualRepayAmount instead of the repayAmount. Alex the Entreprenerd (judge) decreased the severity of the bug to Medium and commented that, due to an oversight, using the type(uint).max to signify a complete repayment will attempt to transfer 2^256-1 units of token. Since a revert will occur instead of the tokens being transferred, a Medium severity is more appropriate. The remediation for this bug requires either using the actualRepayAmount or re-assigning the value of repayAmount.
