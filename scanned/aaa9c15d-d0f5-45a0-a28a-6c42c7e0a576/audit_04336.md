# Incorrect Chain ID Values in opera.ts and params.json

## Summary
This bug report discusses an issue with the chain IDs in the `opera.ts` and `params.json` files in the Opera-Bridge repository. The `sonicChainId` in `opera.ts` is set to the wrong value of 250 instead of the correct value of 64165 for the Sonic test network. Similarly, the `operaChainId` in `params.json` is also incorrect. This can cause deployment failures and incorrect chain interactions. The report suggests updating the chain IDs to the correct values and mentions that the issue has been resolved in a recent pull request. The Sonic team also confirms that a new Chain ID will be used for the Sonic chain and the deployment configuration will need to be updated again before deployment.

## Details
In [`opera.ts`](https://github.com/Fantom-foundation/Opera-Bridge/blob/730e10b341c98ead97c14befd704c03db951fe86/ignition/modules/opera.ts), the [`sonicChainId`](https://github.com/Fantom-foundation/Opera-Bridge/blob/730e10b341c98ead97c14befd704c03db951fe86/ignition/modules/opera.ts#L8) parameter is incorrectly set to 250 (`0xFA`) instead of the correct value 64165 (`0xFAA5`) for the Sonic test network. Similarly, in the [`params.json`](https://github.com/Fantom-foundation/Opera-Bridge/blob/730e10b341c98ead97c14befd704c03db951fe86/ignition/params.json) file, the [`operaChainId`](https://github.com/Fantom-foundation/Opera-Bridge/blob/730e10b341c98ead97c14befd704c03db951fe86/ignition/params.json#L9) parameter is set to 64165 (`0xFAA5`) instead of the correct value 250 (`0xFA`). This discrepancy can cause deployments to fail or result in the contract interacting with the wrong chain.


Consider ensuring that the chain IDs are consistent and accurate at the moment of deployment. In addition, consider updating the [`sonicChainId`](https://github.com/Fantom-foundation/Opera-Bridge/blob/730e10b341c98ead97c14befd704c03db951fe86/ignition/modules/opera.ts#L8) in `opera.ts` to 64165 (`0xFAA5`) if using the test net and then to the correct Sonic mainnet chain ID when it is launched. Similarly, also change the [`operaChainId`](https://github.com/Fantom-foundation/Opera-Bridge/blob/730e10b341c98ead97c14befd704c03db951fe86/ignition/params.json#L9) in `params.json` to 250 (`0xFA`) in order to avoid misconfiguration and deployment failures.


***Update:** Resolved in [pull request \#9](https://github.com/Fantom-foundation/Opera-Bridge/pull/9) at commit [dad2148](https://github.com/Fantom-foundation/Opera-Bridge/pull/9/commits/dad2148517c1c1567ef27259d1e51eae6c0541ff). The Sonic team stated:*



> *A new Chain ID will be used for the Sonic chain. The `0xFAA5` is a temporary testnet. The deployment configuration will need to be updated again before the actual deployment on the target environment, so we did not update it yet. Yet, the current configuration was indeed incorrect and was updated accordingly by the linked pull request.*
