# Missing await in asynchronous validation functions 

## Summary
This bug report discusses an issue with the runTask and sendTask functions in a codebase. These functions use a method called verifyIntent to validate user-provided data, but the code does not properly handle the asynchronous execution of this method. This means that the validation process may not complete before the functions continue, potentially allowing unauthorized actions to occur. The report recommends using the await keyword to properly handle asynchronous validation functions and suggests auditing the codebase for similar issues. The bug has been fixed in the codebase.

## Description
The `runTask` and `sendTask` functions use the `verifyIntent` method to validate the integrity of user-provided data (`signableUserTasks`, `ethereumAddress`, and `tasksSignature`). However, the `verifyIntent` function returns a Promise, and the code does not include the `await` keyword to handle its asynchronous execution. This results in the validation logic continuing execution without waiting for the validation process to complete, potentially allowing actions to proceed without proper authorization.

Similarly, the `verifyQuote` function in the `runTask` function is asynchronous but is also missing the `await` keyword. This omission causes the function to proceed without verifying that the quote meets the expected parameters, leading to an improper validation flow.

## Recommendation
1. **Use `await` for asynchronous validation functions**: Ensure that both `verifyIntent` and `verifyQuote` are awaited in their respective calls so that validation completes before proceeding.
2. **Review all asynchronous validations**: Audit the codebase to ensure that all other asynchronous validation functions are properly awaited where applicable to avoid similar issues.
