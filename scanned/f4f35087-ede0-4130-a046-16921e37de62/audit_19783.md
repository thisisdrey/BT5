# [H-09] treasury is vulnerable to sandwich attack

## Summary
A user with the handle jonah1005 has identified a vulnerability in MochiTreasury, a permissionless function called veCRVlock. This vulnerability allows an attacker to launch a sandwich attack with a flashloan to steal funds from the treasury. This is considered to be a high-risk issue, as it could potentially allow an attacker to steal all of the funds in the treasury. The exploit pattern involves using a flashloan to buy CRV from the uniswap pool, triggering veCRVlock, then the treasury buying CRV at a high price, and finally selling CRV and paying back the loan. No tools were used in the identification of this vulnerability. The recommended mitigation step is to add an onlyOwner modifier to the function.
