# OPERATOR_ROLE can withdraw ETH when the protocol is paused 

## Summary
The bug report is about a function called `operatorWithdrawETHForUser()` in a file called Goldsand.sol. The function is supposed to prevent a specific role called `OPERATOR_ROLE` from withdrawing ETH when the protocol is paused. However, the function does not have a necessary modifier called `whenNotPaused`, which allows the `OPERATOR_ROLE` to still withdraw ETH even when the protocol is paused. The recommendation is to add the `whenNotPaused` modifier to the function to prevent this from happening. The bug has been fixed in a commit called `96b61cc1` and has been confirmed to be resolved.

## Details
## Context
**File:** Goldsand.sol#L324

## Description
When the protocol is paused, `OPERATOR_ROLE` and users should not be able to perform any actions. However, since the `whenNotPaused` modifier is not added to the `operatorWithdrawETHForUser()` function, `OPERATOR_ROLE` can withdraw ETH from the protocol when the protocol is paused.

## Recommendation
It is recommended to add the `whenNotPaused` modifier to the `operatorWithdrawETHForUser()` function.
