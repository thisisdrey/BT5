# No check of parameter values when executing smart contract constructor

## Summary
This bug report concerns the Pickle Finance Protocol codebase, which is available on GitHub. The bug was related to a missing requirement in the code that should have been appended after the constructor execution starts. Specifically, the requirement `require(variableAddress != address(0))` was missing from the following files: strategy-curve-base.sol, strategy-staking-rewards-base.sol, strategy-uni-farm-base.sol, crv-locker.sol, scrv-voter.sol, strategy-curve-scrv-v4.sol, and strategy-curve-scrv-v4_1.sol. Additionally, the bug was present in the pickle-jar.sol file for variables _token, _governance, _timelock, and _controller. The bug has now been fixed in the latest version of the codebase.

## Details
It is recommended to append `require(variableAddress != address(0))` requirement right after the constructor execution starts.
