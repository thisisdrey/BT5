# Inconsistencies In Contract State

## Summary
The bug report is about a variable called $.numExchangableNFT that is not being updated correctly when new NFTs are minted or existing ones are burned. This causes the contract's internal state to not accurately represent the number of exchangeable NFTs. Additionally, the fee is not being refreshed in both the _mint and _burn functions, leading to users potentially receiving an incorrect fee amount. The recommended solution is to ensure that the variable is updated properly whenever NFTs are minted or burned, and to also refresh the fee in a timely manner. The bug has been fixed by removing the _mint and _burn functions from the code base in version ed8ec93.

## Details
## Issue Description

In DN404, the variable `$.numExchangableNFT` remains unchanged when minting new NFTs or burning existing ones within `_mint` and `_burn` respectively. Consequently, when a user mints or burns NFTs, the contract’s internal state fails to represent the number of exchangeable NFTs accurately. Furthermore, the fee remains unrefreshed in both `_mint` and `_burn`, even though the `ownedLength` of users is updated. This creates a situation where users may receive an inaccurate fee amount.

## Remediation

Ensure that the `$.numExchangableNFT` variable is updated appropriately whenever NFTs are minted or burned. Additionally, make sure to refresh the fee on time.

## Patch

`_mint` and `_burn` have been removed from the code base in ed8ec93.
