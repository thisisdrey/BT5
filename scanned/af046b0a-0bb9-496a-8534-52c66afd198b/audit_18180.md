# The value of lastExecutionPrice fails to update if pool.poolUpkeep() reverts

## Summary
The PoolKeeper contract has an issue with the performUpkeepSinglePool() function. This function updates the executionPrice[] with the latest price and calls pool.poolUpkeep() to process the price difference. However, if pool.poolUpkeep() reverts, then the previous price value is lost and the processing will not be accurate. 

To prevent this, the executionPrice[_pool] variable should be updated only if poolUpkeep() does not revert. Alternatively, the administration of the executionPrice s could be done within the called poolUpkeep() function of the LeveragedPool contract. 

The issue has been fixed in PR 327.

## Description
The `performUpkeepSinglePool()` function of the `PoolKeeper` contract updates `executionPrice[]` with the latest price and calls `pool.poolUpkeep()` to process the price difference. However, `pool.poolUpkeep()` can revert, for example, due to the `checkInvariantsBeforeFunction` modifier in `mintTokens()`. 

If `pool.poolUpkeep()` reverts, then the previous price value is lost and the processing will not be accurate. Therefore, it is safer to store the new price only if `pool.poolUpkeep()` has been executed successfully.

```solidity
function performUpkeepSinglePool(...) public override {
    ...
    int256 lastExecutionPrice = executionPrice[_pool];
    executionPrice[_pool] = latestPrice; // previous price can get lost if poolUpkeep() reverts
    ...
    try pool.poolUpkeep(lastExecutionPrice, latestPrice, _boundedIntervals, _numberOfIntervals) {
        ... // executionPrice[_pool] should be updated here
    } catch Error(string memory reason) {
        ...
    }
}
```

## Recommendation
To prevent losing the `latestPrice` value, the `executionPrice[_pool]` variable should be updated only if `poolUpkeep()` doesn’t revert.

```solidity
function performUpkeepSinglePool(...) public override {
    ...
    int256 lastExecutionPrice = executionPrice[_pool];
    // executionPrice[_pool] = latestPrice; // This line is problematic
    ...
    try pool.poolUpkeep(lastExecutionPrice, latestPrice, _boundedIntervals, _numberOfIntervals) {
        executionPrice[_pool] = latestPrice; // Update only after successful execution
    } catch Error(string memory reason) {
        ...
    }
}
```

Alternatively, the administration of the `executionPrice` could be done within the called `poolUpkeep()` function of the `LeveragedPool` contract.
