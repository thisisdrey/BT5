# [M-01] No slippage protection

## Summary
The Burve contract allows users to provide liquidity to both the Island pool and Uniswap V3 pools. However, there is a bug in the contract that affects the number of shares minted to users. This is because the amount of shares depends on a variable called `sqrtRatioX96`, which can fluctuate and result in users receiving fewer tokens than expected. The impact of this bug is considered medium, and the likelihood of it occurring is also medium. To fix this issue, it is recommended to add a `minSharesOut` parameter to the mint/burn functions.

## Details
## Severity

**Impact:** Medium

**Likelihood:** Medium

## Description

The Burve contract allows users to provide liquidity to both the Island pool and Uniswap V3 pools. The number of shares minted to users is determined using the following calculation:

```solidity
    function islandLiqToShares(uint128 liq) internal view returns (uint256 shares) {
        if (address(island) == address(0x0)) {
            revert NoIsland();
        }

        (uint160 sqrtRatioX96,,,,,,) = pool.slot0();

        (uint256 amount0, uint256 amount1) = getAmountsFromLiquidity(
            sqrtRatioX96,
            island.lowerTick(),
            island.upperTick(),
            liq
        );

        (,, shares) = island.getMintAmounts(amount0, amount1);
    }
```

Since the amount of shares depends on `sqrtRatioX96`, any fluctuations in this value will impact the number of shares users receive. As a result, users may end up with fewer tokens than expected.

## Recommendations

Consider adding a `minSharesOut` parameter to the mint/burn functions.
