# Unbounded iteration over balances in InitGenesis can be exploited to prevent chain start 

## Summary
The bug report discusses an issue with the GetAllBalances function in the genesis.go file. This function is used to retrieve balances from a rewards pool, but it has a problem where it can be exploited by users to deposit an unlimited number of tokens. This can potentially block the start of the chain and require a costly upgrade to fix. Although there is no clear motive for someone to exploit this issue, it could still cause significant problems. The recommended solution is to build the rewards pool balances denomination by denomination and compare them using the IsAnyGT check. This issue has been fixed in PR 14 and has been verified by Alpo.

## Details
## Context
genesis.go#L319

## Description
GetAllBalances call in InitGenesis triggers an unbounded iteration over the rewardsPool-Balances address, which any user can deposit an arbitrary number of tokens to. If this is exploited, chain start can be blocked, which would require running an expensive upgrade handler that cleans up the spam token balances prior to chain initialization. Since this would require making direct changes to state and redistributing binaries, the impact is quite high. However, there is no material incentive to do this other than slowing down an upgrade or trying to sneak in a chain halt at upgrade-time.

## Recommendation
Build the rewardsPoolBalances denom by denom, running GetBalance on each rewards denom that needs to be compared against in the IsAnyGT check that follows.
