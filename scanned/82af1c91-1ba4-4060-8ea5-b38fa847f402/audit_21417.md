# [M01] Cannot change zero fee

## Summary
This bug report is about the `StreamingFeeModule` in the SetProtocolV2. This module allows the manager to set a zero fee, however, when it is changed to a non-zero value, the `accrueFee` function will revert, making it impossible to change the fee. To fix this, a pull request (PR#118) was created to update the `lastStreamingFeeTimestamp` and return successfully when attempting to accrue a zero fee. This solved the bug and the module now works as expected.

## Details
The `StreamingFeeModule` [allows the manager to set a zero fee](https://github.com/SetProtocol/set-protocol-v2/blob/b8286f431547823ff3935a1343cd4cf4d77585a1/contracts/protocol/modules/StreamingFeeModule.sol#L154). However, when it is subsequently changed, [the `accrueFee` function will revert](https://github.com/SetProtocol/set-protocol-v2/blob/b8286f431547823ff3935a1343cd4cf4d77585a1/contracts/protocol/modules/StreamingFeeModule.sol#L87), making it impossible to change the fee to a non-zero value. Consider simply updating the `lastStreamingFeeTimestamp` and returning successfully when attempting to accrue a zero fee.


***Update:** Fixed in [PR#118](https://github.com/SetProtocol/set-protocol-v2/pull/118).*
