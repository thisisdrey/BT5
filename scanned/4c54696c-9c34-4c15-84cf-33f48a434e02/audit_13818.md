# A Dark Age Can End Prematurely

## Summary
The client has marked a bug as "Fixed" in the file DoomsdaySettlers.sol. The bug caused a Dark Age to start and end prematurely if enough players abandoned their settlements. This was due to the contract using the supply at the mint of the last token instead of the current supply to calculate if a Dark Age is in progress. The recommended fix is to update the supply to initialize at (hashes.length - firstSettlement).

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `7736eb06074e1de55cccebfc2cbf572e892c08bc`. The client provided the following explanation: Uses supply at mint of the last minted token rather than current supply to calculate isDarkAge.

**File(s) affected:**`DoomsdaySettlers.sol`

**Description:** A Dark Age is intended to continue until only one settlement remains. The contract tracks whether a Dark Age is in progress in `isDarkAge()`:

```
function isDarkAge() public view returns(bool){
    // Returns true if game is currently in Dark Age
    // Ie, if there are more than 2 tokens in existence, and current block is beyond the collapse block
    unchecked{
        uint supply = (hashes.length - abandoned);
        uint collapseBlock = settlements[uint32(hashes.length )].settleBlock;

        if(supply * COLLAPSE_RAMP <  COLLAPSE_INITIAL - COLLAPSE_MIN){
            collapseBlock += ( COLLAPSE_INITIAL - supply * COLLAPSE_RAMP ) / BLOCK_TIME;
        } else{
            collapseBlock +=  COLLAPSE_MIN / BLOCK_TIME;
        }

        return supply > 1 && (block.number > collapseBlock );
    }
}
```

In the current implementation, the `supply` can both increase (via settling) and decrease (via abandoning). It is possible for a Dark Age to begin and end prematurely if enough players abandon their settlements.

**Exploit Scenario:**

1.   Let's say that we have 10,000 settlements and the most recently settled is at block 500. This would mean that a Dark Age starts at:

```
collapseBlock == 500 + ( COLLAPSE_INITIAL - 10000 * COLLAPSE_RAMP ) / BLOCK_TIME 
              == 500 + 533 
              == 1033
```

1.   A Dark Age begins by virtue of reaching block 1050.
2.   Over the next few blocks, 9,000 settlements are all abandoned.
3.   `isDarkAge()` will now calculate:

```
collapseBlock == 500 + ( COLLAPSE_INITIAL - 1000 * COLLAPSE_RAMP ) / BLOCK_TIME 
              == 500 + 6533 
              == 7033
```

1.   With the current block being roughly block 1050, we are now back in a Renaissance.

**Recommendation:** Update `supply` to initialize to `(hashes.length - firstSettlement)`.
