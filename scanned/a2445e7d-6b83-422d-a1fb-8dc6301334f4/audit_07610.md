# Improper fee subtraction on transferFrom function 

## Summary
The transferFrom function in the MEME20.sol contract is not working properly. Instead of subtracting fees from the intended recipient, it is subtracting them from the sender. This can cause problems when trying to transfer tokens through smart contracts. The bug also results in user funds being distributed incorrectly. A test has been provided as proof of concept, and the bug has been fixed by Trugly Labs and Cantina Managed. This is considered a high risk issue.

## Details
## Context
**File:** MEME20.sol  
**Lines:** 99-100

## Description
The `transferFrom` function from the `MEME20.sol` contract subtracts the fees from the `msg.sender` instead of the `to` address as it is designed for. This can have direct implications when attempting to transfer the token through smart contracts of various protocols. Additionally, it inappropriately distributes user funds.

## Proof of Concept
The next test reverts as the router address does not have any token balance.

```solidity
function test_issue22() public {
    MEME20 my20 = new MEME20('MY MEME', 'M', address(this), address(this));
    address router = makeAddr('router');
    address treasury = makeAddr('treasury');
    address pool = makeAddr('pool');
    address[] memory routers = new address[](1);
    address[] memory expmtFees = new address[](0);
    routers[0] = router;
    address bob = makeAddr('bob');
    my20.initialize(address(this), treasury, uint256(80), uint256(50), pool, routers, expmtFees);
    my20.transfer(pool, 9e18);
    vm.prank(pool);
    my20.approve(router, 9e18);
    vm.prank(router);
    my20.transferFrom(pool, bob, 9e18);
}
```

## Recommendation
Replace `super.transfer` with `super.transferFrom` when transferring the fees.
