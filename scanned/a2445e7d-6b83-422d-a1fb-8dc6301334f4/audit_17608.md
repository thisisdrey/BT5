# [M-11] Twav.getTwav() will return a wrong result when twavObservations[TWAVBLOCK_NUMBERS - 1].timestamp = 0.

## Summary
This bug report is about a vulnerability in the "Twav" smart contract, which can affect the main functions like NibblVault.buy() and NibblVault.sell(). The issue lies in the "if" condition of Twav._getTwav(), which is missing some edge cases. This can result in the function returning 0 instead of the correct value. The bug was discovered using the Solidity Visual Developer of VSCode. The recommended mitigation step is to replace "timestamp" with "cumulativeValuation", which is increasing all the time.
