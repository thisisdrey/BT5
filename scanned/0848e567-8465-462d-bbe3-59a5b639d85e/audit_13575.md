# An unsafe transfer

## Summary
In this bug report, it is highlighted that some tokens do not revert when a failed `transferFrom` is attempted. Additionally, some tokens have a fee on transfer, so the real transferred balance should be checked via `balanceOf()` to prevent funds insolvency. To resolve this issue, it is recommended to use the `safeTransfer` library and also add a check that an exact amount of tokens was transferred to the contract. This will help ensure that the correct amount of tokens is transferred, and that any fees associated with the transfer are accounted for.

## Details
##### Description
