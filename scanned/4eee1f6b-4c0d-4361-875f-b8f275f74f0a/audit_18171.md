# [H-01] `IndexLogic`: An attacker can mint tokens for himself using assets deposited by other users

## Summary
A bug has been identified in the mint function of the IndexLogic contract, which can be found at https://github.com/code-423n4/2022-04-phuture/blob/main/contracts/IndexLogic.sol#L48. This bug allows an attacker to monitor the asset balance in the vToken contract, and when the balance is greater than lastBalance, the attacker can call the mint function to mint tokens for himself. No tools were used to identify this bug. The recommended mitigation step is to call the transferfrom function in the mint function of the IndexLogic contract to transfer the user's assets. This bug needs to be addressed as soon as possible to prevent attackers from exploiting the vulnerability.

## Details
_Submitted by cccz, also found by hyh, Kenshin, pedroais, and TrungOre_

In the mint function of the IndexLogic contract, users are required to transfer assets to vToken in advance, and then call the mint function to mint tokens.
The attacker can monitor the asset balance in the vToken contract. When the balance is greater than lastBalance, the attacker can call the mint function to mint tokens for himself.

### Proof of Concept

[IndexLogic.sol#L48](https://github.com/code-423n4/2022-04-phuture/blob/main/contracts/IndexLogic.sol#L48)<br>

### Recommended Mitigation Steps

Call the transferfrom function in the mint function of the IndexLogic contract to transfer the user's assets.
