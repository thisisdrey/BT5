# M-36: _validateRange in Config does not check the input value as described

## Summary
This bug report is about an issue found in the `_validateRange` function in the Config of the GMX-Synthetics project. The issue is that the function does not check the input value against a pre-set boundary as described in the dev note. This means the function does not verify the input value is within an expected range. It was found by 0xGoodess using Manual Review. A code snippet of the function can be found at the link provided. 

The issue was discussed by the team and it was accepted as a valid medium. The team agreed to escalate the issue for 10 USDC. After the 48-hour escalation window closed, the issue's escalation was accepted and the contestants' payouts and scores were updated accordingly. The issue was fixed in the GMX-Synthetics project in a pull request.

## Vulnerability Detail
in the dev note of this function `_validateRange` : it said  `// @dev validate that the value is within the allowed range`. However in the function itself the value is only used in sending revert message, but not used in any check against a pre-set boundary.
