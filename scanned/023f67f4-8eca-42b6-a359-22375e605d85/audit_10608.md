# `ReachDistributionv2` Is Effectively Ownerless

## Summary
The bug report discusses an issue with the `ReachDistributionV2` contract. The contract is currently owned by the `ReachFactory` contract, but it should be owned by the user creating the Reach server. Additionally, the `ReachFactory` contract is not properly configured to call certain functions in `ReachDistributionV2`. The recommendation is to transfer ownership of `ReachDistributionV2` to the correct owner during deployment in `ReachFactory.deployAffiliateDistribution()`.

## Details
**Update**
It is more convenient to use the internal function `_transferOwnership()`. Since the contract now inherits from`Ownable2Step`, by calling the public function `transferOwnership()`, the owner would still be the factory until `msg.sender` accepts ownership.

Fixed in commit: `e87feb8b41f62fb76158dd706dfae5206ed6391f`.

Marked as "Fixed" by the client. Addressed in: `6d5224d`.

**File(s) affected:**`ReachFactory.sol, ReachDistributionV2.sol`

**Description:**`ReachDistributionV2` is deployed by `ReachFactory`, making the `ReachFactory` contract the owner. However, the `ReachDistributionV2` contract is intended to be owned by the user who is creating the Reach server. Furthermore, the `ReachFactory` is not configured to call the guarded `onlyOwner` functions in `ReachDistributionV2`.

**Recommendation:** Transfer the ownership of the `ReachDistributionV2` contract to the correct owner upon deployment in `ReachFactory.deployAffiliateDistribution()`.
