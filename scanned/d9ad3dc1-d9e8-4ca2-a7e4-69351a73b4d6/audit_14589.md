# M-2: maxTrainingDeposit can be bypassed

## Summary
A bug was found in the MainVault.sol code, which enabled users to bypass the maxTrainingDeposit by transferring balance from one user to another. The deposit function included a check that the amount plus the user's balance must be less than or equal to the maxTrainingDeposit. However, if the user transferred their balance to another account, the check would pass and the user could deposit more than allowed. The bug was found by csanuragjain, c7e7eff, and chaduke and was verified manually. The code snippet for the bug can be found at https://github.com/sherlock-audit/2023-01-derby/blob/main/derby-yield-optimiser/contracts/MainVault.sol#L113. The recommendation to fix this bug is to add a check that the amount transferred plus the balance of the receiver must be less than or equal to the maxTrainingDeposit. However, according to sjoerdsommen, this bug is known and there is no solution to it, and it is not a huge problem.

## Summary
It was observed that User can bypass the `maxTrainingDeposit` by transferring balance from one user to another

## Vulnerability Detail
1. Observe the `deposit` function

```solidity
function deposit(
    uint256 _amount,
    address _receiver
  ) external nonReentrant onlyWhenVaultIsOn returns (uint256 shares) {
    if (training) {
      require(whitelist[msg.sender]);
      uint256 balanceSender = (balanceOf(msg.sender) * exchangeRate) / (10 ** decimals());
      require(_amount + balanceSender <= maxTrainingDeposit);
    }
...
```

2. So if User balance exceeds maxTrainingDeposit then request fails (considering training is true)
3. Lets say User A has balance of 50 and maxTrainingDeposit is 100
4. If User A deposit amount 51 then it fails since 50+51<=100 is false
5. So User A transfer amount 50 to his another account
6. Now when User A deposit, it does not fail since `0+51<=100`
## Impact
User can bypass maxTrainingDeposit and deposit more than allowed

https://github.com/sherlock-audit/2023-01-derby/blob/main/derby-yield-optimiser/contracts/MainVault.sol#L113

Manual Review

## Recommendation
If user specific limit is required then transfer should be check below:

```solidity
 require(_amountTransferred + balanceRecepient <= maxTrainingDeposit);
```
