# Improper Selection Of Winners

## Summary
The VRFv2Consumer.sol file has a bug that causes issues with selecting random winners. When the vrfCoordinator triggers the fulfillRandomWords function, it uses the randomWords to select winners from a list of participants. However, this method of storing the indexes of the winners can result in duplicate winners and a smaller number of winners than expected. Additionally, the process for removing participants from the list is faulty and can lead to unexpected behaviors, such as empty strings in the list of winners. To fix this, the chosen participant value should be stored directly in the winners list and the chosen participant should be properly removed from the participants array. This bug has been fixed in version ac66a86.

## Details
## VRFv2Consumer.sol Issues and Remediation

In `VRFv2Consumer.sol`, `fulfillRandomWords` is triggered by the `vrfCoordinator` to fulfill the request for random words. The `randomWords` is then used to select a random list of participants as winners.

## Issues

This leads to two primary issues:

1. Storing the indexes of the winners from the list of participants is not a preferred method. Calculating `randomWords[i] % _RNGInstance.participants.length` may result in duplicate indexes, which then results in duplicate members in the list of winners. Additionally, the removal of duplicates results in a fewer number of winners than expected.

2. The process for which the participants are deleted from the list is faulty, as `delete` sets the array item to its default value and does not reduce the length of the array.

The combination of the two issues may lead to unexpected behaviors. For example, the list of the winners may contain empty strings when `getRNGWinners` is called.

## Remediation

Directly store the chosen participant value in the winners list instead of the index. Additionally, properly remove the chosen participant from the participants array by swapping it with the last element, then utilizing the `pop` method to remove the last element.

## Patch

Fixed in `ac66a86`.
