# Minters can mint double the max allowance in an interval 

## Summary
This bug report discusses an issue with the RateLimit.sol contract where callers are able to mint more tokens than the maximum allowed within a given time interval. This allows for the possibility of minting double the maximum allowance in a single interval. A proof of concept is provided, along with a recommendation to fix the issue by implementing a replenish only after a certain time interval has passed, or setting the initial allowance to 0. The bug has been fixed in the latest commit.

## RateLimit Issue in Token Minting

## Context
**File:** RateLimit.sol

## Description
Callers can mint more tokens than `maxAllowance` in an interval. Let's say the interval is 1 hour and `maxAllowance` is 60 cbBTC. At `t = 0`, the caller's allowance is 60 cbBTC. Consider this scenario (more scenarios can be crafted):

- At `t = 0`, the caller mints 60 cbBTC, and their allowance drops to 0.
- At `t = 0.5` hour, the `amountToReplenish` is 30 cbBTC, so the new allowance becomes 30 cbBTC. The caller mints another 30 cbBTC.
- Just before the 1 hour interval finishes, `amountToReplenish` becomes 30 cbBTC again, and the caller can mint another 30 cbBTC.

So with `maxAllowance = 60 cbBTC`, the caller can mint a total of 120 cbBTC in the same interval. A similar issue happens when a minter doesn't use their allowance in an interval. This allows the minter to mint double the maximum allowance in the next interval.

## Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.6;

import "forge-std/Test.sol";
import "forge-std/mocks/MockERC20.sol";
import "src/cbbtc/wrapped-tokens/MintForwarder.sol";

contract MockToken is MockERC20 {
    constructor() {
        initialize("MockToken", "TKN", 18);
    }

    function mint(address to, uint256 amount) public {
        _mint(to, amount);
    }
}

contract MintForwarderTest is Test {
    address receiver = makeAddr("receiver");
    MintForwarder mintForwarder;
    MockToken mockToken;

    function setUp() public {
        mockToken = new MockToken();
        mintForwarder = new MintForwarder();
        mintForwarder.initialize(address(this), address(mockToken));
    }

    function test_poc_mintDoubleInOneInterval() public {
        mintForwarder.configureCaller(address(this), 1000, 100 seconds); // 1000 tokens in 100 seconds interval
        vm.warp(block.timestamp + 100 seconds);
        mintForwarder.mint(receiver, 1000);
        mintForwarder.mint(receiver, 1000); // minted 2X tokens in one interval
        vm.warp(block.timestamp + 100 seconds);
        mintForwarder.allowanceCurrent(address(this));
        vm.warp(block.timestamp + 100 seconds);
        mintForwarder.mint(receiver, 1000);
        mintForwarder.mint(receiver, 1000); // minted 2X tokens in one interval
        vm.warp(block.timestamp + 100 seconds);
        mintForwarder.allowanceCurrent(address(this));
        vm.warp(block.timestamp + 100 seconds);
        mintForwarder.mint(receiver, 1000);
        mintForwarder.mint(receiver, 1000); // minted 2X tokens in one interval
    }
}
```

## Recommendation
Consider a replenishment only after the last update to allowance is interval seconds apart, or set the initial allowance to 0.
