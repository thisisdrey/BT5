# Improper Normalized Amount Updating On State

## Summary
The report states that there is a bug in SpotEngine's socializeSubaccount function. This function is supposed to subtract the negative balance amount from the total borrow, but instead it is adding it. This is because the function has a minus symbol instead of an add symbol, even though the balance amount is already negative. To fix this, the minus symbol should be changed to an add symbol. This bug has been fixed in version #116.

## Details
## SpotEngine SocializeSubaccount Function Issue

In SpotEngine’s `socializeSubaccount` function, the negative balance amount is being added to the total borrow, instead of being subtracted.

```solidity
contracts/SpotEngine.sol
state.totalBorrowsNormalizedX18 -= balance.amountX18.div(
    state.cumulativeBorrowsMultiplierX18
);
```

Since `socializeSubAccount` is clearing the borrow and adding the loss to all the deposited users, it must subtract from the total borrow. However, it has a minus (`-`) instead of an add (`+`), as the `amountX18` is already negative.

## Remediation

Use the add (`+`) symbol instead of subtraction (`-`), which is adding the balance to the total borrow amount since the amount is already negative.

## Patch

The operation was modified to addition. Fixed in #116.
