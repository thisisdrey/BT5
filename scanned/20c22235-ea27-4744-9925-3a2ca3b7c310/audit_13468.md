# Possible Denial of Service Concerns Around Expired Listings

## Summary
The bug report addresses an issue in the code for a marketplace application. The `validate()` function is supposed to check if a listing has not expired, but the code does not currently do this. This can cause problems for users trying to buy listings that have already expired. However, the code does have a check for expired listings in a different function called `swap()`, so the issue is not as severe. The recommendation is to add the check for expired listings in the `validate()` function to prevent any potential issues.

## Details
**Update**
Addressed in: `ec60dbbf61fc02bbd5bcc045468152bac570331f`. The client added a check for expired listings to `validate()`.

**File(s) affected:**`Marketplace.sol`

**Description:** According to the documentation, `Marketplace.validate()` should validate that “The listing should have `validUntil` greater than the block timestamp”, however this check does not happen in the code. This results in expired listings being returned as valid, which can lead to denial of service issues for users looking to purchase these seemingly valid listings. Fortunately, this check is being performed in `Marketplace.swap()`, preventing the trading of expired listings.

**Recommendation:** Validate whether the listing has expired in `Marketplace.validate()`.
