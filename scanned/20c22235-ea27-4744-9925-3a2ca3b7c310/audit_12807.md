# `withdraw` function is not working if `emergencyWithdraw` is set to true

## Summary
This bug report is about an issue with the `emergencyWithdraw` and `withdraw` functions in a program. If the `emergencyWithdraw` is set to true and the user tries to withdraw, the program will give an error message saying "ReentrancyGuard: reentrant call". This bug has been marked as high severity and has been resolved. The recommendation is to fix the `nonReentrant` modifier usage.

## Details
**Severity**: High

**Status**: Resolved

**Description**

if the `emergencyWithdraw` is true and the user calls `withdraw` function, the call will be reverted with the message: "ReentrancyGuard: reentrant call"

**Recommendations**: 

fix the `nonReentrant` modifiers usage
