# Rocketpool CLI - Insecure SSH HostKeyCallback ✓ Fixed

## Summary
This bug report is about the SSH client factory returning instances that have an insecure `HostKeyCallback` set. This means that SSH servers' public key will not be validated and thus initialize a potentially insecure connection. This could lead to security vulnerabilities and should not be used for production code. To fix this issue, a proper host key callback function to validate the remote party’s authenticity is now defined. An example of this is the code found in the smartnode-2.5-Tokenomics/shared/services/rocketpool/client.go:L87.

## Details
#### Resolution



A proper host key callback function to validate the remote party’s authenticity is now defined.
