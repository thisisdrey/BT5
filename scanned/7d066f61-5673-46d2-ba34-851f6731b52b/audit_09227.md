# Unhandled Native Asset Provision

## Summary
The functions fcnSettleVault and fcnRepayBond are currently not marked as payable, which prevents them from handling the provision of native assets. This is causing issues for FCN products that use native Ether as a deposit asset, as these functions unconditionally revert and compromise the protocol's robustness. To fix this, it is recommended to ensure that fcnSettleVault and fcnRepayBond are marked as payable. This issue has been addressed in the patch 5ab692e. © 2024 Otter Audits LLC. ## Details
## FCN Functions and Their Status

`fcnSettleVault` and `fcnRepayBond` are currently not marked as payable, which prevents them from handling the provision of native assets. In cases where an FCN product is set up to utilize native Ether as a deposit asset, these functions unconditionally revert, compromising the robustness of the protocol.

## Remediation

Ensure `fcnSettleVault` and `fcnRepayBond` are marked as payable.

## Patch

Fixed in `5ab692e`.
