# NFTR-10 | Potential DoS

## Summary
This bug report discusses an issue with the `initiateRetroactiveHoldFarming` function in the Hold Farming contract. The function calls the `initiateHoldFarmingForNFT` function 10,000 times, which could exceed the block gas limit and prevent any collection from getting curated. To address this issue, the NFTR team eliminated the function and suggested that retroactive hold farming might be taken care of in a different way, if at all. Additionally, they recommended ensuring that the block gas limit is not exceeded or executing calls to `initiateHoldFarmingForNFT` in batches.

## Details
**Description**

The `initiateRetroactiveHoldFarming` function calls the `initiateHoldFarmingForNFT` function in the Hold Farming contract 10,000 times. This may exceed the block gas limit and prevent any collection from getting curated.

**Recommendation**

Ensure that the block gas limit limit is not exceed or consider executing calls to `initiateHoldFarmingForNFT` in batches.

**Resolution**

NFTR Team:

- This function has been eliminated.
- Retroactive hold farming might be taken care of in a different way, if at all.
