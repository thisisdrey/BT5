# Wrong logic in `withdrawAll`

## Summary
This bug report covers an issue in the VoterProxy.sol contract, in which the withdrawAll method is counting the balance incorrectly. Instead of counting balance as `balanceOfPool(_gauge)`, the correct logic should be `balanceOfPool(_gauge).add(IERC20(_token).balanceOf(address(this)))`. This is an issue because the withdrawAll method is used by the shutdownSystem, which could lead to tokens remaining in the contract. To fix this issue, the recommendation is to count the amount of tokens as `balanceOfPool(_gauge).add(IERC20(_token).balanceOf(address(this)))`.

## Details
##### Description
At the moment `withdrawAll` counts balance as: `balanceOfPool(_gauge)` (https://github.com/convex-eth/platform/blob/754d9e700693246275b613e895b4044b63ce9ed5/contracts/contracts/VoterProxy.sol#L92)

Correct logic should be: `balanceOfPool(_gauge).add(IERC20(_token).balanceOf(address(this)))`.

The `withdrawAll` method is used by `shutdownSystem` so potentially some tokens could remain in the contract.

##### Recommendation
It is recommended to count amount of tokens as `balanceOfPool(_gauge).add(IERC20(_token).balanceOf(address(this)))`.
