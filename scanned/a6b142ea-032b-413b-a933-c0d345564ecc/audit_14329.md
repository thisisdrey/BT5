# M-27: Virtual impacts can be trivially bypassed via structuring

## Summary
This bug report is about an issue in the GMX-Synthetics codebase where virtual price impacts can be bypassed. Virtual price impacts are meant to make sure that even when a user spreads their orders over multiple pools, they still have a sufficient price impact to deter oracle price manipulation. The bug report found that the code allows smaller orders to go through without incurring this global virtual price impact. 

The impact of this bug is that a user can get around a large virtual price impact by chopping their large orders into multiple smaller orders, below the price impact threshold, manipulate the oracle price, and have all of the small orders executed in the same block, defeating the manipulation protection. The bug was found through manual review and the code snippet provided in the report shows that only changes in position are taken into account when determining whether the threshold should apply. 

The recommendation given is that virtual price impact needs to always apply, in order to solve for the case of splitting a large order over multiple accounts. If orders split over multiple accounts is not a risk you want to address, the price impact should at least take into account the total position size for a single account when determining whether the threshold should apply. The bug was fixed in a pull request on the GMX-Synthetics Github repository.
