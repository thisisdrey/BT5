# Missing chargeFee modifier allows spamming the CL

## Summary
The bug report states that there are three functions, `removeOperator`, `redelegate`, and `redelegateOnBehalf`, in the `IPTokenStaking.sol` contract that do not have the `chargeFee` modifier. This allows a malicious user to spam the contract by sending these functions, which can cause issues with the contract's functionality. The report recommends adding the `chargeFee` modifier to these functions to prevent this from happening. The issue has been fixed by the Story team.

## Details
##### Description

The `removeOperator`, `redelegate` and `redelegateOnBehalf` **do not have** the `chargeFee` modifier. As none of them requires the user sending funds, it allows a malicious user to spam the CL by sending these events, which requires consumption from the CL.
