# Incorrect Version Parsing

## Summary
The vulnerability in parseVersion within db is related to the use of strconv.ParseInt with bitSize set to 32. This limits the parsed integer to the range of an int32, which may result in an inability to parse version numbers greater than Int32.Max. The fix for this is to use strconv.ParseUint with bitSize set to 64 to handle the entire range of Uint32.Max. This issue has been fixed in dec c4f4.

## Details
## Vulnerability in parseVersion

The vulnerability in `parseVersion` within `db` is related to the use of `strconv.ParseInt` with `bitSize` set to 32. This limits the parsed integer to the range of an `int32`. Since version numbers in the code’s context should be of type `int64`, utilizing `strconv.ParseInt` with `bitSize` set to 32 may result in an inability to parse version numbers greater than `Int32.Max`.

> _src/memavl/db.go_

```go
func parseVersion(name string) (int64, error) {
    if !isSnapshotName(name) {
        return 0, fmt.Errorf("invalid snapshot name %s", name)
    }
    v, err := strconv.ParseInt(name[len(SnapshotPrefix):], 10, 32)
    if err != nil {
        return 0, fmt.Errorf("snapshot version overflows: %d", err)
    }
    return v, nil
}
```

Thus, this restricts the valid version numbers to the range of signed integers (`Int32.Max`), whereas the version numbers should be treated as unsigned integers to cover the full range up to `Uint32.Max`.

## Remediation

Use `strconv.ParseUint` with `bitSize` set to 64 to handle the entire range of `Uint32.Max`.

## Patch

Fixed in `decc4f4`.
