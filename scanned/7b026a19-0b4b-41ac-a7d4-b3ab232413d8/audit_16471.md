# The value in the `idInList` mapping can be wrong.

## Summary
This bug report discusses an issue with the finalization of deposit requests in TokensFarm.sol and TokensFarmSDK.sol. When a user doesn't have any deposit requests, they are removed from the waiting list and their ID is given to the last user in the waiting list. However, the value in the 'idInList' mapping is not updated for the last user, which can cause their request to be blocked. The recommendation is to update the value in the mapping and delete it for the user. This issue has been resolved in the latest audit.

However, the value in the `idInList mapping is not updated for 'lastUserInWaiting ListArray`. Due to this, the finalization of the request for 'lastUserInWaitingListArray can be blocked.

**Recommendation**

Update the value in the 'idInList' mapping for 'lastUserInWaitingListArray with deletedUserId'. Delete the value from the 'idInList' mapping for the user. Take into account that 'lastUserInWaitingListArray can be equal to 'user' in case there is only one address in the waiting list.
