# Suspicious manipulation of `totalReserves` and `internalCash`

## Summary
A bug has been reported in the compound protocol contract, CCapableErc20.sol, at line 132. This bug involves the ‘gulp’ function, which changes the ‘totalReserves’ and ‘internalCash’ variables. This function can be re-entered while a flashloan is being executed, which could lead to an unpredictable outcome. The ‘accrueInterest’ function is also affected by this bug. To prevent this bug from occurring, it is recommended to double check the code and add a re-entrancy guard to the ‘gulp’ function.

## Details
##### Description
At the line https://github.com/CreamFi/compound-protocol/blob/e414160eb8a774140457c885bb099baae528df04/contracts/CCapableErc20.sol#L132 contract have `gulp` function that change `totalReserves` and `internalCash` variables. That function can be re-entered from while flashloan execution. For now, in `flashloan` contract captured affected variables and also `accrueInterest` depends on it, to be honest it's really precarious place.

##### Recommendation
We recommend to double check this and add re-entrancy guard to `gulp` function
