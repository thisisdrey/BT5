# Wrongly increased `participants` amount

## Summary
A bug has been reported in the code of the TosDisFinance project on line 120 of the StakingPool.sol file. The bug causes the "participants" amount to increase even if the user provides a zero value for the "_amountToStake" parameter. This could allow someone to increase the "participants" infinitely by calling the "stakeTokens" function with a zero stake.

To fix this bug, it is recommended that the "participants" amount only be increased if the "_amountToStake" is not zero. This will ensure that the participants amount cannot be increased infinitely.

## Details
##### Description
At line https://github.com/tosdis/TosDisFinance/blob/be50dbf8a52a8f919694498bf30394d328d88fbb/StakeMaster/contracts/StakingPool.sol#L120 contract increases participants amount, that happens if `user.amount` is zero, however if user will provide zero `_amountToStake` `participants` also will increased. So somebody can increase `participants` infinitely by calling `stakeTokens` with zero stake.

##### Recommendation
We recommend increase `participants` only if `_amountToStake` not zero
