# Rust-crypto is not recommended for security-critical usage

## Summary
This bug report is about a security issue with the rust-crypto crate, used by Parity, which is a cryptography library. The rust-crypto crate has never received a thorough security assessment and is not recommended for applications where security is important. A malicious actor (Alice) could exploit this issue to decrypt sensitive Parity key files. 

To minimize the use of the rust-crypto crate in the short term, it is recommended to prepare to migrate away from it. In the long term, a more conservative, better reviewed library for cryptographic operations should be used. Consider using a Rust wrapper to OpenSSL or the Rust Ring library.

References to other security issues are provided, such as CVE-2016-7798 and CVE-2014-1266, as well as a link to a document outlining the many ways cryptographic software can fail.

## Details
## Type: Cryptography
## Target: ethstore/src/account/crypto.rs

### Difficulty: Medium

### Description
The rust-crypto crate has never received a thorough security assessment and should not be considered for security-critical usage. Many of the Parity encryption algorithms, encryption modes, and hashing algorithms are based on the rust-crypto crate. The rust-crypto crate homepage explicitly mentions that it is not recommended for applications where security is important.

![Screenshot from rust-crypto crate's homepage](link_to_image)

Cryptography libraries are notoriously difficult to implement correctly and can fail in catastrophic ways due to seemingly minor flaws.

### Exploit Scenario
A critical security issue is discovered in one of rust-crypto’s cryptographic algorithms which Parity uses. Alice, a malicious actor, exploits this issue to decrypt sensitive Parity key files.

### Recommendation
- **Short term:** Minimize the use of the rust-crypto crate and prepare to migrate away from it.
- **Long term:** Use a more conservative, better reviewed library for cryptographic operations. Consider using a Rust wrapper to OpenSSL or the Rust Ring library.
