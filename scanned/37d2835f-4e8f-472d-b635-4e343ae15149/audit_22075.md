## Summary
This bug report is about a patching issue with the OriginToken and TokenMigration contracts in the origin-contracts/contracts/token/ directory. The difficulty level of the bug is medium. 

The bug occurs when a buyer submits a listing offer with a commission value and it is accepted by the seller. The commission is subtracted from the listing's deposit and left in the Marketplace contract balance. When the buyer finalizes the offer, the affiliated listed on the offer receives the allocated commission from the Marketplace's balance. However, if the finalization period passes and the seller finalizes the offer instead, the affiliate does not receive the commission. This is intended behavior, but the commission is trapped in the Marketplace's balance and cannot be returned to the listing's deposit pool. 

The exploit scenario is as follows: Alice creates a listing on the Marketplace. Bob submits an offer for Alice's listing with an affiliate and commission, which is accepted. Bob does not finalize within the finalization window, allowing Alice to finalize the offer. As Alice finalized the offer, the commission is never paid for the offer. 

The recommendation is to refund the listing deposit upon offer finalization by the seller, and to use unit testing to ensure that no token is left once all listings are finalized or canceled.

## Patching

**Target:** `origin-contracts/contracts/token/{OriginToken,TokenMigration}.sol`

**Difficulty:** Medium

## Description

When a buyer submits a listing offer with a commission value and it is subsequently accepted by the seller, the commission is subtracted from the listing’s deposit, leaving the commission escrowed in the Marketplace contract balance. Upon finalization of the offer by the buyer, the affiliate listed on the offer receives the allocated commission from the Marketplace’s balance.

```solidity
function acceptOffer(uint listingID, uint offerID, bytes32 _ipfsHash) public {
    Listing storage listing = listings[listingID];
    Offer storage offer = offers[listingID][offerID];
    ...
    listing.deposit -= offer.commission; // Accepting an offer puts Origin Token into escrow
    ...
}
```
*Figure 1: An excerpt of the `acceptOffer` function, highlighting the commission reduction from the deposit.*

However, if the finalization period passes and the seller finalizes the offer in place of the buyer, the affiliate does not receive the commission. This is intended behavior. However, the commission is trapped in the Marketplace’s balance. There’s no way to return the commission to the listing’s deposit pool.

```solidity
function finalize(uint listingID, uint offerID, bytes32 _ipfsHash) public {
    Listing storage listing = listings[listingID];
    Offer storage offer = offers[listingID][offerID];
    ...
    paySeller(listingID, offerID); // Pay seller
    if (msg.sender == offer.buyer) { // Only pay commission if buyer is finalizing
        payCommission(listingID, offerID);
    }
    ...
}
```
*Figure 2: An excerpt of the `finalize` function, highlighting the lack of commission withdrawal upon seller finalization.*

## Exploit Scenario

Alice creates a listing on the Marketplace. Bob submits an offer for Alice’s listing with an affiliate and commission, which is subsequently accepted. Bob does not finalize within the finalization window, allowing Alice to finalize the offer. Because Alice finalized the offer, the commission is never paid for the offer. The commission cannot be retrieved from the Marketplace’s account balance.

## Recommendation

**Short term:** Refund the listing deposit upon offer finalization by the seller.

**Long term:** Use unit testing to ensure that no token is left once all listings are finalized or canceled.
