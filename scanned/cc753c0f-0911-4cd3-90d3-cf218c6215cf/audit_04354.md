# ClaggBaseAdapter._handleIncentive() withdraws the wrong token from the incentive vault 

## Summary
The bug is in a specific part of the ClaggBaseAdapter code, where the wrong token is being passed when distributing incentive tokens to depositors. This causes the function to fail and revert when trying to withdraw from the incentive vault. The recommended fix is to pass the correct token instead. This bug has been fixed in a recent commit and has been verified by the Cantina Managed team. It is considered to be a low-risk bug.

## Details
## ClaggBaseAdapter Issue Report

## Context
File: `ClaggBaseAdapter.sol`  
Lines: 350-354

## Description
In `ClaggBaseAdapter._handleIncentive()`, when distributing incentive tokens from the `incentiveVault` to depositors, the token is passed to `withdrawTokens`:

```solidity
try
    IIncentiveVault(incentiveVault).withdrawTokens(pool, token, incentiveUsed)
{} catch {
    return 0;
}
```

However, the `token` here is incorrect as it is the pool's token, not the incentive token. As a result, whenever `_handleIncentive()` is called, it will revert when attempting to withdraw from the incentive vault.

## Recommendation
Pass `poolIncentive.token` to `withdrawTokens` instead.

## Clave
Fixed in commit `c3656e53`.

## Cantina Managed
Verified, the recommended fix was implemented.
