# Use of Deprecated Chainlink Function

## Summary
The report highlights an issue with the contracts `BaseOracle` and `PriceProvider` in the `v2-core` repository. These contracts use a function called `latestAnswer` from a platform called Chainlink to retrieve the price of ETH. However, this function is no longer recommended and there is no way to check if the data it returns is up to date. The report suggests replacing the use of `latestAnswer` with a different function called `latestRoundData` and adding checks to ensure the price is positive, up to date, and complete. This issue has been resolved in a recent update to the repository.

## Details
The contracts [`BaseOracle`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/oracles/BaseOracle.sol) and [`PriceProvider`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/oracles/PriceProvider.sol) are using Chainlink’s [deprecated `latestAnswer`](https://docs.chain.link/data-feeds/api-reference/#latestanswer) function to retrieve the price of ETH. Although `latestAnswer` returns the price of the asset, it is not possible to check if the data is fresh.


The following instances of using `latestAnswer` were identified:


* [`latestAnswer`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/oracles/BaseOracle.sol#L68) in `BaseOracle.sol`
* [`getTokenPriceUsd`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/oracles/PriceProvider.sol#L79) and [`getLpTokenPriceUsd`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/oracles/PriceProvider.sol#L103) in `PriceProvider.sol`


Consider replacing calls to `latestAnswer` with Chainlink's [`latestRoundData`](https://docs.chain.link/data-feeds/api-reference#latestrounddata) function, and adding checks on the returned data to ensure the price value is positive, is not stale, and that the round is complete.


***Update:** Resolved in [pull request #220](https://github.com/radiant-capital/v2-core/pull/220) at commit [1612d75](https://github.com/radiant-capital/v2-core/commit/1612d75b11344257fd03fd5d65ec4704c4ae22ca). Calls to the deprecated `latestAnswer` function have been replaced with calls to `latestRoundData`. Additionally, transactions will be reverted if reported prices are older than a day, non-positive, or if the round is not complete.*
