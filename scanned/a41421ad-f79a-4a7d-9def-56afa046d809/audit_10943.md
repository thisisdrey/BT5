# Missing Fund Check

## Summary
The releaseEscrow function in the Escrow.sol file does not properly check if an intent is funded before transferring funds. This allows an attacker to exploit the system by waiting for a legitimate user to initiate an escrow and then using the obtained intentId to call releaseEscrow before the legitimate user has funded the intent. To fix this issue, the function should be updated to check for a funded intent before any transfers are made. This issue has been fixed in the latest patch.

## Details
## releaseEscrow Function Overview

`releaseEscrow` retrieves the intent details using `_intent.getIntent` and checks if the intent exists and its status is acknowledged (`IINTENT.IntentState.ACKNOWLEDGED`). However, it does not explicitly verify if the intent is funded (`funded` is true) before attempting to transfer funds. 

Thus, when a legitimate user initiates an escrow, the attacker may wait for the seller to acknowledge the intent and call `releaseEscrow` with the obtained `intentId` before the legitimate user funds the intent.

## Escrow.sol (SOLIDITY)

```solidity
function releaseEscrow(string memory intentId, bool sendToSeller) external
    onlyRole(RELEASER_ROLE) nonReentrant {
    (
        address receiver,
        address seller,
        ,
        address tokenAddress,
        IINTENT.IntentState status,
        ,
        uint256 amount,
        uint256 taxAmount,
        uint256 feeAmount,
        bool exists,
        ,
        bool funded
    ) = _intent.getIntent(intentId);
    
    require(exists, "Intent does not exist");
    require(status == IINTENT.IntentState.ACKNOWLEDGED, "Intent is not in acknowledged status");
    require(funded, "Intent is not funded");
}
```

## Remediation

Ensure `releaseEscrow` checks for a funded intent before any transfers.

## Patch

Fixed in commit `465f097`.
