# Ensure no reentry

## Summary
The bug report suggests adding a nonReentrant lock to every external method in order to prevent reentrancy attacks. After discussing with the client, the functionality was verified and the issue was marked as resolved.

## Details
**Description**

To make sure, there is no possible reentrancy attacks. It’s recommended to add something
like nonReentrant lock in every external method (as in solidity).

**Recommendation**:

Use nonReentrant lock.
Post-audit: after the converstaion with the client the functionality was verified and issue
marked as Verified.
