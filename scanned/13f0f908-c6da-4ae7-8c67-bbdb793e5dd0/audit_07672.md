# Incorrect RPL Stake Calculation During Withdrawal

## Summary
The bug report is about a problem with a line of code in a program. The code does not take into account the amount of locked RPL balance when calculating the total withdrawable balance. This means that more RPL can be withdrawn than is allowed. The recommendation is to modify the code to account for the locked RPL balance. The issue has been fixed in a recent update to the program.

## Details
## Description
Check on line [415] does not take into account the amount of locked RPL balance, which should not be accounted for in the total withdrawable balance:

```solidity
uint256 rplStake = getNodeRPLStake(msg.sender);
uint256 lockedStake = getNodeRPLLocked(msg.sender);
require(rplStake >= _amount, "Withdrawal amount exceeds node's staked RPL balance");
```

This could result in more RPL being withdrawn than allowed to.

## Recommendations
Modify implementation to account for locked RPL stake, e.g.:

```solidity
require(rplStake - lockedStake >= _amount);
```

## Resolution
The issue has been addressed in commit `85bede4` as per the recommendations.
