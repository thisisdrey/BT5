# `_repayCredit()` Is Publicly Accessible

## Summary
The report states that a function called `_repayCredit()` in the file `credit/CreditCaller.sol` is publicly accessible, but it should only be used by two other functions called `repayCredit()` and `liquidate()`. This means that anyone can call this function and potentially cause issues. The function is missing important checks to make sure that the loan is still valid and has not already been repaid or liquidated. This could result in a malicious user getting more money than they should by repeatedly calling this function. The recommendation is to change the function to be an `internal` function, which means it can only be used within the file and not by anyone else. This will prevent any potential issues caused by external users calling this function.

## Details
**Update**
The issue has been fixed by making `_repayCredit()` an internal function.

**File(s) affected:**`credit/CreditCaller.sol`

**Description:** The function `_repayCredit()` appears to be a helper function to be used by `repayCredit()` and `liquidate()`. However, the function is publicly accessible, as shown below:

```
function _repayCredit(address _recipient, uint256 _borrowedIndex) public returns (uint256)
```

The `_repayCredit()` function lacks the critical validation checks which are done by `repayCredit()` and `liquidate()` including:

*   Whether the `_borrowedIndex` is valid
*   Whether the position has been terminated

Among other possibilities, this could result in loans being repaid or liquidated multiple times, which may result in a malicious user retrieving several multiples of their collateral by repeatedly calling `_repayCredit()` with a `_borrowedIndex` that has already been terminated.

**Recommendation:** Make `_repayCredit()` into an `internal` function.
