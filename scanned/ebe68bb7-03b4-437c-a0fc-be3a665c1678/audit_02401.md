# Zero Value Handling in DIAOracleV2Guardian

## Summary
The `guardianDiffBips()` function in the `DIAOracleV2Guardian` contract has a bug where it does not handle zero values properly. This function calculates the difference between two values in basis points, but it does not check if one of the values is zero before performing division. This can cause the transaction to fail because of division by zero. The severity of this issue is classified as medium, and it is recommended to implement measures to handle zero values properly, such as adding checks for zero input values and implementing safe division and comparison mechanisms.

## Details
##### Description
The `guardianDiffBips()` function of the `DIAOracleV2Guardian` contract lacks handling of zero value scenarios.

The function calculates the difference between two values in basis points (1/1000th of a percent) but fails to check if `value` is zero before performing division. If `value` is zero, the division operation `/ value` will cause the transaction to revert due to division by zero.

The issue is classified as **Medium** severity, focusing on proactive numerical robustness improvements.

##### Recommendation
We recommend implementing comprehensive zero value handling:
- Add explicit checks for zero input values
- Implement safe division and comparison mechanisms
