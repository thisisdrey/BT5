# Lack of Token Whitelisting Leads to Deposit of Fake Tokens

## Summary
This bug report identifies an issue with the depositToken function in a contract implementation. It states that the current implementation does not check for the legitimacy of tokens being deposited into the contract, which leaves the contract vulnerable to malicious actors depositing fake or non-valuable tokens.

The report recommends that a token whitelist should be implemented, where only approved tokens can be deposited into the contract. Additionally, an onlyOwner function should be added to the contract which allows the owner to add and remove tokens from the whitelist. This would ensure that only legitimate tokens are being deposited into the contract, and prevent malicious actors from exploiting the loophole.

## Details
**Description:**

 The depositToken function in the current contract implementation allows the deposit of any ERC20 token without a check for its legitimacy. A malicious actor could use this loophole to deposit fake or non-valuable tokens into the contract.

**Recommendations:**

 A token whitelist should be implemented, where only approved tokens can be deposited into the contract. An onlyOwner function should be added to the contract which allows the owner to add and remove tokens from the whitelist.
