# `ReseedSilo` does not set the necessary user variables

## Summary
The `ReseedSilo` function is not properly setting user variables when migrating deposits. This can lead to unfair amounts of `plenty` for users, potentially causing a loss of funds. It is recommended to fix this issue by setting the variables correctly.

## Details
## Summary
`ReseedSilo` does not set the necessary user variables

## Vulnerability Details
When user deposits are migrated via `ReseedSilo`, user's storage variables such as `lastUpdated`, `lastSop` and `lastRain` are not set accordingly.

As these values would remain 0, this would allow users for unfair amounts of `plenty`

## Impact
Loss of funds

## Tools Used
Manual review


## Recommendations
Set the necessary variables accordingly.
