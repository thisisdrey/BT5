# Infrared contract cannot receive bribes in native BERA 

## Summary
The Infrared contract is unable to receive bribes in its native currency, BERA, because it does not have a method to handle it. This affects the functionality of the contract and may impact other contracts that rely on it. It is recommended to add a method to receive BERA in the contract.

## Details
## Infrared Contract Analysis

## Context
Infrared.sol#L459-L471

## Description
The `harvestBribes()` method is responsible for forwarding bribe assets to the `BribeCollector`. The Infrared contract aims to receive bribes in native BERA, which is then wrapped into WBERA. However, the Infrared contract is unable to receive bribes in native BERA because it lacks an external payable method or a receive/fallback method.

### Code Snippet
```solidity
function harvestBribes(address[] memory _tokens) external whenInitialized {
    for (uint256 i = 0; i < _tokens.length; i++) {
        address _token = _tokens[i];
        if (_token == DataTypes.NATIVE_ASSET) {
            wbera.deposit{value: address(this).balance}(); // conversion of native BERA to WBERA
            _token = address(wbera);
        }
        // ... skipped for simplicity
    }
}
```

## Impact
**Medium:** The intended functionality to receive native BERA bribes is not operational. Additionally, there may be contract implementations relying on Infrared to accept BERA as expected.

## Likelihood
**Medium:** While there is an alternative to send bribes as tokens to the Infrared contract, it is reasonable to expect that BERA, being the native currency, will be used for a significant portion of the total bribe value over time.

## Recommendation
Consider adding a payable receive method to the Infrared contract.

```solidity
receive() external payable {
    // This function is executed when the contract receives native BERA (without data)
}
```
