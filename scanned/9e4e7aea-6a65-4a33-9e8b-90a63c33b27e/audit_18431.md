# Functions withdraw() and directWithdraw() called from initWithdrawOnBehalf()

## Summary
The bug report states that there is an issue with withdrawing funds for the wrong account in the CommonV1.sol file. Specifically, the function initWithdrawOnBehalf() is causing the withdraw() and directWithdraw() functions to withdraw for the owner's address instead of the intended address. The recommendation is to fix this issue by ensuring that the correct address is used when calling these functions from initWithdrawOnBehalf(). 

After the main issue was resolved, auditors recommend verifying that reading balance and burning LP tokens should also be done with the intended address instead of the owner's address. The client has confirmed that the initWithdraw function is meant to burn balance from the contract or external account calling it.
