# Missing out-of-bounds check 

## Summary
This bug report is about a possible error in the LibBytes.sol file. The error occurs when trying to hash memory beyond the bounds of the bytes array. This can happen if the offset is greater than or equal to the length of the array, causing the function to attempt to access memory outside the allocated bounds. To fix this, the report recommends adding a check to validate the offset and length against the data length. This bug has been fixed in PR 10 and has also been fixed by Cantina Managed.

## Details
## Context

**LibBytes.sol#L5**

## Description

A possible out-of-bounds error could arise from attempting to hash memory beyond the bounds of the bytes array.

```solidity
function keccak(bytes memory data, uint256 offset, uint256 length) internal pure returns (bytes32 result) {
    assembly {
        result := keccak256(add(data, add(32, offset)), length)
    }
}
```

If the offset is greater than or equal to the length of the array, the pointer `add(data, add(32, offset))` will point outside the valid memory range. Therefore, the function will attempt to access memory outside the allocated bounds.

## Recommendation

Add the following check validating offset and length against `data.length`:

```solidity
function keccak(bytes memory data, uint256 offset, uint256 length) internal pure returns (bytes32 result) {
    + require(offset + length <= data.length, "Out-of-bounds access");
    assembly {
        result := keccak256(add(data, add(32, offset)), length)
    }
}
```
