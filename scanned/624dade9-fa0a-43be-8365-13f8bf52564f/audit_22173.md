# Duplicated Logs May Lead to Misinterpreted Events

## Summary
This bug report is about Denial of Service in the Ethereum blockchain. It occurs when the logs emitted during the creation of a contract are duplicated, fooling a client based on the blockchain events. The logs are added twice in the Program.java file at lines 637 and 673, resulting in any log in a constructor being duplicated. An exploit scenario is provided, where Bob creates a contract that stores investments which are logged. Due to the duplicated logs, Alice believes that Bob made more investments than he actually did. The recommendation is to remove the duplicated logs and create unit tests for greater code coverage to test the instructions’ behavior. This bug is similar to TOB-RSK-005 and TOB-RSK-006. Appendix D contains a test case to trigger the issue.

## Details
## Type: Denial of Service

## Difficulty: High

## Description

The Ethereum blockchain allows events recording through log operations. These events can be used by light clients to receive information from the blockchain. In RSK, the logs emitted during the creation of a contract are duplicated. As a result, a client based on the blockchain events can be fooled.

The logs are added to the program result during the creation of a contract in `createContract` (Program.java#637):

```java
getResult().merge(result);
```

`merge` is defined in `ProgramResult#203-206`:

```java
public void merge(ProgramResult another) {
    addInternalTransactions(another.getInternalTransactions());
    addDeleteAccounts(another.getDeleteAccounts());
    addLogInfos(another.getLogInfoList());
}
```

The logs are added a second time in `createContract` (Program.java#673):

```java
getResult().addLogInfos(result.getLogInfoList());
```

As a result, any log in a constructor is duplicated. Appendix D contains a test case to trigger the issue.

## Exploit Scenario

Bob creates a contract which stores some investments. The investments are logged. During the creation of the contract, Bob made an initial investment. Due to the duplicated logs, Bob’s investments are announced twice. As a result, Alice believes that Bob made more investments than he actually did.

## Recommendation

Remove the duplicated logs. Similar to **TOB-RSK-005** and **TOB-RSK-006**, create unit tests for greater code coverage to test the instructions’ behavior.
