# Owner can set initial timestamp several times in UnmarshalTokenVesting

## Summary
This bug report is about a problem with a function called "setInitialTimestamp" in a programming language called Solidity. The function is supposed to change a value called "isInitialized" to "true" but it is not working. The code snippet shows that there is a comment that might be causing the issue, so the recommendation is to remove the comment.

## Details
**Description**

Function setInitialTimestamp does not change the value of isInitialized to true.

**Snippet**:
```solidity
function setInitialTimestamp(uint256 _timestamp) external onlyOwner() notInitialized() {
// isInitialized = true;
_initialTimestamp = _timestamp;
```
**Recommendation**:

Remove comment.
