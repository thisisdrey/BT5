# M-1: Wfcash deposit might give incorrect amount of shares (wfcash) in some cases.

## Summary
The bug report discusses an issue with the Wfcash deposit function where it may not give the correct amount of shares in certain cases. This was found by a user named tvdung94 and could result in users receiving less or incorrect shares when using this function. The vulnerability lies in the preview function, which does not take into account the edge case where there is no fcash available on the markets. This means that the code will get the amount of shares from NotionalV2 instead, which is incorrect. The impact of this bug is that users may receive less shares than expected. The code snippet and tool used for the review are also provided. The recommendation is for the developers to handle the case when there is not enough fcash available on the markets. During the judging contest, one comment was left stating that using NotionalV2 is the intended behavior, but another user pointed out that the logic used in the preview function is not consistent with the logic used in the _mintInternal function.

## Found by 
tvdung94
## Summary
Wfcash deposit might not give correct amount of wfcash when there is no fcash available on the markets.
## Vulnerability Detail
The preview function, which calculates the amount of share  for minting, does not take the edge case, when there is not available fcash on the markets, into account. In this case, amount of shares should be equivalent to the amount of asset (ratio 1:1). However, the code will get amount of share on notionalv2 instead, which is incorrect.
## Impact
Users will get less/ incorrect amount of shares when using this deposit function.
## Code Snippet
https://github.com/sherlock-audit/2023-12-notional-update-5/blob/main/wrapped-fcash/contracts/wfCashERC4626.sol#L89-L106
## Tool used

Manual Review

## Recommendation
Consider adding code to handle the case when there is not enough available fcash on the markets.
