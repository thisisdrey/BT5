# Calls to AccessHub.setFeeSplit will revert due to _feeSplits having the wrong type 

## Summary
The AccessHub.sol#L149 function, AccessHub.setFeeSplit, has a bug where it can only facilitate a maximum percentage of 2.55% per pool, even though it is supposed to represent 100%. This means that some fees will not be claimed properly. The recommended solution is to change the type of _feeSplits to uint256[]. This bug has been fixed in commit fbdb3796 and verified by Cantina Managed. It is considered a medium risk bug.

## AccessHub.setFeeSplit Function Overview

## Context
AccessHub.sol#L149

## Description
The `AccessHub.setFeeSplit` function allows `SWAP_FEE_SETTER` to set the fee splits for pools (either Continuous Liquidity or not). The `_feeSplits` parameter is an array that represents percentages. 

In the case of non-Continuous Liquidity (CL) pairs (managed by `poolFactory`), 100% is represented as 10,000. However, since `_feeSplits` is of type `uint8[]`, it can only facilitate a maximum percentage of 255/10,000 = 2.55% per pool. This limitation can result in some fees going unclaimed.

To summarize, calls to the function may lead to reversion or can leave unclaimed tokens.

## Recommendation
Consider changing the type of `_feeSplits` to `uint256[]`.
