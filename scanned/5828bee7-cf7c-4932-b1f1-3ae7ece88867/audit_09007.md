# potCreator can block liquidity rotation 

## Summary
The bug report discusses an issue with the QuailFinance smart contract. The contract has a function called rotateLiquidity(), which can only be called by the potCreator. If the potCreator does not call this function for any reason, the funds in the contract will be permanently locked. The only way to withdraw the funds would be to use a merkle tree, which the report suggests changing to. There are a few reasons why the potCreator may not call rotateLiquidity(), such as losing their private key or being malicious. However, the contract cannot remove the requirement for the potCreator to call this function, as it is necessary for revealing the random number used to declare a winner in createPot(). The report recommends adding a time duration after which rotateLiquidity() cannot be called, allowing participants to withdraw their deposits.

## Details
## QuailFinance.sol#L142-L142

## Description
Only `potCreator` can call `rotateLiquidity()`. If `potCreator` for any reason doesn't call this function, funds are permanently locked. The only way to withdraw it would be to use the merkle tree, which we have recommended to change to so that it can't access user funds. `potCreator` may not call `rotateLiquidity()` for multiple reasons such as losing the private key or maliciousness. This require check can't be removed as only the requestor of the random number in `createPot()` should reveal the random number used to declare the winner; otherwise, the call to `entropy.reveal()` reverts.

## Recommendation
Consider adding a time duration after which `rotateLiquidity()` cannot be called, allowing participants to withdraw their deposits.
