# Locked ETH in Contract

## Summary
The codebase of the Ion Protocol has several contracts that can receive ETH but do not have a function to withdraw it. This means that any ETH sent to these contracts may be permanently lost. Specifically, the constructor of the TransparentUpgradeableProxy contract, the flashLeverageWeth function in the BalancerFlashloanDirectMintHandler contract, and the zapDepositWstEth function in the IonZapper contract are affected. To fix this issue, the payable attribute should be removed or a withdrawal feature should be added. This bug has been resolved in a recent pull request.

## Details
Throughout the [codebase](https://github.com/Ion-Protocol/ion-protocol/tree/98e282514ac5827196b49f688938e1e44709505a/), there are several contracts that include functions that can receive ETH but with no corresponding function for withdrawing ETH. As a result, any ETH sent to these contracts could be permanently lost. In particular:


* The [constructor](https://github.com/Ion-Protocol/ion-protocol/blob/98e282514ac5827196b49f688938e1e44709505a/src/admin/TransparentUpgradeableProxy.sol#L79) of the `TransparentUpgradeableProxy` contract
* The [`flashLeverageWeth` function](https://github.com/Ion-Protocol/ion-protocol/blob/98e282514ac5827196b49f688938e1e44709505a/src/flash/handlers/base/BalancerFlashloanDirectMintHandler.sol#L99-L106) in the `BalancerFlashloanDirectMintHandler` contract
* The [`zapDepositWstEth` function](https://github.com/Ion-Protocol/ion-protocol/blob/98e282514ac5827196b49f688938e1e44709505a/src/periphery/IonZapper.sol#L67) in the `IonZapper` contract


Consider removing the `payable` attribute or adding a withdrawal feature.


***Update:** Resolved in [pull request #23](https://github.com/Ion-Protocol/ion-protocol/pull/23).*
