# Users cannot specify a minimum desired interest

## Summary
This bug report is about the YVaultMarket.sol in the 88mph protocol. The bug is that the DInterest.deposit(amount, maturationTimestamp) function computes and returns an amount of interest that users receive at maturationTimestamp, but users cannot specify a minimum desired interest to guarantee that the amount of interest will be profitable. 

Exploit scenario is that Charlie, a user of the 88mph protocol, calls the DInterest.deposit function and the amount of interest returned is unacceptably low for him. To withdraw his funds for use in a more profitable market, Charlie has to pay the early withdrawal fee and the gas costs for another transaction. 

The short-term recommendation is to add a minInterestAmount parameter to the DInterest.deposit function and insert a require statement - require(interestAmount >= minInterestAmount) - to ensure that the interest amount meets or exceeds the minimum requirement set by the user. The long-term recommendation is to ensure that the sender has control over the minimum desired output amount for cases in which a transaction gives a sender an output amount in exchange for an input amount. Otherwise, state changes outside of the sender’s control, possibly caused by front-running, could result in an unexpectedly low output amount and a loss of funds for the user.

## Details
## Type: Optimization
## Target: YVaultMarket.sol

### Difficulty: Not Applicable

### Description
`DInterest.deposit(amount, maturationTimestamp)` computes and returns the amount of interest users will receive at `maturationTimestamp`. However, users calling this function cannot specify a minimum desired interest to guarantee that the amount of interest will be profitable.

### Exploit Scenario
Charlie, a user of the 88mph protocol, calls `DInterest.deposit`. The amount of interest that is computed and returned is unacceptably low for Charlie. However, he has already deposited his funds into 88mph. To withdraw his funds for use in a more profitable market, Charlie has to pay the early withdrawal fee and the gas costs for another transaction.

### Recommendations
- **Short term:** Add a `minInterestAmount` parameter to `DInterest.deposit` and insert a require statement—`require(interestAmount >= minInterestAmount)`—to ensure that the interest amount meets or exceeds the minimum requirement set by the user.
  
- **Long term:** For cases in which a transaction gives a sender an output amount in exchange for an input amount—during a trade, for example—ensure that the sender has control over the minimum desired output amount. Otherwise, state changes outside of the sender’s control, possibly caused by front-running, could result in an unexpectedly low output amount and a loss of funds for the user.
