# Object credentials retrieval

## Summary
The bug report discusses a vulnerability in the mina_getState function that allows a zkApp to retrieve all credentials stored in the pallad vault without the user's knowledge. This means that the user may unknowingly give access to sensitive information. A proof of concept is provided, showing how the code can be executed to request all object credentials without clearly indicating this to the user. The suggested remediation is to show the user all the credential objects that the zkApp will have access to, so they can make informed decisions. A patch has been implemented to mitigate this vulnerability by displaying the requested credentials to the user in the prompt window. The specific commit that addresses this issue is also mentioned.

## Details
## Security Considerations for zkApps

Using `mina_getState`, it is possible to retrieve all credentials without the user knowing which ones are going to be accessed by the zkApp.

## Proof of Concept

If a zkApp executes the following code, it will request all object credentials stored in the pallad vault. However, the prompt does not make it clear:

```javascript
await mina.request({
  method: "mina_getState",
  params: {
    query: {
      "@context": "https://www.w3.org/2018/credentials/v1"
    },
    props: ["id"]
  }
});
```

## Remediation

Consider retrieving all the credential objects that the zkApp will have access to and showing them to the user, so they can make better decisions.

## Patch

The commit `8f78cba` mitigated the vulnerability by showing the requested credentials to the user inside the prompt window.

```diff
>_ mina_provider.ts diff
+ const { query, props } = params as Validation.GetStateData
+ const credentials = await this.vault.getState(query as any, props)
const confirmation = await this.userPrompt("confirmation", {
  title: "Credential read request",
- payload: JSON.stringify(params),
+ payload: JSON.stringify({ ...params, credentials }),
```

By implementing this patch, users will have better visibility of the credentials being accessed, allowing for more informed decision-making.
