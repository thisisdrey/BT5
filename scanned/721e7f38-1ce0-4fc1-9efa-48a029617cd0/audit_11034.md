# Maximum Transfer Amount Is Bypassable

## Summary
A client has reported a bug in the MaviaToken.sol file where the state variable `tfMaxAmount` is not effectively limiting the transfer amount for non-whitelisted addresses. This means that although there is a maximum transfer cap set, users can still bypass it by making multiple transfers below the cap. The client recommends either removing the maximum transfer cap or implementing additional logic to track transfer amounts over a certain period of time. The client has marked this issue as "Acknowledged" and it is currently being looked into by the team.

## Details
**Update**
Marked as "Acknowledged" by the client. The client provided the following explanation:

> This is the intended behavior. The limit amount is applied during some time and for a single transfer, not accumulation type.

**File(s) affected:**`MaviaToken.sol`

**Description:** The state variable `tfMaxAmount` is used as a maximum Mavia token transfer cap for non-whitelisted addresses; this means that a single transfer of Mavia tokens cannot exceed the `tfMaxAmount`.

While the `tfMaxAmount` cap limits the transfer amount for a single transfer, a user can simply transfer multiple times where the transfer amount of each transfer is less than `tfMaxAmount`. Therefore, the maximum transfer amount cap is easily bypassable by any user.

**Recommendation:** Consider the need for a maximum transfer amount cap as contract admins can already limit when non-whitelisted users can transfer via the enforcement of `tfStartTime`.

If a maximum transfer amount cap is unnecessary, remove the relevant logic from the `_transfer()` function.

Alternatively, if a transfer amount cap is desired, additional logic must be implemented to track transfer amounts by address over an arbitrary amount of time (e.g., non-whitelisted users can only transfer up to 100 Mavia tokens within a single day).
