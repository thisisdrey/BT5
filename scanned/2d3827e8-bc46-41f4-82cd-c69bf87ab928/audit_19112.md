# Possible assets loss

## Summary
The bug report states that there is an issue with the code in the WETHGateway.sol file. Specifically, if the amount to be repaid is greater than the amount available, the code should call revert but it is not currently doing so. The recommendation is to update the code to call revert in this scenario.

## Details
##### Description
If `repayAmount > amount`, this means that Gateway repays credit from its own assets:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/core/WETHGateway.sol#L191
##### Recommendation
We recommend to call revert in case `repayAmount > amount`.
