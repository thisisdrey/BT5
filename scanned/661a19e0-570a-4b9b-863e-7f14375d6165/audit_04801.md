# Pool Fee Manipulation

## Summary
This bug report discusses an issue with the current implementation of the pool::get_fee function. The final fee calculation is based on the maximum value between the sum of fee_adjustment and base_fee, and zero. If the fee_adjustment is a negative value, the final fee will be zero, resulting in users being able to manipulate the token's price in the pool. This can lead to an imbalance in the pool's target ratios over time. The suggested solution is to use the maximum value between fee_adjustment + base_fee and base_fee, ensuring a minimum fee is always charged. This bug has been resolved in the patch 1f7a8f7.
