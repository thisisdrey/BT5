# Missing Zero Address Check

## Summary
The bug report states that there is a problem with the function `setAutomatedMarketMakerPair()`. This function does not have a check to prevent a zero address from being accidentally set to true in the `automatedMarketMakerPairs[]` mapping. This could cause unintended issues. The severity of this bug is low and it has been resolved. It is recommended to add a check for zero address in the function to prevent this issue from occurring.

## Details
**Severity**: Low

**Status**: Resolved

**Description**: 

Missing zero address checks for parameter pair of the function `setAutomatedMarketMakerPair()`. This could lead to zero address being accidentally set to true for `automatedMarketMakerPairs[]` mapping and can lead to unintended issues.
       
**Recommendation**: 

It is advised to add a zero address require check for the same.
