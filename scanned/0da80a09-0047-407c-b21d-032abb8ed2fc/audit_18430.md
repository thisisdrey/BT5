# Anyone can initiate withdrawal on behalf of

## Summary
This bug report is about a function called initWithdrawOnBehalf in a file called CommonV1.sol. The issue is that there is no check to ensure that the person calling the function is the owner, if the transaction sender is not 'onBehalfOf'. This goes against the comments in the code which state that only the owner should be able to initiate a withdrawal. The recommendation is to add a validation to check if the caller is the owner, when the sender is not 'onBehalfOf'. After consulting with the client, it was confirmed that the purpose of this function is to burn balance from the user calling it.

## Details
**Description**

CommonV1.sol: function initWithdrawOnBehalf().
There is no validation that the caller of the function is the owner, in case the sender of the
transaction is not 'onBehalfOf'. The comments to the function say that only the owner can
initiate withdrawal.

**Recommendation**:

Add a validation, that the caller is the owner, in case the sender of the transaction is not
'onBehalfOf'.

**Post-audit:**

balance from the user calling it
