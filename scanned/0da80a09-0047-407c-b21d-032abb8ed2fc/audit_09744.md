# Using updateweight may break the total weight assumption 

## Summary
The report discusses a bug in the Omnipool smart contract's `updateWeight` function, which allows for updating the target weight of a single pool. This function can cause inaccurate calculations in other functions of the contract by breaking the assumption that the total of all target weights should equal `ScaledMath.ONE`. The recommendation is to restrict weight updates to the `updateWeights` function only, or to remove the `updateWeight` function entirely.

## Details
## Omnipool.updateWeight Analysis

## Context
- **File:** Omnipool.sol
- **Line:** 1032

## Description
The `Omnipool.updateWeight` function allows updating the target weight of a single pool. The `updateWeights` function ensures that the total of all target weights should equal `ScaledMath.ONE`. However, using `updateWeight` can break this assumption, leading to inaccurate calculations in several functions of the Omnipool.

## Impact
Breaking the assumption that `sum(target weights) == ScaledMath.ONE` will disrupt Omnipool's calculations. For instance, the `_getDepositPool` function calculates `targetAllocation_` under the assumption through the `mulDown` function:

```solidity
uint256 targetWeight = (pool.targetWeight);
uint256 targetAllocation_ = totalUnderlying_.mulDown(targetWeight);
```

## Recommendation
Consider restricting weight updates to the use of `updateWeights` only, ensuring that the total target weights assumption remains valid. This can be accomplished by deleting the `updateWeight` function.
