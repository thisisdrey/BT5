# M-1: Incorrect assumption of fixed 1e18 scaling in Pendle PT Oracle Rate

## Summary
A bug has been found in the `OraclePendlePT` contract, which assumes that the `getPtToSyRate()` function always returns a rate scaled to 1e18. However, for some Pendle markets, this function returns values with larger decimal scaling, causing incorrect PT pricing. This can have a critical impact on contracts such as `Collateral` and can result in the loss of funds for the protocol. No response or mitigation has been provided yet.

### Summary

The `OraclePendlePT` contract assumes that the `getPtToSyRate()` function from Pendle's oracle always returns a rate scaled to 1e18. However, some Pendle markets (e.g., 0x83916356556f51dcBcB226202c3efeEfc88d5eaA) return values with larger decimal scaling, which causes incorrect PT pricing.

### Root Cause

https://github.com/sherlock-audit/2025-08-usg-tangent/blob/main/tangent-contracts/src/USG/Oracles/Pendle/OraclePendlePT.sol#L45
`getPtToSyRate()` function will return ptRate in larger decimals for some markets.
