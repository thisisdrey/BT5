# `ECDSA` Library Vulnerable to Signature Malleability

## Summary
The team recently updated the Openzeppelin versions in the code, but it has caused a problem in the `PMXBonusNFT.sol`, `ReferralProgram.sol`, and `WhiteBlackListReferral.sol` files. This is because the code is still using an older version of `ECDSAUpgradeable` which has a vulnerability related to signature malleability. This means that the code is at risk of being hacked. To fix this, it is recommended to update the `ECDSAUpgradeable` library to at least version `4.7.3` or the latest stable version.

## Details
**Update**
The team updated the Openzeppelin versions in the following commits: `ffed20b` and `1382573`.

**File(s) affected:**`PMXBonusNFT.sol`, `ReferralProgram.sol`, `WhiteBlackListReferral.sol`

**Description:** The protocol currently utilizes OpenZeppelin's `ECDSAUpgradeable` version `4.3.1` in contracts `PMXBonusNFT`, `ReferralProgram`, and `WhiteBlackListReferral` for validating registration using `ECDSAUpgradeable.recover(bytes, bytes)`. However, it is important to note that version `4.7.3` introduced a high-severity patch to address a [vulnerability related to signature malleability](https://github.com/OpenZeppelin/openzeppelin-contracts/security/advisories/GHSA-4h98-2769-gh6h) in the `ECDSA.recover()` and `ECDSA.tryRecover()` functions. These functions were found to be vulnerable due to accepting EIP-2098 compact signatures in addition to the traditional 65-byte signature format. This vulnerability affected the functions that accept a single bytes argument, not the ones that take `r, v, s` or `r, vs` as separate arguments. As a result, the `PMXBonusNFT`, `ReferralProgram`, and `WhiteBlackListReferral` contracts are impacted since they utilize the single bytes argument version of `ECDSA`, and version `4.3.1` is affected by this vulnerability.

**Recommendation:** We strongly recommend upgrading the `ECDSAUpgradeable` library to at least version `4.7.3` or the latest stable version available to incorporate the security fixes.
