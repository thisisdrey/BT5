# An Approver Can Approve Their Own Submission

## Summary
The client has reported a bug in the `BountyV1.sol` file where an approver can approve their own submission if they call `grantApprover(account)` after making a submission. This can be fixed by keeping track of users with active submissions using an enumerable mapping and validating that they are not in the mapping when promoting users. This fix should not significantly increase gas usage. The client has marked this bug as "Fixed" in a specific commit.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `92204393279e53c51427ac6dfff669c109edb4b7`. The client provided the following explanation:

> We opted to directly disallow approval by an address of submissions from that address.

**File(s) affected:**`BountyV1.sol`

**Description:** An approver can approve their own submission if `grantApprover(account)` is called after the `account` has made a submission.

**Recommendation:** Keep track of users who have active submissions in the protocol. This can be done using an enumerable mapping. Then when promoting users, validate that they are not in the mapping. The lookup is quite cheap and should not spike gas usage.
