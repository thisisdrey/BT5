# Single Signer Failing the Consensus

## Summary
The bug report discusses an issue with the `Reject.sol` function in the code. This function is used to check for consensus failure in a group of signers. However, the function does not properly check for duplicate signers, which can lead to a malicious signer faking multiple submissions and causing the system to pause. The team has fixed this issue by adding extra inputs to the function to ensure signer uniqueness and integrity. The report recommends adding a check to prevent duplicate members in the `custodians` input.

## Details
**Update**
The team fixed the issue by checking signer uniqueness and integrity with extra inputs: `_signatures` and `_indexes` to the `Reject.sol:reject` function. The `_signatures` is required to be sorted by the recovered addresses (verified in the function `validateSignatureOrdering` in L99-136), thus proving the uniqueness of the signers. The `_indexes` input maps the `_signatures` array and the `_submissions` array to ensure the input integrity.

**File(s) affected:**`facets/Reject.sol`

**Description:** The function `Reject.sol:reject` (L56-126) checks whether there is consensus failure or not with the checkpoint `_submissions` input data. The calculation is based on the number of `custodians` in L96-97 retrieved from the `_submissions[i]`. However, the `reject` function does not check whether it counts the specific custodian multiple times. Thus, a single signer can fake several submissions and pretend that there are several groups.

**Exploit Scenario:**

1.   Given a group of 5 signers, suppose the canonical group has 3 signers, and the malicious group has 2.
2.   A malicious signer creates 4 different `_submissions` with its signature. Although the `custodians` in L96 all point to the same attacker in each for loop step, the `submissionCount` (L102) will become 4, and each `submissionCounts[i]` (L98) is 1. The `maximumMissingSubmissions` (L106) will be 1.
3.   The `reject` function will pass and pause the system.

**Recommendation:** Add a check that `custodians` cannot have duplicated members.
