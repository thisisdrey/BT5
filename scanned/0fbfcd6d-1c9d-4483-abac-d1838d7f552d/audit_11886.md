# Centralized Design of `salesAddress`

## Summary
A client has reported a bug in the `UnikuraPhygitalCollection.sol` file, which is responsible for initializing the `UnikuraPhygitalCollection` contract. The bug causes all tokens to be minted to the `salesAddress` without any clear documentation or explanation. This could potentially lead to a security risk if the `salesAddress` is compromised. The client has marked this bug as "Unresolved" and has requested for the development team to review the system design and document the intended behavior clearly in the user-facing documentation.

## Details
**Update**
Marked as "Unresolved" by the client. We believe this item is acknowledged by the development team, as the centralized design risk is well notified. The client provided the following explanation:

> Initially, the sales address will be specified only for velvett's wallet. When we provide the functionality externally, we will add a description of the risk.

**File(s) affected:**`UnikuraPhygitalCollection.sol`

**Description:** Upon the initialization of the `UnikuraPhygitalCollection` contract, all tokens are minted to the `salesAddress`. Such design is not documented and it is unclear why the `salesAddress` receives max token supply. In case of compromise of this address, a malicious user can have access to a vast majority of the token supply.

**Recommendation:** Review the system design. If this is the intended behavior, document it clearly in the user-facing documentation.
