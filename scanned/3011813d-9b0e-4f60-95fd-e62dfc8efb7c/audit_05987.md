# getBidValue() is not always used

## Summary
The report discusses a bug in the code related to sorting bids. The bug is classified as a medium risk and occurs in three different files: Sorter.sol, Atlas.sol, and Escrow.sol. The function getBidValue() is used in Sorter to sort bids, but it is not used in Atlas and Escrow, where solverOp.bidAmount is used instead. This may cause issues in the future when the values are not the same. The recommendation is to either use getBidValue() in all three files or remove it from Sorter. The bug has been solved in PR 372 by removing the call from Sorter. The solution has been verified by Spearbit.

## Details
## Security Advisory

## Severity
**Medium Risk**

## Context
- `Sorter.sol#L118`
- `Atlas.sol#L325-L328`
- `Escrow.sol#L355`
- `SwapIntent.sol#L243-L245`

## Description
`Sorter` uses `getBidValue()` to sort the bids. However, `Atlas / _bidFindingIteration()` and `Escrow / _getBidAmount()` do not utilize this function and instead use `solverOp.bidAmount` directly. 

In the example code, these values are the same because of the following function:

```solidity
function getBidValue(SolverOperation calldata solverOp) public pure override returns (uint256) {
    return solverOp.bidAmount;
}
```

However, in general cases, they might differ.

## Recommendation
- Use `getBidValue()` in `Atlas / _bidFindingIteration()` and `Escrow / _getBidAmount()`.
- If the usage is limited, consider removing it from `Sorter` for consistency.

## Fastlane
Solved in PR 372 by removing the call from the `Sorter`.

## Spearbit
Verified.
