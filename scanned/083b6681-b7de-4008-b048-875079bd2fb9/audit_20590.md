# [M-03] `timelockERC721Keys` could exceed the block size limit

## Summary
The bug report is about a potential issue in the code for the Visor platform, submitted by Sherlock and also found by shw. The problem is on line 504 of the `Visor.sol` file, where looping through the `timelockERC721Keys` could exceed the block size limit. The report recommends transferring by index instead of token ID to avoid this issue. The team at Visor has acknowledged the problem and plans to fix it in their next version by capping the size of the NFT collection. The severity of the bug has been raised to medium because it could occur if the project is successful. The Visor team has since patched the issue by using EnumerableSets.
