# Proposals Can Reach Quorum Without Any Votes

## Summary
The bug report is about an issue with the minting of Anvil tokens, which is causing the total voting power to always be recorded as zero. This is due to an empty value being used to calculate the past total supply of tokens, which is then used to determine the quorum for proposals. As a result, proposals are reaching quorum without any actual votes, allowing them to be executed. The bug can be fixed by modifying the `getPastTotalSupply` function to return the correct total supply of Anvil tokens. The issue has been resolved in a recent pull request on GitHub.

## Details
The minting of Anvil tokens does not update the total voting power recorded in the `_totalCheckpoints` struct, hence `_totalCheckpoints` is always empty. This empty value is used to [calculate the past total supply of tokens](https://github.com/AmperaFoundation/sol-contracts/blob/c6e940c12044c8994f778c978e34582449229df8/contracts/governance/AnvilVotes.sol#L114C16-L114C33) in the `getPastTotalSupply` function of the `AnvilVotes` contract, incorrectly indicating the total supply as zero. The `getPastTotalSupply` function is called in the `GovernorVotesQuorumFractionUpgradeable` contract to [calculate](https://github.com/OpenZeppelin/openzeppelin-contracts-upgradeable/blob/26fa134e226d19762f84130b73ded1927f0327fd/contracts/governance/extensions/GovernorVotesQuorumFractionUpgradeable.sol#L91) the `quorum`, which makes the returned quorum always be equal to zero. If the quorum is zero, proposals would [reach quorum](https://github.com/OpenZeppelin/openzeppelin-contracts-upgradeable/blob/26fa134e226d19762f84130b73ded1927f0327fd/contracts/governance/extensions/GovernorCountingSimpleUpgradeable.sol#L82) even without anyone voting on them. Once the proposal deadline has passed, the proposal can be executed. The only way to prevent a malicious proposal from getting executed is to have [enough votes against it](https://github.com/OpenZeppelin/openzeppelin-contracts-upgradeable/blob/26fa134e226d19762f84130b73ded1927f0327fd/contracts/governance/extensions/GovernorCountingSimpleUpgradeable.sol#L92).


Since the minting of Anvil tokens is [restricted](https://github.com/AmperaFoundation/sol-contracts/blob/c6e940c12044c8994f778c978e34582449229df8/contracts/governance/Anvil.sol#L43-L46) to the constructor and there is no possibility of burning Anvil tokens, consider modifying the `getPastTotalSupply` function so that it returns the total supply of Anvil tokens for the correct calculation of quorum.


***Update:** Resolved in [pull request \#281](https://github.com/AmperaFoundation/sol-contracts/pull/281) at commit [e1352ca](https://github.com/AmperaFoundation/sol-contracts/commit/e1352ca574dd466c37d459f1f776362b24cae0c7).*
