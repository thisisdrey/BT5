# Utilization Of Fixed Size Array

## Summary
The bug report is about a conditional check in the code for automatically claiming staked tokens. The check is supposed to pause the process if the array of locked stakes is empty. However, due to a coding error, the check will always evaluate to false and the code block within the if statement will never execute. This means that even when all liquid stakes are withdrawn and there are no funds left in the locked stakes, the automatic claiming mechanism will continue to operate without pause. The report recommends re-evaluating the condition check to ensure it works properly. The bug has been fixed in the latest patch.

## Details
## Analysis of the `remove_locked_stake` Function

In the `remove_locked_stake`, there is a conditional check to pause the auto-claim process if certain conditions are met. It checks if the array of locked stakes (`locked_stakes`) is empty. However, `user_staking.locked_stakes.is_empty()` will never evaluate to true, as the `locked_stakes` array is defined as a fixed-size array, and its length is fixed at compile-time and does not change.

```rust
// Remove one stake at a time
pub fn remove_locked_stake(
    ctx: Context<RemoveLockedStake>,
    params: &RemoveLockedStakeParams,
) -> Result<()> {
    [...]
    // Pause auto-claim if there are no more staked tokens,
    if !ctx.accounts.stakes_claim_cron_thread.paused
        && user_staking.liquid_stake.amount == 0
        && user_staking.locked_stakes.is_empty()
    {
        let authority_seeds: &[&[&[u8]]] = &[&[b"transfer_authority", &[ctx.accounts.cortex.load()?.transfer_authority_bump],]];
        sablier_sdk::cpi::thread_pause(CpiContext::new_with_signer(
            ctx.accounts.sablier_program.to_account_info(),
            sablier_sdk::cpi::ThreadPause {
                authority: ctx.accounts.transfer_authority.to_account_info(),
                thread: ctx.accounts.stakes_claim_cron_thread.to_account_info(),
            },
            authority_seeds,
        ))?;
    }
    [...]
}
```

This condition being perpetually false means that the code block within the `if` statement will never execute. As a result, even when all liquid stakes are withdrawn (`user_staking.liquid_stake.amount == 0`) and there are no funds left in the locked stakes, the auto-claim mechanism (`stakes_claim_cron_thread`) will continue to operate normally without pause in the `remove_locked_stake` and `remove_liquid_stake` instructions.

## Audit Findings

### Adrena Audit 04 — Vulnerabilities

**Remediation**
Re-evaluate the condition checking for the empty `locked_stakes` vector to ensure it executes properly.

**Patch**
Resolved in commit `d72bd2a`.
