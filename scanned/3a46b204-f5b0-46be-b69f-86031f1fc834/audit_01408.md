# (e *encodedBlobStore) DeleteEncodingRequest() needs a write lock

## Summary
The bug report describes a problem with the `DeleteEncodingRequest()` function in the `encodedBlobStore.go` file. This function holds a read lock on `e.requested`, but it is not enough because the `e.requested` is also modified in the function. This can cause issues with the disperser's performance. The recommendation is to change the function's `Rlock()` to a `Lock()` to prevent this problem.

## Details
## Severity: Medium Risk

## Context
`disperser/batcher/encoded_blob_store.go#L83`

## Description
`(e *encodedBlobStore) DeleteEncodingRequest()` holds a read lock `one.mu` which is guarding `e.requested`. This is not sufficient because `e.requested` is modified with a `delete()` in this function. This is called regularly by `(e *EncodingStreamer) ProcessEncodedBlobs` when the `EncodingResultOrStatus` chan receives encoding requests. A data race here threatens the liveness of the disperser.

## Recommendation
Change this function's `Rlock()` to a `Lock()`.
