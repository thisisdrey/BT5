# canExecute causes ambiguity

## Summary
This bug report is about an issue with a code in Executor.sol file. The code is responsible for executing a task, but it is causing the task to be marked as "canExecute=false". This problem also occurs when a successful task execution is followed by a failed task execution, resulting in the same "canExecute=false" state. To fix this issue, the recommendation is to use an enum instead of a boolean value to represent the different states of execution, and to add necessary checks in the relevant methods to validate the states of execution. This bug has been resolved with a medium severity level.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In Executor.sol - executeTask method executes task that result on marking canExecute=false. It is also the case from the perspective of layer-1 that a failed call to Executor::readyTask after a successful Sender::readyTask that ends up having canExecute=false.

**Recommendation** 

Apply an enum instead of bool canExecute that represents 3 states of execution
NotReady
Ready
Executed
Then apply the required checks that validate the states of execution in the relevant methods.
