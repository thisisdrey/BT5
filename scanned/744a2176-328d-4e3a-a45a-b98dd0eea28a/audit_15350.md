# Incorrect Cumulative Value Used To Update Perp LpState

## Summary
The PerpEngineState contract has a bug in the updateStates function where it is using the wrong variable to update the base balance of lpState. This results in a difference of zero in the vQuoteBalanceX18 if the base balance is greater than zero and a difference in the amount if the base balance is less than zero. The bug can be fixed by using the correct variable lpState.lastCumulativeFundingX18 instead of state.cumulativeFundingLongX18. This has been fixed in the patch #116.

## Details
## PerpEngineState Contract Update

In the `PerpEngineState` contract, the `updateStates` function updates the base balance of `lpState` (pool) by using the `updateBalance` function. For this, it requires a `lastCumulativeFundingX18` so that `lpState` holds a `lastCumulativeFundingX18` value.

### Code Snippet

```solidity
contracts/PerpEngineState.sol
```

```solidity
Balance memory balance = Balance({
    amountX18: lpState.base.fromInt(),
    vQuoteBalanceX18: 0,
    lastCumulativeFundingX18: state.cumulativeFundingLongX18
});
_updateBalance(state, balance, 0, 0);
if (lpState.supply != 0) {
    lpState.cumulativeFundingPerLpX18 += balance.vQuoteBalanceX18.div(lpState.supply.fromInt());
}
lpState.lastCumulativeFundingX18 = state.cumulativeFundingLongX18;
```

### Issue

The function is using `state.cumulativeFundingLongX18` instead of `lpState.lastCumulativeFundingX18`, resulting in a zero `vQuoteBalanceX18` difference if `lpState.base > 0`. The difference `(cumulativeFundingShortX18 - cumulativeFundingLongX18)` is applied on the amount if `lpState.base < 0.

### Remediation

Use `lpState.lastCumulativeFundingX18` instead of `state.cumulativeFundingLongX18`.

### Patch

The usage of `lpState.lastCumulativeFundingX18` has been fixed in #116.
