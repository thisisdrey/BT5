# Unbounded Gas In Queries

## Summary
The bug report describes a vulnerability in the callEVM function that allows for unlimited contract execution during EVM static calls. This means that contracts invoked through external queries can consume unlimited gas, potentially leading to long-running computations. Attackers can exploit this by launching a denial of service attack on the blockchain network, deploying computationally expensive contracts, and querying them to consume excessive resources, causing the EVM to process transactions slowly or even grind to a halt. The suggested fix is to add a gas limit to the query processing and the bug has been fixed in version 21df3fe.

## Details
## EVM Gas Limit Considerations

By setting `seiGasRemaining` to `math.MaxUint64` when the gas limit is zero or negative, `callEVM` essentially allows for unbounded contract execution during EVM static calls. This implies that contracts invoked via external queries may consume unlimited gas, resulting in potentially long-running computations.

```go
func (k *Keeper) callEVM(ctx sdk.Context, from sdk.AccAddress, to *common.Address, val *sdk.Int, data []byte, f EVMCallFunc) ([]byte, error) {
    sender := k.GetEVMAddressOrDefault(ctx, from)
    seiGasRemaining := ctx.GasMeter().Limit() - ctx.GasMeter().GasConsumedToLimit()
    if ctx.GasMeter().Limit() <= 0 {
        // infinite gas meter (used in queries)
        seiGasRemaining = math.MaxUint64
    }
    [...]
    return ret, nil
}
```

Attackers may exploit this by launching a denial of service attack on the blockchain network, deploying contracts with computationally expensive operations, then querying it to consume excessive resources, causing the EVM to process transactions slowly or even grind to a halt.

## Remediation
Ensure to add a gas limit to the query processing.

## Patch
Fixed in commit `21df3fe`.
