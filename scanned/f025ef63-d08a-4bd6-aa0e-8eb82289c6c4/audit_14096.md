# Ownership Can Be Renounced

## Summary
The bug report states that there is an issue with the `renounceOwnership()` function in contracts that inherit from `Ownable`. This function allows the owner to leave the contract without an owner, which would make it impossible to execute any functions that are protected by the `onlyOwner` modifier. This is a problem in the `Erc20Reserve.sol` contract, as the `transfer` function, which is guarded by `onlyOwner`, is the only way to retrieve funds from the contract. If the owner renounces ownership, the funds in the contract will be lost forever. The recommendation is to confirm if this is intended behavior, and if not, to fix it by overriding the `renounceOwnership()` function to always revert.

## Details
**Update**
Acknowledged. The client's response: "We confirm this behaviour is intended."

**File(s) affected:**`Erc20Reserve.sol`, `All contracts inheriting from Ownable or OwnableUpgradeable`

**Description:** Contracts inheriting from `Ownable` contain a function called `renounceOwnership()` which allows the `owner` to leave the contract without an `owner`. This would leave every function guarded by the `onlyOwner` modifier unable to be executed.

In the case of `Erc20Reserve`, the `transfer` function, guarded by `onlyOwner`, is the only way of retrieving funds from the contract. If ownership is renounced in this context, the funds in the contract would be lost forever.

**Recommendation:** Confirm whether this is the intended behavior. If not, override the `renounceOwnership()` function to always revert.
