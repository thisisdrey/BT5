# Curve part is not reduced by negative variable interest growth when closing longs

## Summary
The bug report is about a problem with the HyperdriveMath.sol code, specifically lines 197-202. When closing longs (a type of financial transaction) during a period of negative interest, the trader's proceeds are reduced proportionally to the negative growth. However, this reduction is only applied to one variable (shareProceeds) and not another (shareReservesDelta). This means that the curve part of the proceeds is not being reduced, even though it should be. Additionally, the negative interest region is being compared to the wrong value, which could lead to incorrect calculations. The recommendation is to fix this by scaling down shareReservesDelta by the same factor, which would also reduce the flat part of the proceeds. This issue was discovered by the team during an audit and has already been fixed.

## Details
## Severity: Medium Risk

## Context
**File:** `HyperdriveMath.sol`  
**Lines:** `197-L202`

## Description
When closing longs during a period of negative interest, the trader's proceeds are reduced proportionally to the negative growth. However, this is only done on the `shareProceeds` variable which represents both flat and curve parts. The curve part `shareReservesDelta` is not reduced by this amount but it should be. It also checks the negative interest region compared to the beginning of the deployment (`_initialSharePrice`) instead of the beginning of opening the longs (`_openSharePrice`).

## Recommendation
Consider scaling down `shareReservesDelta` by the same factor. This means the curve part `shareReservesDelta` would be reduced but also the flat part `shareProceeds - shareReservesDelta`. This was independently discovered by the team during the audit and it is already fixed here.
