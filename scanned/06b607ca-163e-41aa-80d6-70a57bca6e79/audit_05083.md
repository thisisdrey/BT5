# M-3: Contracts of the codebase will not strictly compliant with the ERC-1504.

## Summary
The bug report states that the contracts in the codebase are not compliant with the ERC-1504 standard. This is causing issues with the readme. The root cause of this is that the contracts are using openzepplin upgradable contracts as a base, which do not follow the ERC-1504 standard. The standard requires the contracts to have a handler contract, a data contract, and optionally an upgrader contract. However, the contracts in the codebase do not have a data contract and have data stored inside the handler contract. This breaks the readme. There is no response from the internal or external pre-conditions, attack path, or PoC. The suggested mitigation is to make the contracts strictly compliant with ERC-1504.

As per readme:
> Is the codebase expected to comply with any EIPs? Can there be/are there any deviations from the specification?
> 
> Strictly compliant: ERC-1504: Upgradable Smart Contract

But the contracts of the codebase uses openzepplin upgradable contracts as base contract which are not compliant with ERC-1504. As per [ERC-1504](https://eips.ethereum.org/EIPS/eip-1504), the upgradable contract should consists of have handler contract, data contract and optionally the upgrader contract.
But the contracts of the codebase are not compliant with ERC-1504 because they has no data contract and has data inside the handler contract.
