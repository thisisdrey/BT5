# Consider Enforcing Pda Constraint Validation

## Summary
This bug report is about a problem with the code that handles redemption offers and requests in a specific program. The client has marked it as "Fixed", meaning that they believe it has been resolved. The issue was that certain instructions did not properly check the constraints of the accounts involved, which could lead to incorrect accounts being used. The recommendation is to add explicit checks for these constraints in the affected instructions and to review the entire codebase for similar issues. This fix will also address a previous issue identified in an audit.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `a57cc029014452eb31ab13a5478c4d75edd86991`. The client provided the following explanation:

> Added missing PDA seed constraints

**File(s) affected:**`programs/onreapp/src/instructions/redemption/*`

**Description:** The `MakeRedemptionOffer` instruction defines `redemption_offer` as a PDA. However, subsequent instructions that accept the `redemption_offer` account do not revalidate its PDA derivation, which may allow incorrect redemption offer accounts to be passed in.

Similarly, the `redemption_request` account is defined as a PDA in the `CreateRedemptionRequest` instruction, but its PDA constraints are not validated in other instructions that interact with it.

**Recommendation:** To improve safety and consistency, consider adding explicit PDA constraint validation for the `redemption_offer` and `redemption_request` accounts in the `CancelRedemptionRequest`, `CreateRedemptionRequest`, `FulfillRedemptionRequest`, and `UpdateRedemptionOfferFee` instructions. Additionally, perform a comprehensive review of the codebase to identify and remediate any other instances where PDA accounts are used without proper constraint verification. Adding the mentioned PDA constraint validations will also address [ONR-1](https://certificate.quantstamp.com/full/onre-re-audit-2/41c2f85f-90ce-42ee-aac8-a810e0f0ec99/index.html#findings-qs1).
