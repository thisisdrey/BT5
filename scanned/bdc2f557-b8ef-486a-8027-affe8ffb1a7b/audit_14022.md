# Incorrect check for calldata overrun in CALLDATACOPY

## Summary
This bug report is about a critical risk issue in the code of calldata-returndata-code.zkasm#L120. The code is intended to check if offset + size is lower than calldata size, however, the code currently checks if txCalldataLen + size is lower than offset, which is incorrect. The recommendation to fix this issue is to assign offset + size to A and txCalldataLen to B. Polygon-Hermez has already been aware of this issue and it has been fixed in the PR #268.

## Details
## Severity: Critical Risk

## Context
calldata-returndata-code.zkasm#L120

## Description
The intention of the code is to check `offset + size < txCalldataLen`:
```
; if offset + size is lower than calldata size => length
$ :LT,JMPC(opCALLDATACOPYX0, opCALLDATACOPYloop)
```
But at this point `A` contains `txCalldataLen + size` and `B` contains `offset`, so it checks `txCalldataLen + size < offset`, which doesn’t make sense.

## Recommendation
Assign `offset + size` to `A` and `txCalldataLen` to `B`.

## Polygon-Hermez
We were already aware of this issue, it is reported here and fixed in PR #268.
