# Insufficient Documentation

## Summary
The report states that updates have been made to the user-facing documentation for Archi. However, there are some issues with the documentation. The specifications listed are not clearly connected to the codebase and the purpose and function of each contract is not explained. Additionally, the documentation includes links to resources that are not included. The code documentation also lacks sufficient comments. The recommendation is to thoroughly describe how the specifications are implemented in the contracts and to provide details on the economic incentives for lenders, farmers, and liquidators. Architecture diagrams should also be included to show all workflows involving token and asset transfers. It is important to document which contracts hold funds and which assets are at risk. Lastly, all functions, events, and contracts should be documented using the NatSpec standard.

## Details
**Update**
Additional information has been added to Archi's user-facing documentation including architecture diagrams and user guides. Particularly, large improvements have been made in adding natspec and code comments where appropriate.

**Description:** Though documentation was provided, the specifications listed were not mapped to the codebase. The purpose and function of each contract was not detailed. The documentation would refer to links to resources that were not included. The code documentation had minimal comments.

**Recommendation:** The documentation should thoroughly describe how the specifications have been implemented in the contracts. The economic incentives for lenders, farmers, and liquidators should be detailed, with references to code segments. All workflows transferring tokens and other liquid assets should be detailed in architecture diagrams. Document which contracts custody funds and which assets are at risk. All functions, events, and contracts should be documented using the NatSpec standard.
