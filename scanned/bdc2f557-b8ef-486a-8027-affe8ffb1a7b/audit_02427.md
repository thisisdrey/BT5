# Converting a potentially 512-bit number to a 256-bit number, causing panic

## Summary
The bug report is about a code error in a specific file and line of a program. The code in question involves a conversion of numbers and can cause the program to crash, disrupting the proof flow. The suggested solution is to use a different type of number or add error handling. The issue has been resolved in a recent update.

## Details
In [`crates/boojum/src/gadgets/u256/mod.rs`](https://github.com/matter-labs/zksync-crypto/blob/ccf5fa30ee2c94b594126d7b4b75770cafbbab21/crates/boojum/src/gadgets/u256/mod.rs#L391), line 391 has the code:

`let q: U256 = q.try_into().unwrap();`

Here, `q` equals ⌊abm⌋\lfloor \frac{ab}{m} \rfloor⌊mab​⌋, where aaa and bbb are 256-bit numbers. Since `q` can be up to 512 bits, the conversion to `U256` may fail. If it does, `unwrap()` causes the `modmul` function to panic, disrupting the proof flow.

Consider using `U512` for `q` or adding error handling to avoid panics.

***Update:** Resolved in [pull request #92](https://github.com/matter-labs/zksync-crypto/pull/92).*
