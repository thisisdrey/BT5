# Initializers could be front-run

## Summary
This bug report is about the vulnerability of Initializers in a script deployment. This vulnerability can be exploited by an attacker to set their values, take ownership of the contract, and in the best case force a re-deployment. To prevent this from happening, the recommendation is to make the initialize function executed in the same transaction. This will ensure that the attacker cannot front-run the Initializers and exploit the vulnerability.

## Details
**Description:**

Initializers could be front-run, allowing an attacker to either set their values, take ownership of the contract, and in the best case force a re-deployment. The script deployment is permitting this front-run attack.

**Recommendations:** 

Make the initialize function executed in the same transaction.
