# Dust Is Paid to Last Payee in the _payees Array

## Summary
The bug report discusses an issue with the `Splitter.sol` contract at line 117, where the `payment` variable is set to the remaining balance for the last address in the `_payees` array. This can create a fairness issue for other `_payees` as it does not distribute the remaining wei equally. The suggested solution is to pay all payees based on their share of the total balance. This issue has been resolved in a recent pull request at commit 822347b.

## Details
[Line 117](https://github.com/oceanprotocol/vw-cli/blob/0397c74da60ae50db5c6414307849120b899af6e/contracts/Splitter.sol#L117) sets the `payment` variable to the remaining balance for the last address in the `_payees` array. While this keeps the rounded-down wei from remaining in the contract after a single payment, it also creates a fairness issue from the point of view of the other `_payees`.


Consider paying all payees `balance * _shares[payee] / _totalShares`.
