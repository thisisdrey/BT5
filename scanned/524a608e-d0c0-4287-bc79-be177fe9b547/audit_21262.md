# `maxCap` can go under `totalSupply`

## Summary
This bug report is about a problem with the BasketFacet.sol contract in the ExperiPie project. The issue is that if the supply is bigger than the new maxCap, the code should require that the totalSupply is less than or equal to the maxCap. To fix this issue, the code should include a line that requires this, such as `require(totalSupply <= _maxCap, '...')`.

## Details
##### Description
If the supply is bigger than new `maxCap`: https://github.com/pie-dao/ExperiPie/blob/0.0.2/contracts/facets/Basket/BasketFacet.sol#L242

##### Recommendation
We recommend to add `require(totalSupply <= _maxCap, '...')`.
