# [H03] _getImpliedRate does not handle errors

## Summary
This bug report describes an issue in the `_getImpliedRate` function of the Swapnet-lite protocol. This function calls `_getExchangeRate` and uses it in calculations. In certain failure cases, the value `0` will be returned from `_getExchangeRate`, resulting in an underflow due to an unprotected subtraction. This can cause future rate calculations to be too high, resulting in a return of `0` from `getExchangeRate` and perpetuating the problem. Miscalculations of `.lastImpliedRate` can also throw off future `rateAnchor` calculations.

To resolve the issue, the developers considered calling `_getImpliedRateRequire` which uses `sub` to protect against underflow if `_getExchangeRate` returns `0`, or determining a safe “max” rate and branching within `_getImpliedRate` whenever `_getExchangeRate` returns `0`. The issue was fixed in pull request #54.

## Details
The [`_getImpliedRate`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L985) function calls [`_getExchangeRate`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L989) and uses it in calculations. In various failure cases within [`_getExchangeRate`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L1025), the value [`0` will be returned](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L1032).


If `0` is returned, within `_getImpliedRate` the return value will [underflow due to an unprotected subtraction](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L989), resulting in a much higher “implied rate” than in a non-error case.


Importantly, this value will be used within [`_tradeCalculation`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L961) to set a market’s `lastImpliedRate` and within [`_getNewRateAnchor`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L975). This can cause future [`rate` calculations](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L1054) to be too high, resulting in a return of `0` from [`getExchangeRate`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L1057-L1058) and perpetuating the problem. Miscalculations of `.lastImpliedRate` can also throw off future [`rateAnchor` calculations](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L975) again perpetuating the problem.


Consider calling [`_getImpliedRateRequire`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L998) which uses [`sub`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L989) to protect against underflow if `_getExchangeRate` returns `0`, rather than `_getImpliedRate`. Alternatively, if it is not desired to revert, consider determining a safe “max” rate, and branching within [`_getImpliedRate`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/FutureCash.sol#L985) whenever `_getExchangeRate` returns `0`, to avoid returning an erroneous value by subtracting from `0`. See related issue [**[C03] Borrow rates can be severely miscalculated**](#c03).


***Update:** Fixed in [pull request #54](https://github.com/swapnet-protocol/swapnet-lite/pull/54).*
