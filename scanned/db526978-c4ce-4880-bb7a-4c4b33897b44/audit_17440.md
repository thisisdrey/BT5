# [M-13] JBToken: mint function could mint arbitrary amount of tokens

## Summary
A bug has been identified in the JBToken contract that allows the owner to mint arbitrary amounts of tokens. The owner of the JBToken contract is usually the JBTokenStore contract, and the minting of tokens is controlled by the JBController contract. However, when the changeTokenOf function of the JBController contract is called, the owner is transferred to any address, which can then mint any amount of tokens. This is done through the mint function, which has no restrictions on the amount of tokens that can be minted.

To mitigate this vulnerability, it is recommended to set the minter as the JBTokenStore contract and add the onlyminter modifier to the mint function. This will ensure that only the JBTokenStore contract can mint tokens, and the amount of tokens that can be minted will be restricted.
