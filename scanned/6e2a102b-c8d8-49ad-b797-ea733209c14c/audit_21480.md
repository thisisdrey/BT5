# The symbol-cli saves the conﬁg ﬁle as readable for others

## Summary
This bug report is about a Denial of Service vulnerability in the NEM Group Symbol Assessment. The vulnerability occurs when a user imports a profile into symbol-cli, which creates the symbol-cli.config.json file with the fs.writeFileSync function. This function has a default file permission of 0o666, which allows all users to read the created config file. This means that if someone else has access to the user's home directory, they can copy the file and try to brute-force the password, allowing them to steal the user's mosaics. 

The short-term recommendation is to explicitly set the permissions of the symbol-cli.config.json file to 0o600 in the symbol-cli when creating or saving the file. This will prevent users on the same machine from accessing someone else's account if they access the config file and are able to brute-force the password. 

The long-term recommendation is to check the permissions of the symbol-cli.config.json file when reading it in the symbol-cli and warn the user if the file permissions are too broad. This will help prevent situations in which the user accidentally sets permissions that are too broad for the file and never changes them back.

## Details
## Type: Denial of Service

**Target:** catapult-server/../PacketPayloadBuilder.h and PacketHandlers.h

**Difficulty:** High

## Description

Importing a profile into `symbol-cli` creates the `symbol-cli.config.json` file. This file is created with the `fs.writeFileSync` function (Figure TOB-SYM-006.1), which by default uses the `0o666` file permissions. This allows all users to read the created config file.

```javascript
/**
 * Save profiles from JSON.
 * @param { JSON } profiles
 */
private saveProfiles(profiles: ProfileRecord) {
    fs.writeFileSync(this.filePath, JSON.stringify(profiles), 'utf-8');
}
```

**Figure TOB-SYM-006.1:** The `saveProfiles` function that saves the `symbol-cli.config.json` file with permissions that are too broad (symbol-cli/src/repositories/profile.repository.ts#L203-L209).

## Exploit Scenario

Alice imports her account into `symbol-cli`. Eve, who has access to Alice's home directory, copies her `symbol-cli.config.json` file and brute-forces their password. Eve then steals Alice's mosaics.

## Recommendation

**Short term:** Explicitly set the permissions of the `symbol-cli.config.json` file to `0o600` in the `symbol-cli` when creating or saving the file. This will prevent users on the same machine from accessing someone else's account if they access the config file and are able to brute-force the password.

**Long term:** Check the permissions of the `symbol-cli.config.json` file when reading it in the `symbol-cli` and warn the user if the file permissions are too broad. This will help prevent situations in which the user accidentally sets permissions that are too broad for the file and never changes them back.
