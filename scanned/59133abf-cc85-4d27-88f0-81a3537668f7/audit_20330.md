# Router - spawns unauthenticated admin API endpoint listening on all interfaces

## Summary
This bug report is about a router that allows any local or remote user with access to the endpoint to steal the router's liquidity. The code/packages/router/src/index.ts file on line 123-130 shows that the server is listening on allips without authentication. The recommendation for this issue is to require authentication and should only bind to localhost by default. This will ensure that the router's liquidity is secure and only accessible to authenticated users.

## Details
#### Description


* unauthenticated
* listening on allips


pot. allows any local or remote unpriv user with access to the endpoint to steal the routers liquidity `/remove-liquidity -> req.body.recipientAddress`


#### Examples


**code/packages/router/src/index.ts:L123-L130**



```

server.listen(8080, "0.0.0.0", (err, address) => {
  if (err) {
    console.error(err);
    process.exit(1);
  }
  console.log(`Server listening at ${address}`);
});

```
#### Recommendation


* require authentication
* should only bind to localhost by default
