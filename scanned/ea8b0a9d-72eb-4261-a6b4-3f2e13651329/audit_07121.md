# A failed send to `to` address in `recoverEther` leads to over-distribution to non-failing parties

## Summary
The bug report describes a vulnerability in the code of a contract that distributes fees to different parties. If the owner of the contract makes a mistake while trying to send Ether to a specific address, the remaining funds will be over-distributed to other parties. This could result in financial losses for the affected parties. To fix this issue, it is recommended to add a statement that will revert the transaction if the attempt to send Ether to the specified address fails.

An issue has been identified where, if the `owner` calls `recoverEther` with the `to` address failing to receive, they will need to call `recoverEther` again. This leads to the over-distribution of the remaining funds to non-failing parties. This scenario requires the `owner` to make an error. This vulnerability is classified as `medium` because the `owner`'s mistake results in losses for the failing parties of the contract.

##### Recommendation
We recommend adding a revert statement if an attempt to send Ether to the `to` address fails.
