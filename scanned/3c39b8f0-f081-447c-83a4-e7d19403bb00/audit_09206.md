# Closing netted, non-matured shorts can lead to insolvency

## Summary
The bug report is about a potential issue in the code for the HyperdriveShort contract, specifically in lines 190-196. The problem occurs when closing netted shorts, which can increase the long exposure if non-matured longs are not also closed. This can lead to the protocol becoming insolvent. The recommendation is to add a check for solvency when updating the long exposure in the _closeShort function. The report includes a code example to demonstrate the issue and mentions that the bug has been fixed in a recent pull request. It has also been verified by Spearbit.

## Details
## High Risk Vulnerability Report

## Severity: High Risk

## Context
*File:* HyperdriveShort.sol  
*Lines:* 190-L196

## Description
Closing netted shorts can increase the long exposure in case the now non-netted longs are not also closed. This occurs when calling `closeShort` before maturity. As the long exposure increases, a solvency check is required so that all outstanding longs can be fulfilled at maturity. This is not done in `closeShort` and the protocol can end up insolvent.

## Recommendation
Consider checking if the protocol is solvent after updating the long exposure in `_closeShort` for non-matured shorts. (Closing shorts at maturity via `applyCheckpoint` should not require a solvency check as any long exposure created from closing netted shorts is offset by closing the same amount of netted matured longs.)
