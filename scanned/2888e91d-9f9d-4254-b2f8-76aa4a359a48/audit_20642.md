# Allowed zero `seizeInternal`

## Summary
This bug report is related to the `seizeInternal` function in the `CCollateralCapErc20.sol` file. This function is supposed to change the state of supply and collateral tokens, however it only consumes gas and emits an "empty" event. To fix this issue, the developers recommend quitting `seizeInternal` after checks, using the code snippet provided. This code snippet will check if the `seizeTokens` is equal to 0, and if it is, it will return an error code. This should fix the issue and allow `seizeInternal` to successfully change the state of supply and collateral tokens.

## Details
##### Description
Zero `seizeInternal` doesn't change the state of supply and collateral tokens. It only consumes gas and emits an "empty" event
https://github.com/CreamFi/compound-protocol/blob/23a4ae93adc70334553f5a83429a4e967c1eefaa/contracts/CCollateralCapErc20.sol#L654

##### Recommendation
We recommend quitting `seizeInternal` after checks https://github.com/CreamFi/compound-protocol/blob/23a4ae93adc70334553f5a83429a4e967c1eefaa/contracts/CCollateralCapErc20.sol#L669
```solidity=
if (seizeTokens == 0) {
    return uint(Error.NO_ERROR);
}
```
