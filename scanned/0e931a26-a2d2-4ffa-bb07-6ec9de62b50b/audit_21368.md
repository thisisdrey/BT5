# The withdrawal mechanism is overcomplicated ✓ Fixed

## Summary
This bug report is about the withdrawal mechanism of a payment registry. Currently, the user can withdraw funds from the deposit account in two ways: with guardian signature - `withdrawDeposit` or using the “deposit exit” process. The withdrawal process requires the user to wait for a lockdown period of 30 days before they can access the funds. During this period, everyone who has a channel with the user is forced to commit their channels or else they will lose money from that channel. If the withdrawer has N channels, it’s theoretically possible to wait for up to N*(30 days) period and make N+2 transactions. 

The bug report recommends that when committing a payment channel, the lockdown period should not be reset to zero. Instead, it should either be unchanged or extended to `now + depositWithdrawalLockPeriod`. This will help prevent the user from having to wait for up to N*(30 days) period and make N+2 transactions.

#### Description

There may be some minor recommendations on how to improve that without major changes:


* When committing a payment channel, do not reset the lockdown period to zero. Two better option would be either not change it at all or extend to `now + depositWithdrawalLockPeriod`
