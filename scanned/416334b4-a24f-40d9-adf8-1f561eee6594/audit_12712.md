# TypeScript Errors ✓ Fixed

## Summary
This bug report is about errors in the TypeScript configuration of the ethereum-push-notification-service/push-protocol-snaps repository. The resolution was partially addressed in 1a6a32ef760088ca59f73e555f41b5b5d871f761, where `toggleHelper` was not addressed, `popupHelper` was addressed as per recommendation, `fetchAllAddrNotifs` was fixed by forcing `fetchAddress()` to return empty array instead, and `persistedData` was partially addressed. The update was that `toggleHelper` and `persistedData` were addressed with the snap data check wrapper function in 7ee018947303014e8c14e9413a5edd9fd29f9829.

The errors include `toggleHelper` not checking `persistedData` for `null` and defaulting to a sane initial config, `notifcount:Number` should be `notifcount:number`, `let msg = []` should be `let msg = [] as String[];`, `addresses` can be `null`, and `persistedData` can be `null`.

The recommendation is to fix the TypeScript configuration, fix all reported ts-lint errors, avoid using `any` types, and use safe types instead.
