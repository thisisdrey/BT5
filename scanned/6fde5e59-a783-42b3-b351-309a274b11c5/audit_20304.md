# [M01] Errors and omissions in events throughout codebase

## Summary
This bug report is about events in the Fei Protocol codebase. Events are used to signify when changes are made to the contracts. However, many events lack indexed parameters, are missing important parameters, or emit erroneous parameters. Additionally, collisions between events exist, and some sensitive actions are lacking events altogether.

Events emitting erroneous parameters include the constructor of `PCVDripController` erroneously emitting the `_incentiveAmount` in the `DripAmountUpdate` event, and the `dripAmount` emitted in the `Dripped` event of the `PCVDripController` can be the incorrect amount due to the fact that the amount withdrawn from uniswap can differ from the amount requested.

Events missing important parameters include `UpdateReceivingAddress` should include the previous variable value, the 4 events defined in `PCVSwapperUniswap` should include the previous variable value, and `TimerReset` should include the previous variable value.

Events lacking indexed parameters include all events in `Timed`, `Incentivized`, `PCVSplitter`, `IUniswapPCVDeposit`, `PCVSwapperUniswap`, `ReweightMinDistanceUpdate` in `IUniswapPCVController`, `FeiPriceThresholdUpdate` in `ITribeReserveStabilizer`, `DripAmountUpdate` in `IPCVDripController`, and `ScaleUpdate`, `BufferUpdate`, `DiscountUpdate`, and `Reset` in `IBondingCurve`.

Sensitive actions that are lacking events include, but are not limited to, the relevant “update” events in the constructors of `BondingCurve`, `PCVSwapperUniswap`, and `TribeReserveStabilizer`.

Additionally, some contracts have events with identical names, such as the `WithdrawERC20` event in `RatioPCVController` and `IPCVDeposit`, and the `DurationUpdate` event in `Timed` contract and the `IUniswapOracle` interface.

Finally, the `ReweightIncentiveUpdate` event is defined but never used.

## Details
Throughout the codebase, events are used to signify when changes are made to the contracts. However many events lack indexed parameters, are missing important parameters, or emit erroneous parameters. Additionally collisions between events exist, and some sensitive actions are lacking events altogether.


Events emitting erroneous parameters include:


* The constructor of `PCVDripController` erroneously [emits the `_incentiveAmount` in the `DripAmountUpdate`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/PCVDripController.sol#L44) event. It should instead emit the `_dripAmount`.
* The [`dripAmount` emitted in the `Dripped` event](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/PCVDripController.sol#L68) of the `PCVDripController` can be the incorrect amount. The is due to the fact that the amount withdrawn from uniswap can differ from the amount requested. The actual amount dripped is equal to [the `amountWithdrawn` in the `withdraw` function of `UniswapPCVDeposit`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/UniswapPCVDeposit.sol#L91).


Events missing important parameters include:


* [`UpdateReceivingAddress`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/IPCVSwapper.sol#L11) should include the previous variable value.
* The 4 events defined in [`PCVSwapperUniswap`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/PCVSwapperUniswap.sol#L22-L25) should include the previous variable value.
* [`TimerReset`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/utils/Timed.sol) should include the previous variable value.


Events lacking indexed parameters include:


* All events in [`Timed`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/utils/Timed.sol).
* All events in [`Incentivized`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/utils/Incentivized.sol).
* All events in [`PCVSplitter`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/PCVSplitter.sol).
* All events in [`IUniswapPCVDeposit`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/IUniswapPCVDeposit.sol).
* All events in [`PCVSwapperUniswap`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/PCVSwapperUniswap.sol).
* [`ReweightMinDistanceUpdate`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/IUniswapPCVController.sol#L21-L24) in `IUniswapPCVController`.
* [`FeiPriceThresholdUpdate`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/stabilizer/ITribeReserveStabilizer.sol#L14) in `ITribeReserveStabilizer`.
* [`DripAmountUpdate`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/IPCVDripController.sol#L13) in `IPCVDripController`.
* [`ScaleUpdate`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/bondingcurve/IBondingCurve.sol#L10), [`BufferUpdate`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/bondingcurve/IBondingCurve.sol#L12), [`DiscountUpdate`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/bondingcurve/IBondingCurve.sol#L14), and [`Reset`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/bondingcurve/IBondingCurve.sol#L20) in `IBondingCurve`.


Sensitive actions that are lacking events include, but are not limited to:


* The relevant “update” events in the constructors of `BondingCurve`, `PCVSwapperUniswap`, and `TribeReserveStabilizer`.


Additionally, some contracts have events with identical names. For example the `WithdrawERC20` event in [`RatioPCVController`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/RatioPCVController.sol#L19-L24) and [`IPCVDeposit`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/IPCVDeposit.sol#L16-L21). A call to [`RatioPCVController.withdrawRatioERC20`](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/RatioPCVController.sol#L49) emits 2 different events with different parameters, but both named `WithdrawERC20`. Another occurrence is the [`Timed` contract](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/utils/Timed.sol#L14) and the [`IUniswapOracle` interface](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/oracle/IUniswapOracle.sol#L11) both define a `DurationUpdate` event. This may cause confusion for off-chain clients and users alike.


Finally, the [`ReweightIncentiveUpdate` event](https://github.com/fei-protocol/fei-protocol-core-internal/blob/f54d7bb07c55adb78e2e142e7044f60090bb7602/contracts/pcv/IUniswapPCVController.sol#L19) is defined but never used.


Consider making all of the above changes to enable off-chain clients to correctly track changes to the Fei Protocol.


***Update:** Partially fixed in [PR#104](https://github.com/fei-protocol/fei-protocol-core/pull/104). The following items were not fixed:*


* *The `dripAmount` of the `Dripped` event remains potentially inaccurate.*
* *2 of the 4 events in `PCVSwapperUniswap` do not emit the previous variable value.*
* *`TimerReset` does not emit the previous value.*
* *No indexing has been added to the list of events lacking indexed parameters provided.*
