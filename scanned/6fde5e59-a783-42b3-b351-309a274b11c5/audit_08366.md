# MISSING ONLY INIT MODIFIER

## Summary
This bug report is about an error in the `onlyInit` modifier which is causing problems in the **SystemReward** contract. The issue is that the **receiveRewards** function in the contract does not have the `onlyInit` modifier, which means that if the function is called without initialization, all the address balances will be burned in the contract. This can have a high impact and is likely to occur. The recommendation for solving this issue is to add the `onlyInit` modifier, which has been done by the CoreDAO team in the commits [b283001](https://github.com/coredao-org/core-genesis-contract/commit/b283001acc46a4b6d0b536c77385bd827f34d85c) and [9e290d8](https://github.com/coredao-org/core-genesis-contract/commit/9e290d8ed5431ddc2dcde652d8a5e8dc2ad9af56).

## Details
##### Description

The `onlyInit` modifier throws an error If the source contract is not initialized. In the **SystemReward** contract, **receiveRewards** function is missing **onlyInit** modifier. If the function is called without initialization, all address balance will be burned in the contract.
