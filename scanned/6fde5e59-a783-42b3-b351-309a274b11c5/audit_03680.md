# [M-05] `BerabotRouter` Cannot Make Swaps with V3 Pools with Fee Different than 500

## Summary
This bug report discusses an issue with the `uniswapV3SwapCallback()` function in a codebase. The function is responsible for calculating the correct pool for a swap, but it currently only works for pools with a fee of 500. This means that swaps for pools with a different fee will fail. The team has acknowledged the issue and plans to fix it by providing the correct fee variable and taking inspiration from how the Uniswap V3 Router handles it. This will ensure that the function works for all V3 pools, not just those with a fee of 500.

## Severity

Medium Risk

## Description

This causes the incorrect pool to be calculated (for V3 pools with a fee different than 500) and will cause the swap to revert. Moreover the `getAmountOutAndRouter()` function loops over V3 router fee versions which implies that fees will differ.

## Impact

The router does not support V3 swaps other than pools with a fee of 500.

## Recommendation

Provide the correct fee variable. Take inspiration from how the Uniswap V3 Router does it.
