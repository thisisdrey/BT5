# AssetChangeComponent Displays  a Change With Value 0 if fiatValue < 0.005

## Summary
The `toFixed(2)` method is used to round a transaction value string to 2 decimals. However, for transactions with a value less than 0.005, the function returns 0. This means that the user may see a transaction with a value of 0, even though it actually has a small non-zero value. To avoid this confusion, it is recommended to display a minimum value of 0.01 instead of 0 for these transactions. This will ensure that the user is not tricked by seeing a transaction with a value of 0.

## Details
#### Description


The `toFixed(2)` method rounds the transaction value string to 2 decimals. For transactions with `fiatValue < 0.005`, the function returns 0, meaning the component will display a transaction with zero value to the user, even if the transaction has a small yet non-zero value. This is not a good idea as it might trick the user.
In that case, it would be better to default to the smallest value that can represented (i.e. 0.01) instead of 0.


**packages/snap/src/components/stateChanges/AssetChangeComponent.ts:L18**



```
const fiatValue = Number(stateChange.fiatValue).toFixed(2);

```
#### Recommendation


If `fiatValue < 0.005`, consider displaying a value of `0.01` to the user, instead of 0.
