# `ACLFacet.updateRoleGroup()` may change `GROUP_SYSTEM_ADMINS`, bypassing the existing check

## Summary
This bug report is about a function called `updateRoleGroup()` in the `ACLFacet.sol` file of the `contracts-v3` project. This function allows system administrators to change the group membership for a given role. However, there is a bug in the function that allows this check to be bypassed. This means that even though the function checks for the group `LC.GROUP_SYSTEM_ADMINS` and prevents it from being changed, it can still be changed by providing a specific value for the `_group` parameter. The recommendation is to fix this bug by parsing the `_group` parameter and comparing it to the value of `LC.GROUP_SYSTEM_ADMINS` using a specific function called `LibHelpers._stringToBytes32()`. The bug has been marked as "Fixed" by the client and the fix can be found in the `fb0337743dc1a9a17ce3f5dd3d877f30da40013c` update.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `fb0337743dc1a9a17ce3f5dd3d877f30da40013c`.

**File(s) affected:**`contracts-v3/ACLFacet.sol, contracts-v3/LibACL.sol, contracts-v3/LibHelpers.sol`

**Description:** The function `ACLFacet.updateRoleGroup()` allows system administrators to change the group membership for a given role. While the following check prevents changing the membership to group `LC.GROUP_SYSTEM_ADMINS`, it can be bypassed:

```
require(!LibString.eq(_group, LC.GROUP_SYSTEM_ADMINS), "system admins group is not modifiable");
```

In details, this check can be bypassed by providing a `_group` string with the value `LC.GROUP_SYSTEM_ADMINS` padded with enough zero bytes such that the function `eq()` returns `false` but the function `LibACL._updateRoleGroup()` still uses the value of `LC.GROUP_SYSTEM_ADMINS` (i.e. `System Admins\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000\u0000` - The string `System Admins` padded with null-bytes to 32 bytes length, which may be passed like this or may contain arbitrary other bytes afterwards).

**Recommendation:** Consider parsing the parameter `_group` using `LibHelpers._stringToBytes32()`, as done in the underlying `_updateRoleGroup()` function, and compare it against the value of `LC.GROUP_SYSTEM_ADMINS`.
