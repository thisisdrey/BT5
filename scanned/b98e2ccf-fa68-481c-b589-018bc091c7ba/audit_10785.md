# Risk of Frozen Funds Due to Unauthorized Claim on Behalf of Contract in `ERC20Rewarder`

## Summary
The report discusses a security vulnerability in the `claim()` function of the `ERC20Rewarder` contract. This function allows any user to initiate a claim on behalf of another address, which could potentially lead to malicious actors claiming ERC20 tokens and freezing them in a third-party contract. The suggested solution is to revise the logic of the `claim()` function to include proper checks to ensure that only legitimate token holders or authorized parties can trigger withdrawals for a given address. The issue has been resolved in a recent commit and the affected file is `ERC20Rewarder.sol`.

## Details
**Update**
The issue was resolved by the client in commit `d92957f3c8a3979be78a350adc42ce22c43443b3`.

The destination address of the claim is now validated against `tx.origin`.

**File(s) affected:**`ERC20Rewarder.sol`

**Description:** The `claim()` function within the `ERC20Rewarder` contract currently exhibits a security vulnerability. This function allows any user to initiate a claim on behalf of another address. While this could potentially be leveraged for beneficial features, it poses a significant risk in scenarios if a contract interacting with `ERC20Rewarder` does not implement the necessary logic to securely manage such transfer. If exploited, malicious actors could claim ERC20 tokens, potentially freezing the assets in the third-party contract.

**Exploit Scenario:** Assuming the given contract, `VulnerableClaimReceiver`, serves as the destination address for a claim() function in the ERC20Rewarder contract.

```
contract VulnerableClaimReceiver {
    ...
    function claim(bytes calldata claimData) external {
        erc20Rewarder.claim(claimData);
        uint amount = decodeAmount(claimData);
        erc20.safeTransfer(receiver, amount);
    }
    ...
}
```

Furthermore, considering that VulnerableClaimReceiver does not implement any rescue functions, we can consider the following scenario:

1.   A malicious actor calls `ERC20Rewarder.claim()`, executing the claim on behalf of the `VulnerableClaimReceiver` contract.
2.   The reward ERC-20 tokens from the `ERC20Rewarder` contract are transferred to the `VulnerableClaimReceiver` contract.
3.   The funds remain indefinitely within the destination contract.

**Recommendation:** We advised revising the `claim()` function's logic to include proper checks to ensure that only the legitimate token holders or authorized parties can trigger withdrawals for a given address.
