# `setSalesAddress()` Breaks the Token Minting Process

## Summary
Bug Summary: In the contract `UnikuraPhygitalCollection`, the function `setSalesAddress()` does not effectively transfer the ownership of minted tokens to the new address. This causes the `_mintTransferFrom` function to fail, requiring all tokens to be minted again to the new address. The team should either redesign `setSalesAddress()` to properly transfer ownership or remove it and keep the sales address as a constant variable.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `b898120427d9748162dccbb37a79524d40f20f40`. The client has removed the functionality for chaging the sale address.

**File(s) affected:**`UniKuraPhygitalCollection.sol`

**Description:** In the contract `UnikuraPhygitalCollection`, all the tokens are minted to the sales address upon initialization. Later when a token is minted, the function `mint()` is called, and the function `_mintTransferFrom()` directly operates over the `ERC721AStorage` contract. This function compares `prevOwnershipPacked` with `UnikuraPhygitalCollectionStorage.layout()._salesAddress` to ensure that the token owner is the sales address.

The problem here is that the function `setSalesAddress()` only update the `_salesAddress` within `UnikuraPhygitalCollectionStorage.layout()`, but it does not effectively transfer the the minted tokens to the new address and update `ERC721AStorage`. As a result, the `_mintTransferFrom` will fail after `setSalesAddress()` is called. The system has to `mint()` all the tokens again to the new sales address before calling `setSalesAddress()`, which is not only expensive but not comply with the expected system design.

**Recommendation:** The team should carefully inspect the design purpose of the function `setSalesAddress()`. If updating `salesAddress` is a required behavior, the team should redesign `setSalesAddress()` to effectively transfer the ownership of the minted tokens to the new address. Otherwise, the team should remove `setSalesAddress()` and keep the sales address as a constant variable.
