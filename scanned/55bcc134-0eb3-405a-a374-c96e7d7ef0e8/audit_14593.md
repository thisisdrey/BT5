# H-1: pushVaultAmounts can be called multiple times if in the right state

## Summary
This bug report is about an issue found in the XChainController of the Sherlock Audit project. The issue is that the function pushVaultAmounts can be called multiple times on the same vault, which can lead to funds being sent cross chain multiple times and breaking protocol accounting. Manual review was used to discover the issue. The impact of this issue is that the protocol accounting is completely broken, leading to funds being stuck on a chain due to being unaccounted for. The recommendation is to cache the allocations on which pushVaultAmount has already been called, so only the difference is forwarded cross chain, or to find a way to check if a cross chain action has already been sent.
