# No Proper Trusted Setup  Acknowledged

## Summary
This bug report is about the Linea project, which uses the Plonk proof system. The Plonk system requires a preprocessed CRS (Common Reference String) for proving and verification. The security of the system is based on the existence of a trusted setup ceremony to compute the CRS. Currently, the verifier uses a CRS created by one single party, which requires full trust of the party to delete the toxic waste (trapdoor) which can be used to generate forged proof, undermining the security of the entire system.

The recommendation is to conduct a proper MPC to generate CRS, such as the Powers of Tau MPC, or use a trustworthy CRS generated by an existing audited trusted setup, such as Aztec’s ignition. This would ensure the security of the system and reduce the risk of forged proof.
