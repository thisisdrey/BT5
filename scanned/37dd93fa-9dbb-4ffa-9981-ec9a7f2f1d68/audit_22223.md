# Owner can load more refund money than necessary

## Summary
This bug report is about a function called `loadRefund` in the codebase for an Initial Coin Offering (ICO). This function was designed to allow an investor to send money into the contract, which can then be used to refund the same amount that was invested. The bug was that it was possible to load more balance than necessary, and the contract had no mechanism to withdraw the extra balance. 

Hubii, the company responsible for the codebase, decided to assign the excess of the refund ether to the sender address, so that they could refund the excess through the normal mechanism. This fix was implemented in the commit `b6f8f29e8e6578d9efc56457c0aa8ff3a49f2c05`.

## Details
[`loadRefund`](https://github.com/CoinFabrik/ico/blob/142f43f3bec4afd0ffff3fce06946cf67cb0272f/contracts/Crowdsale.sol#L429) allows to send balance into the contract so that it can be used for refunds. This money can only be withdrawn via the [`refund`](https://github.com/CoinFabrik/ico/blob/142f43f3bec4afd0ffff3fce06946cf67cb0272f/contracts/Crowdsale.sol#L437) function, by an investor requesting a refund of the same amount they had invested.


It’s possible to load more balance than necessary, and the contract provides no mechanism to withdraw the extra balance. (Note that it would be possible to withdraw it by abusing the lack of safety checks in `preallocate`, as explained below, but this should not be relied upon.) Consider requiring that `msg.value == weiRaised` in [`loadRefund`](https://github.com/CoinFabrik/ico/blob/142f43f3bec4afd0ffff3fce06946cf67cb0272f/contracts/Crowdsale.sol#L429).


**Update:** Hubii replied *“We decided to assign the excess of the refund ether to the sender address. So they can refund the excess through the normal mechanism.”* Fixed in [`b6f8f29e8e6578d9efc56457c0aa8ff3a49f2c05`](https://github.com/CoinFabrik/ico/commit/b6f8f29e8e6578d9efc56457c0aa8ff3a49f2c05).
