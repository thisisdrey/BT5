# *LevelZeroEdge snapshots are not updated

## Summary
This bug report discusses an issue with the HonestChallengeTree struct in the Arbitrum Chains Challenge Protocol. The problem is that the snapshots of honest level-zero edges in the struct are not being properly updated in the RefreshEdgesFromChain function. This can lead to incorrect children being used in the findAncestorInChallenge function. The report recommends a short term solution of updating the relevant fields in the RefreshEdgesFromChain function and a long term solution of improving the testing suite to catch similar issues in the future.

## Details
## Diﬃculty: Low

## Type: Undeﬁned Behavior

### Description
Snapshots of honest level-zero edges in the `HonestChallengeTree` struct are not refreshed in `RefreshEdgesFromChain`, where only the `edges` field is updated. Since the lower and upper child of a snapshot edge can change, it is possible that incorrect children will be used in `findAncestorInChallenge`.

```go
type HonestChallengeTree struct {
    edges                             *threadsafe.Map[protocol.EdgeId, *threadsafe.Map[protocol.MutualId, *threadsafe.Map[protocol.EdgeId, creationTime]]
    protocol.AssertionId
    topLevelAssertionId               util.Option[protocol.EdgeSnapshot]
    honestBlockChalLevelZeroEdge     honestBigStepLevelZeroEdges *threadsafe.Slice[protocol.EdgeSnapshot]
    honestSmallStepLevelZeroEdges     *threadsafe.Slice[protocol.EdgeSnapshot]
    metadataReader                    MetadataReader
    histChecker                       HistoryChecker
    edgeReader                        EdgeReader
}
```
*Figure 18.1: The HonestChallengeTree struct in tree.go#L52-L62*

The `RefreshEdgesFromChain` function re-fetches each edge currently in the `edges` field, but it does not refetch the `*LevelZeroEdge/s` fields.

```go
// RefreshEdgesFromChain refreshes all edge snapshots from the chain.
func (ht *HonestChallengeTree) RefreshEdgesFromChain(ctx context.Context) error {
    edgeIds := make([]protocol.EdgeId, 0, ht.edges.NumItems())
    if err := ht.edges.ForEach(func(id protocol.EdgeId, _ protocol.EdgeSnapshot) error {
        edgeIds = append(edgeIds, id)
        return nil
    }); err != nil {
        return err
    }
    snapshots := make([]protocol.EdgeSnapshot, len(edgeIds))
    for i, edgeId := range edgeIds {
        edge, err := ht.edgeReader.GetEdge(ctx, edgeId)
        if err != nil {
            return err
        }
        snapshots[i] = edge
    }
    for i, edgeId := range edgeIds {
        ht.edges.Put(edgeId, snapshots[i])
    }
    return nil
}
```

### Exploit Scenario
The lower child of the `honestBlockChalLevelZeroEdge` changes, but it is not updated in `RefreshEdgesFromChain`. Additionally, when used as a starting point in `findAncestorsInChallenge`, it leads to an incorrect tree of edges, returning a wrong result.

### Recommendations
- **Short term:** Update the `honestBlockChalLevelZeroEdge`, `honestBigStepLevelZeroEdges`, and `honestSmallStepLevelZeroEdges` fields in `RefreshEdgesFromChain`.
- **Long term:** Improve the testing suite by simulating a scenario in which the children of an honest-level zero edge are updated and verify the correct behavior.
