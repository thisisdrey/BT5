# Router - handleNewAuction - fragile code

## Summary
This bug report is about the `handleNewAuction` code in the router package of the application. The code and coding style appears fragile and it is suggested that validation should be strictly enforced, debugging code should be removed or disabled in production and protective measures should be taken from abusive clients.

The bug report highlights a collection of issues with the `handleNewAuction` which can potentially broaden the attack surface of this method. These issues include router bids on zero-amount requests, duplicate constant var assignment, actual estimated gas required to fuel transaction is never checked, removing debug code from production build, signer address might be different for different chains, lack of rate limiting which could lead to a potential DoS vector and missing validation of bid parameters.
