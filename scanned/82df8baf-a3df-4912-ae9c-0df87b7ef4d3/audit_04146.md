# False negatives in ECDSA384 can lead to DoS 

## Summary
The current version of the solarity/libs/crypto/ECDSA384.sol library has a bug where it rejects correct signatures in certain cases due to how point arithmetic is handled. This has been reported in two GitHub issues, 126 and 125. To fix this issue, it is recommended to update the library to the latest version and ensure it matches with the wycheproof test suite's output for all cases. Two pull requests, 124 and 127, have been made to address this issue. The bug has been fixed in PR 4 and has also been fixed in Cantina Managed.

## Description
The current pinned version of `solarity/libs/crypto/ECDSA384.sol` is an incomplete implementation, where the correct signature is being rejected in some cases due to how point arithmetic is being handled.

See the following GitHub issues for more details:
- **Issue 126**: False negatives in `ECDSA384.sol`.

## Recommendation
Update the library to the latest version and ensure it matches with the Wycheproof test suite's output for all cases. See **PR 124** and **PR 127**.
