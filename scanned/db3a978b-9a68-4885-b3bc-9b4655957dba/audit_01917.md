# [M-01] Rescue Tokens: Implement a Rescue Funds Function for stucked funds

## Summary
The report states that there is a bug in the Pip.sol code that could potentially cause user funds to become permanently stuck in the contract. This happens when a user deposits and receives a nullifier code, which is required for withdrawing from a different address. However, if the user forgets their nullifier code, they will not be able to withdraw their funds. 

To fix this issue, the recommendation is to add a new function called "rescueFunds" that will allow the recovery of funds in case a user forgets their nullifier code. This function will also include an on-chain view function to check if the nullifier code has been used for withdrawal. This will ensure that the user genuinely forgot their code and facilitate the recovery of their funds. 

The team has acknowledged this issue and will work on implementing the suggested solution. It is important to fix this bug as it could potentially cause users to lose their funds permanently.

## Details
## Severity

Medium Risk

## Description

In Pip.sol, when a user deposits, they receive a nullifier code which is required when
withdrawing from a different address. However, since it is possible for a user to forget their nullifier
code, the funds could become permanently stuck in the contract.


## Recommendation

Add a rescueFunds function along with an on-chain view function to check
whether the nullifier hash (used during withdrawal) has been consumed. This would enable the
protocol to verify that the user genuinely forgot their nullifier code and facilitate the recovery of their
funds.
