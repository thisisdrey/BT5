# Use of static bitcoin fees

## Summary
This bug report is about the fee rate used in Line 10 of bitcoin.js of cosmos/fundraiser-lib. Currently, the fee rate of 200 satoshis per byte is the cheapest and fastest, but it is not recommended to use a static fee rate. If there is network congestion during the fundraiser, transactions could get stuck. To prevent this, the report recommends integrating the bitcoinfees' API to get a recommended fee rate dynamically. This API can be found at bitcoinfees.21.co/api. By integrating this API, the fee rate can be adjusted dynamically to avoid network congestion and ensure transactions go through.

## Details
[Line 10 in bitcoin.js of cosmos/fundraiser-lib](https://github.com/cosmos/fundraiser-lib/blob/426425dfc296060a9b87830e69e19ae8a6d444c0/src/bitcoin.js#L10) defines a fee rate of 200 satoshis per byte. That’s the current cheapest and fastest fee rate according to <https://bitcoinfees.21.co/>, but it’s still not recommended to use a static fee rate. If there’s network congestion during the fundraiser, transactions could be stuck. Consider integrating [bitcoinfees’ API](https://bitcoinfees.21.co/api) to get recommended fee rate dynamically.
