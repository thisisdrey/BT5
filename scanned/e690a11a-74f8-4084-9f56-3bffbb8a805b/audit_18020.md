# [M-07] AutoleverageBase: Must approve 0 first

## Summary
A bug has been found in the AutoleverageBase.sol code on the GitHub repository, code-423n4/2022-05-alchemix. This bug has an impact on tokens such as USDT, which do not work when changing the allowance from an existing non-zero allowance value. To fix the bug, it is recommended that the code be changed to the following: 
```
    function approve(address token, address spender) internal {
    +  IERC20(token).approve(spender, 0);
        IERC20(token).approve(spender, type(uint256).max);
    }
```
This bug should be fixed as soon as possible to prevent any further issues with the code.

## Details
_Submitted by cccz_

Some tokens (like USDT) do not work when changing the allowance from an existing non-zero allowance value.They must first be approved by zero and then the actual allowance must be approved.
