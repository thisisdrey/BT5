# The contract will be paused after liquidation

## Summary
This bug report is about a medium severity issue in a contract called LPMigration. The function being affected is called liquidate and on line #103, there is a call to a function called `IPausableERC20.pause()`. This call is not conditional, meaning that even if the contract was not initially paused, it will be paused after `liquidate()` is called. The recommendation is to add a local variable and assign it to the value of `IPausableERC20.paused` before line #86. This value can then be used as a condition for both the unpause and pause function calls.
