# ClaggSyncAdapter LP staking cannot be disabled without bricking the vault 

## Summary
The ClaggSyncAdapter has a bug where the migration function does not allow for disabling staking by setting an empty address as the staking contract. This means that the staking functionality cannot be turned off without changing the configuration, which can lock the liquidity. The recommendation is to add the ability to migrate from a staking pool to an empty address, so that the owner can disable staking without risking the liquidity. The bug has been fixed in the commit 18047bde and has been verified by Cantina Managed.

ClaggSyncAdapter.sol#L81

## Description
In ClaggSyncAdapter, SyncSwap liquidity can be automatically staked in a contract if this one has been previously configured by the owner. To switch staking contracts, the owner can call `migrate` on the adapter, effectively migrating the assets to the new staking contract. However, the current migration function does not allow for disabling staking (setting an empty address as a staking contract). As a result, staking cannot be disabled without effectively locking the liquidity by changing the configuration.

## Recommendation
Consider implementing the ability to migrate from a staking pool to an empty address, allowing the owner to disable staking functionality when necessary without risking the liquidity.
