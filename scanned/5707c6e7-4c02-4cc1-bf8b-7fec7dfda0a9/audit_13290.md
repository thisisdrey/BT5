# TRST-M-2 SwapHelper.getCalldata should check whitelistedRouters[_router]

## Summary
This bug report is about the `SwapHelper.getCalldata()` function which returns data for swap based on the input and uses **whitelistedRouters** to limit the **_router** param. The issue was that when `setWhitelistedRouters()` sets the **_routers** state to **false**, it does not reset the data in **routerTypes** and **swapInfos**, which resulted in the router still being available in `getCalldata()`. As a result, users could still swap with invalid router data.

The team recommended checking **whitelistedRouters[_router]** in SwapHelper.getCalldata() to address the issue. The team then fixed the issue by setting **routerTypes** to **UNSET** status in whitelist function when delisting.
