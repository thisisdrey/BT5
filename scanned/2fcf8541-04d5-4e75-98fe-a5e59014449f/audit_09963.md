# `createDistribution()` Does Not Support `_paymentType`

## Summary
The bug report is about a recently added issue in a file called `ReachDistribution.sol`. The problem was discovered during a review of a fix, where two functions called `createMission()` and `claimRewards()` were updated to accept a new parameter called `_paymentType`. However, the third function, `createDistribution()`, was not updated and this causes a problem when trying to use reward tokens other than Ether. The recommendation is to create a new code branch in `createDistribution()` to support the case when the payment type is set to `ERC20`.

## Details
**File(s) affected:**`ReachDistribution.sol`

**Description:** This issue is newly added during the fix-review. In the provided commit hash `8afbbf4` for the fix-review, the functions `createMission()` and `claimRewards()` were updated to accept the parameter `_paymentType`. However, `createDistribution()` was not and the check `require(address(this).balance >= _amount, "Insufficient balance.");` will reject reward tokens other than Ether.

**Recommendation:** Create a code branch in `createDistribution()` to support the case when `if (_paymentType == PaymentType.ERC20) {`.
