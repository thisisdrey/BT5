# Changing yield from Claimable cause fund loss

## Summary
The bug report states that there is a critical risk in the ERC20Rebasing contract at line 348. If the mode for claiming yield is changed from "Claimable" to any other mode, users will lose all unclaimed yield. This is because the change only takes into account the fixed part of the balance and ignores the claimable part. The recommendation is to consider the overall balance (fixed plus claimable) as the new balance when configuring from Claimable to any other mode. This should be done while calling the "_setBalance" function.

## Details
## Severity: Critical Risk

## Context
`ERC20Rebasing.sol#L348`

## Description
If Claimable yield is changed to any other mode, then users will lose all unclaimed yield. This happens because changing from Claimable to any other mode only takes the fixed part of balance and ignores the claimable part.

## Recommendation
While configuring from Claimable to any other mode, the overall balance (fixed plus claimable) should be considered as the new balance for the new yield mode (while calling `_setBalance`).
