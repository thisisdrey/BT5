# [M-06] block.timestamp or deadline

## Summary
This bug report concerns the use of the _deadline variable and the block.timestamp variable in the RebalanceManagerV3, SingleTokenJoinV2, and SingleNativeTokenExitV2 smart contracts. The RebalanceManagerV3's rebalance() function uses _deadline as a time limit for swapExactTokensForTokens(), while the SingleTokenJoinV2 and SingleNativeTokenExitV2 functions use block.timestamp, although a deadline field is present in the struct. It is possible that the deadline fields should have been used instead of the block.timestamp.

The bug can be found in the following lines of code:

## Details
_Submitted by gpersoon, also found by kenzo and robee_

#### Impact

Some functions, like rebalance() in RebalanceManagerV3 use \_deadline as a time limit for swapExactTokensForTokens()
Other functions, like \_joinTokenSingle() of SingleTokenJoinV2.sol and \_exit() of SingleNativeTokenExitV2() use block.timestamp, although a deadline field is present in the struct.

Possibly the deadline fields should have been used.
