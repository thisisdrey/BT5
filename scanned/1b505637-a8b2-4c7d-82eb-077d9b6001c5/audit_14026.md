# The function getCurrentPeriod() returns wrong value which results in staking before startDate

## Summary
This bug report describes a problem with a function that is used to calculate the amount of stake held at a certain date. The function returns 0 if block.timestamp is less than the startDate, or if the difference between block.timestamp and startDate is less than the periodLength. This means that it is possible to stake before the startDate, which is not the intended behavior.

The recommendation is to change the function accordingly, so that it does not allow staking before the startDate.

## Details
**Description:** 

If block.timestamp < startDate, then the function returns 0, while it also returns 0 if block.timestamp - startDate < periodLength. Due to return of 0 for block.timestamp < startDate, it is possible to stake before startDate.

**Recommendations** 

Change the function accordingly.
