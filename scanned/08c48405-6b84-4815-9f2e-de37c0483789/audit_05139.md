# Excessive Logging of Sensitive Data

## Summary
The bug report discusses an issue where sensitive information, such as passwords and request details, were being logged without proper protection. This can lead to the exposure of this sensitive data to unauthorized users. The report provides a code snippet as proof of concept and gives the issue a score of 3 for both impact and likelihood. The recommendation is to avoid logging sensitive data or to securely store and redact/encrypt it before logging. The issue has been solved by the zkPass team by reducing the number of output logs and will be fully addressed in a future stable version of the code. The remediation hash for this issue is ffdd2baf089e7835465a21ea74026eace88721af.

## Details
##### Description

Logging sensitive information such as passwords, tokens, or request details can lead to information leakage, especially if logs are exposed to unauthorized users or external systems.

In `httpSession.ts`, sensitive data such as request objects were logged without redaction, increasing the risk of data exposure.

##### Score

Impact: 3

##### Recommendation

Avoid logging sensitive data, or ensure that logs are securely stored, and sensitive information is redacted or encrypted before logging.

##### Remediation

**SOLVED:** The **zkPass team** reduced significantly the number of output logs on the console, keeping some of them until they have a final stable version of the code.
