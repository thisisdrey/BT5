# USAGE OF SELF-DESTRUCT MAY LEAD TO FUNDS LOSS

## Summary
The `selfdestruct` function in the `StakeBNBToken.sol` contract can erase the code from the blockchain, causing users to lose their deposited money and breaking the decentralized application. This can happen if the function is used incorrectly. The impact and likelihood of this issue happening are both rated as 3. The recommended solution is to use the `timelock` contract from openzeppelin, which will give users time to withdraw their funds if the self-destruct function is triggered. The `persistenceOne team` has fixed this issue in their code and will only use the self-destruct function in case of a major upgrade on the Binance ecosystem.

## Details
##### Description

The usage of the `selfdestruct` function erases the contract code from the blockchain. Although it is understandable under certain circumstances, this function if executed inappropriately can lead to the destruction of the token contract making users loose all the deposited money and making the decentralized application broken.

Code Location
-------------

#### StakeBNBToken.sol

```
function selfDestruct(address addr) external onlyRole(DEFAULT_ADMIN_ROLE) whenPaused {
    selfdestruct(payable(addr));
    }

```

##### Score

Impact: 3  
Likelihood: 3

##### Recommendation

**SOLVED**: The `persistenceOne team` fixed the above issue by using the `timelock`contract from openzeppelin on the commit ID [d059bccbb368158a63767107b37894d47009c385](https://github.com/persistenceOne/stkBNB-contracts/commit/d059bccbb368158a63767107b37894d47009c385). As a result, this gives time to the users to call the withdraw function in case the self-destruct function is triggered.

Moreover, the `persistenceOne team` ensures that this function will only be called in case a major upgrade occurs on the Binance ecosystem and gets the compromise to re-balance the stkBNB tokens to every user account if this situation happens.
