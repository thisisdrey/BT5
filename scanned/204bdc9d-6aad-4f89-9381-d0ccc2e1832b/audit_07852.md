# Potential Owner Lockout

## Summary
The report states that the instruction "RemoveGroupUser" allows for the removal of the group owner from the user list. However, the provided code for "AddGroupUser" does not have functionality for adding a user with the owner role. This means that once the owner is removed, there is no apparent way to re-add them with owner privileges. The suggested solution is to introduce a separate instruction for adding a new owner to a group. The issue has been fixed in version 6c7043b.

## Details
## RemoveGroupUser Instruction

`RemoveGroupUser` instruction allows the removal of the group owner from the user list. However, the provided code snippet for `AddGroupUser` does not contain functionality for adding a user with the Owner role. This implies that once the Owner is removed, there is no apparent method of re-adding them with Owner privileges.

```rust
> _src/handlers/handle_remove_user.rs
pub fn process(ctx: Context<RemoveGroupUser>, user: Pubkey) -> Result<()> {
    let mut group = ctx.accounts.group.load_mut()?;
    group.remove_user(user)?;
    Ok(())
}
```

## Remediation

Introduce a separate instruction for adding a new owner to a group.

## Patch

Fixed in 6c7043b.
