# Price and refund changes may cause failures ✓ Addressed

## Summary
A bug was discovered in the code related to the price and refund for gold cards used in the commit, mint, and refund phases. The code was designed to burn the weave tokens during the commit phase, however, this was not happening until after a delay. This created a problem where if the price was increased between the transactions, it may not burn enough weave tokens to cover the cost of the gold cards. Alternatively, if the price was decreased, some weave tokens would be stuck in the contract forever. To resolve this issue, it was recommended that the goldPrice and goldRefund be stored in the GoldOrder. This issue was addressed in the horizon-games/SkyWeaver-contracts#3 pull request.

## Details
#### Resolution



Addressed in [horizon-games/SkyWeaver-contracts#3](https://github.com/horizon-games/SkyWeaver-contracts/pull/3).



> 
> Fix involves burning the weave when the commit occurs instead of when the minting of the gold cards occur.
> 
> 
> 




#### Description


Price and refund for gold cards are used in 3 different places: commit, mint, refund.


Weave tokens spent during the commit phase


**code/contracts/shop/GoldCardsFactory.sol:L274-L279**



```
function \_commit(uint256 \_weaveAmount, GoldOrder memory \_order)
  internal
{
  // Check if weave sent is sufficient for order
  uint256 total\_cost = \_order.cardAmount.mul(goldPrice).add(\_order.feeAmount);
  uint256 refund\_amount = \_weaveAmount.sub(total\_cost); // Will throw if insufficient amount received

```
but they are burned `rngDelay` blocks after


**code/contracts/shop/GoldCardsFactory.sol:L371-L373**



```
// Burn the non-refundable weave
uint256 weave\_to\_burn = (\_order.cardAmount.mul(goldPrice)).sub(\_order.cardAmount.mul(goldRefund));
weaveContract.burn(weaveID, weave\_to\_burn);

```
If the price is increased between these transactions, mining cards may fail because it should burn more `weave` tokens than there are tokens in the smart contract. Even if there are enough tokens during this particular transaction, someone may fail to melt a gold card later.


If the price is decreased, some `weave` tokens will be stuck in the contract forever without being burned.


#### Recommendation


Store `goldPrice` and `goldRefund` in `GoldOrder`.
