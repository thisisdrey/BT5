# Incorrect INDEXER_ROLE value may prevent indexer access 

## Summary
This bug report is about a mistake in a file called `AttestationIndexer.sol` on lines 34 and 80. The mistake is in the `INDEXER_ROLE` which is supposed to control who can access the `index()` function. However, the value of `INDEXER_ROLE` is wrong and needs to be changed from `cbattestations.staticattester.indexer` to `cbattestations.attestationindexer.indexer`. This mistake can cause problems with access control and may require an upgrade to fix. The bug has been fixed by a pull request and reviewed by the Cantina team.

## Details
## Issue Report

## Context
- **File**: `AttestationIndexer.sol`
- **Lines**: 34, 80

## Description
`INDEXER_ROLE` is used to enforce role-based access control to the `index()` function. However, the value of `INDEXER_ROLE` is incorrectly set to `cbattestations.staticattester.indexer` instead of `cbattestations.attestationindexer.indexer`. This appears to be a copy-paste typographical error.

If `AccessControlUpgradeable.DEFAULT_ADMIN_ROLE` correctly assigns `cbattestations.attestationindexer.indexer` role to addresses which are authorized to access the indexer, they will be denied access. The dynamic role assignment will also need to intentionally use the incorrect `cbattestations.staticattester.indexer` role during assignments, which will be confusing and error-prone, or this contract will require upgrading to fix this error.

## Recommendation
Change `cbattestations.staticattester.indexer` to `cbattestations.attestationindexer.indexer`.
