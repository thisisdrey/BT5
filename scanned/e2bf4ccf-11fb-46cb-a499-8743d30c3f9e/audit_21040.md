# Pod: Fees are not limited by a user during the withdrawal

## Summary
A bug report has been filed regarding the Pod contract in the Pods-v3-contracts repository. When withdrawing from the Pod, the shares are burned and the deposit is removed from the Pod. However, if there are not enough deposit tokens in the contract, the remaining tokens are withdrawn from the pool contract with a fee, which is not controlled or limited by the user. The recommendation is to allow users to pass a `maxFee` parameter to control fees. This would give the user more control over the fees associated with withdrawing from the Pod.

## Details
#### Description


When withdrawing from the Pod, the shares are burned, and the deposit is removed from the Pod. If there are not enough deposit tokens in the contract, the remaining tokens are withdrawn from the pool contract:


**code/pods-v3-contracts/contracts/Pod.sol:L523-L532**



```
if (amount > currentBalance) {
    // Calculate Withdrawl Amount
    uint256 \_withdraw = amount.sub(currentBalance);

    // Withdraw from Prize Pool
    uint256 exitFee = \_withdrawFromPool(\_withdraw);

    // Add Exit Fee to Withdrawl Amount
    amount = amount.sub(exitFee);
}

```
These tokens are withdrawn with a fee from the pool, which is not controlled or limited by the user.


#### Recommendation


Allow users to pass a `maxFee` parameter to control fees.
