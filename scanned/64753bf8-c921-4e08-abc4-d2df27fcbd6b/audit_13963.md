# Swap functions with sell orders in LSSVMRouter will fail for property-check enforced pairs

## Summary
This bug report is about an issue with the LSSVMRouter and VeryFastRouter contracts. Specifically, the swap functions with sell orders in the LSSVMRouter will revert for property-check enforced pairs, while the VeryFastRouter swap function supports sell orders to specify property check parameters for pairs enforcing them. The recommendation is to either deprecate the LSSVMRouter or add support to it for feature-parity with the VeryFastRouter. Sudorandom Labs has stated that the VeryFastRouter is intended to be a router rewrite that addresses many of the concerns with the LSSVMRouter, and that the LSSVMRouter will be deprecated once full test coverage for the VeryFastRouter is achieved. Spearbit has acknowledged this.

## Details
## Severity: High Risk

## Context
- LSSVMRouter.sol
- VeryFastRouter.sol#L36-L37
- VeryFastRouter.sol#L118-L140

## Description
Swap functions with sell orders in LSSVMRouter will revert for property-check enforced pairs. While VeryFastRouter's swap function supports sell orders to specify property check parameters for pairs enforcing them, none of the swap functions in LSSVMRouter support the same.

## Recommendation
Deprecate LSSVMRouter or add support to it for feature-parity with VeryFastRouter.

## Sudorandom Labs
The VeryFastRouter is intended to be a router rewrite that addresses many of the concerns with the LSSVMRouter. The LSSVMRouter has been modified very little from the audit in v1, with most of the new features (e.g. property check support, multiple token output support, buying with ETH and ERC20, and partial fill) being all moved to the new router. Acknowledged, as mentioned above, property checking is supported in the new VeryFastRouter. LSSVMRouter will be deprecated once full test coverage for VeryFastRouter is achieved.

## Spearbit
Acknowledged.
