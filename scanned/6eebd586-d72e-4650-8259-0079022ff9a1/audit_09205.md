# Invariants are not checked after calling calculateDistributeExcessIdleShareProceedsNetLongEdgeCaseSafe

## Summary
The bug report discusses a problem with the calculation of distribute excess idle share proceeds in the LPMath.sol file. This issue has a medium risk severity and is found in lines 653-666. The calculation is not accurate and can result in incorrect values. The report suggests two recommendations to fix the issue and mentions that one of the recommendations has already been applied in a previous update. The issue has been verified and is being addressed by the developers.

## Details
## Medium Risk Report

**Severity:** Medium Risk  
**Context:** LPMath.sol#L653-L666  

**Description:**  
`calculateDistributeExcessIdleShareProceedsNetLongEdgeCaseSafe` returns:
- \( z\text{proceed} = z\text{og} \cdot \text{log}\left(\frac{s\text{LP}}{s\text{LP}+s\text{W}}\right)PV_0 - z\text{flat} \)

Or in other words:
- \( z\text{og} \cdot \left(1 - \text{log}\left(\frac{s\text{LP}}{s\text{LP}+s\text{W}}\right)PV_0 - z\text{flat}\right) \)

1. This scaling is independent of the number of iterations performed in the Newton approximation loop. This is not really an issue, just an observation that one determines this specific value only based on the original point.
   
2. After one finds \( z\text{we} \), it is still necessary to check that \( \text{uint256}(_\text{params.netCurveTrade}) \leq \text{maxBondAmount} \) is false on the updated curve \( C(z, \theta, y) \). Otherwise, the assumption where \( z + z\text{curve} - z\text{min} = \text{log}(z\text{og})z \) would be incorrect, which would break:
   \[
   PV_0 \cdot s\text{LP} + s\text{W} = PV_1 \cdot s\text{LP}
   \]
   
3. \( z\text{proceed} \) is not compared to the idle share reserves to ensure it cannot be greater than that value. We need to check:
   \[
   z\text{proceed} \leq I
   \]

**Recommendation:**

1. After updating the point to the new point \( (z, \theta, y) \) by the scaling factor:
   \[
   \eta = \text{log}\left(\frac{s\text{LP}}{s\text{LP}+s\text{W}}\right)PV_0 - z\text{flat}
   \]
   Make sure that \( \text{uint256}(_\text{params.netCurveTrade}) \leq \text{maxBondAmount} \) is false.

2. Check that \( z\text{proceed} \leq I \).

**DELV:**  
1. The second recommendation has been applied in PR 800, which had an error that was fixed in PR 827.  
2. The above PRs, along with PR 771 below, will guarantee the third recommendation.

**Spearbit:** Verified.
