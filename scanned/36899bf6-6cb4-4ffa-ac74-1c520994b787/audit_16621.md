# Too many bytes are checked to verify the function selector

## Summary
This bug report is about a problem in the code of three smart contracts (SwapperV2.sol, Swapper.sol, and LibStorage.sol). The issue is that the function _executeSwaps() slices the callData with 8 bytes, while the function selector is only 4 bytes. This means that additional bytes are being checked unnecessarily, which is not intended. 

To fix this issue, the recommendation is to limit the check on function signatures to 4 bytes in SwapperV2.sol and Swapper.sol, and to also change the type of dexFuncSignatureAllowList to bytes4. In DexManagerFacet.sol, the related functions should be changed from bytes32 to bytes4. 

The bug was fixed with PR #8 and verified by Spearbit.

## Description
The function `_executeSwaps()` slices the `callData` with 8 bytes. The function selector is only 4 bytes. Refer to the documentation for additional details. Thus, the check for extra bytes may be unnecessary and likely not desired.

Present in both `SwapperV2.sol` and `Swapper.sol`:

```solidity
function _executeSwaps(...) { 
    ...
    if (
        !(appStorage.dexAllowlist[currentSwapData.approveTo] &&
          appStorage.dexAllowlist[currentSwapData.callTo] &&
          appStorage.dexFuncSignatureAllowList[bytes32(currentSwapData.callData[:8])]) // should be 4
        ) revert ContractCallNotAllowed();
    ...
    }
}
```

### Definition of `dexFuncSignatureAllowList` in `LibStorage.sol`
```solidity
struct LibStorage {
    ...
    mapping(bytes32 => bool) dexFuncSignatureAllowList; // could be bytes4
    ...
}
```

## Recommendation
Limit the check on function signatures to 4 bytes in `SwapperV2.sol` and `Swapper.sol`:

```diff
-appStorage.dexFuncSignatureAllowList[bytes32(currentSwapData.callData[:8])]
+appStorage.dexFuncSignatureAllowList[bytes4(currentSwapData.callData[:4])]
```

Change the type of `dexFuncSignatureAllowList` to `bytes4`:

```diff
struct LibStorage {
    ...
    - mapping(bytes32 => bool) dexFuncSignatureAllowList;
    + mapping(bytes4 => bool) dexFuncSignatureAllowList;
    ...
}
```

In `DexManagerFacet.sol`, change the related functions from `bytes32` to `bytes4`.
