# owner orwallet recipient can mint tokens for free.

## Summary
This bug report is about a vulnerability in the FantomToken contract which allows the owner to mint tokens for free. This could be done by calling the setWallet() function at any time, which would allow the owner to cyclically purchase tokens during the crowdsale. The owner could also change the wallet address to themselves, purchase tokens a number of times, then change the wallet address back. This would allow the owner to mint tokens for free, including the amount specified by the TOKEN_MAIN_CAP. The vulnerability has a low likelihood of being exploited as it can only be performed by the owner or designated wallet address. However, potential investors could be diluted in the event of a malicious owner or if an attacker gains control of the wallet or owner’s private keys. The authors of this crowdsale deemed the risk to be low and no action was taken. The resolution for this issue is to only allow the ether purchased in the crowdsale to be withdrawn once the crowdsale has completed.

## Details
Description
The owner has the ability to call the setWallet() function at any time without restriction. The wallet
address immediately receives all ether that is deposited into the FantomToken contract.
This allows the owner or the wallet beneficiary to cyclically purchase tokens during the crowdsale. Consider
the cycle: wallet user buys tokens using 100 ether . The 100 ether is immediately returned to the user
and the user is credited with tokens. The user repeats this process, accumulating tokens for free.
Equivalently, the owner during the crowdsale, can change the wallet address to themselves, purchase tokens
a number of times, then change the wallet address back.
Although the owner can already mint tokens for free, they are capped to TOKEN_TOTAL_SUPPLY - TOKEN_MAIN_CAP .
This vulnerability allows the owner to mint tokens for free, including the amount specified by TOKEN_MAIN_CAP .
This is a low likelihood attack as only the owner or designated wallet address can perform this attack.
Note: The level of trust on the owner may be acceptable for the authors of this crowdsale. This issue is raised as
potential investors can be diluted in the event of a malicious owner, or if an attacker gains control of the wallet or
owner’s private keys.
Recommendations
Only allow the ether purchased in the crowdsale to be withdrawn once the crowdsale has completed.
Resolution
As only a small portion of funds are being raised via the crowdsale, the likelihood and the impact of this issue
are low and therefore no action was deemed necessary.
