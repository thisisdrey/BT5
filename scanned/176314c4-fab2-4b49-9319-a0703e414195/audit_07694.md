# Rounding Issue Causing Dust Amount Being Locked

## Summary
The bug report discusses a rounding error in the calculation of the minter payoff in the LeverageSmartTrendVault.sol contract. This error results in a small amount of collateral being locked in the vault forever. The report recommends replacing a certain expression in the code to fix the issue, which has already been implemented in a recent commit.

## Details
## Description

A rounding error in calculating the minter payoﬀ would result in collateral dust amount being locked forever in the vault. The minter payoﬀ has the following expression:

```solidity
payoff = payoffWithFee - fee + (amount * 1e18 - amount * collateralAtRiskPercentage) / 1e18;
```

Reducing `amount * collateralAtRiskPercentage` from `amount * 1e18` and then dividing the result by `1e18` will round down the result by 1 wei. So, for every minter payoﬀ, 1 wei will be locked in the vault.

## Recommendations

Replace 

```solidity
(amount * 1e18 - amount * collateralAtRiskPercentage) / 1e18
```

with 

```solidity
amount - (amount * collateralAtRiskPercentage) / 1e18
```

similar to the function `LeverageDNTVault.getMinterPayoff()`.

## Resolution

The recommendation has been implemented in commit `ef1e9b9`.
