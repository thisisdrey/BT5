# Minter can burn tokens of other accounts

## Summary
This bug report describes a critical issue where the minter is able to burn tokens from any address, even if it does not have the private key for that address. This could potentially lead to the loss of tokens for users. The recommendation is to review the logic and not allow the minter to directly burn tokens from other accounts. A suggested solution is to refer to the Openzeppelin Burnable ERC20 contract, which only allows the tokens of the sender to be burnt.

## Details
**Severity**: Critical

**Status**: Acknowledged

**Description**

The minter can burn tokens from any address, even from addresses whose private key it does not have.

**Recommendation**: 

It is advised to not allow the minter to burn the tokens directly from other accounts. Review the business and operational logic for the same. Refer this Openzeppelin Burnable ERC20 contract, which allows only the tokens of the msg.sender to be burnt.
