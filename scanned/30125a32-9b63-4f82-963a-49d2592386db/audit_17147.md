# Unchecked stake mint in ticket casting

## Summary
The contract has a bug where it does not properly check if the NftAttempts account matches the Stake account provided in the getLotTicketByStaking instruction. This allows an attacker to use another user's stake to create raffle tickets, giving them an unfair advantage. The fix for this bug is to verify that the NftAttempts account's mint_pubkey matches the attempts_nft_mint account. This can be done by adding a check in the code.

## Details
## Vulnerability in getLotTicketByStaking Instruction

In the `getLotTicketByStaking` instruction, the contract accepts an `NftAttempts` account for tracking ticket entries and takes a `Stake` account (`frakt_nft_stake_account`) which is used to compute the parameters for the rate at which raffle tickets can be created. However, the contract fails to check that the `NftAttempts` account actually corresponds to the `Stake` account provided. This allows an attacker to use another user’s stake to cast raffle tickets, giving the attacker additional tickets that they otherwise would not be able to obtain.

## Remediation

Verify that `frakt_nft_stake_account.mint_pubkey` matches the `attempts_nft_mint` account.
