# `calculateMultiplier` Will Always Return `maxMultiplier`

## Summary
The client has marked a bug as "Fixed" and provided an explanation for the fix. The bug was found in the `SapienStaking.sol` file and affects the `calculateMultiplier()` function. The function currently always returns the `maxMultiplier` if the staked amount is greater than or equal to the `BASE_STAKE`, which is always true since the staked amount is required to be greater than the `BASE_STAKE`. The recommendation is to rework the function to prevent this exploit.

**File(s) affected:**`SapienStaking.sol`

**Description:** Currently, the function `calculateMultiplier()` will always return the `maxMultiplier` if the following condition is true:

```
if (amount >= BASE_STAKE) {
            return maxMultiplier;
        }
```

However, this condition will always be true, since during staking, it is required that the amount staked be greater than the `BASE_STAKE`:

`require(amount >= BASE_STAKE, "Amount must be greater than base stake");`

**Recommendation:** Consider reworking this function to prevent the maximum multiplier from being exploited.
