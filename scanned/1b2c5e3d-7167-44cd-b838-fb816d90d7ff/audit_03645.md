# Improper Handling of Empty Items In Circular Buffer

## Summary
The report states that there is a bug in the CircularBuffer code. The function "contains" is supposed to check if a specific item exists in the buffer, but it is not ignoring empty items when doing so. This can cause the function to incorrectly identify empty items as valid entries, leading to false positives. The bug can be fixed by modifying the function to exclude empty items when searching for a specific item. This issue has been resolved in a pull request (PR#14).

## Details
## CircularBuffer::contains

The `CircularBuffer::contains` method checks whether a specific item exists within the buffer. However, it does not ignore empty items when checking for a given item. 

In the context of this `CircularBuffer`, empty items are represented as arrays filled with default values (`[0; M]`). These empty slots may skew the behavior of `contains` if they are not properly handled. Without ignoring empty items, the search may incorrectly match these slots and return a false positive. This is especially problematic when an item (such as `[0; M]`) is used to represent an empty state in the buffer but is mistakenly treated as a valid entry.

## Code Snippet

```rust
>_ api/src/types/circular_buffer.rs rust
pub fn find_index(&self, item: &[u8]) -> Option<usize> {
    self.items.iter().position(|x| x.eq(item))
}

pub fn contains(&self, item: &[u8]) -> bool {
    self.find_index(item).is_some()
}
```

## Remediation

Modify the function to exclude empty items when performing the search, ensuring that only non-empty, valid items are considered in the check.

## Patch

Resolved in PR#14.
