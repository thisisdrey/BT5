# [H-22] Users may unintentionally remove liquidity under a phishing attack.

## Summary
This bug report concerns the `removeLiquidity` function in the `Pools.sol` code. This function uses `tx.origin` to determine the person who wants to remove liquidity. This design is dangerous, as it assumes that this function is called from the router, which may not be true if the user is under a phishing attack, and could unintendedly remove liquidity. To mitigate this risk, it is recommended to make the function `_removeLiquidity` external, which can be utilized by the router, providing information of which person removes his liquidity.

## Details
The `removeLiquidity` function in `Pools.sol` uses `tx.origin` to determine the person who wants to remove liquidity.
