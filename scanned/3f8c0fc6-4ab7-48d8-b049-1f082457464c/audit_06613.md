# swapTokens Function Is Unusable

## Summary
The `_depositToCoreLendingPool` function in the `RevenueManagement.sol` contract has a `nonReentrant` modifier. However, it is only called by the `swapTokens` function, which also has a `nonReentrant` modifier. This causes the second `nonReentrant` modifier to revert, preventing the `swapTokens` function from executing successfully. To fix this issue, the `nonReentrant` modifier should be removed from the `_depositToCoreLendingPool` function. It is also recommended to implement unit testing to ensure proper execution of the function. This issue has been resolved in a recent pull request.

## Details
The [`_depositToCoreLendingPool function`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L302) has a `nonReentrant` modifier. However, it is only called by the [`swapTokens` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L224) which also has a `nonReentrant` modifier. As a result, the second `nonReentrant` modifier will revert since it would detect that the contract has been re\-entered. This means that the `swapTokens` function will never execute successfully.


Consider removing the `nonReentrant` modifier from the [`internal _depositToCoreLendingPool` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/RevenueManagement.sol#L302). Additionally, consider [implementing unit testing](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/test/unit_tests/TestUnitRevenueManagement.t.sol#L310) to ensure that the function executes properly.
