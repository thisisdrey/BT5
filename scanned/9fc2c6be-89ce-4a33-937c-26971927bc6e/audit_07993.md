# Withdraw Message Would Be Replayable On A Different Chain

## Summary
This bug report describes a problem with the calculation of a message digest in a contract. The message digest is used to verify the identity of the person withdrawing funds. However, the current calculation does not include a chainId, which can cause issues if the contract is deployed on a different chain. This could result in users being able to withdraw funds twice. The recommendation is to add a chainId value to the message digest calculation to prevent this issue. The bug has been resolved.

## Details
**Severity**: High

**Status**: Resolved 

**Description**

The message digest calculated for the withdraw includes `msgSender`, `amount`, `expires`, `nonce`, and `address(this)`, and then the `withdrawSigner` is recovered. But this message does not include a chainId due to which if the contract gets deployed on another chain then the same message would be valid there with a different `chainId` and the user would essentially withdraw twice.

**Recommendation:**

Add a chainId value to the message digest calculation.
