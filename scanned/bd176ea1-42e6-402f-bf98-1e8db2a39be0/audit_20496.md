# Too complex irrevertible construction

## Summary
This bug report is about a field called "leader_status" in the User structure, which can only have two values. To save storage and gas usage, it is recommended to use a mapping for leader_status with an address to a boolean connection. The method for setting a user as a leader is currently irreversible, so it is suggested to change it to a toggle-like method where the status can be unset. This will simplify the construction and make it easier to use.

## Details
**Description**

Field “leader_status” of User structure can have only 2 values. In order to save storage and gas
usage, consider usage of mapping leader_status with address to bool connection. Also,
setting user as a leader is irrevertible. Consider changing the method
“setLeaderAddress(address)” to “toggleLeaderAddress(address)” where the status can be
unset.

**Recommendation**:

Simplify the construction and consider usage of toggle-like method for leader setting.
