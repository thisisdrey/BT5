# Possible flashloan attack

## Summary
This bug report describes an issue with the current version of the flashloan function. It is possible for any user to set the exchange rate to 0, which could be used in another protocol to steal assets. The recommendation is to transfer tokens to the user directly without using the doTransferOut function. This bug report is important for developers and users of the flashloan function as it could result in the loss of assets.

## Details
##### Description
Current version of flashloan function gives opportunity to any user to set `exchangeRate = 0`, which can be used in another protocols to steal assets:
https://github.com/CreamFi/compound-protocol/blob/23a4ae93adc70334553f5a83429a4e967c1eefaa/contracts/CCollateralCapErc20.sol#L169

##### Recommendation
We recommend to transfer tokens to user directly without using `doTransferOut` function.
