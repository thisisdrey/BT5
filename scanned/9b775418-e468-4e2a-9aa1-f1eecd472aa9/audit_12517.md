# Incorrect Fee checks

## Summary
The code for the functions initiateBorrow and initiateRepay in the SpokeHandler.sol file is not properly checking for fees, allowing users to bypass them. This can result in an attacker slowly taking Ether from the spoke. The initiateRepay function also uses the entire value of msg.value as the repayment amount without deducting any fees, leading to the appropriation of funds from the spoke. To fix this issue, the fee check condition in initiateBorrow needs to be modified and the repaidAmount field in initiateRepay should be set to msg.value - fee. This bug has been fixed in the code.

## Details
## Vulnerability Report

## Overview
The functions `initiateBorrow` and `initiateRepay` implement fee checks incorrectly, allowing users to partially or entirely bypass required fees.

## initiateBorrow

In `initiateBorrow`, a user may avoid paying the `forwardFee` due to leniency in the highlighted portion of the code below. This vulnerability allows an attacker to slowly deplete Ether from the spoke by an amount equal to `forwardFee` every check.

```solidity
function initiateBorrow(
    address borrower,
    uint256 borrowAmount,
    bool borrowTypeStable,
    uint16 targetChainId,
    address targetAddress,
    uint256 transportFees
)
internal
{
    /* ... */
    if (spokeChainId == hubChainId) {
        /* ... */
    } else {
        (uint256 fee,) = estimateTransportCost(hubChainId, GAS_LIMIT);
        (uint256 forwardFee,) = estimateTransportCost(24, GAS_LIMIT);
        require(transportFees >= fee, Errors.INVALID_ARGUMENTS);
        IGateway(gateway).pike_send{value: fee + forwardFee}(
            DataTypes.Transport.WORMHOLE,
            hubChainId,
            payload,
            payable(borrower),
            address(0)
        );
    }
}
```

## initiateRepay

In `initiateRepay`, the entire value of `msg.value` is utilized as the repayment amount without deducting any fees. This results in the appropriation of funds from the spoke.

```solidity
function initiateRepay(
    address repayer,
    uint256 repayAmount,
    bool borrowTypeStable
)
internal
{
    /* ... */
    bytes memory payload = abi.encode(
        DataTypes.HubRepay({
            action: DataTypes.Action.HUB_REPAY,
            user: payable(msg.sender),
            repaidAmount: msg.value,
            borrowTypeStable: borrowTypeStable,
            targetChainId: spokeChainId
        })
    );
    if (spokeChainId == hubChainId) {
        /* ... */
    } else {
        (uint256 fee,) = estimateTransportCost(hubChainId, GAS_LIMIT);
        require(msg.value >= fee, Errors.INVALID_ARGUMENTS);
        IGateway(gateway).pike_send{value: fee}(
            DataTypes.Transport.WORMHOLE,
            hubChainId,
            payload,
            payable(msg.sender),
            address(0)
        );
    }
}
```

## Remediation

- Modify the fee check condition in `initiateBorrow` to `transportFees >= fee + forwardFee`.
- Set the `repaidAmount` field in `initiateRepay` to `msg.value - fee` instead of `msg.value`.

## Patch

Fixed in commits `964c2df` and `cc64b8b`.
