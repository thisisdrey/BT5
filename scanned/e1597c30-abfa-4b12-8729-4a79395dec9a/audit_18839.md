# Full test suite is necessary

## Summary
This bug report concerns the incomplete test suite for the Gamma system. It is essential that the test suite is comprehensive and includes both edge cases and failure scenarios. This is especially important for complex systems like Gamma, as incomplete test suites can result in issues that are difficult to identify with manual reviews. The report cites an example of an issue that could have been caught by a full-coverage test suite. To ensure the Gamma system is functioning as expected, it is important that the test suite is completed and covers all possible scenarios.

## Details
#### Description


The test suite at this stage is not complete. It is crucial to have a full test coverage that includes the edge cases and failure scenarios, especially for complex system like Gamma.


As we’ve seen in some smart contract incidents, a complete test suite can prevent issues that might be hard to find with manual reviews.


Some issues such as <https://github.com/ConsenSys/gamma-audit-2022-02/issues/5,> [issue 3.2](#uniproxydepositswap---tokens-are-not-approved-before-calling-routerexactinput) could be caught by a full-coverage test suite.
