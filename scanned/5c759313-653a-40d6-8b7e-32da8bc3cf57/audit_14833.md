# TRST-H-3 Additional Satin/$CASH LP locked in an owner-created veSatin can’t be withdrawn

## Summary
A bug was reported in the protocol that allowed owners to create a special veSatin token via `createLockForOwner()` without locking LP. This veSatin token was able to revert when calling `withdraw()`, `merge()`, and `increaseUnlockTime()`. However, the `increaseAmount()` function, which is used to lock extra Satin/$CASH LP, did not revert. This could lead to partners adding Satin/$CASH LP to their position, but not being able to withdraw it, since `withdraw()` would revert.

The recommended mitigation was to revert on calls to `increaseAmount()` for veSatin tokens created via `createLockForOwner()`, like in the other functions. The team responded by fixing the issue, and the mitigation review confirmed that `increaseAmount()` can no longer be called on a veSatin created via `createLockForOwner()`.

## Details

**Team response:**
