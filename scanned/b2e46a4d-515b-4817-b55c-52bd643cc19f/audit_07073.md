# maxMint / maxDeposit can overestimate shares / assets as it ignores totalShares overflow

## Summary
This bug report discusses an issue with the maxMint function in the Vault.sol file. Currently, the function does not take into account the current total supply of shares, resulting in a potential overflow of the maximum sane amount. The recommendation is to modify the function to consider the totalShares reaching the maximum sane amount limit. The bug has been fixed in a pull request (PR 155) and the issue has been mitigated.

## Details
## Severity: Medium Risk

## Context
Vault.sol#L64-L66

## Description
The `maxMint` function currently returns 
```solidity
shares < MAX_SANE_AMOUNT ? shares : MAX_SANE_AMOUNT
```
where shares are the max-deposit assets converted to shares. However, it needs to take the current `totalSupply` into account as `totalSupply + shares <= MAX_SANE_AMOUNT` should hold. 

It can return a larger amount than what can actually be accepted. According to EIP4626, this breaks the behavior: 

- MUST return the maximum amount of shares mint would allow to be deposited to the receiver and not cause a revert, which MUST NOT be higher than the actual maximum that would be accepted (it should underestimate if necessary).

Note that `maxDepositInternal` only looks at cash, and if the vault has high utilization, `maxDepositInternal` might return a large value, indeed resulting in a large shares amount that would overflow the `totalSupply`'s `MAX_SANE_AMOUNT`.

## Recommendation
Consider changing the `maxMint` and `maxDeposit` functions to take into account `totalShares` reaching the `MAX_SANE_AMOUNT` limit.

### Example for maxMint only
```solidity
function maxMint(address account) public view virtual nonReentrantView returns (uint256) {
    VaultCache memory vaultCache = loadVault();
    if (isOperationDisabled(vaultCache.hookedOps, OP_MINT)) return 0;
    
    // Make sure to not revert on conversion
    uint256 shares = maxDepositInternal(vaultCache, account)
        .toAssets()
        .toSharesDownUint256(vaultCache);

    // ! <- old line
    return shares < MAX_SANE_AMOUNT ? shares : MAX_SANE_AMOUNT;
    
    // + <- new line
    uint256 remainingSupply = vaultCache.totalShares - shares;
    return shares < remainingSupply ? shares : remainingSupply;
}
```
