# [M-01] TokenHandler.sol, L174 - `.transfer` is bad practice

## Summary
JMukesh found a vulnerability in the `TokenHandler.sol` code, which uses the `.transfer` method to send ether. This is now considered bad practice, as the gas costs can change and break the code. It is recommended to use `call` instead, and to check for reentrancy. Jeffywu (Notional) commented that this is a best practice recommendation, and GHoul-sol (judge) agreed. Additional conversation can be found in the link provided. 

It is important to be aware of the best practices for coding, as this can help to ensure that the code is secure and will not be broken by changes in gas costs. When sending ether, it is best to use `call` instead of `.transfer` and to check for reentrancy.

## Details
_Submitted by JMukesh, also found by tensors_

The use of `.transfer` in [`TokenHandler.sol` L174](https://github.com/code-423n4/2021-08-notional/blob/4b51b0de2b448e4d36809781c097c7bc373312e9/contracts/internal/balances/TokenHandler.sol#L174) to send ether is now considered bad practice as gas costs can change which would break the code.

See [stop using soliditys transfer now](https://consensys.net/diligence/blog/2019/09/stop-using-soliditys-transfer-now/), and [istanbul hardfork eips increasing gas costs and more](https://chainsecurity.com/istanbul-hardfork-eips-increasing-gas-costs-and-more/).

Recommend using `call` instead, and make sure to check for reentrancy.
