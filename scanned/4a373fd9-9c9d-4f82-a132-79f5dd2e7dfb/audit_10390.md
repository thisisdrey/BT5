# Premature Token Burn In Withdraw

## Summary
The bug report states that there is an issue with the withdraw function in GMIndex. When a user tries to withdraw their GMI tokens, the function burns the tokens at the beginning, which reduces the total supply. This causes a problem because the function then calculates the withdrawal amount based on the reduced supply, resulting in an incorrect calculation. This means that users may be able to withdraw more funds than they originally had. The report suggests that the burn should happen after the calculation to ensure an accurate total supply is used. This issue has been fixed in the latest version of the code.

## Details
## GMIndex Withdraw Issues

In GMIndex, invoking the `withdraw` function burns GMI tokens at the beginning of the function before calculating the withdrawal amount. This may result in an incorrect calculation of the withdrawal amount due to the premature burn since it reduces the total supply. As this occurs before calculating the withdrawal amount, it unintentionally increases the effective withdrawal amount (`amountOut`), enabling users to withdraw more funds than the original amount.

## HGMIndex.sol SOLIDITY

```solidity
function withdraw(bytes32 name, uint256 _amount, address _recipient, uint256 _minAmountOut) external {
    [...]
    burn(_amount);
    uint256 totalValue = totalControlledValue();
    [...]
    {
        uint256 withdrawTokenPrice = _getPrice(name);
        uint256 withdrawValue = (_amount * totalValue) / totalSupply();
        uint256 bankBalance = _bankBalance(name);
        uint256 targetBalance = _targetBalance(name);
        withdrawAmount = (withdrawValue * 10 ** IERC20Metadata(address(token)).decimals()) / withdrawTokenPrice;
        fee = FeeUtils.getFeeBasisPoints(bankBalance, targetBalance, -int256(withdrawValue), feeBase, taxBase);
    }
    uint256 feeAmount = (withdrawAmount * fee) / FEE_DIVISOR;
    uint256 amountOut = withdrawAmount - feeAmount;
    [...]
}
```

## Remediation

Burn GMI tokens after calculating the withdrawal amount, ensuring an accurate total supply is utilized in the calculation.

## Patch

Fixed in `b3d224a`.
