# Code too long and complex

## Summary
This bug report is about the DSTContract code being too long and complex. Security is dependent on the match between the developer's intention and what the code actually does, but this is very difficult to verify when the code is huge and messy. It is therefore important to write code that is simple and modular, and files with more than 700 lines of code, such as DSTContract.sol, should not be used. The contract could be modularized into a parent contract that handles executive authorization and impeachment proposals, and the rest of the code should be separated. This would make the code easier to read and verify, and thus increase security.
