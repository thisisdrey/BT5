# [M-10] Add a timelock to `setPlatformFee()`

## Summary
This bug report concerns two lines of code on two separate files in a GitHub repository: SplitFactory.sol on line 120 and RoyaltyVault.sol on line 67. The vulnerability is that no timelock capabilities are used, which impacts multiple users who may want to be notified ahead of time of any critical changes. The recommended mitigation step is to add a timelock to the function setPlatformFee(). Adding a timelock to this function would provide more guarantees and reduce the level of trust required, thus decreasing risk for users. It would also indicate that the project is legitimate.
