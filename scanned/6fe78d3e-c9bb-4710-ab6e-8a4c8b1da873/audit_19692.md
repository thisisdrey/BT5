# [M-08] LPToken.set_minter() doesn’t check that _minter doesn’t equal zero

## Summary
A bug was identified in the function `LPToken.set_minter()` of the 'pants' system. This bug means that if the wrong address is passed as an argument, the system will lose its minter and won't be able to set a new one. This was identified during a manual code review. To fix this, the system should check that `_minter` doesn't equal zero before it sets it as the new minter.

## Details
## Handle

pants


## Vulnerability details

The function `LPToken.set_minter()` doesn't check that `_minter` doesn't equal zero before it sets it as the new minter.

## Impact
This function can be invoked by mistake with the zero address as `_minter`, causing the system to lose its minter forever, without the option to set a new minter.

## Tool Used
Manual code review.

## Recommended Mitigation Steps
Check that `_minter` doesn't equal zero before setting it as the new minter.
