# No response check after function call to transfer tokens

## Summary
This bug report is about a problem in the Yearn Vaults codebase. According to the ERC-20 standard, the token transfer functions should return a boolean variable. This variable is necessary to process the function response after the token transfer. However, there is no such functionality on the lines of code specified in the report. Therefore, it is recommended to add a return value check after the token transfer in order to fix this bug.

## Details
##### Description
According to the ERC-20 standard, the token transfer functions return a boolean variable.
It is necessary to process the function response after the token transfer.
But now there is no such functionality on the following lines:
- https://github.com/yearn/yearn-vaults/tree/e390c2a6b2ba6e2ecc8f3a72a1ea4adfabd17544/contracts/BaseWrapper.sol#L137
- https://github.com/yearn/yearn-vaults/tree/e390c2a6b2ba6e2ecc8f3a72a1ea4adfabd17544/contracts/BaseWrapper.sol#L171

##### Recommendation
It is recommended to add a return value check after the token transfer.
