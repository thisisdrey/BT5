# avgBorrowRate can blow-up for small linearVariation 

## Summary
The report discusses a bug in the code for the SpeedJumpIrm contract, specifically in the calculation of the average borrow rate. When the variable "linearVariation" is small, the calculated rate has a significant error due to division errors. This is because the code uses an approximation for the exponential function, which is only accurate for larger values of "x". The report recommends a different calculation for small values of "x" to avoid these errors. A proof of concept is provided, along with an implementation note for fixing the bug.

## SpeedJumpIrm.sol Analysis

## Context
Location: `SpeedJumpIrm.sol#L149`

## Description
For small values of `linearVariation`, the average borrow rate calculated has a significant error which causes the result to deviate from the expected value. These errors stem from amplified division errors when divided by `linearVariation`.

The exponential function has a property that for small `x` (the terms/operations below are not in WAD):

\[
\frac{e^x - 1}{x} \approx 1 + x
\]

The `wExp(x)` defined in `MathLib` has the following property for small `x` (up to division errors):

\[
|x| < \text{LN2\_INT}
\]
\[
wExp (x) - \frac{10^{18}}{x} \cdot 10^{18} = 10^{18} + x
\]

This is due to the fact that 

\[
q = \frac{x + \text{roundingAdjustment}}{\text{LN2\_INT}} = 0.
\]

## Analysis of `avgBorrowRate` Calculation

Let's analyze the calculation of `avgBorrowRate` for small non-zero `linearVariation`:

| Parameter               | Description                                                                    |
|-------------------------|--------------------------------------------------------------------------------|
| `ravg`                  | Average borrow rate                                                            |
| `rn`                    | New borrow rate, \( rn = \frac{A \cdot V}{10^{18} - \epsilon_0} \)           |
| `rn-1`                  | `marketIrm[id].prevBorrowRate`                                               |
| `A`                     | `borrowRateAfterJump`, \( A = \frac{J \cdot rn-1}{x} \)                    |
| `x`                     | `linearVariation`                                                              |
| `J`                     | `jumpMultiplier`                                                              |
| `V`                     | `variationMultiplier`                                                          |
| \(\epsilon_2\)         | \(\frac{x^2}{2 \cdot 10^{18}} - \left\lfloor \frac{x^2}{10^{18}} \right\rfloor\), \(\epsilon_2 \in [0, 3)\) |
| \(\epsilon_0\)         | \(\frac{A \cdot V}{10^{18}} - \frac{A \cdot V}{10^{18}} \), \(\epsilon_0 \in [0, 1)\) |
| \(\epsilon_3\)         | \(\frac{x}{2} - \frac{x}{x} \), \(\epsilon_3 \in [0, 1)\)                   |
| \(\epsilon_4\)         | \(\left\{ A \left(10^{18} + \frac{x}{10}\right) \right\} \), \(\epsilon_4 \in [0, 1)\) |

#### `V` Calculation
\[
V = wExp (x) = 10^{18} + x + \frac{x^2}{2 \cdot 10^{18}} - \epsilon_2
\]

#### `ravg` Calculation
\[
ravg = \frac{rn - A}{x \cdot 10^{18}} = \frac{A \cdot V}{10^{18} - \epsilon_0 - A \cdot \frac{x}{10^{18}}}
\]
\[
= A \left( \frac{10^{18} + x + \frac{x^2}{2 \cdot 10^{18}}}{10^{18} - \epsilon_0 - A \cdot \frac{x}{10^{18}}} \right)
\]
\[
\approx \frac{A(10^{18} + x)}{10^{18} - \epsilon_0 + A \cdot \frac{\epsilon_2}{x}}
\]
Thus, as \(x\) approaches 0, `E` becomes significant:

\[
E = \frac{\epsilon_4 + A\epsilon_3}{10^{18} - A\epsilon_2 + 10^{18}\epsilon_0}{x}
\]

Then:
\[
ravg = wMulDown\left(A, 10^{18} + \frac{x}{10}\right) + E
\]

### Recommendation for Small Values
When `x` is small:
\[
|x| < \text{LN2\_INT}
\]

Calculate `ravg` as:
\[
ravg = \text{mulDivDown}\left(A, 2 \cdot 10^{18} + x, 2 \cdot 10^{18}\right)
\]
