# M-9: Treasury cap restriction will not hold and one block per month will be compromised

## Summary
This bug report discusses an issue (M-9) found by user "volodya" in the Allora blockchain project. The issue involves the treasury cap restriction not being properly enforced, which could lead to one block per month being compromised. This is due to a lack of cap checking before the `.AddEcosystemTokensMinted` function is called. The impact of this bug is that the cap restriction will not hold and one block per month will be compromised. The report includes code snippets and a recommendation for how to fix the issue, as well as a discussion among the project team members. The bug has been classified as medium and unique, and the escalation status has been resolved successfully.

```go
func GetEcosystemMintSupplyRemaining(
	ctx sdk.Context,
	k keeper.Keeper,
	params types.Params,
) (math.Int, error) {
	// calculate how many tokens left the ecosystem account is allowed to mint
	ecosystemTokensAlreadyMinted, err := k.EcosystemTokensMinted.Get(ctx)
	if err != nil {
		return math.Int{}, err
	}
	// check that you are allowed to mint more tokens and we haven't hit the max supply
	ecosystemMaxSupply := math.LegacyNewDecFromInt(params.MaxSupply).
		Mul(params.EcosystemTreasuryPercentOfTotalSupply).TruncateInt()
	return ecosystemMaxSupply.Sub(ecosystemTokensAlreadyMinted), nil
}
```
