# No validation of the address parameter value in function before using this parameter in access modifiers

## Summary
This bug report is related to the PieDao ExperiPie smart contract. The bug is that it is possible for a parameter value to be equal to zero, which would block the work of the smart contract. The bug is present in several functions, including setWrappedToProtocol(), setWrappedToUnderlying(), setProtocolToLogic(), and setUnderlyingToProtocolWrapped(). It is also present in the setDefaultController() function. The recommended solution is to add a check of the parameter value to zero before initializing the variables.

## Details
