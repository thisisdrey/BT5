# Insecure privileges for `Owner`

## Summary
A bug has been identified in the `setFactories` code of the Convex ETH Platform. This code allows the owner to change factories, meaning that the owner can change the addresses of the rewardFactory, stashFactory, and tokenFactory before calling `addPool`. This could potentially enable a front-running attack where the owner can gain an advantage. 

To address this bug, the Convex ETH Platform team has recommended that the contracts be constructed in Booster and that a mechanism of migrations be created directly in the factories.

## Details
##### Description
Owner can change factories in `setFactories` (https://github.com/convex-eth/platform/blob/754d9e700693246275b613e895b4044b63ce9ed5/contracts/contracts/Booster.sol#L95):
```solidity
rewardFactory = _rfactory;
stashFactory = _sfactory;
tokenFactory = _tfactory;
```

Via front-running attack `owner` can change these addresses before calling `addPool`.

##### Recommendation
We recommend to construct this contracts in Booster and create mechanism of migrations directly in factories.
