# Ineffective Unused Gas Penalty [core]

## Summary
This bug report discusses an issue with the gas limits in a certain operation. The operation has two gas limits, `callGasLimit` and `paymasterPostOpGasLimit`, which are used during the execution phase. However, if the `postOp` function is not called, the `paymasterPostOpGasLimit` is not penalized and still takes up space in the bundle. This can cause problems for other valid operations. The bug has been resolved in a recent pull request.

## Details
Operations specify [several gas limits](https://github.com/eth-infinitism/account-abstraction/blob/9879c931ce92f0bee1bca1d1b1352eeb98b9a120/contracts/interfaces/PackedUserOperation.sol#L10-L16) for each part of the lifecycle. The `callGasLimit` and `paymasterPostOpGasLimit` can be collectively denoted the "execution gas limit". This is because these values are related to the operation's execution and the amount of gas consumed in this phase cannot be reliably estimated during simulation.


Once an operation is executed, any unused execution gas is [partially confiscated](https://github.com/eth-infinitism/account-abstraction/blob/9879c931ce92f0bee1bca1d1b1352eeb98b9a120/contracts/core/EntryPoint.sol#L687-L701), and given to the bundler. This discourages operations that specify a much larger execution gas limit than they need, which consumes excess space in the bundle and prevents other valid operations from being included.


However, the `paymasterPostOpGasLimit` is [not penalized](https://github.com/eth-infinitism/account-abstraction/blob/9879c931ce92f0bee1bca1d1b1352eeb98b9a120/contracts/core/EntryPoint.sol#L691-L693) when the `context` is empty. Although this corresponds to a situation where the `postOp` function is not called, the gas limit is [still reserved](https://github.com/eth-infinitism/account-abstraction/blob/9879c931ce92f0bee1bca1d1b1352eeb98b9a120/contracts/core/EntryPoint.sol#L387), so it still consumes space in the bundle.


Consider always including the `paymasterPostOpGasLimit` in the penalty calculation.


***Update:** Resolved in [pull request #407](https://github.com/eth-infinitism/account-abstraction/pull/407).*
