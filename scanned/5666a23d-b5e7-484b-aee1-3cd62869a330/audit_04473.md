# Emergency Withdrawal Function Restricted by Timelock

## Summary
The BellumNursery contract has a bug in the emergency withdrawal function. This function has a modifier that enforces a 6-epoch waiting period before allowing withdrawals, which goes against the purpose of an emergency withdrawal mechanism. This means that users must wait 6 epochs (36 hours) even in emergencies, which could be dangerous in situations such as smart contract vulnerabilities, market crashes, protocol exploits, or network issues. The emergency withdrawal function is important for user fund safety and is a standard safety mechanism in DeFi. The recommended solution is to remove the timelock from the emergency withdrawal function. This issue has been solved by removing the function from the contract. The remediation hash for this fix is 357ab688b7677125df61c3661d0865bea0652542.

## Details
##### Description

The `BellumNursery.emergencyWithdrawal` function incorrectly includes the `waitForWithdrawal` modifier, which enforces a 6-epoch waiting period before withdrawals. This fundamentally contradicts the purpose of an emergency withdrawal mechanism:

  

**Current Implementation Issues:**

* Users must wait 6 epochs (36 hours) even in emergencies
* Cannot quickly respond to:

  + *Smart contract vulnerabilities*
  + Market crashes
  + *Protocol exploits*
  + Network issues

  

**Emergency Function Requirements:**

* Should provide immediate exit route
* Critical for user fund safety
* Standard DeFi safety mechanism
* Expected behavior by users
