# TypedMemView.sameType does not use the correct right shift value to compare two bytes29 s

## Summary
This bug report is about the TypedMemView.sol file at line 402. It has been identified that the function sameType() should shift 2 x 12 + 3 bytes to access the type flag (TTTTTTTTTT ) when comparing it to 0. This is due to the fact that when using bytes29 type in bitwise operations and also comparisons to 0, a parameter of type bytes29 is zero padded from the right so that it fits into a uint256 under the hood. The recommendation is to change sameType() to take the zero padding into account. A solution is proposed in the Pull Request 10. Additionally, it is recommended to leave a header comment indicating the source of libraries/contracts used. The bug has been solved in Pull Request 2394 and verified by Spearbit.

## Details
## Severity: Medium Risk

**Context:** TypedMemView.sol#L402

## Description
The function `sameType` should shift `2 x 12 + 3` bytes to access the type flag (`TTTTTTTTTT`) when comparing it to 0. This is due to the fact that when using `bytes29` type in bitwise operations and also comparisons to 0, a parameter of type `bytes29` is zero padded from the right so that it fits into a `uint256` under the hood.

```
0x TTTTTTTTTT AAAAAAAAAAAAAAAAAAAAAAAA LLLLLLLLLLLLLLLLLLLLLLLL 00 00 00
```

Currently, `sameType` only shifts the XORed value `2 x 12` bytes, so the comparison compares the type flag and the 3 leading bytes of memory address in the packing specified below:

- First 5 bytes are a type flag.
  - `ff_ffff_fffe` is reserved for unknown type.
  - `ff_ffff_ffff` is reserved for invalid types/errors.
- Next 12 are memory address.
- Next 12 are length.
- Bottom 3 bytes are empty.

The function is not used in the codebase but can pose an important issue if incorporated into the project in the future.

### Current Implementation
```solidity
function sameType(bytes29 left, bytes29 right) internal pure returns (bool) {
    return (left ^ right) >> (2 * TWELVE_BYTES) == 0;
}
```

## Recommendation
Change `sameType()` to take the zero padding into account:

```solidity
uint256 private constant TWENTY_SEVEN_BYTES = 8 * 27;
...
function sameType(bytes29 left, bytes29 right) internal pure returns (bool) {
    return (left ^ right) >> TWENTY_SEVEN_BYTES == 0;
}
```

See [summa-tx/memview-sol/pull/10](https://github.com/summa-tx/memview-sol/pull/10) for a solution.

We also recommend leaving a header comment indicating the source of libraries/contracts used.
