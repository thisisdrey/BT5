# L2USDCGateway Is Missing Rate Limiter Functionality

## Summary
The `L1USDCGateway` contract has an issue where the `_transferERC20In` function is not calling the rate limiter function `_addUsedAmount` when a user initiates a deposit. This means that USDC withdrawals will not be subject to rate limiting. To fix this, the `_addUsedAmount` function should be called when users make a withdrawal in USDC. This issue has been resolved in a recent pull request.

## Details
The [`L1USDCGateway` contract](https://github.com/scroll-tech/scroll/blob/f6894bb82f78228b349267ed814375cae2fc1483/contracts/src/L1/gateways/usdc/L1USDCGateway.sol) inherits from [`L1ERC20Gateway`](https://github.com/scroll-tech/scroll/blob/f6894bb82f78228b349267ed814375cae2fc1483/contracts/src/L1/gateways/L1ERC20Gateway.sol). When a user initiates a deposit, the `_transferERC20In` function is called, which in turn invokes the rate limiter function [`_addUsedAmount`](https://github.com/scroll-tech/scroll/blob/f6894bb82f78228b349267ed814375cae2fc1483/contracts/src/L1/gateways/L1ERC20Gateway.sol#L163). However, the [`L2USDCGateway` contract](https://github.com/scroll-tech/scroll/blob/f6894bb82f78228b349267ed814375cae2fc1483/contracts/src/L2/gateways/usdc/L2USDCGateway.sol) inherits from [`L2ERC20Gateway`](https://github.com/scroll-tech/scroll/blob/f6894bb82f78228b349267ed814375cae2fc1483/contracts/src/L2/gateways/L2ERC20Gateway.sol) which does not call the rate limiter `_addUsedAmount` function. This means that USDC withdrawals will not be subject to rate limiting.


Consider ensuring that `_addUsedAmount` is called when users make a withdrawal in USDC.


***Update:** Resolved in [pull request #927](https://github.com/scroll-tech/scroll/pull/927) at commit [be6d404](https://github.com/scroll-tech/scroll/pull/927/commits/be6d4041e62817b72fffa555d83c9c0d271b291a).*
