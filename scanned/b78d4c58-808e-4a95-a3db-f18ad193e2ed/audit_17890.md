# [M-20] Strategists can’t be removed

## Summary
This bug report is about a vulnerability in the RubiconPools/BathHouse.sol code. The issue is that there is no option to revoke a strategist's privilege, which is a very strategic role that could potentially steal LP's funds. This is a dangerous security flaw that could have a serious impact if a rogue, compromised, or cancelled strategist cannot have their permissions revoked. The bug report includes a proof of concept which shows that there is a function to approve a strategist, but no option to revoke the access. The recommended mitigation step is to add a function or change the current function to allow setting the strategist's access to false.

## Details
## Lines of code

https://github.com/code-423n4/2022-05-rubicon/blob/main/contracts/rubiconPools/BathHouse.sol#L264


## Vulnerability details

There is no option to revoke strategist's privilege.
As the strategist is a very strategic role which can effectively steal LP's funds, this is very dangerous.

## Impact
A rogue / compromised / cancelled strategist can not be revoked of permissions.

## Proof of Concept
There's a function to [approve](https://github.com/code-423n4/2022-05-rubicon/blob/main/contracts/rubiconPools/BathHouse.sol#L264) a strategist, but no option to revoke the access.

## Recommended Mitigation Steps
Add a function / change the function and allow setting strategist's access to false.
