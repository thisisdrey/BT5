# Maximum slippage shouldn’t be updated for the current batch ✓ Fixed

## Summary
This bug report details an issue that occurs when someone submits a new order in the Batched Bancor Market Maker. When the batch price is updated, it is checked if the price slippage is acceptable. The problem is that the maximum slippage can be updated during the batch, and traders cannot be sure that the price is limited as they initially expected. Additionally, if a maximum slippage is updated to a lower value, some of the orders that should lower the current slippage will also revert. 

To fix this issue, the recommendation is to save a slippage value on batch initialization and use it during the current batch. This was successfully fixed with [AragonBlack/[email protected]`aa4f03e`](https://github.com/AragonBlack/fundraising/pull/155/commits/aa4f03ef86c0f60ed2e82a87845d638738b8cdba).
