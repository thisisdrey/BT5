# `addStrategy()` Is Publicly Accessible

## Summary
A recent update has fixed an issue in the `addStrategy()` function in the `CreditCaller.sol` file. This function is responsible for linking a strategy to a `GMXDepositor` contract, which controls the creation and withdrawal of credit tokens. However, the function currently has no access control, allowing anyone to manipulate the flow of credit tokens. To fix this, the function should only be accessible to the contract owner.

## Details
**Update**
The issue has been fixed by adding the `onlyOwner` modifier to the `addStrategy()`.

**File(s) affected:**`credit/CreditCaller.sol`

**Description:** The `addStrategy()` function associates a strategy with a `GMXDepositor` contract. This strategy controls:

*   Where credit tokens corresponding to the collateral amount are minted when a farmer opens a position.
*   Where credit tokens corresponding to the borrowed amount are minted when a farmer opens a position.
*   Where credit tokens are withdrawn and burned from when a position is closed.

Currently, the function has no access control, so anyone can control the flow of credit tokens and potentially redirect it toward their own accounts or contracts.

**Recommendation:** The `addStrategy()` function should only be callable by the contract owner.
