# Inconsistent Commit Information

## Summary
The report discusses a potential vulnerability in the Query function, specifically related to the consistency of commit information and the queried height. The issue is caused by the use of a function called LastCommitInfo, which retrieves the latest commit information. However, when handling queries at a specific height, it is crucial to ensure that the commit information used is consistent with the queried height. Otherwise, incorrect proofs or verification failures may occur. The suggested solution is to use a different method, such as LoadVersion, to retrieve commit information for the exact height of the query. The issue has been fixed in a recent patch.

## Details
## Potential Vulnerability in Query

A potential vulnerability exists in `Query`, related to the consistency of commit information and the queried height, due to the use of `rs.scStore.LastCommitInfo`. Within `Query`, `LastCommitInfo` retrieves the commit information corresponding to the latest committed block. However, when handling queries at a specific height (`req.Height`), ensuring that the commit information used is consistent with the queried height is crucial.

```go
// store v2/rootmulti/store.go
func (rs *Store) Query(req abci.RequestQuery) abci.ResponseQuery {
    version := req.Height
    [...]
    if res.ProofOps == nil || len(res.ProofOps.Ops) == 0 {
        return sdkerrors.QueryResult(errors.Wrap(sdkerrors.ErrInvalidRequest, "proof is unexpectedly empty; ensure height has not been pruned"))
    }
    commitInfo := convertCommitInfo(rs.scStore.LastCommitInfo())
    commitInfo = amendCommitInfo(commitInfo, rs.storesParams)
    // Restore origin path and append proof op.
    res.ProofOps.Ops = append(res.ProofOps.Ops, commitInfo.ProofOp(storeName))
    return res
}
```

On queries for historical height, the commit information obtained using `LastCommitInfo` may not accurately represent the state at the queried height, resulting in incorrect proofs or verification failures. 

## Remediation

Ensure that the commit information utilized in the query is precisely obtained for the queried height. Instead of relying on `LastCommitInfo`, consider utilizing a method such as `LoadVersion` or an equivalent to retrieve commit information for the exact height of the query.

## Patch

Fixed in commit `88d442`.
