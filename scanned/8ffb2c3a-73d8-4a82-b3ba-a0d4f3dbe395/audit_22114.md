# SimpleInterestTermsContract will silently accept other tokens

## Summary
This bug report is about the function `registerRepayment` implemented in the `SimpleInterestTermsContract` contract. This function is meant to be called by the `RepaymentRouter` whenever a debtor repays part of an issued debt. The terms contract requires repayments to be done in a specific token, but the function could be called with a different token as parameter. The contract would recognize this, since the `valueRepaid` would only be updated when the correct token is used, but the function would silently accept any other token. It was recommended to reject any other token, by returning `false` from `registerRepayment` and this has now been fixed in the code.

## Details
The function [`registerRepayment`](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/examples/SimpleInterestTermsContract.sol#L57) implemented in [`SimpleInterestTermsContract`](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/examples/SimpleInterestTermsContract.sol) is meant to be called by the `RepaymentRouter` whenever a debtor repays part of an issued debt. Although said terms contract requires repayments to be done in a specific token [`repaymentToken`](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/examples/SimpleInterestTermsContract.sol#L32), `registerRepayment` could be called with a different token as parameter. This is recognized by the contract, since the `valueRepaid` is [only updated](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/examples/SimpleInterestTermsContract.sol#L71-L73) when the correct token is used, but the function will silently accept any other token. We would recommend to reject any other token, by returning `false` from `registerRepayment`.


***Update:** Fixed in [`1bfa459`](https://github.com/dharmaprotocol/charta/commit/1bfa459d76f8c8987ebecf9afe035e4ea7b7849e).*
