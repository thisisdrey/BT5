# Stake is locked for the contract owner in `_createPool()`

## Summary
A bug was found in the code of the Permissionless Protocol's PoolFactory.sol contract. The `_createPool()` method is called from the `createPoolInitial()` and `createPool()` methods, which are marked with the `onlyOwner` modifier. This means that when the lines of code `info.staker = msg.sender; info.stakedAmount = staking.lockStake(msg.sender);` are executed, the owner's address is used to lock the stake, instead of the manager who should be staking the tokens according to the whitepaper. 

The recommendation is to correct the lines of code so that the funds are taken from the correct staker and this staker is stored in `info.staker`. This will ensure that the correct person is staking the tokens, and that the bug is fixed.

## Details
##### Description
The `_createPool()` method is called from the `createPoolInitial()` and `createPool()` methods which are marked with the `onlyOwner` modifier.

> <..>
> To become whitelisted, the institution must first complete a due diligence, KYC and AML procedure, agree to the Clearpool Terms & Conditions, and stake an amount of CPOOL tokens before a pool can be launched.

##### Recommendation

If it is not an expected behavior, correct the lines so that the funds were taken from the right staker and this right staker were stored in `info.staker`.
