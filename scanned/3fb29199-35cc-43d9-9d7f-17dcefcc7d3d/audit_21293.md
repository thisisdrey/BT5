# Improper Bit-Length Validation in Reserve Configuration

## Summary
This bug report concerns the Aave protocol-v2 library, specifically the "ReserveConfiguration.sol" file. The issue is that the bit lengths of the provided values are not checked against the bit lengths of the corresponding fields in the "data" field. For example, if a value greater than 65535 is provided for the "ltv" field, it will result in a corruption of the liquidation threshold field. The recommendation is to make sure that the passed values fit in the corresponding fields.

Bit lengths of the provided values are not checked against bit lengths of the corresponding fields in the `data`.

For example, here

```solidity
function setLtv(ReserveConfiguration.Map memory self, uint256 ltv) internal pure {
    self.data = (self.data & LTV_MASK) | ltv;
```

providing a value greater than `65535` as `ltv` will result in a corruption of the liquidation threshold field. 

##### Recommendation
We recommend making sure that passed values fit in the corresponding fields.
