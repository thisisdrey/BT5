# [C-01] Contract is missing a `payable` function which makes it impossible to operate with native assets

## Summary
This bug report is about the lack of a `payable` `fallback` or `receive` function in the `PayrollManager` contract. This will make it impossible for the contract to work with native assets, as all transfers from the Gnosis Safe multisig to the `PayrollManager` will revert. The impact of this bug is high, as all transactions that use native assets will revert, and the likelihood of this bug is also high, as native assets are often used as a `paymentToken`. The recommendation is to add a `payable` `fallback` or `receive` function in `PayrollManager` to allow for native assets transfers.

## Details
**Impact:**
High, as all transactions that use native assets will revert

**Likelihood:**
High, as it is well expected that native assets will be used as a `paymentToken` often

**Description**

The `PayrollManager` does not have a `receive` function that is marked as `payable` neither any `payable` function at all. This will make it impossible for the contract to work with native assets because all transfers from the Gnosis Safe multisig to him will revert.

**Recommendations**

Add a `payable` `fallback` or `receive` function in `PayrollManager` to allow for native assets transfers.
