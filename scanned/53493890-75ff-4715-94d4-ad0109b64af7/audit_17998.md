# Self-trading is broken

## Summary
This bug report is about an error in the Dex's consume_events instruction, which is used for trading on a platform. The instruction has special logic for self-trade fill events, but it fails to increase the free base/quote balance, resulting in loss of funds for users. This is because the instruction is only meant to be used for normal trades and ignores the identities of the maker and taker. The report includes a proof of concept and recommends removing the special logic for self-trades to fix the issue. The bug has been fixed in the latest patch.

## Details
## Dex's Consume Events Instruction

Dex's `consume_events` instruction includes special logic for self-trade Fill events.

```rust
// dex-v4/program/src/processor/consume_events.rs:L170-L300
if taker_info.user_account == maker_info.user_account {
    ...
} else {
    ...
}
```

It correctly increases the maker’s rebate.

```rust
// dex-v4/program/src/processor/consume_events.rs:L171-L176
let maker_rebate = taker_fee_tier.maker_rebate(quote_size);
maker_account.header.quote_token_free = maker_account
    .header
    .quote_token_free
    .checked_add(maker_rebate)
    .unwrap();
```

It also correctly decreases the maker’s locked base/quote balance.

```rust
// dex-v4/program/src/processor/consume_events.rs:L194-L209
match taker_side {
    Side::Bid => {
        maker_account.header.base_token_locked = maker_account
            .header
            .base_token_locked
            .checked_sub(base_size)
            .unwrap();
    }
    Side::Ask => {
        maker_account.header.quote_token_locked = maker_account
            .header
            .quote_token_locked
            .checked_sub(quote_size)
            .unwrap();
    }
};
```

However, it fails to increase the free base/quote balance, resulting in loss of funds. We note that once this is included, the logic is equivalent to what is done for normal trades. Indeed, a self-trade Fill event is only emitted if an order’s self-trade behavior is `DecrementTake`, which ignores the identities of maker and taker.

```rust
// agnostic-orderbook/program/src/orderbook.rs:L159-L206
if self_trade_behavior != SelfTradeBehavior::DecrementTake {
    let order_would_self_trade = &callback_info[..callback_id_len]
        == (&self
            .get_tree(side.opposite())
            .get_callback_info(best_bo_ref.callback_info_pt as usize)[..callback_id_len]
            as &[u8]);
    if order_would_self_trade {
        ...
    }
}
```

## Proof of Concept

1. A user posts an ask of 1A for 1B on the exchange.
2. The user files a bid of 1A for 1B with the `DecrementTake` self-trade behavior. The order is matched, a Fill event is pushed to the event queue, and the user receives 1A minus fees.
3. When the Fill event is consumed, the user's free quote balance is not incremented.

To conclude, the user lost approximately 1B from the self-trade.

## Remediation

Dex's `consume_events` instruction does not need to handle self-trades differently. The entire if branch should be removed.

## Patch

Fixed in #55.
