# [M-21] User will loose funds

## Summary
A bug has been reported in the RubiconRouter contract, which is part of the code-423n4/2022-05-rubicon repository on GitHub. The bug can cause users to lose funds if they accidentally pass a route with only one value (route[0]=X WETH) when calling the swapForETH, swapWithETH, swapEntireBalance, or swap functions.

The bug can be triggered by a user calling the swapForETH function with the following parameters: pay_amt=500, buy_amt_min=0, route[0]=WETH, and expectedMarketFeeBPS=1. When this is done, the user will transfer 500+fees amount to the contract.

The bug occurs when the _swap function is called, as the loop will not run due to a condition failure (route.length - 1 is 1-1=0, so 0<0 is false). As a result, the currentAmount will be 0 and, since buy_amt_min is also 0, the require(currentAmount >= buy_amt_min, "didnt clear buy_amt_min"); statement will pass and 0 will be returned back. This means that the user will not receive anything in return for their transaction.

To fix this bug, the code should be updated to include a check for the route parameter, with the following line added: require(route.length>1, "Invalid route param"). This will ensure that the user is aware of the issue and can avoid the bug.

## Details
_Submitted by csanuragjain, also found by 0x1f8b_

User will lose funds if user accidentally pass route with only 1 value which is route\[0]=X WETH while calling swapForETH or swapWithETH/swapEntireBalance/swap function.

### Proof of Concept

1.  User calls swapForETH function with below params:

<!---->

    pay_amt=500
    buy_amt_min=0
    route[0]=WETH
    expectedMarketFeeBPS=1

2.  User will transfer 500+fees amount to the contract

<!---->

    require(
                ERC20(route[0]).transferFrom(
                    msg.sender,
                    address(this),
                    pay_amt.add(pay_amt.mul(expectedMarketFeeBPS).div(10000))
                ),
                "initial ERC20 transfer failed"
            );

3.  Now \_swap function is called. This function will do nothing and loop will not run due to condition failure

<!---->

    for (uint256 i = 0; i < route.length - 1; i++)

    // here since route.length - 1 is 1-1=0 so loop will not run as i=0 and 0<0 is false

4.  Since currentAmount will be 0 and buy_amt_min is also 0 so require(currentAmount >= buy_amt_min, "didnt clear buy_amt_min"); will pass and 0 will be returned back

5.  Swap is complete and user will not receive anything

#### Note:

The same need to be fixed for swapWithETH,swapEntireBalance,swap function as well

### Recommended Mitigation Steps

Add below check

    require(route.length>1, "Invalid route param");
