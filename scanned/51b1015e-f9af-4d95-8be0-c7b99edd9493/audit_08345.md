# USDC DECIMALS

## Summary
The report explains that the `USDC` token has been set up with `4` decimals instead of the accepted standard of `6` decimals. This can cause unexpected results when performing calculations. The issue has been solved by the `HbarSuite team` by adjusting the decimal value of `USDC` to `6`.

## Details
##### Description

It was found that the `USDC` token has been set up with `4` decimals. This was observed in the `/tokens/list` API endpoint.

The accepted standard for the amount of decimals to be used when working with `USDC` is `6`. By performing calculations using `4` decimal places, It might result in unexpected results.
