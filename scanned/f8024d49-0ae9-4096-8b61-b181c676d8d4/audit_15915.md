# [M-05] Admin privileges are dangerous

## Summary
This bug report describes a malicious or compromised admin executing a 100% rug pull on the `LP` contract. A rug pull is a type of attack where the attacker takes all the funds from the contract. The attacker can do this by calling the `Factory` contract to add a malicious `core` to the `LP`, which will return the `LP` contract balance when its `resolveAffiliateReward` method is called. Then, by calling `claimAffiliateReward` with the fake `core` as an argument, the attacker can steal 100% of the `LP` balance. The same applies to `withdrawPayout`.

The recommendation to make the process of adding a new `coreType` or calling `plugCore` safer is to add a time delay before a core is added to the `LP`, up until which the request will be pending. This will help mitigate the risk of a malicious actor exploiting this vulnerability to execute a rug pull.

## Details
**Likelihood:**
Low, because it requires a malicious/compromised admin

**Impact:**
High, because a rug pull can be executed

**Description**

A malicious or a compromised admin can execute a 100% rug pull in the following way:

1. The `LP` admin calls the `Factory` contract to add a malicious `core` to the `LP`
2. The malicious `core` returns the `LP` contract balance when its `resolveAffiliateReward` method is called
3. Now calling `claimAffiliateReward` with the fake `core` as an argument will result in a 100% of the `LP` balance stolen

Same thing applies to `withdrawPayout`.

**Recommendations**

Make the process of adding a new `coreType` or calling `plugCore` to be safer. One possible approach is by adding a time delay before a core is added to the `LP`, up until which the request will be pending.
