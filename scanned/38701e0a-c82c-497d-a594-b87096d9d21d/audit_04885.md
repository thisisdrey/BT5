# Extended commands read the wrong index from the array

## Summary
This bug report discusses a medium risk issue in a VM.sol file, specifically on line 49. The problem is that the VM commands contain a 1-byte flag argument that is used to identify additional features and calltype. However, when the extbit is set, the next command should be treated differently and read as a 32-byte list of indices. Currently, the increment is not being processed before the memory array read, resulting in incorrect data being returned. The recommendation is to update the _execute() function to ensure the increment is executed before the array read. This issue has been fixed by swapping the weiroll library in commit 07b60a64. The fix has been reviewed and approved by Spearbit.
