# Not checking the ERC20 transfer result on depositing funds by an athlete

## Summary
This bug report concerns the `addDistributionAmount` function, which is used to transfer funds from an athlete to the protocol. The `transferFrom` function is currently used for this purpose, but it has a flaw which allows an athlete to mark an event as `amountPaidIn` without actually depositing any tokens. To fix this, we recommend using `safeTranferFrom` from the OpenZeppelin `safeERC20` implementation. This will ensure that funds are securely transferred and the athlete cannot falsely mark an event as paid without actually depositing any tokens.

## Details
##### Description
There is an `addDistributionAmount` function which transfers funds from an athlete to the protocol. It uses the `transferFrom` function here - https://github.com/FantiumAG/smart-contracts/blob/a2d126453c1105028f12277b8f342d2cdbf01a77/contracts/claiming/FantiumClaimingV1.sol#L277. If `payoutToken` which returns false/true instead of reverting on an unsuccessful transfer would be used there, then an athlete would be able to mark the event as `amountPaidIn` without depositing any tokens.

##### Recommendation
We recommend using `safeTranferFrom` from the OpenZeppelin `safeERC20` implementation.
