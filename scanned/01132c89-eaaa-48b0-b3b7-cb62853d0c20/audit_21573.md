# [M02] Passed withdrawal request might not be withdrawable

## Summary
This bug report is about the `requestWithdrawal` function of the `PricelessPositionManager` contract in the UMA protocol. This function allows sponsors to submit a withdrawal request for a given amount of collateral. The time at which a request is expected to pass is equal or lower than the contract’s expiration time. However, in the corner case where the request’s pass time is equal to the contract’s expiry time, the request will be accepted, even though it will not be possible to actually execute it. This is because the `withdrawPassedRequest` function, which executes passed requests, can only be called before the contract’s expiration time. To avoid unexpected behaviors during withdrawal of collateral, the `requestWithdrawal` function was modified to only accept requests whose pass time will be strictly before the contract’s expiration time. This was fixed in Pull Request #1386. It should be noted that the funds aren’t locked and can be withdrawn at final settlement. The contract can also undergo emergency shutdown, which will result in pending/passed withdrawal requests being locked until settlement anyway.

## Details
The [`requestWithdrawal` function](https://github.com/UMAprotocol/protocol/blob/e6eaa48124ae3f209fb117cf05eb18292cf26d21/core/contracts/financial-templates/implementation/PricelessPositionManager.sol#L232) of the `PricelessPositionManager` contract allows sponsors to submit a withdrawal request for a given amount of collateral. A request is considered passed once a certain period of time elapses, and the function ensures that [the time at which a request is expected to pass is equal or lower than the contract’s expiration time](https://github.com/UMAprotocol/protocol/blob/e6eaa48124ae3f209fb117cf05eb18292cf26d21/core/contracts/financial-templates/implementation/PricelessPositionManager.sol#L238).


In the corner case where the request’s pass time is equal to the contract’s expiry time, the request will be accepted, even though it will not be possible to actually execute it. This is due to the fact that passed requests are executed by the sponsor calling the [`withdrawPassedRequest` function](https://github.com/UMAprotocol/protocol/blob/e6eaa48124ae3f209fb117cf05eb18292cf26d21/core/contracts/financial-templates/implementation/PricelessPositionManager.sol#L254), which can only be called *before* the contract’s expiration time (as it is marked with the [`onlyPreExpiration` modifier](https://github.com/UMAprotocol/protocol/blob/e6eaa48124ae3f209fb117cf05eb18292cf26d21/core/contracts/financial-templates/implementation/PricelessPositionManager.sol#L99)).


To avoid unexpected behaviors during withdrawal of collateral, consider modifying the `requestWithdrawal` function to only accept requests whose pass time will be strictly *before* the contract’s expiration time.


**Update:** *Fixed in [PR#1386](https://github.com/UMAprotocol/protocol/pull/1386). A comment from the UMA team worth highlighting:*



> 
>  It should be noted, however, that the funds aren’t locked and can be withdrawn at final settlement. The contract can also undergo emergency shutdown, which will result in pending/passed withdrawal requests being locked until settlement anyway. We think this change makes sense to make the inequalities consistent, but being able to withdraw after the liveness period is, nonetheless, not guaranteed.
