# Unclaimed Fees Become Inaccessible After `V2Locker.unlock()`

## Summary
The report states that there is a bug in the `Locker.claimFees()` function, which is protected by the `onlyLocked` modifier. This modifier requires `lockedUntil` to not be equal to 0. However, in the `V2Locker.unlock()` function, the contract sets `lockedUntil` to 0 before transferring the LP tokens to the recipient. This causes the fees accumulated by the locker contract to be stranded and unable to be claimed. The recommendation is to claim the swap fees within the `V2Locker.unlock()` function. The bug has been fixed in a recent commit.

## Details
##### Description
`Locker.claimFees()` is protected by the `onlyLocked` modifier, which requires `lockedUntil != 0`. Inside `V2Locker.unlock()`, the contract sets `lockedUntil = 0` before transferring the LP tokens to the recipient. During the transfer, the underlying pool credits all accumulated fees to the locker contract. Once the LP tokens are transferred out, the locker can no longer call `claimFees()` because it is no longer locked, leaving those fees permanently stranded.  

A user who forgets to call `claimFees()` before unlocking will lose the entire fee balance earned up to that moment.

##### Recommendation
We recommend claiming swap fees within the `V2Locker.unlock()` function.
