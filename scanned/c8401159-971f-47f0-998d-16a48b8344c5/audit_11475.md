# L2MessageQueue Stores Incorrect Value if not Initialized Before Appending

## Summary
The `L2MessageQueue` has a bug where the `initialize` function is not always called before the `appendMessage` function. This causes the wrong Merkle tree to be calculated after the 5th message is sent. Even if `initialize` is called afterwards, the bug cannot be fixed because messages cannot be removed and the contract cannot be upgraded. However, this bug is unlikely to occur because `appendMessage` can only be called by the `messenger`. To prevent this bug from happening, it is suggested to add a safeguard that prevents `appendMessage` from being called before `initialize`. This bug has been resolved in a recent update.

## Details
In `L2MessageQueue` it is expected that `initialize` is called before `appendMessage`. This initializes the [`zeroHashes` array](https://github.com/scroll-tech/scroll/blob/3bc8a3f5c6ac816ddffadca41024331dcf4d3064/contracts/src/libraries/common/AppendOnlyMerkleTree.sol#L17). If however, it is not, [`appendMessage`](https://github.com/scroll-tech/scroll/blob/3bc8a3f5c6ac816ddffadca41024331dcf4d3064/contracts/src/L2/predeploys/L2MessageQueue.sol#L33) can still be called. On the 5th message sent, the wrong Merkle tree will be calculated (since at that point the calculation starts using the `zeroHashes[1]` value). After that point, even if `initialize` is called, it is too late, since messages cannot be removed and the contract is not upgradeable.


Fortunately, `appendMessage` can [only be called by the `messenger`](https://github.com/scroll-tech/scroll/blob/3bc8a3f5c6ac816ddffadca41024331dcf4d3064/contracts/src/L2/predeploys/L2MessageQueue.sol#L34), which makes this scenario unlikely.


Consider adding a safeguard so that `appendMessage` cannot be called until `initialize` is called first.


***Update:** Resolved in [pull request #630](https://github.com/scroll-tech/scroll/pull/630) at commit [89814bd](https://github.com/scroll-tech/scroll/pull/630/commits/89814bde4c28901839b659409ed55b57af4ade23).*
