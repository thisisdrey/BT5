# Be careful with small transactions

## Summary
This bug report concerns two pieces of code in the DSTContract.sol file from the ether-camp/virtual-accelerator repository. Transactions executing the fallback function with amounts smaller than 1 finney (0.001 ether) will assign zero tokens to the sender. The same problem has been seen in the HackerGold.sol file, and was fixed in that case. It is recommended to do the same for DSTContract.sol, as well as for the reverse operation.

## Details
[Transactions executing the fallback function](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L140) with amounts smaller than `1 finney` will assign zero tokens to the sender. [Same problem happened with HackerGold.sol](https://github.com/ether-camp/virtual-accelerator/blob/2529ffe5efd5294b44f1bc89dc9a4721a7b16409/contracts/HackerGold.sol#L90) and was fixed in that case. We recommend doing the same for `DSTContract.sol`. Same for [the reverse operation](https://github.com/ether-camp/virtual-accelerator/blob/6d4097a08669c2520e0d5bab317b60f1d13df44e/contracts/DSTContract.sol#L147).
