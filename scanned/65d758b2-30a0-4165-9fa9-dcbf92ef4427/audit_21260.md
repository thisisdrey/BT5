# Require success in search over tokens using for-loop

## Summary
This bug report is about a problem that occurs when a token is not found. In this case, an event is emitted saying that the token was removed, even though this is not true. The recommendation is to either fail the call or not to emit the particular event. This bug report relates to the BasketFacet.sol file from the ExperiPie repository on GitHub.

## Details
##### Description
If a token was not found, it means that something unexpected happen, moreover in that case we also emit event that the token was removed, but it's not true: https://github.com/pie-dao/ExperiPie/blob/0.0.2/contracts/facets/Basket/BasketFacet.sol#L52

##### Recommendation
We recommend to fail the call or not to emit a particular event.
