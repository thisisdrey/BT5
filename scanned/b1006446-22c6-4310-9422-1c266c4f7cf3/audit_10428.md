# Initializers Can Be Called More than Once

## Summary
This bug report discusses an issue with two functions, `VAIVault.setVenusInfo()` and `XVSVault.setXvsStore()`, in the `VAIVault.sol` and `XVSVault.sol` files. These functions are used to set token addresses, but the admin can call them again and change the addresses, causing the tokens to become locked in the vault. The recommendation is to prevent these functions from being called after the contract has been initialized. The bug has been marked as "Fixed" by the client and has been addressed in specific code changes.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `c02ccb8e0bad620ede890c9bf411f98d2c2e8e75`, `53d11560f8a836b999b76405f3842757ebb6fb82`.

**File(s) affected:**`VAIVault.sol`, `XVSVault.sol`

**Description:**`VAIVault.setVenusInfo()` and `XVSVault.setXvsStore()` functions are used to set the different token addresses. However, the admin can call the function again and change the addresses, thereby locking the tokens in the vault.

**Recommendation:** Do not allow calls to `VAIVault.setVenusInfo()` and `XVSVault.setXvsStore()` once the contract has been initialized.
