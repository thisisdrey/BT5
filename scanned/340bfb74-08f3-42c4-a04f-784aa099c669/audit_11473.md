# Lack of Storage Gaps

## Summary
The contract `L2ERC20Gateway` is causing issues because it is being inherited by multiple other contracts. This means that if new storage variables are added to `L2ERC20Gateway`, it will cause problems in all the contracts that inherit from it. This can compromise the functionality of these contracts. The solution is to add a storage gap variable to prevent storage collisions and future-proof against storage changes. This issue has been resolved in a recent pull request and commit.

## Details
The contract [`L2ERC20Gateway`](https://github.com/scroll-tech/scroll/blob/3bc8a3f5c6ac816ddffadca41024331dcf4d3064/contracts/src/L2/gateways/L2ERC20Gateway.sol) is being inherited by multiple upgradeable contracts.


Without adding a storage gap, new storage variables cannot be added to `L2ERC20Gateway` without causing a storage collision in all the contracts that inherit from it. This would cause contracts to malfunction and compromise their functionalities.


Consider adding a [gap variable](https://docs.openzeppelin.com/contracts/4.x/upgradeable#storage_gaps) to future-proof base contract storage changes and be safe against storage collisions.
