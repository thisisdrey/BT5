# Token Transfers May Fail Locking Auctions

## Summary
This bug report is about a problem with ERC20 tokens which can prevent transfers to blacklisted accounts. If the token.transfer() fails during the end() function, it can cause the entire transaction to fail, making it impossible to end the auction or start new ones. The development team has fixed this issue by introducing a balance mechanism inside the start(), placeBid(), and end functions. Instead of transferring tokens with external calls, a balance mechanism is updated and token transfers are made using the deposit() and withdraw() functions. Additionally, a clearBid() function is recommended which can be used to delete existing bids and transfer the bid tokens to the receiver. It is important to note that transfers can fail for genuine reasons such as a token contract being paused.

## Details
## Description

It is possible for token transfers to fail for certain ERC20 tokens. For example, some ERC20 tokens prevent transfers to blacklisted accounts.

If the `token.transfer(bid.bidder, bid.rewardAmount);` fails during `end()`, then the entire transaction will fail. The impact is that it will be impossible to end the auction or start new auctions. Hence, both the amount of bid tokens and any current or future reward tokens (for this specific token) will be locked in the contract.

## Recommendations

Consider adding a `clearBid()` function, which may be called within a specified period of time after a bid has ended. This function would delete the existing bid and transfer the bid tokens to the receiver.

Note that it is possible for transfers to fail for genuine reasons such as a token contract being paused.

## Resolution

The development team has fixed this issue by introducing a balance mechanism inside `start()`, `placeBid()`, and `end` functions. In fact, instead of transferring tokens in these functions using external calls, a balance mechanism is updated. All the token transfers are made using `deposit()` and `withdraw()` functions.
