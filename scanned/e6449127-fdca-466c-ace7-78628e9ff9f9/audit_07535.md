# Method `confirmFailureToTransfer()` doesn’t check for seller funds

## Summary
This bug report discusses a problem with a method called `confirmFailureToTransfer()`. This method does not properly check if the seller has locked funds before transferring funds to the buyer. This can result in the buyer receiving double the amount of funds, which can lead to loss of funds for the protocol. The bug has been resolved, and the recommendation is to make sure that both the seller and buyer funds are locked before transferring funds to the buyer.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

Method `confirmFailureToTransfer()` doesn’t check if the seller has locked funds or not but only checks for buyer funds. 
There could be a case where seller funds are not locked and still the buyer gets `2*amount` when owner calls this method.
This would be loss of funds for the protocol if enough tokens are present in the wallet address to transfer to the buyer.

**Recommendation**: 

Ensure that both seller and buyer funds are locked before transferring 2X funds to the buyer.
