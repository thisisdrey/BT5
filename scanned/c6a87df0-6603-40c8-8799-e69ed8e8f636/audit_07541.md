# Slowloris Attack In HTTP Server Due To Missing ReadHeaderTimeout Configuration

## Summary
The Node HTTP server, as implemented in the file `http_server.go`, is vulnerable to a potential Slowloris attack. This attack can cause a Denial of Service (DoS) by sending partial HTTP requests to the server and keeping them open, exhausting server resources. This is because the server lacks a timeout mechanism for incomplete requests. To fix this, the Spacemesh team has implemented a `ReadHeaderTimeout` setting in the http.Server configuration, which will close incomplete request connections after a certain period of time. This has been done in the `http_server.go` file, specifically in line 82. The Spacemesh team has also provided a recommended code snippet to implement this timeout setting. This issue has been solved and the fix can be found in the referenced GitHub link.

## Details
##### Description

The Node HTTP server, as implemented in the file `http_server.go`, is currently vulnerable to a potential Slowloris attack. This vulnerability stems from the absence of the `ReadHeaderTimeout` setting in the http.Server configuration. The Slowloris attack is a type of Denial of Service (DoS) attack where the attacker sends partial HTTP requests to the server, aiming to open many connections and keep them open as long as possible, thereby exhausting server resources. Without a `ReadHeaderTimeout`, the server lacks a mechanism to time out and close these incomplete request connections, making it susceptible to such attacks.

##### Recommendation

Consider implementing timeout on the HTTP server.

```
s := &http.Server{

    Addr:              tssAddr,

    Handler:           mux,

    ReadHeaderTimeout: 5 * time.Second, // Set to a suitable value

}
```
