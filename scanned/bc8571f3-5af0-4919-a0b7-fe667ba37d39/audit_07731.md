# Limit Order Rounding

## Summary
The bug occurs when users place limit orders with extremely low prices, resulting in a non-withdrawable amount due to rounding issues. If the resulting withdrawable token amount falls between zero and one, it will be rounded down to zero, making the tokens inaccessible. This was illustrated in a test where Alice placed a limit sell order for TokenA at a very low price, Bob placed a limit sell order for TokenB, and Alice attempted to cancel or withdraw her order, resulting in a rounded down amount and loss of the placed tokens. The suggested fix is to disallow limit orders if they result in a withdrawable amount of less than one token. This issue has been fixed in a recent code update.

## Details
## When users place limit orders with extremely low prices

The resulting filled amounts may become non-withdrawable due to rounding issues. Specifically, if the resulting withdrawable token amount falls between zero and one, it will be rounded to zero, effectively rendering the tokens inaccessible.

## Proof of Concept

This is illustrated in the test given below:

> _test.gogo_
```go
func (s *DexTestSuite) TestPlaceLimitOrderEdgeCase() {
    s.fundAliceBalances(100, 100)
    s.fundBobBalances(0, 300)
    trancheKeya := s.limitSellsWithPrice(s.alice, "TokenA", 
        math_utils.MustNewPrecDecFromStr("0.0000009"), 1)
    s.bobLimitSells("TokenB", 10, 2, types.LimitOrderType_IMMEDIATE_OR_CANCEL)
    s.aliceCancelsLimitSell(trancheKeya)
    s.aliceWithdrawsLimitSell(trancheKeya)
}
```

1. Alice is funded with 100000000000 units each of TokenA and TokenB. Bob is funded with 3000000000000000 units of TokenB.
2. Alice places a limit sell order for one TokenA at an extremely low price (0.00000009). This order creates a tranche (batch of limit orders) identified by `trancheKey`.
3. Bob places a limit sell order for ten TokenB with a different price, which is designed to be an immediate or cancel (IOC) order.
4. Alice attempts to cancel her limit sell order or to withdraw the order, but since the filled amount is very small (between zero and one), it gets rounded to zero. As a result, Alice will not be able to withdraw her tokens, effectively losing the placed amount.

## Remediation

Disallow limit orders (both maker and taker) if they result in a withdrawable amount of less than one token.
