# Gamma is never scaled in USUAL token distribution formula 

## Summary
The bug report discusses a problem with the distribution of tokens in the DistributionModule contract. When the function distributeUsualToBuckets() is called, the formula for calculating the total amount of tokens to be distributed does not take into account the correct value for gamma, which is meant to adjust the token minting rate. This is because the calculation for gamma uses the wrong value for timePassed, resulting in incorrect values for kappa and mt. This can lead to underdistribution of tokens. The recommendation is to first call the function _calculateUsualDistribution() and then update the timestamp, which has been fixed in a recent commit. The risk level of this bug is low.

## Details
## Token Distribution Module Analysis

## Context
- `DistributionModule.sol#L637-L655`
- `DistributionModule.sol#L881-L890`

## Description
When `distributeUsualToBuckets()` is called, the formula for the total amount of tokens to be distributed takes into account many factors and controls that can be used to adjust the token minting rate.

One such factor is **gamma**, which is intended to upscale the amount distributed if more than 24 hours have passed since the last distribution. However, the current implementation does not calculate gamma correctly. The value of gamma depends on `timePassed` since `lastOnChainDistributionTimestamp`. The calculation here does not account for the real time duration since the last distribution.

### Function Overview
```solidity
function distributeUsualToBuckets(uint256 ratet, uint256 p90Rate)
external
whenNotPaused
nonReentrant
{
    DistributionModuleStorageV0 storage $ = _distributionModuleStorageV0();
    _requireOnlyOperator($);
    
    if (block.timestamp < $.lastOnChainDistributionTimestamp + DISTRIBUTION_FREQUENCY_SCALAR) {
        revert CannotDistributeUsualMoreThanOnceADay();
    }
    
    $.lastOnChainDistributionTimestamp = block.timestamp;
    (,,,, uint256 usualDistribution) = _calculateUsualDistribution($, ratet, p90Rate);
    
    _distributeToOffChainBucket($, usualDistribution);
    _distributeToUsualXBucket($, usualDistribution);
    _distributeToUsualStarBucket($, usualDistribution);
}
```

The `lastOnChainDistributionTimestamp` is first updated to `block.timestamp` and then `_calculateUsualDistribution()` is called, where many values like gamma, kappa, mt, etc. are calculated.

In `_calculateGamma` (where the gamma value is used in calculating kappa and mt), distribution is scaled based on `timePassed` but `timePassed = block.timestamp - lastOnChainDistributionTimestamp`, which will always be zero because the timestamp has just been updated to the current time. Therefore, the calculations do not account for any time latencies.

As a result, **gamma** will always be equal to **baseGamma**, and the calculated MT and Kappa values will be incorrect, potentially leading to an underdistribution of USUAL token rewards.

## Recommendation
It is advisable to first call `_calculateUsualDistribution`, and then set `$.lastOnChainDistributionTimestamp = block.timestamp;` later to correctly use the previous distribution timestamp.

- **Usual**: Fixed in commit `e0d6557b`.
- **Cantina Managed**: Fixed.
