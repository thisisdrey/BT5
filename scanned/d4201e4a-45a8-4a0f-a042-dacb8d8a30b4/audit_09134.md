# M-1: WETH was never set in baseLeverageExecutor.sol

## Summary
The bug report discusses an issue with the Tapioca protocol's baseLeverageExecutor contract. The problem was discovered by an auditor named AuditorPraise and it involves the WETH state variable not being properly set in the contract. This means that WETH, a type of cryptocurrency, will be set to address zero and cannot be wrapped or unwrapped. The code snippet and manual review tool used are also mentioned. The recommendation is to initialize the WETH state variable through the constructor. The issue was confirmed by other members of the judging contest and has since been fixed by the protocol team.
