# Implement Support for ERC-20 transferFrom Method in ParseEthereumTransaction

## Summary
This bug report describes an issue with the `ParseEthereumTransaction` function, which is used to analyze transactions for ERC-20 tokens. Currently, it only supports one method, `transfer`, but it is missing support for another important method called `transferFrom`. This leads to limited functionality and potential errors in interpreting transactions. The report recommends adding support for `transferFrom` in the `parseCallData` function and provides a link to the solution.

## Details
##### Description

The current implementation of `ParseEthereumTransaction` only supports parsing of the `transfer` method for ERC-20 tokens. It lacks support for the `transferFrom` method, which is a common and important function in ERC-20 contracts.

**Impact**

- Limited functionality in parsing ERC-20 transactions  
- Inability to properly handle and validate `transferFrom` transactions  
- Potential misinterpretation of `transferFrom` calls as unsupported contract calls
