# Malicious user can frontrunning buyMemecoin funtion minting less tokens for the user 

## Summary
The bug report is about a problem with the code for a smart contract called TruglyMemeception. This contract has a function called "buyMemecoin" which allows users to buy a type of cryptocurrency called "Memecoin." However, there is a vulnerability in this function that allows a malicious user to manipulate the transaction and only leave 1 wei (a very small amount) of ETH (another type of cryptocurrency) to buy the Memecoin. This means that the next user who tries to buy Memecoin will end up paying more in gas fees (transaction fees) than the actual value of the Memecoin they receive. The recommendation is to add a minimum amount of tokens that a user can buy to prevent this issue. The team responsible for the contract, Trugly Labs, has acknowledged the problem but will not fix it. Another team, Cantina Managed, has also acknowledged the issue. The risk of this bug is considered low.

## Details
## Context
`TruglyMemeception.sol#L269`

## Description
The `buyMemecoin` function can be frontrun by a malicious user, leaving only 1 wei of ETH to buy. The next user would be paying more gas fees to execute the finalization of the launch transaction than the value of the tokens received.

## Example
- **targetETH**: 100 ETH 
- **left ETH to buy**: 20 ETH

1. A malicious user sent a value of 19.999...999 ETH, resulting in a remaining `targetETH - collectedETH = 1 wei`.
2. The next user who intends to purchase 5 ETH in tokens will only buy 1 wei in tokens, and pay the gas to finalize the launch.

## Recommendation
The recommendation is to add a `minAmountTokens` that the user can use to avoid getting MEV'd.

## Trugly Labs
Acknowledged and won't fix.

Acknowledged.
