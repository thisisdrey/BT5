# What if a wrong rootHash is used?

## Summary
The bug report discusses a medium risk issue in Keystore.sol, a file used to store data. The bug occurs when the wrong `rootHash` is created due to a bug in the creation script. This can result in the account becoming unusable and unable to receive updates or validations. The report recommends creating a recovery mechanism for this situation. The issue has been fixed in PR 57 and the fix has been verified.

## Details
## Security Analysis Report

## Severity
**Medium Risk**

## Context
- Keystore.sol#L17
- Keystore.sol#L33

## Description
Assume a wrong `rootHash` is created, for example due to a bug in the creation script. Once that is updated via `handleUpdates()`, there is no way to do any more updates or validations, and the account would be bricked.

## Recommendation
Consider creating a recovery mechanism for this situation.

## Stackup
Fixed in PR 57.
