# Deflation tokens support

## Summary
This bug report discusses an issue with deflation tokens. When a user calls the `transfer(value)` function, the token balance of the owner may not increase by the specified `value`. This is because deflation tokens "burn" some of the value during the transfer process. To ensure that the token balance is increased by the expected `value`, the user must explicitly check the balance of the token owner after the transfer.

The recommendation is to either add support for deflation tokens or explicitly specify in the docstring that deflation tokens are not supported. This will help ensure that users are aware of the issue and can take the necessary steps to check the balance of the token owner after the transfer.

## Details
##### Description
It might be never known that after calling `transfer(value)` the token will really increase someone's balance by `value`, some deflation tokens "burn" some value on every transfer call. So the only way to know it is explicitly checking balance of the tokens' owner after transfer.

##### Recommendation
Add deflation tokens support or explicitly specify in docstring that deflation tokens are not supported.
