# Potential Overflow

## Summary
The `DexSwapStrategy` contract has a bug in the `bytesToSwapData` function. The function only checks that the input data is at least 96 bytes long, but later on, it is assumed that the data is at least 160 bytes long. This can cause the function to fail silently if the data is between 96 and 160 bytes long. To fix this, the initial check should be changed to at least 160 bytes or the function should be able to handle shorter inputs. This bug has been resolved in a recent update to the contract.

## Details
In the [`bytesToSwapData` function](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/libraries/swap-strategies/DexSwapStrategy.sol#L60) of the `DexSwapStrategy` contract, there is only a [single, insufficient check on `rawData.length`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/libraries/swap-strategies/DexSwapStrategy.sol#L61). This check ensures that `rawData.length >= 96`. However, [later on in the function, it is assumed that `rawData.length >= 160`](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/libraries/swap-strategies/DexSwapStrategy.sol#L78). In the case that `rawData.length` is somewhere between `96` and `160`, [the subtraction will fail silently](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/libraries/swap-strategies/DexSwapStrategy.sol#L78).


Consider [changing the initial check](https://github.com/radiant-capital/riz/blob/65ca7bb55550d30388e2b2277fefe5686ab7e4e8/src/libraries/swap-strategies/DexSwapStrategy.sol#L61) to check that the length is at least `160`. Alternatively, consider handling inputs whose length is shorter than `160`.


***Update:** Resolved in [pull request \#81](https://github.com/radiant-capital/riz/pull/81) at commit [4d61055](https://github.com/radiant-capital/riz/pull/81/commits/4d61055f9679a3209ad28ca68d539420f5d2c692). The length validation now uses 160\.*
