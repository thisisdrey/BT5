# Merkle Tree Leaf Format Could Lead to Users Losing Rewards

## Summary
Summary: The bug report discusses an issue with the computation of rewards in a Merkle tree. There is confusion over whether the rewards should be in ETH or ERC20 tokens, which could result in the wrong beneficiary receiving the rewards. The recommendation is to specify the denomination of the rewards in the leaf to ensure they are sent correctly by the function `claimRewards()`.

## Details
**Description:** A leaf in the Reach’s Merkle tree is computed as `leaf = keccak256(abi.encodePacked(msg.sender, _amount))` and can be interpreted as the caller, `msg.sender`, should get `_amount` of rewards. However, it is unclear if the amount should be in ETH or ERC20 tokens. If the variable `tokenDistribution` is wrongly set, the user might get the rewards in the wrong denomination and the original beneficiary of those rewards might not get them at all.

**Recommendation:** We recommend specifying the denomination of the rewards in the leaf as well: i.e. `keccak256(abi.encodePacked(user_address, amount, token_address))`. This would allow the function `claimRewards()` to send the rewards in the right denomination without requiring to check the value of `tokenDistribution` or to revert if the `tokenDistribution` is wrongly set.
