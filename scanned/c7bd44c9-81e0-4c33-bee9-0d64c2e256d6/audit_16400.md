# MiniGovernance’s senate has almost unlimited validity

## Summary
This bug report is about the MiniGovernance contract in the code/contracts/Portal/MiniGovernance/MiniGovernance.sol:L201. Currently, the validity period argument includes `block.timestamp`, which is going to be added a bit later in the code/contracts/Portal/utils/GeodeUtilsLib.sol:L496. This will cause the senate of MiniGovernance to have much longer validity than it is supposed to. To fix this issue, the report recommends passing only `SENATE_VALIDITY` in the `_refreshSenate` function.

## Details
#### Description


A new senate for the MiniGovernance contract is set in the following line:


**code/contracts/Portal/MiniGovernance/MiniGovernance.sol:L201**



```
GEM.\_setSenate(newSenate, block.timestamp + SENATE\_VALIDITY);

```
The validity period argument should not include `block.timestamp`, because it is going to be added a bit later in the code:


**code/contracts/Portal/utils/GeodeUtilsLib.sol:L496**



```
self.SENATE\_EXPIRY = block.timestamp + \_senatePeriod;

```
So currently, every senate of MiniGovernance will have much longer validity than it is supposed to.


#### Recommendation


Pass only`SENATE_VALIDITY` in the `_refreshSenate` function.
