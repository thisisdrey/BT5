# The risk of wrong vesting calculations

## Summary
The bug report states that there is an issue with the vesting start periods being based on a value called "TGE" without any validation. This means that there is no check to ensure that the value is correct in the functions that use it. The recommendation is to add a separate validation for a non-zero "TGE" value. This issue has been resolved.

## Details
**Description**

All the vesting start periods are based on the “TGE” value but there is no validation of its value
in the functions that are using this value.

**Recommendation**:

Create a separate validation for non-zero “TGE” value.

**Re-audit comment**:

Resolved.
