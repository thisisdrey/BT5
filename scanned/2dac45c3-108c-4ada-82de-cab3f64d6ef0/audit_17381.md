# [M-05] Should use >= instead of >

## Summary
This bug report concerns a vulnerability in the code for VaultTracker, a repository found on Github. The vulnerability is located at lines 86 and 158 of the VaultTracker.sol file. The issue is that the comparison should be 'a >= vlt.notional' instead of a > vlt.notional. If this is not done, dust amounts will always be left in vlt.notional when calling the functions removeNotional() or transferNotionalFrom(). This could lead to an incorrect balance and should be fixed as soon as possible.
