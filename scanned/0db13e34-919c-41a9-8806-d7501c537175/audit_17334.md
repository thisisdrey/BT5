# Mismatches between UMIP and implementation

## Summary
This bug report is regarding the discrepancies between the documentation and implementation of the data structures in the UMIP-152 and the `OptmisticGovenor.sol` file. The differences can be found in the `Transaction` and `Proposal` structs, as well as the data format of `ancilliaryData`. The bug report suggests updating the documentation to correspond to the implementation, as well as adding further documentation to explain how uniqueness of a proposal identifier across all users of the Optimistic Oracle is maintained in light of the removal of the `module` address from `ancilliaryData`. The bug report has not yet been fixed, and UMA has replied that it will be fixed in a separate PR updating the UMIP after the code is finalized.

## Details
In [UMIP-152](https://github.com/UMAprotocol/UMIPs/blob/master/UMIPs/umip-152.md), the documentation of several data structures is significantly different from their implementation in [`OptmisticGovenor.sol`](https://github.com/UMAprotocol/protocol/blob/fca8e24275e928f7ddf660b5651eb93b87f70afb/packages/core/contracts/zodiac/OptimisticGovernor.sol).


Differences can be found in the [`Transaction`](https://github.com/UMAprotocol/protocol/blob/fca8e24275e928f7ddf660b5651eb93b87f70afb/packages/core/contracts/zodiac/OptimisticGovernor.sol#L49-L54) and [`Proposal`](https://github.com/UMAprotocol/protocol/blob/fca8e24275e928f7ddf660b5651eb93b87f70afb/packages/core/contracts/zodiac/OptimisticGovernor.sol#L56-L59) structs as well as the data format of [`ancilliaryData`](https://github.com/UMAprotocol/protocol/blob/fca8e24275e928f7ddf660b5651eb93b87f70afb/packages/core/contracts/zodiac/OptimisticGovernor.sol#L192).


Consider updating the documentation to correspond to the implementation. Additionally, in light of the removal of the `module` address from `ancilliaryData`, consider adding further documentation to explain how uniqueness of a proposal identifier across all users of the Optimistic Oracle is maintained.


***Update:** not fixed, UMA’s reply: “will be fixed in separate PR updating the UMIP after code is finalized”.*
