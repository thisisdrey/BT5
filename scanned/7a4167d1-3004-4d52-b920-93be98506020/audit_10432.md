# Use of Unsafe Cast Operations Leading to Incorrect Voting Power

## Summary
This bug report discusses an issue with the `XVSVault.sol` file in a project. The problem occurs when the amount of XVS, a governance token, exceeds a certain limit. This can lead to incorrect calculations and potentially allow attackers to withdraw more funds than they are entitled to. The client has marked this issue as "Fixed" and recommends using a safe cast library or the `safe96()` function to prevent this problem from occurring.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `3fe0317a06936b1992929e92727c6b005a9fdffc`. The client provided the following explanation:

> Fixed in 3fe0317a. Note, however, that amounts of XVS (which is used as governance token) can’t possibly be larger than type(uint96).max.

**File(s) affected:**`XVSVault.sol`

**Description:** The following functions make use of unsafe cast operations (`uint96()`), exposing them to truncation, when reaching values higher than `type(uint96).max` (`79,228,162,514,264,337,593,543,950,335`):

1.   `deposit()`: Depositor could end up with less votes than he is entitled to.
2.   `requestWithdrawal()`: Withdrawer is debited a too small amount. 
3.   `getStakeAmount()`: May return a truncated (too small) staked amount.

**Exploit Scenario:**

1.   Attacker deposits more than `type(uint96).max` in funds using `deposit()` (at most in chunks of `type(uint96).max`, to circumvent issue 1.).
2.   Attacker requests a withdrawal of more than `type(uint96).max` in funds using `requestWithdrawal()`.
3.   Attacker waits out the corresponding lockup period for the given token pool.
4.   Attacker executes the requested withdrawal using `executeWithdrawal()`.
5.   Attacker ends up back with all his deposited funds, accumulated rewards in reward token and a surplus in votes, that have wrongfuly not been deducted due to issue 2.

**Recommendation:** We recommend the use of a [safe cast library](https://docs.openzeppelin.com/contracts/3.x/api/utils#SafeCast), or the already present function `safe96()`.
