# Unexpected Transaction Revert

## Summary
The report states that there is a bug in the veFLUO deposit function. When a user deposits an amount that exceeds the schedule's released amount, the function will revert. This is due to an overflow error in the Solidity compiler, which is not accurate in this case. The suggested fix is to add the amount to the schedule before subtracting from the released amount. This bug has been fixed in a recent update.

## Details
## Issue in veFLUO's Deposit Function

This issue arises in veFLUO’s deposit, where the execution of a statement results in a revert for a specific case.

## Problem Description

If the user has already deposited some amount for vesting and calls deposit again with a value of `amount` such that it exceeds that of `schedule.releasedAmount`, it will result in a revert due to the following operation:

```solidity
function deposit(uint256 amount, uint256 vestingDuration) public {
    [...]
    if (schedule.amount > 0) {
        schedule.amount += amount - schedule.releasedAmount;
        schedule.initialVEAmount = balanceOf(user) + veFLUOAmount;
    }
    [...]
}
```

This occurs as the Solidity compiler, which is of a version above 0.8, will detect an overflow in this case, which is not true since this amount is being added to `schedule.amount` and will result in a positive amount regardless of what the value of `amount` is.

## Remediation

Ensure that `amount` is added to `schedule.amount` before subtracting from `releasedAmount`.

```solidity
function deposit(uint256 amount, uint256 vestingDuration) public {
    [...]
    if (schedule.amount > 0) {
        schedule.amount += amount;
        schedule.amount -= schedule.releasedAmount;
        schedule.initialVEAmount = balanceOf(user) + veFLUOAmount;
    }
    [...]
}
```

## Patch

Fixed in `fef1670`.
