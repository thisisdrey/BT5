# Attacker can sweep all mint rewards when the system is in Recovery Mode 

## Summary
The report describes a bug in the TroveManager code that allows an attacker to sweep all mint rewards when the system is in Recovery Mode and the day is about to end. This can happen when the system has been in Recovery Mode for the entire day and there are only a few seconds left before the day ends. The attacker can exploit this by opening a large Trove to increase the TCR and take the system out of Recovery Mode, then increasing the debt on an existing Trove and sweeping all mint rewards at no cost. This can also be done by liquidating a few Troves to take the system out of Recovery Mode. To fix this issue, it is recommended to distribute mint rewards even when the system is in Recovery Mode and to not set the borrowing fee to zero, but to lower it instead.

## Description
When the system is in Recovery Mode, the mint rewards are not distributed within TroveManager:

```solidity
if (!_isRecoveryMode) _updateMintVolume(_borrower, _compositeDebt);
```

However, this allows an attacker to sweep all mint rewards when the system is in Recovery Mode and we're in the last seconds of a day. The attack path is the following:

1. The system has been in Recovery Mode (RM) all day and there are a few seconds left to finish the day.
2. An attacker opens a big Trove to move the TCR above the CCR and take the system out of RM:
   - There's no borrowing fee due to RM.
3. The attacker increases the debt on the existing Trove by a few wei so that this amount is stored on `accountLatestMint` and `totalMints`.
4. The day just ends a few seconds later and the attacker has swept all mint rewards for that day at zero cost.

Alternatively, in step 2, the attacker could liquidate a few Troves to take the system out of RM and still execute this attack at no cost. The impact is that the attacker can sweep mint rewards for a whole day at zero cost when the system is in RM and the day is about to end.

## Recommendation
To mitigate this issue, it is recommended to distribute mint rewards even if we're in Recovery Mode. Additionally, to avoid users sweeping mint rewards while we're in RM, it is recommended not to set the borrowing fee to zero but to lower it.
