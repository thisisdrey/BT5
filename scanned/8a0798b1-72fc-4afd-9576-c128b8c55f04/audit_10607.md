# Funds Will Get Locked in the `ReachFactory`

## Summary
The bug report discusses an unnecessary function called `withdrawETH()` that was recently added to a contract. This function is not needed because there is no way for the contract to receive ETH (a type of cryptocurrency). The only way for the contract to receive ETH is if another contract self-destructs and sends its ETH balance to the contract. The client has marked this issue as "Fixed" in a recent update. However, the report suggests that the contract needs to be revised and documented to clarify how user funds will be handled and provide a way for users to withdraw their funds from the contract. This will align with the project's goals.

## Details
**Update**
The newly added function `withdrawETH()` is unnecessary since there is no normal way for the contract to receive `ETH`. The only way is for another contract to self-destruct and provide the factory's address as the receiver of the `ETH` balance of said contract.

**Update**
Marked as "Fixed" by the client. Addressed in: `75aee5e`.

**Description:** The `ReachFactory` implements the `topUp()` and `reserveEthAllocations()`functions, which will transfer user funds to the contract. However, the expected behavior is not defined in the documentation, and there is no way of withdrawing the funds from this contract, thereby locking them forever.

**Recommendation:** Revise and document this mechanism and implement some way of withdrawing the funds from the ReachFactory contract that is aligned with the goals of the project.
