# `airdropUpdateLastClaimTime` not updated

## Summary
The bug report is about a critical issue in a function called `airdropUserClaimPlx` in the PlexusIDO contract. The function has a check that is always true because the code never updates a variable called `airdropUpdateLastClaimTime`. This means that the function will always work as if it is the first time, and the correct distribution will not happen. The recommendation is to make changes to lines #303 and #306 and remove the `airdropUpdateLastClaimTime` declaration from line #48. This will fix the issue.

## Details
**Severity**: Critical

**Status**: Resolved

**Contract**: PlexusIDO

**Function**: airdropUserClaimPlx

**Details**:

The function `airdropUserClaimPlx` has a check on tine #303 for the `airdropUpdateLastClaimTime` variable value to be equal to zero. This check will always be true, because the code does never update this variable. Therefore, each time they call, the function will work as the first time, and the correct distribution will not work.


**Recommendations**:
On lines #303 and #306, use `airdropUser.airdropUpdateLastClaimTime` instead of `airdropUpdateLastClaimTime` variable. And then remove the airdropUpdateLastClaimTime declaration from line #48 as obsolete.
