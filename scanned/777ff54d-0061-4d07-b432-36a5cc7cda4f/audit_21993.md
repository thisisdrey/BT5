# Daily limits may be bypassed via executeTransaction for certain tokens

## Summary
The TokenCard wallet allows users to store their Ether and ERC20 tokens and use these to interact with the TokenCard ecosystem. The wallet enforces daily limits to limit transfers to addresses outside of a user's whitelist. However, due to the executeTransaction function, it may be possible to bypass the daily limit for certain tokens. The executeTransaction function allows users to execute arbitrary transactions through their wallet. To prevent this function being used to bypass the daily limits, the function checks for the identifiers of transfer or approve and enforces the daily limit if they are encountered. However, due to the ERC20 race condition on the approve and transferFrom functions, many tokens have implemented safer auxiliary functions for modifying token allowances. These alternate functions, such as increaseApproval, are not checked for by executeTransaction and can be used to bypass the daily limit if the ERC20 token includes such functions. 

An exploit scenario is that an attacker gains access to the user's private key and uses increaseApproval to remove tokens beyond the daily limit from the user's wallet. The recommendation is to add a check to executeTransaction to detect calls to increaseApproval as well as other extra functions commonly found in ERC20 tokens that may also allow these limits to be bypassed. Additionally, research more flexible approaches to enforce the daily limits in calls to executeTransaction.

## Exploit Scenario

Alice holds several different tokens in her wallet. An attacker gains access to her private key. The attacker uses `increaseApproval` to remove tokens beyond the daily limit from the user's wallet.

## Recommendation

**Short term:** Add a check to `executeTransaction` to detect calls to `increaseApproval` as well as other extra functions commonly found in ERC20 tokens that may also allow these limits to be bypassed.

**Long term:** Research more flexible approaches to enforce the daily limits in calls to `executeTransaction`.
