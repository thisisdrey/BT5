# Absence of Recency Check for Oracle Price

## Summary
This bug report discusses an issue with the validation of prices in the Switchboard feature of the system. The code for retrieving and parsing data from the feed_account does not verify the timestamp, which can lead to outdated prices being accepted. This can result in inaccurate calculations and potential exploitation of the system. To fix this issue, the code needs to be modified to ensure that the price data is current and fails gracefully if it is outdated. This bug has been fixed in the commit 5a35f1f.

## Details
There is a lack of validation for the Switchboard price in `refresh_oracle_price::load_switchboard`. While it retrieves and parses data from the `feed_account` to extract the price, it does not verify the associated timestamp. This omission may result in the system accepting outdated prices if the oracle feed has not been updated for an extended period.

> _src/processors/refresh_oracle_price.rs rust_
>
> ```rust
> fn load_switchboard<'a>(oracle_switchboard: &AccountInfo<'a>) -> Result<Decimal> {
>     let feed_account = oracle_switchboard.data.borrow();
>     let feed = PullFeedAccountData::parse(feed_account).unwrap();
>     msg!("Switchboard unpack start");
>     let price = feed.value().unwrap();
>     Ok(price)
> }
> ```

There is no guarantee that the price corresponds to the current market state or a recent update. A recency check ensures that the price data fetched from the oracle is current and reflects the latest market conditions. A stale oracle price may result in inaccurate calculations of asset values, allowing the possibility of exploiting the system by utilizing old prices for arbitrage attacks.

## Remediation

Modify `load_switchboard` to verify that the price fetched from the oracle is recent. If the price is outdated, the function should fail gracefully with a specific error, preventing the protocol from using invalid data.

## Patch

Fixed in commit `5a35f1f`.
