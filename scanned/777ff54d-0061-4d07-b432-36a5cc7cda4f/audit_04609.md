# Immediately expired bid will replace good bid 

## Summary
There is a bug in the Auction.sol code where bids for BUY_IT_NOW_WITH_BIDS listings are not being accepted when they expire. This is because there is no limit on the minimum valid time for bids, allowing bidders to offer a higher bid with a very short valid time, which replaces the current best bid and expires immediately. This can prevent the seller from accepting a good bid. To fix this, it is recommended to require the expiryTimestamp to be greater than a minimum duration from the current timestamp. The bug has been fixed in PR 8 by Cantina Managed.

## Context
`Auction.sol#L150`

## Description
For listings of type `BUY_IT_NOW_WITH_BIDS`, the bidder bids with an `expiryTimestamp`, and when the bid expires, the bid will not be accepted. Since the protocol does not limit the minimum valid time, a bidder can offer a higher bid with a 1-second valid time, which will replace the current best bid and expire immediately, which may prevent the seller from accepting a good bid.

## Recommendation
It is recommended to require `expiryTimestamp` to be greater than a minimum duration from the current timestamp.
