# Loss of funds due to the method `setTokenRewards`

## Summary
A bug has been found in the Contract WaveContract.sol. The method `setTokenRewards()` allows for setting new token rewards and a new reward token address. However, once the new reward token is set, there is no way to withdraw the old token rewards that were already funded. This can result in the owner losing funds. To fix this, the method `setTokenRewards()` should be updated to return the old reward token to the owner and replace it with the new token rewards. It is also recommended to avoid updating the reward token while the contract is active. This bug has been marked as resolved.

## Details
**Severity**: High

**Status**: Resolved

**Description**

In Contract WaveContract.sol, the method `setTokenRewards()` allows setting the new tokens rewards along with a new reward token address.

Once the new reward token is set, there is no way to withdraw old token rewards that were already funded.

This will lead to the loss of funds for the owner.

**Recommendation**: 

Update the method `setTokenRewards()` to send the old reward token back to the owner and funds with the new token rewards. Also, it is advised not to update the reward token when the contract is active.
