# Incorrect 4-bytes function identifier in ERC865Token contract

## Summary
This bug report is about an issue with the `ERC865Token` contract, which is part of the Props Token Distribution project. The problem is that the 4-bytes identifier used in the `transferPreSignedHashing` function is incorrect and does not match the signature of the function. This mismatch can cause problems in higher application layers or when interacting with other third-party systems. The 4-bytes identifier must be replaced by the correct one, which is `15420b71`. The issue has since been fixed and the 4-bytes identifier now matches the signature of the function, which is `0x0d98dcb1`.

## Details
According to [its docstrings](https://github.com/propsproject/props-token-distribution/blob/eaf0c6fddd320a258a9d7531d72d07a830fae58a/contracts/token/ERC865Token.sol#L223), the [function](https://github.com/propsproject/props-token-distribution/blob/eaf0c6fddd320a258a9d7531d72d07a830fae58a/contracts/token/ERC865Token.sol#L230) [`transferPreSignedHashing`](https://github.com/propsproject/props-token-distribution/blob/eaf0c6fddd320a258a9d7531d72d07a830fae58a/contracts/token/ERC865Token.sol#L230) in the `ERC865Token` contract is in charge of computing the keccak256 hash of the payload that will be used by the [`transferPreSigned`](https://github.com/propsproject/props-token-distribution/blob/eaf0c6fddd320a258a9d7531d72d07a830fae58a/contracts/token/ERC865Token.sol#L30) function. However, the 4-bytes identifier in use (i.e. `48664c16`) is obtained from hashing the signature `transferPreSignedHashing(address,address,address,uint256,uint256,uint256)`, whereas the actual signature of the function being used is `transferPreSignedHashing(address,address,uint256,uint256,uint256)`. This mismatch may potentially cause problems in higher application layers or in the interaction with other third-party systems that use the correct signature.


Hence, the 4-bytes identifier must be replaced by the correct one, which is `15420b71`. Consider updating this value [in line 242](https://github.com/propsproject/props-token-distribution/blob/eaf0c6fddd320a258a9d7531d72d07a830fae58a/contracts/token/ERC865Token.sol#L242) of the `ERC865Token`, as well as modifying [the comment in the previous line](https://github.com/propsproject/props-token-distribution/blob/eaf0c6fddd320a258a9d7531d72d07a830fae58a/contracts/token/ERC865Token.sol#L241) so as to match both the correct identifier and function signature.


***Update:*** *Fixed. The* [*4-bytes identifier now matches the signature of the function*](https://github.com/propsproject/props-token-distribution/blob/7ead8c55da3770c4d39db8c89a8aa20ec62e97b1/contracts/token/ERC865Token.sol#L245-L246)*. Note that the function was renamed, so the signature is now* [*`0x0d98dcb1`*](https://www.4byte.directory/signatures/?bytes4_signature=0x0d98dcb1).
