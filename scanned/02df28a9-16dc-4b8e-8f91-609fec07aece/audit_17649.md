# [M-11] In `Cnote.sol`, anyone can initially become both accountant and admin

## Summary
This bug report is about a function called `_setAccountantContract()` in a contract called CNote. This function is supposed to be called after contract initialization, so that the `accountant` is immediately set. However, this function completely lacks any access control (it’s just `public`) so an attacker can monitor the mempool and frontrun the transaction in order to become both `accountant` and `admin`. To mitigate this, the function should have a guard that regulates access control and not set the `admin` too, which is dangerous and out of scope. After review, the severity of the bug was decreased from High to Medium, as the worst-case scenario is a re-deploy.
