# Preallocate function has no sanity checks

## Summary
This bug report is about the `preallocate` function in the Crowdsale.sol contract. This function had no checks to make sure the amount of tokens being sent was non-zero, the receiver address was non-zero, the state of the Crowdsale was `PreFunding`, or that the cap was not exceeded. Hubii responded by adding checks to ensure the correct values are processed by the `preallocate` function to avoid any problems. This was fixed in the `b4fccc6dee752f9b64b5598270032fd9cec106f6` commit.

## Details
`preallocate` in Crowdsale.sol [L239](https://github.com/CoinFabrik/ico/blob/142f43f3bec4afd0ffff3fce06946cf67cb0272f/contracts/Crowdsale.sol#L239) has no checks regarding the amount of tokens being non-zero, the receiver address being non-zero, the state of the `Crowdsale` being `PreFunding` (allowing preallocations to be sent even on a `Failed` or `Finalized` crowdsale), or the cap not being exceeded. Even if the method is to be invoked only by the `Crowdsale` owner, consider adding checks for all these conditions to prevent inconsistent states produced by human errors.


**Update:** Hubii replied *“Checks were added to ensure correct values are processed by the `preallocate` function to avoid any problems.”* Fixed in [`b4fccc6dee752f9b64b5598270032fd9cec106f6`](https://github.com/CoinFabrik/ico/commit/b4fccc6dee752f9b64b5598270032fd9cec106f6).
