# [M-15] Check _to is not empty

## Summary
This bug report is about the functions claimYield, _withdraw, and _unsponsor in the code. The issue is that these functions do not validate that the address _to is not an empty 0x0 address, which may lead to accidental burns. To prevent this, the recommended mitigation step is to implement a validation that requires _to to not be an empty 0x0 address.

## Details
_Submitted by pauliax_

functions `claimYield`, `\_withdraw`, and `\_unsponsor` should validate that `\_to` is not an empty 0x0 address to prevent accidental burns.

#### Recommended Mitigation Steps

Consider implementing the proposed validation:
require `\_to != address(0)`
