# `head` can't be taken out

## Summary
The bug report states that there is an issue with the `prev` account for head not existing. The report includes a link to the code where the issue is located. The recommendation is to add a check for when `creditAccount` is equal to `head`.

## Details
##### Description
`prev` account for head doesn't exist:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/core/AccountFactory.sol#L257
##### Recommendation
We recommend adding a special check for situation when `creditAccount == head`.
