# Missing whenNotPaused Modifier

## Summary
The LockZap contract has a bug where the function zapAlternateAsset does not have the whenNotPaused modifier, which means it can still be used even when the contract is paused. This could lead to additional funds being locked during an emergency. The bug has been fixed in the latest update.

## Details
The [`LockZap`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/zap/LockZap.sol) contract implements pause functionality that prevents users from locking additional funds in case of emergency. While the functions `zap`, `zapOnBehalf` and `zapFromVesting` have the `whenNotPaused` modifier correctly applied, the function [`zapAlternateAsset`](https://github.com/radiant-capital/v2-core/blob/e8f21f103e17736907b62809d1ad902febc53714/contracts/radiant/zap/LockZap.sol#L221) is missing that modifier. As a result, even when the contract is paused, it will still be possible to use the `zapAlternateAsset` function to stake additional liquidity.


Consider adding the `whenNotPaused` modifier to the `zapAlternateAsset` function so that it cannot be executed while the contract is paused.


***Update:** Resolved in [pull request #234](https://github.com/radiant-capital/v2-core/pull/234) at commit [6e5faf7](https://github.com/radiant-capital/v2-core/commit/6e5faf79dd00a5b5ddaa2af6087071d5ebeccf9f).*
