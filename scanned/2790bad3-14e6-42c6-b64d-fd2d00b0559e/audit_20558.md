# [H-05] `Withdrawable.withdraw` does not decrease `pendingWithdrawals`

## Summary
The bug report is about two issues in the code for the FairSide project. The first issue is related to a storage variable called `pendingWithdrawals` not decreasing when a withdrawal is made, which is causing incorrect behavior in the code. The second issue is with the `getReserveBalance` function, which is not accurately reporting the reserve balance and causing incorrect calculations in the `FSD.mint` function. The report recommends decreasing `pendingWithdrawals` by the withdrawn amount to fix the first issue. The second issue has already been identified and fixed in a previous pull request.

## Details
The name `pendingWithdrawals` indicates that this storage variable tracks the withdrawals that need yet to be paid out. Furthermore, this matches the behavior in `_increaseWithdrawal`. As such, it should be decreased when withdrawing in `withdraw`, but it is not.

The `getReserveBalance` function consistently under-reports the actual reserve balance, which leads to the wrong mint amounts being used in the `FSD.mint` calculation.

Recommend decreasing `pendingWithdrawals` by the withdrawn amount.
