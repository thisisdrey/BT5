# Potential Reentrancy Attack Vector in Token Creation and Launch Flow

## Summary
The `BellumFactory` contract has a potential vulnerability that could allow an attacker to manipulate conditions or steal AVAX from the contract. This is because the contract holds AVAX balances during token creation and launch, and there is a pattern in the `buy()` function that could return excess AVAX. Additionally, there is no protection against reentrancy attacks in the `createToken()` function and state changes occur after external calls. The BVSS score for this vulnerability is 6.3, and the recommended solution is to add a ReentrancyGuard to the `createToken()` function. This has already been implemented by the Bellum Exchange team and the remediation hash is 357ab688b7677125df61c3661d0865bea0652542.

## Details
##### Description

In `BellumFactory` contract, the `createToken()` function combined with the `buy()` functionality exposes a potential reentrancy attack vector that could allow an attacker to manipulate conditions or drain AVAX from the contract.

  

The vulnerability exists because:

1. Contract holds AVAX balances during token creation and launch
2. External call pattern in `buy() -> buy() -> launchToken()` flow that returns excess AVAX
3. No reentrancy guard on `createToken()`
4. State changes occur after external calls

```

```
