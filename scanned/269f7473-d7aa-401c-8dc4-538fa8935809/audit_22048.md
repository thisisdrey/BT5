# SOA record check on the wrong domain ✓ Fixed

## Summary
This bug report is about an issue discovered by the client development team during an audit. The issue was that the SOA record check in `Root.getAddress` was meant to happen on the root TLD but was instead performed on `_ens.nic.<tld>`. This issue was already fixed in [ensdomains/root#25](https://github.com/ensdomains/root/pull/25). The SOA record check is a way to verify the validity of a domain name and is used to ensure that the domain name is registered with the correct authority. The bug report was resolved by the development team fixing the code so that the SOA record check is performed on the root TLD, as it was intended.
