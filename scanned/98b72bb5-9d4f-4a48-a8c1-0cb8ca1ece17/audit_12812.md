# `boostRewards` sends tokens FROM vault instead of TO vault

## Summary
This bug report is about a critical issue that has been resolved. On line 174 of the `boostRewards` function, there is a call that mistakenly sends `accrualToken` from the vault to the caller, instead of the intended action of sending it from the caller to the vault. The recommendation is to replace `safeTransfer` with `safeTransferFrom` to fix this issue.

## Details
**Severity**: Critical

**Status**: Resolved

**Description**

on line 174 in the `boostRewards` function there is a call `IERC20(accrualToken).safeTransfer(msg.sender, amount);` which actually sends `accrualToken` to the caller from the vault. The real function should do vise versa.

**Recommendations**: 

replace `safeTransfer` with `safeTransferFrom` to send tokens from the caller to the vault.
