# M-6: parseAllPrice not support the tokens whose decimal is greater than 18

## Summary
This bug report is about the issue M-6, which is related to the `parseAllPrice` not supporting tokens with more than 18 decimals. This issue was found by kutugu and was observed that due to this issue, users cannot trade tokens larger than 18 decimal, which is not expected as DODOv3 is intended to be compatible with all standard ERC20 tokens. The vulnerability detail is that if the token decimal is greater than 18, the code will revert. This impacts the users as they cannot trade the tokens with more than 18 decimals. The code snippet and the tool used for the bug report are mentioned in the report. The recommendation is to fix the decimal to 36 instead of 18. The discussion was about the two questions raised by kutugu and it was agreed to be a medium vulnerability. The result of the discussion is also mentioned in the report. In the end, it was mentioned that tokens with more than 18 decimals will not be supported.

## Summary

`parseAllPrice` not support the token decimal is greater than 18, such as NEAR with 24 decimal.
Since `buyToken / sellToken` is dependent on `parseAllPrice`, so users can't trade tokens larger than 18 decimal, but DODOv3 is intended to be compatible with all standard ERC20, which is not expected.

## Vulnerability Detail

```solidity
        // fix price decimal
        if (tokenDecimal != 18) {
            uint256 fixDecimal = 18 - tokenDecimal;
            bidDownPrice = bidDownPrice / (10 ** fixDecimal);
            bidUpPrice = bidUpPrice / (10 ** fixDecimal);
            askDownPrice = askDownPrice * (10 ** fixDecimal);
            askUpPrice = askUpPrice * (10 ** fixDecimal);
        }
```

If `tokenDecimal > 18`, `18 - tokenDecimal` will revert

## Impact

DODOv3 is not compatible the tokens whose decimal is greater than 18, users can't trade them.
