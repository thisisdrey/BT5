# maxInvocation limits in two contracts are not synchronized

## Summary
This bug report is about the FantiumMinterV1 and FantiumNFTV1 smart contracts. The issue is that the FantiumMinterV1 implies that the maxInvocation for collections is limited to 1 million. However, the multiple setter functions in FantiumNFTV1 allow for maxinvocation for collections to be above 1 million. To fix this bug, it is recommended that for every function that modifies maxInvocation, it should be checked to ensure that it is limited by 1 million.

## Details
##### Description
FantiumMinterV1 imply that maxInvocation for collection is limited by 1 mln. But multiple setter functions in FantiumNFTV1 allow maxinvocation for collections being above 1 mln.

- https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L346
- https://github.com/metaxu-art/fantium-smart-contracts/blob/cb2d97bc30c40321991fe5ab8fc798babba1610f/contracts/FantiumNFTV1.sol#L493-L505

##### Recommendation

For every function that modifies `maxInvocation`, we recommend checking that it is limited by 1 mln.
