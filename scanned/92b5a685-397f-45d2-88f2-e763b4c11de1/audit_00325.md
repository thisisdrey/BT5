# [M-09] GTL contract is vulnerable to an inflation attack

## Summary
The GTL contract in the code is vulnerable to an inflation attack, which means a user can deposit assets and receive no shares, resulting in a loss of funds. This is because the contract does not implement any protection methods from the solady ERC4626 contract that it inherits. To prevent this, it is recommended to use one of the provided methods such as decimal offset, virtual shares, or a fixed initial conversion rate when minting shares. A detailed proof of concept can be viewed at the given link.

## Details
`perps/GTL.sol` [# L292-L294](https://github.com/code-423n4/2025-08-gte-perps/blob/f43e1eedb65e7e0327cfaf4d7608a37d85d2fae7/contracts/perps/GTL.sol# L292-L294)

GTL contract is vulnerable to an inflation attack. As a result, a user can deposit the asset in the GTL contract and receive no shares. This will cause a loss of funds for users.

The GTL contract inherits the ERC4626 contract from the solady library, but does not implement any of the provided protection methods leaving a newly deployed GTL contract vulnerable to an inflation attack.

Here is an explanation of an inflation attack if the reader is unfamiliar: <https://mixbytes.io/blog/overview-of-the-inflation-attack# rec558238760>

### Recommended mitigation steps

Use one of the provide methods inside the solady ERC4626 contract:

1. Decimal offset
2. Virtual shares
3. Fixed initial conversion rate when minting shares
