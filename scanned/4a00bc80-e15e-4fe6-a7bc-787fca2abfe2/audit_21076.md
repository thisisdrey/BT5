# Ignore failure status for `CToken`

## Summary
This bug report is about the failure of the CToken interface methods. The methods in question are `mint` and `redeemUnderlying`, which can be found in two files on GitHub. These methods return a `uint256` enum of errors, which can range from 'NO_ERROR' to 'TOO_MUCH_REPAY'. The bug report recommends that the return value of every CToken method should be validated and if it is not 'NO_ERROR', the action should be reverted.

## Details
##### Description
There are many reasons for failure `CToken`, but Lenders contracts ignore it in the all places.
Interface methods of `CToken`:
For 
```
function mint(uint256 mintAmount) external returns (uint256);
```

Return value (`uint256`) is enum of errors which may be:

```
enum Error {
    NO_ERROR,
    UNAUTHORIZED,
    COMPTROLLER_MISMATCH,
    INSUFFICIENT_SHORTFALL,
    INSUFFICIENT_LIQUIDITY,
    INVALID_CLOSE_FACTOR,
    INVALID_COLLATERAL_FACTOR,
    INVALID_LIQUIDATION_INCENTIVE,
    MARKET_NOT_ENTERED, // no longer possible
    MARKET_NOT_LISTED,
    MARKET_ALREADY_LISTED,
    MATH_ERROR,
    NONZERO_BORROW_BALANCE,
    PRICE_ERROR,
    REJECTION,
    SNAPSHOT_ERROR,
    TOO_MANY_ASSETS,
    TOO_MUCH_REPAY
}
```

##### Recommendation
We recommend to validate return of every method for `CToken`. If method returns no `NO_ERROR` — revert it.
