# Unaccounted Storage Costs

## Summary
The bug report states that there is an issue with the internal_register_account function, which is used to create new user accounts. This function does not account for storage staking, which means that eventually, users will not be able to create new accounts. To fix this issue, a nominal storage fee should be charged for creating new accounts. This bug has been fixed in version #6.

## Details
## Internal Register Account Function Issue

The `internal_register_account` function does not account for storage staking when creating new users. Eventually, users will no longer be able to create an account due to this mismatch in storage accounting.

## Remediation

Charge a nominal storage fee for the creation of a new user account.

## Patch

Resolved in #6.
