# External query gas limit may be problematic

## Summary
A bug report has been filed for the `DebtKernel` contract, which is part of the Dharmaprotocol Charta repository. The report is related to the functions `getBalance` and `getAllowance`, which are used to query the balance and allowance of any arbitrary token used to fill a debt order. When the query is done, it is done by calling the ERC20 `balanceOf` and `allowance` functions on said token contract with a gas limit of `EXTERNAL_QUERY_GAS_LIMIT`. 

The `EXTERNAL_QUERY_GAS_LIMIT` constant had a value of `4999`, which was justified by the fact that changes to state require at least 5000 gas. However, the team was concerned that an external call such as a balance check could cause a re-entrancy attack. Tests showed that a balance check in a recently cloned MiniMe token cost around 4320 gas, which was dangerously close to the constant limit imposed.

The team looked into the new opcode `STATICCALL`, which would eliminate the risk of re-entrancy by ensuring that an external call performs no state changes. However, due to the unavailability of STATICCALL in high-level Solidity code as of yet, the team decided to stick with a simple gas limit. The limit was raised to 8000 and the missing check was added.

## Details
In [`DebtKernel`](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/DebtKernel.sol), the functions [`getBalance`](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/DebtKernel.sol#L637) and [`getAllowance`](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/DebtKernel.sol#L651) are used to query the balance and allowance of any arbitrary token used to fill a debt order. The query is done by calling the ERC20 `balanceOf` and `allowance` functions on said token contract with a gas limit of [`EXTERNAL_QUERY_GAS_LIMIT`](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/DebtKernel.sol#L68). (Although `getAllowance` doesn’t limit the gas, [a comment](https://github.com/dharmaprotocol/charta/blob/b110959477cf37375bf7e9344d40eb85219c8575/contracts/DebtKernel.sol#L658) implies that it should be doing so.)


The `EXTERNAL_QUERY_GAS_LIMIT` constant has a value of `4999`, justified by the fact that changes to state require at least 5000 gas. Although we share the concern that an external call such as a balance check could cause a re-entrancy attack, the reality is that gas limits are not part of the ERC20 spec, and a token is free to implement a balance check that costs more than 4999 gas. In fact, quick tests show us that a balance check in a recently cloned [MiniMe](https://github.com/Giveth/minime/) token (a popular token implementation) costs around 4320 gas, which is dangerously close to the constant limit imposed.


We would recommend to look into the recent new opcode [`STATICCALL`](https://github.com/ethereum/EIPs/blob/master/EIPS/eip-214.md) to use as an alternative to limiting gas. This opcode will eliminate the risk of re-entrancy by ensuring that an external call performs no state changes, which is fine to assume for ERC20 functions `balanceOf` and `allowance`, as they are constant functions.


***Update:** Due to the unavailability of STATICCALL in high-level Solidity code as of yet, the team has decided to stick with a simple gas limit. However, the limit was raised to 8000 and the missing check was added in [`ff1a22e`](https://github.com/dharmaprotocol/charta/commit/ff1a22ee3974efc1493401a9f3ebb527a2b3af29).*
