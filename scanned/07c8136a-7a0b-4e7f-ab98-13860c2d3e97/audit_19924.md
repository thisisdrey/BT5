# [M-01] Admin is a single-point of failure without any mitigations

## Summary
This bug report is about the absolute power of the admin role across Swivel, Marketplace and VaultTracker contracts. There is no ability to change the admin to a new address or renounce it, which could be useful in cases of lost/compromised admin keys or to delegate control to a different governance/DAO address. The project does not use the OpenZeppelin Ownable library, which provides transfer/renounce functions to mitigate such issues. This makes the admin role/key a single-point of failure. The bug report includes proof of concept links to demonstrate the vulnerability. The recommended mitigation steps are to ensure admins are reasonably redundant/independent (3/7 or 5/9) multisigs and add transfer/renounce functionality for admin. It is also suggested to consider using the OpenZeppelin Ownable library.

## Details
_Submitted by 0xRajeev, also found by 0xsanson and leastwood_.

#### Impact

Admin role has absolute power across Swivel, Marketplace and VaultTracker contracts with several `onlyOwner` functions. There is no ability to change admin to a new address or renounce it which is helpful for lost/compromised admin keys or to delegate control to a different governance/DAO address in future.

The project does not use the widely used OpenZeppelin Ownable library which provides transfer/renounce functions to mitigate such compromised/accidental situations with admin keys. This makes admin role/key a single-point of failure.

#### Recommended Mitigation Steps

Ensure admins are reasonably redundant/independent (3/7 or 5/9) multisigs and add transfer/renounce functionality for admin. Consider using OpenZeppelin’s Ownable library.
