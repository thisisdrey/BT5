# No validation of address parameter value in contract constructor

## Summary
This bug report is about the `vault` variable in a contract. The issue is that the variable cannot be initialized correctly if the value passed to the constructor is not correct. This can cause the contract to not work as intended.

The recommendation is to add a parameter check to ensure that the value passed to the constructor is correct. This would help ensure that the `vault` variable is initialized correctly and the contract will work as expected.

## Details
##### Description
There is no more functionality to initialize the `vault` variable in the contract. If the value of the variable passed to [constructor](https://github.com/iearn-finance/yearn-vaults/blob/54db126821c4d7aaaf5839be935cecb9b1bf088b/contracts/BaseStrategy.sol#L162) is not correct, then the contract will not work.

##### Recommendation
We suggest adding a parameter check.
