# Transfer without enough token on contract balance.

## Summary
The bug report is about a function called "loop()" in a program called "Leverager". The function is not working properly and is causing two issues. First, it is giving an error message saying "isBorrow parameter = true" because there are no tokens on the balance. Second, it seems to be charging fees twice, once before a "for()" cycle and then again during the cycle. The recommendation is to only transfer the fee if there are enough tokens on the balance in the contract.

## Details
**Description**

Leverager: loop() 
The function will revert with isBorrow parameter = true since it cannot transfer fee without any tokens on balance. Also it seems like it is charging funds twice: before the for() cycle and in it. 

**Recommendation**:

Transfer fee when a contract has enough tokens on balance.
