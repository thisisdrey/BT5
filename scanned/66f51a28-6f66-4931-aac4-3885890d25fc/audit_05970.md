# Deadline check skipped in simulation mode

## Summary
This bug report discusses a medium risk issue in the AtlasVerification.sol code, specifically lines 132-139. The problem is that in simulation mode, the deadline check for userOp and dAppOp is being skipped. This is problematic because a successful simulation could lead to an onchain transaction which would then revert, wasting gas for the bundler. The recommendation is to check against the deadline for simulation mode and update the code accordingly. This issue has been resolved in PR 178 and has been verified by Spearbit.

## Details
## Severity: Medium Risk

## Context
`AtlasVerification.sol#L132-L139`

## Description
Deadline check for `userOp` and `dAppOp` is skipped in simulation mode. This shouldn't be the case as a successful simulation will lead to an on-chain transaction which will then revert, wasting gas for the bundler.

## Recommendation
Check against deadline for simulation mode:

```solidity
if (userOp.deadline != 0 && !isSimulation) {
    return (userOpHash, ValidCallsResult.UserDeadlineReached);
}
```
Should be changed to:

```solidity
if (userOp.deadline != 0) {
    return (userOpHash, ValidCallsResult.UserDeadlineReached);
}
```
For `dAppOp`:

```solidity
if (dAppOp.deadline != 0 && !isSimulation) {
    return (userOpHash, ValidCallsResult.DAppDeadlineReached);
}
```
Should be changed to:

```solidity
if (dAppOp.deadline != 0) {
    return (userOpHash, ValidCallsResult.DAppDeadlineReached);
}
```

## Fastlane
Solved in PR 178.

## Spearbit
Verified.
