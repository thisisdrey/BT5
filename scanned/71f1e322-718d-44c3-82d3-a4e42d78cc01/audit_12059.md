# LeverageMacroBase 's _doCheckValueType() condition looks to be incorrectly reverted 

## Summary
This bug report is about a code review for a specific part of the code called LeverageMacroBase.sol. The issue is related to a function called `_doCheckValueType()` which is used to compare two values. The report suggests that the function needs to be updated to compare the values in the correct order. A recommendation is made to change the conditions in the function to ensure that the first value is compared to the second value, rather than the other way around. The discussion between the parties involved confirms that the issue has been addressed and acknowledged.

## Context
LeverageMacroBase.sol#L236-L247

## Description
Given the usage, say for the `debt >= minDebt` check in:
- LeverageMacroBase.sol#L175-L176
```solidity
_doCheckValueType(cdpInfo.debt, checkParams.expectedDebt);
_doCheckValueType(cdpInfo.coll, checkParams.expectedCollateral);
```
It looks like `_doCheckValueType()` needs to compare the first argument against the second according to the operator type.

## Recommendation
Consider making the conditions reverted:
- LeverageMacroBase.sol#L236-L247

```solidity
/// @dev Assumes that
/// >= you prob use this one
/// <= if you don ' t need >= you go for lte
/// And if you really need eq, it ' s third
function _doCheckValueType(uint256 valueToCheck, CheckValueAndType memory check) internal {
    if (check.operator == Operator.skip) {
        // Early return
        return;
    } else if (check.operator == Operator.gte) {
        - require(check.value >= valueToCheck, "!LeverageMacroReference: gte post check");
        + require(valueToCheck >= check.value, "!LeverageMacroBase: gte post check");
    } else if (check.operator == Operator.lte) {
        - require(check.value <= valueToCheck, "!LeverageMacroReference: let post check");
        + require(valueToCheck <= check.value, "!LeverageMacroBase: lte post check");
    }
}
```
