# When recoverToken in DaiGoldAction contract is called, it does not update the status

## Summary
This bug report is about an issue with the recoverToken function in the TempleDAO auction contract. This function is supposed to move a specific amount of TGLD tokens to another contract and delete the last epoch so that it can be reset and restarted. However, while the tokens are successfully moved, the function does not subtract the correct amount from the nextAuctionGoldAmount. This results in an incorrect amount of tokens being available for the next epoch, causing bidders to be unable to claim their TGLD tokens after the auction ends. The recommended solution is to subtract the nextAuctionGoldAmount when tokens are moved in the recoverToken function. This bug has been fixed in PR 1027 by the TempleDAO team and has been verified by Cyfrin.

## Details
**Description:** recoverToken function moves specific amount of TGLD from the auction contract to another and delete last epoch so that it can be reset and restart.
The tokens get moved but the status nextAuctionGoldAmount is not subtracted, and this gives incorrect amount of TLGD tokens to the next epoch, and as a result, the bidders won't be able to claim their TGLD.

**Impact:** The next auction has incorrect number as auction amount which results in users not being able to claim tokens after the auction ends.

**Recommended Mitigation:** The nextAuctionGoldAmount should be subtracted when tokens are moved in recoverToken function.

**TempleDAO:** Fixed in [PR 1027](https://github.com/TempleDAO/temple/pull/1027)

**Cyfrin:** Verified
