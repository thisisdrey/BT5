# [M02][Fixed] Lack of event emissions after sensitive changes

## Summary
This bug report addresses an issue with functions in the Convexity Protocol. These functions, such as updateParameters, transferFee, and transferCollateral, would not emit an event when the collateral was ETH. This is an issue because it is beneficial for critical functions to trigger events for record keeping and filtering. The bug was fixed in the commit 3adfd9afa6d463869d9e0a78cc7f316ae34eb89e, where events were added to the critical functions. It is recommended to double check all critical functions to make sure they are triggering events properly.

## Details
It is beneficial for critical functions to trigger events for purposes like record keeping and filtering. However, functions like [`updateParameters`](https://github.com/opynfinance/Convexity-Protocol/blob/c34598565cba2bfcf824eb2da63d95c7f5dda4fa/contracts/OptionsContract.sol#L166), [`transferFee`](https://github.com/opynfinance/Convexity-Protocol/blob/c34598565cba2bfcf824eb2da63d95c7f5dda4fa/contracts/OptionsContract.sol#L184), and [`transferCollateral`](https://github.com/opynfinance/Convexity-Protocol/blob/c34598565cba2bfcf824eb2da63d95c7f5dda4fa/contracts/OptionsContract.sol#L636) will not emit an event through inherited ERC20 functions, if the collateral is ETH.  

Consider double checking all critical functions to see if those trigger events properly.


**Update**: *Fixed in the follow-up commit [`3adfd9afa6d463869d9e0a78cc7f316ae34eb89e`](https://github.com/opynfinance/Convexity-Protocol/commit/3adfd9afa6d463869d9e0a78cc7f316ae34eb89e) where events were added to critical functions.*
