# The contract WithPools.sol is not compiled

## Summary
The bug report is about a mismatch in types in the WithPools.sol contract. The function "poolExists" has an error at line 51 where the types for comparison do not match. This is because the "pools" variable is an array of type IStakingLockableExternal, while the "pool" variable is a regular address. The recommendation is to make the "pool" variable of type IStakingLockableExternal. This can be done by using the example code provided in the report. The re-audit of this bug has been skipped.
