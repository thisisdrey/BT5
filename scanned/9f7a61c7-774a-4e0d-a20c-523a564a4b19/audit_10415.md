# Missing reentrancy guard in EmergencyWithdraw / WithdrawEscrow

## Summary
This bug report is about a medium severity issue that has been resolved. It suggests using a modifier called "nonreentrant" to prevent reentrancy attacks when interacting with external contracts.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

By adding the nonreentrant modifier, you can help ensure that the function is not vulnerable to reentrancy attacks, especially when interacting with external contracts.
