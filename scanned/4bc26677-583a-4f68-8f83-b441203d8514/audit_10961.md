# Same-Slot Price Update Error

## Summary
This report discusses a bug in the code related to updating and retrieving prices from an oracle in the GooseFX program. The bug occurs when the price changes within the same slot, causing an error to be thrown and resulting in a temporary denial of service for functions using this code. The bug has been fixed by properly handling the error and removing the vulnerable code.

## Details
## Issue Summary

The issue lies in `oracle_price_history` within `get_and_update_price`, specifically in its interaction with `update_price` and `ensure_recency` in cases where the price changes within the same slot. This results in the `ensure_recency` check failing incorrectly, reverting the function with an `OracleThrottled` error.

## Code Snippet

```rust
pub fn get_and_update_price<'info>(
    price_history: &AccountLoader<'info, OraclePriceHistory>,
    token_oracle: &UncheckedAccount<'info>,
    intended_mint: Pubkey,
    curr_slot: Slot,
) -> Result<Decimal> {
    let mut price_hist = price_history.load_mut()?;
    let previously_recorded_slot = price_hist.latest_price()?.slot;

    if price_hist.mint != intended_mint || token_oracle.key() != price_hist.oracle_address {
        #[cfg(feature = "debug-msg")]
        msg!("Invalid oracle address");
        return err!(SSLV2Error::InvalidOracleAddress);
    }

    // price_hist.update_price(&token_oracle.data.borrow()[..])?;
    #[allow(unused_variables)]
    match price_hist.update_price(&token_oracle.data.borrow()[..]) {
        Err(e) => {
            #[cfg(feature = "debug-msg")]
            msg!(
                "Failed to update oracle {} price: {}",
                token_oracle.key(),
                e
            );
            return Err(e);
        }
        _ => {}
    }
    // [...]
}
```

## Explanation

This occurs as `update_price` accepts a new price when the price changes in the same or different slot, or if a slot changes (even if the price is the same). This results in `ensure_recency` throwing an error when it checks for the recency of the price data and fails for any new price updates up to the minimum elapsed slots, thereby triggering the `OracleThrottled` error.

Since `OracleThrottled` is not whitelisted in the error match case in `internal_swap`, this error is not explicitly handled, reverting the entire function, resulting in a temporary denial of service for all functions fetching prices utilizing this function until minimum elapsed slots finish. A similar situation occurs in `Quote::get_and_update_price` where `get_price` reverts identically.

## Remediation

Properly handle the `OracleThrottled` error in the match case by whitelisting this error and taking specific actions or providing information when this error occurs rather than allowing the function to revert without clear handling.

## Patch

The program no longer contains the vulnerable code.
