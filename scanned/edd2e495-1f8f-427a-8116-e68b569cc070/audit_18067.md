# [M-05] Calls inside loops that may address DoS

## Summary
This bug report is about a vulnerability in the code of a software program. This vulnerability could lead to a denial of service (DoS) attack if it is not addressed. The vulnerability is caused by calls to external contracts inside a loop, which can be user-controlled. This can cause an attack if one of the calls reverts or if execution runs out of gas. The recommended mitigation steps to address this vulnerability are to avoid combining multiple calls in a single transaction, especially when calls are executed as part of a loop, to always assume that external calls can fail, and to implement the contract logic to handle failed calls.
