# IClear Text Logging of Sensitive Credentials (client/export.go)

## Summary
This bug report states that there is a medium severity issue with logging sensitive information in clear text, which could potentially expose private keys. It is acknowledged and has a moderate likelihood of being accessed by malicious actors. The recommendation is to sanitize or hash the information before logging, or use a secure logging method that encrypts or obscures the data. The specific code snippet that needs to be addressed is in the "export.go" file, lines 80 and 82.

## Details
**Severity:** Medium

**Status:** Acknowledged

**Impact:** 

Unsanitized input from sensitive credentials is logged in clear text, potentially exposing private keys.

**Likelihood:** 

Moderate, as malicious actors could access this information with access to the logs.

**Recommendation:** 

Sanitize or hash sensitive information before logging. Consider using a secure logging mechanism that encrypts or obscures sensitive data.

**Code Snippet:**

export.go:80 `keyS := strings.ToUpper(hexutil.Encode(privB)[2:])`
export.go:82 `fmt.Println(keyS)`
