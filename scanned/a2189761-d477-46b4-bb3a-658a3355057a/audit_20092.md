# Incorrect change of the root

## Summary
This bug report is about an issue found in the CumulativeMerkleDrop128, CumulativeMerkleDrop and CumulativeMerkleDrop160 contracts. The problem is that it is possible to update the root with the same value. This can cause problems with the contract's functionality, as it is not intended to be able to update the root with the same value.

The recommendation to fix this issue is to add a check into the code, so that the root cannot be updated with the same value. This check should be added to the code for each of the affected contracts. The code to add this check is provided in the bug report. 

Adding this check will help to prevent the root from being updated with the same value, and ensure that the contract functions as intended.

## Details
##### Description
In the following function it is possible to update root with the same value:
https://github.com/1inch/cumulative-merkle-drop/tree/96fb63d0cbfea73603e7500961c71e8ab1fb8c10/contracts/CumulativeMerkleDrop128.sol#L26
https://github.com/1inch/cumulative-merkle-drop/tree/96fb63d0cbfea73603e7500961c71e8ab1fb8c10/contracts/CumulativeMerkleDrop.sol#L27
https://github.com/1inch/cumulative-merkle-drop/tree/96fb63d0cbfea73603e7500961c71e8ab1fb8c10/contracts/CumulativeMerkleDrop160.sol#L26
##### Recommendation
We recommend to add the following check:
```solidity=
require(merkleRoot_ != merkleRoot, "CMD: Same root");
```
