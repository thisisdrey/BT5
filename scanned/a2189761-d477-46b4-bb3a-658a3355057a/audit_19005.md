# [C01] Cannot dispute invalid reward

## Summary
This bug report describes an issue with the `OptimisticRewardBase` contract when disputing a reward request. Specifically, when the contract triggers a proposal on the `SkinnyOptimisticOracle` and then disputes that proposal, the proposal sets the expiration time as an offset from the current dispute time while the dispute specifies the expiration as an offset from the time of the original reward request. This discrepancy will prevent the oracle from identifying the proposal to be disputed, leading to valid disputes not being processed and invalid reward requests being accepted.

The report suggests updating the dispute invocation to correctly specify the proposal to be disputed. This issue has now been fixed as of commit `9e15557` in pull request 3690.

## Details
When disputing a reward request, the `OptimisticRewardBase` contract first [triggers a proposal](https://github.com/UMAprotocol/protocol/blob/0c4cea3c3d5e48da6f8984b8ba3afdfea4ce47cc/packages/core/contracts/financial-templates/optimistic-rewarder/OptimisticRewarderBase.sol#L209) on the `SkinnyOptimisticOracle` and then [disputes that proposal](https://github.com/UMAprotocol/protocol/blob/0c4cea3c3d5e48da6f8984b8ba3afdfea4ce47cc/packages/core/contracts/financial-templates/optimistic-rewarder/OptimisticRewarderBase.sol#L253). However, the proposal [sets the expiration time](https://github.com/UMAprotocol/protocol/blob/0c4cea3c3d5e48da6f8984b8ba3afdfea4ce47cc/packages/core/contracts/oracle/implementation/SkinnyOptimisticOracle.sol#L325) as an offset from the current (dispute) time, while the dispute [specifies the expiration](https://github.com/UMAprotocol/protocol/blob/0c4cea3c3d5e48da6f8984b8ba3afdfea4ce47cc/packages/core/contracts/financial-templates/optimistic-rewarder/OptimisticRewarderBase.sol#L172) as an offset from the time of the original reward request. In most cases, this discrepancy will prevent the oracle from identifying the proposal to be disputed, which means valid disputes will not be processed and invalid reward requests will be accepted.


Consider updating the dispute invocation to correctly specify the proposal to be disputed.


**Update:** *Fixed as of commit [`9e15557`](https://github.com/UMAprotocol/protocol/pull/3690/commits/9e155574458f24d653bf933ee313b94a79a788d7) in [PR3690](https://github.com/UMAprotocol/protocol/pull/3690).*
