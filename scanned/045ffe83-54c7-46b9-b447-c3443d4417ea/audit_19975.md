# [M-11] Owner can steal all Basket funds during auction

## Summary
This bug report is regarding a vulnerability in the Factory contract of a DeFi protocol. The vulnerability allows the owner of the Factory contract to modify the values of 'auctionMultiplier' and 'auctionDecrement' at any time. This manipulation can be used to set 'tokensNeeded' to 0 or close to 0, allowing the owner to buy the underlying tokens from a Basket without transferring any pending tokens. The bug can be found in the 'Auction.sol' file on Github, and was identified using an editor. To mitigate this vulnerability, the Factory contract should consider adding a Timelock to these Factory functions, or a way to not modify them if an auction is ongoing.

## Vulnerability details

## Impact
The owner of Factory contract can modify the values of `auctionMultiplier` and `auctionDecrement` at any time.
During an auction, these values are used to calculate `newRatio` and thereby `tokensNeeded`: specifically, it's easy to set the factory parameters so that `tokensNeeded = 0` (or close to zero) for every token.
This way the owner can participate at an auction, change the parameters, and get the underlying tokens from a Basket without transferring any pending tokens.

## Recommended Mitigation Steps
Consider adding a Timelock to these Factory functions. Otherwise a way to not modify them if an auction is ongoing (maybe Auction saves the values it reads when `startAuction` is called).
