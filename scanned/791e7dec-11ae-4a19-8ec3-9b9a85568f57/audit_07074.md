# IRMSynth 's targetQuote assumes reference asset has 18 decimals

## Summary
This bug report is about an issue with the IRMSynth contract. The targetQuote parameter is set to a fixed value of 1e18, which may cause problems when compared to the output of the oracle.getQuote function. The recommendation is to adjust the targetQuote based on the reference asset's decimals. A fix has been implemented in commit affd8844. Another suggestion is to simplify the code by making the quoteAmount a constant instead of an arbitrary parameter. Additionally, there is no check on the input parameter for targetQuote, which could potentially cause problems. The team has decided to keep the current setup to avoid integration issues, but will add documentation to ensure proper setup of targetQuote in the future.

## Details
## Severity: Medium Risk

## Context
`IRMSynth.sol#L9`

## Description
The `IRMSynth.targetQuote` parameter is set to `1e18`. It is compared against the output of `oracle.getQuote(1e18, synth, referenceAsset)` that returns a reference asset amount which will be in reference asset decimals.

## Recommendation
Consider shifting the `targetQuote` based on the `referenceAsset.decimals()` in the constructor.

## Euler
Fixed in commit `affd8844`.

## Spearbit
The provided commit mitigates the issue. Spearbit suggests considering the following changes:

- `ESynth` that inherits from `ERC20Collateral` does not allow having a custom value for decimals that will be by default equal to 18. If there's not a specific reason to have it as an arbitrary parameter, Euler should consider simplifying the code and declaring `quoteAmount` as a constant equal to `1e18`.
- There is no sanity check on the `targetQuote_` input parameter. Depending on the value passed, the IRM could for example always increase or decrease the rate no matter the quote value in `_computeRate`. Consider adding sanity checks or documenting it.

## Euler
Will keep as it is now to avoid possible integration issues with different synth tokens in the future. Will add to the docs that `targetQuote` should be properly set up.
