# CrossPool staker can get fewer reward tokens in case if RewardVault has less balance than accumulated reward tokens

## Summary
The RewardVault contract is not accurately transferring tokens to the Stacker and CrossPool contracts. This is due to a bug in the code that calculates the amount of tokens to transfer. The bug is located in two snippets of code, one that checks the block number and total pending amount, and one that transfers the tokens. The bug causes the transfer to only send a portion of the intended amount. This needs to be fixed in order for the contracts to function properly.

## Details
**Description**

RewardVault is not returning the actual amount of tokens transferred to Stacker and
CrossPool contract marks that all reward tokens transferred.


Snippet 1:
```solidity
if(block.number >= timeLock && totalPending > 0) {
user.accClaim = 0;
rewardVault.safeRewardTransfer(msg.sender, totalPending);
}
```
Snippet 2:
```solidity
function safeRewardTransfer(
address to,
uint256 amount
) public onlyOwner {
uint256 rewardTokenBalance = rewardToken.balanceOf(address(this));
uint256 transferableAmount = amount > rewardTokenBalance
? rewardTokenBalance
: amount;
rewardToken.safeTransfer(to, transferableAmount);
}
```
