## Summary
This report highlights a vulnerability in the `RefuelAction` and `TransferAction` contracts that could lead to financial losses for Otim executors. The contracts use low-level calls without gas limitations for native token transfers, which allows malicious actors to front run transactions and consume all available gas before reverting. This results in the executor bearing the full gas cost without receiving any compensation. The report recommends implementing a gas limit for external calls in the short term and redesigning the fee mechanism in the long term to prevent gas griefing attacks.

## Details
## Vulnerability Report: Gas Griefing Attacks

**Difficulty:** Low

**Type:** Timing

## Description

The `RefuelAction` and `TransferAction` contracts are vulnerable to gas griefing attacks due to the way they handle native token transfers to the target address, which could lead to financial losses for Otim executors. 

When native tokens are transferred to target addresses, these contracts use low-level calls without gas limitations, and fees are charged only upon successful execution.

```solidity
80    (bool success, bytes memory result) = refuel.target.call{value: refuelAmount}("");
81    // slither-disable-end reentrancy-events
82    // slither-disable-end missing-zero-check
83    // slither-disable-end arbitrary-send-eth
84
85    // if the transfer fails, revert
86    if (!success) {
87        revert NativeTransferFailed(result);
88    }
89
90    // charge the fee
91    chargeFee(startGas - gasleft(), refuel.fee);
```

Since the fee is charged only after a successful call, a malicious target can front run transfer transactions with malicious ones that consume gas and revert, preventing the executor from receiving compensation for their expended resources.

He front runs this transaction with an EIP-7702 transaction that transforms his EOA into a contract by pointing to a malicious delegate contract. This contract implements a receive function that executes an unbounded loop consuming all available gas before reverting. 

When the Otim executorŌĆÖs transaction executes immediately after, it attempts to transfer ETH to what is now BobŌĆÖs contract. The transaction reverts after consuming significant gas but before reaching the fee charging logic. As a result, the executor bears the full gas cost of the failed transaction without receiving any compensation, resulting in financial loss.

## Recommendations

- **Short term:** Implement a gas limit for external calls to prevent complex executions in recipient contracts and protect against gas griefing attacks.
- **Long term:** Redesign the fee mechanism to charge users regardless of transfer success, potentially by implementing a two-phase execution process where fees are charged before attempting risky external calls, or by implementing a prepayment system for gas costs.
