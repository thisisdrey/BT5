# Possible excess of gas limit

## Summary
This bug report is about a function in a Compound Protocol contract. The bug report suggests that a check should be added to the start of the loop in order to make sure there is enough gas left. The recommended check is to add an if statement to the loop that will return if there is less than 40,000 gas left. This check is necessary to make sure that the loop has enough gas to complete the task without running out. By adding this check, the loop will be able to complete its task without running out of gas, ensuring that the function will complete successfully.

## Details
##### Description
In following function it is worth to add check of how much gas left.
https://github.com/CreamFi/compound-protocol/blob/23a4ae93adc70334553f5a83429a4e967c1eefaa/contracts/Comptroller.sol#L1266

##### Recommendation
We recommend add following check in head of `for` loop:
```solidity=
if (gasleft < 40000) { return;}
```
