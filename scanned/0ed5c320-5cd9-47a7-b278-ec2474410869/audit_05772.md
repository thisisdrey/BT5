# InvestmentManager::_deleteOffer should subtract escrowAmount instead of $.offers[id].amount

## Summary
This bug report describes a problem with the InvestmentManager contract where the accounting is incorrect when an investor withdraws their offer after a delay. This results in the excess amount being locked in the contract instead of being returned to the investor. The recommended solution is to subtract the escrowed amount from the current contract balance. This issue has been fixed in a pull request (PR) and by the Spearbit team.

## Details
## High Risk Vulnerability Report

**Severity:** High Risk  
**Context:** InvestmentManager.sol#L360  

## Description
When an investor submits an offer to DealManager, the payment is initially escrowed, waiting for the offer to be reviewed by an Originator. If, after a delay of `offerEscrowPeriod` (initially 5 days), the offer is not reviewed, the investor can call `withdrawOffer` to get his funds back. However, the accounting is incorrect in that case, since the amount is deducted from the escrowed funds, whereas `escrowAmount` (amount + fees) has been escrowed. The surplus amount (equivalent to the fee) stays locked in the contract. 

`hasSufficientAvailableFunds` would deduct `escrowedAmount` from the current contract balance to determine the funds which can be distributed or withdrawn by admin.

## Recommendation
Subtract `$.offers[id].escrowAmount` in InvestmentManager.sol:

```solidity
function _deleteOffer(uint256 id) private {
    IMStorage storage $ = _imStorage();
    - $.escrowedAmount -= $.offers[id].amount;
    + $.escrowedAmount -= $.offers[id].escrowAmount;
    assert($.offerIds.remove(id));
    delete $.offers[id];
}
```
