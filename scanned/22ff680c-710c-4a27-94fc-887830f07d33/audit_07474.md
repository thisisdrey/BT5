# Missing Access Control

## Summary
This bug report is about a function called sfnProcessTradeDispute in the CEGA strategies code. This function does not have proper access control, which means that any user can use it to process disputes for SFN vaults. This function also has the ability to override prices, making it vulnerable to abuse. This oversight could potentially lead to price manipulation and theft of funds. The suggested solution is to add a modifier called onlyCegaAdmin to restrict the use of this function to users with the CEGA_ADMIN role. This bug has been fixed in the latest version of the code.

## Details
## sfnProcessTradeDispute Function Access Control Vulnerability

The `sfnProcessTradeDispute` function lacks proper access control, allowing any user to process disputes for SFN vaults. This function's ability to override prices makes it vulnerable to abuse; therefore, its use should be restricted to users with the `CEGA_ADMIN` role.

This oversight potentially leads to price manipulation by processing a dispute with fabricated prices (via the `newPrices` argument). When processing a dispute, it is possible to override price observation for a certain timestamp. Consequently, the ability to call this function may lead to the theft of funds. For instance, by setting the price at trade expiry to the knock-out price, it is possible to maximize yield for traders.

## Code Snippet

```solidity
function sfnProcessTradeDispute(
    address vaultAddress,
    uint40 timestamp,
    bool accept,
    uint128[] calldata newPrices
) external {
    CegaGlobalStorage storage cgs = getStorage();
    SFNVaultLogic.processDispute(
        cgs,
        addressManager,
        vaultAddress,
        timestamp,
        accept,
        newPrices
    );
}
```

## Remediation

Add the `onlyCegaAdmin` modifier to `sfnProcessTradeDispute`.

## Patch

Fixed in `567e65f`.
