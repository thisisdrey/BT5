# Unchecked return

## Summary
The bug report is about a medium severity issue that has already been resolved. The problem was found in the dpnm_sc.sol file in two functions, buyTurnoverWithGWT and buyEarnLimitWithGWT. The issue was that the return value from the burning operation of gwt was not being validated. The same issue was found in two other functions, buydPNM and selldPNM, where the return value from the minting operation of gwt was also not being validated. The recommendation is to wrap these operations in a require statement to ensure they are properly validated. The bug has been fixed in the latest version of the code.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

dpnm_sc.sol - In buyTurnoverWithGWT(uint) & buyEarnLimitWithGWT(uint) , the return value from the burning operation of gwt is not validated gwt.burn(msg.sender, gwtCost).
In buydPNM(uint) & selldPNM(uint) , the return value from the minting operation of gwt is not validated gwt.mint(msg.sender, gwtCost).

**Recommendation** 

Wrap in require statement: 

```solidity
require(gwt.burn(msg.sender, gwtCost), ...).
require(gwt.mint(msg.sender, gwtCost), ...).
```
**Fix** - Issue fixed in 60761c1.
