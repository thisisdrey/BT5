# Unlimited Minting Of Tokens

## Summary
The report discusses a bug in a function called mint in a smart contract called VariableSupplyERC20Token. This function is used to create new ERC20 tokens, but it has a flaw that allows an unlimited amount of tokens to be created. The function keeps track of the maximum amount of tokens that can be created through a variable called mintableSupply. However, as the value of mintableSupply decreases, the function can still create an unlimited amount of tokens. 

To fix this issue, the report suggests verifying that the total amount of tokens being created does not exceed the mintableSupply. This can be done by adding a check in the function to ensure that the sum of the current amount being minted and the total amount minted until now does not surpass the mintableSupply. This fix has been implemented in a commit called d0716cb.

## Details
## VariableSupplyERC20Token Minting Issue

`VariableSupplyERC20Token::mint` limits the amount of ERC20 tokens that may be minted. This is tracked via the `mintableSupply` value, which determines the maximum amount of tokens mintable. 

To validate that the amount minted does not exceed the max supply of tokens, `mint` reduces `mintableSupply` by the amount being minted. This method is faulty as eventually, as the value of `mintableSupply` decreases and reaches zero, `mint` may effectively mint an unlimited amount of ERC20 tokens.

## Solidity Code

```solidity
function mint(address account, uint256 amount) public onlyAdmin {
    require(account != address(0), "INVALID_ADDRESS");

    // If we're using maxSupply, we need to make sure we respect it
    // mintableSupply = 0 means mint at will
    if(mintableSupply > 0) {
        require(amount <= mintableSupply, "INVALID_AMOUNT");
        // We need to reduce the amount only if we're using the limit, if not, just leave it be
        mintableSupply -= amount;
    }
    _mint(account, amount);
}
```

## Remediation

Verify that the sum of the current amount being minted and the total amount minted until now does not surpass `mintableSupply`.

### Updated Solidity Code

```solidity
function mint(address account, uint256 amount) public onlyAdmin {
    require(account != address(0), "INVALID_ADDRESS");

    // If we're using maxSupply, we need to make sure we respect it
    // mintableSupply = 0 means mint at will
    if(mintableSupply > 0) {
        require(totalSupply + amount <= mintableSupply, "INVALID_AMOUNT");
    }
    _mint(account, amount);
}
```

## Patch

Fixed in commit `d0716cb` by implementing a check to ensure that the sum of `totalSupply` and the newly minting amount does not exceed the `mintableSupply`.
