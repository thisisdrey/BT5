# [M-03] Balance going below BASE_WEARABLE_UNIT

## Summary
The bug report states that there is a problem with the transfer and selling of wearables in a system. It is categorized as a medium severity issue, meaning it is not critical but still important to address. The impact is small, but it can be valuable if there is a low curveAdjustmentFactor or high supply. The likelihood of this bug occurring is also medium, as it only requires sending a number that is not strictly divisible by 0.001 ETH.

The issue is that the function `transferWearables()` only allows for amounts above 0.001 ETH to be sent, but it still allows for amounts like 0.0015 ETH to be sent when the user has 0.002 ETH. This means that 0.0005 ETH will remain, which is below the minimum requirement of 0.001 ETH. The same problem occurs when selling wearables.

To address this issue, the report recommends checking that the user has a balance above the minimum requirement after a transfer or sell operation. This will prevent users from being stuck with a balance below the minimum requirement and having to buy more wearables to pass the limit.

## Details
**Severity**

**Impact:** Medium, some balance cannot be sold and transferred, it is small but potentially valuable enough in case of low curveAdjustmentFactor or high supply

**Likelihood:** Medium, just required sending any number not strictly devisable by 0.001 ETH

**Description**

`transferWearables()` only requires sending amount above `BASE_WEARABLE_UNIT=0.001 ether`.
But allows sending e.g. 0.0015 ether when the user has 0.002 ether.
0.0005 ether will remain in this case, below `BASE_WEARABLE_UNIT`.

The same thing for selling wearables.

```solidity
    function sellPrivateWearables(bytes32 wearablesSubject, uint256 amount, bytes calldata signature)
        external
        payable
    {
            // Check if amount is greater than base unit
            if (amount < BASE_WEARABLE_UNIT) revert InsufficientBaseUnit();
            ...
    }

    function transferWearables(bytes32 wearablesSubject, address from, address to, uint256 amount) external {
        ...

        // Check if amount is greater than base unit
        if (amount < BASE_WEARABLE_UNIT) revert InsufficientBaseUnit();
        ...
    }
```

This new amount will not pass the minimum requirements in `transferWearables()` and sell operations. The only option for the user is to buy more wearables to pass the limit.

**Recommendations**

Consider checking that after transfer and a sell operation a user has the balance above `BASE_WEARABLE_UNIT`
