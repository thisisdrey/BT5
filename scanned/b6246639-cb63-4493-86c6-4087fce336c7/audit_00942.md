# M-23: The protocol doesnt support weird tokens like Tether Gold (that return false on every operation)

## Summary
This bug report is about an issue with the protocol's token transfers. The team has acknowledged the problem but will not fix it at this time. The issue was found by a user named anonymousjoe and it is caused by tokens like Tether gold, which always cause the transfer to fail. This means that these tokens are not supported by the protocol. There are no internal or external pre-conditions for this issue and there is no known attack path. The impact of this bug is that the protocol claims to support tokens that it actually cannot support. There is no proof of concept provided and no suggested mitigation for this issue.

### Summary

The protocol uses the `TransferHelper.safeTransferFrom(token, _currentTokenRequester, msg.sender, uint256(change));` for its token transfers. But this does not work if the token returns false for all the operations.

### Root Cause

Tokens like Tether gold will cause the transfer to always revert. Thus such tokens are not supported by the protocol.
```solidity
            if (change > 0) {
                // Pull tokens from the original caller and send to MAKER_FACET
                TransferHelper.safeTransferFrom(token, _currentTokenRequester, msg.sender, uint256(change));
            } else if (change < 0) {
                // Send tokens from our balance to MAKER_FACET (for fee collection, etc.)
                TransferHelper.safeTransfer(token, msg.sender, uint256(-change));
            }

            // After primary transfer, sweep any dust the contract may still hold for this token
            uint256 residual = IERC20(token).balanceOf(address(this));
            if (residual > 0) {
                TransferHelper.safeTransfer(token, msg.sender, residual);
            }
```
