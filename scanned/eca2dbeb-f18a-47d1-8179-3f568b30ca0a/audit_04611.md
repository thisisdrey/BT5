# Blind auctions cannot be cancelled before auction starts 

## Summary
This bug report discusses an issue with the KimNFTMarketplace protocol where the maker is not able to cancel a blind auction before it starts. The current code does not allow for this, but the recommendation is to change it so that blind auctions can be cancelled before they start. This issue has been fixed in the KIM Exchange PR 8. The new implementation ensures that the blind auction can be cancelled before it starts by checking for bids before the auction begins.

## Details
## KimNFTMarketplace.sol Vulnerability Report

## Context

Refer to lines 160-162 of `KimNFTMarketplace.sol` for the following context.

## Description

In the protocol, the maker is allowed to cancel the auction before it starts:

```solidity
if (listing.start > 0 && block.timestamp >= listing.start && block.timestamp <= listing.end) 
    revert AuctionNotFinished();
```

However, for blind auctions, due to the incorrect check, the maker is not allowed to cancel the auction before it starts:

```solidity
if (listing.listingType == ListingType.BLIND) {
    if (block.timestamp <= (listing.end + BLIND_BID_REVEAL_WINDOW)) 
        revert AuctionNotFinished();
}
```

## Recommendation

It is recommended to change the code as follows to allow blind auctions to be cancelled before they start:

```solidity
- if (block.timestamp <= (listing.end + BLIND_BID_REVEAL_WINDOW)) 
    revert AuctionNotFinished();
+ if (block.timestamp >= listing.start && block.timestamp <= (listing.end + BLIND_BID_REVEAL_WINDOW)) 
    revert AuctionNotFinished();
```

## KIM Exchange

This issue has been fixed in PR 8.

## Cantina Managed

The new implementation is as follows. Before the blind auction starts, `blindBidsForListing[listingId].length == 0`, so it doesn't go into that branch, and the outer check allows the blind auction to be canceled before it starts:

```solidity
if (listing.listingType != ListingType.BUY_IT_NOW && listing.listingType != ListingType.BUY_IT_NOW_WITH_BIDS) {
    // Ensure in the middle of an auction we cannot cancel the listing
    if (block.timestamp >= listing.start && block.timestamp <= listing.end) 
        revert AuctionNotFinished();
    
    // When blind auction enforce the bid reveal window to finish if there are bids present
    if (listing.listingType == ListingType.BLIND && blindBidsForListing[listingId].length > 0) {
        if (block.timestamp >= listing.start && block.timestamp <= (listing.end + BLIND_BID_REVEAL_WINDOW))
            revert AuctionNotFinished();
    }
}
```
