# No change returned in case of extra fee value

## Summary
This bug report discusses an issue with the deposit and burning of tokens in the ChainportMainBridge.sol and ChainportSideBridge.sol contracts. The problem occurs when a user deposits more tokens than necessary, as the contracts do not return any change from the excess amount. This can lead to users leaving more funds than needed, which poses a risk. The recommendation is to either provide clear documentation and warnings for users about this risk, or to implement a strict check for the necessary amount of fee. The Chainport team has acknowledged this issue and has verified it as intended behavior to avoid rounding errors. However, they have ensured that the frontend and any integration partners will receive exact values to avoid this issue.

## Details
**Severity**: Low

**Status**: Acknowledged 

**Description**

ChainportMainBridge.sol: _depositTokens()
ChainportSideBridge.sol: burnTokens()

No change returned from the msg.value in case if deposited more than the needed. Since
the check is not strict, it allows such operation and the user may leave more funds than
necessary.

**Recommendation**:

Verify the approach and provide clear documentation and warning for the user about such
risk OR provide the strict equality against the necessary amount of fee OR return the change
back to the user

**Post audit.**

The Chainport team verified it as intended behavior - as a measure of avoiding the risk of
rounding errors issues (at least on one side) between frontend and backend. Though, the
team ensured that the frontend of Chainport makes sure the values are exact, and any
integration partner will also receive the exact values to send in msg.value from the Chainport
api endpoints as per the docs.
