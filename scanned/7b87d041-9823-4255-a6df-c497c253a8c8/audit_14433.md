# Data Structure Inconsistency

## Summary
The removeBlacklist function in the MultiRewardsBasePoolV3.sol contract has a flaw where it does not remove a blacklisted address from the blacklistAddresses array. This can cause two security issues: an oversized blacklist may cause a denial of service and duplicate entries in the array can lead to an integer overflow. The suggested fix is to remove the blacklistAddresses array and only maintain the blacklistSum variable. This bug has been fixed in two patches.

## Details
## RemoveBlacklist Function Security Flaws

The `removeBlacklist` function removes the blacklisted address from the `inBlacklist` map but fails to remove the same address from the `blacklistAddresses` array.

```solidity
// contracts/multiRewards/gamefi/base/MultiRewardsBasePoolV3.sol
function removeBlacklist(address _address) external onlyAdmin {
    require(
        inBlacklist[_address],
        "MultiRewardsBasePoolV3.removeBlacklist: address not in blacklist, please try to add first"
    );
    inBlacklist[_address] = false;
    /* abridged */
}
```

## Security Implications

This implementation flaw has two security implications:
1. An oversized blacklist may lead to a denial of service condition in `adjustedTotalSupply`.
2. It allows duplicate entries to be present in the array. Redundant entries in the array may violate the invariant that ensures the sum of all blacklisted shares is less than or equal to the total supply. If this invariant is broken, `adjustedTotalSupply` is bound to revert due to integer overflow in the last subtraction.

```solidity
// contracts/multiRewards/gamefi/base/MultiRewardsBasePoolV3.sol
function adjustedTotalSupply() public view returns (uint256) {
    uint256 blacklistSum = 0;
    for (uint256 i = 0; i < blacklistAddresses.length; i++) {
        blacklistSum = blacklistSum + blacklistAmount[blacklistAddresses[i]];
    }
    return super.totalSupply() - blacklistSum;
}
```

## Remediation

Remove the `blacklistAddresses` array and maintain only the `blacklistSum` in a separate member variable.

## Patch

Fixed in commits `8c37830` and `15e0843`.
