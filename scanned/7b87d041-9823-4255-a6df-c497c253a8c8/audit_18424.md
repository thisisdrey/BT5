# Incorrect interface of vault.

## Summary
The bug report is about a problem with two functions in a contract called OwnerProxy_V1.sol. The functions are emergencyWithdraw() and settleStrike_MM(). The emergencyWithdraw() function has a parameter for a token address, but the owner proxy does not pass it. This is causing an issue because the contract is unable to find a storage variable or function called "COLAT_ADDRESS" in the vault contracts. Similarly, it cannot find a variable called "PRICE_FEED". The recommendation is to use the correct interface of the vault contracts to fix this issue.
