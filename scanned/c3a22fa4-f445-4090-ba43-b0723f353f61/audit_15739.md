# Any party can reject receiving funds to prevent escrow from settling/claiming

## Summary
A bug report was submitted regarding a problem with settling an escrow. The system was using a "Push" pattern to handle funds transferring when settling, which meant that if any transfer failed, the escrow couldn't be settled. This allowed any party (buyer, seller, or marketplace) to reject receiving funds (native token or ERC777) and prevent the escrow from settling. 

The bug was observed in a dispute where the arbitrator had decided to split the funds 80% to the buyer, 10% to the seller, and 10% to others. The seller was unhappy with the decision and, when the arbitrator initiated the claim based on his split, the sendEscrowShare function was called to send each party their share. When the function tried to send the split to the seller (who was a contract), the seller simply reverted the payment, causing success to be false and the whole claim to fail. 

The bug was resolved with the recommendation to consider using a "Pull" over "Push" pattern when dealing with settling escrow. It was also recommended to not fail the claim function if a party is trying to dos the receive payment, and to store the failed payment amounts in a mapping variable so that users can use another function to extract them.
