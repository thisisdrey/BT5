# Code is not corresponding comments.

## Summary
This bug report highlights an issue with the lockduration_ variable in the code. The report states that in line 264, it is mentioned that lockduration_ is passed as a number of days and assigned to the lockDuration storage variable. However, in lines 357 and 375, lockDuration is multiplied by 3600, which represents 1 hour. This means that lockDuration is actually representing hours instead of days, which is different from what is stated in the comments. The recommendation given is to either correct the comments to reflect that lockduration_ represents hours or change the multiplication factor to 1 day. This will help ensure consistency and accuracy in the code.
