# Possible for New `main_auth_descriptor` to Be Missing the `A` or `T` Flags

## Summary
A bug has been reported in the `rell/src/lib/ft4/core/accounts/module.rell` file. The bug occurs when creating a new account, where the main auth descriptor does not properly validate the required flags. This could potentially leave the account in a state where no auth descriptors can access the account or perform transfers, resulting in locked funds. The recommendation is to update the `update_main_auth_descriptor()` function to validate the new auth descriptor's flags. The client has marked this bug as "Fixed" and it has been addressed in a specific code change.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `415b7916d5b6b80edc328d8743985b04f56a38c3`.

**File(s) affected:**`rell/src/lib/ft4/core/accounts/module.rell`

**Description:** The most privileged auth descriptor for an account is the main auth descriptor. On account creation, the main auth descriptor requires both the Account flag and the Transfer flag (`A` and `T` flags respectively). This gives it access to all account and transfer operations.

Additionally, the main auth descriptor is the only auth descriptor that can call `update_main_auth_descriptor()`. However, the new main auth descriptor is not validated to have the `A` and `T` flags. This could leave the account in a state where no auth descriptors are capable of account or transfer operations, potentially locking all funds held by the account.

**Recommendation:** Update `update_main_auth_descriptor()` to validate that the new auth descriptor has the required flags.
