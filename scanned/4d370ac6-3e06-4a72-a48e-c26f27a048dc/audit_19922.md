# [M-03] Previously created markets can be overwritten

## Summary
This bug report is about the createMarket function in the MarketPlace.sol contract. It allows accidental overwriting of previously created markets for the same combination of underlying and maturity timestamp (u, m) because there is no zero-address check to see if a previously created market exists for that combination. The bug was discovered through manual analysis and the proof of concept is provided in the report. The recommended mitigation step is to add a zero-address check for markets[u][m] in createMarket before writing to it. This will ensure that the contract will not allow the accidental overwrite of previously created markets.
