# Lack of Source Validation

## Summary
This bug report discusses an issue with the authorization process in the authorize_invoke instruction. The transfer instructions within this instruction do not check the source account, allowing a user to supply any account as long as it is a signer in the wallet program's invoke instruction. This could potentially allow a user to withdraw funds from the session's authority account to a valid destination account. While this may not be a critical issue, it goes against the assumption that withdrawal instructions should only access and withdraw from the wallet_authority account, not the session_authority account. The lack of source account validation could result in misuse of funds. The solution is to explicitly check and validate the source accounts used in transfer instructions to ensure they are legitimate and authorized. This issue has been resolved in the latest patches.

## Details
## Authorization Oversight in `authorize_invoke` Instruction

There is an authorization oversight in the `authorize_invoke` instruction. The `transfer` instructions in `authorize_invoke` do not check the source account. Therefore, a user may supply any account as the source, as long as it is a signer in the wallet program’s `invoke` instruction. The signer in the `invoke` instruction is checked to be the authority of the session.

```rust
> _ wallet-v1/src/utils/authorizer.rs
pub fn require_authorizer_invoked<'info>(
    [...]
) -> Result<()> {
    [...]
    for i in (0..usize::MAX).filter(|&i| i != current) {
        [...]
        let authorizer_signer = instruction
            .accounts
            .get(AUTHORIZER_SIGNER_ACCOUNT_IDX)
            .ok_or(WalletError::InvalidSignerAccount)?
            .pubkey;
        [...]
    }
    err!(WalletError::MissingAuthorizerIx)
}
```

As a result, a user utilizing a wallet program may withdraw funds from the session’s authority account to a valid destination account. This is not a critical issue, since the ability to sign for the account implies that the program has authorization to also withdraw funds from it. However, the wallet program is designed with the assumption that withdrawal instructions should only access and withdraw from the `wallet_authority` account, not from the `session_authority`. The lack of source account validation breaches this assumption, resulting in possible misuse of the funds.

```rust
> _ wallet-v1/src/instructions/invoke.rs
pub fn invoke(ctx: Context<Invoke>, instruction_data: Vec<u8>) -> Result<()> {
    [...]
    invoke_signed(
        &instruction,
        remaining,
        &[&[
            WalletAuthorityInfo::AUTHORITY_SEED,
            &ctx.accounts.wallet_info.uca,
            &[ctx.accounts.wallet_authority_info.wallet_index],
            &[ctx.accounts.wallet_authority_info.authority_bump],
        ]],
    )?;
    Ok(())
}
```

## Remediation
Explicitly check to validate that the source accounts used in transfer instructions are legitimate and authorized.

### Patch
Resolved in `f651ad8a` and `49f7a1a`.
