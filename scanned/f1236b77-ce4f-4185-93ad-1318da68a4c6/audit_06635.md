# Enforce EpochLimit value minimum size of 3 on chain creation

## Summary
This bug report discusses an issue with the blockchain code, specifically in the core/blockchain.go file. The problem occurs when the chain deletes epochs based on a user-configured value called EpochLimit. If this value is set to anything lower than 3, it can accidentally delete important data. The report recommends enforcing a minimum value of 3 for EpochLimit when creating a chain, but still allowing the option for an unlimited value. This bug is considered a medium risk and should be addressed by the developers.

## Details
## Vulnerability Report

**Severity:** Medium Risk  
**Context:** `core/blockchain.go#L1438-L1444`  
**Description:** Each checkpoint block, the chain goes through and deletes epochs based on the configured `EpochLimit`. Any value lower than 3 could delete required data.  
**Recommendation:** Enforce the minimum `EpochLimit` value of 3 on chain creation, while still allowing the value of 0 for unlimited `EpochLimit`.
