# EscapeUnsafeCharacters Requires More Strictness

## Summary
The report highlights a bug in an application's escapeUnsafeCharacters method, which is meant to filter out special characters. However, the method is not properly defined and does not effectively escape all special characters, leaving the application vulnerable to security threats. This bug is also present in the injectedJs constant, which could allow malicious inputs to escape from the context. Additionally, the use of replaceAll() in the method may cause issues when decoding inputs with special characters or specific encoding. The report recommends implementing a replacement for the single quote character and using a regular expression in the replaceAll() method to properly filter encoded special characters. The bug has been fixed in version 4702294 by updating the function name to escapeBundleCharacters(). It is important to carefully consider and handle potential issues when working with input data that may require decoding.

## Security Vulnerability Report

## Overview

The application has defined the `escapeUnsafeCharacters` method intending to filter out special characters. However, the method is insufficiently defined and does not safely escape all special characters, rendering it ineffective. As a result, the application may still be vulnerable to security threats stemming from unescaped special characters.

## Code Snippet

```typescript
function escapeUnsafeCharacters(code: string) {
    return (
        code
        // when the string gets evaluated we want to preserve characters like `\n`
        // instead of literally translating to newlines.
        .replaceAll(`\\`, `\\\\`)
        // escape all backticks
        .replaceAll('`', '\\`')
    )
}
```

This function is also used inside the `injectedJs` constant, but a malicious input could escape from the context with a simple sequence such as `';`.

```typescript
const getPostToWebView = (webviewRef: React.RefObject<WebView>) => (payload: WalletAdapterResponse) => {
    const payloadStr = JSON.stringify(payload);
    const injectedJs = escapeUnsafeCharacters(`
    {
        const embEvt = new Event('${payload.uuid}');
        embEvt.data = '${payloadStr}';
        window.dispatchEvent(embEvt);
    }
    true;
    `);
    return webviewRef.current?.injectJavaScript(injectedJs);
}
```

Furthermore, the use of `replaceAll()` may lead to issues when decoding inputs containing special characters or specific encoding. In such cases, `replaceAll()` may not recognize the input as a special character, leading to unexpected behavior and errors.

## The Easy Company Audit 04 | Vulnerabilities

### Remediation

Implement a replacement for the single quote character in addition to a regular expression inside the `replaceAll()` method. These filters should handle encoded special characters. As this function is sensitive, it is crucial to carefully consider and handle any potential issues that may arise when working with input data that may require decoding.

### Patch

Fixed in commit `4702294` by updating the function name to `escapeBundleCharacters()` as it is not going to escape user input.
