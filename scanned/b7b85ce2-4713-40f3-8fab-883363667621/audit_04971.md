# Weak condition

## Summary
The report states that there is a vulnerability in the `balanceCondition` code, specifically in the `Comparison.EQUAL` function. This vulnerability allows any actor to manipulate the token's balance, making it susceptible to DoS attacks. The report recommends avoiding the use of `Comparison.EQUAL` for token balance checks to prevent this vulnerability from being exploited.

## Details
##### Description
The token's balance can be easily manipulated by any actor, which makes `balanceCondition` vulnerable to DoS attacks:
https://github.com/MetaLex-Tech/borg-core/blob/9074503d37cfa1d777ef16f6c88b84c98b4f54eb/src/libs/conditions/balanceCondition.sol#L37.

##### Recommendation
We don't recommend using `Comparison.EQUAL` for token balance checks as it can be easily manipulated.
