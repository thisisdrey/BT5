# Pool Token Value Is Not Checked Against Total Short Size when Withdrawing, Potentially Resulting in Widespread Protocol Error

## Summary
This bug report is about an issue found in the code for the Pool.sol file. When withdrawing funds from the pool, there is a possibility for the short profits to exceed the value of the token in the pool, causing a negative AUM. This can also happen when opening or increasing a short position. To fix this, the report recommends implementing an error to check for these cases and prevent them from happening in the future. The bug has been fixed by implementing a new function called `_validateGlobalShortSize()`.

## Details
**Update**
Fixed by implementing `_validateGlobalShortSize()`. Total short size cannot exceed the value of the pool amount minus the reserved amount.

**File(s) affected:**`Pool.sol`

**Description:** When withdrawing from the pool via `removeLiquidity()`, an internal call is made to `_decreaseTranchePoolAmount()` which subtracts the withdrawn amount from `asset.poolAmount` and checks whether `asset.poolAmount < asset.reservedAmount`, reverting if this is the case. However, it does not check whether the value of the liquidity in the tranche can accommodate the total open short position size, `asset.totalShortSize`. If this is not the case, then it is possible for the short profits on a token to exceed the value of the new value of the token in the tranche. This would lead to a negative AUM, causing multiple functions to revert in the future.

The same problem arises when `totalShortSize` increases above the value of the tranche's value for a given asset when opening or increasing a short position.

**Recommendation:**

1.   Implement an error for the case where the new value of the tranche (with respect to an asset) is lower than `asset.totalShortSize` when withdrawing liquidity. 
2.   Implement an error for the case where the new `asset.totalShortSize` is greater than the value of the tranche with respect to the asset when opening or increasing a short position.
