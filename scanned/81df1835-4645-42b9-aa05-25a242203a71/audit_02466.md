# Case-Sensitive TDX Module Identity Comparison May Cause Valid Attestations to Fail

## Summary
The bug report describes an issue with how a TDX module identity ID is generated and compared in the `check_tdx_module_tcb_status` function of the `quote-verifier` crate. The ID is currently generated using lowercase hexadecimal formatting, but the reference C++ implementation uses uppercase formatting and performs a case-insensitive comparison. This could cause legitimate TDX attestations to fail verification and impact system functionality. The recommendation is to perform a case-insensitive comparison to avoid this issue.

## Details
**Update**
Marked as "Fixed" by the client. Addressed in: `d15940900350ad5b8170aeaf19b7b94904acbb85`.

**Update:** Note that `tdx_module_identity_id` is made uppercase by `let tdx_module_identity_id = format!("TDX_{:02X}", tdx_module_version)`, where `02x` was changed to `02X`.

**File(s) affected:**`crates/quote-verifier/src/tdx_module.rs`

**Description:** In the `check_tdx_module_tcb_status` function, a TDX module identity ID is generated using lowercase hexadecimal formatting:

```
let tdx_module_identity_id = format!("TDX_{:02x}", tdx_module_version);
```

This identifier is then used in a direct string comparison when filtering module identities:

```
.filter(|m| m.id == tdx_module_identity_id)
```

The reference C++ implementation in Intel's SGX-TDX-DCAP-QuoteVerificationLibrary explicitly normalizes string case before comparison:

```
std::string id = tdxModuleIdentity.getId();
std::transform(id.begin(), id.end(), id.begin(), ::toupper);
return (id == tdxModuleIdentityId);
```

If a mismatch occurs due to case sensitivity:

*   The function will not find a matching TDX module identity
*   It will return `TdxModuleTcbValidationStatus::TcbNotSupported`
*   This will likely cause legitimate TDX attestations to fail verification

While this is not a security vulnerability allowing unauthorized access, it is a correctness issue that could impact system functionality by causing false rejections of valid attestations.

**Recommendation:** Consider performing a case-insensitive comparison:

```
.filter(|m| m.id.to_uppercase() == tdx_module_identity_id.to_uppercase())
```
