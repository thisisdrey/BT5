# [C-03] Number of Available Spins Is Never Decreased On Spin or If Spin Is Fulfilled

## Summary
This bug report is about an issue in a code that allows users to have unlimited spins and rewards. This can cause harm to the protocol. The code does not decrease the amount of available spins for a user, even though there is a function called `getAvailableSpinsOf()` that is supposed to do that. This function is not being used in the code, which means that the available spins are never decreased. This can be seen in the proof of concept section, where the code is tested and the output shows that the number of available spins remains the same even after the user has spun. The recommendation is to fix this issue by implementing a decrease in the available spin count in the code. The team has responded that they have fixed the issue.

## Details
## Severity

Critical Risk

## Description

When a user spins or the user's spin is fulfilled the code has nowhere where it decreases the amount of available spins for a user. However `getAvailableSpinsOf()` is out of scope, thus it may happen elsewhere but in the code, it is never decreased in any way.

## Impact

The user has unlimited spins and rewards available to them which can cause the protocol harm.
