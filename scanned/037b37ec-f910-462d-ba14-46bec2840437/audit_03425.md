# Missing enforcement of fillDeadline check breaks ERC-7683 requirement and may lead to loss of ﬁller rewards 

## Summary
This bug report is about a missing check in the Eco7683DestinationSettler contract that could lead to loss of rewards for users. The ERC-7683 specification requires a check for fillDeadline, which is the timestamp by which an order must be filled. However, this check is not enforced in the fill() function, allowing fillers to accidentally fill orders even after the specified deadline. This could result in loss of rewards for fillers. The impact of this bug is considered high, but the likelihood is low as most fillers are expected to perform their own checks before filling orders. The recommendation is to add a defensive check in the fill() function to prevent accidental loss of rewards. This bug has been fixed in a recent commit.

## Summary
Missing enforcement of `fillDeadline` check in `Eco7683DestinationSettler.fill()` breaks ERC-7683 requirement and may lead to loss of filler rewards.

## Finding Description
ERC-7683 specifies requirements for cross-chain intent orders on behalf of a user, which are filled via `fill()` function. One of the requirements is the enforcement of `fillDeadline`, which is the timestamp by which the order must be filled by the filler on the destination chain. This is part of all the ERC-7683 structs and allows users to specify time-sensitive intents:

- `@param fillDeadline` The timestamp by which the order must be filled on the destination chain.

However, the `fillDeadline` check is not enforced against `block.timestamp` in `Eco7683DestinationSettler.fill()`, which allows fillers to accidentally fill such intents at any time even beyond the `fillDeadline` specified. But such fillers cannot withdraw their rewards because this check is enforced during `withdrawRewards()` within `IntentVault.constructor()` when rewards for expired intents are refunded to the intent creator.

## Impact Explanation
**High**, because this missing check breaks ERC-7683 requirement and may lead to loss of filler rewards.

## Likelihood Explanation
**Low**, because intent fillers are expected to perform their own due-diligence checks on intents before attempting to fill them, and so this can only happen accidentally with fillers who do not self-validate intents correctly.

## Recommendation
Consider adding a defensive `fillDeadline` check such as `block.timestamp <= order.fillDeadline` in `Eco7683DestinationSettler.fill()` to meet ERC-7683 requirements and prevent accidental loss of filler rewards.

## Eco
Fixed in commit `2e7bc2f`.

## Cantina Managed
Reviewed that commit `2e7bc2f` fixes this issue with the below logic:

```solidity
function fill(
    bytes32 _orderId,
    bytes calldata _originData,
    bytes calldata _fillerData
) external payable {
    Intent memory intent = abi.decode(_originData, (Intent));
    if (block.timestamp > intent.reward.deadline) {
        revert FillDeadlinePassed();
    }
    // ...
}
```
