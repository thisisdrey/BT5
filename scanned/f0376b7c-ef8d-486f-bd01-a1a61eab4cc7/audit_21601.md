# Missing Proper Access Control ✓ Fixed

## Summary
A bug was found in the code of two contracts, Auction.sol and AuctionBoard.sol, which are part of the Lien Protocol. These contracts have some functions that do not have proper access control and are set as public, meaning anyone can call them. This could result in a system takeover depending on how critical those functionalities are. As an example, anyone can set the IDOLContract in MainContracts.Auction.sol, which is a critical aspect of the auction contract, and it cannot be changed after it is set. The recommendation is to make the setIDOLContract() function internal and call it from the constructor, or only allow the deployer to set the value. This would ensure that only the deployer can set the IDOLContract and prevent a system takeover.
