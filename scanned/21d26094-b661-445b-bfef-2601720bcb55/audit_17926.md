# BRF-1 | Accidental Magnification

## Summary
This bug report is about an issue with the `distribute` and `withdraw` functions in a program. In the `distribute` function, each `user.earned` amount is multiplied by the `disRate`. However, in the `withdraw` function, the `user.earned` amount is not divided by a divisor that corresponds to the `disRate`. The recommendation is to add a divisor for the `disRate` and use it to adjust `user.earned` values in either the `distribute` or `withdraw` functions as needed. The Bridges Team has responded that this is intended behavior and that they treat the `disRate` as a multiplier.

## Details
**Description**

In the `distribute` function each `user.earned` amount is multiplied by the `disRate` but in the withdraw function, the `user.earned` amount is not divided by some divisor that corresponds to the `disRate`.

**Recommendation**

Add a divisor for the `disRate` and use it to adjust `user.earned` values in either the `distribute` or `withdraw` functions as needed.

**Resolution**

Bridges Team:

- This is intended behavior, we treat the disRate as a multiplier.
