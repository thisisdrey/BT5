# [M-06] Even when the Vault contract is paused, the `rebalance` function is not paused

## Summary
This bug report discusses an issue found in the code for a project called BakerFi. The issue is related to the `rebalance` function, which is used to manage investments in the project's Vault. The problem is that this function can still be called even when the project is paused, which goes against the project's own documentation. This means that while users cannot withdraw their investments when the project is paused, the project can still collect performance fees from interest. The report recommends adding a modifier to the `rebalance` function to prevent it from being called when the project is paused. The project has confirmed the issue and has implemented a solution.

### Finding description and impact

When the contract is paused, `rebalance` is not paused. While users cannot withdraw, performance fees can still be collected from interest.

### Proof of Concept

The `rebalance` should not be callable when paused (according to the [documentation](https://github.com/code-423n4/2024-12-bakerfi/blob/0daf8a0547b6245faed5b6cd3f5daf44d2ea7c9a/doc/core/VaultBase.md# pause)), but it can still be called even when paused. This means that while users cannot withdraw their investments from the Vault when paused, it’s still possible to collect performance fees on interest through the `rebalance` function. Also, MultiStrategyVault has the same issue.
```

    function rebalance(
        IVault.RebalanceCommand[] calldata commands
@>  ) external override nonReentrant onlyRole(VAULT_MANAGER_ROLE) returns (bool success) {
        success = true;
        uint256 numCommands = commands.length;
        for (uint256 i = 0; i < numCommands; ) {
            if (commands[i].action == HARVEST_VAULT) {
                _harvestAndMintFees();
            }
            unchecked {
                i++;
            }
        }
    }
```

### Recommended Mitigation Steps

Add the `whenNotPaused` modifier to the `rebalance` function.
