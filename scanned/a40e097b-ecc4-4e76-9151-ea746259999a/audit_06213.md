# DelegatedStaking contract does not implement any receive or fallback function

## Summary
The report discusses a bug in the `DelegatedStaking` contract, which is used to hold Eon tokens for users to claim rewards. The contract does not have a way to receive funds, which could prevent users from claiming their rewards. The BVSS score for this bug is 5.0, which is considered medium risk. The recommendation is to add a `receive()` function to the contract to allow it to receive funds. The team has accepted the risk and will address it by having the contract receive funds through its address.

## Details
##### Description

The `DelegatedStaking` contract is supposed to hold Eon tokens in order to allow users to execute their claims by calling the `claimReward()` function. However, this contract does not implement any `payable` function, nor a `receive()` or `fallback()` function.

##### Recommendation

Consider implementing a `receive()` function in the `DelegatedStaking` contract.

  

### Remediation Plan

**RISK ACCEPTED:** The **Horizen Labs team** accepted the risk of this finding, and states that the smart contract will receive funds inserting its address as forger rewards receiver. In this case, fallback methods will not be required.
