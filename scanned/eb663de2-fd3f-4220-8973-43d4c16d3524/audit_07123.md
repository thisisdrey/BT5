# Incorrectly setting `s_referrerConfig.basisPoints` will lead to the lock of funds

## Summary
This bug report discusses an issue with a contract called BaseFeeDistributor.sol. The problem occurs when the value for `_referrerConfig.basisPoints` is greater than the value for `_clientConfig.basisPoints`. This causes the `withdraw` function to always fail, resulting in an error. This can happen if a user accidentally sets the value for `_referrerConfig.basisPoints` too high, as there are currently no checks in place to prevent this. The recommendation is to add checks to ensure that the value for `_referrerConfig.basisPoints` is always lower than the value for `_clientConfig.basisPoints` and that it is also lower than 10,000 minus the value for `_clientConfig.basisPoints`. This will prevent the issue from occurring in the future.
