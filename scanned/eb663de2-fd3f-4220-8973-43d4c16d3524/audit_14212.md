# Unsafe practice of managing a user's ERC20 approve

## Summary
This bug report is about a security issue that can be found in the architecture of a project. It states that a user's ERC20 tokens can be spent without their confirmation, reducing the overall security of the project. To fix this issue, it is recommended that every single spending of the user's ERC20 tokens be explicitly signed by the user. This would ensure that the user is aware of and agrees to each spending, making it much harder for malicious actors to take advantage of the system.

## Details
##### Description
As it is stated above in the description of the Critical.1 finding, a user's ERC20 tokens can be spent without the user's confirmation. This bad practice reduces the overall project's security by amplifying the impact of other issues. For example, Critical.1 would have a Low severity, and the Medium.2 would be a no issue, if finding Medium.1 is fixed as recommended.

As a general architecture recommendation, this finding is described as a separate item of the report.
##### Recommendation
It is recommended that every single spending of the user's ERC20 tokens be explicitly signed by the user.
