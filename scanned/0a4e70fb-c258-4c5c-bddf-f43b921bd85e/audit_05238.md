# Risk of fee loss due to transactions through a paused router

## Summary
The `UnifiedRouterV2` contract is currently experiencing a bug where it can be paused. This means that if a user tries to use the contract and their path includes a paused router, their transaction will fail and they may lose money on fees. To prevent this issue, it is recommended to notify users on the front-end when contracts are paused.

## Details
##### Description

The `UnifiedRouterV2` contract can be paused. If the user's path includes a paused router, their transaction can fail, and they might simply lose money on fees.

##### Recommendation
We recommend notifying users on the front-end when contracts are paused.
