# L2 Blocks With Incorrect Timestamp Could Be Finalized

## Summary
This bug report is about the `_finalizeBlocks` function in the `ZkEvmV2` contract. This function should check that the current block's timestamp (`blockInfo.l2BlockTimestamp`) is greater or equal than the last L2 block timestamp and less or equal than the L1 block timestamp when `_finalizeBlocks` is executed. However, the first check is missing, which means that blocks with incorrect timestamps could be finalized, resulting in unintended system behavior. The recommendation is to add the missing timestamp check. This can be done by adding the code snippet provided in the report to the `ZkEvmV2.sol` file at line 158-160.

## Details
#### Description


In `_finalizeBlocks` of `ZkEvmV2`, the current block timestamp `blockInfo.l2BlockTimestamp` should be greater or equal than the last L2 block timestamp and less or equal than the L1 block timestamp when `_finalizeBlocks` is executed. However the first check is missing, blocks with incorrect timestamp could be finalized, causing unintended system behavior

if (blockInfo.l2BlockTimestamp >= block.timestamp) {
 revert BlockTimestampError();

Add the missing timestamp check
