# Message Spoofing

## Summary
The report discusses a bug in the transaction and addChainconfirmation dialog, where a JSON string is shown and parsed after approval. However, the JSON parser may remove duplicated entries, which can lead to malicious requests deceiving the user. To fix this issue, the object should be parsed before being presented to the user and the parameters should be checked for validity. This bug has been fixed in a recent commit by adding a check for valid parameters in all RPC functions.

## Details
## Transaction Confirmation and JSON Parsing in Chain Addition

The transaction and addChain confirmation dialog shows a JSON string that will be parsed after approval:

```ts
confirmation = await snap.request({
  method: "snap_dialog",
  params: {
    type: "confirmation",
    content: panel([
      heading("Confirm Chain Addition"),
      divider(),
      heading("Chain Info"),
      text(
        `${request.params.chain_info}`
      ),
    ]),
  },
});
if (!confirmation) {
  throw new Error("Chain addition was denied.");
}
let new_chain: Chain = JSON.parse(request.params.chain_info);
```

However, the JSON parser may invalidate some keys shown in the dialog since it removes duplicated entries. Thus, attackers may craft a malicious request object that deceives the user into thinking it is a legitimate transaction.

## Remediation

Parse the object before presenting it to the user. In addition, check if the parameters contained in the JSON object are valid parameters.

## Patch

Fixed in commit `36d71c0` by adding a check in all RPC functions, which checks if the passed parameters are valid:

```ts
+ let new_chain: Chain = JSON.parse(request.params.chain_info);
+ if (
+   !(
+     "chain_name" in new_chain &&
+     "chain_id" in new_chain &&
+     typeof new_chain.chain_name == "string" &&
+     typeof new_chain.chain_id == "string"
+   )
+ ) {
+   throw new Error("Invalid Chain Info");
+ }
```
