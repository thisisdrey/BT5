# Return Length of ‘EcPairing’ Does Not Match the Specifications

## Summary
This bug report is about a feature called `EcPairing` on Ethereum, which is supposed to return data of exactly 32 bytes in a specific format. However, the `fallback` function of the `EcPairing` contract returns 64 bytes instead. This could potentially cause issues and does not follow the specifications of the Ethereum Virtual Machine (EVM). The bug has been fixed in a recent update, where the return value has been changed to the correct length of 32 bytes.

## Details
[EIP-197](https://eips.ethereum.org/EIPS/eip-197) introduces the `EcPairing` precompile on Ethereum and states that "The length of the returned data is always exactly 32 bytes and encoded as a 32 byte big-endian number". However, the `fallback` function of the `EcPairing` contract returns [64 bytes](https://github.com/matter-labs/era-contracts/blob/886018aa81bc224b2721982ba9623e129b64b87c/system-contracts/contracts/precompiles/EcPairing.yul#L122).

Consider returning only 32 bytes to avoid potential issues and more closely match the EVM's specifications.

***Update:** Resolved in [pull request #1373](https://github.com/matter-labs/era-contracts/pull/1373) at commit [fab789f](https://github.com/matter-labs/era-contracts/commit/fab789ffd276411463a6aa4bdebfb74a072c84c2). The return value has been updated to 32 bytes.*
