# [M-05] flan can't be transferred unless the flan contract has flan balance greater than the amount we want to transfer

## Summary
The report is about a bug found in a contract called Flan. The bug affects the `safeTransfer()` function, which is used to transfer Flan tokens. The contract needs to have enough Flan tokens in order for the transfer to be successful. However, if there are no tokens in the contract, the transfer will fail. This is because the function is checking the contract's balance instead of the sender's balance. This bug has been confirmed by a developer but they disagree on the severity, with one person saying it is a high severity bug and another saying it is a medium severity bug. The severity has been downgraded to medium as there is no risk to funds.

## Details
_Submitted by CertoraInc_

#### Flan.sol (`safeTransfer()` function)

The flan contract must have balance (and must have more flan then we want to transfer) in order to allow flan transfers. If it doesn't have any balance, the safeTransfer, which is the only way to transfer flan, will call `_transfer()` function with `amount = 0`. It should check `address(msg.sender)`'s balance instead of `address(this)`'s balance.

```solidity
function safeTransfer(address _to, uint256 _amount) external {
  uint256 flanBal = balanceOf(address(this)); // the problem is in this line
  uint256 flanToTransfer = _amount > flanBal ? flanBal : _amount;
  _transfer(_msgSender(), _to, flanToTransfer);
}
```
