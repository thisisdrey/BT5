# Excessive Oracle Requests Without Financial Penalties

## Summary
This bug report discusses a potential issue with the system that could lead to a denial of service. Users are able to submit requests to the system without providing the required data on-chain. This could lead to the system being overloaded and legitimate requests not being processed. The report suggests implementing penalties for users who do not follow through with on-chain submissions and increasing the default timeout for oracle jobs. The issue has been resolved in the ba65e19 patch.

## Details
## Risk of Denial of Service

There is a risk of denial of service due to the lack of financial penalties for users who submit Oracle requests but fail to provide the required data on-chain. Users are expected to submit Oracle-generated values back on-chain. However, if they fail to do so, they do not incur any costs. Thus, users may exploit the system without any immediate repercussions, continuously submitting Oracle requests without following through with on-chain submissions, thereby overloading the system by occupying resources.

The default timeout for Oracle jobs is 5 seconds. Nonetheless, this short rate limit can be easily bypassed by sending requests from several IPs simultaneously. Hence, it is relatively cheap to overload Oracles, preventing them from processing legitimate requests from users intending to post their values on-chain and pay their fees. Additionally, if future API routes or updates involve signing transactions with the payer key, malicious users may abuse these mechanisms to further exploit the system.

## Remediation

- Introduce fees or penalties for users who submit requests but do not follow through with on-chain submissions.
- Consider increasing the default timeout for Oracle jobs to accommodate longer processing times.

## Patch

Resolved in ba65e19.
