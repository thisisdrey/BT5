# Potential Deposit Blockage

## Summary
The bug report describes an issue with the `maxDeposit` function in the `sUSX.sol` file. This function is used to calculate the maximum amount that can be deposited, but it is currently causing the contract to revert due to an underflow error. This error occurs when the `mintCap` value is changed by the contract owner, leading to a negative value when subtracting the `totalSupply`. This bug is marked as medium severity because it can temporarily lock the contract. The recommendation is to fix this issue by returning a value of 0 instead of reverting.

## Details
##### Description
The `maxDeposit` [function](https://github.com/dforce-network/sUSX/blob/2ff5d7a8f9a509006557bf5de72fabf40d1138a5/src/sUSX.sol#L229) will attempt to calculate the maximum amount that can be deposited based on the new `mintCap`, but it will revert due to underflow because `mintCap - totalSupply` is negative. This situation is not triggered by user actions but can occur if  the admin changes the `mintCap` value.

This issue is marked as **medium** because owner actions can lead to a temporary contract lock.

##### Recommendation
We recommend avoiding reverting by returing 0 in maxDeposit.
