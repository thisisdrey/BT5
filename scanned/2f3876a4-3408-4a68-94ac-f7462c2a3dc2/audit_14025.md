# Later claimer might not be able to claim reward

## Summary
This bug report is about an issue with the getAvailableReward function in a contract. If a user claims a reward very late (many years after), the getAvailableReward might revert with an out of gas error. This is because the currentAvailableRewardPeriod might become greater than the rewardPeriods. Since the contract is not checking for this scenario, looping the periods from 0 to currentAvailableRewardPeriod might cause an out of gas error.

To fix this issue, the loop should not go beyond the rewardPeriods, if the currentAvailableRewardPeriod is greater than the rewardPeriods.
