# Wallet is not encrypted on the ﬁlesystem

## Summary
This bug report is about a vulnerability in the ZecWallet program which allows an attacker to steal an unencrypted wallet.dat file from the user's computer. The wallet.dat file is used to store the user's cryptocurrency wallet information. The vulnerability exists in the connection.cpp file of the program.

The vulnerability is caused by the fact that the wallet.dat file is not encrypted when it is backed up. This means that anyone with access to the user's computer can access the unencrypted wallet.dat file and steal the user's cryptocurrency.

The recommendation is to encrypt all copies of the wallet database as soon as possible. This will ensure that any attackers will not be able to access the wallet.dat file and steal the user's cryptocurrency.

References are provided for further reading on the subject of encrypted streams and file encryption.

## Description
When the wallet is backed up, it is not encrypted. Copies of the wallet made by the program are saved to the file system unencrypted.

```cpp
void MainWindow::backupWalletDat() {
    if (!rpc->getConnection())
        return;

    QDir zcashdir(rpc->getConnection()->config->zcashDir);
    QString backupDefaultName = "zcash-wallet-backup-" +
        QDateTime::currentDateTime().toString("yyyyMMdd") + ".dat";

    if (Settings::getInstance()->isTestnet()) {
        zcashdir.cd("testnet3");
        backupDefaultName = "testnet-" + backupDefaultName;
    }

    QFile wallet(zcashdir.filePath("wallet.dat"));
    if (!wallet.exists()) {
        QMessageBox::critical(this, 
            tr("No wallet.dat"), 
            tr("Couldn't find the wallet.dat on this computer") + "\n" +
            tr("You need to back it up from the machine zcashd is running on"),
            QMessageBox::Ok);
        return;
    }

    QUrl backupName = QFileDialog::getSaveFileUrl(this, 
        tr("Backup wallet.dat"),
        backupDefaultName, "Data file (*.dat)");
    if (backupName.isEmpty())
        return;

    if (!wallet.copy(backupName.toLocalFile())) {
        QMessageBox::critical(this, 
            tr("Couldn't backup"), 
            tr("Couldn't backup the wallet.dat file.") +
            tr("You need to back it up manually."), 
            QMessageBox::Ok);
    }
}
```

**Figure 1:** Backups of the wallet, and the main wallet file are never encrypted (mainwindow.cpp#L908-L935)

## Exploit Scenario
Eve has access to Alice’s workstation. Eve knows Alice’s wallet is stored in an unencrypted flat file. Eve steals the wallet effortlessly as it is not encrypted.

## Recommendation
Encrypt all copies of the wallet database immediately. Wallet data must be encrypted at rest as soon as possible.
