# Anyone can block the payout token by changing its amount by 1 wei

## Summary
This bug report is about a check in the `updatePayoutToken()` function in the FantiumClaimingV1.sol file. This check requires that the balance should be 0, but anyone can send 1 wei to the balance and make it impossible to change the payout token. To fix this issue, the report recommends removing the zero balance check from the `updatePayoutToken()` function. This is because the token address is now stored in the snapshot, which means that the check is no longer necessary.

## Details
##### Description
There is a check in `updatePayoutToken()` that the balance should be 0.
https://github.com/FantiumAG/smart-contracts/blob/b4acd978e25289becc640f75eb5921e760b83720/contracts/claiming/FantiumClaimingV1.sol#L749
So, anyone can send 1 wei to the balance and make it impossible to change the payout token.

##### Recommendation
We recommend removing a zero balance check from `updatePayoutToken()`. Now it's not required because the token address is stored in the snapshot.
