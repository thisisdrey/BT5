# Mnemonic is copied to clipboard

## Summary
This bug report is about data exposure in iOS and Android wallets. The wallets allow users to copy the mnemonic (i.e., the recovery phrase) to the clipboard, which can potentially expose the mnemonic either accidentally or due to malicious applications. Both wallets have a copy button in two views that display the mnemonic to the user. The mnemonic is the most sensitive piece of data the wallet has, and both iOS and Android clipboard content can be read from another app. For example, the official Google search application automatically reads clipboard content. Additionally, the clipboard can now be shared between Apple devices, notably macOS, which further increases the possibility of exposure.

The exploit scenario is that a wallet user copies the recovery phrase and another application reads from the clipboard and sends it to the internet. To address this, the recommendation is to not allow users to copy the mnemonic to the clipboard and require them to retype it in a following view during onboarding. To avoid this type of data exposure in the future, it is recommended to avoid copying sensitive data to the clipboard.

## Description
The wallets allow users to copy the mnemonic (i.e., the recovery phrase) to the clipboard, potentially leaking the mnemonic either by accident or due to a malicious application. Both wallets have a copy button in two views that display the mnemonic to the user. The first view is part of onboarding, and the second is an option in settings. The mnemonic is the most sensitive piece of data the wallet has, and both iOS and Android clipboard content can be read from another app. For instance, the official Google search application automatically reads clipboard content. Furthermore, the clipboard can now be shared between Apple devices, notably macOS, which further increases the possibility of exposure.

## Exploit Scenario
A wallet user copies the recovery phrase. Another application reads from clipboard and sends the phrase to the Internet.

## Recommendation
- **Short term:** Don’t allow users to copy the mnemonic to the clipboard. Ensure users store the phrase by requiring them to retype it in a following view during onboarding.
- **Long term:** Avoid copying sensitive data to the clipboard.
