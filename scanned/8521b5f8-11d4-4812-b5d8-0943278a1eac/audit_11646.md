# Lack Of Restriction On Seed Length

## Summary
The bug report states that there is an issue in the code for the ElGamalSecretKey, which is written in the programming language RUST. The problem occurs because there are no checks or limits on the length of the input for the seed, which can cause problems such as stack overflow or panic if the seed is too long. This is because the code recursively calls a function called Self::from_seed(seed)?, which can lead to a panic. The report suggests implementing a check to make sure the seed length is not too long, and this issue has been fixed in version 33700.

## Details
## Issue Summary

The issue arises due to a lack of explicit checks or restrictions on the length of the seed input, which may result in various issues, including a stack overflow or panic if the seed length is very high. The code recursively calls `Self::from_seed(seed)?`, resulting in a panic.

## Code Snippet

```rust
elgamal.rs
```

```rust
impl SeedDerivable for ElGamalSecretKey {
    fn from_seed(seed: &[u8]) -> Result<Self, Box<dyn error::Error>> {
        let key = Self::from_seed(seed)?;
        Ok(key)
    }
}
```

## Remediation

Implement a check that ensures the seed length does not exceed a preset value.

## Patch

Fixed in commit `33700`
