# Unchecked possible zero address

## Summary
The report states that there is a medium severity bug in the contract UmamiPriceFeed.sol. The bug has been resolved, but it is important to understand what the bug was and how it was fixed. The bug was caused by not checking for null or zero input addresses in certain functions. This means that if an input address was accidentally left blank, it could cause the contract to become locked. The report recommends adding a check to ensure that input addresses are not null or zero to prevent this issue from happening in the future.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In contract UmamiPriceFeed.sol there are multiple cases where input addresses are not checked to not be null/zero. For example, the setGov function sets the gov address, so in case the input address is by accident send as zero address it can lock the contract as the setter has the 'onlyGov' modifier. This also occurs in functions setGmxVaultPriceFeed and setSecondaryPriceFeed.

**Recommendation**: 

Add a sanity check for the input addresses to not be null/zero
