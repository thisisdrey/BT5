# Exit pool potentially is impossible

## Summary
This bug report is about a problem with the code found in the lines 163 and 179 of the BasketFacet.sol file in the ExperiPie repository on GitHub. This code includes a requirement that token balance must be greater than a certain minimum amount (MIN_AMOUNT) in order for a user to take all of their tokens and leave the pool. This requirement prevents users from being able to completely leave the pool, as some of their funds will be locked in the pool forever. 

The bug report suggests that the issue likely lies in the MIN_AMOUNT variable not being set to 0 due to the division by totalSupply in some places. It is recommended that this moment should be covered in the source code to make it clear why this problem exists and how it can be resolved.

## Details
##### Description
At the lines https://github.com/pie-dao/ExperiPie/blob/0.0.2/contracts/facets/Basket/BasketFacet.sol#L163 and https://github.com/pie-dao/ExperiPie/blob/0.0.2/contracts/facets/Basket/BasketFacet.sol#L179
Due to `require(tokenBalance.sub(tokenAmount) >= MIN_AMOUNT, "TOKEN_BALANCE_TOO_LOW");` it is impossible to leave the pool completely, so some funds will be locked in the pool forever.

It is not clear why there should be such unsolvable problems if you want just to "take all your tokens and leave".

##### Recommendation
Probably `MIN_AMOUNT` is not 0 because of the division by `totalSupply` in some places. At least this moment should be covered in source code somewhere.
