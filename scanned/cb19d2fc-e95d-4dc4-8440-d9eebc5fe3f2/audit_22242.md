# Burn of token assignment to BAT fund is not implemented

## Summary
This bug report is about the Brave token sale. The Brave team was given a process document that states that if the token sale doesn't sell out, the BAT kept in the multisig wallet should be 'burned' down to a 3/10 ratio of the total BAT. However, the amount of BAT assigned to Brave is currently fixed at deploy time. This means that Brave team has to manually adjust the amount of BAT after the token sale, which is inconsistent with the process document. The bug report suggests that the issue should be addressed by implementing the process document into the contract, so that Brave team does not have to manually adjust the amount of BAT.

## Details
The amount of BAT assigned to Brave is fixed at deploy time. This seems to be inconsistent with the process document we received that states: “In event of a non-sell out, contract will ‘burn’ BAT kept in multisig wallet down to a 3/10 ratio of total BAT.”. Consider implementing this in the contract, as the specification states, so that the Brave team doesn’t have to do this manually after the token sale.
