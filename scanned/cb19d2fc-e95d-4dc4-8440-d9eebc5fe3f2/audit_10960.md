# Faulty Scale Update

## Summary
The function "set_scale" in the "utils" file of the RUST programming language is not working correctly. Instead of modifying the original decimal value, it returns a new decimal with the updated scale. This can cause problems when converting the decimal value to its native representation, as the result from "set_scale" is not captured. To fix this issue, the function needs to be modified so that it updates the original decimal value. This bug has been fixed in the recent version 2c4422d.

## Details
## Issue with `set_scale` in Utils

The `set_scale` function in `utils` fails to modify the original decimal value; instead, it returns a new decimal with the updated scale. This behavior is problematic as the result of `set_scale` is not captured, which causes the conversion of the decimal value to its native representation using `Decimal::to_native` to fail.

## Code Example

```rust
fn set_scale(&self, scale: u32) -> Option<Self> {
    Some(self / 10f64.powi(scale.try_into().unwrap()))
}
```

## Remediation

Modify `set_scale` to update the original decimal value.

## Patch

Fixed in commit `2c4422d`.
