# It's possible to top up a closed distribution

## Summary
This bug report is about a problem that occurs when a manager calls `updateDistribtionTotalEarningsAmounts()` or `updateDistributionEventCollectionIds()` for a closed distribution. If this happens, the athlete's funds will be topped up, but the funds will then be stuck because it's impossible to close the already closed distribution again. 

The recommendation to fix this issue is to add a check to `updateDistribtionTotalEarningsAmounts()` and `updateDistributionEventCollectionIds()` that would require that the distribution is not closed before the function is called. This would prevent the athlete's funds from being stuck.

## Details
##### Description
If the manager calls by mistake `updateDistribtionTotalEarningsAmounts()` or `updateDistributionEventCollectionIds()` for a closed distribution, it will be topped up using the athlete's funds. These funds will be stuck because it's impossible to close the already closed distribution again.

##### Recommendation
We recommend adding a check to `updateDistribtionTotalEarningsAmounts()` and `updateDistributionEventCollectionIds()`:
`require(!distributionEvents[_id].closed, "FantiumClaimingV1: distribution already closed");`.
