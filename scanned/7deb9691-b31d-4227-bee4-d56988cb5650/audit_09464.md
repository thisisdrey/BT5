# Owner Can Override a Bond

## Summary
This bug report is about a function called `listBond()` in a file called `Fintroller.sol`. The function allows the owner to add a new bond token, but if the owner adds an existing bond, it will override the old one. The bug report suggests that the team should check if the bond already exists before adding it to the `bonds` mapping. The team has confirmed that this is the intended behavior.

## Details
**Update**
The Hifi team indicated that this is the intended behavior.

**File(s) affected:**`packages/protocol/core/fintroller/Fintroller.sol`

**Description:** The owner has the ability to add a new bond token using the `listBond()` function. This will create the new bond with the default parameters. However if the owner injected an existing bond he will override the old bond.

**Recommendation:** Consider verifying if the bond already existed before inserting it into the `bonds` mapping.
