# S3 Buckets Cleartext Communication ✓ Fixed

## Summary
This bug report concerns the unencrypted traffic to the assets of the system's S3 buckets. Direct access to S3 buckets through `s3.amazonaws.com` was rejected, while unencrypted HTTP traffic to the previously affected assets consistently redirects to the HTTPS equivalents. The affected assets include seven S3 buckets: `arn:aws:s3:::ropsten-app.fei.money/*`, `arn:aws:s3:::www.fei.money/*`, `arn:aws:s3:::feiprotocol.com/*`, `arn:aws:s3:::www.app.fei.money/*`, `arn:aws:s3:::www.ropsten-app.fei.money/*`, `arn:aws:s3:::app.fei.money/*`, and `arn:aws:s3:::fei.money/*`. To resolve the issue, it is recommended to enforce encryption of data in transit using TLS certificates. This can be done by setting the `aws:SecureTransport` in the S3 bucket’s policies.

## Details
#### Resolution



Direct access to S3 buckets through `s3.amazonaws.com` is now rejected, while unencrypted HTTP traffic to the previously affected assets now consistently redirects to the HTTPS equivalents.
