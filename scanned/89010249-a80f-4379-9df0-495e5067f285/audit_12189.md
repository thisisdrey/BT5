# Top level assertion timer not included in honest path timer calculation

## Summary
This report describes a bug in the Arbitrum Chains Challenge Protocol v2 that affects the data validation process. The bug occurs when calculating the cumulative unrivaled timer of all honest edges. The validator does not account for the unrivaled timer of the honest top-level assertion, which leads to incorrect calculations. This bug could potentially allow a dishonest actor to win the challenge. The report recommends adding the unrivaled timer of the honest top-level assertion to the calculation in the short term and improving unit testing in the long term to ensure the correct behavior of the function.

## Description

Whenever the cumulative unrivaled timer of all honest edges is calculated, the validator does not account for the unrivaled timer of the honest top-level assertion. The validator periodically updates the cumulative unrivaled timer of all edges in the honest path using the `UpdateCumulativePathTimers` function.

```go
func (ht *HonestChallengeTree) UpdateCumulativePathTimers(blockNum uint64) error {
    if ht.honestBlockChalLevelZeroEdge.IsNone() {
        return nil
    }
    blockEdge := ht.honestBlockChalLevelZeroEdge.Unwrap()
    return ht.recursiveTimersUpdate(
        0, // Total honest path timer accumulator, starting at 0.
        blockEdge,
        blockNum,
    )
}
```

This function recursively navigates the honest path of the tree of edges from the honest block level-zero edge down to the lowest edge, calculates the cumulative timer, and updates the edges.


However, as shown in the above snippets, the calculation of the cumulative timer starts at the level-zero block edge and does not account for the unrivaled time of the top-level honest assertion, which is incorrect.

## Exploit Scenario

Because the validator does not account for the unrivaled timer of the top-level honest assertion when calculating the cumulative unrivaled time of the honest edge path, a dishonest actor could potentially win the challenge.

## Recommendations

- **Short term:** Add the unrivaled timer of the honest top-level assertion to the cumulative unrivaled time calculation.
- **Long term:** Improve unit testing to verify the expected behavior of the function.
