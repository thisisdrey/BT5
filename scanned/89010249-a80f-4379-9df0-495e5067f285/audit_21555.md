# Remove commented out code from the repository ✓ Fixed

## Summary
This bug report is about the presence of commented out code in a repository. Commented out code increases the complexity of an already complex system, and it hides the important parts of the system that should get proper attention. It also has to be maintained and will be out of date if other parts of the system are changed, and the tests will not pick that up. The main problem is that commented code adds confusion with no real benefit.

To solve this issue, the code should be removed or dealt with in a separate branch that is later included in the master branch. This will help to reduce the cognitive load on the system, as well as ensure that the code is up to date and the tests will be able to pick up any changes. Additionally, it will help to keep the code and comments separate.

## Details
#### Description


Having commented out code increases the cognitive load on an already complex system. Also, it hides the important parts of the system that should get the proper attention, but that attention gets to be diluted.


There is no code that is important enough to be left commented out in a repository. Git branching should take care of having different code versions or diffs should show what was before.


If there is commented out code, this also has to be maintained; it will be out of date if other parts of the system are changed, and the tests will not pick that up.


The main problem is that commented code adds confusion with no real benefit. Code should be code, and comments should be comments.
