# Deprecate DisperseBlob endpoint still in use 

## Summary
The current implementation includes a feature called "DisperseBlob" which is marked for deprecation according to the API documentation. This means that it will no longer be supported in future releases. The documentation recommends that all production traffic should use the "DisperseBlobAuthenticated" endpoint instead. It is recommended to update client code to use this endpoint and, if possible, implement it in the "DisperserImpl" layer. The bug has been fixed in a recent commit, but it should be noted that the fix also adds a new signing key which is handled insecurely due to a known issue with the handling of admin signing keys.
