# Receipt event has confusing semantics

## Summary
This bug report is about the `Receipt` event in a codebase. This event is emitted in line 29 for each payment processed, and it logs the investor address, payment hash, and amount of ether received. However, it is also emitted with an empty hash when the owner withdraws the collected ether in line 37. This can be confusing, so it is suggested that a separate `Withdrawal` event is emitted in `withdraw` instead. 

This issue has since been resolved, and the event has been removed from the `withdraw` function. It is now only emitted for payments, which is a more clear and understandable semantic.

## Details
The `Receipt` event is emitted, in line 29, for each payment processed. It logs the investor address, payment hash, and amount of ether received. The same event is emitted, with an empty hash, when the owner withdraws the collected ether in line 37. This overloading of the event’s semantics seems confusing. Consider emitting instead a separate `Withdrawal` event in `withdraw`.


**UPDATE:** *The event has been removed from the `withdraw` function. It is now only emitted for payments, which are clearer semantics.*
