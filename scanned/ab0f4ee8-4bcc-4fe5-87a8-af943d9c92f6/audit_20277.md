# [M-02] `safeTransferFrom` in `TransferHelper` is not `safeTransferFrom`

## Summary
The bug report was submitted by jonah1005 and also found by 0xRajeev, shw, JMukesh, and cmichel. It states that there is an issue with a non-standard ERC20 token, where calling the `_safeTransferFrom` function will always result in an error. This bug can be seen when a user creates a USDT/DAI pool and deposits funds into it, but does not see a corresponding deposit in the pool. The `TransferHelper.sol` file, specifically line 19, is mentioned as being related to this issue. It is also noted that the `TransferHelper` does not use the `SafeERC20` library, despite its name suggesting otherwise. The bug can be reproduced with the provided sample code, and the error message received is included in the report. The recommendation is to use the `openzeppelin` `SafeERC20` library in `TransferHelper` and any other contract that uses IERC20. The confirmation of the bug was made by talegift (Wild Credit), and the judge, ghoul-sol, commented that this bug could have a medium impact on deposits.

## Details
_Submitted by jonah1005, also found by 0xRajeev, shw, JMukesh, and cmichel_

A non standard ERC20 token would always raise error when calling `_safeTransferFrom`.  If a user creates a USDT/DAI pool and deposit into the pool he would find out there's never a counterpart deposit. See `TransferHelper.sol` [#L19](https://github.com/code-423n4/2021-07-wildcredit/blob/82c48d73fd27a9d4d5d4a395b3affcef4ef6c5c8/contracts/TransferHelper.sol#L19).

`TransferHelper` does not uses `SafeERC20` library as the function name implies.

A sample POC:
```solidity
usdt.functions.approve(lending_pair.address, deposit_amount).transact({'from': w3.eth.accounts[0]})
lending_pair.functions.deposit(w3.eth.accounts[0], usdt.address, deposit_amount).transact({'from': w3.eth.accounts[0]})
```

Error Message:
```solidity
  Error: Transaction reverted: function returned an unexpected amount of data
      at LendingPair._safeTransferFrom (contracts/TransferHelper.sol:20)
      at LendingPair.deposit (contracts/LendingPair.sol:95)
```

Recommend using `openzeppelin` `SafeERC20` in `transferHelper` (and any other contract that uses IERC20).
