# [M-05] No slippage control on `deposit` of `IbbtcVaultZap.sol`

## Summary
The report is about a bug found in a code called `IbbtcVaultZap.sol`. This bug can potentially expose users to a sandwich attack, which means that their deposits can be manipulated. The proof of concept for this bug can be found on line 174 of the code. To mitigate this issue, it is recommended to add a `minOut` parameter in line with the mint function of other contacts and pass it on line 174.

## Details
_Submitted by gzeon, also found by WatchPug_


#### Impact
There is no slippage control on `deposit` of `IbbtcVaultZap.sol`, which expose user to sandwich attack.

#### Proof of Concept
[`IbbtcVaultZap.sol` L174](https://github.com/Badger-Finance/badger-ibbtc-utility-zaps/blob/6f700995129182fec81b772f97abab9977b46026/contracts/IbbtcVaultZap.sol#L174)
Any deposit can be sandwiched, especially when the pool is not balanced.

#### Recommended Mitigation Steps
Add a `minOut` in line with the mint function of other contacts, and pass it as a parameter on L174
