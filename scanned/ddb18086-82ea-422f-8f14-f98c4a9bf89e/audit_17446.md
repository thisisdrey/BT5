# [M-07] processFees() may fail due to exceed gas limit

## Summary
This bug report is about a vulnerability in the `processFees()` function in `JBPayoutRedemptionPaymentTerminal.sol`. This vulnerability is caused by an unbounded loop over the `_heldFeesOf[_projectId]` array. This array can get very large due to the function `_takeFeeFrom()` where it pushes fees that should be paid to a specific beneficiary onto the array. This can cause a DOS condition where no fees can be distributed due to exceed of gas limit. The proof of concept is the code snippet provided in the report.
