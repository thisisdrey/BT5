# Missing Field Element Check and on Curve Point Check for Proof Elements  Acknowledged

## Summary
This bug report describes an issue with the verifier's algorithm in the Plonk paper. The algorithm does not have a field element check or a curve point check for proof elements, which can lead to unintended behavior of the verification contract. It is recommended that a field element, group element, and curve point check be added for proof elements and the verification contract be reverted if the check fails. This will help prevent any unintended behavior of the verification contract.

## Details

Although there is field element check and curve point check in ECCADD, ECCMUL and ECCParing precompiles on those elements, in which the precompile would revert on failed check but it would consume gas on revert and there is no error information.
