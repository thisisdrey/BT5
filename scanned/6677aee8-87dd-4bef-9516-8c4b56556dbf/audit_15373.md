# Frontrun can create alternative NFT markets when price goes up

## Summary
This bug report is about an issue with FantiumNFTV1, a platform that allows upgrading the price of non-fungible tokens (NFTs). The bug occurs when a user calls the updateCollectionPrice() function to increase the price of a collection. An attacker can take advantage of this and buy many NFT tokens at the old price, before the updateCollectionPrice() function is called, and then resell them at a lower price. This allows the attacker to make a profit and leaves the original user with a lower price than expected. 

The issue is difficult to prevent, but one potential solution is to introduce a workflow that pauses minting before calling updateCollectionPrice(). This would give the user time to buy the tokens before the price increase, preventing the attacker from taking advantage of the situation.

## Details
##### Description
FantiumNFTV1 allows upgrading the price of NFTs. When the price for a collection is set higher by calling updateCollectionPrice(), an attacker can front-run this price increase, buy many NFT tokens, and arrange secondary markets for these tokens. The price can be lower than the one set after updateCollectionPrice().

##### Recommendation

The issue is hard to mitigate. One of the options is to introduce workflow when a caller pauses minting before calling updateCollectionPrice().
