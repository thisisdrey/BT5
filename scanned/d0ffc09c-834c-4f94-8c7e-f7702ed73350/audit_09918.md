# Inability To Load Historical Version

## Summary
The bug report describes an issue where the function "LoadVersionAndUpgrade" in the Store module disregards the version argument and loads the latest version instead. This can lead to unintended consequences, especially when the intention is to load a specific historical version. The report suggests modifying the function to load the specified version and ensuring that historical queries or operations work as expected. The issue has been fixed in a patch released by Otter Audits LLC.

## Details
## LoadVersionAndUpgrade

`LoadVersionAndUpgrade` in the store disregards the version argument, loading the latest version instead. This discrepancy may result in unintended consequences, especially when the intention is to load a specific historical version.

> _store v2/root multi/store.gogo_

```go
func (rs *Store) LoadVersionAndUpgrade(version int64, upgrades *types.StoreUpgrades) error {
    [...]
    // deterministic iteration order for upgrades
    sort.Slice(storesKeys, func(i, j int) bool {
        return storesKeys[i].Name() < storesKeys[j].Name()
    })

    initialStores := make([]string, 0, len(storesKeys))
    for _, key := range storesKeys {
        if rs.storesParams[key].typ == types.StoreTypeIAVL {
            initialStores = append(initialStores, key.Name())
        }
    }

    if err := rs.scStore.Initialize(initialStores); err != nil {
        return err
    }
    [...]
}
```

## Remediation

Modify `LoadVersionAndUpgrade` to load the specified version, ensuring that historical queries or operations work as expected.
