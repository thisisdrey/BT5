# Consider replacing the Buffer implementation ✓ Fixed

## Summary
A bug was recently discovered in the 'Buffer' library. The audit team uncovered two bugs in the library, one of which was a critical memory corruption bug. This calls into question the safety of the library and its use in general. There is currently no immediate fix for this, but the client team is working on collaborating to get a better audited version of the 'Buffer' library in place. As a remediation, it is suggested to use a different library, preferably one that has been fully tested and audited and that minimizes the use of inline assembly, particularly around memory allocation.

## Details
#### Resolution



There will be no immediate fix for this, but the client team is working on collaborating to get a better audited `Buffer` library in place.


#### Description


The audit team uncovered two bugs in the `Buffer` library, one each in the only two functions that were looked at. (The library was in general *not* in scope for this audit.) One bug was a critical memory corruption bug. This calls into question how safe this library is to use in general.


#### Remediation


Consider using a different library, ideally one that has been fully tested and audited and that minimizes the use of inline assembly, particularly around memory allocation.
