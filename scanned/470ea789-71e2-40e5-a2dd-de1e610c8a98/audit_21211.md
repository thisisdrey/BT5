# GenesisGroup.commit overwrites previously-committed values

## Summary
A bug was found in the code for the GenesisGroup.sol contract which allowed users to commit purchased FGEN to a swap that would occur once the genesis group was launched. The bug was that the amount stored in the recipient’s committedFGEN balance overwrote any previously-committed value, which also allowed anyone to commit an amount of “0” to any account, deleting their commitment entirely. The bug was addressed in fei-protocol/fei-protocol-core#16 by ensuring that the committed amount is added to the existing commitment.

## Details
#### Resolution



This was addressed in [fei-protocol/fei-protocol-core#16](https://github.com/fei-protocol/fei-protocol-core/pull/16).


#### Description


`commit` allows anyone to commit purchased FGEN to a swap that will occur once the genesis group is launched. This commitment may be performed on behalf of other users, as long as the calling account has sufficient allowance:


**code/contracts/genesis/GenesisGroup.sol:L87-L94**



```
function commit(address from, address to, uint amount) external override onlyGenesisPeriod {
	burnFrom(from, amount);

	committedFGEN[to] = amount;
	totalCommittedFGEN += amount;

	emit Commit(from, to, amount);
}

```
The `amount` stored in the recipient’s `committedFGEN` balance overwrites any previously-committed value. Additionally, this also allows anyone to commit an amount of “0” to any account, deleting their commitment entirely.


#### Recommendation


Ensure the committed amount is added to the existing commitment.
