# Configuration is crucial (both Nomad and Connext)

## Summary
This bug report is about the Connext and Nomad protocols, which rely heavily on configuration parameters. These parameters are set during deployment and can be updated later. If the configuration is incorrect, it can lead to major issues. Examples of important configurations are: s.promiseRouter, s.connextions, s.approvedSequencers, remotes[], home, replicaToDomain[], and domainToReplica[]. The recommendation is to have rigorous controls when configuring and updating these values, to ensure that the configuration is correct.

## Description
The Connext and Nomad protocol rely heavily on configuration parameters. These parameters are configured during deployment time and are updated afterwards. Configuration errors can have major consequences. Examples of important configurations are:

- **BridgeFacet.sol:** 
  - `s.promiseRouter`
  - `s.connextions`
  - `s.approvedSequencers`
  
- **Router.sol:**
  - `remotes[]`
  
- **XAppConnectionManager.sol:**
  - `home`
  - `replicaToDomain[]`
  - `domainToReplica[]`

## Recommendation
Have rigorous controls when configuring and updating these values.
