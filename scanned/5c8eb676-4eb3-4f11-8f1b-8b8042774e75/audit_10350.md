# DevvE_V2 can not be deployed using `upgradeProxy()`

## Summary
This bug report is about a medium severity issue in a contract called Contract DevvE_V2.sol. The problem is that the constructor is not properly checking for a potential issue that could prevent the contract from being deployed using a plugin called OpenZeppelin's Hardhat. The recommendation is to add a specific comment to the constructor in order to fix this issue and allow for smooth deployment. This bug has been resolved and the recommended fix should be implemented.

## Details
**Severity**: Medium

**Status**: Resolved

**Description**

In Contract DevvE_V2.sol, the constructor is disabling the initializer method without adding the following unsafe check which will not allow this contract to be deployed using upgradeProxy() from the OpenZeppelin’s Hardhat plugin.

**Recommendation**: 

Add the following comment to the constructor to upgrade without any issue.

```solidity
/// @custom:oz-upgrades-unsafe-allow constructor
   constructor() {
       _disableInitializers(); 
   }
```
