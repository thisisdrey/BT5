# [M05] Lack of event emission after sensitive actions

## Summary
This bug report is about two functions in the `Option` contract of the Primitive Finance Protocol, which lack event emissions. The `initRedeemToken` function does not emit an event when the redeem token’s address is set, and the `take` function does not emit an event when callers withdraw underlying, strike, redeem, or option tokens. The recommendation is to emit events after sensitive changes take place to facilitate tracking and notify off-chain clients following the contracts’ activity. After the bug report was submitted, the issue was fixed in PR #19, where an event was emitted in the `initRedeemToken` function when the redeem token’s address is initialized. However, the `take` function was later renamed to `withdrawUnusedFunds` and removed from the code base in PR #34.

## Details
In several parts of the code base there are sensitive functions that lack event emissions.


* The [`initRedeemToken` function](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Option.sol#L78) in the `Option` contract does not emit an event when the redeem token’s address is set.
* The [`take` function](https://github.com/primitivefinance/primitive-protocol/blob/78a8e64b7618e9199203ab84042876c580ae1e90/packages/primitive-contracts/contracts/option/primitives/Option.sol#L102) in the `Option` contract does not emit an event when callers withdraw underlying, strike, redeem, or option tokens.


Consider emitting events after sensitive changes take place to facilitate tracking and notify off-chain clients following the contracts’ activity.


**Update:** *Fixed in [PR #19](https://github.com/primitivefinance/primitive-protocol/pull/19). An event is emitted in `initRedeemToken` function when the redeem token’s address is initialized. Note that while the mentioned PR adds an event to the `take` function, the function was later renamed to `withdrawUnusedFunds` and finally removed from the code base in [PR #34](https://github.com/primitivefinance/primitive-protocol/pull/34).*
