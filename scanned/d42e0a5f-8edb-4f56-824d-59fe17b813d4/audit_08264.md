# API - AUTHENTICATED INTERNAL USERS CAN CREATE ADMIN USER

## Summary
The `create_user` function in the `user_router` is used to create new users, but it has a bug that allows an attacker to create a user with admin privileges. This is because the function only requires an authenticated user, without any special permissions. The `user` parameter, which is based on the `UserCreate` schema, includes the option to specify a user role. This means that an attacker can provide any role in the request and create a user with admin privileges. The impact of this bug is rated as 5 out of 10, with a likelihood of 5 out of 10. To fix this issue, the Playground Labs team added a permission checking class in their code, which can be found in the commit `6e0368d70f706452d4bc940c08beec485ebd6093` on their Github repository for the Kapital-Self-Custody-Node project.

## Details
##### Description

Inside the `user_router` within the `create_user` function which is responsible for creating new users, it was noted that the function requires only an authenticated user without any special permissions.
The `user` parameter within `create_user` is based on the `UserCreate` schema which includes `user_role`, the attacker can provide any role within the request and create a user with any role, including the `admin` role.

##### Score

Impact: 5  
Likelihood: 5

##### Recommendation

**SOLVED:** The `Playground Labs team` solved the issue by adding the permission checking class.
