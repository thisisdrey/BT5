# NO FILE VALIDATION ON UPLOAD DOCUMENTS FOR KYC PROCESS

## Summary
The bug report is about a problem where a user was able to upload a non-image file on a website that should only allow certain types of files. This could potentially be used by a malicious person to harm the website or its users. The report includes steps to reproduce the issue and a screenshot of the uploaded file. It also provides a score for the impact and likelihood of the bug, as well as a recommendation for how to fix it. The issue has been resolved by implementing proper validation on a specific file.

## Details
##### Description

The file types allowed to be uploaded should be restricted to only those that are necessary for business functionality. A malicious actor was able to upload non-image files.

1. Navigate to `https://staging.truffles.one`.
2. Create an account.
3. Select an executable .dmg file and upload as a KYC document.
