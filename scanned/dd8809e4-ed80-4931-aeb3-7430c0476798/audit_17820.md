# depositFor() transfers tokens from target account instead of msg.sender.

## Summary
The report describes a bug in the function depositFor() in the Gauge.sol file. The function is supposed to stake tokens for the account that initiated the transaction, but instead, it takes funds from the account itself. This prevents the function from working correctly and can result in the account's funds being deposited without their knowledge. The recommendation is to transfer funds from the sender of the transaction instead of the account. The client has acknowledged the issue and claims that it is intentionally designed this way to deposit the account's balance if they have approved their tokens.

## Details
**Description**

Gauge.sol: function depositFor().
Function is supposed to stake the amount of tokens for “account” by sender of transaction,
however in function _deposit() funds are transferred from “account” instead of msg.sender
which prevents function from correct execution. In case, “account” had approved tokens but
never deposited, his funds will be deposited without his knowing.

**Recommendation**:

Transfer funds from msg.sender instead of “account”.

**Post-audit**.

Client assured, that they know about this functionality and that it is purposely designed to
deposit the balance of account in case, account has approved his tokens.
