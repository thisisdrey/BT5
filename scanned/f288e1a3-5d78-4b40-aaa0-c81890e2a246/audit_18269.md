# approve() Will Revert on Any Credit Account

## Summary
This bug report is about the Gearbox functions passing the creditAccount variable as the borrower parameter of CreditManager.approveCreditAccount() causing calls to CreditFacade.getCreditAccountOrRevert(borrower) to always revert because the creditAccount does not exist. The development team recommended passing msg.sender as the borrower parameter in the CreditManager.approveCreditAccount() function calls. The team then mitigated the issue in multiple commits, 5e35d31 addressed calls from adapters and 7cfef5f addressed calls from CreditFacade.approve(). This bug report is part of Gearbox v2.0 Detailed Findings with the bug code GRB-09.

## Details
## Description
Various Gearbox functions pass the `creditAccount` variable as the `borrower` parameter of `CreditManager.approveCreditAccount()`, causing calls to `CreditFacade.getCreditAccountOrRevert(borrower)` to always revert because the `creditAccount` does not exist.

## Recommendations
Pass `msg.sender` as the `borrower` parameter in the `CreditManager.approveCreditAccount()` function calls.

## Resolution
The development team mitigated this issue in multiple commits:
- `5e35d31` addressed calls from adapters
- `7cfef5f` addressed calls from `CreditFacade.approve()`
