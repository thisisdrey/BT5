# Consensus Key Re-utilization

## Summary
Summary:

This bug report discusses a vulnerability in blockchain networks where previously used consensus keys are not checked when creating a new validator, potentially compromising security. The report suggests updating the CreateValidator function to verify if the provided key has been previously used. The issue has been resolved in a recent patch.

## Details
## Blockchain Security Issue

Blockchain networks often maintain a history of previously utilized consensus keys for validators to prevent the re-utilization of old keys, which may compromise security if compromised or associated with a decommissioned validator. However, `msg_server::CreateValidator` fails to check if the provided consensus key (`msg.Pubkey`) has been utilized historically by any other validator. This omission allows the creation of a new validator through a previously utilized consensus key.

## Remediation

Update `CreateValidator` to verify whether the provided consensus key (`msg.Pubkey`) has been previously utilized.

## Patch

Resolved in PR #20713.
