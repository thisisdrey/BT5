# The native token transfer is not supported by `borgCore.checkTransaction()` in `blacklist` borg mode

## Summary
The bug report states that there is an issue with the `borgCore.checkTransaction()` function in the `blacklist` branch. This function does not check for the size of data being transferred, making it impossible to transfer native tokens in a regular way. Additionally, there is no way to restrict the call value in this branch, unlike in the `whitelist` branch. The recommendation is to add support for native tokens in the `blacklist` branch, similar to the `whitelist` branch.

## Details
##### Description
There is a zero check of calldata size in `borgCore.checkTransaction()` in the `blacklist` branch. So, it's impossible to transfer native tokens in a regular way in this mode. There is also no way to restrict call value similar to what's done in the `whitelist` branch.
