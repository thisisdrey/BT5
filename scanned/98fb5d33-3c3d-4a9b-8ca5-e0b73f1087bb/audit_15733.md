# Fee could be zero for small decimal tokens

## Summary
This bug report is about a problem related to the calculation of fees for payments. The issue is that the fee can be calculated as zero for tokens with small decimals, such as EURS (2 decimals). This can result in small-volume transactions being rounded down to zero and stuck in the contract. An example of this problem is 50*180 EURS being stuck in the contract. 

The report provides two recommendations to address this issue. The first is to restrict the token by the whitelist. The second is to scale the amount to precision (1e4) amounts. It also provides a link to a similar issue in GitHub.
