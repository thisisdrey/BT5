# No security checks for the final epoch calculation

## Summary
The bug report is about a problem in a specific line of code (line 51) in a file called TokenDistribution.sol. The issue is that there is a risk of getting an underflow (a mathematical error) when certain parameters are not handled correctly. This can happen during a calculation called "epoch calculation". The report suggests adding "require" statements or using SafeMath to prevent this error from happening.

## Details
**Description**

TokenDistribution.sol, line 51.
With incorrectly handled parameters there is a possibility to get underflow during the epoch
calculation, since no SafeMath applied in this place nor necessary checks performed.

**Recommendation**:

Add necessary “require” statements to mitigate the case when “currentRewardRate /
rewardReductionPerEpoch_” equals 0 or add SafeMath.
