# Gas overflow during iteration (DoS)

## Summary
This bug report is about a problem that occurs in the 1inch Farming project. The issue is that when more gas is required than allocated to record one block, all iterations of the loop will fail. This affects four lines of code in the ERC20Farmable.sol file. 

The recommendation is to add a check for the maximum possible number of elements of the arrays. This will help to ensure that the loop can continue running without any issues. 

In conclusion, this bug report is about an issue in the 1inch Farming project where more gas is required than allocated to record one block, causing all iterations of the loop to fail. The recommended solution is to add a check for the maximum possible number of elements of the arrays.

## Details
##### Description
Each iteration of the cycle requires a gas flow.
A moment may come when more gas is required than it is allocated to record one block. In this case, all iterations of the loop will fail.
Affected lines:
- https://github.com/1inch/farming/blob/7a007ec7784cca2899889e99e46cf06d5788a7d9/contracts/ERC20Farmable.sol#L70
- https://github.com/1inch/farming/blob/7a007ec7784cca2899889e99e46cf06d5788a7d9/contracts/ERC20Farmable.sol#L117
- https://github.com/1inch/farming/blob/7a007ec7784cca2899889e99e46cf06d5788a7d9/contracts/ERC20Farmable.sol#L121
- https://github.com/1inch/farming/blob/7a007ec7784cca2899889e99e46cf06d5788a7d9/contracts/ERC20Farmable.sol#L137
##### Recommendation
It is recommended to add a check for the maximum possible number of elements of the arrays.
