# Missed `safeApprove`

## Summary
This bug report is about the `Booster` contract, which uses the `approve` method (https://github.com/convex-eth/platform/blob/754d9e700693246275b613e895b4044b63ce9ed5/contracts/contracts/Booster.sol#L288) from the `IERC20` interface. The `approve` method is used to allow a third-party contract to transfer tokens from an account. However, the bug report recommends using the `safeApprove` method instead.

`safeApprove` is a safer alternative to the `approve` method because it allows a user to set a limit on the amount of tokens that can be transferred in a single transaction. It also allows users to set a time limit on how long the approval is valid for. This makes it more secure and prevents malicious contracts from taking advantage of the `approve` method.

In conclusion, it is recommended to use `safeApprove` instead of `approve` when using the `Booster` contract. This will ensure that the tokens are securely transferred and that malicious contracts cannot take advantage of the `approve` method.

## Details
##### Description
`Booster` uses `approve` method (https://github.com/convex-eth/platform/blob/754d9e700693246275b613e895b4044b63ce9ed5/contracts/contracts/Booster.sol#L288):
```solidity
IERC20(token).approve(rewardContract, _amount);
```
It's better to use `safeApprove`.

##### Recommendation
It is recommended to use `safeApprove`.
