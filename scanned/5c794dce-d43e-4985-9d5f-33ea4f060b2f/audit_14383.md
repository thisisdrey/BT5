# M-3: fillQuote uses transfer instead of call which can break with future updates to gas costs

## Summary
This bug report is about an issue in the code of a smart contract called fillQuote, which uses the transfer function instead of the call function. Transfer will always send ETH with a 2300 gas, which can be problematic for interacting smart contracts if gas cost change, as the interaction may abruptly break. This issue was found by 0x52, 0xAgro, Bauer, Breeje, Saeedalipoor01988, martin, sayan_, and tsvetanovv. The code snippet can be seen at https://github.com/sherlock-audit/2023-03-sense/blob/main/sense-v1/pkg/core/src/Periphery.sol#L902-L932. The impact of this issue is that changing gas costs may break integrations in the future. The recommendation is to use call instead of transfer, as reentrancy isn't a concern since the contract should only ever contain the caller's funds. This issue was fixed by changing transfer to call in three occurrences, two in RollerPeriphery and one in Periphery, which are addressed in PR#32 and PR#346, respectively.
