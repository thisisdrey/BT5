# Broker broadcasts sequentially can potentially timeout upstream block building / verification

## Summary
This bug report describes a problem with the mod/async/pkg/broker/broker.go file, where the broadcast function is blocking for each subscriber before sending a message to the next subscriber. This can lead to out of sync services and partially committed states, but is not likely to occur under normal circumstances. The recommendation is to turn it into a goroutine to prevent these issues. The bug has been fixed in PR 2225 and has been verified by Spearbit.

## Details
## Severity: Medium Risk

## Context
`mod/async/pkg/broker/broker.go`

## Description
```go
func (b *Broker[T]) broadcast(msg T) {
    for client := range b.subscriptions {
        // send msg to client (or discard msg after timeout)
        // we could consider using a go routine for each client to allow
        // for concurrent notification attempts, while respecting the timeout
        // for each client individually
        select {
        case client <- msg:
        case <-time.After(b.timeout):
        }
    }
}
```

* `b.timeout` is set to 1 second.

It blocks for each subscriber before sending a message to the next subscriber:
- If the subscriber doesn't accept for whatever reason (channel buffer limit reached, etc...).
- If timed out by broadcast context.
- If timed out by upstream context.

### Consequences
- Potentially out of sync services leading to non-deterministic & faulty behavior.
- Partially committed states.

However, this behavior is likely not to be triggered under normal circumstances but can occur during:
- If CL and EL are disconnected and reconnected.
- If the network is recovering from failure to propose blocks.
- If more events and/or subscribers are added, risk scales non-linearly.
- When performing initial sync to reach head.

## Recommendation
Turn it into a goroutine:
```go
func (b *Broker[T]) broadcast(msg T) {
    if msg.Context().Err() != nil {
        panic(msg.Context().Err())
    }
    for client := range b.subscriptions {
        go func(client chan T) {
            select {
            case client <- msg:
            case <-time.After(b.timeout):
            }
        }(client)
    }
}
```
