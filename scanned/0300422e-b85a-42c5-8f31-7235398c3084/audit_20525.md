# Default max_loss on underlying vault

## Summary
This bug report is about a function called `withdrawFromSUSDVault()` which is found in the line https://github.com/jmonteer/yearnV2-strat-SNX-staking/blob/91b839df4a350d80cb583795bccafe0836fdb732/contracts/Strategy.sol#L512. This function is not specifying the max_loss parameter which can lead to unavailability of withdrawals. It is recommended to implement a function to change the max_loss parameter by the strategist.

## Details
