# [M06] Unnecessary administrator privileges

## Summary
This bug report is about the `BridgePool` contract which is part of the UMAprotocol. This contract inherits from `ExpandedERC20` which allows the contract deployer to mint and burn LP tokens. This power is not required and could unfairly penalize liquidity providers. The solution proposed is to modify `BridgePool` to inherit directly from `ERC20` instead of `ExpandedERC20`. This bug has now been fixed in commit `370e8b21b660543eadbd764fed984a5bdeddce24` in pull request 3492.

## Details
The `BridgePool` contract [inherits from `ExpandedERC20`](https://github.com/UMAprotocol/protocol/blob/f24ad501c8e813cf685f72217e7f13c8f3c366df/packages/core/contracts/insured-bridge/BridgePool.sol#L34) so that it can issue LP tokens to liquidity providers. This inherits the functionality of OpenZeppelin’s `ERC20` contract and also [provides administrator privileges](https://github.com/UMAprotocol/protocol/blob/f24ad501c8e813cf685f72217e7f13c8f3c366df/packages/core/contracts/common/implementation/ExpandedERC20.sol#L36) to the contract deployer, which allows them to mint and burn LP tokens. However, this power is not required and, if exercised, could unfairly penalize liquidity providers.


Consider modifying `BridgePool` to inherit directly from `ERC20` instead of `ExpandedERC20`.
