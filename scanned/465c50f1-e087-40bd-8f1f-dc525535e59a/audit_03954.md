# Users can continue voting for inactive pools through registerVoteWeight() 

## Summary
This bug report describes an issue in the IncentiveVoting.sol code where users are able to vote for inactive receivers. This is due to a function, registerAccountWeight(), not checking if the receiver is active before allowing users to vote. This means that users can continue voting for receivers even if they have been marked as inactive. The impact of this bug is that users may continue voting for inactive receivers if they are switched to inactive after a user has already voted for them. To fix this issue, the recommendation is to check that the receiver is active in all functions used for assigning voting power to receivers.

## Summary
Users can vote for receivers that have been marked as inactive if they have previously voted for this receiver. Users have a few ways to vote in `IncentiveVoting.sol`. They can call:

- **`registerAccountWeightAndVote()`** → Grabs user's current voting weight from `TokenLocker.sol` and registers the weights for the receivers, then stores the user's vote points.
- **`vote()`** → Votes using user's current registered voting weights.
- **`registerAccountWeight()`** → Grabs user's current voting weight from `TokenLocker.sol` and registers the weights for the receivers.

Both `registerAccountWeightAndVote()` and `vote()` check that the receiver of the voting power is an active receiver; however, this is not the case for `registerAccountWeight()`. Because of this, users can continue voting for inactive receivers.

## Finding Description
`IncentiveVoting.sol` contains safeguards to ensure that users cannot vote for inactive receivers in `_storeAccountVotes()`:

```solidity
function _storeAccountVotes(
    address account,
    AccountData storage accountData,
    Vote[] calldata votes,
    uint256 points,
    uint256 offset
) internal {
    // ...
    for (uint256 i; i < votes.length; i++) {
        // prevent voting for disabled receivers
        require(
            IBimaVault(vault).isReceiverActive(votes[i].id),
            "Can't vote for disabled receivers - clearVote first"
        );
        // ...
    }
}
```

However, this function is not called when a user calls `registerVoteWeight()` as it does not call `_storeAccountVotes()`:

```solidity
function registerAccountWeight(address account, uint256 minWeeks) 
    external callerOrDelegated(account) {
    // get storage reference to account's lock data
    AccountData storage accountData = accountLockData[account];
    Vote[] memory existingVotes;
    
    // if account has an active vote, clear the recorded vote weights prior to updating the registered account weights
    if (accountData.voteLength > 0) {
        existingVotes = getAccountCurrentVotes(account);
        _removeVoteWeights(account, existingVotes, accountData.frozenWeight);
        emit ClearedVotes(account, getWeek());
    }

    // get updated account lock weights and store locally
    uint256 frozenWeight = _registerAccountWeight(account, minWeeks);
    
    // resubmit the account's active vote using the newly registered weights
    _addVoteWeights(account, existingVotes, frozenWeight);
    // do not call `_storeAccountVotes` because the vote is unchanged
}
```

## Impact Explanation
Users may continue voting for inactive receivers if they are switched to inactive after a user has already voted for them.

function test_UserCanKeepVotingForInactiveReceiver() external {

    vm.prank(users.user1);
    incentiveVoting.registerAccountWeightAndVote(users.user1, 10, votes);
    for (uint16 week = startWeek; week <= startWeek + 10; week++) {
        uint256 receiverWeight = incentiveVoting.getReceiverWeightAt(RECEIVER_ID, week);
        console.log("Weight at Week %s: %s", week, receiverWeight);

    _lockMoreTokens(INIT_BAB_TKN_TOTAL_SUPPLY / 2 / 2);
    // User registers weights again
    vm.prank(users.user1);
    incentiveVoting.registerAccountWeight(users.user1, 10);
    // warp time by input
    vm.warp(block.timestamp + 1 weeks * 10);
    // cache state prior to allocateNewEmissions
    uint16 finalWeek = SafeCast.toUint16(bimaVault.getWeek());

        console.log("Weight at Week %s: %s", week, receiverWeight);
    }

## Recommendation
Check that the receiver is active in all functions used for assigning voting power to receivers.
