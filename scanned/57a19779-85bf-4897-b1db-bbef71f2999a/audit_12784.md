# Token amounts are in wrong decimals if useMax parameter is used

## Summary
This bug report is about the allocate and swap functions in Portfolio.sol. The useMax parameter sets the token amounts to be used, but these amounts are in token decimals, while the code that follows expects them to be in WAD units. This can lead to user loss as the tokens remain in the contract after the action. The recommendation is to scale these amounts to WAD units and add tests for both scenarios. The bug has been fixed in PR 387 and is now marked as fixed.

## Details
## Severity: High Risk

## Context
- Portfolio.sol#L227-L237
- Portfolio.sol#L444-L448

## Description
The `allocate` and `swap` functions have a `useMax` parameter that sets the token amounts to be used to the net balance of the contract. This net balance is the return value of a `getNetBalance` call, which is in token decimals. The code that follows (`getPoolMaxLiquidity` for `allocate`, `iteration.input` for `swap`) expects these amounts to be in WAD units.

Using this parameter with tokens that don't have 18 decimals does not work correctly. The actual tokens used will be far lower than the expected amount to be used, which will lead to user loss as the tokens remain in the contract after the action.

## Recommendation
Scale these amounts to WAD units. Add tests for both scenarios.

## Primitive
Fixed in PR 387.

Fixed.
