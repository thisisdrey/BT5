# BRF-2 | Unbounded disRate

## Summary
This bug report describes an issue with the `setDisRate` and `distribute` functions. The `disRate` variable can be set too high, which causes the distributed amount to exceed the provided `amount`. The recommendation is to implement a maximum cap for `disRate`, with a suggested cap of 57%. The resolution proposed is for the Bridges Team to set a disRate cap of 1000.

## Details
**Description**

There is no bound to how high the `disRate` variable can be set in the `setDisRate` function. Therefore, the distributed amount in the `distribute` function may exceed the provided `amount`.

**Recommendation**

Implement a maximum cap for `disRate`. A cap of ~ 57% (1/1.75) would allow for a maximum distribution of `amount`.

**Resolution**

Bridges Team:

- Set a disRate cap of 1000.
