# Possible transfer of 0 funds

## Summary
The bug report is about a problem with a code that is used for liquidation. The issue is that the variable "remainingFunds" can sometimes have a value of zero, which can cause problems. The recommendation is to add a check to make sure that "remainingFunds" is not equal to zero before transferring it.

## Details
##### Description
For liquidation `remainingFunds` can be equal to zero:
https://github.com/Gearbox-protocol/gearbox-contracts/blob/0ac33ba87212ce056ac6b6357ad74161d417158a/contracts/credit/CreditManager.sol#L434
##### Recommendation
We recommend adding a check that `remainingFunds != 0` before transfer.
