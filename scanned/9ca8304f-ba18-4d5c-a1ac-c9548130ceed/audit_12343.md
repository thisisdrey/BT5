# Possible re-entrance

## Summary
This bug report is about a contract called SakeVault that has a function called `withdrawVlp` which is causing some problems. The function has callbacks that could potentially cause issues when other functions are used, specifically `closePosition` and `liquidatePosition`. The bug has been marked as resolved, but the recommendation is to make sure that `withdrawVlp` does not call any external sources or to protect `closePosition` and `liquidatePosition` from re-entrance attacks.

## Details
**Severity**: High

**Status**: Resolved

**Description**

In the contract SakeVault there is a `velaStakingVault` with a function `withdrawVlp` which has a not introduced functionality. In this case, this function has callbacks, that could potentially lead to re-entrance issues in all functions when it used: closePosition and liquidatePosition.

**Recommendation**: 

make sure `withdrawVlp` does not call any external sources or protect `closePosition` and `liquidatePosition` from re-entrance attacks
