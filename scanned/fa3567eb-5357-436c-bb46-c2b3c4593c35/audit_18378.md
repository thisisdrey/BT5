# It is possible to pollute the observations array Status: Undetermined

## Summary
This bug report is regarding the CumulativeTwap.sol, which is part of the Chainlink and Band price feeds. In the new versions of the code, it is possible to pollute the observations array with a single observation because the strict equality in the require's condition changed to a loose equality. The issue was not considered during the retest because the vulnerability is relevant only for a newer version of the code that was not part of the original audit. The statuses used to indicate whether an issue has been sufficiently addressed are Undetermined, Unresolved, Partially Resolved, and Resolved.

## Details
## Target: CumulativeTwap.sol

## Description

In new versions of the Chainlink and Band price feeds, it is possible to pollute the observations array with a single observation because the strict equality in `require`’s condition was changed to a loose equality, as shown in figure 9.1.

```solidity
// add `==` in the require statement in case that two or more price with the same timestamp
// this might happen on Optimism bcs their timestamp is not up-to-date
Observation memory lastObservation = observations[currentObservationIndex];
require (lastUpdatedTimestamp >= lastObservation.timestamp, "CT_IT");
```

**Figure 9.1:** The insecure `require` condition in the `_update` method in the new version of the code (perp-oracle/contracts/CumulativeTwap.sol#43–46).

## Fix Analysis

The finding was not considered during the retest because the vulnerability is relevant only for a newer version of the code that was not part of the original audit.
