# Staking API service can be unavailable due to continuation ﬂood vulnerability in net/http 

## Summary
The report states that the Staking API service is using an older version of Go (1.21.6) which has a known vulnerability in the net/http package. This vulnerability can be checked using the govulncheck tool, which shows that the vulnerability is present in the golang.org/x/net module at version v0.22.0 but has been fixed in version v0.23.0. The report also provides a list of example traces where the vulnerability can be found in the code. The recommendation is to update to the latest version of Go and implement govulncheck in the Github workflows to be alerted of any known vulnerabilities. A specific Github workflow is suggested to be added to the Babylon repository to fix the vulnerability in the staking-api-service commit c2d28281.

## Description
The Staking API service uses Go in version `1.21.6` as configured in the `go.mod` file. However, for this Go version, a vulnerability is known in the `net/http` package. More information about this vulnerability can be found [here](https://pkg.go.dev/vuln/GO-2024-2687).

## Proof of Concept
The `govulncheck` tool shows the vulnerability:

1. Install `govulncheck` if not already done:
   ```bash
   go install golang.org/x/vuln/cmd/govulncheck@latest
   ```
2. Run `govulncheck` on the `api` directory:
   ```bash
   govulncheck ./internal/api/
   ```
3. It can also be run on the compiled binary:
   ```bash
   govulncheck -mode binary ./build/staking-api-service
   ```
