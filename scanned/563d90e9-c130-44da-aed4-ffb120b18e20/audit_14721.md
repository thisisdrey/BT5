# M-8: transferFrom uses allowance even if spender == from

## Summary
This bug report is about an issue found in the code of the Pool#transferFrom method in the Surge Protocol. The bug is that the transferFrom method attempts to use allowance even when the spender and the from are the same. This breaks the compatibility with protocols that use the transferFrom method in order to pull only instead of using both transfer and transferFrom. The ERC20 standard only does an allowance check when the spender and from are not the same. This difference can result in tokens becoming irreversibly stranded across different protocols. 

The code snippet provided shows the transferFrom method which always uses allowance even if the spender and from are the same. The impact of this issue is that tokens won't be compatible with some protocols and will end up stranded. 

The recommendation provided is to only use allowance when the spender is not the same as from. The fix provided looks good and the allowance check is bypassed if the msg.sender is the same as from.
