# H-21: Creating an order of type MarketIncrease opens an attack vector where attacker can execute txs with stale prices by inputting a very extense swapPath

## Summary
This bug report is about a vulnerability found in the create order function of GMX Synthetics. Specifically, when creating an order of type MarketIncrease, an attacker can execute transactions with stale prices by inputting a very extensive swapPath in the params. The way to achieve this is by creating a market increase order and passing a very extensive swapPath in the params, which has to be as long as it gets close to the gasLimit of the block. Then, by using the callback contract that was passed as a param, the attacker can exceed the block.gasLimit in the callback. After a certain amount of blocks, the attacker can then change the gasUsage on the fallback, allowing the transaction to execute at the prior price. A proof of concept on how to execute the transaction with old pricing was also provided.

The impact of this vulnerability is that it would allow the attacker to make free trades in terms of risk. The attacker would be able to trade without any risk by controlling when to execute the transaction. A code snippet of the vulnerability was also provided, as well as the tool used to find it, which was manual review.

The recommendation provided to fix this vulnerability is to have a way to cap the length of the path to control user input. The discussion section of the bug report also includes escalations and rejections of the issue. Finally, a fix was provided in the form of a commit.
