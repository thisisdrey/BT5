# [M-03] NFTs can get locked after the call to `releaseOnEid`

## Summary
The bug report explains that there is a problem with a function called `Beacon.releaseOnEid()` which allows users to lock and unlock tokens between two chains. If the amount of tokens released is too high, it can cause all the gas in the target chain to be used up, resulting in the tokens being locked forever. This is a serious issue and it is recommended to limit the number of tokens that can be released at once to prevent this from happening.

## Details
## Severity

**Impact:** High

**Likelihood:** Low

## Description

Users can call `Beacon.releaseOnEid()` to lock tokens in the source chain and unlock them in the target chain. If the amount of tokens released is too high, it is possible that all the gas of the block is consumed in the target chain when unlocking the tokens. This would cause the tokens to be locked forever.

## Recommendations

Limit the number of tokens that can be released at once.
