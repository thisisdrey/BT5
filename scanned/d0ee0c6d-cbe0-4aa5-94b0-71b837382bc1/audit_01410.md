# No rate limiting for (s *DispersalServer) RetrieveBlob() API requests

## Summary
The bug report discusses a problem with a method called RetrieveBlob in disperser/apiserver/server.go file. The issue is that this method does not have rate limiting, which means it can cause problems with shared resources and affect the proper functioning of the disperser. The recommendation is to implement rate limiting for this method to avoid any potential issues.

## Details
## Medium Risk Alert

**Severity:** Medium Risk  
**Context:** disperser/apiserver/server.go#L530  
**Description:**  
The method `(s *DispersalServer) RetrieveBlob()` does not enforce rate limiting. This request holds mutexes on blob metadata, will cause error logging when requests fail, and makes database lookups. Grieving these shared resources could be detrimental to proper disperser operation.

**Recommendation:**  
Implement rate limiting for the RetrieveBlob API call.
