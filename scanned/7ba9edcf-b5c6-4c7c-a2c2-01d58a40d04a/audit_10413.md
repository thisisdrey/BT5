# No Price Staleness Check

## Summary
The bug report describes a medium severity issue that has been resolved. The function `latestRoundData()` in the ERC20PriceOracle_V2 contract is used to get the price of an asset, but there are no checks to ensure that the price is up-to-date. The recommendation is to add checks for price staleness by using the `latestRoundData()` function and requiring that the `answeredInRound` value is greater than or equal to the `roundID` and that the `timestamp` is not equal to 0. This will help prevent stale prices from being used.

## Details
**Severity** - Medium

**Status** - Resolved

**Description**

The function `latestRoundData()` has been used inside ERC20PriceOracle_V2 (getChainlinkPrice) to fetch the price of an asset , but there are no price staleness checks.



**Recommendation**:

Introduce price staleness checks as follows →
```solidity
(uint80 roundID, int256 answer, , uint256 timestamp, uint80 answeredInRound) = tokenInfos[token].chainlinkOracle.latestRoundData();
require(answeredInRound >= roundID, "Stale price");
require(timestamp != 0,"Round not complete");
```
