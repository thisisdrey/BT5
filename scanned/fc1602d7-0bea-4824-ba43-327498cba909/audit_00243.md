# [M-01] Interface mismatch in `IYuzuILPV2` and `IPSM` can lead to call revert

## Summary
The bug report discusses two issues in the code for `IYuzuILPV2.sol` and `IPSM.sol`. 

The first issue is that the `terminateDistribution()` function in `IYuzuILPV2.sol` has a parameter for the `receiver` address, but the actual function call in the code does not include this parameter. This can cause problems for the pool manager when trying to use the `IYuzuILPV2` interface. 

The second issue is that the `initialize()` function in `IPSM.sol` is missing the `_minRedeemOrder` parameter, which is present in the `PSM.sol` code. 

To fix these issues, it is recommended to remove the `receiver` parameter from `IYuzuILPV2::terminateDistribution()` and add the `_minRedeemOrder` parameter in `IPSM::initialize()`.

It is recommended to remove the `receiver` parameter from `IYuzuILPV2::terminateDistribution()` and add the `_minRedeemOrder` parameter in `IPSM::initialize()`.
