# Eliminate assembly code by using ABI decode ✓ Fixed

## Summary
This bug report focuses on the use of assembly code in several locations to access and decode byte arrays. This code is used for gas optimization, but it reduces the readability and future updatability of the code. In order to resolve this issue, it was suggested to use ABI decode instead of assembly code since it can access 32-byte chunks of data from user input. This should make the code significantly more clean, as well as reduce the size of flags and reduce the complexity of Merkle Root verifications/calculation code.

## Details
#### Resolution



All assembly code was replaced with proper use of `abi.decode()`.


#### Description


There are several locations where assembly code is used to access and decode byte arrays (including uses inside loops).
Even though assembly code was used for gas optimization, it reduces the readability (and future updatability) of the code.

#### Recommendation


As discussed in the mid-audit meeting, it is a good solution to use ABI decode since all uses of assembly simply access 32-byte chunks of data from user input. This should eliminate all assembly code and make the code significantly more clean.
In addition, it might allow for more compact encoding in some cases (for instance, by eliminating or reducing the size of the flags).


This suggestion can be also applied to Merkle Root verifications/calculation code, which can reduce the for loops and complexity of these functions.
