# Missing Access Control on Token Release Functions

## Summary
The bug report discusses an issue with the `releaseTokens` and `batchRelease` functions in a smart contract. These functions do not have proper access control, which means that anyone can release tokens. This goes against the intended purpose of the functions, as stated in the documentation. The report recommends adding appropriate access control to these functions to prevent unauthorized token releases. The BlockDAG team has already solved this issue by adding the `onlyRole(ADMIN_ROLE)` modifier to the functions and ensuring that only authorized admins can trigger token releases. They have also maintained existing checks for nonReentrancy and pausing, aligning with the requirement for admin-only releases. This issue has been resolved in version 3 of the contract.

## Details
##### Description

The `releaseTokens` and `batchRelease` functions lack access control, allowing anyone to release tokens:

```
function releaseTokens(
        address user,
        bytes32 category
    ) external nonReentrant whenNotPaused returns (uint256)

function batchRelease(
        bytes32 category,
        address[] calldata users
    ) external nonReentrant whenNotPaused
```

  

This implementation does not follow the intention as stated in the provided documentation:

> **3.3 Admin-Only Release**
>
> * The admin triggers batch release for each category after verifying the correct vesting stage has arrived.

##### Recommendation

Add appropriate access control to both release functions.

##### Remediation

**SOLVED**: The **BlockDAG team** solved this issue as follows:

* Adds the `onlyRole(ADMIN_ROLE)` modifier to both functions
* Ensures only authorized admins can trigger token releases
* Maintains existing nonReentrant and pause checks
* Aligns with the requirement for admin-only releases
