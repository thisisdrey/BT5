# [M-26] `StakingRewards.setRewardsDuration` allows setting near zero or enormous `rewardsDuration`, which breaks reward logic

## Summary
This bug report is about a vulnerability in the StakingRewards.sol contract in the code-423n4/2022-02-concur Github repository. The vulnerability is that the notifyRewardAmount function will be inoperable if rewardsDuration is set to zero, or if it is set to too small or too big. This is because the setter does not control the value, and division by the duration is used in the notifyRewardAmount function. To mitigate this vulnerability, it is recommended to check for min and max range in the rewardsDuration setter, as too small or too big rewardsDuration will break the logic.

## Details
_Submitted by hyh_

notifyRewardAmount will be inoperable if rewardsDuration is set to zero. If will cease to produce meaningful results if rewardsDuration is too small or too big.

### Proof of Concept

The setter does not control the value, allowing zero/near zero/enormous duration:

[StakingRewards.sol#L178-185](https://github.com/code-423n4/2022-02-concur/blob/main/contracts/StakingRewards.sol#L178-185)

Division by the duration is used in notifyRewardAmount:

[StakingRewards.sol#L143-156](https://github.com/code-423n4/2022-02-concur/blob/main/contracts/StakingRewards.sol#L143-156)

### Recommended Mitigation Steps

Check for min and max range in the rewardsDuration setter, as too small or too big rewardsDuration breaks the logic.
