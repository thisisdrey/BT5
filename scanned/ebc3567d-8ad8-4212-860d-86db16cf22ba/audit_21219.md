# Safe math library isn't used

## Summary
This bug report is about an arithmetic overflow issue in two contracts, Keep3rV1Oracle.sol and Keep3rV1Volatility.sol. These contracts are found in the GitHub repository https://github.com/iearn-finance/yoracle.link/blob/faf1309cbe7a05f70b338351315039eb8e5b9c09/contracts/. The issue is that if the safe math library is not used, an arithmetic overflow may occur which could lead to incorrect operation of the smart contracts.

To resolve this issue, the recommendation is to use the safe math library. This library is already present in the contracts. This library is used to perform arithmetic operations safely and prevent any overflow issues. It is important to note that the safe math library should be used for all arithmetic operations in the contracts in order to ensure that they are performed correctly.
