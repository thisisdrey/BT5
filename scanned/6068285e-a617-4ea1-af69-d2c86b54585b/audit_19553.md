# [M-06] `calcMint` always return poolId=0 and idx=0

## Summary
The bug report states that the function `calcMint` in the file Zap.sol always returns a specific value, regardless of the input it receives. This is different from what is stated in the documentation, which says that it should return the most optimal route. This bug can lead to suboptimal results when using the function. The report provides a link to the specific line of code where the bug can be found. A comment from a member of the team confirms the validity of the finding and mentions that there are two missing functions that could potentially fix the issue. However, the team has shifted to only using a specific pool, so the bug does not affect them.

## Details
_Submitted by gzeon_

#### Impact
`calcMint` in Zap.sol always return poolId=0 and idx=0, while the docstring specified it should return the most optimal route instead. This will lead to suboptimal zap.
