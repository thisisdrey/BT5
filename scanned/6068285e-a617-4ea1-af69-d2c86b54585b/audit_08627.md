# Incorrect DeleteAll Implementation

## Summary
The bug report discusses an issue with the DeleteAll function in the KVStore, which is used to delete all state data for EVM contracts. The original implementation only clears keys in the "dirtyItems" set, which means it may miss other keys not touched in the same transaction. This can result in incorrect destruction of EVM contracts. The suggested fix is to have DeleteAll iterate over the entire range instead of relying on dirtyItems. The issue has been fixed in a recent pull request.

## Details
## DeleteAll Implementation in KVStore

`DeleteAll` is added to `KVStore` to support deletion of all EVM contract states on contract destruct. However, the original implementation for `cachekv` only clears keys in the `dirtyItems` set, thus misses any other keys not touched within the same transaction.

## Code Snippet

> _store/cach e kv/store.gogo_

```go
func (store *Store) DeleteAll(start, end []byte) error {
    store.dirtyItems(start, end)
    // memdb iterator
    cachedIter, err := store.sortedCache.Iterator(start, end)
    if err != nil {
        return err
    }
    defer cachedIter.Close()
    for ; cachedIter.Valid(); cachedIter.Next() {
        // `Delete` would not touch sortedCache so it's okay to perform inside iterator
        store.Delete(cachedIter.Key())
    }
    return nil
}
```

This may result in incorrect EVM contract destruction.

## Remediation

Have `DeleteAll` iterate over the entire range and not rely on `dirtyItems` to provide the key-value set.

## Patch

Fixed in PR #464.
