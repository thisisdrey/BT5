# Unsafe ERC20 operation(s)

## Summary
The current contracts are not handling certain types of ERC20 tokens correctly when used for staking. This means that there is a risk of errors happening without being detected. An example of this is the LDO token, which does not give an error message when there is not enough balance for a transfer. To fix this issue, the Entangle team has implemented safeApprove and safeTransfer methods. This can be seen in the remediation plan link provided.

## Details
##### Description

Currently, the contracts does not adequately handle atypical ERC20 tokens on the staking. According to the `ERC20` specification, tokens should return "false" when a transfer fails, but it does not guarantee that the function will revert. This discrepancy could potentially lead to silent failures, making it difficult to detect issues when they occur.
