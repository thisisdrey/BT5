# Division before multiplication can cause round down issue. Almost in all the places,it is happening

## Summary
This bug report is about an issue with a computation process, where the order of operations was not performed correctly. The computation was done with division first and then multiplication, instead of multiplication first and then division. This caused incorrect results.

The recommendation was to perform multiplication first and then do divide to ensure the correct results. This issue has now been resolved.

## Details
**Description:** 

Computation is done such that the division first and then multiplication. 

**Recommendations:** 

Perform multiplication first and then do divide.

**Status:** Resolved
