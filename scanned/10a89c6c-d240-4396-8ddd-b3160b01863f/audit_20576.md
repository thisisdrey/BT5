# [H-04] ERC20ConvictionScore’s governanceDelta should be subtracted when user is not a governor anymore

## Summary
A bug has been reported in the `ERC20ConvictionScore._updateConvictionScore` contract, which is part of the `TOTAL_GOVERNANCE_SCORE` system. This system is used to track the sum of the credit scores of all governors. The bug causes the `governanceDelta` to be positive when a user does not fulfill the governance criteria anymore and should be removed. This incorrect data leads to issues throughout all contracts, such as wrong `FairSideDAO.totalVotes` data, which can be used to pass proposals in the worst case, or can be arbitrarily inflated and break the voting mechanism.

The recommended mitigation step is to return a negative, signed integer for this case and add it to the new total. This will ensure that the `TOTAL_GOVERNANCE_SCORE` system is tracking the correct data and prevent any further issues.

## Vulnerability Details
The `TOTAL_GOVERNANCE_SCORE` is supposed to track the sum of the credit scores of all governors.

In `ERC20ConvictionScore._updateConvictionScore`, when the user does not fulfill the governance criteria anymore and is therefore removed, the `governanceDelta` should be negative but it's positive.

```solidity
isGovernance[user] = false;
governanceDelta = getPriorConvictionScore(
    user,
    block.number - 1
);
```

It then gets added to the new total:

```solidity
uint224 totalGCSNew =
    add224(
        totalGCSOld,
        governanceDelta,
        "ERC20ConvictionScore::_updateConvictionTotals: conviction score amount overflows"
    );
```

## Impact
The `TOTAL_GOVERNANCE_SCORE` tracks wrong data leading to issues throughout all contracts like wrong `FairSideDAO.totalVotes` data which can then be used for anyone to pass proposals in the worst case.
Or `totalVotes` can be arbitrarily inflated and break the voting mechanism as no proposals can reach the quorum (percentage of `totalVotes`) anymore.

## Recommended Mitigation Steps
Return a negative, signed integer for this case and add it to the new total.
