# The rewardsAdded calculation in extendRewardsInterval is incorrect 

## Summary
This bug report is about a calculation error in a function called extendRewardsInterval. The calculation for rewardsAdded and the new rate is incorrect, which results in the vault pulling less funds from the owner than intended. This is because the current implementation assumes that two time parameters are equal, when they are actually different. The recommendation is to use a different calculation instead, which has been fixed in a recent commit.

## Description
The `rewardsAdded` calculation in `extendRewardsInterval` is incorrect. The `rewardsAdded` and the new rate are calculated as below in `extendRewardsInterval`:

```solidity
uint256 remainingRewards = rewardsInterval.rate * (rewardsInterval.end - newStart);
uint256 rate = (rewardsAdded - frontendFeeTaken - protocolFeeTaken + remainingRewards) / (newEnd - newStart);
rewardsAdded = (rate - rewardsInterval.rate) * (newEnd - newStart) + frontendFeeTaken + protocolFeeTaken;
```

We have:
- \( ar = rold(te,0 - ts,1) \)
- \( rnew = \frac{a0 + ar - f}{te,1 - ts,1} \)

Thus, \( a1 \) should have been calculated as:
- \( a0 \geq a1 = rnew(te,1 - ts,1) - ar + f \)
- or \( a1 = rnew(te,1 - ts,1) - rold(te,0 - ts,1) + f \)

But the current implementation incorrectly calculates \( a1 \) as:
- \( a1,incorrect = (rnew - rold)(te,1 - ts,1) + f \)

This assumes that \( te,1 \) and \( te,0 \) are equal. The effect would be that the vault pulls less funds from the owner of the vault, which would not match the newly set rate. The discrepancy is as follows:
- \( a1 - a1,incorrect = rold(te,1 - te,0) = rold \Delta te \)

## Parameter Description
- `rold`: rewardsInterval.rate
- `rnew`: rate
- `ar`: remainingRewards
- `a0`: rewardsAdded before update
- `a1`: rewardsAdded after update
- `f`: frontendFeeTaken + protocolFeeTaken
- `ts,0`: rewardsInterval.start before update
- `ts,1`: newStart
- `te,0`: rewardsInterval.end
- `te,1`: newEnd

## Recommendation
Use the following calculation instead:
```solidity
rewardsAdded = rate * (newEnd - newStart) - remainingRewards + frontendFeeTaken + protocolFeeTaken;
```
