# Continuous integration builds are not enabled on all branches

## Summary
The Picasso GitHub repository lacks proper continuous integration checks, which can lead to code being merged into protected branches that break the build. This was demonstrated when an error was encountered when running cargo-build due to a WASM target error and a use of undeclared crate or module. 

The exploit scenario involves a Composable Finance developer, Alice, adding new features to the Picasso runtime that break the build. This is possible because continuous integration checks are not run when code is merged. 

In order to prevent this from happening in the future, short term, checks should be added to verify that all builds on the development and main branches are successful. Long term, continuous integration pipelines should be implemented on all branches with sufficient build and runtime checks.

## Exploit Scenario

Alice, a Composable Finance developer, adds new features to the Picasso runtime. The changes break the build. However, because continuous integration checks are not run when code is merged, the build failure is not detected.

## Recommendations

- **Short term:** Add checks to verify that all builds on development and main branches are successful. These checks will ensure that code will not be merged into either branch unless all builds are successful.

- **Long term:** Always add sufficient build and runtime checks to continuous integration pipelines on all branches.
