# Incorrect Pool Balance Calculation

## Summary
The bug report is about an issue in the process_donation function within the donations code. There is a problem with the order of operations when updating the self.matching_pool_balance. The remainder variable should be the remaining amount after deducting fees and storage costs from the total donation amount. However, subtracting the storage cost occurs after adding it to self.matching_pool_balance, resulting in incorrect accounting for total matching pool donations. This requires the admin to fund the difference before being able to distribute payouts. The suggested solution is to subtract the required_deposit from the remainder before updating the self.matching_pool_balance. The bug has been resolved in version e8ef580.

## Details
## Issue in process_donation Function

In `process_donation` within donations, there is an issue in the order of operations when updating the `self.matching_pool_balance`.

> _pot/src/donations.rs

```rust
pub(crate) fn process_donation(
    [...]
) -> DonationExternal {
    [...]
    if matching_pool {
        [...]
        self.matching_pool_balance =
            self.matching_pool_balance
                .checked_add(remainder)
                .expect(&format!(
                    "Overflow occurred when calculating self.matching_pool_balance ({} + {})",
                    self.matching_pool_balance, remainder,
                ));
    }
    [...]
    remainder = remainder.checked_sub(required_deposit).expect(&format!(
        "Overflow occurred when calculating remainder ({} - {})",
        remainder, required_deposit,
    ));
    [...]
}
```

`remainder` should be the remaining amount after deducting fees and storage costs from the total donation amount. However, subtracting the storage cost (`required_deposit`) occurs after adding it to `self.matching_pool_balance`. This would result in incorrect accounting for total matching pool donations and require the admin to fund this difference before being able to distribute payouts.

## Remediation

Subtract `required_deposit` from `remainder` before updating `self.matching_pool_balance`.

### Patch

Resolved in `e8ef580`.
