# userOp validation is skipped in simulation mode for smart contract user accounts

## Summary
This bug report is about a high-risk issue in a specific part of the code (AtlasVerification.sol#L551). The problem occurs when a user has not given permission for a certain action (validateUserOp() returns false), but during testing, it is mistakenly assumed that the user has given permission because the simulation mode is on. This can lead to wasted gas for the bundler if the action is executed on the blockchain. The recommendation is to check for user approval even in simulation mode to avoid this issue. The bug has been solved by two pull requests (PR194 and PR250) and has been verified by Spearbit.

## Details
## Severity: High Risk

**Context:** `AtlasVerification.sol#L551`

**Description:**  
It can happen that the user has not allowed this `userOp` (i.e., returning `validateUserOp()` returns false),  
but during simulation, it's falsely believed that the user has allowed it since `isSimulation` is true:

```solidity
bool validSmartWallet =
IAccount(userOp.from).validateUserOp{ gas: 30_000 }(userOp, _getProofHash(userOp), 0) == 0;
return (isSimulation || validSmartWallet);
```

If this `userOp` goes on-chain, it leads to a revert, wasting gas for the bundler.

**Recommendation:**  
Check that the smart contract user account has approved `userOp` even in simulation mode:  

```solidity
- return (isSimulation || validSmartWallet);
+ return (validSmartWallet);
```

**Fastlane:** Solved by PR194 and PR 250.  
**Spearbit:** Verified.
