# LACK OF END TIME VALIDATION LEADS TO WRONG MARKET INDEX CALCULATION ON THE NEW MARKETS

## Summary
The report describes a bug in the `_addEmissionConfig` function of the Moonwell Finance system. This function is responsible for creating new market emission configurations. The bug is that there is no validation check for the `_endTime` parameter, which can result in the creation of markets with incorrect or unreasonable end times. This can lead to wrong market index calculations and potentially impact the overall functioning of the system. The report includes the code location of the function and a proof of concept of the bug. The report also provides a recommendation to add a validation check for the `_endTime` parameter and a remediation plan stating that the Moonwell Finance team has solved the issue by adding the validation check.

## Details
##### Description

In the `_addEmissionConfig` function, which is responsible for creating new market emission configurations, there is no validation check for the `_endTime` parameter. This oversight may lead to the creation of markets with incorrect or unreasonable end times, resulting in wrong market index calculations and potentially impacting the overall functioning of the system.
