# Hardcoded or Fixed Slippage can lead to Loss of User funds

## Summary
This bug report is about a problem in the `GammaSwapLiquidityWarehouse.sol` contract. The hardcoded Slippage values in the contract can cause users to lose their funds when the token's volatility is high. The function on line 102 only allows the admin to change the slippage, not the users. This means that the admin can change the slippage to a very high or low value without the users knowing, leading to potential loss of funds. The recommendation is to allow users to override the slippage value and use a multisig wallet to prevent any malicious changes to the slippage. More information about this issue can be found at https://dacian.me/defi-slippage-attacks#heading-hard-coded-slippage-may-freeze-user-funds.

## Details
**Severity**: Medium	

**Status**: Acknowledged

**Description**

In the `GammaSwapLiquidityWarehouse.sol` contract the hardcoded Slippage values is not adivsed as it can lead to loss of user funds when the volatility of token is high.
Line: 48 and 97. The function on line: 102 only allows the admin to change the slippage and not the users.

In addition to this, the admin can change the slippage to a very high or low value without user’s notice which could lead to loss of user funds.

**Recommendation**: 

Consider allowing mechanism for user to override the existing slippage value so that he can override it if he wants. And use a multisig wallet to ensure that the ownership of the contract is sufficiently decnetralized to prevent loss of private keys or any malicious changes to the slippage.
