# Credit Manager Entries Can Be Overwritten

## Summary
A bug has been found in the `addCreditManager()` function of the `vaults/AbstractVault.sol` file. This function allows the owner to add new credit managers, but there is a flaw that can be exploited. The `require()` statement meant to prevent adding the same credit manager twice can be bypassed, allowing the owner to overwrite the credit manager's share locker and lose access to it. This can happen if the owner first adds an address as a credit manager, then uses that address to carry out staking and withdrawal activities, and finally calls `forbidCreditManagersCanRepay()` and `addCreditManager()`. To prevent this, a loop should be added to check if the passed address already exists in the list of credit managers.

## Details
**Update**
It is no longer possible to re-add the same credit manager due to the additional check `creditManagersShareLocker[_creditManager] == address(0)`.

**File(s) affected:**`vaults/AbstractVault.sol`

**Description:** The `addCreditManager()` function allows the owner to add new credit managers. The function contains a `require()` statement to make sure the passed address doesn't already exist as a credit manager. However, the statement can be circumvented and the credit manager information can be overwritten. This allows the owner to overwrite the credit manager's share locker, effectively losing all access to it.

**Exploit Scenario:**

1.   The owner adds X (an address) as a credit manager.
2.   X carries out a bunch of staking and withdrawal activities on their share locker through calling `borrow()` and `repay()`.
3.   The owner calls `forbidCreditManagersCanRepay()` on X's address..
4.   The owner calls `addCreditManager()`, the `require()` statement passes since `!creditManagersCanRepay[_creditManager]` is true.
5.   The `creditManagersShareLocker[_creditManager]` entry is overwritten.

**Recommendation:** The `creditManagers` address array keeps track of all credit managers. Run a loop on it and confirm the passed address doesn't exist.
