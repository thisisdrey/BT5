# Insufficient application of trading fee leads to 50% loss for LPs in swapTokenForAnyNFTs()

## Summary
This bug report is about an issue with the protocol in the LSSVMPairERC1155 and LSSVMPairERC721 Solidity contracts. The protocol applies a trading fee of 2*tradeFee on NFT buys from pairs, but LSSVMPairERC1155.swapTokenForAnyNFTs() enforces a trading fee of only tradeFee instead. This would have caused affected LPs of pairs targeted by LSSVMPairERC1155.swapTokenForAnyNFTs() to unexpectedly lose 50% of the trade fees.

The problem has been resolved with the removal of the entire function LSSVMPairERC1155.swapTokenForAnyNFTs() in PR#22. This was acknowledged by Sudorandom Labs and verified by Spearbit.

## Description
The protocol applies a trading fee of `2 * tradeFee` on NFT buys from pairs (to compensate for `0` fees on NFT sells as noted in the comment:

> "// We pull twice the trade fee on buys but don't take trade fee on sells if assetRecipient is set."

While this is enforced in `LSSVMPairERC721.swapTokenForSpecificNFTs()` and `LSSVMPairERC1155.swapTokenForSpecificNFTs()`, `LSSVMPairERC1155.swapTokenForAnyNFTs()` enforces a trading fee of only `tradeFee` (instead of `2 * tradeFee`). 

Affected LPs of pairs targeted by `LSSVMPairERC1155.swapTokenForAnyNFTs()` will unexpectedly lose `50%` of the trade fees.

## Recommendation
This entire function `LSSVMPairERC1155.swapTokenForAnyNFTs()` has been removed in a recent PR#22 (after the start of this review). So there is nothing to fix related to this issue.
