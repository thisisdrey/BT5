# [H05] Wrong value in call can cause money to be incorrectly withdrawn from reserves

## Summary
This bug report is about the code within the `_settleCashBalance` function of `Escrow.sol`. It states that a call to `_convertToETHWithHaircut` is made using the value `settledAmount`, which is declared within the function and has an initial value of `0`. This causes the calculation of the `payer`‘s `freeCollateral` to be incorrect, leading to further liquidation of the `payer`‘s assets and the protocol taking currency out of the protocol’s reserves when it isn’t needed.

The bug report suggests replacing `settledAmount` with `valueToSettle - localCurrencyRequired` on line 772. This has been fixed in pull request #52.

## Details
In [`_settleCashBalance`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L710) within [`Escrow.sol`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol), a call to [`_convertToETHWithHaircut`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L772) is made using the value `settledAmount`. However, `settledAmount` is [declared within the function](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L728) giving it a value of `0` initially. The only place [`settledAmount` is modified](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L735) is within the `if` portion of the branch on [line 730](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L730). However, the call to `_convertToETHWithHaircut` is within the `else` section of the same structure, and since `settledAmount` is not modified within that branch, it will still be `0` when `_convertToETHWithHaircut` is called. This will cause the [calculation of the `payer`‘s `freeCollateral`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L771) to be incorrect, leading to further liquidation of the `payer`‘s assets, and may end up causing the protocol to [take currency out of the protocol’s reserves](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L967) when it isn’t needed.


It appears that the intention was to use the value `valueToSettle - localCurrencyRequired` in [the call to `_convertToETHWithHaircut`](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L772). So, consider replacing `settledAmount` with `valueToSettle - localCurrencyRequired` on [line 772](https://github.com/swapnet-protocol/swapnet-lite/blob/66ce8f3e529410a5272b7a57c8391fcb80a375fd/packages/contracts/contracts/Escrow.sol#L772).


***Update:** Fixed in [pull request #52](https://github.com/swapnet-protocol/swapnet-lite/pull/52).*
