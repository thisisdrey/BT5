# It Is Possible to Cross the `lazyMint` Threshold

## Summary
The client has marked a bug as "Mitigated" and provided an explanation for the fix. The bug affects the `createToken()` function in the `NFT1155` and `NFT721` files. The function does not check if the newly minted token's ID is less than the `LAZY_MINT_TOKENID_START` value, which can result in the token crossing the threshold and causing issues. The recommendation is to add a check to ensure the `newItemId` is allowed.

## Details
**Update**
Marked as "Mitigated" by the client. Addressed in: `7ff94e4acd3947352e8a8404adfbeff36c5f946c`. The client provided the following explanation:

> Used uin256 / 2 for LAZY_MINT_TOKENID_START to reduce the chances of crossing the lazyMint threshold.

**File(s) affected:**`NFT1155`, `NFT721`

**Description:** The function `createToken()` does not check if the current `newItemId` is less than the `LAZY_MINT_TOKENID_START`, this can make newly minted tokens to cross this threshold and start consuming lazy token ids.

**Recommendation:** Add a check to see if the `newItemId` is allowed.
