# Inability to Modify Admin Account

## Summary
GodMode Admin Account and Lamport Transfers Bug

The GodMode Admin Account is not working properly when handling lamport transfers. The issue is that the transfer CPI call in `add_ephemeral_multiplier` tries to deduct lamports from the admin account, but the admin account is currently read-only. This prevents lamport deductions in `add_ephemeral_multiplier` and blocks modifications in `reclaim_rent`. The fix for this bug is to mark the admin account as `mut` and it has been resolved in version `0b87d6e`.

## Details
## GodMode Admin Account and Lamport Transfers

In GodMode, the admin account handles lamport transfers when resizing the staking account. However, the transfer CPI call in `add_ephemeral_multiplier` attempts to deduct lamports from the admin account, while `reclaim_rent` (shown below) transfers excess lamports back to it. Since the admin account is currently read-only, these operations will fail—preventing lamport deductions in `add_ephemeral_multiplier` and blocking modifications in `reclaim_rent`.
