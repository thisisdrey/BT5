# [M-12] Admin rug vectors

## Summary
This bug report is about a vulnerability in the code of the Rubicon protocol. The vulnerability allows the admin to call certain functions that could brick parts of the system, leading to user funds being locked. This could have a negative impact on the protocol’s reputation.

The vulnerability is demonstrated with three examples. In the first example, the admin can overwrite state by calling the function “adminWriteBathToken”. In the second example, the admin can set a new malicious market to steal new funds. In the third example, the admin can add a ton of bonus tokens, which could cause withdrawals to break.

The tools used to identify the vulnerability were code inspection. The recommended mitigation steps are to validate input arguments and require specific upgrade paths for each contract, and to add a limit to the number of bonus tokens.

### Impact

There are multiple functions that the admin can call to brick various parts of the system, leading to user's funds being locked

Even if the owner is benevolent the fact that there is a rug vector available may [negatively impact the protocol's reputation](https://twitter.com/RugDocIO/status/1411732108029181960). See [this](https://github.com/code-423n4/2021-08-realitycards-findings/issues/73) example where a similar finding has been flagged as a high-severity issue. I've downgraded these instances to be a medium since it requires cooperation of the admin.

### Recommended Mitigation Steps

Validate input arguments and require specific upgrade paths for each contract, not just allowing the admin to set whatever they decide, add limit to number of bonus tokens.
