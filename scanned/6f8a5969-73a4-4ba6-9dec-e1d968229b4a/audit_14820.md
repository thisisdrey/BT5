# TRST-M-7 It’s possible to freely detach tokens from a gauge

## Summary
This bug report is about the `withdrawToken()` function. This function takes two inputs, an amount and a tokenId, and detaches the token from the gauge it is associated with and withdraws the amount. It was meant to be called by `withdraw()` internally, which only detaches the token if the full amount deposited is being withdrawn. The bug was that a user could pass 0 as an amount with the tokenId, which would withdraw nothing and detach the token. Additionally, the function was not emitting the Withdraw event.

The recommended mitigation for this bug was to make `withdrawToken()` internal instead of public. This was implemented and the issue was resolved. Now, `withdrawToken()` is declared as an internal function.

## Details
**Description:**

nothing and detach the token. The function was meant to be called by `withdraw()` internally
which only detaches the token if the full amount deposited is being withdrawn.
In addition to this, `withdrawToken()` doesn’t emit the Withdraw event.

**Recommended mitigation:**
