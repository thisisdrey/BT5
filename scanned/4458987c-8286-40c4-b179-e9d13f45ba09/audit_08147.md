# Unprotected event emission functions 

## Summary
This bug report is about a contract called TokenVault.sol that has two functions which emit DepositClaimable and RedeemClaimable events. These events can be easily manipulated by anyone, leading to fraudulent events being emitted or causing issues with the off-chain infrastructure. The recommendation is to remove these functions as they serve no clear purpose and pose a low risk.

## Details
## TokenVault.sol Review

## Context
**File:** TokenVault.sol  
**Line:** L300 - L300

## Description
In the contract, we can see two functions that just emit `DepositClaimable` / `RedeemClaimable` events with parameters received from the call. By doing this, anyone can just emit fraudulent events or can DoS the off-chain infrastructure by emitting a very large number of events or mislead anyone reading the events.

## Recommendation
As we do not see why these functions are needed, consider removing them.
