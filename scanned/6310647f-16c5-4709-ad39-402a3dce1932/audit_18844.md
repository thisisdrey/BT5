# UniProxy - SafeERC20 is declared but safe functions are not used ✓ Fixed

## Summary
This bug report concerns the UniProxy contract, which was declared to use the SafeERC20 library for functions of the IERC20 type. However, instead of using the safe functions, the contract was using unsafe functions. Examples of this included the usage of the 'approve' function instead of the 'safeApprove' function, and the usage of the 'transferFrom' function instead of the 'safeTransferFrom' function. The bug was fixed in GammaStrategies/hypervisor/commit/9a7a3dd88e8e8b106bf5d0e4c56e879442a72181 by implementing the auditor's recommendation.

## Details
#### Resolution



fixed in [GammaStrategies/[email protected]`9a7a3dd`](https://github.com/GammaStrategies/hypervisor/commit/9a7a3dd88e8e8b106bf5d0e4c56e879442a72181) by implementing the auditor’s recommendation.


#### Description


The `UniProxy` contract declares the usage of the `SafeERC20` library for functions of the `IERC20` type. However, unsafe functions are used instead of safe ones.


#### Examples


* Usage of `approve` instead of `safeApprove`
* Usage of `transferFrom` instead of `safeTransferFrom`.
