# Distribution closing shouldn't be allowed during the active distribution

## Summary
This bug report discusses an issue with the FantiumClaimingV1 smart contract. Currently, there are no checks in place to ensure that the distribution ends in the closing function. To address this issue, the report recommends adding a check that the distribution can be closed only when the block timestamp is greater than the closeTime. This would ensure that the distribution ends properly and prevent any potential issues.

## Details
##### Description
There are no checks that the distribution ended in the closing function.
https://github.com/FantiumAG/smart-contracts/blob/a2d126453c1105028f12277b8f342d2cdbf01a77/contracts/claiming/FantiumClaimingV1.sol#L624-L631

##### Recommendation
We recommend adding a check that the distribution can be closed only when `block.timestamp > closeTime`.
