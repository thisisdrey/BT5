# Unsafe Contract Initialization

## Summary
This bug report discusses an issue with insufficient validation in three contracts: WormholeBridge.sol, LayerZeroBridge.sol, and USX.sol. This could potentially lead to a denial of service attack and also pose a risk to cross-chain functionality. The suggested solution is to validate the parameters of the initialize function and restrict access to specific callers. The specific code that needs to be changed is also mentioned. This issue has been resolved in the latest patch.

## Details
## Initialization Vulnerabilities in Smart Contracts

The `initialize` functions in `WormholeBridge.sol`, `LayerZeroBridge.sol`, and `USX.sol` do not perform sufficient validation of the arguments provided. This could result in a denial of service attack if the contract initialization can be front-run. Moreover, depending on the order of initialization, this may also pose a significant risk to cross-chain functionality.

## Remediation

Validate the parameters of the `initialize` function and restrict access to the functions by allowing them to be called only by a specific caller.

### WormholeBridge.sol

```solidity
function initialize(address _wormholeCoreBridge, address _usx) public initializer {
    /// @dev No constructor, so initialize Ownable explicitly.
    require(msg.sender == address(0xdeadbeef), "Invalid caller");
    // replace the address
    require(_wormholeCoreBridge != address(0) && _usx != address(0), "Invalid Parameter");
    __Ownable_init();
    wormholeCoreBridge = IWormhole(_wormholeCoreBridge);
    usx = _usx;
}
```

### LayerZeroBridge.sol

```solidity
function initialize(address _lzEndpoint, address _usx) public initializer {
    /// @dev No constructor, so initialize Ownable explicitly.
    require(msg.sender == address(0xdeadbeef), "Invalid caller");
    // replace the address
    require(_lzEndpoint != address(0) && _usx != address(0), "Invalid Parameter");
    __Ownable_init();
    __NonBlockingLzApp_init_unchained(_lzEndpoint);
    usx = _usx;
}
```

### USX.sol

```solidity
function initialize() public initializer {
    /// @dev No constructor, so initialize Ownable explicitly.
    require(msg.sender == address(0xdeadbeef), "Invalid caller");
    // replace the address
    __Ownable_init();
    __ERC20_init("USX", "USX");
}
```

## Patch

Resolved in commit `777ba1d`.
