# Active LP may not be able to provide more liquidity 

## Summary
The report is about a bug in the code for a lottery system. The specific function, lpDeposit(), is used by LP (a type of user) to deposit initial principal, add more principal, or change their risk percentage. However, there is a requirement check that prevents an active LP from depositing more principal if the number of active LPs has already reached a certain limit. This check should only be applied if the LP's action would actually increase the number of active LPs. This bug has been fixed in a recent update.

## Details
## Context
**File:** Lottery.sol  
**Line:** 572

## Description
The `lpDeposit()` function is called by liquidity providers (LPs) to deposit initial principal, add more principal, or adjust their risk percentage. However, due to the highlighted `require` check, an active LP won't be able to deposit more principal if the number of active LPs reaches `lpLimit`. This action by the LP doesn’t increase the number of active LPs, hence should be allowed.

## Recommendation
The check `activeLpAddresses.length < lpLimit` should only be applied if the action increases the number of active LPs.
