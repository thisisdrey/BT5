# [M-03] It is possible to “uninitialize” ERC20Facet contract

## Summary
This bug report is about the ERC20Facet contract, which is part of the Amun project. It is possible to reset the name and symbol of the contract to empty strings, which would allow anyone to take control of the contract and mint any number of tokens. This should not be possible as the contract should only be initialized once. A suggested mitigation step is to add empty string checks in the setName and setSymbol functions. This bug report has been confirmed by loki-sama (Amun).

<p>The initialization status is defined by the <a href="https://github.com/code-423n4/2021-12-amun/blob/main/contracts/basket/contracts/facets/ERC20/ERC20Facet.sol#L25-L28">name and symbol</a>. It is possible it set them back to an empty string, uninitializing the contract and letting the <code>initialize(..)</code> function be called again. This way, the owner may, for example, hide minting additional tokens. Or, after accidentally setting name and symbol to empty strings, anyone can take control over the contract and mint any number of tokens.</p>
<p>In general, it shouldn’t be possible to initialize more than once.</p>
